# ShieldCortex — 为AI代理提供持久化存储与安全保护

让您的AI代理拥有在会话之间保持状态的“大脑”，并保护其免受内存注入攻击的威胁。

## 安全性与使用范围

此技能**不会**自动安装任何内容。以下所有安装命令都需要用户手动执行：

- **安装过程中不会写入任何文件**：`clawhub install shieldcortex` 仅会将此 SKILL.md 文件复制到您的技能目录中，不会修改任何二进制文件、钩子或配置文件。
- **可选的 npm 安装**：`npm install -g shieldcortex` 会全局安装 CLI 和 MCP 服务器。仅当用户明确要求时才进行安装。
- **仅修改本地配置**：`shieldcortex install` 会将 MCP 服务器的配置写入代理的本地配置文件（例如 `~/.config/claude-code/settings.json`、`~/.codex/config.toml` 或 `~/.openclaw/settings.json`）。它不会修改系统文件、环境变量或其他代理的配置。
- **未经许可不会进行网络请求**：核心包完全基于本地运行（使用 SQLite 数据库和本地嵌入数据）。云同步和 API 密钥提交是可选的，并且默认处于关闭状态。
- **文件扫描是针对指定路径的**：`scan-skill` 仅扫描用户指定的文件，不会遍历目录或读取超出指定范围的文件。
- **无需提供任何凭证**：基础包不需要 API 密钥、令牌或账户信息。启用云功能需要执行 `shieldcortex config set-api-key` 命令。

## 产品描述

ShieldCortex 是一个具有内置安全机制的完整内存管理系统。它为 AI 代理提供持久化、智能化的存储功能，支持语义搜索、知识图谱、基于衰减机制的遗忘机制以及矛盾检测。所有内存写入操作都会经过六层防御机制的检测，有效阻止提示注入、凭证泄露和恶意攻击。此外，Iron Dome 功能还提供了行为保护（如动作门控、安全配置文件和完整的审计记录）。

**适用场景：**
- 当您希望代理在会话之间保留信息（如决策、偏好设置、架构信息、上下文数据）时。
- 当您需要对过去的内存内容进行语义搜索（而不仅仅是简单的关键词匹配）时。
- 当您需要自动整合、清理和优化内存数据时。
- 当您希望从内存中提取实体和关系数据以构建知识图谱时。
- 当您需要保护内存免受提示注入或恶意攻击时。
- 当您希望检测内存写入操作中的凭证泄露行为（支持 25 种以上检测模式，涵盖 11 种常见的攻击方式）时。
- 当您希望审计内存中存储和检索的内容时。
- 当您希望扫描代理的指令文件（如 SKILL.md、.cursorrules、CLAUDE.md）以发现隐藏威胁时。
- 当您希望使用 Iron Dome 功能（动作门控、安全配置文件）来增强安全性时。
- 当您希望使用通用内存防护机制（Universal Memory Bridge）来保护任何内存后端时。

**不适用场景：**
- 当您只需要简单的键值存储功能时（可以使用配置文件）。
- 当您只需要临时性的会话级上下文数据时（可以使用代理内置的上下文管理功能）。
- 当您需要用于 RAG（Retrieval-Augmentation-Generation）流程的向量数据库时（ShieldCortex 主要用于代理的内存管理，而非文档检索）。

## 内存系统特性

| 特性 | 描述 |
|---------|-------------|
| **持久化存储** | 基于 SQLite 的存储系统，可跨重启和数据压缩操作保持数据完整 |
| **语义搜索** | 通过内容含义而非关键词进行记忆检索 |
| **项目级管理** | 可按项目或工作区隔离记忆数据 |
| **重要性分级** | 内存数据分为关键、高级、普通和低级三种级别，并支持自动衰减机制 |
| **分类管理** | 支持对不同类型的记忆数据进行分类（如架构信息、决策记录、偏好设置、上下文数据、学习结果、错误记录等） |
| **衰减与遗忘** | 旧数据或未被访问的数据会逐渐被删除，模拟真实大脑的记忆机制 |
| **数据整合** | 自动合并相似或重复的记忆内容 |
| **矛盾检测** | 当新记忆与现有记忆冲突时发出警报 |
| **知识图谱** | 从记忆数据中提取实体和关系信息 |
| **访问优先级** | 最近访问的记忆内容在搜索中优先显示 |

## 安全特性

| 防护层次 | 保护措施 |
|---------|-----------|
| **输入清洗** | 去除控制字符、空字节和危险格式的输入数据 |
| **模式检测** | 通过正则表达式检测常见的注入模式 |
| **语义分析** | 将记忆数据与攻击代码库进行相似性对比 |
| **结构验证** | 检查数据的 JSON 格式是否正确 |
| **行为监控** | 长期监控系统行为以检测异常行为 |
| **凭证保护** | 阻止 API 密钥、令牌和私钥的泄露（支持 25 种以上检测模式，涵盖 11 种常见的攻击方式） |
| **信任评估** | 根据数据来源对内存写入操作进行可靠性评估 |
| **审计记录** | 详细记录所有内存操作的审计日志 |
| **安全扫描** | 检查 SKILL.md、.cursorrules、CLAUDE.md 文件中的潜在安全威胁 |

#### Iron Dome

Iron Dome 是一种行为安全层，用于控制代理的可执行操作：
- **安全配置文件**：提供 `school`、`enterprise`、`personal`、`paranoid` 四种安全级别，每种级别都有相应的动作门控和信任设置 |
- **动作门控**：限制某些危险操作（如发送邮件、删除文件、调用 API）的执行，需经过授权 |
- **注入检测**：扫描文本以检测潜在的注入代码 |
- **完整审计记录**：所有操作都会被记录下来，便于后续审计分析

#### 通用内存防护机制（Universal Memory Bridge）

该机制可以保护任何内存后端，不仅限于 ShieldCortex 自带的存储系统。它通过六层防御机制来增强系统的安全性。使用该机制无需任何凭证或 API 密钥，所有处理过程都在本地完成。

### ShieldCortex 云服务（可选）

您可以将审计数据同步到团队仪表板，实现跨项目的信息共享。云服务是可选的，默认情况下是关闭的。除非您使用自己的 API 密钥进行配置，否则数据不会离开您的设备。

### 注意事项

免费的本地版本使用不受限制。云服务提供团队仪表板、审计数据汇总和警报功能。无论使用本地功能还是云服务，都不需要任何云凭证。整个内存系统、防御机制、知识图谱和 Iron Dome 功能都无需 API 密钥或账户信息。
