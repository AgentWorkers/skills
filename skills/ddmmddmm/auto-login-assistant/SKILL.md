# 自动登录助手

## 概述

当代理在浏览网站时遇到登录屏幕、会话过期或验证码挑战等阻碍时，可以使用此技能。该技能提供了一套保守的登录辅助工作流程：检测登录障碍、收集用户提供的凭据、填写表单，并在用户明确同意的情况下处理一次性验证码。

请注意，此技能并非用于绕过任何安全机制（如验证码、反机器人系统），也不允许获取用户未明确提供的敏感信息。

## 使用场景

在以下情况下触发此技能：

- 页面跳转到登录、签到或会话过期的界面。
- 用户要求代理登录网站、邮箱、SaaS产品或管理控制台。
- 阅读邮件、发送邮件、查看仪表板或访问设置等操作因身份验证而受阻。
- 用户希望在完成操作后协助获取或输入验证码。

**禁止使用此技能的场景**：

- 解码验证码、绕过二维码登录、使用硬件密钥、进行支付授权或生物特征验证。
- 猜测用户名、密码、安全问题或备用密码。
- 未经用户明确授权，不得读取邮件或消息内容。

## 工作流程

### 1. 确认是否为登录障碍

首先验证页面是否确实需要身份验证。注意以下提示：

- URL 中包含 `login`、`signin`、`auth`、`session-expired`、`verify` 等关键字。
- 存在密码输入框、一次性密码（OTP）输入框或邮箱/用户名输入框。
- 显示“登录”、“继续使用邮箱”、“输入验证码”等按钮或标题。

如果页面信息不明确，请告知用户，并在输入任何凭据之前询问是否应将其视为登录流程。

### 2. 选择凭据来源

凭据的优先级顺序如下：

1. 用户在当前对话中提供的凭据。
2. 用户明确指定的本地文件路径。
3. 用户明确指定的环境变量。

切勿随意扫描文件系统以获取敏感信息。未经用户指示，切勿假设存在保存的凭据。

如果用户提供了文件路径或环境变量名称，请使用 `scripts/read_credentials.py` 将其转换为统一格式。

**支持的标准化字段**：

- `site`（网站地址）
- `login_url`（登录URL）
- `username`（用户名）
- `email`（邮箱地址）
- `phone`（电话号码）
- `otp_email`（用于接收一次性密码的邮箱地址）
- `otp_mode`（验证码生成方式）
- `notes`（备注）

具体示例请参考 `references/config-example.md`。

### 3. 保守地填写登录表单

遵循网站提供的登录流程，避免直接发送请求。

**推荐的字段映射顺序**：

- 用户标识：`email`（邮箱地址）、`username`（用户名）、`account`（账户名）、`phone`（电话号码）
- 密码：`password`（密码）
- 验证码：`otp`（一次性密码）、`code`（验证码）、`verification code`（验证码）

在提交前，请确认用户是否有多个账户可供选择，并在解释过程中对敏感信息进行屏蔽。除非用户明确要求，否则不要勾选“记住此设备”等相关选项。

### 4. 处理验证码

**默认行为**：要求用户手动输入验证码。

**仅在用户明确授权且提供了邮箱访问权限的情况下**，才执行读取邮箱内容的操作。具体步骤如下：

- 仅读取获取最新验证码所需的最少邮箱内容。
- 使用 `scripts/extract_verification_code.py` 提取可能的验证码。
- 如果验证码的准确性较低，先向用户确认后再显示。
- 如果存在多个可能的验证码，请在提交前询问用户。

如果读取邮箱内容失败或无法访问邮箱，则要求用户手动粘贴验证码。

### 5. 验证登录是否成功

提交后，通过以下方式确认登录是否成功：

- 用户头像、账户菜单、收件箱、仪表板或“退出”按钮的显示。
- 登录提示是否消失。
- 是否成功导航到目标功能页面。

如果尝试多次仍失败，请清楚地说明失败原因，并停止尝试。

## 安全规则

- 除非用户明确要求，否则将凭据视为临时性数据，不得长期保存。
- 不得将凭据存储在技能文件夹中。
- 未经用户许可，不得更改多因素认证（MFA）设置或信任设备的相关设置。
- 严禁用于绕过身份验证或反滥用保护机制。
- 如果网站要求输入验证码、扫描二维码、使用物理令牌或密码片，请将控制权交还给用户。

## 建议的交互方式

使用简洁直接的提示语，例如：

- “此页面需要登录。您是想使用您提供的凭据登录，还是使用您指定的本地配置？”
- “我发现了一个密码输入框和邮箱输入框。请提供该网站的账户信息，或告诉我配置文件的路径。”
- “网站需要验证码。如果您同意，可以在这里粘贴验证码。但只有在您明确授权的情况下，我才能读取该邮箱的内容。”
