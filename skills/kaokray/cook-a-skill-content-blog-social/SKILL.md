# 自动内容博客生成流程

**端到端工作流程：**  
热门话题 → 关键词研究 → 内容大纲 → 用户生成内容（UGC）丰富 → 人工撰写初稿 → 视觉元素与链接优化 → SEO与地理位置（GEO）优化 → 质量评估报告。

## 工作流程  

```
[0]   X TREND MONITOR   → auto-run on session start, propose hot keywords
[1]   KEYWORD RESEARCH  → scored, data-backed selection
[2]   CONTENT BRIEF     → outline + keyword map + UGC insertion points
[3]   UGC ENRICHMENT    → mine comments → ready-to-paste blog sections
[4]   DRAFT             → human-style, data-backed, zero AI patterns
[4.5] VISUAL & LINKS    → data charts (matplotlib) + inline hyperlinks on every citation
[5]   SEO + GEO         → on-page checklist + AI-bot-friendly formatting
[6]   QA REPORT         → rubric score + publish verdict
```  

**可以在任何阶段开始执行流程。**  
**对于已有初稿，可以直接从第5阶段开始。**  

**常用命令：**  
- `run full pipeline` → 执行全部流程  
- `start from stage [N]` → 从指定阶段开始  
- `score my draft` → 评估我的初稿（仅限第5阶段）  
- `enrich my draft` → 丰富我的初稿（仅限第4.5阶段）  

**参考文件**（需详细信息时查阅）：  
- `references/scoring-rubrics.md` — 所有评分标准  
- `references/ai-patterns-blacklist.md` — 需避免使用的AI相关表达式列表  
- `references/output-templates.md` — 各阶段的输出格式模板  
- `references/example-run.md` — 实例：关键词“什么是液态质押（What is liquid staking）”

## 输入要求  
**必填：** `project_spec`（.md格式文件）+ `keyword_or_topic`  
**可选：** `ugc_urls`（如未提供可跳过第3阶段）、`blog_draft`、`language`（默认：英语）、`word_count`（默认：1500–2000字）、`tone`（根据文档自动确定）  

**如果缺少`project_spec`文件，请在开始前索取。**  

---

## 第0阶段 — X趋势监控  
**在新会话开始时自动执行，用户输入任何内容之前。**  
1. 解析`project_spec.md`文件，提取：域名/领域、目标受众、产品主题、竞争对手名称  
2. 进行网络搜索（使用文档中的信息）：  
   - `site:x.com [domain keyword] -filter:replies`  
   - `[domain keyword] 2025年Twitter热门话题`  
   - `[domain keyword] 相关讨论或辩论`  
   - `[competitor name] 2025年Twitter情感分析`  
   - `crypto twitter 2025年热门话题`  
   - `[domain keyword] 加密货币Twitter讨论`  
3. 根据`references/scoring-rubrics.md`中的评分标准对每个话题进行评分  
4. 保留排名前三的热门话题，并在用户输入任何内容前显示“趋势警报”（参考`references/output-templates.md`中的第0阶段模板）  
**如果网络搜索失败，**则跳过此阶段，并提示用户“第0阶段不可用，请提供关键词以继续第1阶段”。  
**如果未找到热门话题，**则从文档中列出3个长期流行的主题。  

---

## 第1阶段 — 关键词研究  
1. 生成8–12个关键词：包括核心关键词、长尾关键词及其变体  
2. 对每个关键词执行以下操作：  
   - `google trends [keyword] 2025` → 了解搜索趋势  
   - `site:x.com [keyword]` + `[keyword] 2025年Twitter热门话题` → 了解讨论热度  
   - `[keyword] 搜索量统计` + `ubersuggest [keyword]` → 获取搜索量估算  
   - 搜索`[keyword]` → 查看相关问题  
3. 根据`references/scoring-rubrics.md`中的评分标准对关键词进行评分  
4. 选择排名第一的关键词作为主要关键词，其余5–8个作为次要关键词/长尾关键词  
5. 询问用户是否拥有Ahrefs或SEMRush API密钥（如有提供，请使用）  
6. 输出结果（参考`references/output-templates.md`中的第1阶段模板）  

**特殊情况：**  
- 关键词过于宽泛 → 建议3–5个长尾关键词替代词；  
- 关键词与文档要求不符 → 提醒用户并确认。  

---

## 第2阶段 — 内容大纲与结构设计  
1. 设计内容结构：  
   - 问题 → 解决方案 | 什么是 → 为什么 → 如何 → 对比 | 用户体验 | 论点  
2. 将每个长尾关键词对应到H2或FAQ条目中  
3. 标记需要插入用户生成内容（UGC）的位置：  
   - `← UGC: 社区观点` / `← UGC: 案例研究` / `← UGC: 常见问题`  
4. 输出内容大纲（参考`references/output-templates.md`中的第2阶段模板）  
5. 询问用户“这个大纲是否合适”，等待确认后再进入第3阶段。  
**FAQ部分**：至少包含5个条目（地理位置（GEO）相关内容。  

---

## 第3阶段 — 用户生成内容（UGC）丰富  
**如果未提供`ugc_urls`，则跳过此阶段。**  
**说明：** Reddit上的用户生成内容占Google AI引用内容的约21%；真实用户生成的内容能使搜索结果可见度提高2–3倍。  
**对于每个URL：**  
1. 使用`web_fetch`工具提取帖子正文及所有评论（包含互动数据）  
2. 根据`references/scoring-rubrics.md`中的评分标准对评论进行质量评估  
3. 将评论分为三类：  
   - **A类：反驳观点**：针对原文提出质疑或提供反例（150–300字）  
   - **B类：真实用户问题**：来自评论的直接问题（尤其是被点赞或重复出现的问题，形成3–6对问答）  
   - **C类：个人经验**：包含具体数字或结果的文字描述（2–4个案例研究，每条50–100字，需重新表述但不得伪造）  
4. 对评分高于7分的评论进行分析，找出3种常见写作技巧  
5. 将用户生成的内容对应到第2阶段设计的大纲中  
6. 输出结果（参考`references/output-templates.md`中的第3阶段模板）  

**重要规则：**  
- 禁止伪造内容；禁止使用用户名或垃圾信息；每个FAQ条目不超过6条；禁止使用垃圾信息或表情包。  

**特殊情况：**  
- 如果评论数量少于10条，提示“数据不足”；  
- 如果所有评论的评分均低于3分，提示“无高质量评论，请尝试其他URL”。  

---

## 第4阶段 — 初稿撰写  
**严格按照第2阶段的设计编写完整文章，**并在指定位置插入用户生成的内容。  
**撰写前：**设置元标题（不超过60个字符，包含主要关键词）和元描述（不超过160个字符，包含主要关键词）。  
**引言部分**：使用具体事实或引人注目的开头语句；主要关键词需出现在前100字内；撰写特色段落（40–60字，包含定义或步骤说明）。  
**正文部分**：每个H2标题下先给出直接答案（1–2句话），再详细阐述；每个主要部分至少包含1个数据支持；优先查找真实统计数据（例如：`[topic] 2025年统计数据`）。如果数据无法获取，可使用`[DATA NEEDED: search "[query]"`。  
**数据要求：**每篇文章至少包含3个数据点；禁止捏造数据。  
**段落长度**：每段不超过3–4行；首次出现技术术语时需进行解释。  
**避免使用`references/ai-patterns-blacklist.md`中的表达式**；若被发现错误，需重新撰写。  
**结论部分**：总结2–3个要点；不要将所有H2内容以列表形式重复。  
**内部链接**：建议添加2–3个链接，链接文本使用次要关键词。  
**输出格式**：参考`references/output-templates.md`中的第4阶段模板。  

---

## 第4.5阶段 — 视觉元素与链接优化  
**在第4阶段完成后自动执行。**  
**此阶段不可跳过，**因为需要生成适合发布的最终版本。  

**为什么需要这个阶段：**  
- 数据图表被AI搜索引擎引用的可能性是纯文本的3倍；  
- 每个数据来源的链接能提升内容的可信度；  
- 未添加链接的引用会降低内容的可信度。  

**步骤1：**  
**查找适合制作图表的数据点**：  
   - 数据需满足以下条件之一：  
     - 随时间变化的数据（如“TVL在6个月内从20亿美元增长到80亿美元”）  
     - 比较两种资产或指标的数据  
     - 流量趋势（如“2025年第一季度ETF流入量达12亿美元”）  
     - 指数或分数的变化  
     - 比例变化（如“ETH质押比例在12个月内从18%上升到27%”）  
     - 多个数据点的系列数据  

**图表类型：**  
   - 根据数据类型选择合适的图表类型（例如：折线图、柱状图、颜色编码的条形图等）  
**图表设计规范：**  
   - 背景颜色：`#F7F7F7`（浅灰色）；无粗网格线  
   - 主要颜色：`#E8650A`（橙色，适用于加密货币/Web3领域；可根据文档要求调整）  
   - 去除图表顶部和右侧的边框  
   - 直接在图表上标注关键数据点  
   - 文字说明：`来源：[名称] | 备注`  
   - 字体：DejaVu Sans（matplotlib默认字体）  
   - 分辨率：至少150 DPI  
   - 为每个图表生成alt文本  

**每篇文章最多使用6个图表；**超过6个图表会降低页面加载速度。  

**如果未找到符合条件的数据点，**则用文字说明替代图表。  

**步骤2：**  
**使用Python和matplotlib生成图表。**  
   - 为每个图表生成PNG图片并嵌入到输出文件中；  
   - 添加图例（1–2句话）：`图表[编号]：展示了[内容]。时间范围：[起始时间]至[结束时间]。数据来源：[名称]`  
   - 为CMS生成alt文本：`[图表类型]显示了[指标]，时间范围为[起始时间]至[结束时间]。主要发现：[关键内容]`  

**步骤3：**  
**将所有引用转换为链接。**  
   - 检查文档中的引用格式；  
   - 根据格式替换为链接；  
   - 如果无法找到原始文章，链接到出版物首页；  
   - 如果引用信息不明确（如“分析师称……”），提示用户提供来源。  

**链接规则：**  
- 优先使用原始出版物的链接；  
- 如果有免费版本，避免使用付费链接；  
- 外部链接需在新标签页中打开；  
- 避免引用未经证实的信息。  

## 第5阶段 — SEO与地理位置（GEO）优化  
**SEO检查清单：**  
- 元标题：50–60个字符，包含主要关键词  
- 元描述：150–160个字符，包含主要关键词  
- H1标题包含主要关键词  
- H2标题使用次要关键词或相关变体  
- 主要关键词出现在前100字及结论部分  
- 关键词密度：1–1.5%  
- 每段不超过4行  
- 提供2个以上内部链接  
- 引用2个以上权威来源  
- 图片alt文本建议（至少2条）  
- 必须包含FAQ部分  

**GEO优化规则：**  
- 每个H2标题下先给出直接答案；  
- 所有数据需明确标注单位（如“3.2%年化收益率”）；  
- 必须包含FAQ部分（至少5个问题）  
- 特色段落：40–60字，放在H2标题内或编号列表中  
- 引用信息需清晰易懂  
- 内部链接使用次要关键词  
- 包含用户生成的内容，以增强用户体验（E-E-A-T原则）  

**评分标准：**  
**使用100分的评分标准进行评估（参考`references/scoring-rubrics.md`中的第5阶段模板）。**  
**评分标准：**85–100分 = 可直接发布；70–84分 = 需少量修改；低于70分 = 需重新修改。  

**输出格式：**参考`references/output-templates.md`中的第6阶段模板。

## 特殊情况处理：**  
| 情况 | 处理方式 |  
|---|---|  
| 缺少`project_spec`文件 | 停止操作，要求用户提供文档。  
| 关键词过于宽泛 | 建议3–5个长尾关键词替代词，等待用户确认。  
| 关键词与文档不符 | 提醒用户并确认。  
| 内容涉及复杂技术内容 | 标记为“需要专家审核”；禁止伪造内容。  
| 用户要求使用越南语 | 将所有内容转换为越南语（保持格式不变）。  
| 字数少于500字或超过5000字 | 提醒用户调整字数（建议800–2500字）。  
| SEO评分低于70分 | 提供具体修改建议及参考内容。  
- 网络搜索失败 | 跳过第0阶段；标记受影响的阶段。  
- 用户生成的内容无法获取 | 跳过第3阶段；从文档中列出长期流行的主题。  
- 用户在流程中途介入 | 从适当阶段开始；询问用户已有信息。  
- 第4.5阶段缺乏适合制作图表的数据 | 用文字说明替代图表数据。  
- 第4.5阶段有太多图表数据 | 优先处理对比数据、趋势分析和指数分数；其余数据使用文字说明。  
- 第4.5阶段的部分数据来源无法访问 | 链接到出版物首页；标记所有需要验证的链接。  
- 引用信息不明确 | 提示用户提供来源。  
- 网络搜索失败 | 跳过链接验证步骤；使用文档中的数据生成图表。  

## 限制事项：**  
- 第0阶段：通过网络搜索模拟趋势数据（非实时API）  
- 关键词搜索量需用户提供API密钥才能准确统计  
- 无法自动发布内容或生成图片  
- 第4.5阶段使用matplotlib从文档数据生成图表（不生成装饰性或编辑性图片）  
- 需人工审核`[VERIFY]`、`[DATA NEEDED]`、`[SOURCE NEEDED]`等标记  
- SEO/GEO评分基于内部评分标准（非第三方工具）  
- 用户生成的内容仅限于公开可访问的页面。
