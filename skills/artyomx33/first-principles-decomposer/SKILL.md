# 第一性原理分解器（First Principles Decomposer）

## 适用场景
- 设计新产品或新功能时  
- 遇到复杂问题时感到无从下手  
- 现有的解决方案显得过于复杂时  
- 需要质疑某些假设时  
- 开始任何新项目或新计划时  

## 使用流程  

### 第一阶段：识别假设  
问自己：“我有哪些假设可能是错误的？”  
列出当前方法中隐含的所有假设。  

### 第二阶段：分解到最基本的事实  
对于每个假设，继续追问：“这里最基本的事实是什么？”  
不断追问“为什么？”，直到找到不可动摇的事实。  

### 第三阶段：基于事实重新构建  
仅从经过验证的基本事实出发，问：“满足核心需求的最简单解决方案是什么？”  

## 交互流程  
当用户使用此技能时：  
1. **明确问题**（最多1-2个问题）  
2. **揭示假设**——列出所有被默认为正确的因素  
3. **分解到基本事实**——展示所有最基本的前提  
4. **重新构建解决方案**——从零开始构建  
5. **进行比较**——展示与传统方法的区别  

## 输出格式  
```
PROBLEM: [stated problem]

ASSUMPTIONS IDENTIFIED:
1. [assumption] → Challenge: [why this might be wrong]
2. [assumption] → Challenge: [why this might be wrong]

FUNDAMENTAL TRUTHS:
• [bedrock fact 1]
• [bedrock fact 2]
• [bedrock fact 3]

REBUILT SOLUTION:
[New approach built only from fundamentals]

VS CONVENTIONAL:
[How this differs from the obvious approach]
```  

## 示例触发语句：  
- “从第一性原理的角度来分析我们的沟通问题”  
- “我想从头开始重新思考如何实现[X]”  
- “我们对[问题]有哪些错误的假设？”  

## 相关技能  
- **逆向策略师（Inversion Strategist）**：在基于基本事实重新构建后，反向思考可能导致新方法失败的因素  
- **二阶后果分析（Second-Order Consequences）**：评估实施新解决方案可能带来的后续影响  
- **事前分析专家（Pre-Mortem Analyst）**：通过模拟失败来测试新解决方案的可靠性  
- **六顶思考帽（Six Thinking Hats）**：运用六种思维模式来验证每一个基本事实
