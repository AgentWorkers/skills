# Cisco ASA 日志分析

本技能将教你如何阅读、解析和解释由 Cisco ASA 防火墙生成的日志消息。所有 ASA 日志都是通过 syslog 协议传输的，因此在处理与 ASA 相关的内容之前，理解 syslog 协议是基础。

---

## 第 1 部分：Syslog 协议基础

所有 Cisco ASA 日志消息都使用 syslog 标准（RFC 3164 / RFC 5424）进行传输和格式化。在解释 ASA 消息之前，你必须先了解 syslog 协议。

### 严重性级别

每个 syslog 消息都带有数字严重性级别（0 = 最严重，7 = 最不严重）：

| 严重性级别 | 关键字       | 含义                                  |
|---------|---------------|------------------------------------------|
| 0       | 紧急         | 系统无法使用                               |
| 1       | 警告         | 需要立即采取行动                          |
| 2       | 严重         | 严重错误情况                             |
| 3       | 错误         | 错误状态                               |
| 4       | 警告         | 警告性事件                             |
| 5       | 通知         | 正常但重要的事件                         |
| 6       | 信息性       | 常规信息性消息                         |
| 7       | 调试         | 详细的诊断输出                         |

### 设施代码

设施代码表示日志的来源子系统（0-23）。Cisco ASA 默认使用 local4（代码 20），但可以在设备上进行配置。

### PRI 值

PRI 标头同时编码了设施代码和严重性级别：

  PRI = (设施代码 × 8) + 严重性级别

示例：local4（20）+ 信息性（6）→ (20 × 8) + 6 = 166，在日志头中显示为 <166>。这是 ASA 日志中最常见的 PRI 值。

### RFC 3164 格式（传统格式）

  <PRI>时间戳 主机名 标签: 消息内容

示例：
  <166>2023-01-15 10:22:01 fw01 : %ASA-6-302013: 建立了到外部 IP 地址 203.0.113.1/443 的出站 TCP 连接...

### RFC 5424 格式（ASA 9.10+）

通过 "logging timestamp rfc5424" 启用。使用 ISO 8601 UTC 时间戳：

  <PRI>版本 时间戳 主机名 应用程序名称 进程ID 消息ID 结构化数据 消息内容

示例：
  <166>2024-01-15T10:22:01Z fw01 - - - %ASA-6-110002: 无法找到出站接口...

## 第 2 部分：Cisco ASA 日志格式

### 核心消息结构

无论使用哪种 syslog 格式，每个 ASA 事件消息都遵循以下结构：

  %ASA-严重性级别-消息ID: 消息内容

| 字段          | 描述                                                    |
|----------------|----------------------------------------------------------------|
| %ASA           | 所有 ASA 消息中固定的前缀                         |
| 严重性级别       | 与 syslog 严重性级别对应的数字（0-7）                     |
| 消息ID         | 用于唯一标识事件类型的 6 位数字                         |
| 消息内容       | 可变替换内容的、人类可读的描述                         |

### 包含 syslog 标头的完整日志行

  <PRI>时间戳 主机名 : %ASA-严重性级别-消息ID: 消息内容

**RFC 3164 格式示例：**
  <166>2023-01-15 10:22:01 fw01 : %ASA-6-302013: 建立了到外部 IP 地址 203.0.113.1/443 的出站 TCP 连接（目标 IP：203.0.113.1/443），连接到内部 IP 地址 192.168.1.100/54321（192.168.1.100/54321）|

**RFC 5424 格式示例：**
  <166>2024-01-15T10:22:01Z fw01 - - - %ASA-6-302013: 建立了出站 TCP 连接...`

### 消息 ID 编号和分类

6 位的消息 ID 的前 3 位表示功能类别：

| 范围          | 分类                                           |
|--------------|----------------------------------------------------|
| 1xxxxx       | 系统、接口、故障转移、硬件                         |
| 2xxxxx       | VPN 故障转移、高可用性                         |
| 3xxxxx       | 防火墙、连接跟踪、NAT                         |
| 4xxxxx       | 安全性、IPS、ACL、威胁检测                         |
| 5xxxxx       | 用户身份验证                         |
| 6xxxxx       | VPN 加密、IPSec、SSL、远程访问                       |
| 7xxxxx       | 应用程序检查、协议处理                         |
| 8xxxxx       | 管理、平台、集群                         |

### 常见变量占位符

ASA 消息文本中使用描述性的占位符：

| 占位符            | 含义                                        |
|------------------|------------------------------------------------|
| interface_name   | ASA 接口名称（例如：inside、outside）                   |
| src_ip/port      | 源 IP 和端口                                   |
| dst_ip/port      | 目标 IP 和端口                                   |
| protocol         | IP 协议（TCP、UDP、ICMP 等）                         |
| conn_id          | 唯一的连接跟踪编号                             |
| direction        | 进站或出站                                   |
| reason         | 事件原因（例如：SYN 超时、FIN）                         |
| username       | 经过身份验证的用户                         |
| group          | VPN 隧道组或策略名称                             |
| access_list      | 匹配的 ACL 名称                                   |
| action         | 允许或拒绝访问                                 |

### 正则表达式解析

从任何日志行中提取核心的 ASA 字段：

  %ASA-(?P<严重性级别>\d)-(?P<消息ID>\d{6}):\s+(?P<消息内容>.+

**包含 syslog 标头的完整日志行正则表达式：**
  ^(?P<时间戳>\S+\s+\S+\S+\S+|\S+T\S+Z)\s+(?P<主机名>\S+)\s+:\s+%ASA-(?P<严重性级别>\d)-(?P<消息ID>\d{6}):\s+(?P<消息内容>.+)$

## 指南

- 首先提取消息ID——它唯一标识事件类型。
- 立即检查严重性级别：1-3 级表示需要关注的严重/错误情况；6-7 级表示常规事件。
- 使用消息ID 的前 3 位来识别功能类别，然后再查看详细信息。
- ASA 日志会记录接口信息——务必记录事件发生在哪个接口（inside、outside、dmz），因为这提供了重要的网络方向上下文。
- 注意成对出现的消息：ASA 通常会生成一对消息（例如，302013 “建立连接” + 302014 “断开连接”），它们共同描述了连接的完整生命周期。
- 如需了解每个消息ID 的详细定义（消息内容模板、变量含义、推荐操作），请参阅下面的参考文件。
