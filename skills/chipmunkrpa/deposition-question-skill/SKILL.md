# 相对性证据提取问题生成器

## 概述

该工具用于分析一个或多个从 Relativity 系统导出的 PDF 文件，并根据法律理论生成相应的证据提取问题。这些问题会按照文档 ID 进行组织。每个问题都包含提出问题的原因以及可供证人在否认时使用的文档引用内容。

## 必须满足的功能：

- 在生成分析结果或问题之前，首先询问用户所采用的法律理论。
- 如果尚未提供 PDF 文件路径，需要用户提供相应的路径。
- 从每页的右下角提取文档 ID（这些 ID 通常以数字形式显示）；如果该区域显示两个数字 ID，则选择较小的那个作为文档 ID。
- 确保引用的内容保持原样，并注明引用文件的路径和页码。
- 按文档 ID 对问题进行分组，并将每个问题放在对应的文档 ID 标题下。
- 每个问题应包含以下内容：
  - 提出问题的原因
  - 供证人在否认时使用的文档引用内容

## 工作流程：

1. 收集输入信息：
   - 确认用户所采用的法律理论。
   - 确认 PDF 文件的路径。
   （可选）收集证人的姓名/角色以及需要重点关注的主题。

2. 提取每页的文本内容和文档 ID：
   - 运行以下代码块（````bash
     python scripts/extract_relativity_pages.py \
       --input <pdf-folder-or-file> \
       --recurse \
       --output <relativity_pages.json>
     ````）以提取这些信息。
   - 检查那些 `selected_document_id` 为空的页面，并标记需要手动核对的情况。

3. 为所选法律理论构建证据相关性分析：
   - 对每页内容进行分析，判断其与法律理论的关系（支持、削弱或中立）。
   - 先针对支持法律理论的页面起草问题，然后再针对削弱法律理论的页面起草问题。

4. 按文档 ID 分组生成证据提取问题：
   - 问题应简洁、具体，且针对单一事实。
   - 问题应依次涵盖身份验证/基础信息、事实陈述以及矛盾之处。
   - 使用页面文本中的直接引用内容，以便在证人否认时进行反驳。

5. 以规定的格式输出结果：
   - 遵循 `references/deposition_output_template.md` 的格式要求。
   - 确保文档 ID 按升序排列。

## 输出规则：

- 不要将不同的文档 ID 合并到一个部分中。
- 任何问题都不得缺少“提出问题的原因”和“供证人在否认时使用的文档引用内容”这两个部分。
- 如果找不到可靠的引用内容，应标记为：
  - `供证人在否认时使用的文档引用内容：[未找到直接引用内容 - 需要手动核实]`
- 优先选择简短、具体且与某一事实直接相关的引用内容。

## 所需资源：

- 文本提取工具：`scripts/extract_relativity_pages.py`
- 输出模板：`references/deposition_output_template.md`
