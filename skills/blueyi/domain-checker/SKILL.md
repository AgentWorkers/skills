# 域名检查工具

使用 `whois` 命令结合 DNS 验证来检查域名的可用性。这是查询所有域名可用性的唯一可靠来源。

## 快速入门

运行捆绑好的脚本来检查一个或多个域名：

```bash
bash scripts/check_domains.sh example.com myproject.io brand.ai
```

或者通过管道传递域名列表：

```bash
echo "foo.com bar.ai baz.io" | bash scripts/check_domains.sh
```

## 输出格式

每个域名会得到以下三种结果之一：

| 符号 | 含义 | 可信度 |
|--------|---------|-----------|
| ✅ 可用 | whois 查询结果显示“未找到”且没有 DNS 名服务器记录 | 高 |
| ❌ 被占用 | whois 查询显示创建日期或存在 DNS 记录 | 高 |
| ⚠️ 可能被占用 | 数据矛盾（whois 结果不明确但 DNS 记录存在） | 中 |
| ❓ 未知 | whois 查询未返回任何数据——需手动验证 | 低 |

## 工作原理

系统会交叉验证三个独立的信息来源：

1. **whois 查询的创建日期**：这是最权威的判断依据。如果存在创建日期，则说明该域名已被占用。
2. **DNS 名服务器记录**：已注册的域名几乎总是有名称服务器的。
3. **DNS A 记录**：用于判断域名是否处于“停放”状态或是否处于活跃使用中。

只有当 whois 明确显示“未找到”且没有 DNS 记录时，该域名才会被标记为“可用”。这样可以避免因不可靠的 whois 网页接口导致的误判。

## 重要说明

- **速率限制**：脚本在每次查询之间会等待 1 秒，以避免被 whois 服务器限制请求频率。对于大量域名（超过 50 个），建议分多次执行脚本。
- **whois 网页接口的不可靠性**：像 whois.com 这样的网站经常返回过时或不准确的数据。本脚本直接使用 `whois` 命令行工具。
- `.ai` 域名的特殊性：.ai 域名的 whois 服务器有时返回的信息非常有限。脚本会同时检查 DNS 记录以获取更准确的信息。
- **高级域名/售后市场域名**：某个域名在 whois 中可能显示为“可用”，但在注册商处可能以高价出售。脚本无法检测到这种情况——请直接咨询注册商确认实际价格。
- **必备工具**：需要 `whois` 和 `dig` 命令行工具（大多数 Linux/macOS 系统已预装）。

## 批量域名检查流程

在帮助用户根据域名筛选项目名称时，可以按照以下步骤操作：

1. 根据用户要求生成 15-30 个候选域名。
2. 将所有候选域名一次性提交给脚本进行检查。
3. 仅展示结果为“可用”的域名，并提供相应的分析。
4. 如有需要，可继续对可用域名进行筛选。

示例：

```bash
bash scripts/check_domains.sh myapp.com myapp.ai myapp.io coolname.com coolname.ai
```
