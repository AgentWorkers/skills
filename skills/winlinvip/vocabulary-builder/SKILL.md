# 词汇构建工具

该工具可以读取和写入以下文件（路径相对于工作区）：
- **词汇跟踪文件：**`memory/vocabulary.md` — 所有单词数据都存储在这里
- **音频文件目录：**`docs/tts-fr/` — 仅读；用户可以将发音音频文件放入此目录

请确保这些路径存在，或在首次使用前创建它们。

## 阅读流程

当用户想要阅读或练习阅读某本书时：
1. 询问他们正在阅读哪本书
2. 在词汇跟踪文件中查找该书中的单词 — 阅读“活跃单词”（Active Words）部分的末尾（使用`tail`命令或查看最后几条记录）以找到最后一个单词。注意：`memory_search`可能会返回部分或排序后的结果，因此在检查最新条目时也需要直接读取文件。
3. 告诉用户最后一个添加的单词及其所在页码，以便他们知道从哪里继续阅读
4. 询问用户是想要练习尚未掌握的单词，还是继续阅读并添加新单词

## 添加新单词

当用户提供一个新的单词时：
1. 提供单词的**发音**（国际音标，美式英语）
2. 提供单词的**含义** — 简单明了
3. 提供**同义词** — 用户可能知道的相似单词
4. **立即将其添加到跟踪文件中**，除非用户表示不需要添加
5. 添加单词后显示单词卡片

如果用户表示他们专注于**发音**，询问他们容易混淆哪些单词（例如：`seem`、`same`、`sim`）。在专注于发音的模式下：
- 如果用户的目标是比较发音，将这组单词视为一个**单独的比较条目**，而不是三个独立的词汇条目
- 在回复和单词卡片中分别显示每个单词的发音
- 以组合标题的形式记录该条目，例如`### seem / same / sim`
- 使用**含义**字段来解释**如何发音以及需要注意的发音差异**，而不是单词的常规含义
- 保持发音解释简短、实用，并突出重点
- 如果用户没有提供例句，为每个单词生成**非常简单的上下文句子**
- 使用用户提供的书籍、页码或上下文信息

将新单词添加到“活跃单词”（Active Words）部分的末尾（在`---`分隔符之前）。

### 单词条目格式

```markdown
### [word]
- **Type:** noun/verb/adj/adv
- **Learned:** YYYY-MM-DD HH:MM TZ
- **Book:** [source name]
- **Page:** [number]
- **Pronunciation:** /IPA/
- **Meaning:** [explanation]
- **Synonyms:** [similar words]
- **Context:** "[sentence from source]"
- **Practice History:**
  - YYYY-MM-DD HH:MM TZ: Step N ✓/✗ (notes)
```

### 发音对比条目格式

```markdown
### [word 1] / [word 2] / [word 3]
- **Type:** pronunciation comparison set
- **Learned:** YYYY-MM-DD HH:MM TZ
- **Book:** [source name]
- **Page:** [number]
- **Pronunciation:**
  - **[word 1]** — /IPA/
  - **[word 2]** — /IPA/
  - **[word 3]** — /IPA/
- **Meaning:** Short pronunciation guidance explaining the sound difference (for example: `[word 1]` has /iː/, `[word 2]` has /eɪ/, `[word 3]` has /ɪ/)
- **Synonyms:** pronunciation set, minimal comparison set
- **Context:**
  - **[word 1]:** "[very simple sentence]"
  - **[word 2]:** "[very simple sentence]"
  - **[word 3]:** "[very simple sentence]"
- **Practice History:**
```

### 法语单词

- **上下文：**仅提供法语句子
- **中文翻译：** 使用单独的字段提供中文翻译
- 不要在同一个字段中混合法语和中文
- 用户将音频文件放入`docs/tts-fr/`目录 → 在**音频**（Audio）字段中添加文件路径

## 三步学习流程

在一次对话中完成所有三个步骤（不要分散在几个小时内）：
- **步骤1：** 显示单词及其复习次数（例如：“已复习0次”或“已复习3次”） → 询问“你知道这个单词的发音吗？” → 用户输入单词进行确认
- **步骤2：** 询问“这个词是什么意思？” → 检查答案是否正确或接近正确
- **步骤3：** 要求用户用这个单词造一个句子

基于信任的发音检查 — 不使用语音识别或自动语音识别（ASR）功能。用户需要输入单词进行确认。

### 专注于发音的练习

如果用户希望进行发音练习而不是先学习单词的含义：
- 询问他们容易混淆哪些单词
- 清晰地展示每个单词的发音
- 简要强调关键的发音差异
- 解释**如何发音**（例如嘴型、长元音和短元音的发音方式、舌头或嘴唇的位置），而不是专注于单词的常规含义
- 如果组合这些信息更有用，可以将这些条目记录为一个发音对比条目
- 在“上下文”（Context）字段中使用**非常简单的例句**，每个单词一行
- 对于纯粹的发音对比请求，不必遵循常规的三步学习流程

## 定期复习计划

完成所有三个步骤后，按照以下时间表进行复习：
- 第二天 → 第三天 → 第一周 → 第二周 → 第一个月 → 第三个月
- 经过三个月的复习后，该单词即被视为**掌握**。

### 单词进度

单词在跟踪文件中会经过三个阶段：
1. **活跃单词** — 当前正在学习的单词
2. **长期复习单词** — 完成了所有步骤的单词，需要定期复习
3. **已掌握的单词** — 在三个月内通过了所有复习的单词

## 测验规则
- **每次测验只包含一个单词** — 不允许连续快速提问
- **避免发送重复的测验**：如果今天没有回复之前的测验，则不发送新的测验
- **次日重置**：新的一天可以发送测验，即使昨天的测验没有回答
- **休息时间：** 在用户所在时区的晚上11点至早上7点之间不发送消息
- **优先级**：需要复习的单词 > 新学的单词（未完成步骤的单词） > 复习单词

### 随机选词

1. 统计每个单词的总练习次数（包括所有步骤和复习次数）
2. 按复习次数升序排列所有单词（复习次数最少的单词排在最前面）
3. 从这些单词中随机选取一个单词
4. 在该组内随机选择单词的位置：使用`(current_timestamp_ms % N) + 1`来决定位置
- **避免连续两次选择相同的单词**。检查上次测验的单词状态，跳过已测验过的单词。

**进行测验时始终显示复习次数**：例如：“(已复习0次)”或“(已复习3次)”。

### 按需测验

用户可以随时请求特定测验 — 这些请求会优先于常规的优先级和避免重复发送测验的规则：
- “给我出个测验” → 随机选择一个单词
- “就[单词]出个测验” → 选择特定的单词
- “就[书籍]中的单词出个测验” → 从该书籍中随机选择一个单词
- “就本周的单词出个测验” → 选择过去7天内的单词
- “给我出[书籍]第[N]页到[M]页的单词” → 选择指定页码范围内的单词
- “给我出3个测验” → 连续选择3个单词

**按需复习**：如果没有单词需要复习或待复习，就从所有已学习的单词中随机选择一个单词。仍然会记录测验日期。

## 测验状态跟踪

在词汇跟踪文件的顶部保留一个`## 测验状态**（Quiz State）部分：

```markdown
## Quiz State
- **Pending quiz:** [word] ([review type], [step] sent)
- **Last quiz sent:** YYYY-MM-DD HH:MM TZ
```

每次测验后更新该部分。

## 定时任务设置

当用户请求设置定时词汇测验时：
1. 询问用户希望的测验频率（默认：每小时一次），以及发送渠道和目标
2. **任务名称**：`vocabulary-quiz-{agentName}`（例如：`vocabulary-quiz-english`）
3. 使用以下设置创建一个定时任务：
   - 会话：独立进行
   - 超时时间：120秒
   - 消息：指示代理使用`vocabulary-builder`技能来运行测验
4. **测验规则**（包含在定时任务消息中）：
   - 检查测验状态 — 如果今天的测验尚未回答，则跳过
   - 遵守休息时间：在用户所在时区的晚上11点至早上7点之间不发送消息
