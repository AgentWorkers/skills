# 词汇构建工具

该工具可以读取和写入以下文件（路径相对于工作区）：
- **词汇跟踪器：**`memory/vocabulary.md` — 所有单词数据都存储在这里
- **音频剪辑目录：**`docs/tts-fr/` — 只读；用户可以将发音音频文件保存在这里

在使用前，请确保这些路径存在，如果不存在请创建它们。

## 阅读流程

当用户想要阅读或练习阅读一本书时：
1. 询问他们正在阅读哪本书
2. 在词汇跟踪器中查找该书中的单词 — 阅读“活跃单词”（Active Words）部分的末尾（可以使用`tail`命令或直接查看最后一条记录）以获取最后一个单词。注意：`memory_search`可能会返回部分或排序后的结果，因此在检查最新记录时也需要直接读取文件进行验证。
3. 告诉用户最后一个添加的单词及其所在的页码，以便他们知道从哪里继续阅读
4. 询问用户是想要练习尚未掌握的单词，还是继续阅读并添加新的单词

## 添加新单词

当用户提供一个新的单词时：
1. 提供单词的**发音**（国际音标，美式英语）
2. 提供单词的**含义** — 简单明了
3. 提供**同义词** — 用户可能知道的相似单词
4. 在用户确认后将其添加到词汇跟踪器中（例如：“将[单词]添加到您的词汇表中 — 好吗？”）
5. 添加单词后显示该单词的详细信息

将新单词添加到“活跃单词”（Active Words）部分的末尾（在“---”分隔符之前）。

### 单词条目格式

```markdown
### [word]
- **Type:** noun/verb/adj/adv
- **Learned:** YYYY-MM-DD HH:MM TZ
- **Book:** [source name]
- **Page:** [number]
- **Pronunciation:** /IPA/
- **Meaning:** [explanation]
- **Synonyms:** [similar words]
- **Context:** "[sentence from source]"
- **Practice History:**
  - YYYY-MM-DD HH:MM TZ: Step N ✓/✗ (notes)
```

### 法语单词

- **上下文：**仅提供法语句子
- **中文翻译：**使用单独的字段记录中文翻译
- 不要在同一个字段中混合法语和中文
- 用户将音频剪辑保存在`docs/tts-fr/`目录中 — 在“音频”（Audio）字段中添加文件路径

## 三步学习流程

这三个步骤应在一次对话中完成（不要分散在几个小时内）：
- **第一步：**显示单词及其复习次数（例如：“已复习0次”或“已复习3次”） → 询问“你知道这个单词的发音吗？” → 用户输入单词进行确认
- **第二步：**询问“这个词是什么意思？” → 检查用户的回答是否正确或接近正确
- **第三步：**要求用户用这个单词造一个句子

发音的确认依赖于用户的输入，不使用语音识别或自动语音识别技术。

## 定时复习计划

完成所有三个步骤后，按照以下时间表进行复习：
- 第二天 → 第三天 → 第一周 → 第二周 → 第一个月 → 第三个月
- 经过三个月的复习后，该单词即被视为“掌握”状态

### 单词进度

单词在词汇跟踪器中会经历三个阶段：
1. **活跃单词** — 当前正在学习的单词
2. **长期复习单词** — 完成了所有步骤的单词，需要定期复习
3. **掌握的单词** — 经过三个月的复习

## 测验规则
- **每次测验只包含一个单词** — 不允许连续快速回答多个单词
- **避免发送重复的测验**：如果用户今天没有回答之前的测验，则不发送新的测验
- **次日重置**：新的一天可以发送新的测验，即使昨天的测验没有回答
- **休息时间**：晚上11点至早上7点（用户所在时区）期间不发送消息
- **优先级**：需要复习的单词 > 新学的单词（尚未完成复习的单词） > 需要复习的旧单词

### 随机单词选择

1. 统计每个单词的总练习次数（包括所有步骤和复习次数）
2. 按复习次数从少到多对单词进行排序
3. 选择复习次数最少的单词组
4. 在该组内随机选择一个单词：使用`(current_timestamp_ms % N) + 1`来确定选择的位置
- **避免连续两次选择相同的单词**。通过检查上次测验的状态来避免重复选择相同的单词。

**进行测验时务必显示复习次数**：例如：“(已复习0次)”或“(已复习3次)”。

### 按需测验

用户可以随时请求特定的测验：
- “给我出个测验” → 随机选择一个单词
- “就[单词]出个测验” → 选择特定的单词
- “就[书]中的单词出个测验” → 从该书中随机选择一个单词
- “就本周的单词出个测验” → 从过去7天内学习的单词中随机选择一个
- “就[书]的第[N]页到[M]页的单词出个测验” → 选择指定页码范围内的单词
- “给我出3个测验” → 连续选择3个单词

**按需复习**：如果没有单词需要复习或待复习的单词，可以从所有学过的单词中随机选择一个单词进行测验。同时记录测验日期。

## 测验状态跟踪

在词汇跟踪器文件的顶部保留一个“## 测验状态”（Quiz State）部分：

```markdown
## Quiz State
- **Pending quiz:** [word] ([review type], [step] sent)
- **Last quiz sent:** YYYY-MM-DD HH:MM TZ
```

每次测验后更新该部分的内容。

## 定时任务设置

当用户请求设置定时词汇测验时：
1. 询问用户希望测验的频率（默认：每小时一次），以及测验的发送渠道和目标
2. 为该任务创建一个定时任务（Cron Job），设置如下：
   - 会话隔离（Session: isolated）
   - 超时时间：120秒
   - 消息内容：指示助手使用“vocabulary-builder”技能来运行测验
3. 在定时任务的消息中包含以下要求：
   - 检查测验状态 — 如果今天的测验尚未回答，则跳过本次测验
   - 遵守休息时间：晚上11点至早上7点（用户所在时区）期间不发送消息
