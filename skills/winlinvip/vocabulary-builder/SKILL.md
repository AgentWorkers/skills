# 词汇构建工具

该工具能够读取和写入以下文件（路径相对于工作区）：
- **词汇跟踪器：** `memory/vocabulary.md` — 所有单词数据都存储在这里
- **音频文件目录：** `docs/tts-fr/` — 仅限读取；用户可以将发音音频文件保存在此目录中

在使用前，请确保这些路径存在，如果不存在请创建它们。

## 阅读流程

当用户想要阅读或练习阅读某本书时：
1. 询问用户正在阅读哪本书
2. 在词汇跟踪器中查找该书中的单词——阅读“活跃单词”（Active Words）部分的末尾内容（可以使用 `tail` 命令或直接查看最后一条记录）以确定最后一个单词。注意：`memory_search` 可能会返回部分或排序后的结果，因此还需要直接查看文件以确认最新内容。
3. 告诉用户最后一个添加的单词及其所在页码，以便他们知道从哪里继续阅读
4. 询问用户是想要练习尚未掌握的单词，还是继续阅读并添加新的单词

## 添加新单词

当用户提供一个新的单词时：
1. 提供该单词的**发音**（国际音标，美式英语）
2. 提供单词的**含义**——表述要清晰、简单
3. 提供**同义词**——用户可能已经知道的相似单词
4. 在用户确认后，将该单词添加到词汇跟踪器中（例如：“将 [单词] 添加到您的词汇表中——可以吗？”）
5. 添加单词后，向用户展示该单词的详细信息

新单词应添加到“活跃单词”（Active Words）部分的末尾（在分隔符 `---` 之前）。

### 单词条目格式

```markdown
### [word]
- **Type:** noun/verb/adj/adv
- **Learned:** YYYY-MM-DD HH:MM TZ
- **Book:** [source name]
- **Page:** [number]
- **Pronunciation:** /IPA/
- **Meaning:** [explanation]
- **Synonyms:** [similar words]
- **Context:** "[sentence from source]"
- **Practice History:**
  - YYYY-MM-DD HH:MM TZ: Step N ✓/✗ (notes)
```

### 法语单词

- **上下文：** 仅提供法语句子
- **中文翻译：** 使用单独的字段记录中文翻译
- 不要在同一个字段中混合法语和中文内容
- 用户将音频文件保存在 `docs/tts-fr/` 目录中——请在单词条目中添加相应的**音频**字段，并提供文件路径

## 三步学习流程

这三个步骤应在一次对话中完成（不要分多次进行）：
- **步骤1：** 展示单词及其复习次数（例如：“已复习 0 次”或“已复习 3 次”），然后询问用户是否知道该单词的发音，用户输入单词进行确认
- **步骤2：** 询问单词的含义，确认用户理解是否正确
- **步骤3：** 要求用户用这个单词造一个句子

发音的确认依赖于用户的输入，不使用语音识别或自动语音识别技术。用户需要手动输入单词进行确认。

## 定期复习计划

完成所有三个步骤后，按照以下时间间隔进行复习：
- 第二天 → 第三天 → 第一周 → 第二周 → 第一个月 → 第三个月
- 经过三个月的复习后，该单词被视为被“掌握”

### 单词的学习进度

单词在词汇跟踪器中会经历三个阶段：
1. **活跃单词** — 当前正在学习的单词
2. **长期复习单词** — 完成了所有学习步骤的单词，需要定期复习
3. **已掌握的单词** — 经过三个月的复习

## 测验规则
- **每次测验只包含一个单词** — 不允许连续快速回答多个单词
- **避免发送重复的测验**：如果用户今天没有回答之前的测验，则不发送新的测验
- **次日重置**：新的一天可以发送新的测验，即使昨天的测验未完成
- **休息时间：** 从晚上 11 点到早上 7 点（用户所在时区）不发送消息
- **优先级**：需要复习的单词 > 新学习的单词（尚未完成学习步骤的单词） > 需要复习的旧单词

### 随机选词机制

1. 统计每个单词的总复习次数（包括所有学习步骤和复习记录）
2. 按复习次数从低到高对单词进行排序
3. 选择复习次数最少的单词组
4. 在该组内随机选取单词（使用公式 `(current_timestamp_ms % N) + 1` 确定选取的单词位置）
- **避免连续两次选择相同的单词**。在开始测验前，检查上次测验中涉及的单词，避免重复选择。

**进行测验时务必显示复习次数**：例如：“(已复习 0 次)” 或 “(已复习 3 次)”。

### 按需测验

用户可以随时请求特定内容的测验——这些请求会优先于常规的测验规则：
- “给我出个测验” → 随机选择一个单词
- “就 [单词] 出个测验” → 选择特定的单词
- “就 [书] 中的单词出个测验” → 从该书中随机选择一个单词
- “就本周学过的单词出个测验” → 选择过去 7 天内学过的单词
- “就 [书] 的第 [N] 页到第 [M] 页的单词出个测验” → 选择指定页码范围内的单词
- “给我出 3 个测验” → 连续选择 3 个单词

**按需复习**：如果没有单词需要复习或尚未完成学习步骤，可以从所有学过的单词中随机选择一个单词进行测验。同时记录测验日期。

## 测验状态跟踪

在词汇跟踪器文件的顶部添加一个“## 测验状态”（Quiz State）部分：

```markdown
## Quiz State
- **Pending quiz:** [word] ([review type], [step] sent)
- **Last quiz sent:** YYYY-MM-DD HH:MM TZ
```

每次测验结束后更新该部分的内容。
