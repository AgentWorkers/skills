# comms.md 阅读指南

根据收件人公开的沟通偏好来调整沟通方式和信息内容。

## 相关技能

- **comms-md**：通过引导式访谈创建 comms.md 文件的辅助技能

## 适用场景

在代表用户发送消息、电子邮件或安排通话之前，请先确认收件人是否拥有 comms.md 文件。常见文件存放位置包括：
- 个人网站（例如：`example.com/comms.md`、`example.com/.well-known/comms.md`）
- 个人简介或联系页面中的链接
- 之前的通信记录中直接分享的文件
- 当地的联系人目录或信息库中

如果未找到 comms.md 文件，请跳过此步骤，按常规方式起草信息。

## 文件结构

comms.md 文件最多包含七个部分，但并非所有部分都会出现。请提取与当前任务相关的信息：

| 部分 | 用途 |
| --- | --- |
| **沟通风格与优势** | 了解对方的沟通方式，避免使用可能引起误解的表达方式 |
| **协作模式** | 规划工作关系或合作事宜 |
| **每周沟通节奏** | 确定发送消息的时机或提议会面时间 |
| **同步沟通方式** | 决定是采用同步沟通（电话）还是异步沟通，并制定会议议程 |
| **沟通渠道偏好** | 选择合适的沟通渠道和发送时间 |
| **异步沟通的注意事项** | 调整消息的语气、长度、正式程度和表达方式 |
| **互动协议** | 明确紧急处理流程、沟通时的信号以及偏好的沟通格式 |

## 应用方法

### 选择沟通渠道

1. 根据消息的性质（紧急/非紧急、复杂/简单、正式/非正式）以及沟通场景（高影响力/低影响力）进行分类。
2. 将分类结果与收件人在 comms.md 中指定的 **沟通渠道偏好** 进行匹配。
3. 如果用户要求的沟通渠道与收件人的偏好不符，请提醒他们：“根据他们的 comms.md 文件，这类沟通更适合通过电子邮件进行，请我改用电子邮件发送吗？”

### 确定发送时机

1. 查看收件人的 **每周沟通节奏**，避免在对方休息时间或精力较低的时候发送消息。
2. 注意他们的 **通知与回复习惯**：如果他们下午 3 点前不查看邮件，那么早上发送消息是可以的，但不要期望快速回复。
3. 提议会面时，只选择他们有空的时间段。

### 调整沟通语气

这是最重要的调整环节。请仔细阅读 **异步沟通的注意事项**：
- **匹配对方的沟通风格**：根据双方的关系（亲密朋友、工作关系、新建立的联系或长时间未联系的情况）选择合适的语气。
- **遵循对方的沟通习惯**：如果对方喜欢非正式的沟通方式，就不要使用正式的语言；如果他们不喜欢使用感叹号，就不要在邮件中使用。
- **使用对方认可的沟通方式**：对于新建立的工作关系或正式场合，要提及对方的名字，并引用具体的事例，以表达友好和专业的态度。
- **避免使用对方反感的表达**：如果对方明确表示“不喜欢被打扰”，就不要以“很抱歉打扰您”开头；如果对方不喜欢官方套话，就不要使用类似的表达。

### 拟定会议议程

如果提议进行同步沟通：
- 根据收件人在 comms.md 中指定的沟通目的（如协调工作、解决问题或讨论决策）来制定会议议程。
- 保持请求简洁明了，明确会议议程、预计时长以及你希望从对方那里得到的信息。
- 如果异步沟通更合适，也要明确说明。

## 注意事项

- **不要直接引用 comms.md 文件的内容**：只需根据其中的信息进行相应的调整，避免让对方感到被“机械化”地处理。
- **如有冲突，请向用户说明**：如果你的沟通方式与收件人的偏好不符，要将其作为一种选择而非障碍提出来。
- **注意缺失的信息**：如果需要了解对方的沟通节奏等信息，但 comms.md 文件中未提及，要告知用户：“他们的 comms.md 文件中没有关于沟通时间的说明，你可以询问一下。”

## 示例

用户请求：“为 Alex 撰写一封关于合作撰写健身内容的邮件。”

1. 获取 Alex 的 comms.md 文件。
2. 由于这是工作相关的沟通，因此需要参考 **异步沟通的注意事项** 中关于正式与非正式沟通的部分。
3. Alex 的沟通偏好表明不适合使用“希望您一切安好”这样的开场白。
4. Alex 表示喜欢在邮件中提及对方的名字、引用具体事例，并以友好的方式结束邮件。
5. 从 comms.md 中可以得知 Alex 每周三的会议安排较多，因此周三是一个合适的发送时间。

根据以上信息起草邮件：使用 Alex 最近的工作作为开场白，请求简洁明了，结尾表达友好，避免冗长的内容。
