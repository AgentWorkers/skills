# Aerobase 旅行智能服务

您可以使用 Aerobase 旅行 API 进行航班搜索和分析，并获取航班的时差评分。

## 设置

请设置您的 API 密钥：

```bash
export AEROBASE_API_KEY="ak_..."
```

所有请求的基地址均为 `https://aerobase.app/api`。

## 响应格式

每个响应都会将数据封装在一个标准的响应结构中：

```json
{
  "data": { ... },
  "meta": {
    "request_id": "req_abc123",
    "tier": "free",
    "calls_remaining": 7,
    "latency_ms": 142
  }
}
```

错误信息会以特定的格式返回：

```json
{
  "error": { "code": "VALIDATION_ERROR", "message": "..." },
  "meta": { "request_id": "...", "tier": "free", "calls_remaining": 9, "latency_ms": 12 }
}
```

## 使用限制

免费 tier：每小时 10 次请求。当 `calls_remaining` 达到 0 时，请等待下一个小时再发起请求。

## 可用工具

### 1. 航班时差评分

用户可以输入具体的出发地和目的地时间，系统会评估该航班的时差影响并给出评分。

**必填字段：** `from`（出发地 IATA 代码）、`to`（目的地 IATA 代码）、`departure`（出发时间，格式为 ISO 8601，包含时区偏移量）、`arrival`（到达时间，格式为 ISO 8601，包含时区偏移量）。

**可选字段：** `cabin`（经济舱 | 商务舱 | 头等舱 | 高级经济舱）。

**响应数据：**

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `score` | 数字 | 0-100 分，分数越高时差影响越小 |
| `tier` | 字符串 | 评分等级：优秀 / 良好 / 中等 / 较差 / 严重 |
| `recovery_days` | 数字 | 预计完全恢复所需天数 |
| `direction` | 字符串 | 旅行方向：向东 / 向西 / 相同方向 |
| `timezone_shift_hours` | 数字 | 跨越的时区数量（小时） |
| `breakdown` | 对象 | 时差影响的详细分解（包括生理节律、持续时间、到达后的影响等） |
| `insight` | 字符串 | 便于理解的总结信息 |
| `strategies.departure` | 字符串 | 出发前的建议 |
| `strategies.arrival` | 字符串 | 到达后的建议 |
| `strategies.shift` | 字符串 | 应对时差的方法 |
| `strategies.recovery` | 字符串 | 恢复时间的预估 |
| `tips` | 字符串数组 | 可操作的实用建议 |
| `origin` | 对象 | 出发机场信息（代码、名称、城市、时区） |
| `destination` | 对象 | 目的地机场信息（代码、名称、城市、时区） |

**展示给用户的内容：** “该航班的时差评分为 **X/100**，预计需要约 **Y** 天来恢复。” [包含到达后的建议]

### 2. 搜索航班

根据航线和日期搜索航班，并按时差评分进行排序。

**必填字段：** `from`（出发地 IATA 代码）、`to`（目的地 IATA 代码）、`date`（日期，格式为 YYYY-MM-DD）。

**可选字段：** `return_date`（返回日期）、`max_stops`（最多中转站数，默认为 2）、`sort`（排序方式：时差评分 | 价格 | 航程时长）、`limit`（免费 tier 下最多显示 5 条结果）。

**响应数据：** 一个包含航班信息的数组：

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `id` | 字符串 | 航班标识符 |
| `price` | 数字 | 航班价格（单位：美元，数据库查询结果可能为 null） |
| `duration_minutes` | 数字 | 总旅行时长（分钟） |
| `stops` | 数字 | 中转站数量 |
| `jetlag_score` | 数字 | 航班时差综合评分（0-100 分） |
| `tier` | 字符串 | 时差等级 |
| `recovery_days` | 数字 | 预计恢复所需天数 |
| `direction` | 字符串 | 旅行方向：向东 / 向西 / 相同方向 |
| `booking_url` | 字符串 | 可用的预订链接 |
| `segments` | 数组 | 航班分段信息（包括每个航段的出发地、目的地、航空公司、出发时间、到达时间） |
| `source` | 字符串 | 数据来源：实时数据或数据库 |

**展示给用户的内容：** 以时差评分最高的航班优先显示。如果价格信息可用，也一并展示。突出显示最佳选项与最差选项之间的时差差异。

### 3. 机场信息

获取机场的详细信息，包括有助于缓解时差的设施（如睡眠舱、淋浴间等）以及地面交通方式。

**响应数据：**

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `code` | 字符串 | IATA 机场代码 |
| `name` | 字符串 | 机场名称 |
| `city` | 字符串 | 机场所在城市 |
| `country` | 字符串 | 机场所在国家 |
| `timezone` | 字符串 | IANA 时区代码 |
| `latitude` | 数字 | 机场纬度 |
| `longitude` | 数字 | 机场经度 |
| `facilities` | 数组 | 机场提供的时差缓解设施 |
| `lounges` | 数组 | 可用的休息室及其设施和评分 |
| `transit` | 数组 | 可用的地面交通方式及其时间和费用 |

**展示给用户的内容：** 突出显示有助于缓解时差的设施，并说明转机时的休息室使用情况。

---

### 4. 航线分析

提供两个机场之间的综合航线分析，包括直飞和转机选项。

**响应数据：**

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `origin` | 对象 | 出发机场信息（代码、名称、城市、国家、时区） |
| `destination` | 对象 | 目的地机场信息 |
| `timezone_shift` | 对象 | 跨越的时区数量、方向、实际时差小时数、UTC 偏移量 |
| `direct_routes` | 数组 | 直飞航班信息（包括时差评分、航空公司、飞行距离） |
| `connecting_routes` | 数组 | 经停航班信息（包含中转机场） |
| `route_count` | 数字 | 找到的航线总数 |

每条航线信息包括：`jetlag_score`（0-100 分）、`stops`（中转站数量）、`connections`（转机次数）、`total_distance_km`（总飞行距离）、`total_duration_minutes`（总飞行时长）、`recovery_days`（预计恢复天数）。

**展示给用户的内容：** 首先说明时区跨越情况（例如：“该航线跨越了 **X** 个时区”）。比较直飞和转机选项，注意转机有时能更好地帮助调整生物钟。

### 5. 旅行优惠

浏览经过时差评分的旅行优惠信息。免费 tier 下最多显示 3 条结果。

**查询参数：**

| 参数 | 类型 | 说明 |
|-------|------|-------------|
| `departure` | 字符串 | 出发地 IATA 代码（可选） |
| `destination` | 字符串 | 目的地 IATA 代码（可选） |
| `max_price` | 数字 | 最高价格（单位：美元） |
| `sort` | 字符串 | 排序方式：综合评分 / 价格 / 时差评分 / 最新优惠 |
| `limit` | 数字 | 最多显示的结果数量（免费 tier 下为 3 条） |

**响应数据：** 一个包含优惠信息的数组：

| 字段 | 类型 | 说明 |
| `title` | 优惠标题 |
| `source` | 优惠来源 |
| `price_usd` | 价格（单位：美元） |
| `cabin_class` | 航班舱位等级 |
| `is_error_fare` | 布尔值 | 是否为错误价格 |
| `origin` | 出发机场信息 |
| `destination` | 目的地机场信息 |
| `jetlag` | 航班时差评分、恢复天数、旅行方向及相关建议 |
| `value_score` | 综合评分 |
| `travel_dates` | 旅行日期范围 |
| `booking_deadline` | 预订截止日期 |
| `source_url` | 预订链接 |

**展示给用户的内容：** 首先显示综合评分最高的优惠。同时说明时差影响和价格。错误价格会特别标注（以免用户误选）。

## 展示指南

1. **务必提及时差评分和恢复所需天数**——这些是重要的信息。
2. **在展示多个选项时进行比较**：“航班 A 的时差评分为 82/100，航班 B 为 61/100，后者需要多休息一天。”
3. **突出出行前的建议和到达后的应对策略**——用户需要的是实用的建议，而不仅仅是数字。
4. **将技术性数据转化为易于理解的语言**——例如：“向东飞行时将跨越 **9** 个时区”，而不是直接使用“时区跨越小时数：9，方向：向东”。
5. **上下文很重要**——2 小时内的航班时差评分为 70 分可能不算显著，但在跨太平洋的航班中则属于优秀等级。
6. **使用明确的评分等级**——“优秀”、“良好”、“中等”、“较差”、“严重”等标签便于用户快速理解。

## 评分解读

| 评分 | 评分等级 | 恢复所需时间 | 含义 |
|-------|------|----------|---------|
| 80-100 | 优秀 | 0-1 天 | 时差影响极小，航班时间安排合理 |
| 65-79 | 良好 | 1-2 天 | 通过简单措施可以应对时差 |
| 50-64 | 中等 | 2-3 天 | 需要采取相应措施来缓解时差 |
| 35-49 | 较差 | 3-5 天 | 需要较长时间来适应时差 |
| 0-34 | 严重 | 超过 5 天 | 建议考虑其他出行时间
