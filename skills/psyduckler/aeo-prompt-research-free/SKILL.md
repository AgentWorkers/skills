# AEO Prompt Research（免费）

> **来源：** [github.com/psyduckler/aeo-skills](https://github.com/psyduckler/aeo-skills/tree/main/aeo-prompt-research-free)  
> **所属项目：** [AEO Skills Suite](https://github.com/psyduckler/aeo-skills) — Prompt Research → [内容](https://github.com/psyduckler/aeo-skills/tree/main/aeo-content-free) → [分析](https://github.com/psyduckler/aeo-skills/tree/main/aeo-analytics-free)

通过免费API，发现哪些提示和主题对品牌的人工智能可见性至关重要。

## 需求

- `web_fetch`：用于爬取目标网站  
- `web_search`：Brave Search的免费版本（可选，但推荐使用）  
- LLM推理：由代理自身的模型完成大部分工作  

无需API密钥、付费工具或账户。

## 工作流程

### 输入

用户提供以下信息：  
- **域名**（必填）：例如 `clearscope.io`  
- **细分领域/类别**（可选）：例如 “面向内容团队的SEO软件”  
- **竞争对手**（可选）：例如 “Surfer SEO、MarketMuse、Frase”  

### 第1步：爬取网站  

使用 `web_fetch` 爬取关键页面，以了解品牌信息：  
```
Pages to fetch (try each, skip 404s):
- / (homepage)
- /about or /about-us
- /pricing
- /products or /features or /services
- /blog (index only)
```  

或者，运行 `scripts/crawl_site.sh <domain>` 进行批量爬取。  

从爬取的内容中提取以下信息：  
- 核心产品/服务  
- 目标受众（行业、公司规模、用户画像）  
- 主要差异化优势/价值主张  
- 竞争对手的相关信息  
- 博文标题中的内容主题  

### 第2步：确定主题范围  

根据对品牌的了解，头脑风暴出相关主题类别。有关方法和类别类型，请参阅 `references/aeo-methodology.md`。  

需要生成的核心提示类别包括：  
1. 问题导向型：“我该如何解决[问题]？”  
2. 解决方案导向型：“有哪些[类别]相关的工具？”  
3. 对比型：“[品牌]与[竞争对手]相比如何？”  
4. 最佳实践型：“在[使用场景]下，[类别]的最佳选择是什么？”  
5. 操作指南型：“该产品如何帮助完成[任务]？”  
6. 评估型：“[品牌]是否适合满足[需求]？”  
7. 行业导向型：“[行业]的趋势/最佳实践是什么？”  

### 第3步：生成提示  

为每个类别生成5-15个用户实际可能会向AI助手提出的具体提示。  

生成提示的指导原则：  
- 语言要自然——模仿人们与ChatGPT交流的方式，而非使用谷歌搜索时的语言。  
- 要具体——包含上下文信息（公司规模、行业、使用场景）。  
- 提示的意图要多样化——包括研究、对比、操作指南、购买决策等。  
- 避免使用专业术语或不切实际的提示。  

### 第4步：排序提示  

根据以下标准对每个提示进行评分（1-5分）：  
- **相关性**：与品牌的核心产品/服务关联程度如何？  
- **潜在使用频率**：有多少人可能会提出这样的问题？  
- **竞争力**：该品牌是否有可能成为最佳答案？  
- **购买意向**：该提示是否表明用户的购买或转化意图？  

评分公式：`优先级 = (相关性 × 2 + 使用频率 + 竞争力 + 购买意向) / 5`  

将提示分为三个等级：  
- 第1级（≥3.5分）  
- 第2级（2.5-3.4分）  
- 第3级（<2.5分）  

### 第5步：审核现有内容覆盖情况  

对于第1级的提示，使用 `web_search` 和 `site:domain.com [主题关键词]` 来检查是否已有相关内容。  
评估内容覆盖情况：  
- **完全覆盖**：有专门页面直接回答了该提示。  
- **部分覆盖**：存在相关内容，但未全面解答。  
- **无覆盖**：未找到相关内容。  

### 第6步：输出结果  

生成一份结构化的报告，内容包括：  
1. 品牌简介（2-3句话）  
2. 带有评分和覆盖情况的优先级提示列表  
3. 内容缺口分析（高优先级但未被覆盖的提示）  
4. 需要优先创建的5篇推荐内容  

报告格式参考 `references/aeo-methodology.md`。  

## 提高效果的小贴士：  
- 如果 `web_search` 不可用，该工具仍可正常使用——只需跳过内容覆盖审核步骤，或让用户手动检查。  
- 对于竞争对手分析，也请爬取他们的网站并比较主题覆盖情况。  
- 每季度重新运行一次分析——随着模型和用户行为的变化，AI提示的趋势也会发生变化。  
- 代理对行业的了解也是有效的研究依据——请充分利用这一信息。
