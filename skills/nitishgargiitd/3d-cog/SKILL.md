# 3D Cog – 将创意转化为3D模型

**其他工具需要完美的图片，而CellCog则能将创意直接转化为3D模型。**

大多数3D生成工具都需要一张构图完美的参考图片。但CellCog可以处理任何类型的输入——无论是文字描述、草图、产品照片，甚至是一份包含50个物品的列表——并完成整个转化流程：理解你的需求、生成优化的参考图片，最后将其转换为可用于制作的GLB格式文件。

## CellCog的独特之处

### 任意输入均可转化为3D模型

CellCog的真正优势在于其“任意输入到任意输出”的能力：
| 输入类型 | CellCog的处理过程 | 输出结果 |
|--------------|-------------------|--------------|
| 文字描述 | 分析物体特征 → 生成优化后的参考图片 → 转换为3D模型 | 可用于制作的GLB文件 |
| 草图 | 将草图细化为清晰、详细的参考图像 → 转换为3D模型 | 可用于制作的GLB文件 |
| 产品照片 | 评估照片质量并进行必要的优化 → 转换为3D模型 | 可用于制作的GLB文件 |
| 高质量概念图 | 直接将其转换为3D模型 | 可用于制作的GLB文件 |
| 多个物品列表 | 为每个物品生成参考图片 → 将所有图片转换为3D模型 | 共10个GLB文件 |

### 批量生成

你需要为RPG游戏制作10件低多边形武器？为室内设计师准备20件家具模型？还是为电子商务网站生成50件产品模型？

只需一个指令，CellCog就能生成多个3D模型。它会根据需求自动调整每个模型的构图、角度和细节层次，然后将其转换为带纹理的3D模型。

## 输出格式

所有3D模型均以**GLB格式**（二进制glTF格式）提供——这是通用的3D文件格式：
- 支持Unity、Unreal、Godot、Three.js、Babylon.js等引擎
- 可在网页浏览器中通过`<model-viewer>`或Three.js插件查看
- 也与Blender、Maya、3ds Max等软件兼容，便于进一步编辑
- 文件中包含纹理和材质信息

## 3D任务的处理方式

| 任务类型 | 推荐使用模式 |
|----------|------------------|
| 根据清晰描述或图片创建单个3D模型 | 使用“agent”模式 |
| 批量生成多个模型（5-20个） | 使用“agent”模式 |
| 需要跨资产保持风格一致性的复杂项目 | 使用“agent team”模式 |

**大多数3D任务建议使用“agent”模式**，它可以处理从单个模型到批量生成的各类需求。

**当需要确保所有资产风格一致时（例如，生成一套具有统一艺术风格的奇幻武器），请使用“agent team”模式**。

## 提高3D模型质量的建议

1. **明确指定材质**：如“抛光铝制”、“仿古皮革”、“抛光大理石”等。这些信息有助于CellCog生成更准确的参考图片和纹理。
2. **指定目标平台**：例如“为移动游戏生成低多边形模型”或“为电影级渲染生成高细节模型”，这会直接影响生成效果。
3. **尽可能提供参考图片**：即使图片不完美，也能为CellCog提供更多参考信息。
4. **在批量任务中统一风格描述**：例如“所有模型采用统一的手绘奇幻风格”，以确保资产的一致性。
5. **请求使用PBR材质**：如果需要金属质感、粗糙度等效果，请明确说明。这些参数对游戏引擎和真实感渲染至关重要。
