# Game Cog – 构建游戏世界，而不仅仅是精灵图像

**其他工具主要用于生成精灵图像，而CellCog则专注于构建整个游戏世界。** 在DeepResearch Bench的评估中，CellCog在深度游戏设计方面排名第一（2026年2月）。

游戏开发是一个跨学科的任务——机制设计、艺术风格、音乐、用户界面以及关卡设计都需要保持高度的一致性。CellCog首先会深入理解你的游戏理念，然后根据这些理念生成角色相关的艺术资源、瓦片集、音乐、音效、用户界面元素、3D模型以及完整的游戏设计文档——所有这些内容都源自同一个项目需求。

**快速使用指南（v1.0及以上版本）：**
```python
# Fire-and-forget - returns immediately

)
# Daemon notifies you when complete - do NOT poll
```

## 你可以创建的内容

### 角色设计

让你的游戏角色栩栩如生：

- **玩家角色**：“设计一个具有多种姿势的赛博朋克武士主角”
- **非玩家角色（NPC）**：“为奇幻角色扮演游戏创建一个友好的商人角色”
- **敌人**：“设计一个 boss 怪物——一个被腐蚀的树之守护者”
- **角色资料表**：“创建包含闲置状态、行走状态和攻击姿势的角色资料表”
- **肖像**：“为我的视觉小说角色生成对话相关的肖像图片”

**示例提示：**
> “为一款温馨的农场游戏设计一个主角：
> 
> 风格：受《星露谷物语》启发，但分辨率更高
> 角色：年轻农民，可自定义性别，表情友善
> 
> 需要的内容：
> - 正面、背面和侧面的角色图像
- 闲置姿势
- 行走动画（四个方向）
- 持有工具的姿势（锄头、浇水壶）
> 
> 色彩搭配：温暖、自然色调”

### 环境与瓦片

构建你的游戏世界：

- **瓦片集**：“为俯视视角的角色扮演游戏创建一个森林瓦片集”
- **背景图**：“为横版游戏设计具有视差效果的背景图”
- **关卡概念**：“为恐怖庄园关卡创作概念艺术图”
- **道具**：“为中世纪酒馆生成装饰性道具”
- **用户界面元素**：“设计生命值条、物品栏和按钮”

**示例提示：**
> “为一款地下城探索游戏创建一个瓦片集：
> 
> 风格：受16位游戏风格启发，具有黑暗奇幻氛围
> 
> 包括的内容：
- 地板瓦片（石头、泥土、水）
- 墙壁瓦片（砖块、洞穴、装饰性图案）
- 门（木制、铁制、魔法效果）
- 道具（火把、箱子、桶）
- 陷阱（尖刺、压力板）
> 
> 所有瓦片之间需要无缝衔接。”

### 游戏概念

发展你的游戏创意：

- **游戏设计文档**：“为这款随机生成内容的角色扮演游戏创建设计文档”
- **故事大纲**：“为这款科幻角色扮演游戏编写主要剧情”
- **机制设计**：“为我的动作游戏设计独特的战斗系统”
- **世界观构建**：“为末日世界构建背景故事”
- **项目提案文档**：“为我的独立游戏制作项目提案文档，以吸引发行商”

**示例提示：**
> “为这款手机解谜游戏创建游戏设计文档：
> 
> 核心概念：结合了匹配消除游戏和城市建造元素
> 目标玩家：休闲玩家，游戏时长约5分钟
> 
> 包括的内容：
- 核心游戏循环机制
- 游戏进度系统
- 营利策略（公平的免费游戏模式）
- 前10个关卡的设计
- 艺术风格建议
> 
> 参考游戏：《Gardenscapes》和《SimCity》**

### 3D模型与资源

为你的游戏引擎准备GLB格式的3D模型：

- **角色**：“为我的角色扮演游戏主角创建3D模型”
- **武器与物品**：“生成3D武器模型（剑、斧头、弓、法杖）”
- **道具**：“创建3D地下城道具（箱子、桶、火把）”
- **载具**：“为我的手机游戏制作低多边形的宇宙飞船模型”
- **环境元素**：“为游戏世界生成3D树木、岩石和建筑物”

CellCog可以处理整个制作流程——你只需描述你的需求，它就会生成优化后的参考图像，并将其转换为带纹理的3D模型。支持批量生成（例如：“生成10个武器模型”）。

GLB格式的模型兼容Unity、Unreal、Godot、Three.js和Blender等游戏引擎。请根据目标平台指定模型的多边形数量和PBR材质属性。

对于专门的3D模型生成工作流程，也可以查看`3d-cog`。

## 艺术风格

| 风格 | 适用场景 | 特点 |
|-------|----------|-----------------|
| **像素艺术** | 复古风格、独立游戏 | 令人怀旧，色彩简洁 |
| **手绘艺术** | 角色扮演游戏、奇幻游戏 | 色彩丰富、细节精美 |
| **矢量/平面艺术** | 手机游戏、休闲游戏 | 简洁、可扩展、现代感强 |
| **低多边形3D** | 风格化的3D游戏 | 几何形状鲜明、独特 |
| **动漫/漫画风格** | 视觉小说、日式角色扮演游戏 | 表现力强、风格独特 |
| **写实风格** | 3A级游戏 | 细节丰富、沉浸感强 |
| **3D模型（GLB格式）** | 适用于游戏引擎、AR/VR技术 | 具有纹理效果，可自定义多边形数量 |

## 游戏开发中的交流方式

| 交流场景 | 推荐的交流模式 |
|----------|------------------|
| 单个资源、精灵图像、角色设计、用户界面元素 | `"agent"` |
| 完整的游戏概念、复杂的世界构建、叙事设计 | `"agent team"` |

**大多数游戏资源使用``agent`模式`**。角色模型、瓦片集和用户界面元素在`agent`模式下效果最佳。**

**对于需要深入的游戏设计，可以使用``agent team`模式`**——例如需要完整的游戏设计文档、复杂的叙事结构，或者需要多角度的创意讨论时。

## 提高游戏资源质量的建议

1. **明确尺寸要求**：指定“32x32像素的瓦片”或“1920x1080像素的背景图”，以避免资源不匹配的问题。
2. **参考现有游戏**：例如“模仿《Hollow Knight》或《Celeste》的风格，可以提供明确的创作方向。”
3. **考虑实现细节**：请求的资源格式应符合你的游戏引擎要求。如果需要透明度、图层效果或特定的文件格式，请务必说明。
4. **保持一致性**：在请求多个资源时，描述你的游戏整体设计风格，以确保所有资源风格统一。
5. **关于动画帧**：明确所需的帧数，以及是需要精灵图像集还是单个动画帧。
6. **合理规划工作流程**：从使用占位符资源开始，逐步完善。完美无瑕的成果往往难以实现。
