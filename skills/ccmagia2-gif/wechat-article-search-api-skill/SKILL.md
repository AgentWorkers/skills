# 微信文章搜索API

## 📖 介绍  
该功能通过BrowserAct的微信文章搜索API模板，为用户提供自动化的微信文章提取服务。用户只需提供搜索关键词和可选的日期筛选条件，即可直接获取包含完整内容的结构化微信文章数据。

## ✨ 特点  
1. **数据稳定且准确**：预配置的工作流程避免了人工智能生成的不准确结果。  
2. **无需处理验证码**：无需应对reCAPTCHA或其他验证挑战。  
3. **无IP限制或地域封锁**：不受地区IP限制的影响。  
4. **执行速度更快**：相比纯AI驱动的浏览器自动化方案，执行速度更快。  
5. **极具成本效益**：与消耗大量API令牌的AI解决方案相比，显著降低了数据获取成本。  

## 🔑 API密钥使用指南  
在运行脚本之前，请检查`BROWSERACT_API_KEY`环境变量。如果未设置该变量，请等待用户提供API密钥。  
**代理应告知用户**：  
> “由于您尚未配置BrowserAct API密钥，请访问[BrowserAct控制台](https://www-browseract.com/reception/integrations)获取您的密钥。”  

## 🛠️ 输入参数  
在调用脚本时，代理应根据用户需求灵活配置以下参数：  

1. **keywords（搜索关键词）**  
   - **类型**：`string`  
   - **说明**：用于查找微信文章的搜索关键词，可以是行业术语、主题或具体短语。  
   - **示例**：`openclaw`、`AI agent`、`browser automation`  

2. **date_limit（提取限制）**  
   - **类型**：`number`  
   - **说明**：允许提取的最大文章数量。首次使用时建议使用较小的默认值。  
   - **默认值**：`10`  
   - **建议**：测试时使用`5`到`10`条；批量研究时使用较大数值。  

3. **publication_date（发布日期筛选）**  
   - **类型**：`string`  
   - **说明**：按文章发布日期进行筛选。  
   - **示例**：`3月11日`、`March 10`、`2026-03-11`  

## 🚀 调用方法  
代理应执行以下脚本以实现“一次命令、直接获取结果”的目标：  

```bash  
# 示例调用  
python -u ./scripts/wechat_article_search_api.py "keywords" limit "publication_date"  
```  

### ⏳ 运行状态监控  
由于该任务涉及自动化浏览器操作，可能需要较长时间（几分钟）。脚本运行期间会持续输出带有时间戳的状态日志（例如：`[14:30:05] 任务状态：正在运行`）。  
**代理操作指南**：  
- 在等待脚本返回结果时，请持续监控终端输出。  
- 只有当终端持续输出新的状态日志时，才表示任务正在正常运行；切勿误判为死锁或无响应状态。  
- 仅当状态长时间未变化，或脚本停止输出且未返回结果时，才考虑触发重试机制。  

## 📊 输出数据说明  
脚本成功执行后，会直接从API响应中解析并打印结果。输出内容包括：  
- `url_link`：文章的原始URL  
- `publication_date`：文章的发布日期  
- `author`：文章作者或发布账号名称  
- `image_url`：文章的主图片URL或封面图片URL  
- `body_content`：文章的全文内容  
- `title`：文章的完整标题  

## ⚠️ 错误处理与重试  
在脚本执行过程中，如果出现错误（如网络波动或任务失败），代理应按照以下逻辑处理：  

1. **检查输出内容**：  
   - 如果输出中包含“Invalid authorization”，说明API密钥无效或已过期。此时**不要重试**，应引导用户重新检查并提供正确的API密钥。  
   - 如果输出中不包含“Invalid authorization”，但任务仍然失败（例如输出以“Error:”开头或返回空结果），代理应**自动尝试再次执行脚本**。  

2. **重试次数限制**：  
   - 自动重试仅允许**一次**。如果第二次尝试仍然失败，则停止重试，并向用户报告具体的错误信息。  

## 🌟 典型应用场景  
1. **内容监控**：追踪微信文章中特定品牌或主题的提及情况。  
2. **媒体研究**：分析顶级微信账号发布的文章全文。  
3. **趋势追踪**：收集关于热门行业趋势（如AI相关主题）的文章以供深入阅读。  
4. **知识库构建**：将文章内容导入内部数据库。  
5. **竞争对手分析**：查看竞争对手账号发布的完整文章。
