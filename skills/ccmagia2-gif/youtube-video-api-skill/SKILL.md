# YouTube 视频 API 技能

## 📖 介绍
该技能利用 BrowserAct 的 YouTube 视频 API 模板，为用户提供一站式 YouTube 视频数据提取服务。通过一次 API 请求，即可直接从指定的 YouTube 频道中提取结构化的频道级数据以及视频详细信息。只需输入 YouTube 频道 URL 和视频类型（最新、热门或最早），即可获得清晰、可直接使用的数据指标。

## ✨ 特点
1. **数据提取稳定且准确**：预设的工作流程避免了生成式 AI 造成的数据错误。
2. **无需处理 CAPTCHA**：无需应对 reCAPTCHA 或其他验证挑战。
3. **无 IP 访问限制和地域封锁**：无需处理区域性的 IP 限制。
4. **执行速度更快**：相比纯 AI 驱动的浏览器自动化解决方案，任务执行速度更快。
5. **极具成本效益**：与消耗大量 Tokens 的 AI 解决方案相比，显著降低了数据获取成本。

## 🔑 API 密钥使用指南
在运行之前，必须检查 `BROWSERACT_API_KEY` 环境变量。如果该变量未设置，请勿执行其他操作。应请求用户提供 API 密钥。
**此时，代理应告知用户**：
> “由于您尚未配置 BrowserAct API 密钥，请先访问 [BrowserAct 控制台](https://www.browseract.com/reception/integrations) 获取密钥。”

## 🛠️ 输入参数
在调用脚本时，代理应根据用户需求灵活配置以下参数：

1. **YouTube_channel_url**
   - **类型**：`string`
   - **描述**：用于加载频道视频列表的目标 YouTube 频道 URL。
   - **示例**：`https://www.youtube.com/@BrowserAct`

2. **Video_type**
   - **类型**：`string`
   - **描述**：在遍历频道视频列表时使用的排序方式。
   - **可选值**：
     - `Latest`
     - `Popular`
     - `Earliest`
   - **默认值**：`Popular`

## 🚀 调用方法
代理应通过执行以下独立脚本来实现“一次命令获取结果”的目标：

```bash
# Invocation example
python -u ./scripts/youtube_video_api.py "YouTube_channel_url" "Video_type"
```

### ⏳ 运行状态监控
由于此任务涉及自动化浏览器操作，可能需要较长时间（几分钟）。脚本在运行过程中会**持续输出带有时间戳的状态日志**（例如：`[14:30:05] 任务状态：运行中`）。
**代理注意事项**：
- 在等待脚本返回结果时，请密切关注终端输出。
- 只要终端仍在输出新的状态日志，说明任务正在正常运行。不要误认为是死锁或无响应。
- 如果状态长时间未发生变化或脚本停止输出且未返回结果，可以考虑使用重试机制。

## 📊 数据输出说明
脚本成功执行后，会直接从 API 响应中解析并打印结果。结果包括：
### 频道字段
- `channel_title`：频道页面上显示的频道名称
- `channel_url`：频道 URL
- `subscribers`：频道页面上显示的订阅者数量
### 视频字段
- `video_title`：视频页面上显示的视频标题
- `video_url`：视频 URL
- `publish_date`：YouTube 上显示的发布日期或时间
- `view_count`：YouTube 上显示的观看次数
- `video_duration`：视频时长
- `comment_count`：总评论数（如果可用）
- `like_count`：点赞数（如果可用）

## ⚠️ 错误处理与重试
在脚本执行过程中，如果出现错误（如网络波动或任务失败），代理应按照以下逻辑处理：

1. **检查输出内容**：
   - 如果输出中包含 “Invalid authorization”，说明 API 密钥无效或已过期。此时，**不要重试**，而是引导用户重新检查并提供正确的 API 密钥。
   - 如果输出中不包含 “Invalid authorization”，但任务执行失败（例如，输出以 “Error:” 开头或返回结果为空），代理应**自动尝试再次执行脚本**。

2. **重试限制**：
   - 自动重试次数限制为 **一次**。如果第二次尝试仍然失败，则停止重试，并向用户报告具体的错误信息。

## 🌟 典型应用场景
1. **竞争对手跟踪**：跟踪竞争对手频道的性能趋势和发布频率。
2. **创作者研究**：分析内容创作者的互动信号和热门视频。
3. **内容运营报告**：监控频道视频和性能指标以用于报告。
4. **增长分析**：了解哪些视频类型（最新/热门）推动了频道增长。
5. **数据库自动化**：将频道视频直接导入 CRM 或数据库，无需手动导出。
6. **市场研究**：汇总特定行业内不同频道的视频指标。
7. **趋势识别**：识别特定科技或游戏频道中最受欢迎的视频。
8. **受众互动分析**：将订阅者数量与视频观看次数和点赞数关联起来。
9. **内容策略**：查看频道的最早视频，了解其起源和增长路径。
10. **自动化社交监控**：关注行业领先者发布的新内容。
