# YouTube 评论 API 自动化技能

## 📖 介绍  
本技能通过 `BrowserAct YouTube 评论 API` 模板，提供一站式提取 YouTube 视频及评论数据的服务。它可以直接从 YouTube 中提取结构化的视频信息及其对应的评论。只需提供搜索关键词、评论数量和滚动次数，即可获得干净、可直接使用的数据集。

## ✨ 特点  
1. **零错误率，确保数据稳定且准确**：预先配置的工作流程避免了 AI 生成错误。  
2. **无需处理验证码**：无需应对 reCAPTCHA 或其他验证挑战。  
3. **无 IP 访问限制或地理封锁**：无需处理区域 IP 限制。  
4. **执行速度更快**：相比纯 AI 驱动的浏览器自动化解决方案，执行速度更快。  
5. **极高成本效益**：与消耗大量令牌的 AI 解决方案相比，显著降低了数据获取成本。  

## 🔑 API 密钥获取指南  
在运行之前，必须先检查 `BROWSERACT_API_KEY` 环境变量。如果该变量未设置，请勿执行其他操作；应请求用户提供 API 密钥。  
**此时，代理应告知用户**：  
> “由于您尚未配置 BrowserAct API 密钥，请先访问 [BrowserAct 控制台](https://www.browseract.com/reception/integrations) 获取密钥。”  

## 🛠️ 输入参数  
在调用脚本时，代理应根据用户需求灵活配置以下参数：  

1. **keywords**  
   - **类型**：`string`  
   - **描述**：用于在 YouTube 上搜索视频的关键词或短语。  
   - **示例**：`AI`、`automation`、`web scraping`  
   - **默认值**：`AI`  

2. **Comments_limit**  
   - **类型**：`number`  
   - **描述**：每个视频最多提取的评论数量。  
   - **示例**：`10`、`20`、`50`  
   - **默认值**：`10`  

3. **Scroll_count**  
   - **类型**：`number`  
   - **描述**：在提取评论前，滚动评论区以加载更多评论的次数。  
   - **示例**：`1`、`2`、`5`、`10`  
   - **默认值**：`2`  

## 🚀 推荐调用方式  
代理应执行以下独立脚本来实现“一次命令，获取结果”：  

```bash  
# 示例调用  
python -u ./scripts/youtube_comments_api.py "keywords" "Comments_limit" "Scroll_count"  
```  

### ⏳ 运行状态监控  
由于此任务涉及自动化浏览器操作，可能需要较长时间（几分钟）。脚本运行期间会**持续输出带时间戳的状态日志**（例如：`[14:30:05] 任务状态：运行中`）。  
**代理提示**：  
- 在等待脚本返回结果时，请持续监控终端输出。  
- 只要终端仍在输出新的状态日志，说明任务正常运行。切勿误认为是死锁或无响应。  
- 仅当状态长时间不变或脚本停止输出且未返回结果时，才考虑触发重试机制。  

## 📊 数据输出说明  
脚本成功执行后，会直接解析并打印 API 响应的结果。结果包含两个数据集：  

**视频字段：**  
- `video_name`：列表中显示的视频标题  
- `video_url`：视频 URL  
- `video_publication_time`：发布时间  
- `video_view_count`：观看次数  

**评论字段：**  
- `commenter_name`：评论作者的显示名称  
- `commenter_url`：评论作者的频道 URL  
- `comment_text`：评论内容  
- `comment_publish_date`：评论发布时间  
- `comment_likes`：评论的点赞数  
- `reply_count`：评论的回复数  

## ⚠️ 错误处理与重试  
在脚本执行过程中，如果出现错误（如网络波动或任务失败），代理应按照以下逻辑处理：  

1. **检查输出内容**：  
   - 如果输出中包含 “Invalid authorization”，说明 API 密钥无效或已过期。此时**不要重试**；应引导用户重新检查并提供正确的 API 密钥。  
   - 如果输出中不包含 “Invalid authorization”，但任务仍然失败（例如，输出以 “Error:” 开头或返回空结果），代理应**自动尝试再次执行脚本**。  

2. **重试次数限制**：  
   - 自动重试仅限于**一次**。如果第二次尝试仍失败，则停止重试，并向用户报告具体错误信息。  

## 🌟 典型应用场景  
1. **受众洞察**：根据特定关键词将评论转化为产品反馈和情感信号。  
2. **内容研究**：了解观众在热门视频话题下的讨论内容。  
3. **竞争监控**：跟踪竞争对手 YouTube 频道的评论和反馈。  
4. **社区洞察**：分析用户在自动化或 AI 等特定领域的关注点。  
5. **话题追踪**：监控热门搜索词的公众反馈和互动情况。  
6. **情感分析**：从评论中收集原始文本数据以评估观众意见。  
7. **问题识别与需求收集**：从产品相关视频评论中识别用户痛点。  
8. **自动化数据集成**：通过 API 将视频和评论数据直接发送到 CRM 或 BI 工具。  
9 **互动指标收集**：跟踪热门评论的点赞数和回复数。  
10. **市场研究**：提取大量视频元数据及用户讨论内容，用于市场分析。
