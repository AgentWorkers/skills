# YouTube搜索API技能

## 📖 介绍  
该技能通过BrowserAct的YouTube搜索API模板，为用户提供一站式YouTube搜索数据提取服务。它可以直接从YouTube搜索结果列表中提取结构化的数据。只需提供搜索关键词和限制条件，即可获取清晰、可用的视频、短视频、频道或播放列表数据。

## ✨ 特点  
1. **无数据生成错误，确保数据提取的稳定性和准确性**：预设的工作流程避免了人工智能生成错误（即“幻觉”现象）。  
2. **无需处理验证码**：无需应对reCAPTCHA或其他验证挑战。  
3. **无IP访问限制或地理屏蔽**：无需处理区域IP限制。  
4. **执行速度更快**：相比纯AI驱动的浏览器自动化解决方案，任务执行速度更快。  
5. **极高的成本效益**：与消耗大量令牌的AI解决方案相比，显著降低了数据获取成本。  

## 🔑 API密钥设置流程  
在运行之前，必须先检查`BROWSERACT_API_KEY`环境变量。如果该变量未设置，请勿执行其他操作；应请求用户并提供API密钥。  
**此时，代理应告知用户**：  
> “由于您尚未配置BrowserAct API密钥，请先访问[BrowserAct控制台](https://www.browseract.com/reception/integrations)获取密钥。”  

## 🛠️ 输入参数  
在调用脚本时，代理应根据用户的需求灵活配置以下参数：  

1. **KeyWords**  
   - **类型**：`string`  
   - **描述**：在YouTube上使用的搜索关键词。可以是任意关键词或短语。  
   - **示例**：`AI`、`automation`、`n8n`、`web scraping`  

2. **Video_type**  
   - **类型**：`string`  
   - **描述**：从哪个结果页面提取数据。  
   - **支持的值**：`Videos`、`Shorts`、`Channels`、`Playlists`  
   - **默认值**：`Videos`  

3. **Date_limit**  
   - **类型**：`number`  
   - **描述**：从搜索结果列表中提取的最大项目数量。  
   - **示例**：`20`、`50`、`100`  
   - **默认值**：`100`  

## 🚀 使用方法（推荐）  
代理应通过执行以下独立脚本来实现“一键获取结果”：  

```bash  
# 调用示例  
python -u ./scripts/youtube_search_api.py "KeyWords" "Video_type" Date_limit  
```  

### ⏳ 执行状态监控  
由于此任务涉及自动化的浏览器操作，可能需要较长时间（几分钟）。脚本在运行过程中会**持续输出带有时间戳的状态日志**（例如：`[14:30:05] 任务状态：运行中`）。  
**给代理的提示**：  
- 在等待脚本返回结果时，请密切关注终端输出。  
- 只要终端仍在输出新的状态日志，说明任务正在正常运行。请不要误判为死锁或无响应。  
- 只有当状态长时间保持不变或脚本停止输出且没有返回结果时，才考虑触发重试机制。  

## 📊 数据输出  
成功执行后，脚本会直接从API响应中解析并打印结果。提取的数据包括：  
- `title`：搜索结果中显示的标题  
- `description`：简短的描述片段（如有提供）  
- `view_count`：结果中显示的观看次数  
- `published_at`：结果中显示的发布时间  
- `url`：结果项的URL  

## ⚠️ 错误处理与重试机制  
在脚本执行过程中，如果发生错误（如网络波动或任务失败），代理应按照以下逻辑处理：  

1. **检查输出内容**：  
   - 如果输出中包含“Invalid authorization”，说明API密钥无效或已过期。此时**不要重试**，应引导用户重新检查并提供正确的API密钥。  
   - 如果输出中不包含“Invalid authorization”，但任务执行仍然失败（例如，输出以`Error:`开头或返回的结果为空），代理应**自动尝试再次执行脚本**。  

2. **重试次数限制**：  
   - 自动重试仅限于**一次**。如果第二次尝试仍然失败，则停止重试，并向用户报告具体的错误信息。  

## 🌟 典型用例  
1. **基于关键词的内容发现**：直接从搜索意图构建主题库和内容数据集。  
2. **竞争对手监控**：搜索竞争对手的品牌名称并提取相关视频。  
3. **内容监控**：定期提取特定行业关键词的搜索结果，了解热门趋势。  
4. **频道研究**：搜索特定领域的频道并收集其URL。  
5. **教程聚合**：查找和提取特定软件或工具的教育视频。  
6. **短视频跟踪**：监控YouTube短视频中的热门标签或主题。  
7. **播放列表提取**：查找特定主题的精选播放列表。  
8. **市场研究**：构建搜索结果的结构化数据集以用于市场分析。  
9. **创作者联系**：寻找特定领域的新兴创作者进行合作。  
10. **观看次数分析**：比较不同关键词的热门视频的观看次数。
