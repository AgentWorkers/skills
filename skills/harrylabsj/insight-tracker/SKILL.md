# 洞察追踪器（Insight Tracker）

## 概述

该功能为 OpenClaw 工作流程提供了结构化的洞察管理机制。它可以捕获会议中出现的观察结果、模式以及学习成果，使其具备可搜索性、可参考性，并便于采取行动。

## 使用场景

在以下情况下使用该功能：
- 用户表示“请记下这一点”或“记住这个发现”；
- 在多个会议中发现了需要追踪的模式；
- 研究或分析揭示了值得保存的发现；
- 用户要求“记录”某些内容以供将来参考；
- 需要将这些洞察与现有知识进行交叉引用。

## 核心概念

### 洞察（Insight）
洞察是指具有以下属性的独立观察结果、模式或学习成果：
- **内容**：实际观察到的内容或发现的结果；
- **来源**：信息来源（会议记录、研究结果或用户输入）；
- **标签**：用于分类的标签；
- **优先级**：重要性等级（高/中/低）；
- **状态**：当前状态（活动/已验证/已归档）；
- **创建时间**：记录创建的时间戳；
- **参考链接**：指向相关洞察或知识的链接。

### 标签（Tags）
常用的分类标签包括：
- `pattern`：重复出现的行为或结构；
- `learning`：新获得的理解或技能；
- `decision`：所做的选择及其理由；
- `risk`：潜在的问题或风险；
- `opportunity`：潜在的改进点或优势；
- `user-preference`：用户特定的偏好；
- `technical`：技术上的发现或限制；
- `process`：工作流程或方法论方面的洞察。

## 输入方式
支持多种形式的洞察输入：
- 直接文本输入；
- 会议记录摘录；
- 记忆文件链接；
- 研究成果。

## 输出结果
- 带日期的洞察记录（Markdown 格式）；
- 带标签的洞察摘要；
- 相关洞察的交叉引用报告；
- 洞察搜索结果。

## 工作流程

### 记录洞察
1. **捕获**：提取核心观察内容；
2. **添加标签**：为洞察内容分配相应的分类标签；
3. **设定优先级**：标明其重要性；
4. **建立链接**：将新洞察与相关洞察关联起来；
5. **保存**：将记录保存为带有日期的文件。

### 查阅洞察
1. **搜索**：可通过标签、关键词、日期或优先级进行查找；
2. **筛选**：根据状态或类别进行筛选；
3. **展示**：以上下文的形式展示匹配的洞察内容。

### 分析洞察
1. **聚类**：将相关的洞察归类在一起；
2. **分析趋势**：识别随时间变化的趋势；
3. **验证**：根据新证据更新洞察内容；
4. **归档**：将过时的洞察标记为已归档状态。

## 命令操作
- **添加洞察**：```
insight add "Content of the insight" --tags pattern,learning --priority high
```
- **列出所有洞察**：```
insight list --tag pattern --since 2024-01-01
```
- **搜索洞察**：```
insight search "keyword" --priority high
```
- **查看洞察详情**：```
insight show <insight-id>
```
- **归档洞察**：```
insight archive <insight-id>
```

## 输出格式示例
- 带日期的洞察记录（Markdown 格式）：```markdown
# Insights - YYYY-MM-DD

## New Insights

### INS-001: Title
- **Content**: The insight content
- **Source**: Session/user/research
- **Tags**: pattern, learning
- **Priority**: high
- **Status**: active
- **Created**: 2024-01-15T10:30:00Z
- **References**: INS-003, knowledge-distillation-2024-01-10

## Summary
- Total insights: 5
- High priority: 2
- New tags: user-preference
```

## 质量要求
- **具体性**：避免使用模糊的表述；
- **记录来源**：务必注明洞察信息的来源；
- **统一标签使用**：遵循标准的标签规范；
- **建立链接**：确保相关洞察之间能够相互关联；
- **定期审查**：及时归档过时的洞察；
- **客观设定优先级**：并非所有洞察都具备高优先级。

## 使用示例：
- “请记录这个洞察：用户更喜欢选项 X 而不是选项 Y”；

- “搜索关于部署问题的洞察”；
- “显示上周所有高优先级的洞察记录”；
- “将洞察 INS-005 归档，因为它已经不再相关”。
