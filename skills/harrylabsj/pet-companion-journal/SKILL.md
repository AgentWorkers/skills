# 宠物伴侣日记

## 概述

使用此功能可以为您的一只或多只宠物创建一个以本地数据为主的数据档案。您可以记录宠物的基本信息、日常点滴、饮食变化、照片、健康状况以及各种提醒事项，确保所有信息按宠物分类存储，便于日后查询。

此功能适用于以下场景：
- 创建或更新宠物档案
- 记录宠物的日常行为、温馨瞬间和成长历程
- 为宠物时间线添加照片及说明文字
- 跟踪宠物的饮食情况、补充剂使用及饮食调整
- 存储宠物的症状、就诊记录、用药信息、疫苗接种和驱虫记录
- 查看即将到期的护理提醒
- 查看宠物的近期健康记录或导出摘要

## 工作流程

### 1. 确定宠物身份
首先明确记录内容所属的宠物。

规则：
- 如果家中只有一只宠物，可以在需要时通过简单确认来确定宠物身份。
- 如果用户有多只宠物且未指明具体是哪只宠物，请在开始操作前询问用户。
- 在查看数据时，优先显示可能的匹配结果；只有在信息不明确时才需要进一步确认。

可以使用 `scripts/pet_manager.py list` 或 `view` 命令来查看现有的宠物信息。

### 2. 创建或更新宠物档案
在创建宠物档案时，首先收集以下基本信息：
- 名字
- 种类
- 出生日期或大致年龄
- 已知的品种
- 已知的性别

可选字段：
- 昵称
- 收养日期
- 是否已绝育
- 毛色特征
- 性格特点
- 备注
- 头像照片的引用路径

使用 `scripts/pet_manager.py create` 或 `update` 命令来创建或更新宠物档案。

### 3. 按类型添加记录
使用 `scripts/record_add.py` 命令将记录保存到对应的宠物文件夹下。

支持的记录类型包括：
- `daily`：普通的日常笔记
- `moment`：温馨的场景、有趣的瞬间或成长里程碑
- `photo`：附带说明文字的照片（可选添加媒体文件）
- `feeding`：宠物的饮食情况、补充剂的使用及饮食调整记录
- `health`：宠物的症状、就诊记录、用药信息、疫苗接种和驱虫记录
- `reminder-note`：与提醒或护理计划相关的备注

如果用户提供了半结构化的文本，可以将文字内容保存在主体部分，并将结构化信息（如症状列表、药物名称或场景标签）存储在 `extra` 字段中。

### 4. 管理提醒事项
使用 `scripts/reminder_manage.py` 命令为每只宠物创建或查看提醒事项。
使用 `scripts/reminder_check.py` 命令来查找即将到期的提醒。

常见的提醒类型包括：
- 生日
- 疫苗接种
- 驱虫
- 体检
- 梳理毛发
- 洗澡
- 补充食物
- 用药
- 后续护理

### 5. 查询和查看记录
使用 `scripts/record_query.py` 命令进行结构化查询。
支持按以下条件进行筛选：
- 宠物
- 记录类型
- 时间范围
- 关键词

使用 `scripts/export_report.py` 命令生成指定时间范围内的摘要报告。

## 输出规则
- 保持回复内容简洁、清晰且条理分明。
- 对于写入操作，需确认记录的内容及其所属宠物。
- 对于查看操作，提供简洁的摘要以及最相关的记录内容。
- 对于健康相关的记录，仅记录和整理相关信息，不得提供兽医诊断结果。
- 如果照片路径或详细信息不明确，应将其标记为备注或进行估算，而非假装信息准确无误。

## 存储规则
默认存储路径：`~/.pet-companion/`

文件结构如下：
- `pets/`：每个宠物对应一个 JSON 档案
- `records/YYYY/MM/`：包含 markdown 格式记录的文件夹（文件前缀为 `.md`，文件内容为 JSON 格式）
- `reminders/`：每个宠物对应的提醒文件（文件扩展名为 `.json`）
- `media/YYYY/MM/`：宠物照片或附件文件
- `reports/`：根据需要生成的摘要报告文件

请参阅以下参考文档：
- `references/data-schema.md`
- `references/query-patterns.md`
- `references/template-examples.md`
- `references/safety-boundaries.md`

## 隐私与安全
此功能仅存储本地数据。未经用户明确许可，不得分享宠物的照片、就诊记录或身份信息。虽然此功能可用于整理宠物健康记录，但它不能替代兽医的诊断或治疗建议。如果用户描述了紧急症状，请建议用户立即寻求兽医帮助。
