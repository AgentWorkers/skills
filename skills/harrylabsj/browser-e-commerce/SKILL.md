# 浏览器商务基础

将此技能作为电子商务任务的通用浏览器工作流程使用。

## 配置文件选择

根据风险和需求选择浏览器配置文件：

- `openclaw`：默认配置，适用于公共浏览、搜索结果、产品页面、截图以及跨平台比较。
- `user`：仅在需要保留用户登录状态时使用（例如购物车、优惠券、订单、地址或会员价格）。
- `chrome-relay`：仅在用户明确要求使用扩展程序的标签页功能时使用。

默认规则：
- 公共商务页面 → `openclaw`
- 已登录的私人页面 → `user`

## 基本工作流程

1. 根据需要执行 `browser status` 和 `browser start` 操作。
2. 在一个稳定的标签页中打开/聚焦目标网站。
3. 在执行任何操作之前先运行 `snapshot` 命令。
4. 优先使用 `snapshot --interactive` 或 `actionable refs`，以避免依赖不稳定的 CSS 表现。
5. 在页面导航、弹出窗口或商品库存发生变化后，重新运行 `snapshot` 命令。
6. 当按钮、商品卡片或价格信息不明确时，使用 `highlight` 功能进行标注。
7. 当结果对后续操作至关重要时，使用截图作为证据。
8. 对于动态变化较大的页面，检查 `requests`、`errors` 和 `response body` 的内容。

## 商务信息提取顺序

对于大多数购物任务，按以下顺序提取信息：

1. 平台
2. 页面类型：搜索结果、产品详情、购物车、订单、优惠券、店铺
3. 标题
4. 当前价格
5. 最终价格、优惠券价格、会员价格
6. 店铺或卖家信息
7. 销售情况、评分、评论数量
8. 运输或配送信息
9. 商品编号（SKU）/规格选项
10. 风险提示或注意事项
11. 链接及截图证据

## 标准页面工作流程

### 搜索结果页面

目标：快速比较多个搜索结果。

操作步骤：
- 打开搜索页面
- 对交互式元素进行截图
- 识别商品卡片、价格信息、店铺名称、标签、促销信息及优惠券
- 将符合条件的商品信息整理到结构化笔记中
- 如果排名结果重要，截图整个搜索结果页面

### 产品详情页面

目标：为购买决策做好准备。

操作步骤：
- 提取商品标题、价格、最终价格、店铺信息、运输方式和服务标识
- 检查商品编号（SKU）/规格选择器
- 记录优惠券、折扣信息、限时促销活动及会员专享价格
- 如果商品规格发生变化，在重新查看价格前重新运行 `snapshot` 命令
- 在推荐购买时，截图产品详情页面作为证据

### 购物车/订单/优惠券页面

目标：安全地使用用户的登录状态。

操作步骤：
- 仅在必要时切换到 `profile="user"` 配置
- 优先执行读取或检查操作，避免执行不可逆的操作（如支付或提交订单）
- 在执行支付或提交订单等不可逆操作前停止操作
- 总结折扣信息、总价、优惠券适用情况以及配送详情

### 地理位置相关的商务页面

目标：根据用户位置判断配送或库存情况。

操作步骤：
- 如果可用，利用地理位置或浏览器上下文信息
- 检查配送区域、预计到达时间（ETA）、店铺可用性或运费是否因位置而变化
- 清晰地标注信息，确保输出结果依赖于用户的位置信息

## 安全注意事项

- 不要完成支付操作。
- 未经明确确认，不要提交最终订单。
- 除非用户明确要求，否则不要更改账户设置、地址或支付方式。
- 对于已登录的页面，优先进行观察和提供购买建议，而非直接执行操作。

## 调试流程

当自动化流程出现异常时：
1. 重新运行 `snapshot` 命令。
2. 使用 `highlight` 功能标记出问题所在的位置。
3. 检查 `errors`（错误信息）。
4. 检查 `requests`（请求数据）。
5. 当可视化用户界面不完整时，通过 `response body` 获取商务相关数据。
6. 仅在必要时使用 `trace`（调试工具）。
