# Naver 搜索技能规范

本文档旨在为 OpenClaw 代理提供指导，帮助其最有效地调用 Naver 搜索功能并分析搜索结果。

## 🎯 技能目的
通过精细的参数控制，获取 Naver 的实时新闻、购物数据、图片以及地点信息。

## 🛠️ 调用规范 (Command Reference)

### 1. 综合聚合器 (Recommended)
当代理需要同时获取多个类别的信息时使用。
```bash
python3 scripts/naver_search.py "<검색어>"
```
- **返回数据**: 最新新闻（3条）、购物趋势（3条）、图片缩略图、视频摘要、热门网页结果。

### 2. 高性能新闻引擎
```bash
python3 scripts/naver_news.py "<검색어>" --sort <sort> --time <time>
```
- `--sort`: `0`（相关性排序）、`1`（最新排序-推荐）、`2`（按时间排序）
- `--time`: `all`（全部）、`1h`（1小时）、`1d`（1天）、`1w`（1周）、`1m`（1个月）
- **提示**: 在分析实时热点时，`--sort 1 --time 1d` 的组合最为有效。

### 3. 购物/图片/视频专用引擎
| 功能 | 脚本 | 特殊选项 |
| :--- | :--- | :--- |
| **购物** | `naver_shopping.py` | 包含价格和销售商信息 |
| **图片** | `naver_images.py` | `--num [n]`（控制结果数量） |
| **视频** | `naver_video.py` | 包含来源和播放时间信息 |

### 4. 预订及地点引擎
当需要获取地点的物理位置或预订链接时使用。
```bash
python3 scripts/naver_booking.py "<맛집/장소명>"
```
- **返回数据**: 商家名称、地址、类别、Naver 预订产品信息及链接。

## 📊 输出格式控制
在所有命令后添加 `--format` 选项，以选择数据格式：
- `compact`（默认）：适合代理阅读的简洁摘要文本。
- `json`：适用于需要在代码层面进行详细分析的原始 JSON 数据。

## 🔐 安全性与环境变量
- 本技能需要 **`SERPAPI_API_KEY`** 环境变量。
- 如果环境变量未设置，请在技能根目录下的 `.env` 文件中输入该密钥。

---
**注意事项**：根据 Naver 引擎的特性，在搜索词后直接添加“新闻”、“预订”等关键词可以获得更准确的结果。（`naver_news.py` 和 `naver_booking.py` 中已包含部分自动优化逻辑。）
