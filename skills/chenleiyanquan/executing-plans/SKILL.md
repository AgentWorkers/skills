# 执行计划

## 概述

加载计划，进行严格审查，分批执行任务，并在批次之间提交报告以供审核。

**核心原则：** 分批执行，并设置检查点以便架构师进行审核。

**开始时声明：** “我将使用‘执行计划’技能来实施这个计划。”

## 执行流程

### 第一步：加载并审查计划
1. 读取计划文件
2. 严格审查计划内容，识别任何问题或疑虑
3. 如果有疑虑：在开始执行前与人类协作者沟通
4. 如果没有疑虑：创建待办事项（TodoWrite），然后继续执行

### 第二步：执行任务
**默认执行前3个任务**

对于每个任务：
1. 将任务标记为“进行中”（inProgress）
2. 严格按照计划中的步骤操作
3. 按照规定执行验证步骤
4. 将任务标记为“已完成”（completed）

### 第三步：提交报告
当所有任务执行完毕并完成验证后：
- 展示已实施的内容
- 显示验证结果
- 说：“准备接受反馈。”

### 第四步：继续执行
根据反馈：
- 如有需要，进行修改
- 执行下一批任务
- 重复此过程，直到所有任务全部完成

### 第五步：完成开发工作
当所有任务都已完成并经过验证后：
- 宣布：“我将使用‘完成开发分支’技能来完成这项工作。”
- **所需子技能：** 使用“完成开发分支”（finishing-a-development-branch）技能
- 按照该技能的步骤进行测试、展示选项并做出选择

## 何时停止并寻求帮助

**在以下情况下立即停止执行：**
- 在执行过程中遇到障碍（缺少依赖项、测试失败、指令不明确）
- 计划中存在严重问题，导致无法继续执行
- 不理解某条指令
- 验证多次失败

**遇到问题时，请寻求澄清，不要自行猜测。**

## 何时重新审查早期步骤
**在以下情况下返回步骤1进行重新审查：**
- 协作者根据你的反馈更新了计划
- 需要重新考虑整个执行策略

**不要强行克服障碍**——遇到问题时，请立即停止并寻求帮助。

## 注意事项：
- 首先严格审查计划内容
- 严格按照计划步骤执行
- 不要跳过验证步骤
- 在计划要求时使用相应的技能
- 在批次之间，只需提交报告并等待下一步指示
- 遇到障碍时立即停止，不要自行猜测
