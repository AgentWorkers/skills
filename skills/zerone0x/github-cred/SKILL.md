# GitHub Cred

用于分析GitHub用户的贡献质量。

## 触发条件

- `github cred <username>`
- `analyze <username>'s github`

---

## 四级评分系统

### 第0级 — 拥有的高星级仓库（30%）

| 指标 | 分数 |
|--------|-------|
| 拥有≥10k星仓库 | +30 |
| 拥有≥5k星仓库 | +20 |
| 拥有≥1k星仓库 | +10 |

*上限：30分。分数优先给项目创建者*

### 第1级 — 外部贡献（30%）

| 指标 | 分数 |
|--------|-------|
| 在≥100k星仓库中合并的PR | 每个PR +4分 |
| 在≥10k星仓库中合并的PR | 每个PR +2分 |
| 在≥1k星仓库中合并的PR | 每个PR +1分 |
| 非平凡的贡献（代码行数超过20行） | 分数×1.5 |

*上限：30分*

### 第2级 — 代码质量（20%）

| 指标 | 分数 |
|--------|-------|
| 合并成功率 ≥70% | +10分 |
| 合并成功率 ≥50% | +7分 |
| 代码贡献占比 ≥80% | +5分 |
| 贡献的仓库数量 ≥10个 | +5分 |

*上限：20分*

### 第3级 — 社交影响力（20%）

| 指标 | 分数 |
|--------|-------|
| 关注者数量 ≥5k | +20分 |
| 关注者数量 ≥2k | +15分 |
| 关注者数量 ≥1k | +12分 |
| 关注者数量 ≥500 | +8分 |
| 关注者数量 ≥200 | +5分 |
| 关注者数量 ≥100 | +3分 |
| 关注者数量 <100 | +1分 |

---

## 评分等级

| 等级 | 分数 | 描述 |
|-------|-------|-------------|
| S | 95+ | 顶级项目创建者、核心贡献者及具有较高影响力的用户 |
| A+ | 85-94 | 项目所有者或顶级贡献者 |
| A | 70-84 | 经验丰富的开发者，有实际贡献 |
| B | 50-69 | 积极贡献的开发者 |
| C | 30-49 | 初级开发者，有一定贡献 |
| D | <30 | 新手 |

---

## 输出格式

```
📊 GitHub Cred: @username

🏆 XX/100 (Grade)

▸ Tier 0 Owned Repos (XX/30)
  👑 repo1 ⭐XXk

▸ Tier 1 External Contributions (XX/30)
  🔨 org/repo ⭐XXk × N PRs

▸ Tier 2 Output Quality (XX/20)
  📈 Merge rate XX% | Code ratio XX%

▸ Tier 3 Social Influence (XX/20)
  👥 X,XXX followers
```

---

## 达到A+等级的两种途径

- **项目创建者路径**：创建一个拥有10k星以上的项目。
- **贡献者路径**：对多个拥有100k星以上的项目做出深度贡献。

结合这两种途径的贡献效果最佳。
