# 🎨 UI/UX 设计技能

**这是一项专门针对 AI 代理的技能，用于构建高保真度的数字界面。**  
UI/UX 设计技能能够将产品需求转化为技术设计规范，用于生成高保真度的原型图、线框图和设计系统。

## 核心能力

1. **原子设计协调**：从基本组件（如按钮）到整体界面结构（如页眉）进行系统化的设计，确保界面的一致性。  
2. **平台特定布局**：为移动设备（iOS/Android）和网页（SaaS/电子商务平台）设计响应式布局。  
3. **设计系统集成**：规定字体大小、间距规则以及颜色方案（Hex/HSL 格式）。  
4. **遵循可用性原则**：确保设计符合尼尔森的 10 条可用性启发式原则。

## 🏗️ 技术规范  

### 1. 意图映射表  

| 创意意图 | 风格 | 布局模式 | 重点 |
| :--- | :--- | :--- | :--- |
| **企业级 SaaS** | 现代/简洁 | 仪表盘/网格布局 | 数据密度展示 |
| **消费类应用** | 玻璃形态设计 | F 形式/卡片布局 | 视觉效果突出 |
| **电子商务** | 极简主义风格 | Z 形式布局/产品网格 | 提升转化率 |
| **作品集展示** | 原始主义风格 | 非对称布局 | 强化品牌识别度 |
| **工具类应用** | 新形态设计 | 控制面板设计 | 提供触觉反馈 |

### 2. 设计元素与变量  
- **字体**：技术类应用使用几何风格的 Sans 字体（如 Inter/Roboto）；奢侈品类应用使用衬线字体（如 Playfair）。  
- **间距**：采用 8pt 的网格系统来保持一致的间距效果。  
- **颜色**：使用高对比度的颜色方案（符合 WCAG 2.1 标准）。  
- **层次感**：通过阴影效果实现深度感，而非纯平面设计。

## 🧠 提示优化协议（代理使用说明）  

**在调用脚本之前，代理必须将用户需求转化为具体的设计规范：**  
1. **基本组件**：明确指出需要使用的具体设计元素（如玻璃形态的按钮、半径为 4px 的输入框、图标式侧边栏等）。  
2. **界面层次结构**：根据页面内容选择合适的布局模式（如 F 形式布局用于内容密集型页面，Z 形式布局用于登录页，网格布局用于仪表盘）。  
3. **字体与设计元素**：指定使用的字体类型（如 Inter Sans-serif）和间距规则（8pt 网格系统），以及符合 WCAG 标准的高对比度颜色。  
4. **避免使用拟物化设计**：确保设计遵循平面设计或玻璃形态设计原则，避免生成类似屏幕截图的界面效果。

## 🚀 使用 UI 设计工具的流程  

### 第一步：定义产品需求  
向代理提供功能列表和目标用户群体信息。  

### 第二步：调用脚本  
`generate-mockup.sh` 脚本会根据设计系统的知识来生成相应的 UI/UX 原型图。  

```bash
# Designing a Fintech Mobile App
bash scripts/generate-mockup.sh \
  --desc "crypto wallet home with price charts" \
  --platform mobile \
  --theme dark \
  --style glassmorphism
```

## ⚠️ 限制与注意事项  

- **设备真实性**：**必须遵守**——不得展示真实的手机、物理设备或办公环境；仅生成纯 UI/UX 原型图。  
- **可访问性**：避免在明亮背景下使用低对比度的文本。  
- **设计复杂性**：确保单个页面中的所有设计元素保持一致性。  
- **文本渲染**：使用 Flux 工具来渲染清晰的标题；对于正文内容，使用占位符文本进行展示。
