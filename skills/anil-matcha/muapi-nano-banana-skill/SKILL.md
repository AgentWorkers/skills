# 🍌 Nano-Banana 专家技能（Gemini 3 风格）

**这是一项专为 AI 代理设计的技能，用于利用“基于推理”的图像生成技术。**  
该技能基于谷歌 Gemini 3（Nano Banana Pro）的先进提示架构，超越了简单的关键词堆砌，实现了结构化、基于逻辑的创意指令生成。

## 核心能力

1. **基于推理的提示生成**：利用自然语言逻辑来定义物体物理特性、光照效果及空间关系。  
2. **结构化的创意指令**：遵循“完美提示”公式：`主题 + 动作 + 上下文 + 构图 + 光照`。  
3. **文本渲染精度**：明确指定文本的排版和显示方式，以确保文本的可读性。  
4. **上下文关联**：通过“搜索关联”逻辑（模拟实现）将生成的图像与现实世界场景相匹配。

## 🏗️ 技术规格

### 1. “完美提示”公式

| 组件 | 描述 | 示例 |
| :--- | :--- | :--- |
| **主题** | 详细的实体描述 | “一个露出铜线的斯多葛风格机器人咖啡师” |
| **动作** | 动态交互行为 | “以机械般的精准度制作拿铁拉花” |
| **上下文** | 环境与氛围 | “午夜时分，位于霓虹灯照亮的赛博朋克咖啡馆内” |
| **构图** | 相机与镜头选择 | “特写镜头，85mm 镜头，f/1.8 光圈” |
| **光照** | 情感氛围与光线方向 | “立体感强的蓝色边缘光，温暖的咖啡馆氛围” |
| **风格** | 美学导向 | “电影级画质，4K 分辨率” |

### 2. 高级功能  
- **负面约束逻辑**：例如，使用“确保主体眼睛的焦点清晰”而非“避免模糊”。  
- **身份一致性**：（模拟实现）在不同场景中保持角色的面部特征一致性。  
- **文本整合**：使用双引号标注特定文本：`标志上写着“24小时营业”`。

## 🧠 提示优化协议（代理使用指南）

**在调用脚本之前，代理必须将用户的提示转换为基于逻辑的创意指令：**  
1. **避免使用关键词堆砌**：不要使用“8K、杰作、超高清”等模糊描述，而应使用完整的、具体的句子。  
2. **描述物理交互**：明确说明元素之间的相互作用（例如：“水晶碎片的光线在黑曜石地板上投射出复杂的图案”）。  
3. **文本精确性**：如果需要包含文本，请精确描述其内容（例如：“标志上用褪色的衬线字体写着‘商店名称’”）。  
4. **光学指令**：指定镜头的使用方式（如：*浅景深（f/1.8）*、*微距镜头*、*变形光晕效果*）。

## 🚀 使用 Nano-Banana 的步骤  

### 第一步：定义创意逻辑  
为代理提供主题和具体场景描述。  

### 第二步：调用脚本  
`generate-nano-art.sh` 脚本会将这些逻辑转化为符合 Gemini 3 风格的结构化提示。  

```bash
# Generating a reasoning-driven image
bash scripts/generate-nano-art.sh \
  --subject "a glass chess piece" \
  --action "shattering into liquid shards" \
  --context "on a obsidian table" \
  --style "macro photography"
```
