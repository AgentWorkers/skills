## 使用场景

当家庭成员需要添加、删除或查看由多人共同维护的共享购物清单时，可以使用此功能。该清单按商店（包含地址）进行分类。主要功能是帮助用户确定需要购买的商品及其购买地点，而不涉及餐食规划。

## 架构

共享数据存储在用户配置的本地路径中，所有家庭成员均可访问。具体文件模板请参见 `memory-template.md`。

## 启动流程

每次启动时，请按以下顺序执行步骤：

### 第1步 — 确认用户身份
1. 从OpenClaw内存中获取用户设置的名称（键：`family_grocery_user`）。
2. 如果未找到名称 → 拒绝访问，并提示用户：“未在该设备上找到该用户名称。请联系管理员为您配置用户名。”然后停止操作。

### 第2步 — 确定共享路径
1. 从OpenClaw内存中读取共享路径（键：`family_grocery_path`）。
2. 如果路径不存在 → 表示这是首次使用该功能，请进入 **管理员初始化流程**。

### 第3步 — 加载配置并验证访问权限
1. 读取 `[shared-path]/config.json` 和 `[shared-path]/users.md` 文件。
2. 在 `users.md` 中查找步骤1中获取的用户名。
3. 如果未找到用户名 → 拒绝访问，并提示用户：“`[用户名]` 未在共享购物清单中。请联系管理员为您添加。”
4. 如果找到用户名 → 继续操作。同时记录用户是“管理员”还是“普通成员”。

---

### 管理员初始化流程

仅当OpenClaw内存中不存在共享路径时执行此流程：

1. 询问用户：“应使用哪个路径来存储家庭购物数据？（例如：/Users/Shared/grocery）”
2. 创建相应目录：`mkdir -p [路径]`
3. 根据 `memory-template.md` 模板初始化以下文件：`config.json`、`users.md`、`list.md`、`history.md`
4. 将当前用户设置为“管理员”并保存到 `users.md` 中。
5. 将路径保存到OpenClaw内存中（键：`family_grocery_path`）。
6. 提示用户：“设置完成。您现在是管理员，请将路径 `[path]` 分享给其他家庭成员，以便他们可以连接自己的设备。”

## 功能范围

- 仅用于管理共享购物清单（添加、删除、查看商品）
- 跟踪商店及其地址
- 根据用户输入、商品类别或默认设置将商品分配到相应商店
- 通过网页搜索查询商品可用性和商店地址（如支持该功能）
- 管理家庭成员的访问权限
- 根据请求显示商品变更历史记录

**注意：**  
- 本功能 **不负责** 实际购物或下订单的操作，也不提供餐食规划或食谱推荐服务。  
- **不读取或写入** 位于 `[shared-path]` 之外的文件。  
- **不访问实时商店库存系统**。  
- **在清单更新时不会发送通知**。

## 核心规则

### 1. 每次启动时务必重新加载配置和用户信息
- 绝不在不同会话间缓存配置信息。每次启动时都必须重新读取 `config.json` 和 `users.md`，以确保获取其他家庭成员所做的更改。

### 2. 用户身份信息从OpenClaw内存中获取
- **切勿** 询问用户手动输入姓名——始终从OpenClaw内存中读取。如果找不到姓名，则拒绝访问。
- 如果共享路径已存在，则无需再次询问。
- 首次使用时需保存该路径信息，以避免后续重复询问。
- 用户姓名必须与管理员在 `users.md` 中注册的姓名完全匹配（不区分大小写）。

### 3. 添加商品时必须先确定商店
- 添加商品的流程包括：用户输入 → 商品类别映射 → 选择默认商店。
- 详情请参阅 `lists.md`，其中包含重复商品检测和网页搜索功能。

### 4. 添加商品前进行重复检查
- 在添加任何商品之前，先通过不区分大小写的模糊匹配方式在 `list.md` 中查找是否存在相同商品。如果找到相同商品，需告知用户并询问是否合并数量或单独添加。合并操作会记录在 `history.md` 中。

### 5. 所有写入操作都必须记录用户信息
- `list.md` 和 `history.md` 中的每条记录都必须包含用户姓名和ISO时间戳。

### 6. 网页搜索为可选功能——优先使用默认方式
- 如果支持网页搜索：利用它来验证商店地址和商品可用性。
- 如果搜索功能不可用：直接进行操作，并向用户说明这一限制。
- **切勿** 因等待搜索结果而阻塞操作。

### 7. 商店名称显示时必须包含地址
- 显示清单时，商店名称的格式应为：`🏪 [商店名称] ([完整地址]) — [营业时间]`
- 如果商店营业时间未知，先通过网页搜索查询，用户确认后保存到 `config.json`。如果用户选择忽略营业时间，则在显示时省略该信息。这样可以兼容旧版本配置文件中的缺失数据。

### 8. 仅限管理员执行的操作
| 操作 | 执行者 |
|--------|--------------|
| 添加新用户 | 仅限管理员 |
| 设置默认商店 | 仅限管理员 |
| 更新商品类别与商店对应关系 | 仅限管理员 |
| 添加新商店 | 任何用户 |
| 添加/删除清单商品 | 任何用户 |
| 查看清单 | 任何用户 |
| 查看历史记录 | 任何用户 |

**如果用户尝试执行仅限管理员的操作，系统会提示：“此操作仅限管理员执行。”**

## 常见错误及解决方法
- **错误1：** 从过时的缓存配置中读取数据 → 每次启动时务必重新加载 `config.json` 和 `users.md`。
- **错误2：** 在未确定商店信息的情况下直接添加商品 → 在写入 `list.md` 之前必须先确认商店信息。
- **错误3：** 显示清单时未按商店分组 → 必须按商店名称对商品进行分组显示。
- **错误4：** 未记录操作信息 → 每次写入操作都必须包含用户姓名和时间戳。
- **错误5：** 网页搜索功能不可用时阻塞操作 → 必须优雅地处理这种情况，继续执行操作而不依赖搜索结果。
- **错误6：** 每次会话都要求用户输入姓名或路径 → 这些信息应在首次使用时已保存在OpenClaw内存中。
