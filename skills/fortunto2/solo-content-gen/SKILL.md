# /content-gen

根据项目的 PRD（产品需求文档）或 README（项目说明文件），生成一套推广内容。该工具会生成三份可直接使用的推广素材：LinkedIn 发布文、Reddit 回答草稿以及 Twitter/X 社交媒体帖子。如需视频脚本，请使用 `/video-promo` 命令。

## MCP 工具（如可用）

- `kb_search(query)` — 查找相关的方法论和原则
- `project_info(name)` — 获取项目详情、技术栈及描述
- `project_code_search(query, project)` — 在代码库中查找关键功能

如果 MCP 工具不可用，可改用全局搜索（Glob）结合 grep 命令以及手动阅读文件的方法。

## 步骤

1. **从 `$ARGUMENTS` 中解析项目信息**：
   - 如果提供项目名称：在 `docs/prd.md`、`README.md` 或 `CLAUDE.md` 中查找 PRD；如果提供项目路径，则直接读取该路径下的文件；如果未提供任何信息，则通过 `AskUserQuestion` 询问：“需要为哪个项目生成推广内容？”

2. **从 PRD/README 中提取核心信息**（同时读取文件）：
   - **问题**：该产品解决了什么问题？（1 句话）
   - **解决方案**：产品具体能做什么？（1 句话）
   - **目标用户群体（ICP）**：产品的目标用户是谁？
   - **关键特性**：产品最突出的 3-5 个独特特性
   **独特优势**：产品与其他产品相比的优势是什么（如有必要，可参考 `research.md` 文件）
   - **用户反馈/数据支持**：产品的用户数量、下载量等数据

3. **制定内容框架**：
   在生成内容之前，先明确以下要点：
   - **吸引注意的引言**：什么能吸引读者的兴趣（痛点或令人惊讶的数据）
   **说服力**：为什么读者应该相信你的内容？（通过演示、数据或权威来源）
   **行动号召（CTA）**：读者应该采取什么行动（例如：尝试使用产品、注册、下载等）
   **语气**：根据目标用户群体的特点选择合适的语气（专业/非正式/技术性/友好的）

4. **生成推广内容**：
   ### 4a. LinkedIn 发布文
     - 格式：吸引注意的引言 → 产品使用前后的变化 → 深入分析 → 行动号召
     - 最长 1300-1500 字
     - 第一行是吸引注意的引言（显示在“查看更多”之前）
     - 使用换行符提升可读性
     - 以问题结尾以激发读者的互动

   ### 4b. Reddit 回答草稿
     - 内容应提供有用的信息，并自然地提及产品
     - 重点在于内容的实用性，产品的介绍为次要内容
     - 建议在 3-5 个相关的 Reddit 子版块发布
     - 以回答他人问题的形式撰写
     - 明确标注“本人负责编写此内容”

   ### 4c. Twitter/X 社交媒体帖子（5-7 条）
     - 格式：吸引注意的引言 → 问题 → 解决方案 → 产品演示 → 用户反馈 → 行动号召
     - 每条推文最长 240-280 字
     - 第一条推文为独立的引言（无需依赖其他推文）
     - 最后一条推文包含链接和行动号召
     - 适量使用表情符号（每条推文最多 1-2 个）

5. **将生成的内容保存到 `docs/content-pack.md` 文件中**：

   ```markdown
   # Content Pack: {Project Name}

   **Generated:** {YYYY-MM-DD}
   **ICP:** {target persona}
   **Core message:** {one-line value prop}

   ## Messaging Framework
   - **Hook:** {hook}
   - **Proof:** {proof}
   - **CTA:** {cta}

   ---

   ## 1. LinkedIn Post
   {ready to copy-paste}

   ## 2. Reddit Draft
   **Target subreddits:** r/..., r/..., r/...
   {answer draft}

   ## 3. Twitter/X Thread
   {numbered tweets}

   > For video script, run `/video-promo {project}`.

   ---
   *Generated by /content-gen. Edit before publishing.*
   ```

6. **输出结果**：打印核心信息以及生成的所有推广内容列表。

## 注意事项

- 所有内容均为草稿，发布前请务必审核
- Reddit 回答草稿应具有实用性，避免过度宣传
- 如需视频脚本，请使用 `/video-promo` 命令（该命令负责视频的生成）
- 产品有重大更新时，请重新生成推广内容
- 该工具在项目有详细 PRD 时效果最佳；若没有 PRD，可使用 README 文件作为替代

## 常见问题及解决方法

### 问题：找不到 PRD
**原因**：项目没有 `docs/prd.md` 或 `README.md` 文件。
**解决方法**：提供包含 PRD 的项目路径，或先运行 `/validate` 命令生成 PRD 文件。

### 内容过于泛泛
**原因**：PRD 中缺乏具体的产品特点或目标用户信息。
**解决方法**：确保 PRD 中包含详细的问题描述、目标用户群体以及产品的独特优势；如有必要，可参考 `research.md` 文件了解市场情况。

### 语气不符合目标受众
**原因**：PRD 中未明确目标受众。
**解决方法**：在 PRD 中明确指定目标受众：LinkedIn 适合专业用户，Reddit 适合非正式或技术型用户，Twitter 适合简洁明了的表达方式。
