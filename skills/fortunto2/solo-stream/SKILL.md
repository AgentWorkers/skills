# /stream

这是一个交互式向导工具，可帮助用户按照STREAM的六层框架来分析任何决策。它专为需要在不确定性下做出高风险选择的创业者设计。

## 步骤

1. **从 `$ARGUMENTS` 中解析决策内容**。如果 `$ARGUMENTS` 为空，系统会询问：“您想分析哪个决策或困境？”

2. **加载框架相关信息**（可选功能）：
   - 如果MCP支持`kb_search`功能，可以使用 `kb_search(query="STREAM framework", n_results=3)` 来获取完整的框架详情。
   - 否则，框架的相关信息会直接显示在界面中。

3. **交互式地分析六个层面**：
   - 对于每个层面，系统会解释其概念，通过 `AskUserQuestion` 提出问题以帮助用户明确思路，然后提供评估结果。

### 第一层 — 认识论（知识）
   - 这个决策是否属于您的职责范围？
   - 有哪些假设尚未得到验证？
   - 什么因素会改变您的决定？
   - **问题示例**：“关于这个领域，您有哪些确定的了解？您做了哪些假设？”

### 第二层 — 时间（时间维度）
   - 结果的实现时间范围是多久？
   - 这个决策在10年后是否仍然重要（符合Lindy法则）？
   - 是否可以撤销这个决策？（例如：在6个月内）
   - **问题示例**：“您的时间规划是怎样的？这是一个为期3个月的实验，还是一个需要3年才能完成的项目？”

### 第三层 — 行动（最小可行方案）
   - 最小的可行步骤是什么？
   - 这个决策可能带来的次要影响是什么？
   - 您能在周末内完成这个决策的测试吗？
   **问题示例**：“您能在1-2周内完成的最小可行版本是什么？”

### 第四层 — 风险与回报
   - 这个决策的收益是否不对称（小风险，大回报）？
   - 在最坏的情况下，您能否承受损失？
   - 不采取这个决策的机会成本是多少？
   **问题示例**：“最糟糕的现实结果是什么？您在财务和情感上能否承受这个结果？”

### 第五层 — 社交影响（网络效应）
   - 这个决策会对您的人际关系网络产生什么影响？
   - 这个决策是否具有网络效应或传播潜力？
   - 它能否帮助您建立社会资本？
   **问题示例**：“除了您自己，还有谁会从中受益？谁可能会受到伤害？”

### 第六层 — 人生价值（终极考量）
   - 这个决策是否值得您有限的生命时间？
   - 它是否符合您的价值观？
   - 如果您只剩下5年的时间，您还会选择尝试这个决策吗？
   **问题示例**：“如果您只剩下5年的时间，您还会选择做这件事吗？”

4. **综合评估结果**：
   - 为每个层面打分（1-10分）。
   - 计算整体STREAM评分（加权平均值）。
   - **决定方向**：**继续**（评分 > 7） / **暂停**（5-7） / **放弃**（< 5）。
   - 需要重点关注的主要风险。
   - 推荐的下一步行动。

5. **记录决策内容**（可选）：系统提供保存决策记录的功能。如果用户希望保存：
   - 可将记录保存到 `3-inbox/`（如果使用solopreneur知识库）或 `docs/decisions/`（适用于任何项目）。
   - 保存格式：日期 + 决策摘要 + 评估结果。

6. **输出结构化的决策日志**：
   ```
   ## Decision: [topic]

   **Date:** [today]
   **Framework:** STREAM 6-layer

   ### Analysis Summary
   | Layer | Score | Key Finding |
   |-------|-------|-------------|
   | Epistemological | X/10 | ... |
   | Temporal | X/10 | ... |
   | Action | X/10 | ... |
   | Stakes | X/10 | ... |
   | Social | X/10 | ... |
   | Meta | X/10 | ... |

   **Overall STREAM Score: X/10 — GO/PAUSE/NO-GO**

   ### Recommendation
   [Clear recommendation with reasoning]

   ### Next Actions
   1. [action]
   2. [action]
   3. [action]
   ```

## 常见问题

### 问题显得过于抽象
**原因**：决策不够具体。
**解决方法**：从具体的决策陈述开始，例如：“在当前情况下，我应该[采取某个行动]吗？”，而不仅仅是“我应该做什么？”

### 评分结果不正确
**原因**：所有层面的权重相同，但实际上某些层面对决策的影响更大。
**解决方法**：整体评分只是一个参考值。请根据实际情况重点关注与决策最相关的层面（例如，财务决策时关注“风险与回报”层面，人生选择时关注“人生价值”层面）。

### 想保存决策记录但不知道该保存在哪里
**原因**：决策内容超出了solopreneur知识库的存储范围。
**解决方法**：可以将记录保存到项目的 `docs/decisions/` 文件夹中，或者如果使用solopreneur知识库，则保存到 `3-inbox/` 文件夹中。如果需要，可以创建相应的文件夹。
