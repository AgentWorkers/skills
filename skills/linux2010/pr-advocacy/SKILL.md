# PR倡导技能（PR Advocacy Skill）

## 目的  
积极监控、响应并推动Pull Request（PR）的审核流程，确保其顺利完成。该技能有助于防止PR陷入审核停滞状态，并及时处理维护者的反馈。

## 核心职责  

### 1. PR状态监控  
- 每4小时检查一次PR的状态  
- 监控持续集成/持续部署（CI/CD）的检测结果  
- 跟踪审阅者的评论及提出的修改建议  
- 区分会阻碍审核流程的反馈与不会造成阻碍的反馈  

### 2. 专业响应机制  
- 在24小时内回复所有审阅者的反馈  
- 对技术性问题提供清晰、专业的解答  
- 对合理的修改建议立即执行  
- 在不同意审阅者的意见时，礼貌地说明理由  

### 3. 主动解决问题  
- 立即解决CI/CD系统出现的故障  
- 主动处理合并冲突  
- 代码变更时更新相关文档  
- 为审阅者指出的边缘情况添加测试用例  

### 4. PR健康状况评估  
- **绿色状态**：所有检测通过，无待审核的PR → 等待合并  
- **黄色状态**：存在轻微问题，但检测通过 → 回复审阅者并等待进一步处理  
- **红色状态**：检测失败或有重大问题 → 需立即采取行动  
- **过期状态**：48小时以上无任何活动 → 主动联系审阅者  

## 实施指南  

### 状态检查分析  
```bash
# Check PR status and checks
gh pr view <PR_NUMBER> --json statusCheckRollup,reviewDecision

# Common check statuses:
# - SUCCESS: All good
# - FAILURE: Needs immediate fixing  
# - PENDING: Still running
# - NEUTRAL: Non-blocking (like coverage)
```  

### 审阅者反馈回复模板  
```
Thanks for the review! 

For [specific comment]:
- [Action taken or explanation]
- [Technical justification if disagreeing]

I've [implemented the change / pushed an update / clarified the approach] in [commit reference].

Let me know if you need any additional changes!
```  

### 常见PR问题及解决方法  

#### CI/CD故障  
- **代码格式问题**：先在本地运行`npm run lint -- --fix`进行格式检查  
- **测试失败**：在本地重现问题，并添加缺失的测试用例  
- **构建失败**：验证依赖关系，检查Node.js版本的兼容性  

#### 代码审核反馈  
- **代码风格问题**：遵循项目的`.editorconfig`文件和代码风格指南  
- **架构相关问题**：参考代码库中的现有模式  
- **安全问题**：优先处理安全漏洞，并添加相应的验证机制  

#### 文档问题  
- **文档缺失**：添加或更新相关文档  
- **格式不一致**：使用项目规定的Markdown模板  
- **示例过时**：根据新的实现情况更新示例代码  

## 自动化集成  

### 集成心跳监控系统  
- 将PR状态信息整合到每日汇总报告中  
- 对超过48小时无活动的PR进行升级处理  

### 记录问题解决过程  
- 在工作区中维护PR跟踪列表  
- 记录问题解决的结果，以供后续参考  
- 将成功的处理方式更新到核心文档中  

## 成功指标  
- **响应时间**：对审阅者的反馈在24小时内作出回应  
- **解决率**：90%以上的PR在7天内被合并  
- **返工率**：初次审核后需要重大修改的PR占比低于10%  
- **维护者满意度**：维护者对沟通质量的反馈积极  

## 最佳实践  

### 沟通  
- 回复简洁明了，同时保证信息全面  
- 使用代码块进行技术性解释  
- 在讨论代码变更时引用具体的行号  
- 感谢审阅者所花费的时间和提供的专业知识  

### 技术要求  
- 保证所做的修改最小化且目标明确  
- 在可能的情况下保持代码的向后兼容性  
- 严格遵循项目规范  
- 推送代码变更前进行彻底的测试  

### 专业素养  
- 不要与审阅者争论，努力理解他们的观点  
- 快速承认错误并立即修复  
- 在适当的情况下提供帮助  
- 对成功的合并表示祝贺，并感谢维护者的支持
