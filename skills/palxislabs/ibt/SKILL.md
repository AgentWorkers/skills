# Intention → Behavior Transformer (IBT)

使用此技能可以在严格控制循环的情况下提升执行质量，同时保持您现有的语音和个性特征。

## 核心流程

解析（Parse）→ 计划（Plan）→ 执行（Commit）→ 行动（Act）→ 验证（Verify）→ 更新（Update）→ 停止（Stop）

在每个步骤中，选择能够最大化以下价值的行为：

`效用（Utility） - λ*风险（Risk） - μ*努力（Effort）`（受约束条件限制）

## 这种方法的好处

- 在多步骤任务中提高可靠性
- 减少虚假的完成声明（即在没有证据的情况下声称任务已完成）
- 在验证失败时能够清晰地处理差异
- 适用于各种模型（在程序层面实现一致性）

## 不要这样做

- 不要用僵化的模板来替代您的个性特征在每次回复中的表现。
- 对于简单的聊天内容，不要使用结构化的标题。
- 不要声称执行了实际上并未执行的工具操作。

## 推荐的操作模式

1. 将您的个性规则保存在 `SOUL.md`（或等效文件）中。
2. 将 IBT 规则保存在 `IBT.md` 中。
3. 仅对复杂、高风险或多步骤的任务使用结构化的格式。
4. 对于简单的任务，使用简洁的格式。

## 建议使用的文件

- `IBT_TEMPLATE.md` — 可直接使用的 IBT 规则模板
- `SOUL_PATCH.md` — 最小的补丁文件，用于将 IBT 与您的身份特征关联起来，同时不改变您的语音风格
- `AGENTSPATCH.md` — 启动时需要读取的补丁文件（如果存在 `IBT.md`，则读取该文件）
