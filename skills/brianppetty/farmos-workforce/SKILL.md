# FarmOS 劳动力管理

包括员工资料、考勤记录、技能跟踪、员工请求以及团队日程安排。

## 使用场景

**该功能支持的功能：** 考勤记录（签到/签出）、日程安排、员工可用性、休假申请、员工资料、加班记录以及工资单导出。

**常用语句：** “我需要[某天]的休假”、“谁在上班”、“安排日程”、“签到/签出”、“谁已经签到？”、“这周有加班吗？”、“有哪些待处理的休假申请？”、“员工列表”、“CDL认证”

**不支持的功能：** 任务分配或工作订单（请使用 farmos-tasks 功能）、设备调度或维护（请使用 farmos-equipment 功能）、薪资标准或财务相关问题（仅限管理员通过 farmos-finance 功能处理）。

**最低有效输入：** 任何与日程安排、员工可用性或上班时间相关的内容。例如：“我需要周五的休假”即可。

## 数据完整性要求：**

1. **必须明确说明返回的员工数量、考勤记录或请求的数量**：例如：“找到 8 名在职员工” 或 “本薪资周期内有 12 条考勤记录”。
2. **考勤记录查询需要指定时间范围**——务必包含 `start_date` 和 `end_date` 参数，以便获取指定时间段内的完整信息。
3. **如果某个 API 端点出现故障，请如实报告**——不要默默地显示空结果（如“没有员工”或“没有考勤记录”）。
4. **在导出工资单之前，请确认日期范围覆盖了整个薪资周期。**

## 认证要求

该功能需要使用 JWT 令牌来访问受保护的 FarmOS 端点。

**获取 JWT 令牌的方法：**
```bash
TOKEN=$(~/clawd/scripts/farmos-auth.sh manager)
```

**角色权限：** 查看 `~/.clawdbot/farmos-users.json` 文件以确定发送者的角色：
- **管理员/经理：** 可查看所有员工数据、工时记录及工资信息。
- **员工：** 仅能查看自己的资料和考勤记录。请使用员工专属的令牌进行查询，以确保 API 能正确识别用户权限。

## API 基础地址：**
http://100.102.77.110:8006

## 集成端点（无需认证）：

### 员工列表（用于任务分配）
GET /api/integration/employees
返回：所有员工的 ID、姓名和角色信息，可用于任务分配选择。

### 在职员工列表
GET /api/integration/employees/active
返回：当前在职的员工列表。

## 需要认证的 API 端点（必须使用 JWT 令牌）：

### 员工资料
GET /api/employees — 显示所有员工信息（管理员可查看）
GET /api/employees/{id} — 查看特定员工的详细信息（管理员或本人可查看）
GET /api/employees/me — 查看当前用户的个人资料

### 考勤记录
POST /api/time/clock-in — 签到（可选上传 GPS 数据）
POST /api/time/clock-out — 签出
GET /api/time/status — 获取用户的当前考勤状态
GET /api/time/entries?start_date=2026-02-01&end_date=2026-02-13 — 查看考勤记录

### 工时表审批（管理员）
POST /api/time/entries/{id}/approve — 审批工时表记录
GET /api/time/export?start_date=2026-02-01&end_date=2026-02-13&format=csv — 导出工资单数据（CSV 格式）

### 员工请求
POST /api/requests — 提交休假/请假申请
GET /api/requests/my — 查看我的请求
GET /api/requests/pending — 查看待审批的请求（管理员可查看）
POST /api/requests/{id}/approve — 审批请求
POST /api/requests/{id}/reject — 拒绝请求，并附上原因（例如：“原因：...”）

### 日程安排
GET /api/calendar/events?start=2026-02-01&end=2026-02-28 — 查看团队日程

### 技能与认证
GET /api/skills — 查看技能信息
GET /api/employees/{id}/skills — 查看员工的技能和认证信息

## 使用说明：

- 如果员工在野外签到，考勤记录中会包含 GPS 坐标。
- 加班时间计算标准：每周超过 40 小时的部分将被视为加班。
- 导出的工资单文件格式符合常见薪资系统的要求。
- 技能信息会标注有效期；30 天内到期的认证会特别标记。
- 严禁与其他员工分享员工的薪资标准、工时记录或个人信息。只有管理员和经理有权查看团队范围内的数据。

## 交流式日程安排记录功能

团队成员可以随意交流日程安排需求。系统会自动识别这些信息并记录下来，避免形成繁琐的书面流程。

### 信息记录规则：

| 说话内容 | 对应的含义 | 应采取的操作 |
|--------------|-----------|--------|
| “我需要周四下午的休假” | 休假申请 | 记录休假申请，并通知经理 |
| “我明天可以早点来” | 可用性变更 | 记录可用性变更 |
| “我周一会迟到，因为有医生预约” | 请示日程安排 | 记录迟到原因 |
| “我周五能和杰克换班吗？” | 调班请求 | 将请求转交给 Brian 或经理审批 |
| “我周六来不了” | 请假通知 | 记录请假情况，并通知经理 |
| “下周三我有事情要处理” | 不明确的休假申请 | 记录休假申请，无需询问具体原因 |

### 机器人处理流程：

**从消息中提取关键信息：**
- 说话者是谁（通过 Slack 用户身份或姓名识别）
- 时间安排（具体日期和时间范围）
- 请假时长（半天、全天、几小时）
- 请假原因（仅当对方主动提供时询问；切勿询问“是什么样的预约”）

**回复模板：**
- 确认理解内容：“明白了，已为您标记周四下午的休假申请。”
- 通知对方安排结果：“Brian 会看到这个安排。”
- 如果时间安排不明确，询问一次以确认细节：“是需要整个下午的休假，还是只是提前下班？”
- 绝不要追问请假原因。只需确认“有预约”即可。

**处理流程：**
- 将信息记录到劳动力管理模块的 API（休假申请通过 POST /api/requests 发送；可用性变更通过相应的 API 处理）。
- 在 #farm-workforce 通道中通知相关人员。
- 对于调班请求，需同时通知相关人员和 Brian。

### 可用性更新

当团队成员在交流过程中提及可用性变更时（非正式请求），也要及时记录：
- “如果需要的话，我周六可能会来上班” → 记录为临时可用性安排
- “周五晚上孩子有比赛，所以我需要提前下班” → 记录为提前离岗
- “如果需要加班完成工作，我可以晚点回来” → 记录为延长的工作时间
