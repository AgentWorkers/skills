# FarmOS Finance

提供前瞻性的现金流规划和成本管理功能。根据计划成本和预期收入来预测项目的未来现金流。

## 数据完整性

1. 在报告财务数据时，务必明确列出各项总额：“总运营成本为892,000美元，分为12个类别。”
2. 现金流预测应结合多种数据来源。如果营销收入方面出现问题，必须如实说明——切勿仅展示成本而忽略收入，因为这无法提供完整的财务情况。
3. 如果某个接口返回错误，应向用户报告错误情况，而不要提供不完整的财务数据。不完整的数据比没有数据更糟糕。
4. 对于成本项目的查询，可以使用 `/api/cost-items` 接口，并指定 `crop_year` 参数——该接口会返回所有成本项目，无需分页。

## 该功能的适用场景

- “我们的现金流情况如何？”
- “今年的每英亩成本是多少？”
- “玉米的盈亏平衡价格是多少？”
- “每月的支出预测”
- “显示成本类别”
- “我们的主要支出有哪些？”

## 认证要求

**仅限管理员使用。** 该功能会访问敏感的财务数据，因此必须使用管理员级别的认证。

```bash
TOKEN=$(~/clawd/scripts/farmos-auth.sh admin)
```

**角色映射：** 请查看 `~/.clawdbot/farmos-users.json` 文件。如果请求者不是管理员，回复：“财务数据仅限农场所有者查看。我无法为您的账户访问这些信息。”

## API 基础地址

http://100.102.77.110:8010

## 集成接口（无需认证——如果启用了AI访问功能）

### 成本汇总
GET /api/integration/summary?crop_year=2025

返回：指定作物年份的各类成本总额。

### 简化版现金流
GET /api/integration/cash-flow-simple?crop_year=2025

返回：每月的支出预测。

### 盈亏平衡分析
GET /api/integration/breakeven?crop_year=2025

返回：按作物和实体划分的每英亩成本及每蒲式耳成本。这可以帮助您了解覆盖成本所需的最低价格。

## 需要认证的接口（需要JWT令牌）

### 成本类别
GET /api/categories
授权方式：Bearer {token}

返回：成本类别的定义（如肥料、种子、化学品、燃料、保险等）。

### 成本项目
GET /api/cost-items?crop_year=2025
授权方式：Bearer {token}

返回：详细的成本项目信息，包括金额、时间安排及所属实体。

### 完整版现金流预测
GET /api/cash-flow/projection?crop_year=2025
授权方式：Bearer {token}

返回：包含成本和收入的完整月度现金流信息，显示资金的具体收支情况。

### 现金流汇总
GET /api/cash-flow/summary?crop_year=2025
授权方式：Bearer {token}

返回：按类别和月份汇总的财务数据。

## 关键概念

- **成本类别**：分为单位成本（如肥料、种子，按使用量×英亩计算）和年度总成本（如保险、人工费用）。
- **时间安排**：各项成本会分配到具体的月份；如果涉及多个月份，费用会平均分摊。
- **实体分配**：成本可以分配给某个特定实体，也可以按作物面积在所有实体间分摊，或手动进行分配。
- **盈亏平衡**：总成本除以预期产量，得出每蒲式耳的最低价格，以确保成本得到覆盖。

## 使用注意事项

- 请务必指定 `crop_year` 参数。
- 盈亏平衡问题是用户最常询问的内容，请快速且清晰地回答。
- 现金流预测需要结合成本数据（本模块提供）和收入数据（营销模块提供）。
- 绝不要与非管理员用户共享任何财务数据，包括每英亩成本、盈亏平衡信息、现金流数据或任何成本细节。
