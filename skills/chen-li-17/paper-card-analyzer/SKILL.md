# 论文卡片生成器

该工具根据 `paper-parse` 的解析结果，通过直接的自然语言分析生成一份面向研究的论文卡片。

## 输入要求

需要读取 `paper-parse` 生成的以下文件：
- `*_content.md`（完整的Markdown格式论文内容）
- `*_parsed.json`（元数据和图表信息）

## 输出

默认情况下，输出结果将以英文形式生成，并保存在与选定的 `*_content.md` 和 `*_parsed.json` 相同的文件夹中。生成的文件包括：
- `paper-card.md`
- `paper-card.json`
- `paper-card-feedback.md`（反馈日志和修订历史）

论文卡片的章节结构如下：
1. 论文概览
2. 研究问题与动机
3. 核心贡献
4. 方法概述
5. 实验设置
6. 主要结果与证据
7. 消融分析与发现
8. 限制性与有效性威胁
9. 可复现性说明
10. 开放性问题与未来研究方向

## 工作流程

1. 确定目标文件对：
   - 最佳情况：同一文件夹中包含一个 `*_content.md` 和一个 `*_parsed.json` 文件。
   - 如果有多个候选文件，请让用户选择一对。
2. 从 `*_parsed.json` 中读取元数据：
   - `title`（标题）、`paper_name`（论文名称）、`num_pages`（页数）、`figures`（图表数量）。
3. 阅读 `*_content.md` 并按章节提取相关内容：
   - 摘要/引言/方法/实验/结果/消融/局限性/结论。
4. 编写论文卡片内容：
   - 重点关注科学创新性、方法逻辑、证据强度、有效性威胁以及可复现性。
5. 将初稿保存到同一文件夹中：
   - `paper-card.md` 和 `paper-card.json`。
6. 请求用户反馈并进行修订：
   - 询问需要修改或加强的部分。
   - 更新卡片内容并重新保存（覆盖原有文件）。
   - 将每轮修订记录添加到 `paper-card-feedback.md` 中，包括修订轮次、用户要求及关键修改内容。
7. 重复修订过程，直到用户明确表示满意为止。
8. 明确标注不确定的内容：
   - 如果某个章节在解析后的内容中未明确提及，应标注为“未在解析内容中找到”。

## 可靠性协议

对于论文卡片中的每一条陈述：
- 仅使用来自 `*_content.md` 或 `*_parsed.json` 的证据。
- 如果证据不足或不存在，应标注为“未在解析内容中找到”。
- 区分“作者报告的结果”和“分析者的评估”。
- 未经直接文本支持，不得推断具体的数字、数据集或基准值。
- 采用保守的表述方式，避免过度推测。

在完成每一轮修订之前，进行自我检查：
- 确保没有未经证实的事实性陈述。
- 所有指标数值均来源于原始内容或已被删除。
- 限制性内容至少包含一个明确的有效性威胁。
- 可复现性说明中应包含已知和未知的信息。
- JSON 键值对及 Markdown 章节顺序必须完整且一致。

## 各章节的具体要求

1. **论文概览**
   - 包括标题、论文名称、会议/发表年份（如果可确定的话）、页数、图表数量。
   - 如果会议/发表年份不确定，应标注为“未知”。

2. **研究问题与动机**
   - 说明研究任务、现有研究的不足之处，以及这些不足为何重要。
   - 如果作者提供了研究范围的信息，也应包括在内。

3. **核心贡献**
   - 列出2-5个明确的创新点。
   - 每项贡献都必须独立理解且不重复。

4. **方法概述**
   - 解释主要组成部分、数据/模型流程及设计理由。
   - 除非对理解至关重要，否则避免涉及实现细节。

5. **实验设置**
   - 记录文本中提到的数据集、基准值、指标及实验方案细节。
   - 指出影响比较性的缺失设置信息。

6. **主要结果与证据**
   - 在有数据支持的情况下，报告最显著的结果及对应的指标。
   - 区分总体收益与每个数据集或每个指标的具体收益。

7. **消融分析与发现**
   - 总结消融分析或研究结果对各个组成部分必要性的验证。
   - 如果没有相关的消融分析结果，应明确说明。

8. **限制性与有效性威胁**
   - 至少涵盖数据/基准偏差风险、方法假设、外部有效性风险。
   - 区分这些限制是作者提出的还是分析者推断的。

9. **可复现性说明**
   - 记录代码/数据链接、超参数相关信息、缺失的文件以及影响可复现性的因素。
   - 说明独立复现所需的努力程度或风险水平。

10. **开放性问题与未来研究方向**
    - 根据观察到的证据缺口，提出2-4个具体的研究问题。
    - 问题应具有可验证性，并且与实验相关。

## 文风要求

- 采用简洁、客观的科学写作风格。
- 不要编造未在原始文本中提及的指标、数据集或结论。
- 区分作者的观点与分析者的评估结果。
- 保持各章节之间的顺序一致。
- 使用精确的语言，避免使用夸张的表述，并在有确凿依据的情况下才使用绝对肯定的表述。

## JSON 数据结构

使用以下顶级键来组织卡片内容：
- `paper_snapshot`
- `research_problem_and_motivation`
- `core_contributions`
- `method_overview`
- `experimental_setup`
- `main_results_and_evidence`
- `ablation_and_analysis_findings`
- `limitations_and_threats_to_validity`
- `reproducibility_notes`
- `open_questions_and_future_work`
- `figures`

每次修订后，都会将更新后的 `paper-card.json` 文件保存下来，而不仅仅是根据请求进行保存。
