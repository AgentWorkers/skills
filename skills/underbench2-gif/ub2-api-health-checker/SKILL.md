# API健康检查器（API Health Checker）

这是一个功能强大的技能，它允许Claw测试API端点、验证其响应内容、测量性能，并生成健康报告。

## 该技能的功能

该技能提供了系统的API测试和监控流程：
1. **端点测试** — 向指定的API端点发送HTTP请求（GET、POST、PUT、DELETE）。
2. **响应验证** — 检查状态码、响应时间和响应体结构是否符合预期值。
3. **身份验证支持** — 处理API密钥、Bearer令牌以及基本的身份验证头部信息。
4. **性能测量** — 记录响应时间，并标记响应缓慢的端点。
5. **健康报告** — 生成一份综合报告，总结所有被测试端点的状态。

## 使用方法

只需提供API端点的地址，Claw就会自动进行测试：
- “检查https://api.example.com/v1/status是否响应正常”
- “测试该API中的所有端点，并报告哪些端点出现了故障”
- “监控这5个端点的平均响应时间”
- “验证/user端点是否返回包含预期字段的JSON数组”

## 配置选项

您可以指定以下配置参数：
- **端点（Endpoints）** — 需要测试的URL列表。
- **预期状态码（Expected status codes）** — 每个端点应返回的HTTP状态码（默认值：200）。
- **超时时间（Timeout）** — 每个请求的最大等待时间（默认值：10秒）。
- **头部信息（Headers）** — 包括身份验证令牌在内的自定义头部信息。
- **请求体（Request body）** — 用于POST/PUT请求的数据内容。

## 输出结果

健康报告包含以下内容：
- **状态概览** — 被测试的端点总数、通过的数量、失败的数量以及超时的数量。
- **每个端点的详细信息** — URL、请求方法、状态码、响应时间以及测试结果（通过/失败）。
- **性能指标** — 响应时间的平均值、最小值、最大值以及第95百分位的值。
- **建议** — 对于响应缓慢或出现错误的端点，提供相应的优化建议。
