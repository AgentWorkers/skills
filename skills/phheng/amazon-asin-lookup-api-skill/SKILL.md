# 亚马逊ASIN查询技能

## 📖 介绍
该技能利用BrowserAct的亚马逊ASIN查询API模板，提供了一种从亚马逊获取产品信息的便捷方式。只需提供ASIN，即可直接将产品信息（包括标题、价格、评分、品牌和详细描述）提取到您的应用程序中，无需手动进行数据抓取。

## ✨ 特点
1. **无错误生成**：预设的工作流程避免了AI生成的错误信息，确保数据提取的稳定性和准确性。
2. **无需处理验证码**：无需处理reCAPTCHA或其他验证挑战。
3. **无IP限制**：不受地区IP限制或地理围栏的影响。
4. **执行速度快**：相比纯AI驱动的浏览器自动化解决方案，执行速度更快。
5. **成本效益高**：与高token消耗的AI解决方案相比，显著降低了数据获取成本。

## 🔑 API密钥设置
在运行之前，请检查`BROWSERACT_API_KEY`环境变量。如果未设置，请等待用户提供API密钥。
**代理应告知用户**：
> “由于您尚未配置BrowserAct API密钥，请访问[BrowserAct控制台](https://www-browseract.com/reception/integrations)获取您的密钥。”

## 🛠️ 输入参数
代理应根据用户需求配置以下参数：

1. **ASIN（亚马逊标准识别码）**
   - **类型**：`string`
   - **描述**：亚马逊产品的唯一标识符。
   - **是否必需**：是
   - **示例**：`B07TS6R1SF`

## 🚀 使用方法
代理应执行以下脚本以一次性获取结果：

```bash
# Example Usage
python -u ./scripts/amazon_asin_lookup_api.py "ASIN_VALUE"
```

### ⏳ 执行监控
由于此任务涉及自动化的浏览器操作，可能需要一些时间（几分钟）。脚本会**持续输出带有时间戳的状态日志**（例如：`[14:30:05] 任务状态：运行中`）。
**代理说明**：
- 在等待脚本结果时，请持续监控终端输出。
- 只要终端仍在输出新的状态日志，说明任务正在正常运行；不要将其误认为是死锁或无响应。
- 仅当状态长时间保持不变或脚本停止输出且未返回结果时，才考虑触发重试机制。

## 📊 数据输出
成功后，脚本会解析并打印来自API响应的结构化产品数据，包括：
- `product_title`：产品的完整标题。
- `ASIN`：提供的ASIN。
- `product_url`：亚马逊产品页面的URL。
- `brand`：品牌名称。
- `price_current_amount`：当前价格。
- `price_original_amount`：原价（如适用）。
- `price_discount_amount`：折扣金额（如适用）。
- `rating_average`：平均评分。
- `rating_count`：评分总数。
- `featured`：如“Amazon's Choice”之类的徽章。
- `color`：颜色版本（如适用）。
- `compatible_devices`：兼容设备列表（如适用）。
- `product_description`：产品详细描述。
- `special_features`：突出显示的功能。
- `style`：样式属性（如适用）。
- `material`：使用的材料（如适用）。

## ⚠️ 错误处理与重试
如果脚本执行过程中出现错误（例如网络波动或任务失败），代理应按照以下逻辑处理：
1. **检查输出内容**：
   - 如果输出中包含“Invalid authorization”，则表示API密钥无效或已过期。**不要重试**；引导用户重新检查并提供正确的API密钥。
   - 如果输出中不包含“Invalid authorization”，但任务仍然失败（例如，输出以`Error:`开头或返回空结果），代理应**自动尝试重新执行脚本一次**。
2. **重试限制**：
   - 自动重试次数限制为**一次**。如果第二次尝试仍然失败，请停止重试，并向用户报告具体的错误信息。

## 🌟 典型用途
1. **产品数据丰富**：获取一系列ASIN的完整详细信息，以更新电子商务数据库。
2. **价格比较**：查询特定ASIN的当前亚马逊价格，以便与其他零售商进行比较。
3. **评论监控**：跟踪关键产品的评分平均值和评论数量的变化。
4. **库存检查**：自动验证特定产品是否在亚马逊上有库存。
5. **品牌分析**：根据ASIN识别产品的品牌和制造商。
6. **详细规格**：获取材料、样式和颜色信息，用于目录管理。
7. **功能突出显示**：提取“特殊功能”和详细描述，用于营销文案。
8. **兼容性验证**：检查电子产品或配件的“兼容设备”。
9. **市场研究**：分析特定产品ID的“Amazon's Choice”等徽章。
10. **URL转换**：将ASIN列表转换为完整的亚马逊产品页面URL。
