# 简化代码

重构代码，使其更易于阅读、更易于维护，并与周围的代码库保持一致，同时不改变代码的功能。

使用 `/simplify` 命令来调用此功能。

## 目的

当用户希望代码更加简洁、实现更加直接，或者希望在不改变代码功能的前提下提升代码的可读性时，可以使用此功能。

## 工作原理

- **保留功能**：除非用户明确要求，否则不要更改代码的行为、输出、副作用或公共接口。
- **遵循项目标准**：使代码符合现有的编码规范、模式、命名风格和架构要求。
- **减少不必要的复杂性**：避免不必要的嵌套结构，移除多余的抽象层，选择最直接、最清晰的实现方式。
- **确保变量的使用有明确的目的**：除非能显著提升代码的可读性或变量被频繁重用，否则不要引入额外的状态变量。
- **改进命名和结构**：使用清晰明了的变量名和函数名，并将相关的逻辑组织在一起，使代码易于理解。

## 默认范围

默认情况下，该功能会针对最近修改过的代码进行优化；除非用户指定了特定的文件、模块或差异文件。
