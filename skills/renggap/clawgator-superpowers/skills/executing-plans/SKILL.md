# 执行计划

## 概述

加载计划，进行仔细审查，分批执行任务，并在批次之间提交报告以供审核。

**核心原则：** 分批执行，并设置检查点以供架构师审核。

**开始时声明：** “我将使用‘执行计划’技能来实施这个计划。”

## 执行流程

### 第1步：加载并审查计划
1. 读取计划文件
2. 仔细审查计划，发现任何问题或疑虑
3. 如果有疑虑：在开始之前与人类合作伙伴沟通
4. 如果没有疑虑：创建待办事项（TodoWrite），然后继续执行

### 第2步：分批执行任务
**默认执行前3个任务**
对于每个任务：
1. 将任务标记为“进行中”（inProgress）
2. 严格按照计划中的步骤操作
3. 按照规定执行验证步骤
4. 将任务标记为“已完成”（completed）

### 第3步：提交报告
当批次任务全部完成后：
- 显示已执行的内容
- 显示验证结果
- 说明：“准备接收反馈。”

### 第4步：继续执行
根据反馈：
- 如有需要，进行修改
- 执行下一批次任务
- 重复此过程，直到所有任务完成

### 第5步：完成开发
当所有任务都完成并通过验证后：
- 宣布：“我将使用‘完成开发分支’技能来完成这项工作。”
- **所需子技能：** 使用“完成开发分支”（finishing-a-development-branch）技能
- 按照该技能的步骤进行测试、展示选项并做出选择

## 何时停止并寻求帮助

**在以下情况下立即停止执行：**
- 在执行过程中遇到障碍（缺少依赖项、测试失败、指令不明确）
- 计划中存在严重问题，导致无法继续执行
- 不理解某条指令
- 验证多次失败

**遇到问题时，请寻求澄清，不要自行猜测。**

## 何时重新审查早期步骤
**在以下情况下返回步骤1进行重新审查：**
- 合作伙伴根据你的反馈更新了计划
- 需要重新考虑整个执行方法

**不要强行克服障碍**——遇到问题时请停止并寻求帮助。

## 注意事项
- 首先仔细审查计划
- 严格按照计划步骤操作
- 不要跳过验证步骤
- 在计划要求时使用相应的技能
- 在批次执行期间：只需提交报告并等待
- 遇到障碍时立即停止，不要猜测
- 未经用户明确同意，切勿在主分支（main/master branch）上开始执行任务

## 集成所需的工作流程技能：
- **必备技能：** 使用Git工作树（using-git-worktrees）——在开始前设置隔离的工作环境
- **必备技能：** 编写计划（writing-plans）——创建该技能所执行的计划文档
- **必备技能：** 完成开发分支（finishing-a-development-branch）——在所有任务完成后完成开发工作
