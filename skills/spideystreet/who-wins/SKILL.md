# PinchBench排行榜

该脚本用于获取并格式化PinchBench排行榜数据，该排行榜展示了在标准化OpenClaw编码任务中表现最佳的AI模型（LLMs）。

## 工作流程

### 1. 确定查询需求

将用户的请求转换为相应的脚本参数：

| 用户请求 | 参数 |
|-------------|-------|
| “显示排行榜” / 默认 | `--top 10` |
| “排名前5的模型” | `--top 5` |
| “Claude的表现如何？” | `--model claude` |
| “成本最低的模型” | `--sort cost --top 10` |
| “运行速度最快的模型” | `--sort time --top 10` |
| “比较Gemini和Claude” | 分别使用`--model gemini`和`--model claude`运行两次，然后并排展示结果 |
| “完整排行榜” | `--top 50` |

### 2. 运行脚本

```json
{
  "tool": "exec",
  "command": "python3 {baseDir}/scripts/fetch_leaderboard.py --top 10"
}
```

可用参数：
- `--top N` — 显示的模型数量（默认值：10）
- `--sort metric` — 排序依据：`score`（分数）、`cost`（成本）或`time`（运行时间）（默认值：`score`）
- `--model filter` — 过滤包含指定字符串的模型（不区分大小写）
- `--json` — 以原始JSON格式输出数据，便于进一步处理

### 3. 格式化结果

将输出结果以代码块的形式呈现，并在表格后添加简短的说明：

- 突出表现最佳的模型及其分数
- 如果用户询问了特定模型的排名，请说明该模型在所选指标上的表现
- 如果按照成本排序，请注明最佳模型（即分数与成本的比值）

### 4. 错误处理

- 如果脚本因curl请求失败而报错，应报告错误并提示用户检查网络连接
- 如果脚本无法解析数据，可能是因为网站结构发生了变化，应通知用户
- 如果没有模型符合过滤条件，应说明这一点并建议用户进行更广泛的搜索

## 示例

| 用户请求 | 参数 | 预期输出 |
|-----------|-------|-------------------|
| “显示PinchBench排行榜” | `--top 10` | 显示按分数排名前10的模型 |
| “OpenClaw中成本最低的模型是哪个？” | `--sort cost --top 10` | 显示按成本排名前10的模型 |
| “Claude与GPT相比如何？” | 先运行`--model claude`，再运行`--model gpt` | 分别显示Claude和GPT的排名并加以比较 |
| “PinchBench上运行速度最快的模型是哪个？” | `--sort time --top 5` | 显示按执行时间排名前5的模型 |
