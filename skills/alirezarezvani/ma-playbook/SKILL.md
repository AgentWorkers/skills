# 并购操作手册

本手册涵盖了并购过程中的各方（收购方和被收购方）所需遵循的框架和流程。

## 关键词
并购（M&A）、合并与收购（mergers and acquisitions）、尽职调查（due diligence）、收购（acquisition）、收购整合（acqui-hire）、整合（integration）、交易结构（deal structure）、估值（valuation）、意向书（LOI，Letter of Intent）、条款清单（term sheet）、绩效激励计划（earnout）

## 快速入门

**作为收购方：**
- 首先明确战略目的 → 筛选目标公司 → 进行尽职调查 → 评估公司价值 → 谈判交易条款 → 实施整合。

**作为被收购方：**
- 先进行自身准备情况评估 → 准备数据共享平台（data room） → 选择并购顾问 → 谈判交易条件 → 平稳过渡。

## 作为收购方时需要考虑的事项

### 战略目的（在采取任何行动之前必须明确）
- **自建 vs 收购**：自己开发是否更快/更经济？如果是，就不要选择收购。
- **收购的内容**：你真正想要的是什么？是人才、技术还是客户资源？
- **整合难度**：将这家公司整合到你的体系中有多困难？

### 尽职调查 checklist
| 领域 | 关键问题 | 需警惕的警示信号 |
|--------|--------------|-----------|
| 财务 | 收入质量、客户集中度、资金消耗速度 | 超过30%的收入来自单一客户 |
| 技术 | 代码质量、技术债务、系统架构的兼容性 | 代码缺乏测试且架构陈旧 |
| 法律 | 知识产权归属、潜在法律纠纷、合同情况 | 关键知识产权由个人持有 |
| 人员 | 关键人员的去留风险、企业文化匹配度 | 创始人没有锁定条款或绩效激励计划 |
| 市场 | 市场地位、竞争威胁 | 市场份额下降 |
| 客户 | 客户流失率、净推荐值（NPS）、合同条款 | 客户流失率高或合同期限较短 |

### 估值方法
- **收入倍数法**：根据行业情况确定（SaaS企业通常为2-15倍年收入）
- **类似交易参考**：参考类似公司的成交价格
- **折现现金流法（DCF）**：仅适用于盈利企业（大多数初创企业适用收入倍数法）
- **收购整合成本**：在热门市场中，每位工程师的收购成本约为100万至300万美元

## 作为被收购方时需要关注的事项

### 收购方的兴趣信号
- 收购方表现出强烈的收购意愿
- 市场出现整合趋势
- 筹资变得比日常运营更困难
- 创始人愿意接受收购并配合过渡

### 准备工作（收购前6-12个月）
1. 整理财务状况（如可能的话请进行审计）
2. 归档所有知识产权和合同文件
3. 降低客户集中度
4. 确保关键员工不会流失
5. 准备好数据共享平台
6. 聘请并购顾问

### 谈判要点
| 交易条款 | 需重点关注的内容 | 你的谈判筹码 |
|------|--------------|---------------|
| 估值 | 需警惕的估值陷阱（如无法实现的目标） | 多个竞争性报价 |
| 绩效激励计划 | 目标的定义、评估周期 | 现金支付与绩效激励的比例 |
| 人员锁定条款 | 期限和条件 | 你的替代可能性 |
| 售后服务和保修条款 | 责任范围 | 保证金与赔偿上限 |
| 员工保留 | 哪些员工会获得补偿、补偿条件是什么 | 关键人员的去留问题 |

## 需警惕的警示信号（双方都应警惕）
- 没有明确的战略理由，只是简单地认为“这是一笔好交易”
- 在尽职调查过程中发现明显的企业文化冲突却被忽视
- 在交易完成前关键人员未得到妥善安排
- 没有制定整合计划，或者只是简单地说“我们会想办法解决”

## 与高管团队的协作
| 高管角色 | 在并购中的职责 |
|------|-------------------|
| CEO | 明确战略目的、主导谈判 |
| CFO | 负责估值、交易结构设计和融资安排 |
| CTO | 负责技术方面的尽职调查和整合方案制定 |
| CHRO | 负责人员方面的尽职调查和员工保留计划 |
| COO | 负责整合工作的执行和流程整合 |
| CPO | 评估产品路线图对并购的影响以及客户资源的重叠情况 |
