## QMS实施工作流程

从差距分析到认证，实施符合ISO 13485:2016标准的质量管理体系。

### 工作流程：初始QMS实施

1. 根据ISO 13485:2016的要求进行差距分析
2. 记录每个条款的当前状态与要求状态
3. 按照以下标准对差距进行优先级排序：
   - 监管要求的紧迫性
   - 对产品安全的风险
   - 资源需求
4. 制定包含里程碑的实施路线图
5. 根据第4.2.2条制定质量手册：
   - 明确QMS的范围及合理的排除项
   - 过程间的交互关系
   - 程序引用
6. 创建所需的文件化程序——详见[强制要求的文件化程序](#quick-reference-mandatory-documented-procedures)以获取完整列表
7. 部署程序并进行培训
8. **验证：**差距分析完成；质量手册获得批准；所有必要的程序均已记录并完成培训

> 使用[qms-process-templates.md](references/qms-process-templates.md)中的差距分析矩阵模板，逐条记录当前状态、差距、优先级及应对措施。

### QMS结构

| 级别 | 文档类型 | 示例 |
|-------|---------------|---------|
| 1 | 质量手册 | QM-001 |
| 2 | 程序 | SOP-02-001 |
| 3 | 工作指导书 | WI-06-012 |
| 4 | 记录 | 培训记录 |

## 文件控制工作流程

根据ISO 13485第4.2.3条建立并维护文件控制。

### 工作流程：文件创建与审批

1. 确定需要新文件或文件修订的必要性
2. 按照编号规则分配文件编号：
   - 格式：`[类型]-[领域]-[序列]-[版本]`
   - 例如：`SOP-02-001-01`
3. 使用已批准的模板起草文件
4. 将文件提交给相关专家审核
5. 收集并处理审核意见
6. 根据文件类型获取必要的批准
7. 更新文件主列表
8. **验证：**文件编号正确；所有审核人员均已签字；主列表已更新

### 文件编号规则

| 前缀 | 文档类型 | 批准权限 |
|--------|---------------|-------------------|
| QM | 质量手册 | 管理层代表 + 首席执行官 |
| POL | 政策 | 部门负责人 + 质量保证 |
| SOP | 程序 | 过程负责人 + 质量保证 |
| WI | 工作指导书 | 监督员 + 质量保证 |
| TF | 模板/表格 | 过程负责人 |
| SPEC | 规范 | 工程部门 + 质量保证 |

### 领域代码

| 代码 | 领域 | 示例 |
|------|------|----------|
| 01 | 质量管理 | 质量手册、政策 |
| 02 | 文件控制 | 本程序 |
| 03 | 培训 | 能力要求程序 |
| 04 | 设计 | 设计控制 |
| 05 | 采购 | 供应商管理 |
| 06 | 生产 | 制造 |
| 07 | 质量控制 | 检验、测试 |
| 08 | CAPA | 纠正措施 |

### 文件变更控制

| 变更类型 | 批准级别 | 示例 |
|-------------|----------------|----------|
| 行政性 | 文件控制 | 打字错误、格式问题 |
| 轻微变更 | 过程负责人 + 质量保证 | 说明性修改 |
| 重大变更 | 全面审查周期 | 过程变更 |
| 紧急变更 | 加急处理 | 安全问题 |

### 文件审查周期

| 文档类型 | 审查周期 | 无需计划审查的触发条件 |
|---------------|---------------|-------------------------------|
| 质量手册 | 每年 | 组织结构变更 |
| 程序 | 每年 | 审计发现、法规变更 |
| 工作指导书 | 每两年 | 过程变更 |
| 表格 | 每两年 | 用户反馈 |

## 内部审核工作流程

根据ISO 13485第8.2.4条计划并执行内部审核。

### 工作流程：年度审核计划

1. 确定需要审核的过程和领域
2. 评估审核频率的风险因素：
   - 之前的审核发现
   - 法规变更
   - 过程变更
   - 投诉趋势
3. 指定具备资格的审核员（不得审核其负责的区域）
4. 制定年度审核计划
5. 获得管理层批准
6. 将审核计划通知过程负责人
7. 跟踪审核进度并根据需要重新安排
8. **验证：**所有过程均被覆盖；审核员具备资格且独立；审核计划获得批准

> 使用[qms-process-templates.md](references/qms-process-templates.md)中的审核计划模板，按条款和季度安排文件控制（4.2.3/4.2.4）、管理评审（5.6）、设计控制（7.3）、生产（7.5）和CAPA（8.5.2/8.5.3）等过程的审核。

### 工作流程：单次审核执行

1. 准备包含范围、标准和时间的审核计划
2. 至少提前一周通知被审核方
3. 审查程序和之前的审核结果
4. 准备审核检查表
5. 通过以下方式收集证据：
   - 文件审查
   - 记录抽样
   - 过程观察
   - 人员访谈
7. 对发现的问题进行分类：
   - 重大不符合项（NC）：系统缺失或完全失效
   - 轻微不符合项（NC）：单个失误或偏差
   - 观察结果：未来可能出现的问题
8. 召开总结会议
9. 在5个工作日内发布审核报告
10. **验证：**所有检查表项目均得到处理；发现的问题有证据支持；报告已分发

### 审核员资格要求

| 标准 | 要求 |
|-----------|-------------|
| 培训 | 了解ISO 13485标准 + 掌握审核技巧 |
| 经验 | 至少有一次作为审核员的参与经验 |
| 独立性 | 不得审核自己负责的区域 |
| 能力 | 理解被审核的过程 |

### 发现问题分类指南

| 分类 | 标准 | 处理时间 |
|----------------|----------|---------------|
| 重大不符合项（NC） | 系统缺失、完全失效、违反法规 | 30天内启动CAPA流程 |
| 轻微不符合项（NC） | 单次出现、部分不符合 | 60天内启动CAPA流程 |
| 观察结果 | 潜在风险、改进机会 | 在下一次审核中跟进 |

## 过程验证工作流程

根据ISO 13485第7.5.6条对特殊过程进行验证。

### 工作流程：过程验证协议

1. 确定需要验证的过程：
   - 无法通过检验来验证输出结果的过程
   - 仅在使用过程中出现缺陷的过程
   - 如灭菌、焊接、密封、软件相关过程
2. 组建包含相关专家的验证团队
3. 编写验证协议，内容包括：
   - 过程描述和参数
   - 使用的设备与材料
   - 接受标准
   - 统计方法
4. 执行初始验证（IQ）：验证设备安装正确性并记录相关规格
5. 执行操作验证（OQ）：测试参数范围并验证过程控制
6. 执行性能验证（PQ）：在生产条件下运行过程并验证输出是否符合要求
7. 编写验证报告并总结结论
8. **验证：**IQ/OQ/PQ完成；接受标准得到满足；验证报告获得批准

### 验证文档要求

| 阶段 | 内容 | 证据 |
|-------|---------|----------|
| 协议 | 目标、方法、标准 | 经批准的验证协议 |
| IQ | 设备验证 | 安装记录 |
| OQ | 参数验证 | 测试结果 |
| PQ | 性能验证 | 生产数据 |
| 报告 | 总结、结论 | 批准签名 |

### 重新验证的触发条件

| 触发因素 | 所需行动 |
|---------|-----------------|
| 设备变更 | 评估影响，重新验证相关阶段 |
| 参数变更 | 至少进行OQ和PQ验证 |
| 材料变更 | 评估影响，至少进行PQ验证 |
| 过程故障 | 全面重新验证 |
| 定期验证 | 根据验证计划（通常为3年）

### 特殊过程示例

| 过程 | 验证标准 | 关键参数 |
|---------|--------------------|--------------------|
| EO灭菌 | ISO 11135 | 温度、湿度、EO浓度、时间 |
| 蒸汽灭菌 | ISO 17665 | 温度、压力、时间 |
| 辐射灭菌 | ISO 11137 | 辐照剂量、剂量均匀性 |
| 密封 | 内部标准 | 温度、压力、停留时间 |
| 焊接 | ISO 11607 | 加热、压力、速度 |

## 供应商资格评估工作流程

根据ISO 13485第7.4条评估并批准供应商。

### 工作流程：新供应商资格评估

1. 确定供应商类别：
   - A类：关键供应商（影响安全/性能）
   - B类：重要供应商（影响质量）
   - C类：次要供应商（间接影响）
2. 请求供应商信息：
   - 质量认证
   - 产品规格
   - 质量历史记录
3. 根据以下标准评估供应商：
   - 质量管理体系（ISO认证）
   | 技术能力 |
   | 质量历史记录 |
   | 财务稳定性 |
4. 对于A类供应商：
   - 进行现场审核
   - 要求签署质量协议
5. 计算供应商评估分数
6. 作出审批决定：
   - 分数 >80：批准
   - 分数在60-80之间：有条件批准
   - 分数 <60：不批准
7. 将供应商添加到合格供应商名单
8. **验证：**评估标准得分完成；供应商分类完成

### 供应商评估标准

| 标准 | 权重 | 分值 |
|-----------|--------|---------|
| 质量管理体系 | 30% | ISO 13485占30%，ISO 9001占20%，文件化要求占10%，无则占0 |
| 质量历史记录 | 25% | 拒收率：<1%占25%，1-3%占15%，>3%占0 |
| 交付表现 | 20% | 准时交付率：>95%占20%，90-95%占10%，<90%占0 |
| 技术能力 | 15% | 超出标准占15%，符合标准占10%，勉强符合占5 |
| 财务稳定性 | 10% | 财务状况良好占10%，一般占5%，存疑占0 |

### 供应商类别要求

| 供应商类别 | 需要的评估内容 | 监控频率 | 协议要求 |
|----------|---------------|------------|-----------|
| A类（关键供应商） | 现场审核 | 每年 | 签署质量协议 |
| B类（重要供应商） | 问卷调查 | 每半年 | 质量要求 |
| C类（次要供应商） | 评估 | 根据问题情况 | 标准条款 |

### 供应商绩效指标

| 指标 | 目标值 | 计算方法 |
|--------|--------|-------------|
| 接收率 | >98% | （接受批次 / 总批次）× 100 |
| 准时交付率 | >95% | （准时交付批次 / 总订单数）× 100 |
| 响应时间 | <5天 | 解决问题的平均时间 |
| 文件完整性 | 100% | （完整的质量控制文件 / 必需文件）× 100 |

### CAPA启动标准

| 来源 | 是否需要启动CAPA | 处理方式 |
