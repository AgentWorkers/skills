## 文档控制流程

从文档创建到废弃的整个生命周期，系统均实施严格的控制措施：

1. 按照规定的编号规则为文档分配唯一编号。
2. 使用标准模板创建文档。
3. 将文档发送给指定的审核人员进行审核。
4. 处理审核意见并回复相关问题。
5. 确保获得所有必要的审批签名。
6. 设置生效日期并分发文档。
7. 更新文档主列表。
8. **验证：** 确保文档在需要使用时可被访问；过时的版本会被及时删除。

### 文档生命周期阶段

| 阶段 | 定义 | 需要执行的操作 |
|-------|------------|------------------|
| 草稿 | 正在创建或修订中 | 作者进行编辑，不得使用 |
| 审核 | 分发给审核人员 | 审核人员提供反馈 |
| 批准 | 收到所有签名 | 可以用于培训或分发 |
| 生效 | 培训完成，文档正式发布 | 可以投入使用 |
| 被替代 | 被新版本取代 | 停止使用 |
| 过时 | 不再适用 | 按照保留计划进行归档 |

### 文档类型及前缀

| 前缀 | 文档类型 | 通常包含的内容 |
|--------|---------------|-----------------|
| QM | 质量手册 | 质量管理体系概述、范围、政策 |
| SOP | 标准操作程序 | 过程级别的操作指南 |
| WI | 工作指令 | 任务级别的详细步骤 |
| TF | 模板/表格 | 控制用的填写表格 |
| SPEC | 规范书 | 产品/过程的规格要求 |
| PLN | 计划 | 质量/项目计划 |

### 不同文档类型所需的审核人员

| 文档类型 | 所需审核人员 | 所需审批人员 |
|---------------|-------------------|-------------------|
| SOP | 过程负责人、QA | QA经理、过程负责人 |
| WI | 区域主管、QA | 区域经理 |
| SPEC | 工程团队、QA | 工程经理、QA |
| TF | 过程负责人 | QA |
| 设计文档 | 设计团队、QA | 设计控制部门 |

## 文档编号系统

为文档分配一致的编号，以便于识别和查找。

### 编号格式

标准格式：`前缀-类别-序列号[-修订版本]`

--- 

### 类别代码

| 代码 | 功能领域 | 说明 |
|------|-----------------|-------------|
| 01 | 质量管理 | 质量管理体系相关程序、管理审核 |
| 02 | 文档控制 | 与文档控制相关的流程 |
| 03 | 人力资源 | 培训、能力要求 |
| 04 | 设计与开发 | 设计控制流程 |
| 05 | 采购 | 供应商管理 |
| 06 | 生产 | 制造流程 |
| 07 | 质量控制 | 检验、测试 |
| 08 | CAPA | 纠正/预防措施 |
| 09 | 风险管理 | ISO 14971相关流程 |
| 10 | 法规事务 | 与法规提交相关的记录 |

### 编号流程

1. 作者向文档控制部门申请编号。
2. 文档控制部门确认所属类别。
3. 分配下一个可用的序列号。
4. 将编号记录在文档主列表中。
5. 作者使用分配的编号创建文档。
6. **验证：** 确保编号格式符合标准；主列表中无重复编号。

### 修订版本标识

| 变更类型 | 修订版本增量 | 例 |
|-------------|-------------------|---------|
| 主要修订 | 增量编号 | Rev 01 → Rev 02 |
| 小幅修订 | 子版本增量 | Rev 01 → Rev 01.1 |
| 行政性变更 | 无内容变更或使用字母后缀 | Rev 01 → Rev 01a |

详细编号指南请参阅 `references/document-control-procedures.md`。

## 审批与审核流程

在文档发布前，必须完成必要的审核和审批流程。

### 审核流程

1. 作者完成文档草稿。
2. 通过指定途径将文档提交给审核人员。
3. 根据文档类型分配审核人员。
4. 审核人员在规定时间内（5-10个工作日内）提供反馈。
5. 作者处理审核意见并回复相关问题。
6. 作者提交修订后的文档。
7. 审批人员签字并注明日期。
8. **验证：** 所有指定审核人员均已完成审核；所有意见均已处理并有记录。

### 意见处理方式

| 处理方式 | 需要执行的操作 |
|-------------|-----------------|
| 同意 | 直接采纳审核意见 |
| 同意但需修改 | 采纳修改意见并记录修改理由 |
| 拒绝 | 不采纳修改意见并记录拒绝理由 |
| 延期处理 | 在后续修订中处理该意见并记录原因 |

## 变更控制流程

通过系统的审核和审批流程，系统化地管理文档的变更。

### 变更控制流程

1. 确定需要变更的文档。
2. 填写变更请求表并附上变更理由。
3. 文档控制部门分配变更编号并记录请求。
4. 将文档发送给审核人员以评估变更影响。
5. 根据变更类型获取必要的审批。
6. 作者实施经批准的变更。
7. 更新修订版本号和变更历史记录。
8. **验证：** 变更内容在批准范围内；变更历史记录完整。

### 变更分类

| 变更类型 | 定义 | 审批级别 | 例 |
|-------|------------|----------------|----------|
| 行政性变更 | 不影响文档内容 | 由文档控制部门处理 | 如拼写错误、格式问题 |
| 小幅变更 | 内容有少量修改 | 由过程负责人和QA共同审批 | 例如澄清说明 |
| 主要变更 | 内容有重大修改 | 需要完整的审核流程 | 例如新增要求 |
| 紧急变更 | 影响安全或合规性 | 加快审批流程并需追溯记录 | 例如安全相关问题 |

### 变更影响评估 checklist

| 变更影响领域 | 评估内容 |
|-------------|---------------------|
| 培训 | 变更是否需要重新培训？ |
| 设备 | 变更是否影响设备或系统？ |
| 验证 | 变更是否需要重新验证？ |
| 法规 | 变更是否影响法规文件？ |
| 其他文档 | 哪些相关文档需要更新？ |
| 记录 | 哪些记录会受到影响？ |

## 21 CFR Part 11合规性

为符合FDA的法规要求，系统实施电子记录和签名控制。

### Part 11的适用范围

| 适用对象 | 不适用对象 |
|------------|-------------------|
| FDA法规要求的记录 | 纸质记录 |
| 提交给FDA的记录 | 内部非法规要求的记录 |
| 需要电子签名的记录 | 一般的电子邮件通信 |

### 电子记录控制

1. 确保系统的准确性和可靠性。
2. 为所有变更生成安全的审计跟踪记录。
3. 仅允许授权人员访问系统。
4. 生成人类可读的电子副本。
5. 在整个保留期间保护记录的安全。
6. **验证：** 审计跟踪记录了所有变更的执行人、变更内容及时间。

### 审计跟踪要求

| 要求 | 实施措施 |
|-------------|----------------|
| 安全性 | 用户无法修改记录 |
| 系统自动生成记录 | 系统自动记录所有操作 |
| 时间戳 | 记录每个操作的日期和时间 |
| 保留原始数据 | 保留之前的所有数据 |
| 用户身份 | 记录每个变更的执行人 |

### 电子签名要求

| 要求 | 实施措施 |
|-------------|----------------|
| 签名唯一性 | 每个用户都有唯一的签名 |
| 至少包含两部分 | 用户ID和密码 |
| 签名显示信息 | 显示签名者的姓名、日期和时间 |
| 与记录关联 | 签名与记录紧密关联，不可删除或复制 |

### 签名显示要求

每个电子签名必须包含以下信息：

| 元素 | 例 |
|---------|---------|
| 姓名 | John Smith |
| 日期和时间 | 2024-03-15 14:32:05 EST |
| 签名含义 | 表示“批准发布” |

### 系统控制要求

**访问控制：**
- 为每个用户分配唯一的用户ID。
- 实施密码复杂性要求。
- 失败登录后锁定账户。
- 不活动一段时间后自动关闭会话。

**审计跟踪：**
- 记录所有操作。
- 记录所有修改内容及原始值。
- 记录用户的操作时间。

**安全性：**
- 实施基于角色的访问控制。
- 对存储和传输的数据进行加密。
- 定期备份数据并测试恢复机制。

详细合规性要求请参阅 `references/21cfr11-compliance-guide.md`。

## 文档控制指标

用于监控文档控制系统的性能。

### 关键绩效指标

| 指标 | 目标值 | 计算方法 |
|--------|--------|-------------|
| 文档周期时间 | <30天 | 从草稿到正式发布的平均天数 |
| 审核完成率 | >95% | 完成的审核数量占总审核数量的百分比 |
| 未处理的变更请求数量 | <10个 | 每月底未处理的变更请求数量 |
| 过期审核率 | <5% | 过期未审核的文档数量占总有效文档的数量 |
| 审计中发现的问题数量 | <2个 | 每次内部审计中发现的文档控制问题数量 |

### 定期审查计划

| 文档类型 | 审核频率 |
|---------------|------------------|
| 政策 | 每3年一次 |
| SOP | 每2年一次 |
| WI | 每2年一次 |
| 规范书 | 根据需要或产品更新情况 |
| 模板/表格 | 每3年一次 |

## 法规要求

### ISO 13485:2016标准第4.2条

| 子条款 | 要求 |
|------------|-------------|
| 4.2.1 | 质量管理体系文档 |
| 4.2.2 | 质量手册 |
| 4.2.3 | 医疗设备文件（技术文档） |
| 4.2.4 | 文档控制 |
| 4.2.5 | 记录控制 |

### FDA 21 CFR 820标准

| 条款 | 要求 |
|---------|-------------|
| 820.40 | 文档控制 |
| 820.180 | 一般记录要求 |
| 820.181 | 设备主记录 |
| 820.184 | 设备历史记录 |
| 820.186 | 质量体系记录 |

### 常见的审计问题及预防措施

| 审计问题 | 预防措施 |
|---------|------------|
| 使用过时的文档 | 实施文档分发控制 |
| 缺少审批签名 | 在发布前确保完成审批流程 |
| 变更历史记录不完整 | 每次修订时更新历史记录 |
| 未制定定期审查计划 | 建立并执行审查计划 |
| 审计跟踪记录不完善 | 确保DMS系统符合Part 11的合规性要求 |
