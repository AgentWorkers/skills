## 法规策略工作流程

制定与业务目标和产品特性相匹配的法规策略。

### 新产品法规策略工作流程

1. 收集产品信息：
   - 预期用途和适应症
   - 设备分类（风险等级）
   - 技术平台
   - 目标市场和时间表
2. 确定每个目标市场适用的法规：
   - 美国FDA：21 CFR Part 820, 510(k)/PMA/De Novo
   - 欧盟：MDR 2017/745, 合规性评估机构（Notified Body）的要求
   - 其他市场：加拿大卫生部（Health Canada）、PMDA、NMPA、TGA
3. 确定最佳的法规路径：
   - 比较不同的提交类型（510(k) vs De Novo vs PMA）
   - 评估现有类似产品的可用性
   - 评估临床证据要求
4. 制定包含关键节点的法规时间表
5. 估算资源需求和预算
6. 识别潜在的法规风险及应对策略
7. 获得利益相关者的同意和批准
8. **验证：**策略文件获得批准；时间表得到确认；资源得到分配

### 法规路径选择矩阵

| 因素 | 510(k) | De Novo | PMA |
|--------|--------|---------|-----|
| 是否有类似产品 | 是 | 否 | 不适用 |
| 风险等级 | 低至中等 | 低至中等 | 高 |
| 是否需要临床数据 | 通常不需要 | 可能需要 | 必需 |
| 审查时间 | 90天（MDUFA） | 150天 | 180天 |
| 用户费用 | 约2.2万美元（2024年） | 约13.5万美元 | 约44万美元 |
| 适用情况 | 仿制产品 | 新型低风险产品 | 高风险、新型产品 |

## FDA提交工作流程

准备并提交FDA的法规申请。

### 510(k)提交工作流程

1. 确认适用510(k)路径：
   - 确定有类似的产品
   - 能够证明实质等同性
   - 无新的预期用途或技术问题
2. 如有必要，安排并召开预提交（Pre-Submission）会议
3. 编制提交材料包：
   - 介绍信和行政信息
   - 设备描述和预期用途
   - 实质等同性比较
   - 性能测试数据
   - 生物相容性（如果产品与患者接触）
   - 软件文档（如适用）
   - 标签和使用说明书（IFU）
4. 进行内部审查和质量检查
5. 按照FDA的格式要求准备电子文件
6. 通过FDA的ESG门户提交并支付用户费用
7. 监控MDUFA的进度，并回应AI/RTA（Administrative Review Request）请求
8. **验证：**提交被接受；收到MDUFA通知；更新跟踪系统

### PMA提交工作流程

1. 确认适用PMA路径：
   - 产品属于III类设备或没有类似产品
   - 明确临床数据策略
2. 如有必要，完成IDE（Investigational Device Examination）临床研究：
   - 获得IDE批准
   - 执行临床研究
   - 完成研究报告
3. 召开预提交会议
4. 编制PMA申请材料：
   - 行政和设备信息
   - 制造信息
   - 非临床研究结果
   - 临床研究结果
   - 标签
5. 提交PMA申请
6. 回答FDA的疑问和指出不足
7. 准备FDA的现场检查
8. **验证：**PMA获得批准；收到批准函；记录批准后的要求

### FDA提交时间表

| 关键节点 | 510(k) | De Novo | PMA |
|-----------|--------|---------|-----|
| 预提交会议 | 提前90天 | 提前90天 | 提前120天 |
| 提交 | 提交当天 | 提交当天 | 提交当天 |
| AI/RTA审查 | 提前15天 | 提前15天 | 提前45天 |
| 实质性审查 | 15-90天 | 15-150天 | 45-180天 |
| 审批决定 | 提前90天 | 提前150天 | 提前180天 |

### FDA常见的缺陷

| 类别 | 常见问题 | 预防措施 |
|----------|---------------|------------|
| 实质等同性 | 类似产品比较不足 | 提前提供有力的实质等同性论证 |
| 性能测试 | 测试方案不完整 | 遵循公认的标准 |
| 生物相容性 | 缺少测试数据 | 进行ISO 10993风险评估 |
| 软件 | 文档不充分 | 遵守IEC 62304标准 |
| 标签 | 声明不一致 | 提前进行标签审查 |

更多信息请参阅：[references/fda-submission-guide.md](references/fda-submission-guide.md)

## 欧盟MDR提交工作流程

根据欧盟MDR 2017/745获得CE标志。

### MDR技术文档工作流程

1. 根据MDR附件VIII确认设备分类
2. 根据设备类别选择合规性评估路径：
   - I类设备：自我声明
   - IIa/IIb类设备：需要合规性评估机构的参与
   - III类设备：需要全面的合规性评估
3. 选择并联系相应的合规性评估机构（针对IIa+类设备）
4. 根据附件II编制技术文档：
   - 设备描述和规格
   - 设计和制造信息
   - 一般安全性和性能要求（GSPR）检查表
   - 利益-风险分析及风险管理
   - 根据附件XIV进行临床评估
   - 市后监督计划
5. 建立并记录符合ISO 13485标准的质量管理体系（QMS）
6. 向合规性评估机构提交申请
7. 回答机构的疑问并协调审计
8. **验证：**获得CE证书；签署符合性声明；完成EUDAMED注册

### MDR分类决策树

```
Is the device active?
        │
    Yes─┴─No
     │     │
     ▼     ▼
Is it an   Does it contact
implant?   the body?
  │            │
Yes─┴─No   Yes─┴─No
 │    │     │     │
 ▼    ▼     ▼     ▼
III  IIb  Check   Class I
         contact  (measuring/
         type     sterile if
         and      applicable)
         duration
```

### 不同类别的临床证据要求

| 类别 | 临床要求 | 需提交的文件 |
|-------|---------------------|---------------|
| I类 | 临床评估（CE认证） | CE认证报告 |
| IIa类 | 基于文献的CE认证 | CE认证报告 + PMCF（Post Market Clinical Feedback）计划 |
| IIb类 | 基于临床数据的CE认证 | CE认证报告 + PMCF + 部分临床研究 |
| III类 | 基于临床调查的CE认证 | CE认证报告 + PMCF + 临床调查 |

### 合规性评估机构选择标准

| 选择标准 | 考虑因素 |
|-----------|---------------|
| 评估范围 | 机构在设备类别上的专业知识 |
| 评估能力 | 评估机构的可用性和审查时间表 |
| 经验 | 机构在相关技术领域的经验 |
| 地理位置 | 机构距离公司的距离（便于审计） |
| 费用 | 费用结构的透明度 |
| 沟通能力 | 机构的响应速度和清晰度 |

## 全球市场准入工作流程

协调国际市场的法规审批工作。

### 多市场提交策略

1. 根据业务优先级确定目标市场
2. 按顺序提交申请，以高效利用现有临床数据：
   - 第一阶段：FDA和欧盟（参考市场）
   - 第二阶段：加拿大、澳大利亚等认可市场
   - 第三阶段：日本、中国等主要市场
   - 第四阶段：新兴市场
3. 确定每个市场的具体要求：
   - 临床数据的接受标准
   - 当地代理/代表的需求
   - 标签和语言要求
4. 制定包含本地化计划的技术文件
5. 建立当地的法规支持团队
6. 并行或依次提交申请
7. 跟踪审批进度并协调产品上市
8. **验证：**所有目标市场的审批完成；更新注册数据库

### 市场优先级矩阵

| 市场 | 规模 | 复杂性 | 合规性要求 | 优先级 |
|--------|------|------------|-------------|----------|
| 美国 | 规模大 | 复杂度高 | 需要FDA批准 | 1 |
| 欧盟 | 规模大 | 复杂度高 | 需要欧盟批准 | 1-2 |
| 加拿大 | 规模中等 | 复杂度中等 | 需要MDSAP（Canadian Medical Devices Agency）批准 | 2 |
| 澳大利亚 | 规模中等 | 复杂度较低 | 需要欧盟批准 | 2 |
| 日本 | 规模大 | 复杂度高 | 需要本地临床评估 | 3 |
| 中国 | 规模大 | 复杂度非常高 | 需要本地测试 | 3 |
| 巴西 | 规模中等 | 复杂度高 | 需要GMP（Good Manufacturing Practice）检查 | 3-4 |

### 文档效率策略

| 文档类型 | 是否需要单一来源 | 是否需要本地化 |
|---------------|---------------|----------------------|
| 核心技术文件 | 是 | 需要格式调整 |
| 风险管理文件 | 是 | 不需要 |
| 临床数据 | 是 | 需要额外评估 |
| QMS证书 | 是（ISO 13485要求） | 需要针对市场的审计 |
| 标签 | 主要标签 | 需要翻译和符合当地要求 |
| 使用说明书 | 主要内容 | 需要翻译和符合当地要求 |

更多信息请参阅：[references/global-regulatory-pathways.md](references/global-regulatory-pathways.md)

## 法规情报工作流程

监控并应对影响产品组合的法规变化。

### 法规变化管理工作流程

1. 监控法规来源：
   - FDA联邦公报、指导文件
   - 欧盟官方期刊、MDCG（Medical Device Coordination Group）的指导
   - 合规性评估机构的沟通
   - 行业协会（如AdvaMed、MedTech Europe）
2. 评估法规变化对产品组合的影响：
   - 合规性要求的变更时间表
   - 所需资源
   - 产品可能需要做出的调整
3. 制定合规行动计划
4. 与受影响的利益相关者沟通
5. 实施必要的变更
6. 记录合规状态
8. **验证：**合规行动计划获得批准；变更按计划实施

### 法规监控来源

| 来源 | 类型 | 监控频率 |
|--------|------|-----------|
| FDA联邦公报 | 法规和指导文件 | 每日 |
| FDA设备数据库 | 510(k)、PMA相关信息、召回通知 | 每周 |
| 欧盟官方期刊 | MDR/IVDR更新 | 每周 |
| MDCG指导 | 欧盟的法规实施情况 | 按发布时间 |
| ISO/IEC标准 | 标准更新 | 每季度 |

### 预提交会议决策

| 因素 | 是否需要召开预提交会议 | 是否可以跳过预提交会议 |
|--------|------------------|--------------|
| 新技术 | 是 | |
| 新的预期用途 | 是 | |
| 复杂的测试 | 是 | |
| 不确定的类似产品 | 是 | |
| 需要临床数据 | 是 | |
| 技术成熟度较高 | 否 | 是 |
| 有明确的类似产品 | 否 | 是 |
| 标准化的测试流程 | 否 | 是 |

### 法规问题升级标准

| 情况 | 升级级别 | 应对措施 |
|-----------|------------------|--------|
| 申请被拒绝 | 法规事务副总裁 | 进行根本原因分析，修订策略 |
| 重大缺陷 | 高管 | 跨部门响应团队 |
| 时间表受影响 | 管理层 | 重新分配资源 |
| 法规变化 | 法规事务副总裁 | 评估对产品组合的影响 |
| 安全问题 | 高层管理人员 | 立即采取控制措施并报告 |
