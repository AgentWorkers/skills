# 产品分析

定义、跟踪并解读产品在不同发展阶段（发现期、成长期和成熟期）的指标数据。

## 使用场景

适用于以下场景：
- 指标框架的选择（AARRR、North Star、HEART）
- 根据产品阶段定义关键绩效指标（KPI）（产品市场发布前、成长期、成熟期）
- 仪表盘设计和指标层次结构
- 群组分析及用户留存分析
- 功能采用情况与用户转化路径的分析

## 工作流程

1. **选择指标框架**：
   - 使用 AARRR 框架来跟踪产品成长过程及用户转化路径。
   - 采用 North Star 框架实现跨职能团队的战略协同。
   - 使用 HEART 框架评估用户体验质量。

2. **定义适合各阶段的 KPI**：
   - **产品市场发布前（Pre-PMF）**：用户激活率、早期留存率、定性成功指标。
   - **成长期**：用户获取效率、产品扩展能力、转化速度。
   - **成熟期**：用户留存深度、收入质量、运营效率。

3. **设计仪表盘层级**：
   - **高层管理仪表盘**：展示5-7个关键指标。
   - **产品健康状况仪表盘**：用户获取情况、激活率、留存率、用户参与度。
   - **功能使用仪表盘**：功能采用率、用户使用深度、重复使用频率、功能效果相关性。

4. **进行群体分析与用户留存分析**：
   - 按注册群体或功能使用群体进行数据分段。
   - 比较不同群体的留存曲线，而非单一时间点的数据。
   - 识别用户入职阶段和首次产生价值的节点。

5. **解读数据并采取行动**：
   - 将指标变化与产品更新及发布时间线关联起来分析。
   - 通过环比数据区分重要信号与干扰因素。
   - 针对每个主要指标风险或机会提出明确的改进措施。

## 不同阶段的 KPI 指导

### 产品市场发布前（Pre-PMF）
- 用户激活率
- 第一周用户留存率
- 首次产生价值的平均时间

### 成长期
- 各阶段的用户转化率
- 每月留存用户数量
- 新用户群体中的功能采用情况

### 成熟期
- 与收入相关的用户留存指标
- 重度用户占比及使用深度
- 不同用户群体的流失风险指标
- 产品可靠性和用户支持需求指标

## 仪表盘设计原则：
- 展示趋势，而非孤立的数据点。
- 每个 KPI 都应指定负责人。
- 为每个 KPI 设定目标值、阈值和决策依据。
- 默认使用群体和数据分段功能。
- 优先选择具有可比性的时间周期（例如：周对比周、月对比月）。

## 群组分析方法

1. **定义群体基准事件**：用户注册、激活、首次购买。
2. **定义用户留存行为**：活跃天数、关键操作、重复使用行为。
3. **按群体和时间段构建留存矩阵**。
4. **比较不同群体的留存曲线**。
5. **识别用户流失的早期迹象并分析使用过程中的障碍**。

## 留存曲线解读：
- **早期流失严重、后期平台稳定**：可能是入职流程不完善或初始产品价值不足。
- **流失率适中、平台稳定**：说明核心用户群体稳定，流失率可预测。
- **留存率长期处于较低水平**：表明产品使用频率较低，需要评估产品价值。
- **新用户群体的留存率提升**：说明入职流程或产品定位有所改进。

## 工具支持

### `scripts/metrics_calculator.py`

这是一个命令行工具（CLI），用于：
- 按用户群体年龄计算留存率。
- 生成群体数据表。
- 进行基础的用户转化路径分析。

**示例代码**：
```bash
python3 scripts/metrics_calculator.py retention events.csv
python3 scripts/metrics_calculator.py cohort events.csv --cohort-grain month
python3 scripts/metrics_calculator.py funnel funnel.csv --stages visit,signup,activate,pay
```
