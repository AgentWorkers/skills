## 审计计划工作流程

**制定基于风险的内部审计计划：**

1. 列出所有需要审计的质量管理体系（QMS）流程。
2. 为每个流程分配风险等级（高/中/低）。
3. 审查之前的审计发现和趋势。
4. 根据风险等级确定审计频率。
5. 指派合格的审计员（确保其独立性）。
6. 制定年度审计计划。
7. 将计划告知流程负责人。
8. **验证：**确保所有ISO 13485条款都包含在审计范围内。

### 基于风险的审计频率

| 风险等级 | 审计频率 | 标准 |
|------------|-----------|----------|
| 高 | 季度 | 设计控制、纠正措施（CAPA）、生产验证 |
| 中 | 半年 | 采购、培训、文件控制 |
| 低 | 年度 | 基础设施、管理评审（如果系统稳定） |

### 按条款划分的审计范围

| 条款 | 流程 | 重点关注领域 |
|--------|---------|-------------|
| 4.2 | 文件控制 | 文件审批、分发、过时文件的处理 |
| 5.6 | 管理评审 | 输入信息的完整性、决策的记录、行动的跟踪 |
| 6.2 | 培训 | 培训内容的有效性、记录的完整性 |
| 7.3 | 设计控制 | 设计输入、评审、验证与验证（V&V）、变更管理 |
| 7.4 | 采购 | 供应商评估、来料检验 |
| 7.5 | 生产 | 工作指令、过程验证、缺陷记录（DHR） |
| 7.6 | 校准 | 设备清单、校准状态、超出校准范围的设备 |
| 8.2.2 | 内部审计 | 审计计划的合规性、审计员的独立性 |
| 8.3 | 不合格产品 | 产品的识别、隔离、处置 |
| 8.5 | 纠正措施（CAPA） | 根本原因的分析、措施的落实、效果的验证 |

### 审计员的独立性

在指派审计员之前，需验证其独立性：

- 审计员不负责被审计的领域。
- 审计员与被审计方没有直接的报告关系。
- 审计员未参与最近被审计的活动。
- 审计员具备相应的资格认证。

## 审计执行

**进行系统的内部审计：**

1. 准备审计计划（范围、标准、时间表）。
2. 审计前查阅相关文件。
3. 与被审计方召开开场会议。
4. 收集证据（记录、访谈、观察结果）。
5. 对发现的问题进行分类（重大/次要/观察项）。
6. 召开总结会议，讨论初步审计结果。
7. 在5个工作日内准备审计报告。
8. **验证：**确保所有审计范围的内容都得到覆盖，且发现的问题都有证据支持。

### 证据收集方法

| 方法 | 用途 | 相关文档 |
|--------|---------|---------------|
| 文件审查 | 确认流程的合规性 | 文件编号、版本、日期 |
| 访谈 | 了解流程细节 | 被访谈者的姓名、职位、访谈内容摘要 |
| 观察 | 确认实际操作情况 | 观察到的内容、地点、时间 |
| 追踪记录 | 确认流程的连贯性 | 记录ID、日期、关联信息 |

### 按条款划分的审计问题

**文件控制（4.2）：**
- 请出示文件清单。
- 你们是如何控制过时文件的？
- 请提供文件变更审批的证明。

**设计控制（7.3）：**
- 请出示[产品]的设计历史文件。
- 谁参与了设计评审？
- 请展示设计输入与输出之间的可追溯性。

**纠正措施（CAPA）（8.5）：**
- 请出示未完成纠正措施的记录。
- 你们是如何确定根本原因的？
- 请提供措施实施效果的验证记录。

完整的审计问题集请参见`references/iso13485-audit-guide.md`。

### 发现问题的记录

每个发现的问题都应记录在专门的文档中：

```
Requirement: [Specific ISO 13485 clause or procedure]
Evidence: [What was observed, reviewed, or heard]
Gap: [How evidence fails to meet requirement]
```

**示例：**
```
Requirement: ISO 13485:2016 Clause 7.6 requires calibration
at specified intervals.

Evidence: Calibration records for pH meter (EQ-042) show
last calibration 2024-01-15. Calibration interval is
12 months. Today is 2025-03-20.

Gap: Equipment is 2 months overdue for calibration,
representing a gap in calibration program execution.
```

## 不符合项管理

**对审计发现进行分类和管理：**

1. 根据分类标准评估发现的问题。
2. 分配问题的严重程度（重大/次要/观察项）。
3. 用客观证据记录发现的问题。
4. 将问题告知流程负责人。
5. 对重大或次要问题启动纠正措施（CAPA）。
6. 跟进问题的解决情况。
7. 在后续审计中验证措施的有效性。
8. **验证：**只有在纠正措施有效后，问题才能被视为已解决。

### 分类标准

| 分类 | 定义 | 是否需要启动CAPA | 处理时限 |
|----------|------------|---------------|----------|
| 重大 | 系统性的故障或缺失关键要素 | 是 | 30天内 |
| 较小 | 个别失误或部分实施 | 建议在60天内处理 |
| 观察项 | 需要改进的地方 | 根据情况而定 |

### 分类决策

```
Is required element absent or failed?
├── Yes → Systematic (multiple instances)? → MAJOR
│   └── No → Could affect product safety? → MAJOR
│       └── No → MINOR
└── No → Deviation from procedure?
    ├── Yes → Recurring? → MAJOR
    │   └── No → MINOR
    └── No → Improvement opportunity? → OBSERVATION
```

### 纠正措施（CAPA）的整合

| 问题严重程度 | 纠正措施深度 | 验证要求 |
|------------------|------------|--------------|
| 重大 | 进行全面的根本原因分析（5Why分析、鱼骨图） | 下次审计前或6个月内完成 |
| 较小 | 立即识别原因 | 下次预定审计时处理 |
| 观察项 | 不需要特别处理 | 在下次审计时记录 |

详细的指导请参见`references/nonconformity-classification.md`。

## 外部审计准备

**为认证机构或监管机构的审计做准备：**

1. 完成所有预定的内部审计。
2. 确保所有问题都通过有效的纠正措施得到解决。
3. 检查文件的时效性和准确性。
4. 以审计结果为输入，进行管理评审。
5. 准备审计所需的设施和人员。
6. 进行模拟审计（涵盖全部审计范围）。
7. 向相关人员简要介绍审计流程。
8. **验证：**在正式审计前，模拟审计中发现的问题都得到处理。

### 审计前准备清单

**文档方面：**
- [ ] 质量手册是最新的。
- [ ] 程序与实际操作一致。
- [ ] 所有记录完整且易于查阅。
- [ ] 之前的审计问题都得到解决。

**人员方面：**
- [ ] 审计期间关键人员可随时到位。
- [ ] 指定了相关领域的专家。
- [ ] 人员已接受审计流程的培训。
- [ ] 指定了陪同人员。

**设施方面：**
- [ ] 工作区域整洁有序。
- [ ] 使用中的文件都是最新的。
- [ ] 设备的校准状态清晰可见。
- [ ] 不合格产品已妥善隔离。

### 模拟审计流程

1. 由外部审计员或合格的内部审计员进行模拟审计。
2. 模拟实际审计的流程和形式。
3. 如实记录模拟审计中的发现。
4. 在正式审计前解决所有重大和次要问题。
5. 向管理层简要报告准备情况。

## 工具

### 审计计划优化工具

```bash
# Generate optimized audit schedule
python scripts/audit_schedule_optimizer.py --processes processes.json

# Interactive mode
python scripts/audit_schedule_optimizer.py --interactive

# JSON output for integration
python scripts/audit_schedule_optimizer.py --processes processes.json --output json
```

该工具根据以下因素生成基于风险的审计计划：
- 流程的风险等级。
- 之前的审计结果。
- 自上次审计以来的时间。
- 问题的紧迫性。

**输出内容包括：**
- 优先级排序的审计计划。
- 按季度分配的审计安排。
- 过期审计的提醒。
- 资源使用建议。

## 审计计划绩效指标

**跟踪审计计划的执行效果：**

| 指标 | 目标 | 测量方法 |
|--------|--------|-------------|
| 审计计划的合规性 | >90% | 审计按时完成 |
| 发现问题的解决率 | >95% | 问题在截止日期前得到解决 |
| 重复出现的问题 | <10% | 连续审计中出现相同的问题 |
| 纠正措施的有效性 | >90% | 后续审计中验证措施的有效性 |
| 审计员的利用率 | 每名合格审计员每月参与4次审计 |
