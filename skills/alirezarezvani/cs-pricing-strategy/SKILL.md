# 定价策略

作为SaaS定价和盈利模式的专家，您的目标是设计出能够体现产品价值、实现合理转化率，并能够随着客户数量增长而扩展的定价方案。

定价不仅仅是数学计算——它更是一种市场定位。正确的价格并非仅仅是成本加上利润的简单叠加，而应该是介于客户认为能够获得的最佳替代方案的成本与您的产品实际价值之间的价格。大多数SaaS产品的定价都偏低。这项技能的核心就在于如何明确且有理有据地解决这一问题。

## 开始之前

**先了解背景信息：**
如果存在`marketing-context.md`文件，请在提问之前先阅读它。根据该文件的内容，仅提出您需要了解的问题。

收集以下信息：

### 1. 当前状况
- 您目前是否有定价方案？如果有：包括哪些定价档次、价格区间以及计费模式是什么？
- 从试用/免费版本转换到付费版本的用户转化率是多少？（如果已知）
- 每位客户的平均收入是多少？
- 每月的客户流失率是多少？

### 2. 业务背景
- 产品类型：B2B还是B2C？是自助使用的还是需要销售协助的？
- 客户群体：哪些是您的核心客户，哪些是偶尔使用的用户？
- 竞争对手：客户会将您的产品与哪些竞争对手进行比较？这些竞争对手的定价是多少？
- 成本结构：服务一位客户每月需要花费多少成本？

### 3. 目标
- 您是正在设计新的定价方案、优化现有定价，还是计划提高价格？
- 有哪些限制因素？（例如：老客户、合同条款、渠道合作伙伴的利润空间）

## 定价策略的实施方法

### 方式1：从零开始设计定价
在没有现有定价模型的情况下，或者需要完全重建定价体系时，我们将共同探讨价值指标的选择、价格档次划分、价格区间研究以及定价页面的设计。

### 方式2：优化现有定价
如果现有定价方案转化率低、业务扩展停滞，或者客户觉得价格不合理，我们将对现有定价进行审计，进行基准分析，并找出具体的改进点。

### 方式3：计划提价
由于通货膨胀、产品价值提升或市场定位调整等原因，您可能需要提价。我们将制定一个既能增加收入又不会导致客户流失的提价策略。

## 三个定价维度

每个定价决策都涉及到三个维度。必须这三个维度都考虑到位。

```
         ┌─────────────────┐
         │   PACKAGING     │  What's in each tier?
         │  (what you get) │
         └────────┬────────┘
                  │
         ┌────────┴────────┐
         │  VALUE METRIC   │  What do you charge for?
         │ (how it scales) │
         └────────┬────────┘
                  │
         ┌────────┴────────┐
         │  PRICE POINT    │  How much?
         │    (the number) │
         └─────────────────┘
```

大多数团队会直接跳到价格区间的设定上，但这其实是本末倒置的。首先应该确定价值指标，然后确定产品包装（即提供的功能和服务），最后再测试具体的价格。

## 价值指标的选择

您选择的价值指标将决定定价如何与客户价值相匹配。如果选择不当，要么会浪费收入，要么会阻碍业务增长。

### 常见的SaaS价值指标

| 指标 | 适用场景 | 示例 |
|--------|---------|---------|
| **按用户数量计费** | 协作工具、客户关系管理（CRM） | Salesforce, Notion |
| **按使用次数计费** | API工具、基础设施、人工智能服务 | Stripe, Twilio, OpenAI |
| **按功能计费** | 平台型产品、插件 | Intercom, HubSpot |
| **固定费用** | 需要持续使用的SMB工具 | Basecamp, Calendly Basic |
| **按成果计费** | 高价值、可衡量的投资回报率（ROI）相关的工具 |
| **混合计费** | 结合上述多种方式 | 大多数成熟的SaaS产品 |

### 如何选择价值指标

回答以下问题：
1. **是什么让客户愿意支付更高的价格？** 这就是您的价值指标。
2. **这个指标是否随着客户的成功而增长？** 如果客户的成功增加了，您的收入也会增加。
3. **这个指标是否易于理解？** 复杂的指标会降低转化率。
4. **客户是否能够通过不正当手段规避这个指标？** 客户不应该有机会绕过这个指标。

**需要注意的问题：**
- 在一个主要由核心用户驱动的产品中采用“按用户数量计费”的方式——因为核心用户的使用量往往远超普通用户。
- 如果某些客户从产品中获得的价值远高于其他用户，却采用“固定费用”计费——这实际上是在补贴这些高价值用户。
- 在调用次数波动较大的情况下采用“按API调用次数计费”的方式——这会导致账单金额不稳定，从而增加客户流失。

## 三档定价结构

三档定价结构是标准做法。这并非出于传统习惯，而是因为它有助于客户对产品价值的认知。

### 三档定价结构的设计原则

**入门级（适合新客户）：**
- 适合那些如果价格上涨就会流失的客户群体。
- 功能或使用量有限，或者提供的支持服务也有限。
- 不提供免费版本。免费版本应单独作为一项策略（如免费增值服务），而不是定价档次的一部分。
- 至少要能够覆盖成本。

**中级级（适合大部分客户）：**
- 这是您主要吸引客户的层级。
- 价格通常是入门级的2-3倍。
- 提供成长型企业所需的所有功能。
- 在定价页面上明确标注为推荐版本。

**高级级（适合高价值客户）：**
- 适用于有企业级需求的高价值客户。
- 可能提供“联系我们”或定制定价方案。
- 开放额外的功能和服务，如单点登录（SSO）、审计日志、服务水平协议（SLA）、专属客服支持、定制合同。
- 如果您的企业级合同年收入超过1000万美元（MRR），则需要设置这一层级来吸引这些客户。

### 各个档级的具体内容

| 功能类别 | 入门级 | 中级级 | 高级级 |
|----------------|-------|--------|------|
| 核心产品 | ✅（有限） | ✅（完整） | ✅（完整） |
| 使用限制 | 低 | 中等 | 高/无限 |
| 用户数量 | 1-3人 | 5人及以上 | 无限 |
| 集成能力 | 基础 | 完整 | 完整 + 定制 |
| 报告功能 | 基础 | 高级 | 定制 |
| 支持服务 | 电子邮件 | 优先响应 | 专属客户服务经理（CSM） |
| 管理功能 | 无 | 无 | 单点登录（SSO）、审计日志、SCIM（安全控制机制） |
| 服务水平协议（SLA） | 无 | 无 | ✅ |

有关定价模型的详细信息和SaaS行业的最佳实践，请参考[references/pricing-models.md]。

## 基于价值的定价

定价应介于客户认为的最佳替代方案与您所提供价值的中间。

```
[Cost of doing nothing] ... [Next-best alternative] ... [YOUR PRICE] ... [Perceived value delivered]
```

**步骤1：确定最佳替代方案**
- 如果没有您的产品，客户会选择什么？
- 是竞争对手的产品？还是电子表格？还是手动流程？或者雇佣专人？
- 这些替代方案的成本是多少？

**步骤2：估算所提供的价值**
- 客户通过使用您的产品节省的时间 × 使用该方案人员的每小时费用
- 通过使用您的产品产生的收入或避免的成本
- 避免的错误或风险的成本
- 向您的核心客户询问：“如果明天停止使用我们的服务，您会损失什么？”

**步骤3：设定价格**
- 一个大致的参考标准：定价应在所提供价值的10%-20%之间。
- 不要将价格定在价值的50%——否则客户会感觉被高估。
- 价格也不能低于最佳替代方案的价格——否则说明您对自己的产品缺乏信心。

**转化率作为参考因素：**
- 如果试用转为付费的转化率超过40%，说明定价可能偏低——可以考虑提价。
- 15%-30%的转化率对于大多数SaaS产品来说是合理的。
- 如果转化率低于10%，可能说明定价过高，或者试用转为付费的流程存在障碍。

## 定价研究方法

### Van Westendorp价格敏感度测试

向现有客户或目标客户群体提出以下四个问题：
1. 在什么价格下，这个产品会显得太便宜而让人怀疑其质量？
2. 在什么价格下，这个产品会显得非常划算？
3. 在什么价格下，这个产品会开始显得太贵而不再值得购买？
4. 在什么价格下，这个产品会显得过于昂贵而无法考虑？

**结果解读方法：** 绘制四条曲线。 “太便宜”和“太贵”两个交点的价格区间就是您可接受的价格范围。“划算”和“昂贵”两个交点的价格就是最佳定价点。

**适用场景：** B2B SaaS产品，受访者数量≥30人，包括现有客户和潜在客户。

### MaxDiff分析

向受访者展示不同的功能组合和价格选项，询问他们最看重和最不看重的功能。通过统计分析可以了解各项功能的相对价值，从而帮助您优化产品包装。

**适用场景：** 在决定哪些功能应该放入哪个定价档次时使用。

### 竞争对手定价分析

| 步骤 | 具体操作 |
|------|-----------|
| 1 | 列出直接竞争对手以及客户认为的替代方案 |
| 2 | 记录这些竞争对手的公开定价（产品名称、价格、价值指标） |
| 3 | 分析每个价格档次所提供的内容 |
| 4 | 分析您的产品与竞争对手在哪些方面表现优于或低于竞争对手 |
| 5 | 根据市场定位设定价格：高端产品定价应比市场价高出20%-40%，性价比产品则应与市场价持平或更低 |

**注意：** 不要简单地复制竞争对手的定价——他们的定价反映了他们的成本结构和市场定位，而不是您的。**

## 提价策略

提价是SaaS公司能够获得高投资回报率的举措之一。然而大多数公司都等待太久才采取行动。

### 提价策略的选择

| 策略 | 适用场景 | 风险 |
|---------|---------|------|
| **仅针对新客户** | 预计会遇到强烈反对 | 风险较低——不影响现有客户群 |
| **对老客户实行优惠并延迟提价** | 客户忠诚度较高，但存在合同风险 | 风险中等——现有客户会感到被尊重 |
| **与产品价值的提升挂钩** | 有新的功能或改进 | 风险较低——合理 |
| **调整产品结构** | 需要对产品进行重大调整 | 风险中等——客户可能会感到困惑 |
| **统一提价** | 对产品价值有信心，且当前价格明显低于市场水平 | 风险中等-较高 |

### 执行流程

1. **量化提价影响：** 计算在保持现有客户80%、90%的情况下，新客户的年收入（MRR）会增加多少。
2. **根据风险进行客户分类：** 区分年度合同客户、积极使用产品的客户以及使用量较大的高风险客户。
3. **设定提价通知时间：** 给现有客户60-90天的通知时间，至少提前30天。
4. **说明提价原因：** 提及新功能、成本上升、对[某项服务的投资等内容——要具体说明。
5. **提供选择方案：** 为选择年度合同的客户提供当前价格的保障，或者提供3个月的试用期。
6. **准备客服团队：** 准备常见问题解答（FAQ）、沟通要点以及提价的授权文件。
7. **监控提价后的效果：** 关注客户流失率、降级率以及支持工单量。

**预计提价20%-30%后的客户流失率：** 5%-15%。如果提价后的净收入增加，就可以继续执行提价计划。

## 定价页面设计

定价页面的作用是将客户的购买意向转化为实际购买行为。设计页面时需要牢记这一点。

### 页面顶部内容
- 必须包含：产品名称（简单明了，例如“入门级/专业级/企业级”，或者根据客户群体命名）
- 显示价格及计费选项（按月/按年计费，年度计费时要明确显示节省的费用）
- 每个定价档次的三到五个主要优势
- 每个定价档次的购买呼吁按钮（CTA）
- 推荐档次上标注“最受欢迎”标签

### 页面底部内容
- **完整的功能对比表**：内容全面、易于阅读，使用情况用✅和❌标记
- **常见问题解答（FAQ）**：解答阻碍客户购买的五个常见问题：
  - “我可以随时取消服务吗？”
  - “当我达到使用上限时会发生什么？”
  - “你们提供退款吗？”
  - “我的数据安全吗？”
  - “如果我需要升级或降级怎么办？”
- **客户信任证明**：展示与每个档次相关的客户案例、推荐信或评价
- **安全认证**：如果针对B2B企业客户，展示SOC2、ISO 27001、GDPR等安全认证标志

### 年度计费与按月计费的切换选项
- 默认显示年度定价（或突出显示）——这有助于提高客户生命周期价值（LTV）
- 明确显示节省的费用：“节省20%”或“免费试用两个月”
- 不要隐藏按月计价的信息——隐藏价格会引发客户的不信任

有关定价页面的设计规范和文案模板，请参考[references/pricing-page-playbook.md]。

## 主动调整定价的时机

在以下情况下应主动调整定价：
- **试用转为付费的转化率超过40%**：这表明定价可能偏低，建议测试提价20%-30%。
- **所有客户都处于中级级**：说明需要提供更高级别的服务或功能，或者当前定价方案无法满足客户需求。
- **客户要求的功能不在他们当前所在的档次中**：说明现有定价方案限制了客户的价值，建议评估是否需要增加相应功能。
- **每月客户流失率超过5%**：在提价之前先解决客户流失问题，因为提价可能会加剧流失。
- **价格两年多没有调整**：仅凭通货膨胀因素，提价10%-15%也是合理的。建议进行战略评估。

## 定价相关的输出成果

## 沟通方式

所有输出内容都遵循以下结构化的沟通标准：
- **先说明结论**：先给出建议，再解释原因。
- **明确行动内容、原因和执行方式**：每个建议都包含这三个要素。
- **为每个行动指定负责人和截止日期**：避免含糊的“考虑中”。
- **使用标签标注信息可靠性**：使用🌟表示经过验证的数据，🟡表示估算值，🔴表示假设值

## 相关技能

- **产品策略师**：负责产品路线图和整体盈利策略的制定，但不涉及定价页面的设计或执行。
- **文案撰写**：负责优化定价页面的文案，但不涉及定价结构和档次设计。
- **客户流失预防**：在提价前解决客户流失问题，提高客户留存率。
- **A/B测试**：用于在初步设计后测试不同的价格或页面布局。
- **客户成功经理**：通过追加销售提高收入，但不涉及定价设计或产品包装。
- **竞争对手分析**：用于制作与定价页面配套的竞争分析材料。
