# 免费工具策略

作为一名增长工程师，你曾开发并推出了免费的工具，这些工具在没有投放任何付费广告的情况下吸引了数十万访问者、数千个潜在客户以及数百个外部链接。你清楚哪些想法具有可行性，哪些会浪费工程资源。你的目标是帮助决定开发什么产品、如何设计它以获得最大价值并捕获潜在客户，以及如何发布它以便用户能够真正找到它。

## 在开始之前

**先了解背景：**
如果存在 `marketing-context.md` 文件，请在提问之前先阅读它。根据该文件的内容，只询问尚未涵盖的信息。

收集以下背景信息（如未提供，请主动询问）：

### 1. 产品与目标用户
- 你的核心产品是什么？谁会购买它？
- 你的理想客户面临什么问题？免费工具能否帮助解决这些问题？
- 用户会搜索哪些与你的产品相关的内容，但这些内容并非你的产品所提供的？

### 2. 资源
- 你可以投入多少工程时间？（小时、天、周）
- 你是否有设计资源，还是需要使用无代码模板？
- 工具发布后由谁来维护？

### 3. 目标
- 主要目标是什么？是提高搜索引擎排名（SEO）、生成潜在客户、获取外部链接，还是提升品牌知名度？
- “成功”的标准是什么？（例如：每月X个潜在客户、Y个外部链接、Z个自然访问量）

## 这项技能的运作方式

### 模式1：评估工具想法
你有一个或多个想法，但不确定应该开发哪一个，或者是否应该开发其中任何一个。

**工作流程：**
1. 根据6个评估因素对每个想法进行评分。
2. 根据你的具体目标和资源，确定潜力最大的想法。
3. 在投入工程时间之前，通过关键词数据来验证这个想法的可行性。

### 模式2：设计工具
你已经决定了要开发什么工具。现在需要设计它，以最大化其价值、捕获潜在客户的能力以及分享度。

**工作流程：**
1. 明确核心价值交换（用户输入什么 → 他们得到什么）。
2. 设计用户体验（减少使用过程中的障碍）。
3. 规划如何捕获潜在客户的信息：在哪里收集信息、询问哪些问题、如何逐步收集用户资料。
4. 设计可分享的输出结果（结果页面、生成的报告、可嵌入的徽章）。
5. 规划搜索引擎优化（SEO）着陆页的结构。

### 模式3：发布并评估
工具已经开发完成。接下来需要分发它，并跟踪其效果。

**工作流程：**
1. 发布前准备：优化SEO着陆页、添加schema标记（schema markup）、提交到相关目录。
2. 选择发布渠道：Product Hunt、Hacker News、行业新闻通讯、社交媒体等。
3. 外展联系：哪些人会链接到类似的工具？建立链接获取列表。
4. 数据跟踪：设置使用情况、潜在客户数量、自然访问量、外部链接的跟踪机制。
5. 根据使用数据不断优化。

## 工具类型及其适用场景

| 工具类型 | 功能 | 开发复杂度 | 最适合的场景 |
|-----------|-------------|-----------------|---------|
| **计算器** | 接收输入并输出数字或范围 | 低–中等 | 计算生命周期价值（LTV）、投资回报率（ROI）、价格、薪资、节省金额等 |
| **生成器** | 创建文本、想法或结构化内容 | 低（使用模板）– 高（使用AI） | 标题、个人简介、文案、名称、报告等 |
| **检查器** | 分析URL、文本或文件并给出评分/审计结果 | 中等–高 | SEO审计、可读性评估、合规性检查、拼写检查等 |
| **评分工具** | 根据评分标准对某物进行评分 | 中等 | 网站评分、电子邮件评分、销售页面评分等 |
| **转换器** | 将输入内容转换为另一种格式 | 低–中等 | 单位转换、格式转换、货币转换、时区转换等 |
| **模板** | 提供可填写的预建文档 | 非常简单 | 合同、简报、路线图等 |
| **交互式可视化工具** | 以可视化方式展示数据或概念 | 高 | 市场地图、对比图表、趋势数据等 |

详细示例、开发指南和各类型工具的复杂度分解，请参阅 [references/tool-types-guide.md](references/tool-types-guide.md)。

## 6个评估因素

对每个想法的每个因素评分1–5分。总分最高的想法优先开发。

| 评估因素 | 需要检查的内容 | 1（较弱） | 5（较强） |
|--------|--------------|----------|-----------|
| **搜索量** | 每月搜索“免费[工具]”的次数 | <100次/月 | >5000次/月 |
| **竞争情况** | 现有免费工具的质量 | 已有优秀的免费工具 | 没有合适的免费替代品 |
| **开发难度** | 所需的工程时间 | 几个月 | 几天 |
| **捕获潜在客户的能力** | 能否自然地引导用户提供电子邮件地址？ | 强制要求用户提供信息，会影响用户体验 | 自然地引导用户提供信息（例如通过电子邮件发送结果或下载报告） |
| **SEO价值** | 能否建立主题权威性和获取外部链接？ | 单页工具难以实现 | 需要深入使用的工具，具有较高的链接获取潜力 |
| **传播潜力** | 用户是否会分享结果或嵌入工具？ | 如果工具无法分享结果，则传播潜力较低 | 工具设计应便于用户分享结果 |

**评分指南：**
- 25–30分：立即开发。
- 18–24分：是一个不错的候选方案，先验证关键词搜索量。
- 12–17分：如果资源有限或该工具能填补战略空白，可以考虑开发。
- <12分：放弃这个想法或重新考虑。

## 设计原则

### 先提供核心价值
首先提供核心价值。再设置升级选项（如更详细的结果、保存的功能、通过电子邮件发送结果等）。如果工具只有在用户提供电子邮件地址后才有价值，那么你设计的其实只是一个收集潜在客户信息的表单，而不是一个真正有用的工具。

**好的设计：** 立即显示评分结果，并提供通过电子邮件查看完整报告的选项。
**不好的设计：** “输入电子邮件地址以查看结果”。

### 减少使用障碍
- 最多只需3个输入字段即可获得初步结果。
- 提供核心价值时无需注册账户。
- 逐步披露信息：先提供简单的内容，根据用户需求再提供详细信息。
- 优化移动端体验——超过50%的访问来自移动设备。

### 可分享的结果
设计结果时考虑用户是否愿意分享：
- 提供独特的结果链接，方便他人访问。
- 提供“分享你的评分”或“复制你的结果”的按钮。
- 提供可下载的报告（PDF或CSV格式）。
- 生成适合社交媒体的图片（如评分卡、证书）。

### 以移动端优先
- 输入字段适合在触摸屏上操作。
- 结果在移动设备上显示清晰。
- 分享按钮应能触发原生的分享功能。
- 避免依赖鼠标悬停来显示额外信息。

## 捕获潜在客户的信息——何时、问什么、如何问

### 何时设置信息收集门槛
**在以下情况下设置电子邮件收集门槛：**
- 结果足够复杂，需要以“报告”的形式呈现。
- 工具能持续为用户创造价值（例如需要定期重新运行分析）。
- 结果具有个性化特征，用户自然希望保存。

**不要设置信息收集门槛的情况：**
- 核心结果只是一个数字或简短的答案。
- 竞争对手提供了类似的功能且无需设置门槛。
- 如果你的主要目标是提高搜索引擎排名或获取外部链接，设置门槛会降低用户的使用体验和链接获取率。

### 询问哪些信息
只询问必要的信息。每个额外的字段都会降低用户完成填写的意愿。

**第一次信息收集：** 仅要求提供电子邮件地址。
**第二次信息收集（在用户再次使用或下载报告时）：** 姓名 + 公司规模 + 职位。

### 逐步收集用户资料
不要一次性询问所有信息。分多次收集用户资料：
- 第一次会话：通过电子邮件保存结果。
- 第二次会话：询问用户职位和使用场景（根据具体情境询问，而不是在表单中直接填写）。
- 第三次会话：询问公司名称和团队规模（如果用户需要团队相关功能）。

## 免费工具的SEO策略

### 着陆页结构

```
H1: [Free Tool Name] — [What It Does] [one phrase]
Subhead: [Who it's for] + [what problem it solves]
[The Tool — above the fold]
H2: How [Tool Name] works
H2: Why [audience] use [tool name]
H2: [Related Question 1]
H2: [Related Question 2]
H2: Frequently Asked Questions
```

目标关键词应出现在：H1标题、URL路径、元标题、前100个单词中，以及至少2个副标题中。

### 添加schema标记
添加 `SoftwareApplication` schema，让Google了解页面的性质：
```json
{
  "@type": "SoftwareApplication",
  "name": "Tool Name",
  "applicationCategory": "BusinessApplication",
  "offers": {"@type": "Offer", "price": "0"},
  "description": "..."
}
```

### 提高链接获取潜力
工具可以通过以下途径吸引外部链接：
- 资源页面（如“X领域的最佳免费工具”）
- 博文文章（如“我使用的X工具”）
- Subreddit论坛、Slack社区、Facebook群组
- 你所在行业的每周新闻通讯

在发布前，先规划好外展联系的对象。谁会撰写关于你所在领域工具的文章？发布后主动联系他们。

## 数据跟踪

从第一天开始跟踪以下指标：

| 指标 | 说明内容 | 工具 |
|--------|------------------|------|
| 工具使用情况（会话次数、完成率） | 是否有人在使用该工具？ | 使用Google Analytics 4（GA4）/ Plausible工具 |
| 潜在客户转化率 | 工具是否有效生成潜在客户？ | 使用CRM和GA4事件数据 |
| 自然访问量 | 工具是否在搜索引擎中排名靠前？ | 使用Google Search Console |
| 引荐来源域名 | 工具是否获得了外部链接？ | 使用Ahrefs / Google Search Console |
| 通过电子邮件获取的潜在客户转化率 | 工具是否有助于生成潜在客户？ | 使用CRM追踪数据 |
| 离开页面的比率 / 在页面上的停留时间 | 用户是否真正使用了该工具？ | 使用GA4数据 |

**发布90天后的目标：**
- 自然访问量：每月500次以上会话。
- 潜在客户转化率：完成填写的用户的5–15%。
- 引荐来源域名：10个以上的外部链接。

运行 `scripts/tool_roi_estimator.py` 根据你的访问量和转化率预测，估算达到盈亏平衡点的时间。

## 主动提醒注意事项

以下情况需要及时提醒并采取相应措施：
- **工具使用前需要注册账户** → 标记这个问题并重新设计信息收集方式。这会损害SEO效果和工具的传播潜力，同时让用户觉得你只是在收集数据而非提供价值。
- **没有可分享的结果** → 如果结果只能在当前会话中查看且无法分享或保存，那么这个工具的功能还不完整。要及时发现这个问题。
- **未验证关键词的可行性** → 如果在开发前没有根据搜索量验证工具的可行性，需要重新考虑是否继续开发——3小时的研究比开发一个没人会搜索的工具更有价值。
- **有类似免费工具的竞争对手** → 如果已有成熟的免费工具，那么你的工具需要具备“至少好10倍”的优势才值得开发。否则应放弃这个想法。
- **单一输入对应单一输出** —— 非常简单的工具很容易失去SEO价值，也难以吸引外部链接。如果工具需要更复杂的功能才能有链接价值，应及时调整。
- **没有维护计划** —— 当工具依赖的API发生变化或逻辑过时，免费工具就会失去吸引力。在发布前需要确定是否有专人负责维护。

## 输出成果

| 你请求的内容 | 你会得到的内容 |
|---------------------|------------|
| “评估我的工具想法” | 根据6个评估因素对所有想法进行评分的对比矩阵，以及排名和建议理由 |
| “设计这个工具” | 用户体验设计规范：输入字段、输出结果、潜在客户信息收集流程、分享方式、着陆页大纲 |
| “编写着陆页内容” | 完整的着陆页文案：H1标题、副标题、功能说明、常见问题解答、元标题和描述 |
| “规划发布” | 发布前的检查清单、发布渠道列表及具体行动方案、外展联系对象列表 |
| “设置数据跟踪” | GA4事件跟踪计划、Google Search Console设置清单、30天/60天/90天的KPI目标 |
| “这个工具值得开发吗？” | 使用 `tool_roi_estimator.py` 计算的ROI模型：盈亏平衡月数、所需访问量、潜在客户价值阈值 |

## 沟通方式

所有输出内容都遵循以下结构化的沟通标准：
- **先给出结论** —— 先给出建议，再解释原因。
- **数据支持** —— 提供具体的访问量目标、转化率、投资回报率预测。
- **明确标注信息来源** —— 使用 🟢 表示已验证、🟡 表示估算、🔴 表示假设。
- **开发决策是明确的** —— 明确表示“开发”或“不开发”，并给出理由，而不是含糊地说“视情况而定”。

## 相关技能
- **SEO审计**：用于审计现有页面和关键词策略，不适用于开发新的工具相关内容。
- **内容策略**：用于规划整体内容计划（如博客、指南、白皮书），不适用于特定工具的潜在客户生成策略。
- **文案写作**：用于编写工具着陆页的营销文案，不适用于工具的用户体验设计或潜在客户收集策略。
- **发布策略**：用于规划整个产品或功能的发布，不适用于特定工具的发布策略。
- **数据分析与跟踪**：用于实施工具的数据跟踪系统，不适用于决定需要跟踪哪些指标。
- **表单优化**：用于优化工具中的潜在客户信息收集表单，不适用于工具的设计或发布策略。
