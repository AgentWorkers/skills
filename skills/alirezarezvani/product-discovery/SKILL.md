# 产品发现

通过结构化的发现过程来识别高价值的机会，并降低产品开发的风险。

## 适用场景

适用于以下场景：
- 机会解决方案树的构建
- 假设映射与测试计划制定
- 问题验证访谈及证据收集
- 基于原型或实验的解决方案验证
- 发现冲刺的规划与成果总结

## 核心发现工作流程

1. 明确期望的结果
- 确定一个可衡量的改进目标。
- 设定基准线和目标时间范围。

2. 构建机会解决方案树（OST）
- 结果 -> 机会 -> 解决方案思路 -> 实验
- 确保机会的依据是用户需求，而非内部观点。

3. 假设映射
- 识别产品的可取性、可行性、可实现性和可用性等方面的假设。
- 根据风险和确定性对假设进行评分。

**使用示例：**
```bash
python3 scripts/assumption_mapper.py assumptions.csv
```

4. 验证问题
- 进行访谈和行为分析。
- 确认问题的发生频率、严重程度以及用户解决问题的意愿。
- 及时淘汰不具潜力的机会。

5. 验证解决方案
- 在正式开发前先制作原型。
- 进行概念测试、可用性测试和价值测试。
- 关注用户实际行为，而不仅仅是用户表达的偏好。

6. 规划发现冲刺
- 设定1-2周的周期，并明确假设。
- 每天审查收集到的证据。
- 最后做出决策：继续推进、调整方向或终止项目。

## 机会解决方案树（Teresa Torres）

结构如下：
- 结果：你希望实现的指标
- 机会：未满足的客户需求或痛点
- 解决方案：可能的干预措施
- 实验：能够最快带来反馈的行动

质量检查标准：
- 在最终确定方案之前，至少需要识别出3个不同的机会。
- 每个主要机会至少对应2个实验。
- 确保每个分支都有相应的证据支持。

## 假设映射

假设分类：
- 可取性：用户是否真的需要这个功能
- 可行性：该功能是否具有商业价值
- 可实现性：团队是否有能力开发或运营它
- 可用性：用户是否能够顺利使用该功能

优先级规则：
- 风险较高且不确定性较大的假设应优先进行验证。

## 问题验证方法

- 通过访谈了解用户当前的行为习惯
- 绘制用户使用产品的流程图（Journey Friction Mapping）
- 综合分析支持工单和销售电话中的信息
- 运用行为分析方法进行验证

**证据收集标准：**
- 多个目标用户中存在相同的痛点
- 可观察到的用户应对策略
- 当前问题带来的可量化成本

## 解决方案验证方法

- 概念测试（理解产品的价值主张）
- 原型可用性测试（评估任务完成情况）
- 通过模拟用户场景进行测试（Fake Door或Concierge Test）
- 小规模测试用户群体（评估用户留存率和激活情况）

## 发现冲刺规划

建议的10天工作计划：
- 第1-2天：明确目标与机会范围
- 第3-4天：假设映射与测试设计
- 第5-7天：问题与解决方案的验证
- 第8-9天：证据收集与决策选项分析
- 第10天：相关方决策讨论

## 工具支持

### `scripts/assumptionmapper.py`

这是一个命令行工具（CLI），用于：
- 从CSV文件或直接输入中读取假设信息
- 根据风险和确定性对假设进行评分
- 生成优先级排序的测试计划，并建议相应的测试类型

更多关于该工具的详细信息，请参阅 `references/discovery-frameworks.md`。
