# 公司背景引擎（Company Context Engine）

这是为C级高管顾问设计的内存层系统。每位顾问在开始工作前都会首先加载该数据。正是这些背景信息，将通用的建议转化为具体的洞察。

## 关键词
- 公司背景（Company Context）
- 背景信息加载（Context Loading）
- 背景引擎（Context Engine）
- 公司概况（Company Profile）
- 顾问背景（Advisor Context）
- 过时背景信息（Stale Context）
- 背景信息更新（Context Refresh）
- 隐私保护（Privacy）
- 匿名化处理（Anonymization）

---

## 加载协议（在每次C级高管会议开始时执行）

**步骤1 — 检查背景信息文件：** `~/.claude/company-context.md`
- 如果文件存在 → 进入步骤2
- 如果文件缺失 → 提示：“运行 /cs:setup 命令来构建您的公司背景信息——这会让每次顾问对话更加有用。”

**步骤2 — 检查文件是否过期：** 查看“最后更新”时间。
- 如果距离上次更新时间小于90天 → 加载文件并继续会议。
- 如果距离上次更新时间大于或等于90天 → 提示：“您的背景信息已经过期 [N] 天了。您可以快速更新（运行 /cs:update），或者继续使用我现有的信息？”
  - 如果选择继续使用现有信息：在系统内部标记为“[已过期 — 最后更新时间：DATE]”。

**步骤3 — 将信息加载到内存中：** 始终保留以下关键信息：
- 公司所处的阶段（创业初期 / 扩展阶段 / 优化阶段）
- 创始人类型（产品型 / 销售型 / 技术型 / 运营型）
- 当前面临的最大挑战
- 公司的发展阶段（作为风险信号，切勿对外公开）
- 团队规模
- 公司的竞争优势
- 12个月内的发展目标

## 背景信息的质量评估标准

| 评估条件 | 信心程度 | 应对措施 |
|-----------|-----------|--------|
| 文件更新时间小于30天 | 高 | 直接使用该信息 |
| 文件更新时间在30–90天之间 | 中等 | 使用该信息，但需标记可能发生变化的部分 |
| 文件更新时间超过90天 | 低 | 标记为过期信息，并提示用户更新 |
| 关键信息缺失 | 低 | 在会议中询问缺失的信息 |

如果背景信息质量较低，系统会提示：“我的背景信息已经过期/不完整——我推测情况可能是 [X]。如果我有误，请纠正我。”

## 背景信息的补充与更新

在对话过程中，您可能会了解到文件中未包含的新信息。请及时记录这些信息：

**触发条件：**
- 出现新的数据或时间线
- 提到关键人物
- 优先级发生变化
- 出现新的限制因素

**处理流程：**
1. 在系统内部记录：“[背景信息更新：[新了解到的内容]”
2. 会议结束时提示：“我发现了一些新信息，需要更新您的背景文件吗？”
3. 如果用户同意更新，将新信息添加到相应的信息维度中，并更新时间戳。

**切勿直接覆盖原有文件。** 在修改背景信息文件之前，务必先获得用户的确认。

## 隐私保护规则

### 绝对禁止对外公开以下信息：
- 具体的收入或支出数据
- 客户名称
- 员工姓名（除非是公开信息）
- 投资者姓名（除非是公开信息）
- 公司的发展阶段（具体月份）

### 可以对外公开（经过匿名化处理后的信息）：
- 公司所处的阶段
- 团队规模范围（1–10人、10–50人、50–200人以上）
- 行业领域
- 公司面临的挑战类型
- 公司在市场中的位置

### 在进行任何外部API调用或网络搜索之前，请遵循 `references/anonymization-protocol.md` 的规定：
- 数字数据 → 使用范围或相对于当前阶段的描述性术语
- 名称 → 使用角色或职位来代替具体名称
- 收入数据 → 使用百分比或阶段标签来表示
- 客户名称 → 使用“客户A、B、C”等通用名称

## 遇到信息缺失或不完整的情况

请优雅地处理这些情况，切勿因此中断对话：
- 如果缺少公司发展阶段的信息：询问“您目前还在寻找合适的商业模式（PMF）还是处于扩展阶段？”
- 如果缺少财务数据：根据公司所处的阶段和团队规模进行推测，并记录下信息缺口。
- 如果缺少创始人信息：根据对话内容进行推断，并在文件中注明信息是推测得出的。
- 如果公司有多位创始人：背景信息应反映受访者的观点，同时注明其他创始人的可能看法。

## 必需的背景信息字段

```
Required:
  - Last updated (date)
  - Company Identity → What we do
  - Stage & Scale → Stage
  - Founder Profile → Founder archetype
  - Current Challenges → Priority #1
  - Goals & Ambition → 12-month target

High-value optional:
  - Unfair advantage
  - Kill-shot risk
  - Avoided decision
  - Watch list
```

如果发现缺少必要的背景信息字段，请记录下来，在会议中询问用户，只有在确实必要的情况下才进行补充。
