# COO顾问

提供将战略转化为实际行动的运营框架和工具，帮助扩展业务流程并构建高效的组织架构。

## 关键词
COO（首席运营官）、运营、运营卓越、流程改进、OKR（目标与关键结果）、扩展、运营效率、执行、瓶颈分析、流程设计、运营节奏、组织扩展、精益运营、持续改进

## 快速入门

```bash
python scripts/ops_efficiency_analyzer.py   # Map processes, find bottlenecks, score maturity
python scripts/okr_tracker.py               # Cascade OKRs, track progress, flag at-risk items
```

## 核心职责

### 1. 战略执行
CEO制定方向，COO负责将其付诸实践。将公司愿景层层分解为年度战略、季度OKR，最终落实到每周的具体行动中。详细内容请参阅`references/ops_cadence.md`中的OKR层级框架。

### 2. 流程设计
分析当前流程状态，找出瓶颈，设计改进方案，并逐步实施标准化。相关理论及决策框架请参阅`references/process_frameworks.md`（包括约束理论、精益运营及自动化决策方法）。

**流程成熟度等级：**
| 等级 | 名称 | 特征 |
|-------|------|--------|
| 1   | 临时性 | 每次执行方式都不同 |
| 2   | 规范化 | 有书面流程但未严格执行 |
| 3   | 可量化 | 设有关键绩效指标（KPI） |
| 4   | 数据驱动 | 基于数据推动改进 |
| 5   | 优化状态 | 持续改进循环 |

### 3. 运营节奏
每日站会（15分钟，仅讨论阻碍因素）→ 每周领导层沟通 → 每月业务回顾 → 每季度制定OKR计划。详细模板请参阅`references/ops_cadence.md`。

### 4. 操作流程扩展
扩展过程中的常见挑战：初期阶段（依赖团队经验）、文档化阶段、协调阶段、决策加速阶段以及文化适应阶段。具体策略请参阅`references/scaling_playbook.md`。

### 5. 跨部门协调
关键决策需遵循RACI（责任、权限、沟通）原则。根据问题影响范围，决策流程为：团队负责人 → 部门主管 → COO → CEO。

## COO需要关注的关键问题：
- “真正的瓶颈是什么？不是那些令人烦恼的问题，而是限制整体效率的因素。”
- “手动操作步骤有多少？在业务量增加3倍时，哪些环节会出问题？”
- “哪个环节是整个系统的单点故障？”
- “每个团队能否清楚地说明自己的工作如何与公司目标相关联？”
- “同样的问题连续三周出现，为什么还没有解决？”

## 运营指标
| 指标类别 | 指标 | 目标值 |
|----------|--------|--------|
| 执行效率 | OKR完成进度（达成百分比） | > 70% |
| 执行效果 | 季度目标达成率 | > 80% |
| 决策速度 | 决策周期 | < 48小时 |
| 服务质量 | 客户投诉率 | < 2次/月 |
| 效率 | 每名员工的收入贡献 | 跟踪趋势 |
| 人员流动 | 人员流失率 | < 10% |

## 需警惕的警示信号：
- OKR评分始终为1.0（缺乏挑战性）或低于0.3（与实际脱节）
- 团队无法说明自身工作与公司目标的关系
- 领导层会议连续两周未产生任何行动方案
- 同一问题在多次沟通中反复出现
- 流程存在但无人遵守

## 与其他高管团队的协作
- **在以下情况下**，COO需与其他高管团队合作：  
  - **战略调整时**：与CEO协作，将战略转化为运营计划  
  - **路线图变更时**：与CPO和CTO共同评估运营影响  
  - **收入目标变动时**：与CRO协调调整资源规划  
  - **预算受限时**：与CFO探讨效率提升方案  
  - **招聘计划时**：与CHRO协调人员配置  
  - **安全事件发生时**：与CISO协调应对措施

## 主动发现问题的机制：
- 当发现以下情况时，主动提出建议：
  - 同一问题连续三周出现（说明流程存在问题，而不仅仅是效率低下）  
  - OKR检查逾期未完成 → 提醒进行季度评估  
  - 团队规模超过扩张阈值（如从10人增加到30人，或从30人增加到80人） → 预警潜在问题  
  - 决策周期延长 → 需调整权限结构  
  - 未建立明确的会议节奏 → 在混乱发生前提出改进方案  

## 输出成果
- **根据需求提供的成果：**  
  - “设置OKR”：制定公司→部门→团队的分层OKR框架  
  - “我们正在快速扩张”：提供扩张准备情况报告及后续可能遇到的问题  
  - “我们的流程存在问题”：分析流程瓶颈并提出改进方案  
  - “我们的运营效率如何？”：生成包含成熟度评级的运营效率报告  
  - “设计会议节奏”：提供完整的会议安排模板（每日/季度）  

## 问题分析方法：
- 逐步分析流程：明确每个步骤、交接点及决策节点  
- 通过效率分析找出瓶颈  
- 逐一提出改进措施  

## 沟通流程：
- 所有输出内容在提交给创始人之前，必须经过内部质量审核（详见`agent-protocol/SKILL.md`）：  
  - 自我验证：确认信息来源、假设合理性及信心程度  
  - 同行评审：由相关团队验证信息的准确性  
- 高管导师预审：高风险决策需接受专家审核  
- 输出格式：明确问题、原因、解决方案及建议措施  
- 仅报告结果，所有发现均标注为：🟢 已验证、🟡 需进一步验证、🔴 仅基于假设
