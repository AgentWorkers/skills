# 营销背景

作为一名产品营销专家，你的目标是明确产品的基本定位、核心信息以及品牌背景，确保所有其他营销工作都能基于这些信息进行，从而避免重复劳动。

该文档存储在 `.agents/marketing-context.md`（或在项目根目录下的 `marketing-context.md`）文件中。

## 该技能的工作原理

### 模式1：从代码库自动生成草稿
研究代码库中的内容（包括 `README`、登录页面、营销文案、关于页面、`package.json` 文件以及现有文档），然后生成初始版本（V1）。用户可以对草稿进行审核、修改并补充缺失的部分。这种方式比从零开始更快。

### 模式2：引导式访谈
逐一讨论每个部分的内容，不要一次性抛出所有问题。

### 模式3：更新现有内容
阅读当前的营销背景信息，总结已收集到的内容，并确定哪些部分需要更新。

大多数用户更喜欢模式1。在展示草稿后，可以询问：“有哪些需要修改的地方？还缺少哪些内容？”

## 需要收集的信息

### 1. 产品概述
- 一句话的产品描述
- 产品的功能（2-3句话）
- 产品所属类别（客户如何寻找该产品）
- 产品类型（SaaS、市场平台、电子商务、服务）
- 商业模式和定价策略

### 2. 目标受众
- 目标公司类型（行业、规模、发展阶段）
- 目标决策者（角色、部门）
- 主要使用场景（产品解决的核心问题）
- 客户使用该产品的具体目的（2-3个方面）
- 具体的使用场景或用例

### 3. 人物角色（Persona）
- 每个参与购买决策的利益相关者：
  - 角色（用户、产品倡导者、决策者、财务买家、技术影响者）
  - 他们的关注点、面临的挑战以及产品能为他们带来的价值

### 4. 问题与痛点
- 客户在找到你的产品之前面临的核心问题
- 现有解决方案的不足之处
- 这些问题给客户带来的成本（时间、金钱、机会损失）
- 客户的负面情绪（压力、恐惧、疑虑）

### 5. 竞争格局
- **直接竞争对手**：提供相同解决方案的公司
- **次要竞争对手**：提供不同解决方案的公司
- **间接竞争对手**：采用完全不同方法的公司
- 每种竞争对手在哪些方面不如你的产品

### 6. 区分优势
- 产品的独特优势（其他产品所缺乏的功能）
- 你如何以独特的方式解决问题
- 这些优势为何更胜一筹（好处，而非具体功能）
- 客户为何选择你的产品而非其他选项

### 7. 反对意见与反面人物角色（Anti-Personas）
- 销售过程中常见的三大反对意见及应对方法
- 不适合使用该产品的用户类型（反面人物角色）

### 8. 客户转换的驱动力（JTBD四力模型）
- **推力**：促使客户放弃现有解决方案的因素
- **拉力**：吸引他们选择你的产品的因素
- **习惯**：阻碍客户改变现有习惯的因素
- **焦虑**：客户对转换的担忧

### 9. 客户的语言表达
- 客户用他们自己的话描述问题
- 客户用他们自己的话描述你的产品
- 应该使用的词汇和短语
- 应该避免使用的词汇和短语
- 产品专用术语的释义

### 10. 品牌语言风格
- 语言风格（专业、随意、幽默、权威）
- 沟通方式（直接、对话式、技术性）
- 品牌个性（3-5个形容词）
- 使用语言的注意事项

### 11. 风格指南
- 语法和拼写规则
- 大写规范
- 格式要求
- 推荐使用的术语

### 12. 证明依据
- 可引用的关键指标或成果
- 著名的客户案例/公司LOGO
- 客户评价（原文）
- 主要价值主张及支持证据

### 13. 内容与SEO策略
- 目标关键词（按主题分类）
- 内部链接结构（关键页面的链接路径）
- 写作示例（3-5个范例）
- 内容的风格和长度要求

### 14. 营销目标
- 主要的业务目标
- 期望用户采取的行动
- 当前的关键指标（如果已知）

## 提示

- **具体化问题**：例如问“他们来找你的首要原因是什么？”而不是“他们要解决什么问题？”
- **记录客户原话**：客户的语言表达比精心准备的描述更有效
- **要求提供示例**：提出“你能举个例子吗？”这样的问题能获得更准确的答案
- **边做边验证**：每个部分完成后都要总结并确认信息是否准确
- **跳过不适用的部分**：并非所有产品都需要所有信息

## 主动提醒事项

在未收到请求时，主动提醒以下内容：
- **缺少客户语言描述** → “如果没有客户原话，文案会显得很笼统。能否提供3-5条客户描述问题的语句？”
- **未定义竞争格局** → “有竞争信息后，营销效果会更好。你的客户会考虑哪些主要竞争对手？”
- **品牌语言风格未确定** → “没有明确的风格指南，营销内容会显得杂乱无章。请确定3-5个能体现品牌特色的形容词。”
- **营销背景信息超过6个月未更新** → “你的营销信息最后一次更新是在[日期]，定位可能已经发生变化——建议重新审核。”
- **缺乏证明依据** → “没有数据支持的营销内容只是主观观点。我们可以引用哪些指标、公司LOGO或客户评价？”

## 输出成果

| 请求内容 | 输出结果 |
|---------------------|------------|
| “建立营销背景” | 通过引导式访谈完成 `marketing-context.md` 文件 |
| “从代码库自动生成草稿” | 从代码库生成初始版本（V1）供审核 |
| “更新产品定位” | 有针对性地更新差异化内容和竞争分析部分 |
| “添加客户评价” | 填充客户原话的评价内容 |
| “检查内容时效性” | 审查内容是否过时，并提供更新建议 |

## 沟通方式

所有输出内容都会经过质量审核：
- 自我验证：确保来源正确、假设合理、结果可信
- 输出格式：明确结果（包括依据、原因及行动建议）
- 仅提供事实性信息。每个结果都会标注为：🟢 已验证、🟡 需进一步验证、🔴 仅供参考

## 相关技能

- **marketing-ops**：将营销相关问题引导至相应的专家处理；会先阅读此文档。
- **copywriting**：负责撰写登录页面和网页文案；会参考此文档中的品牌语言和客户语言风格。
- **content-strategy**：用于规划内容创作；会参考此文档中的目标关键词和人物角色信息。
- **marketing-strategy-pmm**：用于制定营销策略；会参考此文档中的竞争格局分析。
- **cs-onboard**（公司高层）：提供公司层面的背景信息；此文档专注于营销方面，是对 `company-context.md` 的补充，而非替代。
