# 战略协同引擎

战略的失败通常发生在执行层面，而非董事会会议中。本技能能够在问题恶化之前发现协同失调的情况，并构建系统以确保从首席执行官到基层员工的战略目标保持一致。

## 关键词
战略协同、战略执行流程、OKR（关键绩效指标）协同、孤立的目标、目标冲突、部门壁垒、沟通障碍、部门对齐、对齐检查工具、战略清晰度、跨职能协作、目标层级结构、协同度评估

```bash
python scripts/alignment_checker.py    # Check OKR alignment: orphans, conflicts, coverage gaps
```

## 核心框架

协同问题在于：**目标与制定该目标的战略之间的差距越大，该目标就越可能偏离最初的意图。**这种情况在组织的各个层级都会发生。关键在于问题的严重程度以及如何解决它。

### 第一步：战略清晰度测试

在检查战略执行流程之前，先询问五个不同团队的成员：
**“公司目前最重要的战略优先事项是什么？”**

**评分标准：**
- 如果所有五个人的答案都相同：✅ 战略清晰度良好
- 如果有3-4个人答案相似：🟡 协同度较低——需要进一步明确和沟通
- 如果少于3个人同意：🔴 战略不够清晰，无法继续执行下一步。

**测试要求：**战略应该能够用一句话表达清楚。如果需要用一段话来说明，说明领导层可能没有将战略真正传达给员工。
- 示例错误表述：“我们专注于产品驱动的增长，同时维护与企业关系、拓展国际业务并投资平台建设”
- 正确表述：“在B轮融资之前，赢得DACH地区的中端医疗市场”

### 第二步：战略执行流程映射

绘制从公司战略到各个层级的执行流程图。

```
Company level:  OKR-1, OKR-2, OKR-3
    ↓
Dept level:     Sales OKRs, Eng OKRs, Product OKRs, CS OKRs
    ↓
Team level:     Team A OKRs, Team B OKRs...
    ↓
Individual:     Personal goals / rocks
```

**对于每个层级的目标，需要回答以下问题：**
- 这个目标支持哪个公司层面的战略目标？
- 如果这个目标完全实现，会对公司整体目标产生多大的推动作用？
- 这种关联是直接的还是理论上的？

### 第三步：协同度检测

存在三种典型的协同失调模式：

**孤立的目标：**某个团队或个人的目标与公司整体目标无关。
- 症状：“我们已经为此工作了一个季度，但上级似乎并不关心。”
- 原因：目标可能是自下而上设定的，或者基于上个季度的优先事项，没有与当前的OKR对齐。
- 解决方法：为每个目标指定一个上级目标。

**目标冲突：**当两个团队的目标同时实现时，可能会导致更糟糕的结果。
- 典型例子：销售团队追求销售量（收入），而客户服务团队关注客户满意度；结果可能是销售业绩不佳，客户服务团队满意度下降。
- 解决方法：在季度开始前进行跨职能的OKR审查，并设定共同的评估指标。

**覆盖范围不足：**公司有三个OKR，5个团队支持OKR-1，2个团队支持OKR-2，0个团队支持OKR-3。
- 症状：OKR-3始终无法实现；没有人负责推动它的实现。
- 解决方法：明确指定负责团队。如果没有任何团队负责某个公司OKR，那么这个目标就无法实现。

有关自动化检测工具的详细信息，请参阅`scripts/alignment_checker.py`（基于JSON格式的OKR数据）。

### 第四步：部门壁垒识别

当团队为了局部指标而优化工作时，就会形成部门壁垒。
- 确认壁垒的信号：
- 某个部门始终能够完成自己的目标，但公司整体目标未能实现
- 团队之间不了解其他团队的工作内容
- 常见的回应是“那不是我们的问题”
- 问题只会向上级反映，而不会在团队之间进行横向协调
- 依赖彼此的团队之间不共享数据

**壁垒的成因：**
1. **激励机制不匹配：**团队根据局部指标获得奖励，因此不会优化公司整体目标
2. **缺乏共同目标：**当团队有共同目标时，他们会协同工作；否则就会各自为政
3. **缺乏共同的语言：**技术团队不了解销售团队的指标，销售团队也不理解技术团队的工作负担
4. **地理或时区差异：**当团队之间缺乏自然互动时，壁垒会加剧

**评估方法：**
- 团队之间相互请求帮助的频率与独立工作的频率
- 解决跨职能问题所需的时间
- 团队成员能否描述相邻团队的当前工作重点

### 第五步：沟通障碍分析

首席执行官传达的战略与团队实际理解的战略之间存在差距。这种差距会随着公司规模的扩大而加剧。
**信息传递的衰减过程：**
- 首席执行官向全体员工传达战略 → 经理们根据自己的理解进行过滤 → 团队接收到经过修改的信息 → 员工进一步根据自己的理解进行解读

**沟通障碍的来源：**
- **信息模糊：**战略表述过于抽象（如“发展业务”），导致团队自行解读
- **沟通频率：**每季度仅进行一次全体会议不足以改变员工的行为
- **沟通方式不匹配：**书面战略文件过长，而团队更习惯于视觉化的沟通方式
- **信任缺失：**团队认为战略不真实（“我们之前听过类似的表述”）

**检测方法：**
- 在所有层级重复进行第一步的战略清晰度测试
- 比较领导层认为传达的内容与团队实际听到的内容
- 进行调查：“自上次战略更新以来，你们的工作方式发生了哪些变化？”

### 第六步：协同调整机制

如何在不引起恐慌的情况下解决协同失调问题？

**步骤6a：不要从问题本身开始**
直接指出问题只会引发抵触情绪。应该先说明“我们的目标方向是什么，以及我们需要确保所有团队保持一致”。

**步骤6b：通过研讨会进行调整，而非通过备忘录**
研讨会比书面文件更有效。召集公司层面的OKR负责人和各部门负责人，共同梳理目标之间的关联，找出存在的问题。

**步骤6c：调整激励机制**
如果部门负责人因为追求局部指标而获得奖励，而这些指标与公司整体目标相冲突，那么仅仅调整目标是无法解决问题的。必须先改变激励机制。

**步骤6d：建立季度性协同检查机制**
调整后，需要防止类似问题的再次发生。有关季度性检查的详细信息，请参阅`references/alignment-playbook.md`。

## 协同度评估

这是一个快速的评估工具，每个方面的得分范围为0-10分：

| 评估项 | 评估内容 | 分数 |
|------|----------|-------|
| 战略清晰度 | 不同团队的五个人能否一致地描述公司的战略？ | /10 |
| 执行流程完整性 | 所有团队的目标是否都与公司目标一致？ | /10 |
| 冲突检测 | 跨团队之间的OKR冲突是否已被识别并解决？ | /10 |
| 目标覆盖范围 | 每个公司的OKR是否有明确的负责团队？ | /10 |
| 沟通效果 | 团队的行为是否真正反映了战略意图（而不仅仅是口头表述）？ | /10 |

**总分：__ / 50**

| 分数 | 评估结果 |
|-------|--------|
| 45–50 | 非常好。继续保持现有机制。 |
| 35–44 | 较好。需要针对薄弱环节进行改进。 |
| 20–34 | 协同失调正在造成损失。需要立即采取措施。 |
| < 20 | 战略出现严重偏差。视为危机。 |

## 协同相关的关键问题：

- “询问新加入的团队成员：公司目前最重视的目标是什么？”
- “你们团队的首要任务支持哪个公司的OKR？你能说明这种关联吗？”
- “当A团队和B团队都完成目标时，公司是否总能取得成功？是否存在例外情况？”
- “自上次战略更新以来，你们团队的工作方式发生了哪些变化？”
- “请举例说明上周做出的、受到公司战略影响的决策。”

## 需警惕的警示信号：

- 团队始终能够完成目标，但公司整体目标未能实现
- 跨职能项目的完成时间远超预期（沟通协调失败）
- 战略每季度更新一次，但团队优先事项却没有变化
- 团队层面存在“那是领导层的问题，与我们无关”的态度
- 新举措的推出没有与现有的OKR相结合

## 与其他高管角色的协作

| 在以下情况下 | 与谁协作 | 为了什么目的 |
|---------|-------------|-------|
| 新战略制定时 | 首席执行官 + 首席运营官 | 将新战略分解为季度性的具体目标再宣布 |
| OKR周期开始时 | 首席运营官 | 在最终确定目标前进行跨团队冲突检测 |
| 团队持续未能完成目标 | 人力资源总监 | 判断是能力不足还是协同问题 |
| 发现部门壁垒时 | 首席运营官 | 设计共同的评估指标或跨职能的OKR |
| 合并后 | 首席执行官 + 文化架构师 | 检测合并后的组织之间的战略冲突 |
