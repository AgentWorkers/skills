# 战略一致性引擎

战略的失败通常发生在执行过程中，而非在董事会会议上。该工具能够在问题变得严重之前检测到策略执行中的不一致性，并构建系统以确保从首席执行官到基层员工的战略目标保持一致。

## 核心框架

**一致性问题：**目标与制定该目标的战略之间的差距越大，其体现原始意图的可能性就越小。这种情况在组织的各个层级都会发生。问题在于这种不一致性的严重程度以及如何解决它。

### 第一步：战略清晰度测试

在检查战略执行流程之前，先从源头开始。询问来自五个不同团队的五个人：
**“公司目前最重要的战略重点是什么？”

**评分标准：**
- 如果五个人都给出相同的答案：✅ 战略清晰度良好
- 如果有3-4个人给出相似的答案：🟡 一致性较弱——需要进一步澄清和沟通
- 如果少于3个人同意：🔴 战略不够清晰，无法继续执行。

**测试要求：**战略应该能够用一句话概括。如果需要用一段话来描述，那么团队很可能无法真正理解并执行这一战略。
- 示例：❌ “我们专注于产品驱动的增长，同时维护与企业关系、拓展国际业务并投资平台能力”
- 示例：✅ “在B轮融资之前，赢得DACH地区的医疗保健市场”

### 第二步：战略执行流程映射

绘制从公司战略到组织各层级的执行流程图。

```
Company level:  OKR-1, OKR-2, OKR-3
    ↓
Dept level:     Sales OKRs, Eng OKRs, Product OKRs, CS OKRs
    ↓
Team level:     Team A OKRs, Team B OKRs...
    ↓
Individual:     Personal goals / rocks
```

**对于每个层级的目标，需要回答以下问题：**
- 这个目标支持哪个公司层面的战略目标？
- 如果这个目标完全实现，会对公司整体目标产生多大的推动作用？
- 这种目标之间的联系是直接的还是理论上的？

### 第三步：不一致性检测

存在三种常见的失败模式：

**孤立的目标：**某个团队或个人的目标与任何公司层面的战略目标都没有关联。
- **症状：**“我们已经为此工作了一个季度，但上级似乎并不关心。”
- **根本原因：**这些目标可能是自下而上设定的，或者基于上个季度的优先事项，而没有与当前的OKR对齐。
- **解决方法：**为每个目标指定一个上级目标，确保它们与整体战略保持一致。

**目标冲突：**当两个团队的目标同时实现时，可能会产生相反的效果。
- **典型例子：**销售团队追求销售量（收入指标），而客户服务团队关注客户满意度。结果可能是销售团队接手了不符合公司需求的客户，导致客户满意度下降。
- **解决方法：**在季度开始前进行跨职能的OKR审查，并制定共同的评估指标。

**覆盖范围不足：**公司有三个OKR，5个团队支持OKR-1，2个团队支持OKR-2，0个团队支持OKR-3。
- **症状：**OKR-3始终无法实现，且没有团队负责推动它的执行。
- **解决方法：**明确指定负责该目标的团队。如果没有任何团队负责某个OKR，那么这个目标就无法得到执行。

有关自动化检测工具的详细信息，请参阅`scripts/alignment_checker.py`（该工具支持JSON格式的OKR数据）。

### 第四步：部门壁垒识别

当团队为了局部指标而优化工作，而忽视公司整体目标时，就会形成部门壁垒。
- **部门壁垒的迹象：**
  - 一个部门持续实现自己的目标，但公司整体目标未能达成
  - 团队之间不了解其他团队的工作内容
  - 常见的回应是“那不是我们的问题”
  - 问题只向上级汇报，缺乏横向协调
  - 依赖彼此的团队之间不共享数据

**部门壁垒的根源：**
  1. **激励机制不一致：**团队因追求局部指标而获得奖励，因此不会优化公司整体目标
  2. **缺乏共同目标：**当团队有共同的目标时，他们会进行协作；否则就会各自为政
  3. **缺乏共同的语言：**技术人员不理解销售团队的指标，销售人员也不了解技术上的问题
  4. **地理或时区差异：**当团队之间缺乏自然交流时，壁垒会加剧

**衡量部门壁垒的方法：**
  - 团队之间相互请求帮助的频率与独立工作的频率
  - 解决跨职能问题所需的时间
  - 团队成员能否描述相邻团队的当前工作重点

### 第五步：沟通障碍分析

首席执行官传达的战略与团队实际理解的战略之间存在差距。这种差距会随着公司规模的扩大而加剧。
- **信息传递过程：**首席执行官向全体员工传达战略，经理们根据自己的理解进行过滤，团队再根据自己的理解进行解读

**沟通障碍的来源：**
  - **表述模糊：**战略目标表述得过于抽象（如“发展业务”），导致每个团队自行解读
  - **沟通频率：**每季度只进行一次全体会议不足以改变团队的行为
  - **沟通方式不匹配：**书面战略文件过于冗长，而团队更习惯于视觉化的沟通方式
  - **信任缺失：**团队不相信战略的真实性（“我们之前也听过类似的内容”）

**检测沟通障碍的方法：**
  - 对所有层级进行第一步的战略清晰度测试
  - 比较领导层认为传达的内容与团队实际理解的内容
  - 调查：“自上次战略更新以来，你们的工作方式发生了哪些变化？”

### 第六步：重新调整机制

如何在不引起恐慌的情况下解决策略执行中的不一致性问题？

**步骤6a：不要从问题开始**
直接指出问题只会引发防御心理。应该先说明“我们的目标是什么，以及如何确保各部门之间的目标保持一致”。

**步骤6b：通过研讨会而不是备忘录来重新调整战略执行流程**
研讨会比书面文件更有效。召集公司层面的OKR负责人和各部门负责人，共同梳理目标之间的联系，找出存在的问题。

**步骤6c：调整激励机制**
如果部门负责人因追求局部指标而获得奖励，而这些指标与公司整体目标相冲突，那么仅仅调整目标是无法解决问题的。必须先改变激励机制。

**步骤6d：建立季度性一致性检查机制**
调整后，需要防止类似问题的再次发生。有关季度性检查的详细信息，请参阅`references/alignment-playbook.md`。

## 一致性评分

这是一个快速评估工具，用于评估各个方面的情况（评分范围0-10分）：

| 评估项 | 问题 | 评分 |
|------|----------|-------|
| 战略清晰度 | 不同团队的五个人能否一致地描述公司的战略？ | /10 |
| 战略执行流程完整性 | 所有团队的目标是否都与公司目标一致？ | /10 |
| 冲突检测 | 跨团队之间的OKR冲突是否已被识别并解决？ | /10 |
| 目标覆盖范围 | 每个公司的OKR是否有明确的负责团队？ | /10 |
| 沟通效果 | 团队的实际行为是否与战略目标一致（而不仅仅是口头上的理解）？ | /10 |

**总分：__ / 50**

| 评分 | 状态 |
|-------|--------|
| 45–50 | 优秀。继续保持现有机制。 |
| 35–44 | 良好。需要针对具体问题进行改进。 |
| 20–34 | 不一致性问题正在造成损失，需要立即处理。 |
| < 20 | 战略方向出现偏差，视为危机。 |

## 用于评估一致性的关键问题：

- “问你的新员工：公司目前最想实现的目标是什么？”
- “你们团队的首要任务支持哪个公司的OKR？你能说明这种关联吗？”
- “当A团队和B团队都达成目标时，公司是否总能取得成功？是否存在例外情况？”
- “自上次战略更新以来，你们团队的工作方式发生了哪些变化？”
- “请列举一个上周根据公司战略做出的决策。”

## 需警惕的警示信号：

- 团队持续实现目标，但公司整体目标未能达成
- 跨职能项目的完成时间比预期长三倍（协调失败）
- 战略每季度更新一次，但团队优先事项却没有变化
- 团队层面存在“那是领导层的问题，与我们无关”的态度
- 新举措的制定没有与现有的OKR相结合

## 与其他高管团队的协作

| 在以下情况下... | 与...合作... | 为了... |
|---------|-------------|-------|
| 新战略制定时 | 首席执行官 + 首席运营官 | 在正式宣布前，先将新战略分解为季度性的具体目标 |
| OKR周期开始时 | 首席运营官 | 在最终确定目标前，进行跨团队冲突检测 |
| 团队持续未能达成目标 | 人力资源总监 | 判断是能力不足还是协调问题 |
| 发现部门壁垒时 | 首席运营官 | 设计共同的评估指标或跨职能的OKR |
| 合并后 | 首席执行官 + 文化架构师 | 检测合并后的组织之间的战略冲突 |
