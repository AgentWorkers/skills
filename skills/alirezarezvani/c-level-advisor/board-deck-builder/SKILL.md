# 平板展示模板构建工具

使用该工具可以创建能够讲述故事（而不仅仅是展示数据）的平板展示材料。每个部分都有负责人、相应的叙述内容以及“那么接下来该怎么做”的建议。

## 关键词
- 平板展示模板（Board Deck Template）
- 投资者更新（Investor Update）
- 董事会会议（Board Meeting）
- 投资者关系（Investor Relations）
- 季度回顾（Quarterly Review）
- 董事会演示（Board Presentation）
- 筹资材料（Fundraising Material）
- 投资者演示文稿（Investor Deck）
- 董事会报告（Board Report）
- 季度业务回顾（Quarterly Business Review）

## 快速入门

```
/board-deck [quarterly|monthly|fundraising] [stage: seed|seriesA|seriesB]
```

提供可用的指标数据。该工具会使用占位符来填补数据空白——绝不会凭空编造数字。

## 模板结构（标准顺序）

每个部分的结构如下：**标题 → 数据 → 叙述 → 下一步行动**

### 1. 高管总结（CEO）
**最多3句话。**
- 第一句话：业务现状（我们目前所处的位置）
- 第二句话：本季度发生的最重要事件
- 第三句话：下一季度的发展方向

*不好的例子：*“我们本季度表现良好，各个领域都取得了进展。”
*好的例子：*“我们第三季度的年收入达到了240万美元（环比增长22%），签署了最大额的企业合同，并且以14个月的资金储备进入第四季度。向中端市场的战略转型正在取得成效——平均客户价值（ACV）增长了40%，销售周期缩短到了3周。第四季度的重点是完成300万美元的A轮融资，并实现280万美元的年收入。”

### 2. 关键指标仪表盘（COO）
**最多6-8个指标。使用表格展示。**

| 指标          | 本季度      | 上一季度      | 目标        | 状态       |
|---------------|-----------|-----------|------------|-----------|
| 年收入（ARR）      | 240万美元   | 197万美元   | 230万美元   | ✅        |
| 月度增长率      | 8.1%      | 7.2%      | 7.5%      | ✅        |
| 资金消耗倍数     | 1.8倍      | 2.1倍      | <2倍      | ✅        |
| 新客户留存率（NRR）    | 112%      | 108%      | >110%      | ✅        |
| 平均客户获取成本（CAC）回收期 | 11个月     | 14个月     | <12个月     | ✅        |
| 员工人数        | 24人      | 21人      | 25人      | 🟡        |

选择董事会实际关注的关键指标。如果某些指标对董事会来说不重要，可以省略。

### 3. 财务更新（CFO）
- 利润与损失（P&L）摘要：收入、销售成本（COGS）、毛利率、运营费用（OpEx）、净资金消耗
- 现金状况及资金储备时间（以月为单位）
- 资金消耗趋势（过去三个季度）
- 与计划的偏差（差异及原因）
- 下一季度的预测更新

**每个偏差都需要用一句话解释。**董事会不喜欢只说“收入低于目标”而不给出原因。

### 4. 收入与销售管线（CRO）
- 年收入（ARR）的分布情况：初期阶段、新客户、扩展阶段、流失客户、最终阶段
- 新客户留存率（NRR）和客户流失率
- 按阶段划分的销售管线（以美元计，而非数量）
- 下一季度的预测及置信度

**预测必须附带置信度。**“我们预计年收入为280万美元”这样的表述不够具体。“我们有90%的信心达到260万美元，如果两个后期阶段的交易能够完成，收入可能会达到290万美元”这样的表述更为有用。

### 5. 产品更新（CPO）
- 本季度发布的功能：列出3-5项，并说明每项功能对用户的影响
- 下一季度计划发布的功能：列出3-5项，并提供目标发布日期
- 产品市场反馈（PMF）指标：净推荐值（NPS）趋势、日活跃用户（DAU）/月活跃用户（MAU）比例、功能采用情况
- 从客户调研中得出的关键发现

**不要列出所有功能。**只展示那些对用户有实际影响的功能。

### 6. 成长与营销（CMO）
- 按渠道划分的平均客户获取成本（CAC）
- 各渠道对销售管线的贡献（以美元计）
- 与当前阶段相关的品牌/知名度指标（流量、市场影响力）
- 哪些策略有效，哪些需要调整，哪些正在测试中

### 7. 工程与技术（CTO）
- 过去四个季度的交付速度趋势
- 技术债务比率及应对计划
- 基础设施状况：运行时间、故障情况、成本趋势
- 安全性状况（简要说明，如有待解决的问题需特别标注）

**除非有重大问题，否则这部分内容应保持简短。**董事会不需要了解详细的开发进度。

### 8. 团队与人员（CHRO）
- 实际员工人数与计划人数对比
- 招聘情况：发出的招聘信息、招聘进度、填补空缺所需时间
- 人员流失情况：哪些流失是不可避免的，哪些是可以避免的
- 员工满意度：最近一次调查的结果及趋势
- 本季度的重要招聘人员、当前空缺职位

### 9. 风险与安全（CISO）
- 安全性状况：关键控制措施的执行情况
- 合规性：正在进行的认证项目及截止日期
- 本季度发生的安全事件（如有）：影响、解决方案、预防措施
- 三大风险及应对措施

### 10. 战略展望（CEO）
- 下一季度的重点任务：按优先级排列的3-5项任务
- 需要董事会做出的关键决策
- 提出的请求：预算、人员推荐、建议、投票事项

**“请求”这部分最为重要。**要具体明确。“我们希望获得3位B轮融资公司的CFO的推荐”比“希望得到任何帮助”更具针对性。

### 11. 附录
- 详细的财务模型
- 完整的销售管线数据
- 客户案例研究
- 详细的员工人数分布

## 叙述框架

董事会每个季度会收到10份以上的展示材料，因此你的材料需要有一个清晰的主题主线。

**四步叙述结构：**
1. **我们原本的目标**（上一季度设定的目标）
2. **我们实际的情况**（客观的评估）
3. **产生差异的原因**（每个偏差都要有一个具体原因，而不是找借口）
4. **我们的应对措施**（具体且带有时间安排的行动计划）

这种结构适用于好消息和坏消息的展示。因为它能够诚实地反映实际情况。

**开场方式：**从最重要的信息开始——董事会应在第三张幻灯片之前就了解核心内容。

## 如何传达坏消息

永远不要隐瞒坏消息。董事会迟早会知道，延迟透露只会让情况更糟。

**沟通框架：**
1. **直接说明**——“我们第三季度的年收入目标未能实现，差了30万美元（12%的差距）”
2. **承认原因**——“主要原因在于企业客户群体的销售周期比预期更长”
3. **表明理解问题**——“我们分析了8笔未能成交或进展缓慢的交易；问题在于……”
4. **提出解决方案**——“我们已经做出了3项调整：[具体措施及实施时间]”
5. **更新预测**——“修订后的第四季度目标是260万美元；以下是详细的预测依据”

**不要这样做：**
- 不要用好消息来缓和坏消息——董事会会察觉到这种做法，并对沟通方式产生怀疑
- 不要在不承担责任的情况下解释问题——“市场环境”只是背景信息，而非真正的原因
- 不要在没有数据支持的情况下提出解决方案
- 不要在不说明假设的情况下展示修订后的预测

## 常见的平板展示模板错误

| 错误                | 应对措施                |
|------------------|----------------------|
| 幻灯片过多（超过25张）       | 严格删减不必要的幻灯片——如果无法在会议中解释某个内容，那么这个幻灯片就是多余的 |
| 指标没有设定目标        | 每个指标都需要设定目标及相应的状态        |
| 没有叙述内容          | 仅展示数据会让董事会自行推断情况        |
| 隐藏坏消息            | 应首先揭示坏消息，然后承担责任并提出解决方案 |
| 请求不明确            | 提出的请求必须具体、可执行，并明确责任人       |
| 不解释差异原因        | 每个与目标之间的差异都需要用一句话说明原因     |
| 附录过时            | 附录只有在其内容最新时才有用           |
| 为阅读者设计而非为现场观众设计    | 幻灯片是用来口头展示的，因此设计时必须考虑现场情况    |

## 发布节奏说明

**季度报告（标准格式）：**完整的展示材料，包含所有部分，共20-30张幻灯片。提前48小时发送。
**月度报告（针对初创企业）：**精简内容——重点展示指标仪表盘、财务数据、销售管线和主要风险，共8-12张幻灯片。
**筹资报告：**以市场概况和战略愿景开头，以提出请求结尾。具体格式请参考`references/deck-frameworks.md`。
