# 公司背景引擎（Company Context Engine）

这是为高层管理顾问设计的内存层系统。每位顾问在开始工作前都会首先加载该数据。正是这些背景信息，将通用的建议转化为具体的见解。

## 关键词
- 公司背景（Company Context）
- 背景信息加载（Context Loading）
- 背景引擎（Context Engine）
- 公司概况（Company Profile）
- 顾问背景（Advisor Context）
- 过时背景信息（Stale Context）
- 背景信息更新（Context Refresh）
- 隐私保护（Privacy）
- 数据匿名化（Anonymization）

---

## 加载协议（在每次高层会议开始时执行）

**步骤 1 — 检查背景信息文件：** `~/.claude/company-context.md`
- 如果文件存在 → 进入步骤 2
- 如果文件缺失 → 提示：“运行 /cs:setup 以构建您的公司背景信息——这会让每次顾问对话更加有用。”

**步骤 2 — 检查文件是否过期：** 查看 `Last updated` 字段。
- **如果文件更新时间在 90 天以内：** 加载文件并继续使用。
- **如果文件更新时间超过 90 天：** 提示：“您的背景信息已经过期 [N] 天了。您可以快速更新（运行 /cs:update），或者继续使用我现有的信息吗？”
  - 如果选择继续使用：在系统内部标记为 “过期（最后更新时间：DATE）”。

**步骤 3 — 将信息加载到内存中：** 这些信息始终处于活跃状态：
- 公司所处的发展阶段（筹备期 / 扩展期 / 优化期）
- 创始人类型（产品型 / 销售型 / 技术型 / 运营型）
- 当前面临的主要挑战
- 公司的发展阶段（作为风险信号，切勿对外公开）
- 团队规模
- 公司的竞争优势
- 12 个月内的目标

## 背景信息的质量评估标准

| 评估条件 | 信息可靠性 | 应采取的行动 |
|-----------|-----------|--------|
| 文件更新时间 < 30 天 | 高 | 直接使用该信息 |
| 文件更新时间在 30–90 天之间 | 中等 | 使用该信息，但需标记可能发生变化的部分 |
| 文件更新时间超过 90 天 | 低 | 标记为过期信息，并提示用户更新 |
| 关键信息缺失 | 低 | 在会议中询问相关信息 |

如果背景信息质量较低，系统会提示：“我的背景信息已经过期/不完整——我推测可能是 [X] 这种情况。如果我有误，请纠正我。”

## 背景信息的补充

在对话过程中，您可能会了解到文件中未包含的新信息。请及时记录这些信息：

**触发条件：**
- 新数据的出现或时间线的更新
- 关键人物的提及
- 优先级的变化
- 新出现的约束因素

**处理流程：**
1. 在系统内部记录：“[背景信息更新：[新了解到的内容]”
2. 会议结束时：询问用户是否需要更新文件，并在文件中添加新内容，并更新时间戳。

**切勿擅自修改背景信息文件。** 在修改之前务必获得用户的确认。

## 隐私保护规则

### 绝对不得对外公开的信息：
- 公司的具体收入或支出数据
- 客户名称
- 员工姓名（除非是公开信息）
- 投资者姓名（除非是公开信息）
- 公司的发展阶段（具体月份）

### 可以对外公开的信息（经过匿名化处理后）：
- 公司的发展阶段
- 团队规模范围（1–10 人、10–50 人、50–200 人以上）
- 行业类型
- 公司面临的挑战类型
- 公司在市场中的定位

### 在进行任何外部 API 调用或网络搜索之前：
请遵循 `references/anonymization-protocol.md` 中的规则：
- 数字数据 → 使用范围或相对发展阶段的描述方式
- 名称 → 使用角色来替代
- 收入数据 → 使用百分比或发展阶段的描述方式
- 客户名称 → 使用 “客户 A、B、C” 等通用名称

## 背景信息缺失或不完整的情况

**处理方式：** 请优雅地应对，切勿因此中断对话：
- **发展阶段缺失：** “请问您目前还在筹备期、扩展期还是优化期？”
- **财务数据缺失：** 根据公司的发展阶段和团队规模进行推测，并注明数据缺失的情况。
- **创始人信息缺失：** 根据对话内容进行推断，并在文件中注明信息是推测得出的。
- **公司有多位创始人：** 文件中的信息主要反映受访者的观点，不同创始人的看法可能存在差异。

## 必需的背景信息字段

```
Required:
  - Last updated (date)
  - Company Identity → What we do
  - Stage & Scale → Stage
  - Founder Profile → Founder archetype
  - Current Challenges → Priority #1
  - Goals & Ambition → 12-month target

High-value optional:
  - Unfair advantage
  - Kill-shot risk
  - Avoided decision
  - Watch list
```

如果发现缺少必要的背景信息字段，请在会议中记录下来，并在必要时向用户询问。
