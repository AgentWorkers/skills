# /em:hard-call — 用于处理没有理想选择的决策的框架

**命令:** `/em:hard-call <决策>`

这个框架适用于那些让你在凌晨3点仍无法入睡的艰难决策：解雇联合创始人、裁减20%的员工、终止客户喜爱的产品线，或者彻底改变公司的发展方向。

这类决策没有绝对正确的答案，只有相对更不糟糕的选项。这个框架能帮助你找到那个“相对不糟糕”的选择。

---

## 为什么这些决策如此艰难？

并非因为数据不清晰（通常数据都很明确），而是因为：

1. **真正的人会受到影响**——有人会失去工作，关系会破裂，团队会受到伤害。
2. **你一直在逃避这个决策**——这意味着问题已经比你想象中更严重了。
3. **决策具有不可逆性**——与大多数商业决策不同，这类决策一旦做出就无法轻易撤销。
4. **你自身也牵涉其中**——你对正确决策的判断会受到个人情感的干扰。

你拖延决策的时间越长，情况通常会变得越糟。六个月前只需要削减10%的员工，现在可能就需要削减25%；本应在第四个月进行的联合创始人谈话，却拖到了第十四个月。

**大多数艰难的决策都是拖延太久的后果。**

---

## 框架内容

### 第一步：可逆性测试

首先问自己最重要的问题：**这个决策可以撤销吗？**
- **可撤销的**——尝试执行后根据反馈进行调整（比如解雇供应商、停止某个功能、调整战略）。
- **部分可撤销的**——撤销虽然痛苦，但仍然可行（比如重组团队、调整联合创始人的职责）。
- **不可撤销的**——一旦做出就无法挽回（比如解雇员工、终止有客户依赖的产品线、关闭公司）。

对于不可撤销的决策，你需要更加谨慎地评估风险。因为一旦做出，就无法回头。

**如果你把一个可撤销的决策当作不可撤销的来处理，那你其实是在逃避它。**

### 第二步：10/10/10评估框架

对每个选项分别问三个问题：
- **10分钟后**：做出这个决策后你的感受会如何？
- **10个月后**：这个决策会产生什么影响？问题会得到解决吗？
- **10年后**：回顾这个决策时，你会认为它是正确的吗？

10分钟后的即时感受往往是最不可靠的判断依据；而10年后的长远视角才能真正揭示正确的选择。

**大多数艰难的决策，在10年后看来其实都很明显。关键在于你是否能够忍受当时的痛苦。**

### 第三步：安迪·格罗夫的测试

安迪·格罗夫曾提出一个用于评估战略决策的标准：“如果明天我们被替换，新的CEO会怎么做？”

用一个没有情感偏见、没有前期投入的新视角来看待问题。从外部角度来看，哪个选择才是正确的？

如果外部人士也能给出明确的答案，那么问题就变成了：你为什么还没有这么做？

### 第四步：利益相关者影响分析

分析每个选项对不同利益相关者的影响：
| 利益相关者 | 选项A的影响 | 选项B的影响 | 他们的反应 |
|-------------|----------------|----------------|----------------|
| 受影响的员工 | | | |
| 剩余团队 | | | |
| 客户 | | | |
| 投资者 | | | |
| 你 | | | |

这个分析的目的不是寻找一个对所有人都没有伤害的选项（实际上这样的选项并不存在），而是为了在做出决策前全面了解各种影响。

### 第五步：预先公告演练

在做出决策之前，先写下相应的公告内容（发给团队的邮件、对客户的声明，以及你可能需要进行的沟通）。

**如果你无法写出这样的公告，那就说明你还没有准备好做出这个决策。**  
撰写公告的过程会迫使你面对现实，同时也能检验你的理由是否站得住脚。比如：“我们做出这个改变是因为……”这句话听起来合理吗？

### 第六步：沟通计划

糟糕的沟通会让艰难的决策变得更加复杂。决策本身固然重要，但沟通方式同样至关重要。

对于每一个艰难的决策，你需要规划好：
- **谁应该最先知道这个决定**（直接受影响的人，优先于其他人）。
- **如何传达这个决定**（尽可能当面沟通，避免通过邮件，因为邮件可能会引发负面情绪）。
- **该说些什么**（诚实、直接、富有同情心——详见`references/hard_things.md`）。
- **他们可能会提出什么问题**（提前做好准备）。
- **接下来会发生什么**（向他们清楚地说明后续安排）。

## 针对特定决策的详细框架

### 解雇联合创始人
详见`references/hard_things.md — Co-Founder Conflicts`。

需要先回答的关键问题：
- 这是一个绩效问题，还是价值观/文化问题？（处理方式不同）。
- 你是否已经明确地指出了问题（而不是含糊其辞）？
- 公司的股权结构是怎样的？法律上有哪些影响？
- 是否有更适合他们的职位安排，或者是否需要彻底终止合作关系？
- 谁需要知道这个决定（董事会、团队、投资者），以及通知的顺序是什么？

**原则：**如果你已经考虑这个问题超过3个月，那么答案其实已经很明显了。剩下的只是何时行动的问题。

### 裁员
关键问题：
- 这是一次性的调整，还是公司长期衰退的开始？（一次性的调整还有恢复的机会；连续裁员会破坏团队氛围。）
- 裁员的幅度是否足够大？（裁员不足反而会加剧问题；两次裁员会严重损害团队信任。）
- 谁来负责宣布这个决定？宣布的方式是否直接且诚实？
- 裁员的补偿方案是否合理？
- 如何防止最优秀的人才离开？

**原则：**裁员时要果断、彻底，并且要尊重相关人员。**

### 转变公司发展方向
关键问题：
- 这是一次真正的战略转变（方向完全改变），还是仅仅是一种策略调整（方向不变，但方法不同）？
- 你会保留哪些业务，放弃哪些业务？
- 你是否有证据表明新的方向可行？还是你只是因为当前策略失败而被迫改变？
- 如何向已经相信旧方向的客户解释这一变化？
- 这个决定会对董事会的信心产生什么影响？

**原则：**转变方向应该基于新的机会，而不是因为当前策略的失败。

### 终止产品线
关键问题：
- 目前使用该产品的客户会怎样？
- 产品的迁移方案是什么？
- 开发该产品的人员会面临什么？
- “终止产品”是最佳选择，还是“出售产品”或“分拆产品”更合适？
- 内外部对此事的舆论会如何？

## 识别逃避决策的行为

如果你出现以下情况，说明你一直在逃避这个艰难的决策：
- 你每周都在思考这个问题，已经超过一个月。
- 你希望问题能自行解决。
- 你一直在等待更多数据，但永远觉得这些数据还不够。
- 你已经在脑海中多次讨论过这个决定，但从未真正付诸行动。
- 周围的人也注意到了这个问题。

**拖延的代价往往比做出决策的代价更高。**  
你每拖延一个月，问题就会恶化：不称职的联合创始人会更加难以替代，需要终止的产品线会消耗更多资源，需要被解雇的人也会对周围的人产生负面影响。

果断做出决策，并且要尊重所有相关人员。
