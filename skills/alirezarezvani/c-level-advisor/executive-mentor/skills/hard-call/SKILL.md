# /em:hard-call — 用于处理没有理想选择的决策框架

**命令:** `/em:hard-call <决策>`

适用于那些让你在凌晨3点仍无法入睡的艰难决策：解雇联合创始人、裁减20%的员工、终止客户喜爱的产品线，或是进行业务转型、关闭公司。

这类决策没有绝对正确的答案，只有相对“错误”或“正确”的选择。这个框架能帮助你找到那个相对更好的答案。

---

## 为什么这些决策如此艰难？

并非因为数据不明确（通常数据都很清晰），而是因为：

1. **真正的人会受到影响**——有人会失去工作，人际关系会破裂，团队会受到打击；
2. **你一直在回避这个决策**——这意味着问题已经比想象中更严重了；
3. **决策具有不可逆性**——与大多数商业决策不同，这些决策一旦做出就无法轻易撤销；
4. **你自身也牵涉其中**——你对正确决策的判断会受到个人情感的干扰。

你回避这个决策的时间越长，情况往往就会越糟。六个月前只需要削减10%的成本，现在可能就需要削减25%；本应在第四个月进行的联合创始人谈话，却拖到了第十四个月才进行。

**大多数艰难的决策都是拖延太久的后果。**

---

## 框架内容

### 第一步：可逆性测试

首先问自己最重要的问题：**这个决策可以撤销吗？**

- **可撤销的**——尝试执行后根据反馈进行调整（比如解雇供应商、停止某个功能、改变策略）；
- **部分可撤销的**——撤销虽然痛苦但仍然可行（比如重组团队、调整联合创始人的职责）；
- **不可撤销的**——一旦做出就无法挽回（比如解雇员工、终止有客户依赖的产品线、关闭公司）。

对于不可撤销的决策，你需要更加谨慎地评估风险。在行动之前，必须进行更彻底的尽职调查。这不是因为你可能会犯错，而是因为一旦做出决定就无法回头。

**如果你把一个可撤销的决策当作不可撤销的来处理，那你其实是在逃避它。**

### 第二步：10/10/10框架

对每个选项分别回答以下三个问题：

- **10分钟后**：做出这个决策后，你的感受会如何？
- **10个月后**：这个决策会产生什么影响？问题会得到解决吗？
- **10年后**：回首往事，你会认为这是正确的选择吗？

10分钟后的即时感受往往是最不可靠的判断依据；而10年后的长远视角通常能帮助你看清正确的方向。

**大多数艰难的决策，在10年后看来其实都很明显。关键在于你是否能够忍受当时的痛苦。**

### 第三步：安迪·格罗夫的测试

安迪·格罗夫提出的战略决策判断标准：“如果明天我们被取代，新的CEO会怎么做？”

用一个没有情感偏见、没有前期投入的新视角来看待问题。从外部来看，哪个选择才是最正确的？

如果外部人士的答案很明确，那么问题就是：你为什么还没有采取行动？

### 第四步：利益相关者影响分析

分析每个选项会对哪些利益相关者产生什么影响：

| 利益相关者 | 选项A的影响 | 选项B的影响 | 他们的反应 |
|-------------|----------------|----------------|----------------|
| 受影响的员工 | | | |
| 剩余团队 | | | |
| 客户 | | | |
| 投资者 | | | |
| 你 | | | |

这个分析的目的不是找到一个对所有人都没有伤害的选项（实际上这样的选项并不存在），而是要在做出决策前全面了解各种影响。

### 第五步：预先宣布测试

在做出决策之前，先写下相应的公告内容（发给团队的邮件、对客户的声明，以及你需要进行的沟通）。

**如果你写不出这样的公告，那就说明你还没有准备好做出这个决定。**  
撰写公告的过程会迫使你直面现实，同时也能检验你的理由是否站得住脚。比如：“我们做出这个改变是因为……”这句话听起来合理吗？

### 第六步：沟通计划

糟糕的沟通会让艰难的决策变得更加复杂。决策本身固然重要，但沟通方式同样关键。

对于每一个艰难的决策，你需要规划好：
- **谁应该最先知道这个决定**（直接受影响的人，优先于其他人）；
- **如何传达这个决定**（尽可能当面沟通，避免通过邮件，因为邮件可能会引发负面情绪）；
- **该说什么**（诚实、直接、富有同情心）；
- **他们可能会提出什么问题**（提前准备好应对）；
- **之后会发生什么**（向他们清楚地说明后续安排）。

## 针对特定决策的详细框架

### 解雇联合创始人
请参考 `references/hard_things.md — Co-Founder Conflicts` 以获取完整的处理流程。

需要首先回答的关键问题：
- 这是绩效问题还是价值观/文化问题？（需要采取不同的沟通方式）；
- 你是否已经明确地指出了问题（而不是含糊其辞）？
- 公司的股权结构是怎样的？法律后果是什么？
- 是否有更适合他们的职位安排，还是应该彻底终止合作关系？
- 谁需要知道这个决定（董事会、团队、投资者），以及告知的顺序是什么？

**原则**：如果你已经考虑这个问题超过3个月，那么答案其实早已明确。剩下的只是何时行动的问题。

### 裁员
需要回答的关键问题：
- 这是一次性的调整，还是公司长期衰退的开始？（一次性的调整还有恢复的可能；连续裁员会破坏团队文化）；
- 裁员的幅度是否足够大？（裁员不足反而会加剧问题；连续两轮裁员会严重损害团队信任）；
- 谁来负责宣布这个决定？宣布的方式是否直接且诚实？
- 裁员的补偿方案是否合理？
- 如何防止最优秀的人才离职？

**原则**：裁员要果断、彻底，并且要尊重相关人员。

### 业务转型
需要回答的关键问题：
- 这是一次真正的业务转型（方向改变），还是仅仅是对现有策略的优化？
- 你会保留哪些部分，放弃哪些部分？
- 你有证据证明新的方向可行吗？还是只是因为现有策略失败而被迫改变？
- 如何向购买过旧产品版本的客户解释这一变化？
- 这个决策会对董事会的信心产生什么影响？

**原则**：业务转型应基于新的机会，而不是因为现有策略的失败。

### 终止产品线
需要回答的关键问题：
- 目前使用该产品的客户该怎么办？
- 产品的迁移方案是什么？
- 开发该产品的人员未来会怎样？
- “终止产品”是最佳选择吗？还是“出售产品”或“分拆产品”更合适？
- 内外部应该如何对外传达这个决定？

## 识别回避行为的测试

如果你出现以下情况，说明你一直在回避这个艰难的决策：
- 你已经思考这个问题超过一个月了；
- 你希望情况能自行解决；
- 你一直在等待更多数据，但永远觉得这些数据还不够；
- 你只是在脑海中反复讨论这个问题，从未真正付诸行动；
- 周围的人已经注意到了问题的存在。

**拖延的代价通常比做出决策的代价更高。**  
你拖延得越久，问题就会越严重：不称职的联合创始人会变得更加难以处理，需要终止的产品线会消耗更多资源，需要被解雇的人也会对周围的人产生负面影响。

果断做出决策，并且要做得体面。
