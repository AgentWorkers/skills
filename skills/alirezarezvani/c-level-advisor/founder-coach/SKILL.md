# 创始人发展教练

公司的成长速度取决于创始人的能力。本技能将创始人发展视为一种战略重点，而不仅仅是个人享受。

## 关键词
创始人、首席执行官（CEO）、创始人模式、授权、职业倦怠、冒名顶替综合症、领导力提升、能量管理、日程审计、执行团队管理、继任计划、从初始团队成员（IC）到经理的转变、领导风格、创始人陷阱、盲点、个人目标与关键结果（OKRs）、CEO反思

## 核心理念

创始人始终是公司发展的瓶颈。这并非有意为之，而是由组织结构决定的。你是公司的创建者，你掌握着所有信息，所有决策都通过你做出。但这种模式在团队规模扩大时会带来问题。

当团队成员达到约15人时，第一个瓶颈就会出现：你无法同时参加所有会议并保持高效思考；当团队规模达到约50人时，你的领导风格可能会引发团队文化问题；当团队规模达到约150人时，你需要一个真正的执行团队，否则公司将无法继续扩张。

越早解决这些问题，效果越好。

## 1. 创始人类型识别

大多数创始人属于某种特定的类型。了解自己的类型有助于预测可能遇到的挑战：

| 类型 | 强项 | 盲点 | 需要的帮助 |
|---------|--------|---------|-------------------|
| **构建者**（Builder）| 产品开发、技术能力 | 市场推广、故事讲述能力 | 需要一位销售或市场推广合作伙伴 |
| **销售者**（Seller）| 收入生成、人际关系处理、愿景传达 | 运营管理、执行能力 | 需要一位运营经理或首席运营官（COO） |
| **运营者**（Operator）| 执行能力、流程管理、可靠性 | 愿景规划、产品直觉 | 需要一位有远见的联合创始人或战略合作伙伴 |
| **远见者**（Visionary）| 战略规划、故事讲述能力、模式识别 | 执行能力、细节处理 | 需要一位整合者或首席运营官 |

**自我评估问题：**
- 你有空闲时间时通常会做什么？
- 你最常拖延的事情是什么？
- 合伙人或早期团队成员经常抱怨你没有做的事情是什么？
- 关于你的领导能力，你收到的最佳反馈是什么？

大多数创始人属于“构建者”或“远见者”类型。大多数公司扩张问题源于他们没有及时招聘到互补类型的人才。

## 2. 授权框架

创始人未能有效授权的原因有四个：
1. “没有人能做得比我好”（短期内可能成立，但长期来看是致命的）
2. “解释事情比亲自去做花费的时间更长”（第一次可能如此，但第十次就不会了）
3. “如果我不亲自做，就会失去控制”（但在大规模运营中，控制其实是一种错觉）
4. “如果事情失败了，那就是我的错”（如果你从不允许他人尝试，失败就是你的责任）

### 技能与意愿矩阵

| 技能水平 | 意愿水平 | 授权方式 |
|---------|---------|-------------------|
| 高技能 | 高意愿 | 完全授权并指导发展 |
| 高技能 | 低意愿 | 激励或重新分配任务 |
| 低技能 | 高意愿 | 直接管理或重新设计角色 |

**规则：**
- 高技能 + 高意愿 → 完全授权并退出管理岗位 |
- 高技能 + 低意愿 | 投资于团队成员，帮助他们成长 |
- 低技能 + 低意愿 | 不要授权给他们，而是解决他们的绩效问题 |

### 授权阶梯

并非所有的授权都是一样的。应逐步进行：
1. “严格按照我的指示去做”——这不算真正的授权，只是指导 |
2. “研究这个问题然后汇报结果”——这属于信息收集 |
3. “提出解决方案，我来决定”——这属于初步授权 |
4. “自己做出决定后告诉我结果”——这属于决策授权 |
5. “完全负责处理，超出范围时再向我汇报”——这属于完全授权 |

从第二或第三级开始授权。随着信任的建立，逐渐提升团队的授权级别。大多数创始人无法让团队达到第三级，这往往是公司发展的瓶颈。

### 首先应该授权的任务

**首先授权（工作量大、风险低的任务）：**
- 每次都按相同方式重复执行的运营任务 |
- 信息收集与整理 |
- 会议协调与安排 |
- 定期生成的报告与更新

**接下来授权（可以通过培训提升技能的任务）：**
- 与客户的互动（需明确原则） |
- 招聘筛选（在培训了判断力之后） |
- 合作伙伴关系管理 |
- 在规定范围内管理预算 |

**最后授权（具有战略意义、不可逆的任务）：**
- 重大战略调整 |
- 高管招聘 |
- 大额财务决策 |

## 3. 能量管理

创始人需要管理的不仅是时间，还有能量。时间是固定的，但能量是可以再生的——前提是你能够有效管理它。

### 能量审计

按照能量的消耗情况来安排一周的工作。详细模板请参考 `references/founder-toolkit.md`。

**能量分类：**
- 🟢 **提神的活动**：完成这些活动后你会感到精力充沛 |
- 🟡 **中性活动**：既不提神也不消耗能量 |
- 🔴 **消耗能量的活动**：完成这些活动后你会感到疲惫 |

**常见创始人能量类型：**
- **构建者**：通过创造活动获得能量，因政治斗争和流程管理而耗能 |
- **销售者**：通过与人互动和取得成果获得能量，因细节工作和行政事务而耗能 |
- **运营者**：通过解决问题获得能量，因不确定性而耗能 |
- **远见者**：通过战略规划和创新获得能量，因执行细节而耗能 |

**原则：** 尽量减少消耗能量的活动，增加提神的活动。**

### 能量管理实践：
- **保护深度思考时间**：每周安排2-4小时的不间断思考时间。 |
- **批量处理琐事**：每天最多处理两次电子邮件和Slack消息等行政任务。 |
- **在精力低谷时专注单一任务**：如果感到疲惫，不要尝试完成最重要的工作，而是处理那些不需要全力以赴的任务。 |
- **识别你的最佳工作时段**：大多数人每天有4-6个高效工作时段，将最重要的工作安排在这些时段内。

## 4. CEO日程审计

日程安排是创始人最真实的“记录”，它反映了你的实际优先事项，而不仅仅是你口头上说的优先事项。

### 审计方法：

查看过去4周的日程数据，对每一项会议或任务进行分类：

| 类别 | 描述 | 占比 |
|---------|---------|---------|
| 战略规划 | 战略思考、计划制定、方向设定 | 20–25% |
| 人际关系 | 一对一沟通、辅导、招聘 | 20–25% |
| 外部事务 | 客户、投资者、合作伙伴 | 20% |
| 执行工作 | 直接工作、决策制定 | 15% |
| 行政事务 | 电子邮件处理、日程安排 | < 15% |
| 恢复时间 | 锻炼、用餐、思考 | 10–15% |

**审计中的警示信号：**
- 行政事务占比超过20%：你更像是一个协调者，而不是CEO。需要改进你的工作流程。 |
- 执行工作占比超过30%：你仍然像初始团队成员一样工作。需要建立更强大的执行团队。 |
- 人际关系相关事务占比低于10%：说明你的团队缺乏支持，需要更多你的关注。 |
- 没有安排恢复时间：你可能一直在超负荷工作，这会导致不良后果。 |
- 战略规划占比低于10%：你只是在管理公司，而不是真正领导它。

### CEO在不同阶段的重点任务

| 发展阶段 | CEO应重点关注的任务： |
|---------|-------------------|
| 初创期 | 产品开发与客户关系。 |
| A轮融资阶段 | 招聘执行团队。 |
| B轮融资阶段 | 文化建设、战略规划、外部关系（投资者/合作伙伴/客户）。 |
| C轮融资及以上阶段 | 愿景规划、董事会管理、外部沟通、团队领导力培养 |

如果你还在忙于前两个阶段的任务，说明你还没有完成角色的转变。

## 5. 领导风格演变

随着公司的发展，领导者的角色也在不断变化。大多数创始人未能及时调整自己的领导风格。

**从初始团队成员到经理（0-10人）：**
你需要教导团队并建立信任。团队会观察你如何处理失败。关键技能是：提供清晰的指导、设定明确的期望、频繁沟通。

**从经理到领导者（10-50人）：**
你无法直接管理所有人，需要培养能够管理他人的团队成员。关键技能是：招聘值得信任的经理，并让他们独立工作。

**从领导者到执行者（50-200人）：**
你的主要任务是塑造企业文化和发展方向，而不再是直接管理具体工作。关键技能是：持续沟通、在合适的层面做出决策、培养领导团队。

**从执行者到机构级CEO（200人以上）：**
此时你更像是一个象征性角色，需要建立无需你亲自参与的运营体系；重点关注董事会、投资者和外部关系。

**最艰难的转变：** 从经理转变为领导者。你需要停止亲自处理所有事务，开始信任那些你还在了解中的团队成员。

## 6. 盲点识别

每个人都有盲点，创始人尤其如此——因为在公司早期，没有人有权力或勇气告诉你这些盲点。

### 常见的创始人盲点：
- **沟通问题**：“我说过一次，他们应该记住了”——你说了，但他们可能没听进去或不相信。 |
- **决策速度**：发展速度过快，导致团队无法跟上你的步伐。 |
- **信息垄断**：自己掌握所有信息却不分享，然后对团队的错误决策感到沮丧。 |
- **乐观偏见**：总是低估时间表、成本和难度。 |
- **创始人特权思维**：认为某些规则只适用于他人，不适用于自己。 |
- **回避反馈**：营造一个没有人敢给你诚实反馈的环境。

### 如何发现自己的盲点：
- **360度反馈**：每年一次，向直接下属、同事和董事会成员征求意见。 |
- **离职员工访谈**：分析离职员工的反馈，找出规律。 |
- **失败总结**：分析最糟糕的决策，找出共性。 |
- **能量审计**：分析哪些活动会让你感到疲惫。

## 7. 冒名顶替综合症应对策略

这种心理现象不会消失，只会随着公司的发展而演变。曾经害怕向投资者展示自己的创始人，现在可能害怕管理董事会；曾经害怕招聘的创始人，现在可能害怕解雇员工。

**正确的认知：** 冒名顶替综合症的程度与你的成长速度成正比。如果你从未感到这种压力，说明你还没有真正成长。

**实用工具：**
- **成就记录**：记录自己的成功经历、收到的表扬以及有效的决策。在感到怀疑时阅读这些记录。 |
- **正视这种感觉**：“我觉得自己准备不足”并不等于“我就是个冒名顶替者”。感受和事实是不同的。 |
- **勇敢行动**：行动才能带来能力提升，而不是等待准备就绪。 |
- **坦诚面对**：向信任的人表达“我对这次投资者会议感到焦虑”，可以减轻这种心理负担。

## 8. 创始人的心理健康

职业倦怠并非软弱的表现，而是高压力、低恢复能力和缺乏控制权导致的必然结果。

**职业倦怠的迹象：**
- 早期症状：易怒、睡眠困难、决策变得困难、对使命失去热情。 |
- 中期症状：出现身体不适（头痛、生病）、对公司产生怀疑、社交退缩、所有任务都显得同样重要（优先级混乱）。 |
- 晚期症状：无法正常工作、决策能力下降、团队成员会先于你察觉问题。

**如果出现职业倦怠：** 应立即停止工作并寻求支持。公司需要一个能够正常工作的创始人，而不是一个精疲力尽的领导者。

**预防措施：**
- **保护恢复时间**：在工作日安排固定的恢复时间。 |
- **寻求心理治疗或辅导**：这对创始人来说至关重要。这份工作具有高度压力。 |
- **加入同行交流群**：与其他处于相似发展阶段的创始人交流，他们能真正理解你的处境。 |
- **设定明确的界限**：知道“今天的工作量”应该是什么样的，避免无限工作。

## 9. 创始人模式陷阱

保罗·格雷厄姆在《创始人模式》一文中提出，优秀的创始人应该深入参与公司运营，跳过中层管理环节。这一观点有时确实适用。

**创始人模式适用的情况：**
- 公司处于危机中，需要直接领导。 |
- 在寻找产品与市场的匹配点时（速度至关重要）。 |
- 需要做出高价值、不可逆的决策时。 |
- 公司规模较小时。

**创始人模式不适用的情况：**
- 当这种模式妨碍了已招聘的经理的工作时。 |
- 当这种模式源于不信任而非战略考虑时。 |
- 当它阻碍了团队的自主判断能力时。 |
- 当你只是因为习惯而继续做这些事情，而不是因为公司真的需要你这样做时。

**判断标准：** 你是出于工作需要而深入参与，还是因为害怕失去控制？前者是领导力的体现，后者则是陷入了陷阱。

## 10. 继任计划

建立一个无需你也能正常运作的公司，并不是不忠诚的表现，而是领导力的体现。

**继任计划不仅仅是关于离职后的安排，更是关于公司的韧性。** 需要考虑如果你生病、休假或公司被收购时的情况。

**继任准备程度：**
- 第一级：你已经记录下了自己的关键知识和流程。 |
- 第二级：至少有一个人能够独立承担你的部分职责两周。 |
- 第三级：你的领导团队能够在没有你的情况下运作一个季度。 |
- 第四级：你已经找到了潜在的继任者并进行了培养。

大多数创始人目前还处于第一级。达到第二级是一个合理的目标，第三级则是重要的战略资产。

## 创始人发展的关键问题：**
- “上周你做了哪些本可以由他人处理的决策？” |
- “还有哪些任务是你六个月前就应该授权给别人的？” |
- “你上次收到诚实、建设性的反馈是什么时候？来自谁？反馈内容是什么？” |
- “公司在没有你的情况下能否正常运作？” |
- “哪些事情正在消耗你的能量，而你却认为这是不可避免的？”
