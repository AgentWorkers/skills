# M&A 战略手册

本手册涵盖了 M&A 双方的框架，包括收购方和被收购方的行动指南。

## 关键术语
M&A（并购）、合并与收购、尽职调查、收购、人才收购（acqui-hire）、整合、交易结构、估值、意向书（LOI）、条款清单（term sheet）、绩效奖励（earnout）

## 快速入门

**作为收购方：**
- 首先明确战略目标 → 筛选目标公司 → 进行尽职调查 → 评估公司价值 → 谈判交易条款 → 实施整合。

**作为被收购方：**
- 先评估自身准备情况 → 准备数据室（data room） → 选择并购顾问 → 谈判收购条款 → 进行过渡。

## 作为收购方时需要考虑的要点

### 战略考量（在开始任何行动之前必须明确）
- **自建 vs 收购：** 自建是否能够更快/更经济地实现目标？如果是，那么就不必进行收购。
- **收购的内容：** 你真正想要的是什么？是人才、技术还是客户？
- **整合难度：** 将这家公司整合到你的公司中有多困难？

### 尽职调查 checklist
| 领域 | 关键问题 | 需警惕的信号 |
|--------|--------------|-----------|
| 财务 | 收入质量、客户集中度、资金消耗速度 | 超过 30% 的收入来自单一客户 |
| 技术 | 代码质量、技术债务、架构兼容性 | 代码质量差且缺乏测试 |
| 法律 | 知识产权归属、诉讼情况、合同条款 | 关键知识产权由个人持有 |
| 人员 | 关键人员流失风险、企业文化匹配度 | 创始人没有锁定条款或绩效奖励机制 |
| 市场 | 市场地位、竞争威胁 | 市场份额下降 |
| 客户 | 客户流失率、净推荐值（NPS）、合同条款 | 客户流失率高或合同期限较短 |

### 估值方法
- **收入倍数法：** 取决于行业（SaaS 企业通常为年收入（ARR）的 2-15 倍 |
- **可比交易法：** 参考类似公司的收购价格 |
- **折现现金流（DCF）法：** 仅适用于盈利企业（大多数初创企业适用倍数法） |
- **人才收购（acqui-hire）：** 在热门市场中，每位工程师的收购价格约为 100 万至 300 万美元 |

## 作为被收购方时需要做的准备

### 表明被收购意向的信号
- 战略买家表现出收购兴趣 |
- 周围市场正在发生整合 |
- 筹资变得比日常运营更困难 |
- 创始人愿意接受收购过渡

### 提前 6-12 个月的准备工作
1. 清理财务状况（如有可能，请进行审计）
2. 归档所有知识产权和合同文件
3. 降低客户集中度
4. 确保关键员工不会离职
5. 准备数据室（data room）
6. 聘请并购顾问

### 谈判要点
| 交易条款 | 需重点关注的内容 | 你的谈判优势 |
|------|--------------|---------------|
| 估值 | 注意估值陷阱（如无法实现的目标） | 多个竞争性报价 |
| 绩效奖励（earnout） | 目标的定义、评估周期 | 现金支付与绩效奖励的比例 |
| 员工锁定条款 | 锁定期、条件 | 你的替代性（即收购方是否容易找到替代人员） |
| 售后服务与保修条款 | 责任范围 | 保证金（escrow）与赔偿上限 |
| 员工保留 | 谁可以获得优惠条件、条件是什么 | 关键人员的去向 |

## 需警惕的信号（双方都应留意）
- 没有明确的战略理由，只是简单认为“这是一笔好交易”
- 在尽职调查过程中发现明显的企业文化冲突但被忽视
- 关键人员在交易完成前未得到妥善安排
- 没有制定整合计划，或者只是简单地说“我们会想办法解决”

## 与高层管理团队的协作
| 高层角色 | 在 M&A 过程中的职责 |
|------|-------------------|
| CEO | 制定战略方向、主导谈判 |
| CFO | 负责估值、交易结构及融资安排 |
| CTO | 负责技术方面的尽职调查和整合架构设计 |
| CHRO | 负责人员评估和保留计划 |
| COO | 负责整合执行和流程整合 |
| CPO | 评估产品路线图对并购的影响以及客户资源的重叠情况 |
