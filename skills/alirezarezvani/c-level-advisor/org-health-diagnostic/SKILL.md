# 组织健康诊断

该诊断工具从八个维度全面评估组织的健康状况，通过“交通灯”式可视化方式（绿色、黄色、红色）直观展示各项指标的现状。这些指标能帮助你发现那些你尚未意识到的问题。

## 关键词
- 组织健康（Organizational Health）
- 健康诊断（Health Diagnostic）
- 健康仪表盘（Health Dashboard）
- 健康检查（Health Check）
- 公司健康（Company Health）
- 团队健康（Team Health）
- 初创企业健康（Startup Health）
- 健康评分卡（Health Scorecard）
- 健康评估（Health Assessment）
- 风险仪表盘（Risk Dashboard）

## 快速入门

```bash
python scripts/health_scorer.py        # Guided CLI — enter metrics, get scored dashboard
python scripts/health_scorer.py --json # Output raw JSON for integration
```

### 描述你的关键指标：
```
/health [paste your key metrics or answer prompts]
/health:dimension [financial|revenue|product|engineering|people|ops|security|market]
```

## 八个维度

### 1. 💰 财务健康（CFO）
**评估内容：** 公司是否有足够的资金来维持运营并投资于业务增长？

**关键指标：**
- **运营周期**：以当前资金消耗速度计算，可维持运营的月数（绿色：>12个月；黄色：6-12个月；红色：<6个月）
- **资金消耗倍数**：净资金消耗与新增收入的比例（绿色：<1.5倍；黄色：1.5-2.5倍；红色：>2.5倍）
- **毛利率**：SaaS产品的目标毛利率（绿色：>70%；黄色：55-70%；红色：<55%）
- **月度增长率**：根据公司发展阶段进行评估（参考基准数据）
- **收入集中度**：最大客户对总收入的贡献比例（绿色：<15%；黄色：15-25%；红色：>25%）

### 2. 📈 收入健康（CRO）
**评估内容：** 客户是否持续使用我们的产品，并愿意推荐给我们？

**关键指标：**
- **净客户留存率（NRR）**：绿色：>110%；黄色：100-110%；红色：<100%
- **客户流失率（年化）**：绿色：<5%；黄色：5-10%；红色：>10%
- **产品管线覆盖率**：下一季度的潜在客户数量（绿色：>3倍；黄色：2-3倍；红色：<2倍）
- **客户获取成本（CAC）回收期**：绿色：<12个月；黄色：12-18个月；红色：>18个月
- **平均客户价值（ACV）趋势**：呈增长、稳定或下降趋势

### 3. 🚀 产品健康（CPO）
**评估内容：** 客户是否喜欢并真正使用我们的产品？

**关键指标：**
- **净推荐值（NPS）**：绿色：>40；黄色：20-40；红色：<20%
- **日活跃用户（DAU）与月活跃用户（MAU）比例**：反映用户参与度（绿色：>40%；黄色：20-40%；红色：<20%）
- **核心功能采用率**：使用核心功能的用户比例（绿色：>60%）
- **从注册到首次使用核心功能的时间**：越短越好
- **客户满意度（CSAT）**：绿色：>4.2/5；黄色：3.5-4.2；红色：<3.5

### 4. ⚙️ 工程健康（CTO）
**评估内容：** 我们能否可靠地发布新功能并保持开发速度？

**关键指标：**
- **发布频率**：绿色：每日；黄色：每周；红色：每月或更少
- **变更失败率**：导致问题的发布占比（绿色：<5%；红色：>15%
- **平均恢复时间（MTTR）**：绿色：<1小时；黄色：1-4小时；红色：>4小时
- **技术债务比率**：开发任务中受技术问题影响的比例（绿色：<20%；黄色：20-35%；红色：>35%
- **问题发生率**：每月的P0/P1级别问题数量（绿色：<2个；黄色：2-5个；红色：>5个）

### 5. 👥 人员健康（CHRO）
**评估内容：** 团队是否稳定、高效，并且能够持续成长？

**关键指标：**
- **年度人员流失率**：绿色：<10%；黄色：10-20%；红色：>20%
- **员工参与度**：（使用类似指标如eNPS）绿色：>30；黄色：0-30；红色：<0%
- **填补职位空缺所需时间**：平均天数（绿色：<45天；黄色：45-90天；红色：>90天）
- **经理与员工比例**：绿色：1:5–1:8；黄色：1:3–1:5或1:8–1:12；红色：低于此比例
- **内部晋升率**：高级职位至少有25-30%由内部员工填补

### 6. 🔄 运营健康（COO）
**评估内容：** 我们是否能够有条不紊地执行战略？

**关键指标：** **关键目标完成率（OKR）**：绿色：>70%；黄色：50-70%；红色：<50%
- **决策周期**：从制定决策到执行决策所需的时间（绿色：<48小时；黄色：48小时-1周）
- **会议效率**：有明确结果的会议占比（绿色：高）
- **流程成熟度**：按1-5级进行评估（参考COO顾问的建议）
- **跨部门项目的完成率**：按时按范围完成的占比

### 7. 🔒 安全健康（CISO）
**评估内容：** 我们是否有效保护客户数据并遵守法规？

**关键指标：**
- **过去90天的安全事件**：绿色：0；黄色：1-2个轻微事件；红色：1个以上严重事件
- **合规状态**：当前持有的认证或正在办理中的认证与逾期认证的比例
- **漏洞修复及时性**：关键漏洞在规定的时间内修复的比例（绿色：100%）
- **安全培训完成率**：团队成员的安全培训完成比例（绿色：>95%）
- **安全测试频率**：最近一次安全测试的时间（绿色：<12个月；黄色：12-24个月；红色：>24个月）

### 8. 📣 市场健康（CMO）
**评估内容：** 我们是否在市场中取得成功并实现高效增长？

**关键指标：**
- **客户获取成本（CAC）趋势**：季度环比是改善、稳定还是恶化
- **有机客户与付费客户的占比**：有机客户占比越高，说明业务越健康（抗风险能力越强）
- **转化率**：合格商机中成功转化的比例（绿色：>25%；黄色：15-25%；红色：<15%
- **针对主要竞争对手的竞争转化率**
- **品牌净推荐值（Brand NPS）**：在客户调查中的品牌认知度和偏好得分

## 评分与可视化展示

每个维度的评分范围为1-10分，并通过“交通灯”颜色进行可视化：
- **绿色（7-10分）**：健康——需要维持并优化
- **黄色（4-6分）**：需要关注——趋势如何？是在改善还是恶化？
- **红色（1-3分）**：需要立即行动——30天内解决

**整体健康评分：**
根据公司发展阶段对各项指标进行加权平均（具体权重请参考`references/health-benchmarks.md`）。

## 维度之间的关联性（一个问题如何引发另一个问题）

| 如果某个维度显示为红色... | 需要关注的相关维度： |
|-----------------------------|----------------------------|
| 财务健康 | 人员健康（暂停招聘）→ 工程健康（暂停基础设施升级）→ 产品健康（缩减项目范围） |
| 收入健康 | 财务健康（资金缺口）→ 人员健康（人员流失风险）→ 市场健康（市场份额下降） |
| 人员健康 | 工程健康（开发速度下降）→ 产品健康（产品质量下降）→ 收入健康（客户流失率上升） |
| 工程健康 | 产品健康（功能更新延迟）→ 收入健康（交易成交受阻） |
| 产品健康 | 收入健康（净客户留存率下降，客户流失率上升）→ 市场健康（客户获取成本上升；推荐减少） |
| 运营健康 | 如果所有维度持续恶化，整体运营将陷入混乱（问题会像多米诺骨牌一样蔓延） |

## 数据处理策略

进行诊断时，并不需要所有指标的数据。该工具可以处理部分缺失的数据：
- 缺失的指标将从评分中排除，并标记为“[数据缺失]”
- 即使部分数据缺失，现有指标的评分仍然有效
- 报告会指出下一周期需要补充的数据项
