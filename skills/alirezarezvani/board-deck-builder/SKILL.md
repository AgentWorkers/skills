# 平板展示套件构建器（Board Deck Builder）

构建能够讲述故事的报告，而不仅仅是展示数据。每个部分都有负责人、叙述内容以及“那么接下来该怎么做”的建议。

## 关键词
- 平板展示套件（Board Deck）
- 投资者更新（Investor Update）
- 董事会会议（Board Meeting）
- 投资者关系（Investor Relations）
- 季度回顾（Quarterly Review）
- 董事会演示（Board Presentation）
- 筹资套件（Fundraising Deck）
- 投资者演示文稿（Investor Deck）
- 董事会叙述（Board Narrative）
- 季度业务回顾（Quarterly Business Review）

## 快速入门

```
/board-deck [quarterly|monthly|fundraising] [stage: seed|seriesA|seriesB]
```

提供可用的指标数据。该工具会使用占位符来填补缺失的信息——绝不会随意编造数字。

## 套件结构（标准顺序）

每个部分的结构如下：**标题 → 数据 → 叙述 → 下一步行动**

### 1. 高管总结（CEO）
**不超过3句话**
- 第1句：业务现状（我们目前所处的位置）
- 第2句：本季度发生的最重要事件
- 第3句：下个季度的发展方向

*不好的示例：“本季度业务表现良好，各个领域都取得了进展。”*
*好的示例：“我们第三季度的收入达到了240万美元（同比增长22%），签署了最大的企业合同，并且拥有14个月的资金储备。向中端市场的战略转型正在取得成效——平均客户价值（ACV）增长了40%，销售周期缩短至3周。第四季度的重点目标是完成300万美元的A轮融资，使年收入达到280万美元。”

### 2. 关键指标仪表盘（COO）
**最多6-8个指标，使用表格展示**

| 指标          | 本季度      | 上一季度    | 目标        | 状态       |
|---------------|-----------|---------|-----------|-----------|
| 年收入（ARR）      | 240万美元   | 197万美元   | 230万美元   | ✅        |
| 每月增长率      | 8.1%      | 7.2%      | 7.5%      | ✅        |
| 营运成本（Burn Rate） | 1.8倍      | 2.1倍      | <2倍      | ✅        |
| 新客户转化率（NRR）    | 112%      | 108%      | >110%      | ✅        |
| 客户获取成本（CAC）回收期 | 11个月     | 14个月     | <12个月     | ✅        |
| 员工人数        | 24人      | 21人      | 25人      | 🟡        |

选择董事会实际关注的指标。如果董事会表示不关心的指标，可以省略。

### 3. 财务更新（CFO）
- 利润表（P&L）摘要：收入、销售成本（COGS）、毛利率、运营费用（OpEx）、净支出（Net Burn）
- 现金状况及资金储备（持续时间）
- 营运成本趋势（过去三个季度）
- 与计划的偏差（差异及原因）
- 下一个季度的预测更新

**每个偏差都需要用一句话解释。**董事会不喜欢只说“收入低于目标”而不给出原因。

### 4. 收入与销售管线（CRO）
- 收入流水图：从开始到结束的完整过程
- 新客户转化率（NRR）和客户流失率（Logo Churn Rate）
- 按阶段划分的销售管线（以美元计，而不仅仅是数量）
- 预测：附带置信度
- 前三大交易项目：名称、金额、截止日期、风险

**预测必须附带置信度。**“我们预计收入为280万美元”这样的表述不够具体。“我们有90%的信心收入为260万美元，如果两个后期交易能够完成，收入可能达到290万美元”则更有参考价值。

### 5. 产品更新（CPO）
- 本季度发布的功能：列出3-5项，并说明每项功能对用户的影响
- 下季度即将发布的功能：列出3-5项，并提供预计发布日期
- 产品市场反馈（PMF）信号：净推荐值（NPS）趋势、日活跃用户（DAU）/月活跃用户（MAU）比率、功能采用情况
- 从客户调研中得出的关键发现

**不要列出所有功能。**只列出那些对用户有实际影响的功能。

### 6. 成长与营销（CMO）
- 按渠道划分的客户获取成本（CAC）
- 各渠道对销售管线的贡献（金额）
- 与当前阶段相关的品牌/知名度指标（流量、市场份额）
- 哪些策略有效，哪些需要调整，哪些正在测试中

### 7. 工程与技术（CTO）
- 交付速度趋势（过去四个季度）
- 技术债务比率及应对计划
- 基础设施状况：运行时间、故障情况、成本趋势
- 安全性状况（简要说明，如有待解决的问题）

**除非有重大问题，否则这部分内容要简短。**董事会不需要详细的技术细节。

### 8. 团队与人员（CHRO）
- 实际员工人数与计划人数对比
- 招聘情况：发出的招聘信息、招聘流程、填补空缺所需的时间
- 人员流失情况：哪些流失是不可避免的，哪些是可以避免的
- 员工满意度：最近一次调查的结果及趋势
- 本季度的重要招聘人员、当前空缺职位

### 9. 风险与安全（CISO）
- 安全性状况：关键控制措施的执行情况
- 合规性：正在进行的认证、截止日期
- 本季度发生的安全事件（如有）：影响、解决方法、预防措施
- 三大风险及应对措施

### 10. 战略展望（CEO）
- 下一个季度的重点任务：按优先级排列的3-5项
- 需要董事会做出的关键决策
- 提出的请求：预算、人员推荐、建议、投票事项

**“请求”这部分非常重要。**要具体明确。“我们希望获得3位B轮融资公司的CFO的介绍”比“希望得到任何帮助”更具体。

### 11. 附录
- 详细的财务模型
- 完整的销售管线数据
- 客户案例研究
- 详细的人员构成分析

## 叙述框架

董事会每个季度会收到10多份报告。你的报告需要有一个连贯的主题。

**四步叙述结构：**
1. **我们之前设定的目标**（上一季度的目标）
2. **我们实际达到的情况**（客观评估）
3. **产生差距的原因**（每个偏差都有具体原因，而非借口）
4. **我们正在采取的措施**（具体、有时间表的行动）

这种结构适用于好消息和坏消息。因为它承认了现实情况，因此更具可信度。

**开场方式：**从最重要的信息开始——董事会应在第3张幻灯片之前就了解核心内容，而不是等到第30张幻灯片。

---

## 如何传达坏消息

永远不要隐瞒坏消息。董事会迟早会知道。如果隐瞒得太晚，只会让情况更糟。

**框架：**
1. **直接说明**——“我们第三季度的收入目标未能实现，差了30万美元（12%的差距）”
2. **承认原因**——“主要原因是在企业领域销售周期比预期更长”
3. **表明理解问题**——“我们分析了8笔未能成交或进展缓慢的交易；问题在于……”
4. **提出解决方案**——“我们已经做出了3项调整：[具体调整内容及时间表]”
5. **更新预测**——“修订后的第四季度目标是260万美元；以下是详细的预测过程”

**不要这样做：**
- 不要用好消息来缓和坏消息——董事会会察觉到这种做法，并对报告的真实性产生怀疑
- 不要在不承担责任的情况下解释问题——“市场环境”只是背景信息，不是根本原因
- 不要在没有数据支持的情况下提出解决方案
- 不要在不说明假设的情况下展示修订后的预测

## 常见的董事会报告错误

| 错误                | 应对措施                |
|------------------|----------------------|
| 幻灯片太多（超过25张）      | 严格删减——如果无法在会议中解释清楚，那么这张幻灯片就没有意义 |
| 指标没有目标值        | 每个指标都需要设定目标值和状态          |
| 没有叙述内容          | 仅有数据会让董事会自行推断           |
| 隐藏坏消息            | 应首先公布坏消息，然后承担责任并提出解决方案 |
| 请求模糊不清          | 请求必须具体、可执行，并明确负责人员        |
| 不解释偏差原因        | 每个与目标之间的差距都需要用一句话说明原因     |
| 附录过时            | 附录只有在信息最新时才有用           |
| 为读者设计而非为会议准备      | 报告是用来口头展示的，因此设计时需要考虑口头表达的需要 |

## 发布节奏说明

**季度报告（标准格式）：**完整的报告，包含所有部分，20-30张幻灯片。提前48小时发送。
**月度报告（针对初创企业）：**精简内容——重点包括指标仪表盘、财务数据、销售管线、主要风险。8-12张幻灯片。
**融资报告：**开头介绍市场情况和公司愿景，结尾提出请求。具体格式请参考`references/deck-frameworks.md`。
