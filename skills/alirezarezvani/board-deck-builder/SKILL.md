# 板块资料库构建工具（Board Deck Builder）

这个工具用于创建能够讲述故事（而不仅仅是展示数据）的板块资料库。每个板块都有其负责人、相应的叙述内容以及“那么接下来该怎么做”的建议。

## 关键词
- 板块资料库（Board Deck）
- 投资者更新（Investor Update）
- 董事会会议（Board Meeting）
- 投资者关系（Investor Relations）
- 季度回顾（Quarterly Review）
- 董事会演示（Board Presentation）
- 筹资资料（Fundraising Deck）
- 投资者演示文稿（Investor Deck）
- 董事会叙述（Board Narrative）
- 季度业务回顾（Quarterly Business Review）

## 快速入门

```
/board-deck [quarterly|monthly|fundraising] [stage: seed|seriesA|seriesB]
```

提供可用的指标数据。该工具会使用占位符来填补数据空白——绝不会随意编造数字。

## 资料库结构（标准顺序）

每个板块的结构如下：**标题 → 数据 → 叙述 → 下一步行动**

### 1. 高管总结（CEO）
**不超过3句话。**
- 第一句话：公司现状（目前所处的位置）
- 第二句话：本季度发生的最重要事件
- 第三句话：下一季度的发展方向

**示例（不佳）：**“我们本季度表现良好，各个领域都取得了进展。”
**示例（优秀）：**“第三季度的收入达到240万美元（环比增长22%），签订了最大的企业合同，并为第四季度准备了充足的资金。向中端市场的战略转型取得了成效——平均客户价值（ACV）增长了40%，销售周期缩短至3周。第四季度的重点是完成300万美元的A轮融资，力争实现280万美元的收入。”

### 2. 关键指标仪表盘（COO）
**最多6-8个指标，使用表格展示。**

| 指标          | 本季度      | 上一季度    | 目标        | 状态       |
|----------------|-----------|---------|------------|-----------|
| 平均收入（ARR）     | 240万美元   | 197万美元   | 230万美元   | ✅        |
| 每月增长率（MoM）    | 8.1%      | 7.2%      | 7.5%      | ✅        |
| 营运成本（Burn Rate） | 1.8倍     | 2.1倍     | <2倍      | ✅        |
| 新客户留存率（NRR）    | 112%      | 108%      | >110%      | ✅        |
| 客户获取成本（CAC）回收期 | 11个月     | 14个月     | <12个月     | ✅        |
| 员工人数       | 24人       | 21人      | 25人      | 🟡        |

选择董事会实际关注的关键指标。如果某些指标无关紧要，可以省略。

### 3. 财务更新（CFO）
- 利润表（P&L）摘要：收入、销售成本（COGS）、毛利率、运营费用（OpEx）、净支出（Net Burn）
- 现金状况及资金使用期限（以月为单位）
- 营运成本趋势（过去三个季度）
- 实际情况与计划之间的差异（差异原因）
- 下一季度的预测更新

**对于每个差异，都要用一句话解释。**董事会不喜欢只说“收入未达到目标”而不给出原因。

### 4. 收入与销售管线（CRO）
- 收入流水图：从开始到结束的完整过程
- 新客户留存率（NRR）及客户流失率（Logo Churn Rate）
- 按阶段划分的销售管线（以美元计，而非数量）
- 下一季度的预测及置信度
- 前三大交易项目：名称、金额、预计完成日期、风险等级

**预测必须附带置信度。**“预计收入为280万美元”这样的表述不够具体。“有90%的信心达到260万美元，如果两个后期交易成功，收入可能达到290万美元”则更具参考价值。”

### 5. 产品更新（CPO）
- 本季度发布的功能：列出3-5项，并说明对用户的影响
- 下一季度计划发布的功能：列出3-5项，并提供预计发布日期
- 产品市场反馈（PMF）指标：净推荐值（NPS）趋势、每日活跃用户（DAU）/月活跃用户（MAU）比率、功能采用情况
- 从客户调研中得出的关键发现

**不要列出所有功能，只展示有实际用户反馈的功能。**

### 6. 成长与营销（CMO）
- 按渠道划分的客户获取成本（CAC）
- 各渠道对销售管线的贡献（以美元计）
- 与当前阶段相关的品牌/知名度指标（流量、市场影响力）
- 哪些策略有效，哪些需要调整，哪些正在测试中

### 7. 工程与技术（CTO）
- 过去四个季度的交付速度趋势
- 技术债务比率及应对计划
- 基础设施状况：运行时间、故障情况、成本趋势
- 安全性状况（简明说明，如有待解决的问题需特别标注）

**除非有重大问题，否则这部分内容应简短。**董事会不需要详细的技术细节。

### 8. 团队与人员（CHRO）
- 实际员工人数与计划人数对比
- 招聘情况：发出的招聘邀请、待填补职位、招聘周期
- 员工流失情况（分为可接受与不可接受的情况）
- 员工满意度：最新调查结果及趋势
- 本季度的重要招聘人员、空缺职位

### 9. 风险与安全（CISO）
- 安全性状况：关键控制措施的执行情况
- 合规性：正在进行的认证项目及截止日期
- 本季度发生的安全事件（如有）：影响、解决措施、预防措施
- 三大风险及应对措施

### 10. 战略展望（CEO）
- 下一季度的重点任务：按优先级排列的3-5个项目
- 需要董事会做出的关键决策
- 提出的请求：预算需求、人员推荐、建议、投票事项

**“请求”部分至关重要。**要具体明确。“我们希望获得B轮融资公司的三位CFO的推荐”比“希望得到任何帮助”更有效。

### 11. 附录
- 详细的财务模型
- 完整的销售管线数据
- 客户案例研究
- 详细的人员构成分析

## 叙述框架

董事会每个季度会收到10多份资料库。你的资料库需要有一个清晰的主题主线。

**四步叙述结构：**
1. **我们原本的目标**（上一季度设定的目标）
2. **我们实际的情况**（客观评估）
3. **产生差异的原因**（针对每个差异给出具体原因，而非借口）
4. **我们的应对措施**（具体且带有时间表的行动计划）

这种结构适用于好消息和坏消息的呈现。因为它承认了现实情况，因此更具可信度。

**开场方式：**从最重要的信息开始——董事会应在第三张幻灯片之前就了解核心内容。

---

## 如何传达坏消息

永远不要隐瞒坏消息。董事会迟早会知道，隐瞒只会让情况更糟。

**传达方式：**
1. **直接说明**——“我们本季度的收入目标未能实现，差了30万美元（差距为12%）”
2. **承认原因**——“主要原因是企业客户群体的销售周期比预期更长”
3. **表明理解问题**——“我们分析了8个失败或停滞的交易案例；问题在于……”
4. **提出解决方案**——“我们已经做出了3项调整：[具体调整内容及时间表]”
5. **更新预测**——“修订后的第四季度目标是260万美元；以下是详细的预测依据”

**切勿这样做：**
- 不要用好消息来缓和坏消息——董事会会察觉到这种做法，并对其真实性产生怀疑
- 不要在不承担责任的情况下解释问题——将“市场状况”作为借口
- 不要在没有数据支持的情况下提出解决方案
- 不要在不说明假设的情况下展示修订后的预测

## 常见的资料库制作错误

| 错误           | 应对措施       |
|-----------------|--------------|
| 幻灯片过多（超过25张） | 严格删减——如果无法在会议上解释，那这张幻灯片就没有必要 |
| 指标没有目标       | 每个指标都需要设定目标及状态 |
| 没有叙述内容       | 仅展示数据会让董事会自行推断 |
| 隐藏坏消息       | 应首先公布坏消息，然后承担责任，再提出解决方案 |
- 请求不明确       | 提出的请求必须具体、可执行，并明确负责人 |
| 不解释差异原因     | 每个与目标不符的差异都需要用一句话说明原因 |
| 附录过时         | 附录只有在信息最新时才有用 |
- 为阅读者而非听众设计资料库 | 资料库是用于口头展示的，必须便于大声朗读 |

## 发布节奏说明

**季度报告（标准格式）：**完整的资料库，包含所有板块，20-30张幻灯片。提前48小时发送。
**月度报告（适用于初创企业）：**精简内容——重点包括指标仪表盘、财务数据、销售管线、主要风险，共8-12张幻灯片。
**筹资报告：**以市场展望和战略目标开头，以提出资金需求结束。具体格式参考`references/deck-frameworks.md`。
