# /em:stress-test — 业务假设压力测试

**命令:** `/em:stress-test <假设>`

在任何市场出现问题之前，先对业务假设进行验证。这些假设可能包括收入预测、市场规模、竞争优势、招聘速度以及客户留存率等。

---

## 为什么大多数假设都是错误的

创始人天生具有乐观主义精神，这确实是一种优势——因为只有乐观才能从零开始创业。然而，当商业模式中的假设因这种乐观情绪而被夸大时，它们就会变成一种负担。

**最危险的假设往往是那些所有人都认同的假设。**

当整个团队都认为某个市场价值5000万美元是真实存在的，当每次投资者会议都进展顺利、你因此认为轮融资能够成功时，当你的模型预测到12月时年收入将达到200万美元且没有人对此提出质疑时——这时你才真正处于最危险的位置。

压力测试并非悲观主义，而是一种校准机制。

---

## 压力测试的方法论

### 第一步：明确假设

将假设具体化。不要只是说“我们的市场很大”，而应该说“德国中小企业在B2B支出管理软件领域的总可寻址市场规模为23亿欧元”。

假设越具体，就越容易进行测试；模糊的假设是无法被验证的，因此也就毫无意义。

**常见的假设类型包括：**
- **市场规模**：总可寻址市场（TAM）、服务可寻址市场（SAM）、可操作市场（SOM）；增长率；客户群体
- **客户行为**：支付意愿、客户流失率、客户推荐率
- **收入模型**：转化率、单笔交易金额、销售周期、客户获取成本（CAC）
- **竞争地位**：竞争优势的持久性、竞争对手的响应速度、客户转换成本
- **执行能力**：团队效率、招聘时间表、产品开发周期、运营扩展能力
- **宏观因素**：监管环境、经济状况、技术可用性

### 第二步：寻找反证

对于每一个假设，都要积极寻找证明其错误的证据。

可以问以下问题：
- 有人尝试过这个假设但失败了吗？
- 有哪些数据与这个假设相矛盾？
- 最坏的情况会是什么样的？
- 如果一个理智的怀疑者看到这个假设，他们会指出什么问题？
- 这种假设的基准数据是多少？

**反证的来源包括：**
- 在类似市场失败过的竞争对手
- 类似企业的客户流失数据
- 历史预测的准确性
- 行业报告中相互矛盾的数据
- 其他竞争对手的经验

测试的目的不是为了找到停止前进的理由，而是为了揭示你尚未了解的信息。

### 第三步：模拟最坏的情况

大多数计划只模拟正常情况和最佳情况，而压力测试则需要明确模拟最坏的情况。

**对于定量假设（如收入、增长、转化率）：**

| 情景 | 假设值 | 概率 | 影响 |
|----------|-----------------|-------------|--------|
| 正常情况 | [原始值] | ? | |
| 最坏情况 | -30% | ? | |
| 压力情况 | -50% | ? | |
| 灾难性情况 | -80% | ? | |

在每个阶段，关键问题是：**公司能否生存？这个计划是否合理？**

**对于定性假设（如竞争优势、产品与市场的匹配度、团队能力）：**
- 这个假设最早会在什么情况下被证明是错误的？
- 你需要多长时间才能发现这个错误？
- 从错误发生到被发现之间会发生什么？

### 第四步：分析敏感性

某些假设比其他假设更重要。敏感性分析的目的是为了了解：**如果这个假设发生变化，结果会如何改变？**

**示例：**
- 如果客户获取成本（CAC）翻倍，公司的现金流会如何受到影响？
- 如果客户流失率从5%上升到10%，24个月后的净收入增长率（NRR）会如何变化？
- 如果销售周期从3个月变为6个月，第三季度的收入会如何变化？

高敏感性意味着这个假设是影响公司成败的关键因素；如果假设错误，后果可能会非常严重。

### 第五步：制定应对策略

对于每个高风险假设，都应制定相应的应对措施：
- **验证策略**：在做出决策前先进行测试（如小规模试点、与客户沟通、开展小实验）
- **应急策略**：如果假设错误，备选方案是什么？
- **预警策略**：有哪些指标可以提前提示这个假设即将失败？

## 按假设类型划分的压力测试内容

### 收入预测

**常见的错误：**
- 自下而上的模型假设所有潜在客户都会转化
- 未考虑交易进度延迟、客户流失和季节性因素
- 在大规模应用之前就假设新渠道会有效

**压力测试问题：**
- 你过去在类似项目中的实际转化率是多少？
- 如果最重要的3笔交易推迟到下个季度，收入会受到影响吗？
- 如果新销售代表需要4个月才能投入工作，而不是2个月，模型预测会如何变化？
- 如果扩张带来的收入没有实现，增长率会是多少？

**测试方法：**根据历史转化率来构建收入模型，而不是基于预期。

### 市场规模

**常见的错误：**
- 仅根据行业报告自上而下计算总可寻址市场（TAM），而未进行实际验证
- 将总市场与可服务市场（SAM）混为一谈
- 假设100%的目标客户群体（ICP）都能被覆盖

**压力测试问题：**
- 你的目标客户群体中实际有多少家公司存在？你能列举出它们的名字吗？
- 在第1到第3年，你的可服务市场规模是多少？
- 目前有多少目标客户正在使用你的解决方案？
- “赢得”这些客户需要达到什么样的市场份额？

**测试方法：**列出目标客户名单，统计数量，然后乘以每个客户的平均收入（ACV），即可得到可服务市场规模（SAM）。

### 竞争优势

**常见的错误：**
- 认为竞争优势（如技术优势）可以在6个月内建立
- 未考虑到网络效应（这些效应尚未显现）
- 假设自己拥有竞争对手尚未具备的数据优势

**压力测试问题：**
- 如果一个资金充足的竞争对手在90天内复制了你的最佳功能，客户会怎么做？
- 在尝试过其他解决方案的客户中，你的留存率是多少？
- 这种竞争优势在扩大规模后是否仍然存在？
- 竞争对手需要付出多少成本才能达到你的功能水平？

**测试方法：**询问流失的客户他们离开的原因，以及竞争对手是否有可能留住他们。

### 招聘计划

**常见的错误：**
- 招聘时间基于标准周期，而非当前市场实际情况
- 未考虑招聘人员的成长周期（通常需要3到6个月才能达到满负荷工作）
- 计划的成败依赖于某个特定人员的招聘

**压力测试问题：**
- 如果销售副总裁的招聘需要5个月而不是2个月，计划会受到影响吗？
- 如果只招聘到计划人数的70%，执行效果会如何？
- 如果关键人员明天离职，会对计划造成多大的影响？
- 如果招聘停滞，现有团队还能完成计划吗？

**测试方法：**在假设没有新招聘的情况下模拟计划，看看哪些部分仍然可行。

### 竞争对手的反应

**常见的错误：**
- 假设现有竞争对手不会做出反应（但实际上他们很可能会采取行动）
- 低估了竞争对手的响应速度
- 未考虑到资源分配的不对称性

**压力测试问题：**
- 如果市场领导者在6个月内复制了你的产品，价格会如何变化？
- 如果竞争对手筹集3000万美元来进攻你的市场，你会如何应对？
- 你的客户中有哪些与竞争对手有合作关系？

## 压力测试的输出结果

```
ASSUMPTION: [Exact statement]
SOURCE: [Where this came from — model, investor pitch, team gut feel]

COUNTER-EVIDENCE
• [Specific evidence that challenges this assumption]
• [Comparable failure case]
• [Data point that contradicts the assumption]

DOWNSIDE MODEL
• Bear case (-30%): [Impact on plan]
• Stress case (-50%): [Impact on plan]
• Catastrophic (-80%): [Impact on plan — does the business survive?]

SENSITIVITY
This assumption has [HIGH / MEDIUM / LOW] sensitivity.
A 10% change → [X] change in outcome.

HEDGE
• Validation: [How to test this before betting on it]
• Contingency: [Plan B if it's wrong]
• Early warning: [Leading indicator to watch — and at what threshold to act]
```
