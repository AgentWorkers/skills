# /em:stress-test — 业务假设压力测试

**命令:** `/em:stress-test <假设>`

在任何市场问题出现之前，先对业务假设进行验证。这些假设可能包括收入预测、市场规模、竞争优势、招聘速度以及客户留存率等。

---

## 为什么大多数假设都是错误的

创始人天生具有乐观主义精神，这是启动一项事业所必需的。然而，当业务模型中的假设因这种乐观情绪而被夸大时，它们反而会成为一种负担。

**最危险的假设往往是那些所有人都认同的假设。**

当整个团队都认为某个市场价值5000万美元是真实存在的，当每次投资者会议都进展顺利，从而你认为融资轮次能够成功完成；当你的模型预测到12月时年收入将达到200万美元且没有人对此提出质疑时——这时你才真正处于风险之中。

压力测试并非悲观主义，而是一种校准机制。

---

## 压力测试方法论

### 第一步：明确假设

将假设具体化。不要只是说“我们的市场很大”，而应该明确指出“德国中小企业在B2B支出管理软件领域的总可寻址市场规模为23亿欧元”。

假设越具体，就越容易进行测试；模糊的假设是无法被验证的，因此也就毫无意义。

**常见的假设类型包括：**
- **市场规模**：总可寻址市场（TAM）、服务可寻址市场（SAM）、可操作市场（SOM）；增长率；客户细分
- **客户行为**：支付意愿、客户流失率、客户推荐率
- **收入模式**：转化率、单笔交易金额、销售周期、客户获取成本（CAC）
- **竞争地位**：竞争优势的持久性、竞争对手的响应速度、客户转换成本
- **执行能力**：团队执行速度、招聘时间表、产品开发时间表、运营扩展能力
- **宏观因素**：监管环境、经济状况、技术可用性

### 第二步：寻找反证

针对每个假设，积极寻找证明其错误的证据。

可以问以下问题：
- 有没有人尝试过类似的事情但失败了？
- 有哪些数据与这个假设相矛盾？
- 最坏的情况会是什么样的？
- 如果一个理智的怀疑者看到这种情况，他们会指出什么问题？
- 这种假设的基准数据是多少？

**反证的来源包括：**
- 在类似市场中失败过的可比公司
- 类似企业的客户流失数据
- 历史预测的准确性
- 行业报告中相互矛盾的数据
- 尝试过类似策略的竞争对手的经验

**测试的目的不是为了找到停止的理由，而是为了揭示你尚未了解的信息。**

### 第三步：模拟最坏情况

大多数计划只模拟理想情况（基线情况）和最佳情况。压力测试则需要明确模拟最坏的情况。

**对于定量假设（如收入、增长率、转化率）：**

| 情景 | 假设值 | 概率 | 影响 |
|----------|-----------------|-------------|--------|
| 基线情况 | [原始值] | ? | |
| 最坏情况 | -30% | ? | |
| 压力情况 | -50% | ? | |
| 灾难性情况 | -80% | ? | |

在每个阶段，关键问题是：**公司能否生存？这个计划是否合理？**

**对于定性假设（如竞争优势、产品市场匹配度、团队能力）：**
- 这个假设出错的最早迹象是什么？
- 你需要多长时间才能发现这个问题？
- 从假设出错到被发现之间会发生什么？

### 第四步：分析敏感性

某些假设的重要性高于其他假设。敏感性分析用于回答：**如果这个假设发生变化，结果会改变多少？**

**示例：**
- 如果客户获取成本（CAC）翻倍，公司的盈利周期会如何变化？
- 如果客户流失率从5%上升到10%，24个月后的净收入增长率（NRR）会如何变化？
- 如果销售周期从3个月变为6个月，第三季度的收入会如何受到影响？

**高敏感性的假设意味着该假设对公司至关重要；如果假设错误，将导致严重问题。**

### 第五步：制定应对策略

对于每个高风险假设，都应制定相应的应对措施：
- **验证策略**：在做出决策前先进行测试（例如小规模试点、与客户沟通或开展小实验）
- **应急策略**：如果假设错误，备选方案是什么？
- **早期预警策略**：有哪些指标可以帮助你在为时已晚之前发现问题的迹象？

## 按假设类型划分的压力测试内容

### 收入预测

**常见错误：**
- 自下而上的模型假设所有潜在客户都会转化
- 未考虑交易进度延误、客户流失和季节性因素
- 在大规模应用之前就假设新渠道会有效

**压力测试问题：**
- 你过去在潜在客户中的实际转化率是多少？
- 如果最重要的3笔交易推迟到下个季度，收入会受到影响吗？
- 如果新销售代表需要4个月才能适应工作，模型预测会如何变化？
- 如果扩张带来的收入没有实现，增长率会是多少？

**测试方法：**根据历史转化率来构建收入模型，而不是基于预期值。

### 市场规模

**常见错误：**
- 仅根据行业报告自上而下计算总可寻址市场（TAM），而未进行验证
- 将总市场与实际可服务市场混淆
- 假设100%的服务可寻址市场（SAM）都能被覆盖

**压力测试问题：**
- 你的目标客户群体中实际有多少家公司存在？你能列举出来吗？
- 在前3年内，你的服务可寻址市场规模是多少？
- 目前有多少目标客户正在使用类似解决方案？
- “赢得”这些客户需要达到什么样的市场份额？

**测试方法：**列出目标客户名单，统计数量，然后乘以每个客户的平均价值（ACV），即可得到服务可寻址市场（SAM）。

### 竞争优势

**常见错误：**
- 认为竞争优势（如技术优势）可以在6个月内建立
- 未考虑到网络效应的影响
- 假设数据优势可以在没有足够规模的情况下实现

**压力测试问题：**
- 如果一个资金充足的竞争对手在90天内复制了你的最佳功能，客户会怎么做？
- 在尝试过其他解决方案的客户中，你的留存率是多少？
- 这种竞争优势在扩大规模后是否仍然存在？
- 竞争对手需要付出多少成本才能达到你的功能水平？

**测试方法：**询问流失的客户他们离开的原因，以及竞争对手是否有可能留住他们。

### 招聘计划

**常见错误：**
- 招聘时间基于标准周期，而非当前市场实际情况
- 未考虑招聘人员的适应期（通常需要3-6个月才能达到满负荷工作）
- 计划的有效性依赖于特定人员的招聘

**压力测试问题：**
- 如果销售副总裁的招聘需要5个月而不是2个月，计划会受影响吗？
- 如果只招聘到计划人数的70%，执行效果会如何？
- 如果关键人员明天离职，会对计划造成多大的影响？
- 如果招聘冻结，现有团队能否完成计划？

**测试方法：**在假设没有新招聘的情况下模拟计划，看看哪些部分仍然可行。

### 竞争对手的反应

**常见错误：**
- 假设现有竞争对手不会做出反应（但实际上他们很可能会）
- 低估了竞争对手的响应速度
- 未考虑资源不对称性

**压力测试问题：**
- 如果市场领导者在6个月内复制了你的产品，价格会如何变化？
- 如果竞争对手筹集3000万美元来进攻你的市场，你会如何应对？
- 你的客户中有多少与竞争对手有合作关系？

## 压力测试的输出结果

```
ASSUMPTION: [Exact statement]
SOURCE: [Where this came from — model, investor pitch, team gut feel]

COUNTER-EVIDENCE
• [Specific evidence that challenges this assumption]
• [Comparable failure case]
• [Data point that contradicts the assumption]

DOWNSIDE MODEL
• Bear case (-30%): [Impact on plan]
• Stress case (-50%): [Impact on plan]
• Catastrophic (-80%): [Impact on plan — does the business survive?]

SENSITIVITY
This assumption has [HIGH / MEDIUM / LOW] sensitivity.
A 10% change → [X] change in outcome.

HEDGE
• Validation: [How to test this before betting on it]
• Contingency: [Plan B if it's wrong]
• Early warning: [Leading indicator to watch — and at what threshold to act]
```
