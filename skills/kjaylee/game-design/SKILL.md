# 🎮 游戏设计技能

> 新游戏设计时需要遵循的框架、检查清单和模式汇总
> 学习背景：请参考 specs/game-design-study.md

## 1. 游戏设计框架

### 第一步：定义游戏概念（30分钟）
```
1. 한 줄 설명 (엘리베이터 피치)
   → "플레이어는 [행동]을 해서 [목표]를 달성하는 [장르] 게임"

2. MDA 정의
   - Target Aesthetics: 어떤 감정을 주고 싶은가? (최대 3개)
   - Core Dynamics: 어떤 행동 패턴을 유도하는가?
   - Core Mechanics: 어떤 규칙/액션으로 구현하는가?

3. 레퍼런스 게임 2-3개
   → 유사한 성공 게임 분석 (뭘 배울 것인가?)

4. 차별화 포인트 1개
   → "이 게임이 [레퍼런스]와 다른 점은..."
```

### 第二步：设计核心游戏循环（1小时）
```
1. Moment Loop (순간, 1-5초)
   [행동] → [결과] → [피드백] ↻

2. Session Loop (세션, 2-5분)
   [시작] → [핵심플레이] → [결과] → [보상] → [다음?] ↻

3. Day Loop (일일)
   [데일리보상] → [세션] → [진행] → [내일 동기] ↻

4. Meta Loop (장기)
   [컬렉션/진행] → [새 콘텐츠] → [목표 갱신] ↻
```

### 第三步：制作原型（2-4小时）
```
1. 핵심 메카닉만 구현 (최소 플레이 가능)
2. 아트 없음 OK — 도형/색상으로
3. 10초 규칙 테스트: "10초 안에 재미?"
4. 3인 이상 플레이테스트
```

### 第四步：平衡性调整（反复进行）
```
1. 난이도 곡선 설계 (S-커브)
2. 보상 스케줄 설정 (FR + VR 혼합)
3. 세션 길이 측정 (목표: 2-5분)
4. DDA 파라미터 정의
5. 데이터 기반 조정
```

### 第五步：盈利模式设计
```
1. 광고 배치 계획 (타이밍, 빈도)
2. IAP 상품 설계 (있을 경우)
3. Stars 연동 계획 (텔레그램)
4. 무과금 경험 보장 확인
```

### 第六步：优化与发布
```
1. FTUE 최적화
2. UI/UX 점검 (체크리스트 참조)
3. QA 테스트 (specs/game-test-case-template.md)
4. 성능 최적화 (모바일)
5. 출시 & 데이터 모니터링
```

## 3. 各游戏类型的规划模板

### 3.1 益智游戏（匹配类、拼图类等）
```markdown
## 퍼즐 게임 기획 요소

### 핵심 메카닉
- 그리드 크기: _×_
- 매칭/배치 규칙: ___
- 클리어 조건: ___
- 특수 아이템/콤보: ___

### 난이도 설계
- 레벨 수: ___
- 난이도 곡선: S-커브 (초반 완만→중반 가파름→후반 완만)
- 새 메카닉 도입 주기: 매 _레벨마다
- DDA 적용 여부: Y/N
- 풀이 보장: 역방향 생성 / 솔버 검증

### 레벨 구조
- Level 0 (튜토리얼): 실패 불가
- Level 1-5: 기본 메카닉
- Level 6-10: 첫 번째 도전
- Level 11+: 복합 메카닉

### 힌트/보조 시스템
- 힌트 타이밍: 무행동 _초 후
- 셔플 기능: Y/N
- Undo 기능: Y/N
- 부스터: ___

### 성공 기준
- 레벨당 평균 플레이 시간: 1-3분
- 레벨 클리어율: >80%
- D1 리텐션 목표: >35%
```

### 세션 설계

## 4. 平衡性设计指南

### 4.1 难度曲线设置
```
# 레벨별 난이도 (1-100 스케일)
Level 1-3:   난이도 5-15  (튜토리얼, 실패 불가)
Level 4-10:  난이도 15-30 (기본기 학습)
Level 11-20: 난이도 30-50 (본격 도전 시작)
Level 21-50: 난이도 50-75 (핵심 도전 구간, 가장 가파름)
Level 51-80: 난이도 75-90 (고급 도전)
Level 81+:   난이도 90-95 (마스터 구간, 상승 완화)
```

### 4.2 奖励系统设计
```
# 보상 빈도
- 행동 보상: 매 1-5초 (시각/청각 피드백)
- 라운드 보상: 매 2-5분 (점수, 코인)
- 세션 보상: 매 10-15분 (레벨업, 해금)
- 일일 보상: 매 24시간 (데일리 보상)
- 마일스톤: 매 주/월 (큰 보상)

# 보상 가치 곡선
- 데일리 Day 1: 100
- 데일리 Day 2: 120
- 데일리 Day 3: 150
- 데일리 Day 4: 200
- 데일리 Day 5: 250
- 데일리 Day 6: 350
- 데일리 Day 7: 500 (보너스 포함)
```

### 4.3 关卡时长设计

| 游戏类型 | 目标关卡时长 | 最短关卡时长 | 最长关卡时长 |
|------|-----------|-----------|-----------|
| 益智游戏 | 3分钟 | 1分钟 | 10分钟 |
| 街机游戏 | 2分钟 | 30秒 | 5分钟 |
| 生存游戏 | 15分钟 | 5分钟 | 30分钟 |
| 儿童游戏 | 2分钟（主动玩法） | 30秒 | 5分钟 |

### 4.4 经济系统平衡性公式
```
# 인플레이션 방지
업그레이드 비용 증가율 > 수입 증가율

# 기본 공식
Level N 비용 = Base × (Growth_Rate ^ N)
Level N 보상 = Base_Reward × (Reward_Rate ^ N)
where Growth_Rate > Reward_Rate (살짝)

# 무과금 진행 보장
전체 콘텐츠 클리어 필요 시간: 무과금 ≤ 과금 × 3
(과금이 시간을 3배 이상 단축하지 않도록)
```

## 5. 新手引导设计模式

### 模式1：**“Level 0”阶段（推荐）**
```
- 첫 화면: 즉시 게임 화면 (메뉴 아님!)
- 핵심 조작 1개만 강조 (예: "탭하세요")
- 실패 불가능 (적 약함, 시간 무제한)
- 성공 → 격려 메시지 → 다음 조작 소개
- 2-3개 조작 학습 후 "진짜" 레벨 1 시작
```

### 模式2：**剧情引导式新手引导**
```
- 짧은 스토리로 시작 (10초 이내)
- "캐릭터를 도와주세요!" → 행동 유도
- 스토리와 튜토리얼 자연스럽게 통합
- 스킵 가능 필수
```

### 模式3：**逐步解锁机制**
```
- 첫 세션: 기본 기능만 표시
- 2-3 세션 후: 메타 기능 해금 (상점, 인벤토리)
- 1주 후: 고급 기능 해금 (길드, 이벤트)
- 각 해금 시 미니 튜토리얼
```

### 模式4：**先展示再操作**
```
1. 시연: NPC/가이드가 한 번 보여줌
2. 따라하기: 플레이어가 동일 행동
3. 응용: 약간 다른 상황에서 스스로
4. 독립: 가이드 없이 수행
```

### 新手引导效果评估指标
- 教程开始率：>90%
- 教程完成率：>70%
- 第一关通过率：>85%
- 教程完成时间：<90秒
- 教程后的用户流失率：<20%

## 6. 盈利模式设计模式

### 模式A：纯广告模式（Block Blast模式）
```
적합: 캐주얼 퍼즐, 대량 유저 목표
구조:
- 배너: 게임 외 화면 하단
- 인터스티셜: 레벨 종료/게임오버 후
- 리워디드: 리트라이, 보상 2배, 힌트
- 빈도: 인터스티셜 3분+ 간격
장점: 심플, 유저 부담 없음
단점: 대량 DAU 필요 ($0.05-0.15/DAU/day)
```

### 模式B：IAP + 广告模式（Candy Crush模式）
```
적합: 에너지 시스템 게임
구조:
- 에너지 시스템 (라이프)
- 부스터/파워업 판매
- No-Ads 옵션
- 리워디드 광고 = IAP 체험
장점: 고수익 (고래 유저)
단점: 밸런스 어려움, Pay-to-Win 리스크
```

### 模式C：Telegram Stars（Mini App模式）
```
적합: 텔레그램 Mini App 게임
구조:
- 기본 게임 무료
- Stars로 코스메틱 구매
- Stars로 프리미엄 콘텐츠 해금
- Stars로 부스터/편의 기능
- Mini App Store 피처 조건 충족
연동: Bot API sendInvoice / WebApp openInvoice
주의: Apple/Google 30% 수수료 고려
```

### 模式D：一次性付费模式
```
적합: 고품질 인디, 코어 타겟
구조:
- 기본 버전 무료 (데모)
- 풀 버전 일회성 구매
장점: 심플, 유저 신뢰
단점: 모바일에서 전환율 낮음
```

### 盈利模式设计原则
1. **提供无付费体验** — 即使不付费也能享受游戏乐趣
2. **减少首次游戏中的广告** — 优先建立用户关系
3. **明确价值交换** — “购买这个功能后会有什么好处”
4. **避免过度强制支付** — 禁止使用弹窗广告
5. **广告应在自然的游戏流程中出现** — 绝不在游戏过程中弹出广告

## 8. 快速参考

### 关键数据指标

| 指标 | 目标值 |
|------|--------|
| 首次游戏启动时间 | <10秒 |
| 单关时长 | 2-5分钟 |
| 触控区域大小 | ≥44×44像素 |
| 输入响应时间 | <100毫秒 |
| 教程完成率 | >70% |
| D1日留存率 | >35% |
| D7日留存率 | >15% |
| D30日留存率 | >5% |
| 关卡通过率 | >80% |
| 关卡间隔时间 | >3分钟 |

### 需避免的设计缺陷 🚩
- 无法用一句话描述核心游戏循环
- 用户在10秒内无法感受到游戏乐趣
- 首次游戏中出现3个以上弹窗
- 连续失败3次后没有帮助提示
- 文本教程时长超过30秒
- 关卡结束后没有引导用户进入下一关的提示
- 无法保存游戏进度
- 用户没有继续游戏的理由
