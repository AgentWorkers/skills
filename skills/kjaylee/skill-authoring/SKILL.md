**一、SKILL.md 文件的结构与内容要求**  
SKILL.md 文件应遵循以下结构：  
1. **文件头部**（文件开头）：包含文件名称、版本号、创建日期等元信息。  
2. **技能描述**：简要介绍该技能的功能、用途及适用场景。  
3. **技能实现细节**：详细说明技能的具体实现逻辑，包括使用的编程语言、技术栈、依赖库等。  
4. **使用说明**：提供如何使用该技能的步骤和注意事项。  
5. **示例代码**：展示技能的核心功能实现代码（可选）。  
6. **测试与调试**：说明如何进行技能的测试和调试过程。  
7. **文档更新**：记录技能的更新历史和版本信息。  

**二、编写技能描述时的注意事项**  
- 使用简洁明了的语言描述技能的核心功能。  
- 强调技能的独特性和优势。  
- 提供适用场景和限制条件（如系统要求、环境配置等）。  

**三、编写实现细节时的注意事项**  
- 详细列出所使用的编程语言和技术栈。  
- 说明代码的结构和逻辑流程。  
- 包括所有必要的依赖库和配置信息。  
- 提供代码示例（如果适用），并附上注释以解释代码的作用。  

**四、编写使用说明时的注意事项**  
- 以清晰的步骤引导用户了解如何使用该技能。  
- 提供故障排除和问题解决的指导。  
- 避免使用过于复杂的术语或命令。  

**五、示例代码编写指南**  
- 使用 Markdown 格式编写代码示例。  
- 为代码添加注释，解释其作用和实现原理。  
- 确保代码示例能够准确反映技能的功能。  

**六、测试与调试建议**  
- 提供测试用例和测试环境的信息。  
- 鼓励开发者进行充分的测试，以确保技能的稳定性和可靠性。  
- 提供调试技巧和常见问题解决方法。  

**七、文档更新与管理**  
- 定期更新 SKILL.md 文件，以反映技能的最新功能和修改内容。  
- 记录每次更新的信息（如版本号、修改日期、修改内容等）。  

**八、总结**  
通过遵循上述指南，您可以编写出高质量、易于维护的 SKILL.md 文件，从而有效提升代理的功能和性能。

# 技能编写指南

本指南介绍了如何根据 agentskills.io 的标准编写高效、易于使用的技能。

## 核心原则

### 1. 简洁性是关键
“上下文窗口”是所有技能共享的资源。请确保每个技能的描述都简洁明了。

**基本假设：**代理已经具备较高的智能水平。因此，只需补充代理未知的信息。在编写任何描述时，请不断问自己：
- “这个解释真的有必要吗？”
- “这段文字是否能够合理地解释所需的操作步骤？”

相比冗长的说明，简洁的示例往往更有效。

### 2. 自由度的匹配

| 自由度 | 适用场景 | 表现形式 |
|--------|--------|--------|
| **高** | 多种实现方法均可行，需根据具体情境选择 | 以文本形式提供的指导说明 |
| **中** | 存在推荐的实现模式，允许一定程度的变通 | 包含伪代码或参数的脚本 |
| **低** | 对操作要求较高，需要保持一致性 | 具体化的脚本，参数数量最少 |

**低自由度**：例如在狭窄的路径上行走（如悬崖旁），需要明确的护栏来确保安全；**高自由度**：则像在广阔的平原上，可以有多种路径可供选择。

## SKILL.md 文件结构

```yaml
---
name: my-skill-name        # 필수, 소문자+하이픈, 1-64자
description: >             # 필수, 1-1024자
  무엇을 하는지 + 언제 사용하는지 + 관련 키워드
license: Apache-2.0        # 선택
metadata:                  # 선택
  author: misskim
  version: "1.0"
---

# 스킬 제목

[에이전트가 따를 지침 - 마크다운 자유형식]
```

### 必填字段
- **name**：必须与目录名匹配，仅允许使用小写字母和连字符（-）。
- **description**：该字段用于触发技能的执行，因此应包含丰富的关键词。

### 目录结构
```
skill-name/
├── SKILL.md          # 필수
├── scripts/          # 실행 코드 (Python/Bash/JS)
├── references/       # 상세 문서 (필요 시 로드)
└── assets/           # 정적 리소스 (템플릿, 이미지, 데이터)
```

## 针对当前环境的特殊注意事项

1. **Clawdbot 代理**：技能文件应放置在 `skills/` 或 `~/.clawdbot/skills/` 目录下。
2. **MiniPC 的集成**：可以通过 `nodes.run` 命令将技能任务委托给外部系统。
3. **子代理**：技能可以被设计为能够自动创建子代理来执行特定任务。
4. **安全性**：`scripts/` 目录中仅应存放可信任的脚本代码。
5. **SKILL.md 文件的长度应控制在 500 行以内**——详细内容可另存于 `references/` 目录中。

## 差劲的技能与优秀的技能

**差劲的技能示例：** “能够处理 PDF 文件”（描述过于简略）
**优秀的技能示例：** “能够从 PDF 文件中提取文本和表格内容，填写表单，并合并多个 PDF 文件。适用于需要处理 PDF 文件或提取文档信息的场景。”

**差劲的技能示例：** 重复讲解代理已经掌握的 Python 语法知识。
**优秀的技能示例：** 提供与代理领域相关的专业知识或工作流程指导。
