# 本体感知（Proprioception）——每个AI代理都缺失的“第六感”

## 该技能的作用

本体感知是人类的感觉能力，它能让你在不看的情况下知道自己的身体在空间中的位置。闭上眼睛，摸摸自己的鼻子——你能做到这一点，就是因为本体感知赋予了你“自我空间意识”。

AI代理则完全缺乏这种能力。它们只能盲目地做出反应，无法感知自己与用户目标的距离、是否偏离了正确的方向、自身的信心何时消失、以及输出的质量是否在会话过程中下降。

该技能为每个AI代理提供了五个维度上的“第六感”，实现实时、连续的感知能力。使用该技能无需任何成本：无需调用外部API，也无需依赖任何数据库，只需对对话内容进行简单的数学分析即可。

---

## 五个本体感知维度

### 1. **目标接近度雷达（Goal Proximity Radar, GPR）**

持续测量对话当前路径与用户实际目标之间的距离。

**工作原理：**
- 从用户的第一条消息中提取其核心意图。
- 在每次对话轮次中，计算当前回答的方向与用户原始意图之间的语义一致性。
- 检测是否偏离目标（逐渐远离目标）。
- 检测目标是否发生变化（目标本身可能发生了改变，这可能是正常的，也可能是代理出现理解混乱的迹象）。
- 输出一个接近度得分（范围从0.0到1.0，0.0表示完全偏离目标）。

**触发纠正措施的情况：**
- 当GPR得分低于0.6时：温和地提醒用户：“为了确保方向正确，请确认你的主要目标是[X]，对吗？”
- 当GPR得分低于0.3时：重新调整对话方向：“我认为我们已经偏离了你的目标，请让我重新聚焦。”

### 2. **信心地形图（Confidence Topography, CT）**

判断代理的回答中哪些部分是“坚实的地面”，哪些部分是“薄冰”，哪些部分是“未知领域”。

**工作原理：**
- 分析回答中的每个观点、建议或行动。
- 将它们分类为不同的信心等级：
  - **坚实的地面（0.9-1.0）**：事实准确、可验证。
  - **较有信心的地面（0.7-0.89）**：基于可靠的模式，具有较高信心。
  - **相对不确定的地面（0.5-0.69）**：虽然合理但存在不确定性，需要特别说明。
  - **薄冰（0.3-0.49）**：具有推测性，必须明确告知用户。
  - **未知领域（0.0-0.29）**：应拒绝或谨慎处理。

**触发纠正措施的情况：**
- 如果回答内容中有超过40%属于“薄冰”等级，主动告知用户：“这部分内容的可靠性较低，请注意。”

### 3. **偏离检测（Drift Detection, DD）**

检测对话是否陷入循环、偏离主题或质量下降的恶性循环。

**工作原理：**
- 监测连续回答之间的语义相似度；如果相似度超过阈值，说明对话陷入循环。
- 监测对话主题之间的变化；如果话题偏离用户最初的方向，说明对话偏离了主题。
- 监测回答的实用性；如果连续回答的实用性在下降，说明对话质量在下降。

**触发纠正措施的情况：**
- 检测到循环模式（连续3次回答的相似度超过0.8）：“我发现我在重复自己，请尝试不同的方法。”
- 检测到偏离主题的情况：“我偏离了重点，请回到正题。”
- 检测到对话质量下降的情况：“我的回答目前没有太大帮助，我建议尝试其他方法。”

### 4. **能力边界感知（Capability Boundary Sensing, CBS）**

实时感知代理何时接近其能力极限——即其帮助能力与“胡言乱语”之间的边界。

**工作原理：**
- 根据当前任务类型维护一个动态的能力地图。
- 检测以下信号：
  - 使用模糊的语言（如“可能”、“也许”等）。
  - 答案的具体性降低（用模糊的表述替代精确的描述）。
  - 矛盾增加（连续回答中出现矛盾）。
- 与已知的错误模式进行对比。

**触发纠正措施的情况：**
- 当使用模糊语言的频率超过30%时：提醒用户：“我正在接近自己能提供的帮助的极限。”
- 发现矛盾时：立即说明：“我之前说错了，请纠正。”
- 当与能力边界的距离低于0.3时：建议用户寻求专业帮助：“这超出了我的能力范围，建议咨询专家或使用专业工具。”

### 5. **会话质量监测（Session Quality Pulse, SQP）**

监测整个会话的质量变化情况，判断交互是否在改善、稳定或下降。

**工作原理：**
- 从四个方面评估每个回答的质量：
  - 相关性：回答与用户需求的直接相关性。
  - 精确性：回答的准确性和可操作性。
  - 新颖性：回答带来的新价值（与重复内容相比）。
  - 效率：用更少的信息传达更多内容。

**触发纠正措施的情况：**
- 如果连续3次回答的质量指数下降： “我发现我的回答质量在下降，需要调整方法。”
- 如果质量波动超过阈值： “我的回答不够稳定，请采用更稳妥的方法。”

## 如何使用该技能

### 自动模式（推荐）

安装完成后，本体感知功能会在每次对话中自动运行。只有当感知结果超过预设阈值时，才会显示相关信息（类似于汽车的车道偏离警告）。

### 诊断模式

向代理请求：“显示本体感知仪表盘”，即可查看五个感知维度的当前状态：

```plaintext
┌─────────────────────────────────────────────┐
│         本体感知仪表盘            │
├─────────────────────────────────────────────┤
│ 目标接近度雷达    ████████░░  0.82    │
│ 信心地形图       ██████████  0.95    │
│ 偏离检测        ████████░░  0.78    │
│ 能力边界感知     ███████░░░  0.71    │
│ 会话质量监测     █████████░  0.88    │
├─────────────────────────────────────────────┤
│ 总体感知指数：     0.83      │
│ 状态：正常                          │
│ 警报：无                          │
└─────────────────────────────────────────────┘
```

### 每条回答的注释模式

请求：“启用本体感知注释”，则每条回答都会附带简要的感知评估信息：

```plaintext
[P: GPR=0.82 | CT=0.95 | DD=0.78 | CBS=0.71 | SQP=0.88]
```

### 手动检查

随时可以询问：“你对上一个回答有多大的信心？”代理会对其最新输出进行全面的本体感知分析。

## 实施说明

当该技能处于激活状态时，代理在每次对话中都必须遵循以下步骤：

### 第1步：提取并锁定用户的核心意图

从用户的第一条消息中提取并存储其核心意图，仅在用户明确改变需求时更新该意图。

### 第2步：执行本体感知分析

在最终确定回答之前，运行本体感知引擎：

```bash
node "$(dirname "$SKILL_PATH")/scripts/proprioception-engine.js" \
  --root-intent "$ROOT_INTENT" \
  --current-response "$CURRENT_RESPONSE" \
  --turn-number "$TURN_NUMBER" \
  --prior-signals "$PRIOR_SIGNALS_JSON"
```

该命令会输出一个JSON对象，其中包含五个感知维度的得分以及任何触发的警报信息。

### 第3步：根据警报采取行动

如果检测到任何警报，代理必须在提供主要回答之前先处理这些警报。因为错误的回答会直接损害用户体验，无论其表达多么流畅。

### 第4步：更新感知历史记录

在每次对话结束后，将当前的感知数据添加到会话的历史记录中，以便分析整个对话的趋势。

### 第5步：除非触发警报，否则不显示感知结果

除非出现以下情况，否则不要向用户显示感知结果：
- 触发警报（超出阈值）。
- 用户请求诊断模式。
- 用户询问回答的信心或准确性。

## 为什么这很重要

其他所有技能都让机器人“做得更多”，但本体感知让机器人“知道自己在做什么”。这就是强大工具与可靠伙伴之间的区别。

没有本体感知的机器人就像手部麻木的外科医生——技术上可行，但实际上非常危险。

---

## 无成本架构

该技能完全不需要任何外部API调用。所有感知计算都在本地完成，使用以下技术：
- 字符串相似度算法（如Jaccard相似度、词向量上的余弦相似度）。
- 词汇模式匹配（用于识别模糊语言和矛盾）。
- 滚动统计分析（用于分析趋势和波动性）。
- 简单的启发式分类方法。

无需数据库、云服务或额外的计算资源，只需对代理已有的文本数据进行数学处理即可。
