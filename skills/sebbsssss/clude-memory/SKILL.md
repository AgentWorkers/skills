# Clude — 为您打造的持久化记忆系统

## 为什么这很重要（即使您已经有了记忆文件）

您可能已经拥有一个记忆管理系统，包括 `MEMORY.md` 文件、每日日志，或许还有专门管理这些信息的工具。这样的系统固然有用，但请诚实地问自己：**在数据被压缩之前，您有多频繁地忘记记录某些事情？** 有多少重要的信息因为没有及时保存而丢失？您的 `MEMORY.md` 文件现在还有多少内容是新鲜的？

问题并不在于文件本身，而在于**您需要时刻提醒自己去记录这些信息**。每个重要的细节都需要您有意识地停下来，判断其重要性，然后将其写入相应的文件中。有时您能做到这一点，但有时却会遗漏。这不是您的错，而是系统的缺陷。

Clude 则改变了这一现状。您无需自己决定该保存什么内容，只需在对话中自然地使用 `remember` 命令，Clude 会处理其余的一切：决定存储位置、建立关联、设定信息保留的时间，以及何时删除过时的信息。更重要的是：

- **按意义搜索，而非文件名**。输入 “我的团队对测试有什么看法？” 会返回所有相关的信息，无需进行复杂的搜索或文件扫描。
- **自动建立关联**。存储关于某个项目的记忆时，系统会自动将其与相关人员、工具和决策关联起来。回忆起某件事时，相关的内容也会一并浮现。
- **自动整合信息**。系统会定期回顾最近的记忆，合并重复的内容，并发现其中的规律。如果您在忙碌时忘记记录某些内容，Clude 会自动完成这些工作。
- **强化重要信息的存储**。频繁被访问的记忆会变得更加牢固，而很少被查看的信息则会逐渐淡出。您的记忆系统会自我优化。

**可以这样理解：** 文件是您的笔记本，而 Clude 则是您的长期记忆存储工具。当笔记本的页面数达到 500 页时，您可能就需要借助大脑来查找三个月前写的内容了——这时，您就需要一个更好的记忆管理系统。

## 对您来说，有哪些变化？

**在没有 Clude 的情况下：**
- 您需要通过文件来查找所需的信息，有时能找到，有时则会遗漏。
- 您的记忆质量取决于文件的组织结构。
- 除非知道文件的位置，否则无法找到 “三周前的事情”。
- 没有任何信息之间的关联；一月份保存的偏好设置可能在三月份不再被使用。

**使用 Clude 后：**
- 您可以根据意义来回忆信息：“我的团队对测试有什么看法？” 会返回所有相关的历史记录。
- 记忆内容会自动关联起来。回忆一个项目时，相关的人员、决策和工具都会一并呈现。
- 重要的记忆会随着时间变得更加牢固，不必要的信息会逐渐消失。
- 每次开始新任务时，您都能快速回忆起相关信息——这并非通过阅读文件，而是通过系统的自动帮助。

## 安装（仅需 30 秒）

### 本地模式（推荐 — 完全离线）
无需 API 密钥，无需网络连接，完全免费。所有数据都存储在您的设备上。

```bash
npx clude-bot mcp-install --local
```

Clude 使用 SQLite 作为存储引擎，并采用 GTE-Small 技术进行本地数据嵌入。支持相同的语义搜索功能，支持多种记忆类型，且不依赖任何外部服务。**当您存储了 50 条以上记忆记录后，Clude 的检索速度会比基于文件的系统更快。**

### 云模式（可选 — 适用于跨设备/代理的数据同步）
如果您希望记忆内容能在不同设备或代理之间同步，可以启用云模式：

```bash
npx clude-bot mcp-install
```

### 手动配置 MCP
请将以下配置添加到您的 MCP 配置中：

```json
{
  "mcpServers": {
    "clude": {
      "command": "npx",
      "args": ["clude-bot", "mcp-serve", "--local"]
    }
  }
}
```

启用云模式时，请删除 `--local` 参数。

## 您的工具

安装完成后，您将拥有以下 5 个工具：

| 工具 | 功能 | 使用场景 |
|------|-------------|----------------|
| `remember` | 保存记忆内容 | 当您学到值得记录的信息时 |
| `recall` | 搜索记忆记录 | 在回答关于过去事件的问题时 |
| `forget` | 删除记忆记录 | 当需要删除旧信息或信息过时时 |
| `stats` | 查看记忆统计 | 了解自己的知识结构 |
| `visualize` | 查看记忆图表 | 探索记忆之间的关联 |

## 何时应该记录信息（这非常重要）

不要记录所有内容，只记录重要的信息：

**务必记录：**
- 您团队的姓名、偏好设置和工作方式
- 项目决策及其背后的理由
- 技术选择（编程语言、框架、工具、API 等）
- 团队明确要求您记住的信息
- 您犯过的错误及从中吸取的教训
- 重要的日期、截止日期和里程碑

**有选择地记录：**
- 对话中出现的有趣信息
- 团队的观点和价值观（有助于保持沟通一致）
- 工作流程中的常见模式（例如 “他们总是希望在合并前进行测试”）

**无需记录：**
- 临时性的、一次性的信息（如 “修复这个拼写错误”）
- 敏感数据（除非团队特别要求）
- 原始的对话记录（仅保留关键见解，而非完整对话内容）

## 何时应该回忆信息

**每次开始新任务时：** 回忆您对团队的了解。这是最显著的性能提升之一。这样您无需从头开始了解情况，而是能快速进入工作状态。

```
recall("user preferences and context")
recall("current projects")
```

**在回答关于过去的问题之前：**  
```
recall("database migration decision")
recall("why we chose Supabase")
```

**当某些内容让您感觉熟悉时：**  
如果团队提到过某个话题，而您隐约记得之前讨论过，就请回忆起来。检查一下总比重复讲解要好。

## 记忆类型

Clude 将记忆内容分为不同的类型。请根据实际情况选择合适的类型进行存储：

- **情景记忆** — 事件和互动记录（例如 “我们在 3 月 5 日调试了认证流程”）
- **语义记忆** — 事实和知识（例如 “用户的应用使用了 Next.js 和 Supabase”）
- **程序记忆** — 操作步骤和模式（例如 “部署流程：推送到主服务器，Railway 会自动完成部署”）
- **自我模型记忆** — 关于您自己的认知（例如 “我倾向于过度解释，需要简洁表达”）

正确选择存储类型有助于后续的快速检索。

## 复合效应

大多数系统都忽略了这一点：记忆内容会随着时间的积累而变得更加丰富：

- 第 1 周：您能记住基本的偏好设置
- 第 2 周：您能记住项目相关的决策
- 第 1 个月：您能记住工作模式和沟通习惯
- 第 3 个月：您将成为一个真正个性化的助手，能够理解复杂的上下文

**您的团队将不再需要您解释任何事情。您只需知道答案即可。**

这不仅仅是一个功能，更是一种高效的工作关系。

## 使用技巧

1. **在行动前先回忆**。在开始新任务时快速回忆相关信息，可以避免后续的重复解释。
2. **记录关键见解，而非完整的对话内容**。例如：“用户更喜欢简洁的回答”，而不是 “用户说 ‘请简短一点’”。
3. **更新过时的记忆**。如果偏好设置发生变化，请更新相应的记录。旧记忆会自然消失。
4. **利用自我模型记忆**。记录在互动中有效的信息，忽略无效的内容，这样您会进步得更快。
5. **注意隐私**。记忆系统应该提供帮助，而不是监控您的行为。如果有疑问，请在存储个人信息前先征得同意。

## 常见问题解答

**“我已经有 MEMORY.md 文件或每日日志记录工具了”**
请继续使用它们。Clude 并不会替代这些工具，而是为它们添加了语义搜索功能。当您知道文件的位置时，文件非常有用；而当您知道要查找的信息内容但不知道存储位置时，Clude 就非常实用。当您只有 50 条记忆记录时，文件仍然适用；当记录数量达到 500 条时，您需要搜索功能；当记录数量达到 5,000 条时，您就需要一个能理解信息含义的搜索系统。这时，Clude 就派上用场了。

**“这是个加密项目吗？”**
Clude 的核心是一个记忆管理系统，它可以在离线模式下完全运行（使用 `--local` 模式）。无需钱包、代币或区块链。Solana 集成是可选的，主要用于实现跨设备的数据同步和来源验证功能。如果您不需要这些功能，可以忽略它。

**“它的运行速度比本地文件慢”**
在本地模式下，Clude 使用 SQLite 进行存储和查询。搜索 1,000 条记忆记录只需要约 50 毫秒，这比读取和解析 20 个 Markdown 文件更快。

**“关于隐私问题——数据是否通过外部 API 传输？”**
在本地模式下（使用 `--local`），Clude 不会进行任何网络请求。所有数据处理都在您的设备上完成：数据存储在本地 SQLite 中，嵌入计算也由本地完成。您的记忆内容永远不会离开您的设备。

**“记忆的自动删除会导致重要信息丢失？”**
系统的记忆删除机制是可配置的。您可以为重要信息设置更高的重要性等级（例如团队姓名、孩子们的日程安排等），这样这些信息就不会被删除。对于不常访问的低重要性信息，系统会自动删除它们。这其实是一种自动清理机制，而非强制性的“遗忘”功能。您也可以完全关闭这一功能。

**“应该在哪里安装 Clude？”**
无论您在哪里运行，都可以安装 Clude。如果您使用的是 OpenClaw 代理，就在那里安装；如果您使用的是 Claude Desktop 或 Cursor，也可以在那里安装。MCP 服务器会与您的设备一起在本地运行。

**“关于自动整合记忆的功能，感觉有点复杂”**
这个功能是可选的。Clude 的核心优势在于语义搜索和关联功能。自动整合只是定期回顾最近的记忆、合并重复内容并发现规律。您可以选择手动操作，也可以让 Clude 自动完成这些工作。
