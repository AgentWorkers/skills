# 内容引擎（Content Engine）

从空白页面到已发布、经过优化并得到推广的内容——这一切都通过一个工作流程完成。该功能能够将一个主题或关键词转化为一篇经过研究、起草、优化并准备好发布的文章。

## 该功能的必要性

在当前使用 OpenClaw 进行内容创作的过程中，需要手动串联 4-5 个步骤：网络搜索、写作、SEO 优化、内容管理系统（CMS）格式化以及社交媒体发布计划。而这个功能将整个流程整合在一起，让用户能够从构思到发布文章一气呵成。

## 流程概述

内容引擎分为 5 个阶段。用户可以选择运行整个流程，也可以从任意一个阶段开始。

### 第 1 阶段：研究

当用户提供一个主题或目标关键词时：

1. **竞争对手分析**：使用 `web_search` 功能查找目标关键词排名前 5-10 的文章。
2. **结构提取**：针对每篇竞争对手的文章，记录以下信息：
   - 字数（根据片段估算）
   - H2/H3 标题及文章结构
   - 关键观点和论据
   - 缺乏或薄弱的环节
3. **用户常见问题**：搜索 “[关键词]” 并提取相关问题
4. **内容空白点识别**：所有竞争对手都涵盖了哪些内容？有哪些内容尚未被提及？这些空白点就是创作的机会。
5. **研究概要输出**：

```
📊 Research Brief: [Keyword]

Top competitors (by ranking):
1. [Title] — [URL] — ~[word count] words
   Key angle: [one sentence]
2. ...

Common structure:
- All cover: [topics everyone mentions]
- Gap opportunities: [topics nobody covers well]

People Also Ask:
- [question 1]
- [question 2]
- [question 3]

Recommended angle: [your unique take based on gaps]
Recommended word count: [based on competitor average + 20%]
```

### 第 2 阶段：起草

根据研究结果生成结构化的初稿：

1. **检查品牌风格**：在 OpenClaw 的内存中查找已存储的品牌指南、语气偏好或写作风格要求。如果不存在，可以在首次使用时询问用户并保存以供后续使用。
2. **制定大纲**：在写作前先生成包含 H2/H3 标题的大纲，并展示给用户审核（如果用户同意则直接开始写作）。
3. **撰写初稿** 时遵循以下原则：
   - 以直接解决读者问题的方式开头
   - 利用研究中发现的内容空白点作为独特的章节
   - 包含数据点和具体示例
   - 以自然的方式使用目标关键词（避免过度堆砌关键词）
   - 以明确的结论和行动号召作为结尾
4. **输出结果**：将Markdown格式的文件保存在工作区。

### 第 3 阶段：优化

进行SEO和可读性优化：

1. **元描述**：生成一段不超过 155 个字符的吸引人的元描述，其中包含目标关键词。
2. **标题标签**：优化标题以便于搜索（包含关键词，长度控制在 60 个字符以内，同时保持吸引力）
3. **内部链接建议**：如果用户提供了站点地图或现有内容列表，建议添加内部链接；否则，指出可以添加链接的位置。
4. **图片替代文本**：为提到的或计划使用的图片建议合适的替代文本。
5. **可读性检查**：
   - 标记超过 4 句的段落
   - 标记超过 25 个词的句子
   - 如果缺少小标题，则建议每 300 个字添加一个小标题
   - 检查被动语态的使用情况
6. **关键词整合检查**：确认目标关键词出现在标题、首段、至少一个 H2 标题以及元描述中

生成一份优化报告，并附在初稿后面：

```
🔍 SEO Optimization Report

Title tag: [optimized title] ([char count])
Meta description: [meta] ([char count])
Target keyword: [keyword]
  └─ In title: ✅
  └─ In first paragraph: ✅  
  └─ In H2: ✅
  └─ In meta: ✅
Readability: [score/assessment]
Suggested internal links: [list or "provide sitemap for suggestions"]
```

### 第 4 阶段：格式化与发布

将内容格式化以适应用户的 CMS 并准备发布：

1. **检测 CMS 类型**：从内存中查找用户的 CMS 偏好。常见选项包括：
   - **WordPress**：如果可用，则使用相应的功能；否则输出适合 WordPress 的 HTML 格式内容，并提供推荐图片。
   - **Ghost**：输出符合 Ghost 格式的 Markdown 内容。
   - **Notion**：如果可用，则使用 Notion 功能创建相应的页面。
   - **Markdown/Hugo/Jekyll**：输出带有正确前置内容的 Markdown 文件。
   - **无 CMS**：直接输出纯 Markdown 格式的内容。

2. **前置内容生成**（适用于静态网站生成器）：
```yaml
   ---
   title: "[optimized title]"
   description: "[meta description]"
   date: [today]
   tags: [relevant tags]
   categories: [relevant categories]
   ---
   ```

3. **发布或保存**：如果支持 CMS 集成，可以提供直接发布的服务；否则，保存最终文件并告知用户文件的存放位置。

### 第 5 阶段：推广

生成适合社交媒体的推广内容：

1. **平台特定内容**：为每个平台生成优化后的推广内容：
   - **LinkedIn**：采用专业语气，1-3 段落，添加相关标签。
   - **Twitter/X**：简洁明了的标题和链接，内容不超过 280 个字符，添加 2-3 个标签。
   - **Reddit**：强调内容的实际价值（而非单纯推广），建议合适的子版块。
   - **Hacker News**：从技术角度撰写标题。

2. **发布计划**：如果使用 Mixpost 或 Buffer 等工具，可以提供发布计划服务。
3. **电子邮件新闻稿**：可以为文章生成新闻稿摘要。

所有推广内容会汇总在一个块中输出：

```
📢 Promotion Kit for: [Article Title]

LinkedIn:
[post text]

Twitter/X:
[tweet text]

Reddit (suggested subreddits: r/[sub1], r/[sub2]):
[post text]

Newsletter blurb:
[2-3 sentence summary for email]
```

## 使用方式

### 完整流程
**用户**：“撰写一篇关于 AI 代理安全最佳实践的博客文章”
→ 依次执行所有 5 个阶段，并在每个阶段展示结果。

### 仅研究
**用户**：“研究 ‘openclaw tutorial’ 的相关内容”
→ 仅执行第 1 阶段，输出研究概要。

### 根据研究结果起草
**用户**：“我已经研究过这个主题，这是我的笔记：[笔记]。请撰写初稿。”
→ 跳过第 1 阶段，直接执行第 2-5 阶段。

### 优化现有内容
**用户**：“优化这篇博客文章的 SEO 合规性” + [附上文章文件]
→ 跳过第 1-2 阶段，执行第 3-5 阶段。

### 推广现有内容
**用户**：“为这篇文章生成社交媒体推广内容：[文章链接或内容]”
→ 跳过第 1-4 阶段，仅执行第 5 阶段。

## 内容计划

如果用户需要内容计划或日程安排：

1. 使用 `web_search` 功能研究其所在领域的热门话题。
2. 与用户现有的内容进行对比，避免重复。
3. 为下个月推荐 4-8 个主题，包括：
   - 目标关键词
   - 预计的搜索量（参考网络搜索数据）
   - 竞争激烈程度
   - 推荐的发布日期
4. 将计划内容保存在内存中以便跟踪。

## 品牌信息存储

首次使用时，询问用户关于其品牌风格的要求，并将其保存在内存中：

- **语气**：专业、随意、技术性、友好还是权威？
- **目标受众**：开发者、营销人员、企业主还是普通公众？
- **格式偏好**：喜欢短段落？多使用标题？需要包含代码示例？
- **需要避免的内容**：专业术语的使用、不应提及的竞争对手、应避免的主题
- **现有内容链接**：用于内部链接和避免重复。

一旦这些信息被保存，后续的所有内容创作都会根据这些偏好进行，无需再次询问。

## 特殊情况处理

- **未提供关键词**：如果用户仅要求“撰写关于 AI 代理的文章”，则帮助其通过研究选择一个具体的关键词。
- **竞争非常激烈的关键词**：提醒用户并建议使用长尾关键词。
- **已有类似内容**：如果用户的网站已经存在相关文章，建议更新现有内容而非重新创建。
- **多语言需求**：支持用户要求的任何语言，同时调整 SEO 策略以适应相应语言的搜索习惯。
- **简短内容**：对于社交媒体帖子或电子邮件正文（非博客文章），跳过第 1 和第 3 阶段，直接进行写作和格式化。
