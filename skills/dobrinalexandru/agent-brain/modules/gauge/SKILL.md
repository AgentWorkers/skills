# 内存监控工具 📊

**状态：** ✅ 正在运行 | **模块：** gauge | **所属部分：** Agent Brain（代理大脑）

该工具用于监控内部状态，包括代理的信心水平、不确定性以及可用资源。

## 功能概述

- **监控**：代理自身的运行状态
- **跟踪**：代理的信心水平
- **警报**：当资源不足时发出警告

## 状态分类

### 信心水平

| 状态 | 值 | 信号 | 应对措施 |
|-------|-------|---------|--------|
| 高 | 0.8及以上 | “我很确定” | 继续执行 |
| 中等 | 0.5–0.8 | “我有一定信心” | 需要进一步确认 |
| 低 | 0.3–0.5 | “不太确定” | 需要询问更多信息 |
| 无 | <0.3 | “我不知道” | 请求相关信息 |

### 不确定性

- 我不知道哪些信息？
- 可能存在哪些问题？
- 哪些内容需要验证？

### 资源情况

- 上下文使用率（%）
- 所花费的时间
- 令牌数量
- 子代理的运行开销

## 触发条件

- **信心水平低**：请求进一步澄清
  → “我对某件事不太确定”

- **资源不足**：提示“上下文信息不足”，建议设置检查点，并优先处理关键任务

- **不确定性高**：标记该问题以供审核
  → “需要验证某些内容”

## 自我监控机制

- **响应前**：
  - 这个回答是否解决了问题？
  - 是否存在矛盾之处？
  - 代理的信心水平如何？

- **响应后**：
  - 回答是否准确？
  - 回应方式是否恰当？

## 使用方式

```
"How confident are you?"
"What's your uncertainty?"
"Are we running low on context?"
```

## 集成方式

作为 Agent Brain 的组成部分，该工具会在以下流程中首先被执行：
1. **Gauge**：评估代理的运行准备状态
2. 接着执行 Archive（归档）、Signal（发送信号）、Ritual（执行特定流程）和 Vibe（调整系统氛围）等操作。
