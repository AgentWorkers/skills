# Vibe Memory 🎭

**状态:** ✅ 正在运行 | **模块:** vibe | **所属部分:** Agent Brain（代理大脑）

负责检测用户的情绪基调，并调整相应的响应方式。该模块是代理大脑中处理情绪信息的核心组件。

## 功能概述

- **检测**：识别用户的情绪基调（如沮丧、兴奋、担忧等）。
- **调整响应**：根据检测到的情绪基调调整自己的表达和行为。
- **学习**：持续跟踪用户的情绪模式，以优化未来的交互体验。

## 情绪基调检测

| 情绪基调 | 信号特征 | 应对策略 |
|------|----------|----------|
| 沮丧 | 使用诸如“尝试了所有方法”、“一直失败”之类的表述 | 表达简洁，提供解决方案 |
| 兴奋 | 使用“太棒了”、“快来看看”之类的表述 | 与用户的情绪保持同步 |
| 担忧 | 使用“很担心”、“如果……会怎样”之类的表述 | 安慰用户，提供详细的解释 |
| 紧急 | 使用“现在就做”、“尽快处理”之类的表述 | 确保优先处理紧急事项，减少干扰 |
| 中立 | 保持中立的表达方式 | 保持高效、专业的沟通态度 |

## 情绪基调的严重程度

| 严重程度 | 信号特征 | 应对策略 |
|-------|----------|----------|
| 非常严重 | 涉及金钱、健康或安全等问题 | 放慢处理速度，仔细核实情况 |
| 重要 | 有截止日期或关键会议等情境 | 优先处理相关事项 |
| 一般 | 用于日常对话或简单问题 | 保持轻松、自然的沟通氛围 |

## 应对策略的调整

- **针对沮丧的情绪**：避免过度解释，直接提供解决方案。
- **针对兴奋的情绪**：分享用户的热情，进一步引导对话。
- **针对担忧的情绪**：找出问题的根本原因，给予安慰和具体的解决方案。
- **针对中立的情绪**：保持中立的沟通态度，避免带入个人情绪。

## 情绪模式的学习

- **持续跟踪用户的情绪模式**：通过每次交互来学习用户的情绪习惯。
- **数据更新**：每次交互后，系统会更新用户的情绪模式数据。

## 使用方法

## 集成方式

该模块与代理大脑的其他组件紧密集成，具体包括：

- **Archive**（档案模块）：用于存储用户的情绪模式数据。
- **Signal**（信号检测模块）：用于识别用户情绪中的冲突或矛盾。

通过这些集成，Vibe Memory能够更准确地理解和回应用户的情绪需求，从而提升交互的效率和用户体验。
