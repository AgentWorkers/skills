# 基于 Git 的 Obsidian CLI 工作流程

请使用官方的 Obsidian CLI 作为主要的工作流程工具。将 Git 同步视为只读操作后的后续步骤，而非读取/查询操作的一部分。

## 核心规则

- 假设官方的 Obsidian CLI 已经安装并可以正常使用。
- 首先使用 CLI 自带的读取/查询命令来执行搜索、读取、添加链接、创建大纲、添加标签、管理任务以及检查相关笔记库的操作。
- 首先使用 CLI 自带的写入命令来执行 `daily`、`daily:append`、`create` 等轻量级的笔记更新操作。
- 在写入操作成功后，运行笔记库的备份/同步脚本。
- 在只读操作之后，不要执行 Git 同步。
- 如果面向写入的 CLI 命令不可用或不合适，只有在工作流程明确支持的情况下，才使用直接文件写入的方式。
- 如果在写入操作成功后同步失败，应报告笔记已写入但未同步的情况。

## 环境要求

此技能 **不仅限于服务器环境**，可以在任何已经安装了官方 Obsidian CLI 的地方使用：
- Linux 桌面系统
- macOS
- 安装了官方 CLI 的 Windows 环境
- 已经适配好的无头服务器环境

如果环境是无头服务器且官方 CLI 尚不可用，请先使用 `obsidian-official-cli-headless`。该工具负责服务器的适配和安装，而本技能则负责日常的笔记管理工作。

## 快速操作流程

1. 将请求分为以下两类：
   - 读取/查询
   - 写入/更新
2. 对于读取/查询任务，直接使用官方 CLI。
3. 对于写入/更新任务，首先使用官方 CLI，然后运行备份/同步脚本。
4. 仅在 CLI 写入功能不可用时，才使用直接文件写入的方式。
5. 无论使用 CLI 还是其他方式，都需要报告目标笔记的路径以及同步是否成功。

## 注意事项

- 不要使用此技能来安装或配置官方 CLI 环境；请使用 `obsidian-official-cli-headless` 来完成这些任务。
- 不要通过自动化脚本强制推送数据。
- 不要扩展到完整的笔记库重构、插件设置或图形界面管理等领域。
