## 何时使用此技能

此技能适用于 CloudBase 项目中的 **微信小程序（Mini Program）认证**。在以下情况下可以使用它：

- 实现微信小程序与 CloudBase 的登录功能；
- 在云函数（cloud functions）中获取用户身份信息（如 `openid`、`unionid`）；
- 了解微信认证如何与 CloudBase 集成；
- 开发需要用户身份验证的小程序功能。

**主要优势：** 使用 CloudBase 进行微信小程序认证的过程是 **无缝且自动的**，无需复杂的 OAuth 流程。当小程序调用云函数时，用户的 `openid` 会自动被微信注入并验证。

**不适用场景：**

- 基于 Web 的微信登录（请使用 **auth-web** 技能）；
- 使用 Node.js SDK 进行服务器端认证（请使用 **auth-nodejs** 技能）；
- 非微信认证方式（请使用相应的认证技能）。

   - 微信小程序认证是 **原生的且自动完成的**；
   - 在大多数情况下，无需显式调用登录 API；
   - 用户身份信息会自动在云函数中可用；
   - CloudBase 负责所有的认证验证工作。

3. **从文档中选择合适的场景**：
   - 对于在云函数中获取基本用户身份信息，使用 **场景 2**；

   - 对于从小程序调用云函数并获取用户身份信息，使用 **场景 3**；

   - 在云函数中使用 `wx-server-sdk`；

5. **如果对某个 API 不确定**：

## 核心概念

### 微信小程序认证与 CloudBase 的集成方式

1. **自动认证**：
   - 当小程序用户调用云函数时，微信会自动将用户身份信息传递给云函数；
   - 无需复杂的 OAuth 流程或令牌管理；
   - CloudBase 负责验证用户身份的真实性。

2. **用户标识符**：
   - `OPENID`：该用户在当前小程序中的唯一标识符；
   - `APPID`：小程序的 App ID；
   - `UNIONID`：（可选）同一微信开放平台账户下所有小程序的唯一标识符；
     - 仅当小程序关联到微信开放平台账户时才可用；
     - 有助于识别跨多个小程序或官方账户的同一用户。

3. **安全性**：
   - `openid`、`appid` 和 `unionid` 经过微信验证，具有较高的可信度；
   - 微信已经完成了身份验证过程；
   - 开发者可以直接使用这些标识符，无需额外验证。

4. **无需显式登录**：
   - 用户在使用小程序时会被自动认证；
   - 在大多数情况下，无需调用登录 API；
   - 用户身份信息会立即在云函数中可用。

## 总结

使用 CloudBase 进行微信小程序认证非常简单且安全：

- **无需显式登录**，认证过程是自动完成的；
- 用户身份信息经过验证（`OPENID`、`APPID` 和 `UNIONID` 都具有较高的可信度）；
- **易于使用**，只需在云函数中调用 `cloud.getWXContext()` 即可；
- **默认情况下非常安全**，因为微信负责所有的认证验证工作。

**关键要点：**
- 在小程序中通过 `wx.cloud.init()` 初始化 CloudBase；
- 在云函数中使用 `cloud.getWXContext()` 获取用户身份信息；
- 使用 `OPENID` 进行用户识别和授权；
- 适当处理 `UNIONID` 的可用性；
- 无需显式调用登录 API，认证过程是自动完成的。

对于更复杂的认证场景或与其他系统的集成，可以考虑结合使用 CloudBase 的自定义登录功能与微信认证。
