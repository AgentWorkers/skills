# 非营利组织资源基础管理（RBM）逻辑模型

## 目标

制定一套决策支持型、符合捐赠者要求的RBM方案，将各项活动与可衡量的成果和影响联系起来，并确保有可行的监测机制。

## 执行流程

1. 在起草方案之前收集必要的背景信息：
   - 问题陈述及干预措施概要
   - 目标人群（包括优先考虑的群体）
   - 地理范围及实施范围
   - 时间框架（例如：12个月内的成果、3-5年内的影响）
   - 捐赠者或报告要求（如USAID、UN、EU的标准模板或自定义模板）
   - 基线数据及数据收集的可行性

2. 如果关键信息缺失，可提出最多五个具有指导性的问题进行澄清。
   - 如果细节仍不明确，需基于现有信息做出明确的假设。

3. 构建成果链，明确各层级之间的关系：
   - 输入（Inputs）
   - 活动（Activities）
   - 输出（Outputs）
   - 成果（Outcomes，短期/中期）
   - 影响（Impact，长期）

4. 确保因果逻辑可验证：
   - 不应将活动直接定义为成果；
   - 不应将交付物直接视为影响；
   - 使用具有时间限制且可观察的成果描述。

5. 为每个成果定义具体的指标（每个成果3-5个指标）：
   - 指标名称及定义/计算公式
   - 基线和目标值
   - 数据分类（如性别/年龄/地区，如适用）
   - 数据来源及收集频率

6. 仅在有证据支持的情况下，将成果和影响与可持续发展目标（SDGs）相对应起来。

7. 制定实用的监测计划：
   - 基线和终线数据收集的时间安排
   - 定期监测的频率
   - 后续数据收集的时间点（例如3个月、6个月、12个月）
   - 数据质量检查及负责人员

8. 以规定的结构呈现最终成果。

## 所需的输出格式

1. 变革理论（如果/那么逻辑 + 假设）
2. 执行摘要（2-3句话）
3. 逻辑模型（输入 → 活动 → 输出 → 成果 → 影响）
4. 成果指标（按成果分类）
5. 与可持续发展目标的对应关系（目标及具体指标）
6. 监测与数据收集计划（方法、频率、负责人）
7. 假设、风险及应对措施

## 质量标准

- 优先使用可量化的、有时间限制的目标，而非定性的描述。
- 明确区分“输出”（Activities）和“成果”（Outcomes）。
- 除非用户另有要求，否则将影响（Impact）定义为长期性的。
- 明确指出所有不确定性和假设。
- 对缺失的基线数据进行标注，并提出数据收集方案。
