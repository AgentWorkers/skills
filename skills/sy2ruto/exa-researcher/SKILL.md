# Exa Researcher

## 概述

Exa Researcher 提供了强大的研究功能，这些功能通过 Exa AI 的 MCP 工具实现，包括基础和高级的网络搜索、公司和人员信息查询、代码上下文提取、网页爬取以及深度研究工作流程。

## 核心功能

### 1. 基础和高级网络搜索

**基础网络搜索** (`web_search_exa`)  
- 快速搜索通用信息  
- 适用于：事实性查询、新闻、快速获取信息  

**高级网络搜索** (`web_search_advanced_exa`)  
- 具有更复杂的搜索功能，支持过滤和排名  
- 适用于：详细研究、查找特定类型的内容、按新鲜度、地区等筛选结果  

### 2. 公司研究 (`company_research_exa`)  
- 收集关于公司的全面信息  
- 适用于：竞争分析、合作伙伴调研、尽职调查  
- 使用场景：公司概况、财务数据、最新新闻、领导层信息、产品/服务  

### 3. 人员搜索 (`people_search_exa`)  
- 通过姓名或查询条件查找人员信息  
- 适用于：背景调查、建立联系网络、寻找专家  
- 使用场景：专业背景、专长领域、近期活动  

### 4. 代码上下文提取 (`get_code_context_exa`)  
- 提取代码上下文和技术信息  
- 适用于：技术研究、理解代码库、查找代码示例  
- 使用场景：理解 API、寻找实现模式、技术文档研究  

### 5. 网页爬取 (`crawling_exa`)  
- 系统性地爬取网页以收集全面信息  
- 适用于：深入网站分析、内容聚合、特定领域的研究  
- 使用场景：网站分析、内容整理、特定领域的数据收集  

### 6. 深度研究 (`deep_researcher_start`, `deep_researcher_check`)  
- 长期运行的研究工作流程，支持进度跟踪  
- 适用于：多源信息的综合研究、趋势分析、信息整合  
- 使用场景：研究论文、市场报告、可行性研究、竞争情报  

## 使用指南

### 如何选择合适的工具  

| 任务 | 推荐工具 | 原因 |  
|------|------------------|--------|  
| 快速获取信息/新闻 | `web_search_exa` | 快速、简单的查询方式 |  
| 带有过滤条件的详细研究 | `web_search_advanced_exa` | 更强的控制能力，更好的搜索结果 |  
| 公司分析 | `company_research_exa` | 结构化的数据来源 |  
| 查找人员/专家 | `people_search_exa` | 专注于人员的搜索工具 |  
| 技术/代码研究 | `get_code_context_exa` | 专注于代码和 API 的工具 |  
| 研究网站/领域 | `crawling_exa` | 系统性的网站探索工具 |  
| 长期研究项目 | `deep_researcher_start` → `deep_researcher_check` | 多源信息、耗时较长的研究流程 |  

### 研究工作流程  

1. **确定研究目标** – 需要哪些信息？  
2. **选择合适的工具** – 根据任务选择相应的工具  
3. **执行搜索/研究** – 调用 Exa AI 工具  
4. **分析结果** – 整合研究结果  
5. **必要时进行迭代** – 根据初步结果优化查询或使用其他工具  

## 高级使用案例  

### 竞争分析  
- 使用 `company_research_exa` 查询多个竞争对手的信息  
- 结合 `web_search_advanced_exa` 查找最新新闻  
- 以结构化格式记录研究结果  

### 市场研究  
- 使用 `web_search_advanced_exa` 查找市场趋势  
- 通过 `crawling_exa` 获取行业报告和白皮书  
- 将结果整合成市场概述  

### 技术文档研究  
- 使用 `get_code_context_exa` 查阅 API 文档  
- 查找代码示例和实现模式  
- 提取技术规格  

### 背景调查（人员/公司）  
- 使用 `people_search_exa` 查找符合要求的专业人士  
- 使用 `company_research_exa` 获取公司组织结构信息  
- 结合使用以获取全面的信息  

### 长期研究项目  
- 使用 `deep_researcher_start` 启动研究流程  
- 通过 `deep_researcher_check` 监控研究进度  
- 适用于复杂的多源研究需求  

## 获取最佳结果的技巧  

- **明确查询条件** – 更精确的查询能获得更好的结果  
- **组合使用工具** – 多种工具可以提供更全面的信息覆盖  
- **合理使用过滤条件** – 尤其是在高级搜索中（如时间范围、地区等）  
- **迭代优化** – 根据初步结果调整查询  
- **保存重要结果** – 将关键发现保存下来  
- **考虑搜索量** – 复杂的搜索可能需要更多时间
