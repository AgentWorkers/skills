# Git 提交规范

## 提交信息格式

使用语义化的提交格式：`<类型>(<范围>): <主题>`

### 提交类型

- `feat`：添加新功能
- `fix`：修复错误
- `docs`：更新文档
- `style`：格式化代码（无功能变更）
- `refactor`：重构代码
- `test`：添加/修改测试用例
- `chore`：更新构建任务

## 工作流程规则

1. **语言**：Git 提交信息必须使用英文。

2. **强制推送**：在执行 `git push --force` 之前，务必先确认。

3. **签名**：所有提交都必须包含 `--signoff` 标志。

4. **合并变更**：将多个变更合并为一个有意义的提交。

5. **推送确认**：在完成本地提交后，务必询问用户是否要将更改推送到远程仓库。

6. **显示 URL**：成功推送至远程仓库后，显示仓库的访问地址。
