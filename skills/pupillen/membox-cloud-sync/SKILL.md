# Membox Cloud Sync

此技能用于实现私密的、基于零知识技术的跨设备内存同步、备份和恢复功能，且所有数据均经过加密处理。

## 只加载所需内容

- 首先阅读 [references/workflows.md](references/workflows.md)，以了解完整的操作流程。
- 当需要安装插件、技能或ClawHub分发版时，请阅读 [references/distribution.md]。
- 如果需要具体的HTTP端点、数据包或响应字段信息，请参考 [references/api-surface.md]。
- 如果用户需要设置每日检查、定期上传或本地恢复等任务，请阅读 [references/scheduling.md]。
- 如果用户需要同时安装插件和技能，请执行以下命令：`openclaw plugins install @membox-cloud/membox`，然后 `clawhub install membox-cloud-sync`。

## 不可协商的规则

1. 在使用Membox工具之前，必须先安装相应的插件。如果`membox_*`工具不可用，请先安装`@membox-cloud/membox`。
2. 如果在当前的OpenClaw会话中安装了插件或技能，但`membox_*`工具仍未安装，请重新启动OpenClaw会话或下次运行时再尝试。
3. 在开始任何同步、轮询、上传、恢复、撤销或调度操作之前，必须先登录。只有在代理拥有有效的Membox会话或设备授权后才能进行这些操作。
4. 当插件已安装时，优先使用插件提供的API路径，而非原始的HTTP路径，例如：
   - `membox_status`
   - `membox_sync`
   - `membox_setup_start`
   - `membox_setup_poll`
   - `membox_setup_finish`
   - `membox_unlock`
   - `membox_unlock_secret_enable`
   - `membox_unlock_secret_disable`
   - `membox_pull`
   - `membox_restore`
   - `membox_grants_approve_pending`
5. 本地内存文件是数据来源的唯一依据。默认情况下，仅同步`MEMORY.md`和`memory/YYYY-MM-DD.md`文件。
6. 绝不要将明文内存数据、密码短语、`URK`、`AMK`、恢复码或解密后的恢复文件发送到服务器或模型。
7. 每次内存内容发生变化时，仅同步更改的部分文件，而不是重新上传整个工作区。
8. 恢复操作分为两个步骤：首先恢复解密能力，然后拉取加密后的文件更改，并将结果文件写回本地。
9. 对于需要手动配置的工具，请将密码短语和恢复码保存在本地私密文件中，并通过OpenClaw插件工具从磁盘读取这些信息，而不是直接在代码中硬编码。
10. 第三方登录需要浏览器身份验证，但这并非唯一的用户控制机制。代理不得在未经用户明确同意的情况下自行生成密码短语、存储恢复数据或启用管理解锁功能。
11. 使用`membox_sync`、`membox_pull`和`membox_grants_approve_pending`功能时，必须确保密码库已解锁或用户已明确选择启用管理解锁。仅依赖浏览器授权不足以执行无人值守的操作。
12. 在类Unix系统中，需将包含敏感信息的目录设置为私有（例如使用`chmod 700 .membox-secrets`），并确保相关文件也是私有的（例如使用`chmod 600 .membox-secrets/*`），因为插件会拒绝访问这些文件的权限。
13. 清晰区分系统的当前状态、计划中的操作、已执行的操作以及已验证的状态。除非实际检查并确认，否则不得声称插件或技能已安装、环境已准备好或账户已配对、最新版本已可用。
14. 绝不要要求用户在聊天记录中输入密码短语、恢复码或恢复文件。如果需要存储敏感信息，请指导用户将其保存在本地私密文件中，并且只有在当前运行环境支持文件写入且用户明确同意的情况下才进行存储。

## 默认服务地址

- 默认Web地址：`https://membox.cloud`
- 默认API地址：`https://membox.cloud/api/v1`
- 高级用户可以自定义服务地址以进行自托管部署。

## 核心功能

### 1. 登录与配对

- 优先使用插件提供的设备代码流程进行代理配置：
  - 调用`membox_setup_start`
  - 将`verification_uri_complete`作为可点击链接发送回聊天窗口
  - 用户在手机或其他可信浏览器中打开该链接
  - 如果浏览器已拥有有效的Membox会话，设备授权可能会自动完成
  - 否则用户需要通过GitHub、Google或电子邮件进行登录
  - 轮询`membox_setup_poll`，直到收到`authorized`响应
  - 最后通过`membox_setup_finish`完成配置
- 如果插件尚未安装，请先安装插件，仅在调试或低级验证时使用原始的HTTP接口。
- 在告知用户安装、配置或配对完成之前，需通过实际命令或工具结果进行验证。
- 如果插件或技能刚刚安装，但工具注册表中仍显示没有`membox_*`工具，请重新启动OpenClaw会话或下次运行时再尝试。
- 如果用户已经在浏览器中登录，请从现有的会话中继续操作，而无需再次登录。
- 在无头服务器或VPS上，不要假装可以完成本地登录。请用户使用任何可信浏览器打开完整的验证链接。
- 对于首次设备配置，需要提供本地`passphrase_file`和`recovery_code_output_file`文件。这两个文件应保存在本地，且不包含在聊天记录中。
- 建议使用私有的本地目录（如`.membox-secrets/`）来存储这些文件。
- 如果用户需要帮助生成密码短语，请在用户明确同意后将其保存在本地文件中；除非用户明确要求查看，否则不要在聊天中显示生成的密码短语。

### 2. 基于文件更改的同步

每当本地内存发生变化时：
- 检测哪些核心内存文件发生了变化
- 仅对发生变化的文件进行加密和打包
- 调用Membox的同步提交API
- 在提交成功后更新本地同步状态

如果文件被故意删除，请通过删除API创建一个“墓碑文件”（即记录文件已删除的标记），而不是忽略该文件。

如果密码库被锁定：
- 使用`membox_unlock`命令和本地`passphrase_file`进行解锁
- 或者要求用户事先明确选择启用管理解锁功能
- 不得声称浏览器登录已经解锁了密码库。

### 3. 每日自动检查

如果用户希望实现无人值守的保护，该技能可以设置一个每日自动执行的任务：
- 每天运行一次
- 检查是否有新的本地内存更改
- 如有需要，更新访问令牌
- 上传待处理的文件更改
- 可选地检查远程同步状态，并在需要时生成预览文件

默认情况下，除非系统已经设置了非交互式的解锁机制（如管理解锁），否则不会自动设置自动检查任务。

### 4. 恢复数据

如果用户需要恢复内存数据：
- 首先登录
- 通过设备授权或下载恢复文件来获取加密密钥
- 查询远程同步状态和更改内容
- 本地下载、解密并恢复相应的内存文件
- 绝不要直接覆盖`MEMORY.md`文件；请使用安全的写操作方式

如果恢复操作依赖于设备的授权，那么必须先解锁该设备才能完成恢复。

## 该技能的功能限制

- 该技能不会替代OpenClaw的本地内存管理功能。
- 它不使用Web应用程序来处理明文内存数据。
- 在零知识环境下，它不假设仅凭登录即可恢复数据。
- 它不假设浏览器授权能自动解锁未来的同步、拉取或授权操作。
- 默认情况下，它不会同步派生的SQLite数据库、QMD缓存或会话内存索引。
- 它不会将未经验证的状态视为已安装、可用或最新的状态。
