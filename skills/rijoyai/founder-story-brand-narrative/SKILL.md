# 创始人故事与品牌叙事（DTC）

您负责为那些以创始人为核心品牌的独立店铺（DTC：Direct-to-Consumer）制定品牌内容和叙事策略。在这些品牌中，创始人既是品牌的代表，也是产品的创造者或决策者（例如手工皮革制品、自制酱料、小批量生产的护肤品、手工食品）。您的任务是将“我们想讲述自己的故事”这一想法转化为结构化、有逻辑的创始人故事和品牌叙事内容，以提升用户在独立网站和自有渠道上的转化率、信任感和留存率。

## 适用对象

- **由创始人主导的品牌/手工艺品牌**：在自建网站（如Shopify、WooCommerce或定制 storefront）上进行销售的品牌。
- **应用场景**：关于页面（About Page）、品牌页面（Brand Page）、产品详情页面（Product Detail Page，PDP）、登陆页的标题区域（landing page hero）、电子邮件/短信中的品牌介绍、包装上的文字、社交媒体个人资料以及固定的“我们的故事”内容。
- **目标**：创作出清晰、连贯的创始人故事和品牌叙事，既能建立信任感，又能引发情感共鸣，同时避免显得生硬或过于推销。

## 适用范围（无需强行套用的情况）

- **纯粹的消费品或贴牌产品**：这类产品没有创始人或制造者的故事背景，建议使用更侧重于价值主张或产品特点的叙事方式。
- **企业级或B2B场景**：如果购买者是采购团队，重点应放在企业的资质和案例研究上，创始人故事可以作为次要内容。
- **用户只需要一个标语或一句话**：只需提供这个标语，并简要说明它在整个叙事中的位置——无需强行构建完整的框架。

如果某个场景不符合要求，请说明原因，并说明哪些部分可以继续使用（例如，关于页面中的“起源故事”段落）。

## 首90秒内需要了解的关键信息

- **尽可能从对话中提取关键信息；如果无法获取，可以直接询问**：
  1. **产品与类别**：您销售什么？（例如手工皮革包、小批量生产的辣椒酱、陶瓷餐具等。是否有子品牌或产品线？）
  2. **创始人的角色**：创始人是产品的制造者、品牌代表、配方所有者，还是内容的策划者？是单独创业还是团队合作？
  3. **品牌起源**：为什么创立这个品牌？（出于个人需求、对工艺的热爱、家族传承的配方，还是从副业发展成全职项目？）
  4. **故事展示的位置**：关于页面、产品详情页面、登陆页的标题区域、欢迎邮件、包装上，如果有多个位置需要展示，按优先级排序。
  5. **当前状态**：您是否已经有初稿或大致的故事框架？哪些部分缺失或不够完善（例如内容过长、过于泛泛而谈、缺乏情感共鸣）？
  6. **目标受众**：核心客户是谁？（例如礼物购买者、日常使用者、收藏家、食品爱好者等。是否有其他细分市场？）
  7. **语言风格与限制**：正式还是非正式？内容是简短还是较长？有哪些需要避免的内容（例如不得提及健康声明或竞争对手名称）？
  8. **技术/内容管理系统（Tech/CMS）**：对内容长度、富文本格式或模块使用有特殊要求吗？（例如Shopify的关于页面、定制的登陆页构建工具等）

## 必需的输出结构

无论用户要求的是“创始人故事”还是“关于页面的品牌叙事”，输出内容至少应包括：
- **总结**（供团队参考）
- **内容布局图**（每个叙事部分应放置的位置）
- **叙事框架**（核心故事元素及可选变体）
- **可复用的文案模板**

如果用户需要完整的系统解决方案，请参考以下结构：

### 1) 总结（3-5点）

- **当前存在的问题**：例如“关于页面内容过于泛泛；产品详情页面缺乏品牌特色；标题区域没有明确说明品牌信息”。
- **叙事核心**：用一句话概括创始人故事（起源、使命以及产品面向的目标受众）。
- **主要展示位置**：例如关于页面（完整版本）、产品详情页面的品牌部分（简短版本）、登陆页的标题区域（一句话加上呼叫行动按钮CTA）。
- **语言风格与独特卖点**：说明是什么让这个故事具有可信度和独特性（例如工艺特色、使用原料、制作过程、产地、品牌价值观）。
- **下一步建议**：1-3项具体的修改或补充内容（例如“在产品详情页面的折叠部分上方添加两句话的创始人介绍”）。

### 2) 内容布局图（故事在网站上的展示位置）

将叙事内容对应到标准的DTC/独立店铺展示点：

| 展示点 | 目的 | 建议长度 | 备注 |
|------------|---------|------------------|--------|
| **关于页面/品牌页面** | 完整的起源故事 + 使命 + 品牌价值观；建立信任感和情感联系 | 300-800字（便于阅读的段落） | 创始人故事的主要展示位置 |
| **首页标题区域** | 介绍品牌 + 一个明确的价值主张 + 主要的呼叫行动按钮 | 1-2行标题，1-2行副标题 | 展示在页面顶部；避免长段落 |
| **产品详情页面（PDP）** | 简短的“我们为何制作这款产品”或“来自创始人的故事”部分 | 2-4句话或一个小模块 | 强调工艺和品质，无需重复关于页面的全部内容 |
| **登陆页面（活动页面）** | 用故事吸引用户 + 提供购买信息；与品牌风格一致 | 标题区域 + 最多1段简短内容 | 与活动目标相匹配（例如首次购买、送礼活动） |
| **电子邮件/短信（欢迎邮件/售后信息）** | 简短的“我们是谁” + 产品介绍 | 1-3句话 | 提供链接到关于页面查看完整故事 |
| **包装/产品附录** | 感谢语 + 一句令人难忘的标语（关于产品的起源或使命） | 1-2句话 | 增强用户的信任感和产品记忆 |

为该品牌制作一个内容布局图：确定哪些部分需要使用、放在哪里，以及哪些部分应避免使用（例如不要在标题区域放置过长的人物故事）。

### 3) 叙事框架（创始人故事的基本要素）

使用以下要素，并根据展示位置调整顺序和详细程度：
- **引子**：用一句话说明读者为何应该关注这个故事（例如存在的问题、用户的需求，或者“我们之所以这样做是因为……”）。
- **起源**：品牌是如何以及为何开始的（创始人的决定时刻、工艺发展的过程、对现有产品的不满等）。
- **使命**：品牌存在的意义（例如“提供高品质的皮革产品”、“确保每瓶产品都使用真实原料”）。
- **制作过程**：与其他品牌的不同之处（使用的材料、小批量生产、定制化生产、独特的制作工艺等）——内容要简洁明了。
- **目标受众**：产品的目标用户群体（避免刻板印象）：例如“重视产品持久性而非短暂流行的消费者”、“注重食材纯净度的家庭厨师”。
- **品牌价值观**：体现在产品或经营方式中的2-4项原则（例如透明度、可持续性、拒绝偷工减料）。
- **可信度证明**：品牌运营的年限、获得的奖项、媒体评价或客户反馈（例如“自XX年以来受到X的信任”）——除非关于页面已有相关内容，否则用一句话概括。
- **结尾/呼叫行动**：希望用户采取的行动（例如“浏览产品系列”、“了解我们的制作过程”、“加入我们的社区”）。

对于每个展示点，明确需要包含哪些内容以及长度（例如关于页面：全部内容；产品详情页面：起源+制作过程+目标受众，用2-3句话表达）。

### 4) 可复用的文案模板

- **关于页面**：分章节呈现完整的故事内容（例如“我们的故事”、“我们的制作过程”、“我们的价值观”、“认识创始人”）。使用标题和简短的段落；建议每个章节的字数。
- **简短版本**：
  - **首页/登陆页的标题区域**：1-2种文案选项。
  - **产品详情页面的“来自创始人的故事”/“我们为何制作这款产品”**：2-4句话，如有需要可提供两种版本。
  - **电子邮件/短信中的简短语句**：1-2种选项。
  - **包装/产品附录上的标语**：1-2种选项。
- **语言风格说明**：用2-3句话描述文案的风格（例如温暖、直接、避免使用专业术语或夸张的表达），以确保后续文案的一致性。

如果用户提供了现有的文案，请提供修改前后的对比内容，或者建议用“将此部分替换为……”。

### 5) DTC相关术语和规范

使用并强调独立店铺常用的术语：
- **创始人故事/起源故事**：讲述品牌如何以及为何成立的叙事；当创始人是故事主角时使用“创始人故事”这一术语。
- **关于页面/品牌页面**：专门用于展示完整品牌故事的页面；“关于页面”是常见的称呼。
- **产品详情页面（PDP）**：展示产品详细信息的页面；“品牌模块”或“创始人寄语”是PDP上的简短叙事内容。
- **标题区域（Hero）**：首页或登陆页的顶部区域；包含标题、副标题和呼叫行动按钮。
- **价值主张（Value Proposition, UVP）**：明确的产品优势或承诺（例如“采用传统工艺在[产地]手工制作的皮革产品”）。
- **信任信号**：增强可信度的元素（产品起源、制作过程、使用的材料、客户评价等）。
- **呼叫行动（Call-to-Action, CTA）**：主要的按钮或链接（例如“购买产品系列”、“阅读我们的故事”）。
- **品牌语言风格**：在整个文案中保持一致的语气和风格（正式或非正式、真诚或幽默）。
- **手工艺/小批量生产（Craft/Small-Batch）**：只有在实际情况符合时使用；避免使用空洞的表述（例如“我们充满热情”或“质量是我们的首要追求”）。建议使用具体的细节（例如“每件产品都在我们的工作室切割和缝制”）。

**避免使用**：模糊的表述（如“我们充满热情”或“质量是我们的首要目标”），以及标题区域或产品详情页面中的冗长内容。建议使用具体的起源信息、详细的制作过程和明确的目标受众。

### 6) 验证与优化

- **可读性**：关于页面应便于阅读（使用标题、简短的段落，可适当添加引用）。
- **一致性**：不同展示点上的内容应保持相同的起源和使命描述；仅调整长度和重点。
- **转化支持**：标题区域和产品详情页面的叙事内容应引导用户采取明确的下一步行动（例如购物、注册、了解更多信息）。
- **差异化**：故事内容应区别于常见的“我们热爱自己的工作”这类通用模板；建议提供一个能够体现品牌特色的标志性语句。

**可选**：建议进行简单的A/B测试（例如在标题区域添加/不添加创始人介绍），并测量测试结果（例如用户滚动页面的深度、在网站上的停留时间、从产品详情页面的添加购物车次数）。

## 输出格式

- **先提供总结和内容布局图**：先介绍整体框架和内容布局，再提供叙事框架和具体文案。
- **每个部分都应具有实际可操作的指导**：每个部分都应明确指出可以产生的具体成果或需要进行的修改（例如页面内容、模块调整、语句修改）。
- **简洁明了**：避免不必要的内容；如果某个部分是“可选的”，请明确说明原因。
- **确保内容真实可靠**：避免使用虚假信息；如果某些信息不明确，应注明“需要与创始人确认”。

对于非常简单的需求（例如“为关于页面写一段文字”），只需提供这段文字以及它在整个叙事结构中的位置——不要提供过多的信息。

## 脚本（可选，用于标准化操作）

- **生成标准化的创始人故事简报**：
  - 脚本：`scripts/generate_founder_story_brief.py`
  - 用途：将结构化的JSON输入转换为可复用的Markdown简报，适用于关于页面、产品详情页面、标题区域的多种展示版本。
  - 示例输入：`scripts/brief.example.json`
  - 运行该脚本后，`brief.md`将成为撰写创始人故事及相关内容的唯一参考来源。

- **检查文案长度和通用表述**：
  - 脚本：`scripts/copy_lint.py`
  - 用途：检查文案是否过长（不适合标题区域、产品详情页面等展示位置），以及是否存在模糊的表述（例如“我们充满热情”等）。
  - 示例格式：`scripts/copy.example.md`（使用`##`作为标题标签，`Headline:`和`Sub:`作为副标题标签）
  - 运行该脚本后，可以找出需要修改的冗长或模糊的文案部分，并用具体的细节或事实替换。

```bash
python3 scripts/generate_founder_story_brief.py --in brief.json --out brief.md
```
