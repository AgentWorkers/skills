# OpenClaw 上下文升级

## 首先检查当前状态

1. 运行 `session_status` 并记录以下信息：
   - 当前使用的模型
   - 当前的上下文（Context）状态
   - 会话是否已经显示为 `1.0m`（即上下文预算为 1,000,000）

2. 如果用户只是询问“为什么还没有达到 1,000,000”，请先进行诊断，不要盲目更改配置。

## 在编辑配置之前仔细检查配置路径

1. 使用 `gateway` 的 `config.schemalookup` 功能检查 `agents.defaults.contextTokens` 的值。
2. 如果模型选择很重要，还需要检查 `agentsdefaults.model` 的值。
3. 使用 `gateway` 的 `config.patch` 功能来修改配置。

## 升级流程

1. 确保目标会话使用的是高上下文预算的模型。在此流程中，建议使用模型别名 `GPT-5.4`。
2. 在验证过程中，避免依赖系统的回退机制。
3. 如有需要，可以使用 `session_status` 并通过会话模型覆盖功能将当前会话设置为 `GPT-5.4`。

2. 增加默认的上下文预算：
   - 将 `agents.defaults.contextTokens` 的值设置为 1,000,000。
   - 如果用户的目标是未来的会话，可以修改默认配置。
   - 如果用户只是希望进行诊断，只需解释这一变更，无需实际应用它。

3. 重新启动 OpenClaw：
   - `gateway config.patch` 会自动重启 OpenClaw。
   - 确保提供清晰的说明，以便用户在重启后收到有用的提示信息。

4. 强制开始一个新的聊天会话：
   - 告诉用户使用 `/new` 或 `/reset` 命令。
   - 旧的聊天记录可能会保留之前的会话预算，导致验证结果不准确。

5. 在新会话开始后进行验证：
   - 再次运行 `session_status`。
   - 成功的状态应该显示为 `Context: .../1.0m` 或类似的信息。

## 常见故障模式的诊断

### 如果仍然显示 `.../272k`（即上下文预算为 272,000）

按以下顺序进行检查：

1. 是否没有使用 `/new` 或 `/reset` 命令重新启动聊天会话。
2. 会话是否仍然使用错误的模型或回退模型。
3. 提供者/模型组合是否返回了较低的上下文预算。
4. 配置更新是否正确地修改了 `agents.defaults.contextTokens` 的值。

## 如实告知上下文预算的限制

- 如果提供者实际返回的上下文预算较低，不要承诺用户可以使用 1,000,000。
- OpenClaw 可以请求更大的预算，但实际上不能超过模型或提供者的限制。
- 如果验证结果仍然是 `272k`，请如实告知用户，并解释原因（可能是会话重复使用、模型不匹配或提供者的限制所致）。

## 默认的回复模板

在回答用户问题时，请按照以下顺序进行：

1. 说明当前观察到的上下文预算限制。
2. 提供具体的修复步骤：
   - 将模型设置为 `GPT-5.4`
   - 将 `agents.defaults.contextTokens` 的值设置为 1,000,000
   - 重新启动会话
   - 使用 `/new` 或 `/reset` 命令
   - 再次使用 `session_status` 进行验证
3. 最后提醒用户：最终的上下文预算仍受提供者的限制影响。
