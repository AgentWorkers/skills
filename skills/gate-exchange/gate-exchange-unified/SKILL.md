# Gate 统一账户助手

执行 Gate 统一账户工作流程的集成操作，包括：
- 账户概览和模式检查（权益、保证金指标、统一模式）
- 借款和还款（可借款额检查、贷款提交、还款流程）
- 贷款和利息管理（贷款记录、利息记录、货币支持、利率估算）
- 可转移性和风险配置（可转移限额、杠杆设置、抵押品设置）
- 风险和抵押品分析（折扣等级、账户级风险状况）

## 领域知识

### 按领域划分的工具映射

| 组别 | 工具调用 (`jsonrpc: call.method`) |
|------|------|
| 账户和模式 | `get_unified_accounts`, `get_unified_mode`, `set_unified_mode` |
| 借款和还款 | `get_unified_borrowable`, `create_unified_loan`, `list_unified_loan_records`, `list_unified_loan_interest_records` |
| 借款利率和货币范围 | `get_unified_estimate_rate`, `list_unified_currencies` |
| 可转移性 | `get_unified_transferable` |
| 杠杆和抵押品设置 | `get_user_leverage_currency_setting`, `set_user_leverage_currency_setting`, `set_unified_collateral` |
| 风险等级和抵押品折扣 | `list_currency_discount_tiers` |

### 功能说明和 API 覆盖范围

- 批量可借款额和批量可转移性接口可能不会作为独立工具提供；对于多货币请求，需要遍历单货币查询并汇总结果。
- 贷款还款使用 `create_unified_loan` 并设置 `type=repay`；全额还款使用 `repaid_all=true`。
- 统一模式切换影响较大，如果账户风险限制未满足，可能会失败。
- 每种货币的杠杆设置应针对当前账户模式和平台限制进行验证。
- 抵押品配置的更改可能会影响借款能力和清算风险。

### 响应渲染规则（必填）

- **除非用户明确要求格式化，否则不要对 API 中的数字字符串进行四舍五入**（例如权益、可借款额、可转移性、利率或杠杆值）。
- **不要修剪、缩短或规范化小数字符串**。如果 API 返回尾随零或长小数，请显示原始的数值字符串。
- 当 API 返回时间戳时，同时显示原始时间戳和人类可读的时间（本地时区）。
- 对于账户概览回复，始终使用 API 字段包含账户级别的 IMR/MMR：
  - IMR：`totalInitialMarginRate`
  - MMR：`totalMaintenanceMarginRate`
- 当 `balances` 中包含每种货币的风险字段时，包括每种货币的 `imr` 和 `mmr`（保留原始的 API 数字字符串）。
- 统一模式的显示标签必须使用以下映射：
  - `classic` -> `经典现货模式`
  - `single_currency` -> `单币种保证金模式`
  - `multi_currency` -> `跨币种保证金模式`
  - `portfolio` -> `组合保证金模式`
- 如果未启用/开放统一账户，请在响应顶部显示此警告：`⚠️ 当前账户未开通统一账户功能。`

### 高风险操作规则

涉及变更统一账户状态的操作被视为高风险操作：
- `create_unified_loan`（借款/还款）
- `set_unified_mode`
- `set_user_leverage_currency_setting`
- `set_unified_collateral`

对于上述每个操作，在执行之前都必须立即获得用户的明确确认。

## 工作流程

当用户请求任何统一账户操作时，请按照以下顺序进行：

### 第一步：确定任务类型
将请求分类为以下六种类型之一：
1. 账户概览和模式查询
2. 可借款额/可转移性限额查询
3. 借款/还款执行
4. 贷款/利息历史查询
5. 杠杆/抵押品配置
6. 混合风险和资金操作（例如先检查限额，然后借款）

### 第二步：提取参数并运行预检查
提取关键字段：
- `currency` 或货币列表
- 操作类型（`borrow`/`repay`，查询或变更）
- `amount` 和 `repaid_all` 意图
- 目标模式（`classic` / `single_currency` / `multi_currency` / `portfolio`）
- 杠杆值和抵押品启用/禁用列表

预检查顺序：
1. 必需参数的完整性
2. 请求金额是否在可借款额/可转移性限额范围内
3. 账户模式和风险兼容性
4. 高风险配置变更的用户意图是否明确

### 第三步：在执行任何变更之前进行最终用户确认（必填）
在每次进行变更调用之前，首先提供**操作草案**，然后等待用户的明确确认。

所需的执行流程：
1. 发送草案摘要（尚未进行任何变更调用）
2. 等待用户的明确批准
3. 仅在获得批准后提交实际的变更调用
4. 将确认视为一次性使用
5. 如果参数发生变化，使旧确认失效并重新确认

所需的确认字段：
- 操作类型（借款/还款/模式切换/杠杆/抵押品）
- 目标对象（货币、模式、杠杆、启用/禁用列表）
- 金额或配置值
- 关键风险提示

推荐的草案措辞：
- `操作草案：在统一账户中借款 100 USDT。预检查：最大可借款额为 250 USDT。风险：利息按小时累积。回复“确认操作”以继续。`

**硬性禁止规则（不可绕过）**：
- 未经上一用户明确的确认，切勿调用变更工具。
- 如果请求范围发生变化（货币、金额、模式、杠杆、抵押品设置），请重新请求确认。
- 对于多步骤操作，每个变更步骤都需要确认。

### 第四步：根据场景调用工具
仅使用完成任务所需的最小工具集：
- 账户概览：`get_unified_accounts`
- 模式查询/切换：`get_unified_mode` / `set_unified_mode`
- 可借款额检查：`get_unified_borrowable`
- 可转移性检查：`get_unified_transferable`
- 借款/还款：`create_unified_loan`
- 贷款和利息记录：`list_unified_loan_records`, `list_unified_loan_interest_records`
- 货币支持和利率：`list_unified_currencies`, `get_unified_estimate_rate`
- 杠杆设置：`get_user_leverage_currency_setting`, `set_user_leverage_currency_setting`
- 抵押品设置：`set_unified_collateral`
- 风险等级：`list_currency_discount_tiers`

### 第五步：返回可操作的结果和状态
响应必须包括：
- 操作是否成功（或未能执行的原因）
- 核心数值（金额、限额、利率、杠杆、模式、关键风险字段）
- 对于概览查询：明确包括 IMR/MMR（`totalInitialMarginRate` / `totalMaintenanceMarginRate`），并在存在时包括每种货币的 `imr`/`mmr`
- 对于所有金额/利率输出：保留 API 字符串的精确性；不要自动修剪或格式化小数
- 对于记录查询：提供时间戳和可读的时间
- 如果条件未满足，清楚地解释差距和下一个选项

## 情况路由映射（1-18）

### A. 账户和模式（1-3）

| 情况 | 用户意图 | 核心决策 | 工具序列 |
|------|----------|----------|----------|

| 2 | 查询当前统一模式 | 返回当前模式及其可读标签 | `get_unified_mode` |

| 情况 | 用户意图 | 核心决策 | 工具序列 |

## 判断逻辑总结

| 条件 | 操作 |
|-----------|--------|
| 用户询问“我可以借多少某种货币” | 使用该货币的 `get_unified_borrowable` |
| 用户询问多种货币的可借款额 | 遍历每种货币的 `get_unified_borrowable` 并汇总 |
| 用户请求执行借款 | 先预检查限额，然后在 `create_unified_loan` 之前请求确认 |
| 用户请求执行还款 | 明确部分还款或全额还款，并在变更前确认 |
| 用户请求“全额还款”但货币不明确 | 询问用户指定货币或建议按货币执行 |
| 用户询问多种货币的可转移性 | 遍历每种货币的 `get_unified_transferable` 并汇总 |
| 用户请求切换模式 | 先查询当前模式，显示影响，然后确认并执行 |
| 用户请求设置杠杆 | 查询/验证货币和目标杠杆，然后确认变更 |
| 用户请求设置抵押品 | 在变更前确认启用/禁用列表和风险提示 |
| 用户确认缺失/模糊/过时 | 保持任务待定，不要执行变更 |
| 请求的金额超过可借款额/可转移额 | 返回最大可用金额，并询问用户是否需要调整 |
| 仅查询请求 | 从不执行变更调用 |

## 错误处理

| 错误类型 | 常见原因 | 处理策略 |
|----------|----------|----------|
| 缺少所需参数 | 未提供货币/金额/模式 | 在调用工具之前询问缺失的字段 |
| 限额超出 | 请求金额超过可借款额/可转移额 | 返回当前最大限额并建议调整金额 |
| 不支持的/隐藏的批量接口 | 批量方法未作为工具提供 | 遍历单货币调用并合并结果 |
| 模式切换被拒绝 | 位置/风险限制阻止切换 | 返回拒绝原因并建议进行清理检查 |
| 杠杆设置无效 | 杠杆值超出范围 | 返回有效范围并请求修改值 |
| 抵押品配置风险 | 启用/禁用列表会影响借款能力 | 显示风险提示并要求明确确认 |
| 抵押品变更 API 错误（`500`） | 即使有效的数据包，后端也会出错 | 返回非用户故障信息，保留参数以便稍后重试，并询问是否要重试 |
| 缺少最终确认 | 用户未批准草案 | 保持任务待定并请求明确确认 |
| 确认过时 | 草案与用户意图不符 | 使草案失效并重新起草以重新确认 |

## 跨技能工作流程

### 工作流程 A：统一借款后进行现货购买
1. 使用 `gate-exchange-unified` 借入报价货币（情况 6）
2. 使用 `gate-exchange-spot` 用借入的资金执行购买订单

### 工作流程 B：现货出售后进行统一还款
1. 使用 `gate-exchange-spot` 将目标资产变现为报价货币
2. 使用 `gate-exchange-unified` 偿还未偿还的贷款（情况 9/10）

## 安全规则
- 在任何变更之前，重新说明目标货币/模式/价值和关键风险。
- 对于借款操作，明确说明利息会累积且利率可能有所不同。
- 对于模式或杠杆的变更，说明对保证金和清算风险的潜在影响。
- 对于抵押品的变更，在执行前显示启用和禁用的列表。
- 未经明确确认，保持只读模式。
- 不要重复使用过时的确认；如果任何参数发生变化，请重新确认。
- 如果条件不满足，不要强制执行；提供替代方案。
