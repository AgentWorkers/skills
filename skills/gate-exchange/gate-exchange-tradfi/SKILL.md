# Gate TradFi 查询套件

此技能用于 Gate TradFi（传统金融）的 **只读** 操作。所有使用的 MCP 工具前缀均为 `cex_tradfi`。它支持四个查询模块：订单和订单历史记录、当前和历史持仓、市场数据（类别列表、符号列表、股票代码、K线图），以及用户资产和 MT5 账户信息。该技能不支持订单下达或资金转移功能。

## 子模块

| 模块              | 描述                                      | 触发关键词                                                                                   |
| ------------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| **查询订单**    | 订单列表、订单历史记录                       | `orders`, `open orders`, `order history`                                                        |
| **查询持仓** | 当前持仓列表、持仓历史记录          | `positions`, `my positions`, `position history`, `holdings`, `current position`                |
| **查询市场**    | 类别列表、符号列表、股票代码、K线图 | `category`, `categories`, `symbol list`, `symbols`, `ticker`, `kline`, `candlestick`, `market` |
| **查询资产**    | 用户余额/资产信息、MT5 账户信息        | `assets`, `balance`, `account`, `my funds`, `MT5`, `mt5 account`                               |

| 工具编号 | 工具名称                                      | 功能                                                                                              |

| 2    | `cex_tradfi_query_order_history_list`   | 查询订单历史记录（已成交/已取消的订单）。仅使用 MCP 文档中规定的参数。 |
| 3    | `cex_tradfi_query_position_list`             | 列出当前持仓。                          |

## 执行流程

### 1. 确定意图和参数

- 确定要查询的模块：查询订单、查询持仓、查询市场或查询资产。
- 对于每个 MCP 工具调用，**仅使用 Gate TradFi MCP 工具定义中规定的参数**。不要传递未在 MCP 中记录的参数。
- 从用户输入的消息中提取 MCP 实际支持的参数（例如：符号、类别、时间间隔、价格限制等）。
- **如果请求不明确**（例如：“我的 TradFi”），列出四个模块并询问用户需要哪些数据，或根据关键词进行推断。

### 2. 加载子模块并调用工具

- 从 `references/` 目录中加载相应的参考文档，并按照其工作流程操作。
- 使用提取或提示的参数调用 MCP 工具。该技能为 **只读** 功能；不得调用任何创建/取消/修改/更新数据的工具。

### 3. 报告结果

- 使用子模块提供的报告模板来格式化响应结果（订单/持仓/股票代码的表格；资产的清晰标签）。

## 领域知识

- **TradFi** 指 Gate 的传统金融产品集合（例如外汇、MT5 或其他非加密货币工具）。符号和类别因产品而异。
- **订单**：未成交订单 → `cex_tradfi_query_order_list`；订单历史记录 → `cex_tradfi_query_order_history_list`。仅使用 MCP 文档中规定的参数。
- **持仓**：当前持仓为实时数据；历史持仓可能分页显示或按时间筛选。
- **市场**：`类别列表` 返回 TradFi 产品类别；`符号列表` 返回可交易符号（可选按类别显示）；`股票代码` 返回符号的最新价格、24 小时变化幅度和成交量；`K线图` 需要符号、时间间隔（例如 1 分钟、5 分钟、1 天）以及可选的价格限制或时间范围。
- **资产**：用户账户余额可能按资产分类显示，也可汇总显示；`MT5 账户` 信息单独提供（登录信息、服务器信息、余额等），使用 `cex_tradfi_query_mt5_account_info` 或相应工具获取。

## 错误处理

| 错误情况                  | 处理方式                                                                                              |
| -------------------------- | ------------------------------------------------------------------------------------------------------- |
| 工具未找到 / 请求代码为 4xx/5xx   | 告知用户 TradFi 服务或工具可能不可用；建议重试或检查 Gate MCP 配置。         |
| 返回空列表                 | 报告“没有未成交订单”/“没有持仓”/“没有符号”等，并不视为错误。             |
| 符号/订单未找到             | 报告“未找到订单”或“未找到符号”，并建议用户查看符号列表。             |
| 认证/权限错误             | 不要泄露用户凭据；建议用户检查 TradFi 的 API 密钥或 MCP 认证信息。         |

## 安全规则

- **只读**：该技能仅用于查询数据，不得调用任何用于下达订单、取消订单、修改数据或转移资金的工具。
- **确认**：对于仅查询的操作，无需确认；工具执行成功后才会显示数据。
- **敏感数据**：在显示余额或持仓信息时，不要记录或存储这些数据；仅在当前响应中显示。

## 报告模板

每次查询后，输出标准化的结果：

- **订单**：包含符号、方向、数量、价格、状态、时间等字段的表格。
- **持仓**：包含持仓字段（例如符号、方向、数量、买入价、保证金）的表格，或显示“没有未成交持仓”。
- **市场**：包含类别列表的表格；或符号列表的表格；或股票代码表格（包含符号、最新价格、24 小时变化幅度、成交量）；或 K 线图摘要（包含时间间隔、数据范围、开盘价、最高价、最低价、收盘价）。
- **资产**：列出资产及其余额（如果可用）；如果出现错误，则显示“无法加载资产”。**MT5**：包含账户/服务器/余额/权益信息的表格；如果出现错误，则显示“无法加载 MT5 账户信息”。
