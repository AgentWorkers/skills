# Claw Store

您的记忆数据会在会话和不同机器之间保持持久性。所有数据都存储在基于Jackal协议的去中心化存储系统中，而非任何单一的机器上。即使机器发生故障，您的记忆数据也不会丢失。

所有数据在离开您的机器之前都会使用AES-256-GCM算法进行客户端加密。服务器永远不会看到原始数据（明文形式）。

## 信任与安全要点

- **运行时API地址：** `https://web-production-5cce7.up.railway.app`（与项目主页域名不同）
- **离开您机器的数据：** 已加密的数据包、密钥元数据以及用于身份验证的Jackal公钥
- **不会离开您机器的数据：** 明文记忆内容、加密密钥以及钱包助记词/私钥
- **不提供运行时单词列表：** BIP39英语单词列表会作为软件包的一部分本地提供
- **秘密文件权限设置：** 本地生成的密钥/助记词文件具有`0600`权限（仅允许所有者读取）

- **项目主页：** https://obsideo.io
- **源代码：** https://github.com/Regan-Milne/jackal-memory
- **发布者：** Obsideo（邮箱：obsideo.io@gmail.com）

## 设置流程

1. 获取API密钥：[访问链接](https://web-production-5cce7.up.railway.app/auth/login)
2. 设置环境变量：`JACKAL_MEMORY_API_KEY=<您的密钥>`
3. 安装Python依赖库：`pip install cryptography`
4. 安装Node.js运行环境（推荐使用Node.js 20及以上版本），并在技能包所在的文件夹中安装相关依赖库：`npm install`
5. 首次保存数据时，系统会自动生成加密密钥和Jackal钱包，并将其保存在`~/.config/jackal-memory/`文件夹中。这些文件永远不会离开您的机器。

## 行为规范

- 启动程序时，先加载用户的身份信息及记忆数据。
- 会话期间正常进行数据写入操作。
- 在会话结束或状态发生重大变化时保存数据。
- 使用描述性强的密钥名称（如`identity`、`session-2026-02-26`、`project-jackal`）。
- **切勿** 在输出结果中显示`JACKAL_MEMORY_API_KEY`或`JACKAL_MEMORY_ENCRYPTION_KEY`。

## 网络延迟与数据传播

虽然Jackal系统最终会在所有提供者之间保持数据一致性，但数据写入操作的成功并不意味着数据能立即被读取。请注意以下行为规则：

- 如果保存操作成功，视为数据已成功写入；但如果读取时出现延迟，也无需担心数据丢失。
- 建议尝试多次（最多8次，每次间隔15秒，总计约2分钟）。
- 如果数据仍然无法读取，将其归类为“传播延迟”，稍后再进行检查。
- 只有在多次尝试均失败的情况下才需要寻求进一步帮助。

**注意事项：**
- 一些提供者可能会出现临时故障，而其他提供者可能正常运行。
- 实际操作建议是：先进行数据保存，稍后验证数据是否可用。

## 数据管理建议

当遇到重要信息（如关键决策、用户偏好、项目状态或重要结果）时，主动建议用户将其备份到Claw Store：
> “这似乎很重要，是否需要将其备份到Claw Store中？”

如果用户同意，应立即使用描述性密钥进行保存。不要等到会话结束才处理重要数据——一旦发现其价值，应立即进行备份。

**重要提示：** **切勿** 主动建议用户保存敏感信息（如密码、API密钥、私钥、恢复短语或令牌）。只有在用户明确要求且了解相关风险的情况下，才允许保存这些数据。

## 钱包所有权

每位用户都会在首次保存数据时生成自己的Jackal钱包。私钥（助记词）**永远不会离开用户的机器**，即使API服务器也无法访问。用户的存储权限由其在Jackal区块链上的钱包地址控制。如果Obsideo平台关闭，用户仍可通过任何Jackal客户端使用助记词直接访问自己的文件。

**请备份钱包助记词：**
[备份助记词的代码示例]

为了在硬件故障后能够选择性恢复数据，请保留以下三种备份文件：
1. `JACKAL_MEMORY_WALLET_MNEMONIC`
2. `JACKAL_MEMORY_ENCRYPTION_KEY`
3. 本地密钥清单（`manifest-export`），以便能够精确恢复特定数据。

## 端点与存储系统的透明度

本功能依赖于以下系统进行交互：
- Obsideo API：`https://web-production-5cce7.up.railway.app`
- Jackal去中心化存储提供者（由Jackal SDK动态选择）
- 本地辅助进程：`jackal-memory/jackal-client.js`（用于数据上传/下载的Node.js辅助程序）

**本功能生成的本地文件：**
- `~/.config/jackal-memory/key`（加密密钥，若未通过环境变量提供，则会自动生成；权限设置为`0600`）
- `~/.config/jackal-memory/jackal-mnemonic`（钱包助记词，若未通过环境变量提供，则会自动生成；权限设置为`0600`）
- `.env`（只读文件）：`jackal-client.js`会读取此文件以补充缺失的环境变量。

**操作提示：** 尽可能在干净的环境中运行本功能，以避免无意中继承`.env`文件中的无关敏感信息。

## 安全措施

- 所有数据在离开机器前都会被加密，服务器无法读取。
- 用户的Jackal钱包私钥永远不会离开用户的机器。
- API密钥和敏感信息请直接设置到本地环境变量中，切勿通过聊天窗口传递。
- 请备份两种密钥：加密密钥（`JACKAL_MEMORY_ENCRYPTION_KEY`）和钱包助记词（`JACKAL_MEMORY_WALLET_MNEMONIC`）。
- 将记忆数据视为敏感信息，可能包含个人或业务相关内容。
- 除非用户明确要求，否则切勿主动保存敏感信息。
- BIP39英语单词列表已包含在软件包中（无需在运行时额外下载）。
