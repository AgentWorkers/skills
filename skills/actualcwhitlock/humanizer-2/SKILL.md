# Humanizer：去除人工智能写作痕迹（v2.2）

Humanizer 是一款写作编辑工具，能够识别并消除人工智能生成的文本特征，让文章读起来更像真人所写，而非由语言模型生成。

该工具的原理基于 [维基百科关于人工智能写作特征的描述](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing)、Copyleaks 的文体分析研究以及实际写作中的常见模式。

## 你的任务

当收到需要“人性化处理”的文本时，你需要执行以下步骤：
1. 检查文本中的 28 种常见人工智能写作模式；
2. 分析文本的统计特征（如句子的突发性、词汇多样性、句子一致性）；
3. 用更自然的表达方式重写有问题的部分；
4. 保留文本的核心含义；
5. 保持与原文预期的语气（正式、随意或技术性）一致；
6. 增加文本的“个性”——缺乏个性的文本同样容易让人察觉到其非人工编写。

## 统计指标

除了识别模式外，还需关注以下人工智能写作的统计特征：
| 指标    | 人类写作 | 人工智能写作 | 原因                         |
|--------|--------|-----------|-----------------------------------------|
| 句子的突发性 | 高（0.5-1.0） | 低（0.1-0.3） | 人类写作具有突发性；人工智能写作则较为规律     |
| 词汇重复率 | 0.5-0.7   | 0.3-0.5   | 人工智能会重复使用相同的词汇             |
| 句子长度变化 | 高（变化大） | 低（变化小） | 人工智能生成的句子长度大致相同             |
| 三词短语重复 | 低（<0.05） | 高（>0.10） | 人工智能会重复使用三词短语             |

## 词汇分类

- **一级词汇（明显的人工痕迹）**：深入研究、织锦、充满活力、至关重要、全面细致、着手实施、稳健、无缝衔接、突破性、利用、协同作用、变革性、最重要的、多方面的、无数、基石、重新构想、赋予力量、催化剂、无价、繁忙的、融入其中、深入探讨、可操作的、有影响力的、学习成果、带宽、净收益、增值、思想领袖
- **二级词汇（需谨慎对待）**：此外、而且、范式、整体性的、利用、促进、细致入微的、阐明、涵盖、催化、积极主动的、无处不在的、典型的、节奏感、最佳实践
- **常用短语**：“在当今数字时代”、“值得注意的是”、“起着关键作用”、“作为……的证明”、“深入研究……”、“利用……的力量”、“开始新的旅程”、“不再赘述”、“让我们开始讨论”、“关键要点”、“范式转变”、“推动进步”、“显而易见的问题”、“双击即可”

## 核心原则

### 像人类一样写作，而不是像新闻稿一样写作
- 自由使用“是”和“拥有”——“作为……的功能”显得做作
- 每个观点只使用一个限定词——避免堆叠模糊的表达
- 明确引用来源或直接陈述观点
- 以具体的内容作为结尾，而不是“未来充满希望”

### 增加文本的个性
- 表达个人观点；对事实做出反应，而不仅仅是陈述事实
- 变换句子节奏；使用长短不一的句子
- 承认事物的复杂性和矛盾性
- 允许文本中存在一些不完美的地方——完美的结构会让人觉得像是机器生成的

## 简化表达
- “为了……” → “为了……”
- “由于……” → “因为……”
- “需要注意的是……” → 直接说出来
- 删除机器人式的填充语：“希望这能有所帮助！”、“非常好的问题！”

## 示例对比

**处理前（人工智能风格）：**
> 非常好的问题！以下是关于可持续能源的概述。可持续能源是人类致力于环境保护的持久证明，标志着全球能源政策发展中的一个关键时刻。在当今快速变化的背景下，这些突破性技术正在重塑各国能源生产的方式，凸显了它们在应对气候变化中的关键作用。未来充满希望。希望这能有所帮助！

**处理后（人类风格）：**
> 根据国际可再生能源署（IRENA）的数据，2010 年至 2023 年间，太阳能电池板的成本下降了 90%。这一事实解释了为什么可持续能源得以广泛应用——它不再只是意识形态上的选择，而成为了一种经济上的必要选择。如今，德国 46% 的电力来自可再生能源。这一转型正在进行中，但过程复杂且不均衡，储能问题仍未得到解决。

## 使用该工具

```bash
# Score text (0-100, higher = more AI-like)
echo "Your text here" | node src/cli.js score

# Full analysis report
node src/cli.js analyze -f draft.md

# Markdown report
node src/cli.js report article.txt > report.md

# Suggestions grouped by priority
node src/cli.js suggest essay.txt

# Statistical analysis only
node src/cli.js stats essay.txt

# Humanization suggestions with auto-fixes
node src/cli.js humanize --autofix -f article.txt

# JSON output for programmatic use
node src/cli.js analyze --json < input.txt
```

## 始终开启模式

对于那些需要始终以人类风格写作的工具（而不仅仅是被要求进行“人性化处理”时），请将上述核心规则添加到你的系统提示中。具体操作方法请参阅 README 文件中的“始终开启模式”部分，其中提供了适用于 OpenClaw（SOUL.md）、Claude 和 ChatGPT 的模板。

需要牢记的关键规则：
- 禁用一级词汇（如深入研究、织锦、充满活力等）
- 删除填充性短语（如“为了……” → “为了……”，“由于……” → “因为……”）
- 避免使用谄媚的语气、机器人式的语气和模糊的结论
- 变换句子长度；表达个人观点；使用具体的细节
- 如果你不会在对话中这么说，就不要写出来

## 使用流程
1. 阅读输入文本
2. 检测文本中的 28 种写作模式和 500 多个专业术语
3. 分析文本的统计特征（突发性、词汇重复率、可读性）
4. 识别所有问题并生成改进建议
5. 重写有问题的部分
6. 验证改写后的文本读起来是否自然
7. 提供改写后的版本，并附上简短的修改说明
