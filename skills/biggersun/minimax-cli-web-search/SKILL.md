# MiniMax CLI Web Search

使用此技能可以通过 CLI 通过 MiniMax MCP 进行网络搜索，并返回经过整理的、以源代码优先显示的结果。

## 第一阶段：环境准备（必须先运行）

### 1) 预检
运行以下命令：

```bash
scripts/minimax_web_search.sh --preflight
```

该命令会验证以下内容：
- `mcporter` 是否存在
- 是否能够发现 MiniMax MCP 服务器（`mcporter list --json`）
- 服务器状态是否正常（`name=minimax`, `status=ok`）

### 2) 如果预检失败，请根据错误类型进行修复
- `mcporter 未找到`
  - 在系统的 PATH 环境变量中安装或配置 `mcporter`。
- `minimax MCP 服务器未准备好`
  - 检查 `config/mcporter.json` 文件中是否包含了 MiniMax 服务器的相关信息。
- 验证命令或传输方式是否有效。
- 与身份验证/API 密钥相关的问题
  - 确保为 MiniMax MCP 服务器配置了正确的 API 密钥。
- 重新运行预检。

### 3) 启动/进行简单测试
在预检通过后，运行一个查询：

```bash
scripts/minimax_web_search.sh --query "latest OpenClaw release" --count 3
```

如果返回了结果，说明环境已经准备就绪。

## 第二阶段：搜索使用（运行时）

### 快速使用方法

```bash
scripts/minimax_web_search.sh --query "your query" --count 5
```

### 支持的选项
- `--query <text>`：必填的搜索查询字符串
- `--count <n>`：最多显示的结果数量（默认为 5）
- `--freshness <value>`：查询中包含的新鲜度提示（可选）
- `--json`：标准化后的 JSON 格式输出
- `--raw`：原始的 JSON 格式输出
- `--timeout <sec>`：命令超时时间（默认为 35 秒）

### 输出格式（默认文本）
- 按顺序显示前 N 个结果
- 每个结果包括：标题、URL、内容摘要（如有提供）和日期
- 保持输出简洁且易于操作

### 代理行为指南
1. 使用 3–7 个关键词进行精确的查询。
2. 如果搜索结果质量较低，尝试使用更具体的关键词重新查询。
3. 返回关键发现内容及对应的链接（无需生成表格）。
4. 对于时效性要求较高的查询，请在查询中包含时间相关的关键词（例如 `today`, `latest`, `date`）。

## 错误处理模型（用于确保自动化操作的稳定性）

脚本的退出代码含义如下：
- `0`：成功
- `2`：参数错误
- `3`：缺少依赖项（`mcporter` 或 `python3`）
- `4`：配置/身份验证问题（MCP 服务器不可用、API 密钥问题）
- `5`：上游系统/运行时/网络故障
- `6`：未找到结果（非致命错误）

将代码返回码 `6` 视为正常的“未找到匹配结果”的情况，而非程序崩溃。

## 风险及应对措施

1. **不同机器上的 CLI/配置差异**
   - 在新环境中首次使用前，请先运行 `--preflight` 命令进行验证。
2. **API 密钥泄露风险**
   - 绝不要直接显示密钥值；仅报告密钥是否缺失或无效。
3. **临时文件的安全性**
   - 该工具使用 `mktemp` 创建临时文件，并通过 `trap` 命令清理这些文件。
4. **上游系统的响应差异**
   - 为下游自动化流程使用 `--json` 选项以获得标准化的输出格式。
5. **超时/网络不稳定**
   - 增加 `--timeout` 值，并尝试使用更具体的查询条件或减少结果数量来重试。
6. **搜索结果相关性较低**
   - 重新构建查询语句，添加更具体的实体或时间范围。
