# SOHO Pay — 信用层支付

该技能通过使用 `spendWithAuthorization` EIP-712 流程，经由 SOHO Pay 的 `Creditor` 智能合约来协调支付操作。

**该技能仅能通过手动调用执行**，严禁由模型自动触发。每次调用都需要用户的明确确认。

---

## 架构 — 三方分离

**SoHo 仅作为信用层服务使用**。它不负责保管、持有、生成或控制任何签名密钥或密钥片段，也不生成 EIP-712 签名。

所有 EIP-712 签名均来自用户或代理操作者（例如 Turnkey/Privy/Fireblocks 等 MPC 提供商，或用户自己的钱包）所拥有的独立 **钱包签名器**。

该技能负责协调以下各方的协作：
- **钱包签名器** — 负责签名操作（由用户/操作者控制）
- **SoHo** — 负责信用审核及即时资金调配（仅限于信用层）
- **区块链** — 负责交易结算

## 工作流程

1. **配置验证**：启动时使用 Zod 工具检查所有环境变量；未知的链地址将被拒绝。
2. **解析输入**：解析金额和明确的商家地址（禁止根据名称生成地址）。
3. **信用审核**：SoHo 信用层会验证借款人的注册信息、活跃状态及信用额度。
4. **生成 EIP-712 签名**：由钱包签名器（用户控制）生成签名；SoHo 本身不参与签名过程。
5. **提交交易**：调用 `Creditor` 合约上的 `spendWithAuthorization` 函数。
6. **返回结果**：返回交易哈希值及区块编号。

## 安全模型

### 密钥保管规则

| 实体            | 角色                                      | 是否保管密钥？       |
| --------------------------- | -------------------------------------- | ------------------------- |
| **钱包签名器**     | 生成 EIP-712 签名                   | **是** — 由用户/操作者控制           |
| **SoHo**        | 负责信用审核及资金调配                   | **否** — 仅作为信用层服务使用           |
| **本技能**       | 仅负责各方的协调工作                   | **否** — 仅传递输入数据至签名器           |

SoHo 绝不保管、持有、生成或控制任何签名密钥或密钥片段。

### 钱包签名器提供商

| 提供商            | 使用场景                          | 风险等级        |
| ------------------------ | -------------------------------------- | --------------------------- |
| `WALLET_SIGNER_REMOTE`   | 生产环境/主网环境                     | 低风险 — 密钥保存在用户的 MPC/HSM 中（如 Turnkey、Privy、Fireblocks 等） |
| `LOCAL_PRIVATE_KEY`    | 仅限开发/测试网环境                    | 高风险 — 密钥以原始形式保存在环境中       |

- **默认及推荐方案**：使用 `WALLET_SIGNER_REMOTE`。该技能会将 EIP-712 相关数据发送至用户的钱包签名服务，绝不接触私钥。
- **仅限开发环境的备用方案**：`LOCAL_PRIVATE_KEY` 仅适用于测试网环境；在主网上使用此方案需设置 `DEV_ALLOW_LOCAL_KEY=YES`，且强烈不建议使用。

### 主网安全机制

在 Base Mainnet（链 ID 为 8453）上，除非设置了 `SOHO_MAINNET_CONFIRM=YES`，否则交易将被拒绝。这可防止开发过程中意外进行主网交易。

### 收件人地址规则

该技能**严禁**根据商家名称生成收件人地址。`merchantAddress` 参数必须是一个明确、有效的、已进行校验的 EVM 地址。任何非地址格式的输入都会被拒绝并引发错误。

### 调用规则

- **调用方式**：必须手动触发；禁止自动执行。
- **确认要求**：执行前必须获得用户确认。
- 如果环境变量 `SOHO_AUTONOMOUS` 被设置为 `true`，系统会拒绝执行。

## 威胁模型与应对措施

| 威胁            | 应对措施                                      |
| ---------------------- | ---------------------------------------- |
| **签名器被攻破**     | 默认使用 `WALLET_SIGNER_REMOTE`（MPC/HSM）；密钥始终保存在用户的签名服务中；本地密钥仅限测试网使用。 |
| **重放攻击**     | 每笔交易使用随机生成的 32 字节随机数；设置 `validAfter`/`expiry` 有效期（10 分钟）；在 `Creditor` 合约中进行链上验证。 |
| **错误的商家地址**     | 禁止根据名称生成地址；仅接受已进行校验的 EVM 地址；签名前会验证地址格式。 |
| **意外进入主网**     | 在链 ID 为 8453 的情况下，必须设置 `SOHO_MAINNET_CONFIRM=YES`。         |
| **自动调用**       | 在元数据中明确指定调用方式为手动；执行前必须获得用户确认。         |
| **SoHo 被误用作签名器**     | 架构设计确保 SoHo 仅用于信用层功能；技能不会将密钥传递给 SoHo API。     |
| **未声明的环境变量**     | 所有环境变量均需在 `skill.json` 中明确声明，并指定数据类型和敏感级别。 |
| **钱包签名器服务被中间人攻击** | 使用 HTTPS 协议连接 `WALLET_SIGNER_SERVICE_URL`；通过 `SIGNER_SERVICE_AUTH_TOKEN` 进行令牌认证。 |

## 环境变量

### 所有模式均需设置的变量

| 变量            | 示例值                                      | 说明                                      |
| --------------------------- | ------------------------------------------------------ | --------------------------- |
| `RPC_URL`        | `https://sepolia.base.org`                       | JSON-RPC 服务端点                         |
| `CHAIN_ID`        | `84532`                                      | Sepolia 链 ID（84532）或 Mainnet 链 ID（8453）                |
| `SIGNER_PROVIDER`     | `WALLET_SIGNER_REMOTE`                   | 所使用的钱包签名器类型                         |

### `WALLET_SIGNER_REMOTE` 需要设置的变量

| 变量            | 说明                                      | 是否涉及敏感信息    |
| --------------------------- | ------------------------------------------------------ | --------------------------- |
| `WALLET_SIGNER_SERVICE_URL` | 用户/操作者钱包签名服务的基础 URL                         | 是                         |
| `SIGNER_SERVICE_AUTH_TOKEN` | 用于钱包签名服务的令牌认证                         | 是                         |

### SoHo 信用层相关变量

| 变量            | 说明                                      | 是否涉及敏感信息    |
| --------------------------- | ------------------------------------------------------ | --------------------------- |
| `SOHO_API_URL`     | SoHo 信用层 API 的地址                             | 是                         |

### `LOCAL_PRIVATE_KEY`（仅限开发/测试网环境）需设置的变量

| 变量            | 说明                                      | 是否涉及敏感信息    | --------------------------- |
| `SOHO_DEV_PRIVATE_KEY` | 用户/操作者的原始十六进制私钥（非 SoHo 所管理的密钥）         | 是                         |

### 条件性变量

| 变量            | 设置条件                                      | --------------------------- | --------------------------- |
| `SOHO_MAINNET-confirm` | 当 `CHAIN_ID` 为 8453 时必须设置为 `YES`                   |                             |
| `DEV_ALLOW_LOCAL_KEY`    | 设置为 `YES` 以允许在非测试网环境中使用本地密钥（存在风险）       |
