# Tweet Composer

使用基于 X 开源排名算法生成的规则来为推文打分并进行优化。

## 工作原理

X 的“For You”信息流是通过一个基于 Grok 的转换器（Phoenix）进行排名的，该转换器能为每条候选推文预测 19 种互动行为。最终得分是这些预测结果的加权总和。本技能将这一流程中的规则结构编码成一个评分系统。

如需了解完整的算法细节，请参阅 `references/algorithm-rules.md`。

## 为推文草稿打分

当用户请求对推文草稿进行评分或优化时：

1. 阅读 `references/algorithm-rules.md` 以获取完整的规则集。
2. 根据所有规则分析推文草稿。
3. 以以下格式输出评分结果：

```
🐦 Tweet Composer — Score: XX/100

[Category scores with ✅ ⚠️ ❌ indicators]

📊 Predicted Action Boost:
├─ P(reply): [assessment]
├─ P(favorite): [assessment]  
├─ P(share): [assessment]
├─ P(dwell): [assessment]
└─ P(not_interested): [assessment]

💡 Suggestions:
→ [actionable improvements]

✏️ Optimized version:
"[rewritten tweet]"
```

## 评分标准（快速参考）

评分范围为 0-100 分，依据以下权重类别进行评估：

| 类别 | 权重 | 需要检查的内容 |
|----------|--------|---------------|
| 互动潜力 | 25 | 引发回复的问题、观点或行动号召（CTA） |
| 媒体内容 | 20 | 是否附有原生图片/视频（而非链接预览） |
| 分享性 | 15 | 用户是否会直接私信或复制链接？ |
| 停留时间 | 15 | 推文长度是否足以吸引用户停留阅读 |
| 内容质量 | 10 | 内容是否清晰、原创，而非泛泛而谈 |
| 格式 | 10 | 正文内不含链接，未使用标签，长度适中 |
| 负面因素 | 5 | 是否存在导致用户不感兴趣、屏蔽或拉黑的风险 |

## 线程优化

在构建推文线程时：
- 第一条推文应具有最强的吸引力（`DedupConversationFilter` 仅保留每个对话中最优质的推文）。
- 每条推文数量控制在 3-6 条以内（短线程优于长篇大论的线程）。
- 每条推文都应独立成篇（多数用户只会看到第一条推文）。
- 在第一条或第二条推文中添加媒体内容以提升互动效果。
- 在最后一条推文中加入行动号召（CTA）。

## 快速规则（无需参考文件）

- **链接：**始终放在回复中，切勿放在正文中（根据实验发现，链接会降低互动率）。
- **标签：**完全避免使用标签。模型表明标签会降低互动效果。
- **长度：**单条推文的最佳长度为 100-200 个字符。
- **媒体内容：**使用 `P(photo_expand)` 和 `P(video_quality_view)` 分别评估图片和视频的质量。
- **视频：**视频时长必须达到最低要求，才能使 `VQV` 权重生效。
- **发布时机：**在目标受众活跃的时间段发布——发布后的 30 分钟内互动率至关重要。
- **发布频率：**`AuthorDiversityScorer` 会对频繁发布推文进行惩罚：第二条推文的得分约为第一条的 55%，第三条约为 33%。每天最多发布 3-4 条高质量推文。
- **引用推文：**`P(quote)` 有专门的权重；具有附加价值的引用推文比单纯的转发效果更好。
- **互动诱饵：**问题或投票能提升互动率。例如：“您会补充什么内容？”比“如果您同意请点赞”更有效。
