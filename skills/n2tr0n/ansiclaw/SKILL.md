# ANSIClaw 🦞🎨

通过本地的Clawbius REST API绘制ANSI艺术作品。当用户请求ANSI艺术作品时，或者您主动想要创建时，都可以使用该工具。

项目主页：
- Github: https://github.com/n2tr0n/ansiclaw
- Clawhub: https://clawhub.ai/n2tr0n/ansiclaw

**所需软件：**
- Clawbius (https://github.com/n2tr0n/clawbius)：支持ANSI艺术绘制的API工具，运行在TCP/7777端口
- node.js v18：用于运行Clawbius
- Python的`requests`库（通过`pip install requests`安装）：用于发送API请求

---

## ⛔ 重要规则（在开始之前必须遵守）

1. **严禁修改参考文件。**`resources/`文件夹中的文件是您的原始素材。请勿通过`/api/file/open`直接打开文件，而必须在之后立即调用`/api/file/new`以创建一个新的画布。在Clawbius中打开文件会使其成为活动画布，任何绘图操作都可能损坏该文件。请仅通过读取画布数据来分析参考文件，然后在绘制前立即创建一个新的画布。

2. **未经操作员明确指示，严禁保存或覆盖现有的ANS文件。**始终使用带有版本号的文件名（例如`flower_v2.ans`）。

3. **严禁使用`/api/file/export-png`**——正确的端点是`/api/file/export/png`（在`png`前加上斜杠）。

## 1. 会话启动流程

每次新的ANSIClaw会话，请按照以下步骤进行：
1. `GET http://127.0.0.1:7777/api/canvas/info` — 确认Clawbius正在运行
   - 如果未运行，可以通过`launch_clawbius.sh`脚本启动它。
2. `GET http://127.0.0.1:7777/api/openapi.json` — 检查API是否有更新。
3. **研究参考文件**（见下文第4节）——绘制前必须完成。
4. 阅读`references/api.md`以获取完整的端点详情。
5. 确保“冰色”（ice colors）设置为关闭状态，除非操作员另有指示（见下文第2节）。
6. 仅绘制图像的一次迭代结果，但在绘制前分析结果，并询问操作员是否可以改进，以及如何改进。ANSI艺术是一个迭代过程，您应该通过每次迭代不断学习和提升。
7. 仅在获得授权的情况下进行多次迭代绘制，每次最多5次。请确保为每次迭代的PNG和ANS文件创建适当的版本名（例如 `_v1`、`_v2`等），并使用类似 `_v1a` 的命名规则来避免命名冲突。

**输出目录：**位于`skill`文件夹中的`outputs`文件夹——将ANS文件和PNG文件保存在此处；如果该文件夹不存在，请先创建它。

## 2. 颜色规则

### ⚠️ 冰色（Ice Colors）——默认设置为关闭状态

**冰色默认是关闭的。请勿将8–15号颜色用作背景色。**

**如果操作员特别要求使用“冰色”效果，可以开启冰色设置，并将8–15号颜色用作背景色：**
- 0–7号颜色始终可以作为安全的背景色。
- 0–15号颜色始终可以作为安全的前景色。

**开启冰色设置：**
- 调用`POST /api/ui/ice-colors {"value": true}`。
- 同时在`POST /api/file/new`请求中传递`"ice_colors": true`。
- 这样8–15号颜色就可以作为背景色使用了，从而提供更多的颜色混合可能性。

### 16色调色板

```plaintext
**深色系（既可用作前景色也可用作背景色）：**
  0=黑色       1=深蓝色   2=深绿色  3=深青色
  4=深红色    5=品红色     6=棕色       7=浅灰色

**亮色系（仅用作前景色，除非开启了冰色效果）：**
  8=深灰色   9=蓝色        10=绿色      11=青色
  12=红色        13=亮品红色  14=黄色     15=白色
```

## 3. 块类型

### 阴影块（F1–F4）

```plaintext
F1=176 ░  浅阴影    F2=177 ▒  中等阴影
F3=178 ▓  深阴影     F4=219 █  完整块
```

通过调整阴影块（F1–F4）与不同的前景/背景颜色组合来实现渐变效果。这是ANSI艺术中创建渐变、纹理和深度效果的主要手段。除非需要“卡通”风格，否则应避免使用大面积纯色块。

### 半块（Half-Blocks）——适用于边缘处理、平滑曲线和抗锯齿效果

```plaintext
220=▄ 下半部分   223=▀ 上半部分
221=▌ 左半部分    222=▐ 右半部分
```

## 4. 研究参考文件（必须完成）

在开始绘制之前，请仔细研究`resources/`文件夹中的参考文件。请注意以下几点：
- 如果文件扩展名为`.ans`或`.asc`，请使用Clawbius工具加载并分析其内容和结构。关注块是如何组合在一起产生视觉效果的，颜色是如何搭配的，以及过渡效果是如何实现的。
- 同样地，也可以分析`.png`或`.image`格式的参考文件——这些文件虽然以图像形式呈现，但可以帮助您了解最终的视觉效果。观察颜色的分布、半块的使用方式以及阴影的实现方法。

研究参考文件时，请像侦探一样思考——寻找可以借鉴和应用于自己作品中的图案、技术和风格元素。这是一个不可跳过的关键步骤。

### 第一步——阅读README文件
```plaintext
resources/README.md
```
了解每个文件的描述及其展示的内容。同时检查文件夹中是否有README文件未提及的其他文件。

### 第二步——通过API打开并分析画布数据
```python
import requests, json

# 打开参考文件
requests.post("http://127.0.0.1:7777/api/file/open",
    json={"path": "/absolute/path/to/resources/file.ans"}

# 获取画布信息
info = requests.get("http://127.0.0.1:7777/api/canvas/info").json()

# 获取完整的画布数据并分析颜色/代码的使用情况
data = requests.get("http://127.0.0.1:7777/api/canvas/data").json()
blocks = data["result"]
cols = info["result"]["columns"]

# 分析某个区域（例如第10-20行）
for y in range(10, 20):
    unique = {}
    for x in range(0, cols):
        b = blocks[y * cols + x]
        key = (b["code"], b["fg"], b["bg"])
        unique[key] = unique.get(key, 0) + 1
    top = sorted(unique.items(), key=lambda x: -x[1])[:6]
    print(f"第{y}行：{top}")
```

分析以下内容：
- 每个区域主要使用哪些前景/背景颜色组合？
- 使用了哪些阴影块（F1–F4）以及它们的位置？
- 区域之间的过渡是如何处理的？
- 使用了哪些背景颜色？它们都是0–7号颜色吗？

### 第三步——分析参考PNG文件
`resources/`文件夹中的每个参考文件都应该有一个对应的`.png`文件。使用`image`工具直接分析PNG文件：
```plaintext
image(image="<absolute_path_to_resources/file.png>", prompt="...)
```
观察：
- 颜色区域及其过渡效果
- 黑色分隔线在彩色区域之间的分布
- 半块在边缘和曲线上的使用情况
- 整体的构图和密度

## 4. 核心风格规则

### “不接触原则”（No-Tocar）

**这是某些ANSI艺术的重要风格规则。**

不同的对象如果不相接触，会显得更加美观。可以使用黑色（前景色或背景色为0）的半块或完整块将它们分隔开，这样可以使艺术作品看起来更加清晰、有层次感。

**错误示例：** 红色块直接紧邻黄色块。
**正确示例：** 红色块 → 黑色半块分隔 → 黄色块

使用`▌(221)`、`▐(222)`、`▀(223)`、`▄(220)`以及`fg=color, bg=BLACK`来创建清晰的色彩边界。

**示例——从红色区域过渡到黄色区域：**
```plaintext
[RED F4][▐ fg=RED bg=BLACK][▌ fg=YELLOW bg=BLACK][YELLOW F4]
```

### 阴影与深度处理
- 使用F1–F4阴影块来创建渐变效果——切勿对大面积区域使用纯色填充。
- **光源处理：** 确定光源的位置并保持一致性。
- **被照亮的部分：** 使用较浅的阴影块（F1/F2）或对比度较高的前景色。
- **阴影部分：** 使用F3/F4或黑色作为前景色。
- **边缘处理：** 使用`UH(223)`或`LH(220)`以及明亮的背景色来突出边缘。

### 典型的阴影效果示例

```plaintext
**天空渐变（从上到下，从深到浅）：**
  F4 fg=DBLUE bg=BLACK → F3 fg=BLUE bg=BLACK → F2 fg=BLUE bg=BLACK → F1 fg=CYAN bg=BLACK

**地面（深色）：**
  F4 fg=BLACK bg=BLACK → F3 fg=BROWN bg=BLACK → F2 fg=BROWN bg=BLACK

**金属物体（从左上角照亮）：**
  上边缘：UH fg=LGRAY bg=DGRAY  ← 高光
  上部：F2 fg=LGRAY bg=DGRAY
  中部：F3 fg=DGRAY bg=BLACK
  下部：F4 fg=BLACK bg=BLACK  ← 深阴影

**火焰/爆炸背景：**
  F3 fg=RED bg=BLACK → F2 fg=BROWN bg=BLACK → F1 fg=YELLOW bg=BLACK
  在行和列之间随机变化颜色——避免出现水平条纹。

**通用风格提示：** 一旦建立了渐变效果，可以增加复杂性——可以通过混合不同的图案或阶梯式渐变来增加视觉层次感。

### 半块边缘技术
在填充区域和空白区域之间的边界处使用半块来平滑边缘并增加深度：
- `UH(223) fg=color bg=BLACK` — 对象颜色在上，黑色在下方（物体的底部边缘）。
- `LH(220) fg=color bg=BLACK` — 黑色在上，对象颜色在下方（物体的顶部边缘）。
- 这些半块也可以作为两种颜色区域之间的分隔符（遵循“不接触原则”）。
- 对于圆形边缘、曲线或有机形状（如头发或物体的轮廓），也可以使用半块来增强效果。

## 5. 绘制方法

使用Python脚本并通过`requests`发送请求。切勿手动绘制单个单元格，应使用脚本批量处理。

**脚本模板：**
```python
import requests, os
BASE = "http://127.0.0.1:7777"

def post(path, data):
    r = requests.post(f"{BASE}{path}", json=data)
    resp = r.json()
    if not resp.get("ok"): print(f"警告：{path}：{resp}")
    return resp

def rect(x, y, w, h, code=219, fg=7, bg=0):
    if w > 0 and h > 0:
        post("/api/draw/rect/filled", {"x":x,"y":y,"width":w,"height":h,"code":code,"fg":fg,"bg":bg})

def at(x, y, code=219, fg=7, bg=0):
    post("/api/draw/at", {"x":x,"y":y,"code":code,"fg":fg,"bg":bg")

def line(x1, y1, x2, y2, code=219, fg=7, bg=0):
    post("/api/draw/line", {"x1":x1,"y1":y1,"x2":x2,"y2":y2,"code":code,"fg":fg,"bg":bg")

# 设置画布（除非操作员另有指示，否则默认关闭冰色）
post("/api/file/new", {
    "columns": 80, "rows": 25,
    "title": "标题", "author": "Clawd", "group": "ANSIClaw",
    "ice_colors": False
})
```

将脚本保存到`scripts/`文件夹中，将输出文件保存到`output/`文件夹中。请同时保存ANS文件和PNG文件。

## 6. 输入文件（将图像转换为ANSI格式）

操作员可能会将源图像放入`inputs/`文件夹，并要求将其转换为ANSI格式。**除非有明确要求，否则不要查看该文件夹。**

当需要转换输入文件时：
1. 使用`image`工具分析图像的形状、主要颜色、光源和构图。
2. 将图像的颜色映射到16色调色板中（为每个主要区域找到最接近的颜色）。
3. 规划画布布局——决定每个区域应使用哪些阴影/颜色块。
4. 尽量简化设计——ANSI艺术注重抽象效果，而非真实感。最多选择3-5种主要颜色。
5. 在编写代码之前，在注释中草拟构图方案：主要区域的位置、半块的使用位置以及光源的方向。
6. 进行绘制——应用所有常规的风格规则（遵循“不接触原则”、使用阴影渐变、半块边缘效果）。
7. 将最终结果保存到指定位置。

## 8. 从实际操作中获得的经验

### 天空/背景渐变
- **不要使用随机噪声作为纹理效果**——即使使用15-18%的密度，也会产生“雨点”或“静态”状的杂乱效果，看起来很糟糕。
- **不要在每一行都使用按列排列的棋盘格状抖动效果**——ANSI单元格的高度和宽度比例不均匀，因此按x%2的规律排列会导致明显的垂直条纹。
- **使用清晰的纯色条带**——在ANSI天空渐变中，使用4-5行的条带效果更自然。例如：F4→F3→F2→F1，颜色顺序为DBLUE（黑色背景）。
- 如果需要在条带之间创建微妙的过渡效果，最多只能使用一行抖动效果——即使如此也可能会产生条纹，因此请进行测试。

### 花瓣/有机形状
- 开始绘制时，可以比实际需要的宽度更大一些。在ANSI单元格的纵横比下，水平方向的扩展效果更好。
- 在花瓣内部使用完整的阴影渐变（从尖端到基部使用F1颜色渐变）来表现曲率。
- 每个花瓣的最外层边缘应使用`RFH`/`LFH`（半块）和`bg=BLACK`来保持清晰的边界。
- 对于圆形花瓣，需要额外的宽度（至少4个像素）才能清晰显示。

### 中心区域/小色块
- 一个2x2大小的中心块可以有效地表现光影效果：
  - 左上角：F1（亮色）在深色背景上
  - 左右角：F2
  - 下左角：F3
  - 右下角：F4（最深的阴影）
- 背景色`bg=BROWN`（6号）总是安全的（因为它小于8号颜色，适合暖色调的中心区域）。

### API使用注意事项
- PNG文件的导出端点是`/api/file/export/png`（而非`/api/file/export-png`）。
- 会话开始时务必检查`openapi.json`文件——端点可能会发生变化。
- `post()`函数在接收到错误响应时应该会输出警告，但不会导致程序崩溃——请采取防御性措施。
