# 子图注册表

该子图注册表支持用户在 The Graph Network 上便捷地查找 15,500 多个经过分类的子图。用户可以通过域名、网络类型、协议类型或自然语言查询来搜索子图，并获得带有可靠性评分的结果（结果会附带相应的查询 URL）。

## 工具

- **search_subgraphs**：根据域名（如 DeFi、NFT、DAO、游戏领域）、网络类型（如 Ethereum、Arbitrum、Base）、协议类型（如 dex、借贷、桥梁协议）、实体类型或关键词来过滤子图。
- **recommend_subgraph**：用户可以通过自然语言指令（例如“在 Arbitrum 上查找 DEX 交易”）来获取最匹配的子图。
- **get_subgraph_detail**：提供特定子图的详细信息，包括其分类信息、包含的实体、可靠性评分以及查询使用说明。
- **listRegistry_stats**：展示注册表中的所有可用域名、网络类型和协议类型的概览。

## 系统要求

- **运行环境**：Node.js >= 18（通过 `npx` 命令进行安装）。
- **环境变量**：无需设置任何环境变量。该注册表为预构建的独立模块，仅支持读取操作，无需 API 密钥。

## 安装方法

```bash
npx subgraph-registry-mcp
```

## 网络与数据处理机制

- 在首次运行时，服务器会从 [GitHub 仓库](https://github.com/PaulieB14/subgraph-registry) 下载预构建的 `registry.db` 数据库文件（大小约为 5 MB）。该文件会存储在本地，并在后续运行中重复使用。
- 所有工具查询均基于这个本地数据库进行，不会在查询过程中发起任何外部 API 调用。
- 如果使用 SSE 传输协议（通过 `--http` 或 `--http-only` 参数），系统会在本地启动一个运行在 3848 端口的 HTTP 服务器（端口号可通过 `MCP_HTTP_PORT` 环境变量进行配置）。

## 使用场景

- 在查询 The Graph 数据之前，先通过该注册表找到合适的子图。
- 按链路类型筛选高可靠性的 DeFi、NFT、DAO 或治理相关子图。
- 无需手动探索即可获取子图的查询 URL 和实体结构信息。
- 可根据子图的可靠性评分（如查询费用、内容审核质量、索引器质押情况等指标）来对比不同子图。
