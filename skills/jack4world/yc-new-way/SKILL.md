# YC “新方法”（发布 → 分发 → 学习）

## 目标
利用能够**衡量实际需求**的最小产品，将一个想法快速转化为一个**持续运行的分发循环**（几天内完成，而非几周）。

## 工作流程（默认）
### 0) 明确目标（2分钟）
完成以下句子：
- **针对[目标用户群体]**的**[具体痛点]**，我们通过**[传播渠道]**提供**[解决方案]**，当**[关键指标]**达到**[预设阈值]**时，我们就知道这个方案有效。

如果信息缺失，请参考 `references/experiment-brief.md`。

### 1) 选择一个具体的切入点
优先选择以下类型的切入点：
- 用户有明确购买意愿（已经在搜索或购买相关产品）
- 产品信息清晰明确（如SKU、产品列表、模板等）
- 能够通过证据（链接、截图、时间戳等）来验证效果

输出结果：
- 1个主要切入点 + 1个备用切入点
- 3个用户故事（User Stories）

### 2) 首先选择一个分发渠道
仅选择一个渠道：
- **SEO**（内容库 + 内部链接）
- **新闻通讯**（每周摘要）
- **社区**（允许发布的平台，如Reddit/Discord/论坛）

在某个渠道显示出效果之前，**不要**创建其他渠道。

### 3) 发布能够衡量效果的MVP（最小可行产品）
常见的MVP示例：
- SEO网站：10个页面（2个产品介绍页面 + 8个支持页面）
- 新闻通讯：1期内容 + 注册功能 + 归档
- 工具页面：1个计算器/检查表，并包含用户邮箱收集功能

包含以下内容：
- 清晰的呼叫行动（Call to Action, CTA）
- 分析工具
- 有效的证据链接（避免使用“AI显示……”这样的表述）

参考文档：`references/landing-page-checklist.md` 和 `references/seo-10-page-plan.md`。

### 4) 监控指标与决策规则
仅跟踪以下关键指标：
- **流量 → CTA点击率 → 外部链接点击量 → （可选）购买量**
- 发布频率 / 每周页面浏览量（吞吐量）

定义停止/迭代规则：
- 如果CTA点击率低 → 优化解决方案或宣传内容
- 如果CTA点击率很高但无转化 → 优化产品或价格/信任度相关因素
- 如果7天后仍无流量 → 问题出在分发环节（而非产品本身）

参考文档：`references/weekly-loop.md`。

### 5) 每周迭代循环
每周执行以下操作：
1) 加大对有效策略的投入
2) 停止无效的策略
3) 开始新的实验

保持变更日志；避免重复建设。

## 输出模板（请严格按照这些模板使用）
- 实验概述：`references/experiment-brief.md`
- 网页设计检查表：`references/landing-page-checklist.md`
- SEO 10页计划：`references/seo-10-page-plan.md`
- 每周迭代流程：`references/weekly-loop.md`

## 规范与注意事项：
- **先分发再完善系统**：在没有明确效果之前，不要过度自动化。
- **以证据为依据**：所有建议都必须引用来源。
- **从单一切入点、单一渠道、单一指标开始**。
- **法律/服务条款**：优先使用公开可用的资源或允许使用的API；避免绕过登录流程或验证码。
