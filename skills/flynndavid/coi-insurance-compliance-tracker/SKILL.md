## 产品简介  
覆盖范围审核矩阵是一个分为四个阶段的系统，用于管理供应商的保险凭证（COIs），从收集到问题解决的全过程。该系统明确了需要收集哪些文件、要求多少保险覆盖范围、何时跟进，以及当供应商不符合规定时应采取的措施。  

**解决的问题：** 大多数运营团队在出现问题后才会被动地管理COIs，例如在发现供应商的保险过期数月后才采取行动。覆盖范围审核矩阵通过将COI管理转变为一个具有明确触发条件和行动方案的主动式系统，填补了这一空白。  

**输出结果：** 为每个供应商提供完整的COI合规性记录，包括当前状态、保险覆盖范围评分、到期提醒以及详细的整改措施。

## 第一阶段：收集  
### 1.1 — 需要收集的文件（通用要求）  
所有供应商都必须提供以下文件：  
1. **保险凭证（ACORD 25表格）** — 标准的COI文件  
2. **附加被保险人声明**（明确列出您的组织名称）（适用于所有具有物理访问权限或服务交付责任的供应商）  
3. **主要责任险和非共同责任险**（适用于具有重大责任风险的供应商）  
4. **放弃代位权声明**（适用于建筑、设施和高风险服务供应商）

## 第二阶段：验证  
### 2.1 — COI验证检查清单  
收到COI后，请执行以下10项验证：  

**身份验证**  
- [ ] 被保险人名称与供应商的法定实体名称完全一致  
- [ ] 保险公司具有在您所在州经营的许可（通过州保险部门查询确认）  
- [ ] 证书持有人名称正确（与您的法定名称一致）  

**覆盖范围验证**  
- [ ] 所有必需的保险类型均已列出  
- [ ] 每次事故的保险限额符合类别最低要求  
- [ ] 总保险限额符合类别最低要求  
- [ ] 保单生效日期为最新（未过期）  
- [ ] 保单到期日期至少在30天后  

**附加被保险人声明**  
- [ ] 附加被保险人声明已正确附加  
- [ ] 如果适用，包含放弃代位权的条款  
- [ ] 包含主要责任险和非共同责任险的相关条款  

### 2.2 — 常见的COI问题  
如果COI存在以下问题，请标记并采取相应行动：  
| 问题 | 应采取的行动 |  
|----------|--------|  
| 保险到期时间在30天内 | 立即要求续保 — 进入第四阶段 |  
| 保险限额低于类别最低要求 | 要求供应商增加保险覆盖范围 — 暂停合作或给予有条件的批准 |  
| 保险公司不在您所在的州开展业务 | 确认是否需要购买额外保险或拒绝合作 |  
| 附加被保险人名称错误 | 要求提供正确的证书 |  
| 缺少某些保险类型 | 要求供应商在激活前补充相应的保险 |  

### 2.3 — 验证最佳实践  
对于年支出超过10万美元的高价值供应商或高风险类别的供应商：  
- **直接联系保险公司**，确认保险覆盖范围是否有效（拨打ACORD表格上列出的保险公司电话）  
- 对于建筑项目或重大项目，**要求提供完整的保险文件包**而不仅仅是证书  
- **增加额外的验证步骤** — 要求提供实际的附加被保险人声明页面，而不仅仅是摘要证书

## 第三阶段：评分  
### 3.1 — 供应商合规性评分（0-100分）  
从五个维度对每个供应商进行评分。每季度更新评分，或在收到新的COI时更新评分。  

#### 评分维度  
**维度1：保险覆盖完整性（25分）**  
- 所有必需的保险类型均齐全：25分  
- 缺少1种保险类型：15分  
- 缺少2种保险类型：5分  
- 缺少3种或更多保险类型：0分  

**维度2：保险覆盖充分性（25分）**  
- 所有保险限额均达到或超过最低要求：25分  
- 1-2项保险限额低于最低要求（小差距<20%）：15分  
- 1-2项保险限额低于最低要求（大差距>20%）：5分  
- 3项或更多保险限额低于最低要求：0分  

**维度3：保单有效性（20分）**  
- 保单有效，且在90天内未过期：20分  
- 保单有效，但在60-90天内到期：15分  
- 保单有效，但在30-60天内到期：8分  
- 保单有效，但在30天内到期：2分  
- 保单已过期：0分  

**维度4：文件质量（15分）**  
- 所有必需的附加声明均齐全且正确：15分  
- 缺少1项附加声明：10分  
- 缺少2项附加声明：5分  
- 缺少3项或更多附加声明或证书持有人信息错误：0分  

**维度5：响应合规性（15分）**  
- 在首次请求后5天内提交相关文件：15分  
- 在10天内提交相关文件：10分  
- 需要1次跟进时在15天内提交：5分  
- 需要2次或更多次跟进时在15天内提交：0分

## 第四阶段：整改  
### 4.1 — 不同等级的整改措施  

**🟢 绿色（85-100分）：无需采取行动**  
- 将评分记录在跟踪系统中  
- 设置下一次审核日期（90天后或COI到期时，以较早者为准）  
- 如果供应商的合规情况有所改善，向供应商发送确认通知  

**🟡 黄色（70-84分）：监控并通知**  
- 记录评分和具体问题  
- 向供应商发送问题通知（见下方模板A）  
- 设置30天的跟进提醒  
- 如果问题在30天内未解决，升级处理  

**🟠 橙色（50-69分）：需要立即整改**  
- 记录评分和所有问题  
- 向供应商发送整改通知（见下方模板B）  
- 通知内部负责供应商关系的负责人  
- 给予供应商15个工作日的时间来解决问题  
- 如果供应商的年支出较高，考虑暂时暂停新的采购订单  
- 如果问题仍未解决，向上级经理报告  

**🔴 红色（低于50分）：不合规**  
- 立即通知内部供应商负责人和上级经理  
- 向供应商发送不合规通知（见下方模板C）  
- 暂停新的采购订单  
- 给予供应商10个工作日的时间来解决问题  
- 如果保险已过期，建议高风险类别的供应商立即停止工作  
- 将此情况记录在供应商的合规历史记录中  

### 4.2 — 沟通模板  
#### 模板A — 问题通知（黄色等级）  
#### 模板B — 需要整改的通知（橙色等级）  
#### 模板C — 不合规通知（红色等级）  
---

### 4.3 — 升级处理流程  
如果供应商在规定时间内未回应：  
| 天数 | 应采取的行动 |  
|----|--------|  
| 第1天 | 发送相应的通知模板（A、B或C） |  
| 第5天 | 如果仍未回应：向同一联系人发送跟进通知，并抄送供应商的高级联系人 |  
| 第10天 | 如果仍未回应：向内部上级经理报告，并进行法律审查 |  
| 第15天 | 如果仍未回应：发送正式的不合规通知，并引用合同条款 |  
| 第20天 | 进行合同审查，考虑终止合作关系或采取其他法律措施 |
