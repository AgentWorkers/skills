## 什么是VendorGate系统  

VendorGate系统是一个专为中型企业运营团队设计的完整供应商入职管理系统。它取代了传统的邮件沟通方式和分散的电子表格，提供了一种结构化、可重复执行的入职流程，任何团队都可以从第一天开始使用。  

**解决的问题：** 大多数组织在新供应商入职过程中会浪费3-5小时的时间，因为需要处理来回的邮件、丢失的文件以及审批责任不明确的问题。VendorGate系统会在所有要求都满足之前阻止流程的继续，只有在所有条件都符合后才会正式开启入职流程。  

**最终结果：** 新供应商完成入职流程，所有文件收集齐全，所有审批手续完成，同时会发送欢迎邮件，并保留完整的审计记录。

## VendorGate系统的工作流程  

VendorGate系统包含**5个顺序阶段**。供应商必须通过当前阶段才能进入下一个阶段。每个阶段都有明确的负责人、检查清单和退出条件。  
```
GATE 1: Intake & Qualification
        ↓ (Pass/Fail)
GATE 2: Document Collection
        ↓ (Complete/Incomplete)
GATE 3: Internal Approval Routing
        ↓ (Approved/Rejected/Conditional)
GATE 4: System Setup & Access
        ↓ (Configured/Pending)
GATE 5: Welcome & Activation
        ↓
    VENDOR ACTIVE ✓
```

## 第一阶段：信息收集与资格审核  

**负责人：** 采购部门/运营部门  
**时间线：** 第1天  
**退出条件：** 供应商通过基本资格审核  

### 1.1 — 供应商信息收集表  

在收集信息时，请填写以下内容（可以使用Typeform、JotForm、Google Form或自定义的邮件模板）：  

**公司信息**  
- 法人实体名称  
- 如果适用，注册代理人名称（DBA）  
- 注册州/公司所在地  
- 实际地址和邮寄地址  
- EIN（美国企业识别号）/税务识别号  
- 网站地址  
- 成立年份  
- 员工人数（1-10人 / 11-50人 / 51-200人 / 200人以上）  

**主要联系人**  
- 姓名、职位、电子邮件地址、电话号码  
- 应付账款联系人（如有不同）  
- 紧急联系人/升级联系人  

**服务/产品**  
- 服务或产品类别（下拉选项：技术类 / 专业服务类 / 设施类 / 市场营销类 / 物流类 / 其他）  
- 服务内容描述  
- 预计年支出金额  
- 合同预计开始日期  

**认证与合规性（初步自我报告）**  
- 你的公司在[我们所在的州]是否有经营许可？（是/否）  
- 你是否购买了一般责任保险？（是/否）如果有，保险金额是多少？  
- 你是否购买了员工工伤保险？（是/否）  
- 你的公司是否由少数族裔/女性/退伍军人拥有？（是/否——用于报告）  
- 你的公司是否有未解决的诉讼案件？（是/否）  

### 1.2 — 资格审核标准  

根据以下标准对供应商的回复进行评分：  

| 项目 | 取消资格的条件 |  
|--------|------------------------|  
| EIN/税务识别号 | 缺失或无效 → 无法继续流程 |  
| 经营许可 | 对于需要许可的服务类别，如果没有许可 → 暂停流程 |  
| 有未解决的诉讼 | 有 → 在继续流程前需进行法律审查 |  
| 保险自我报告 | 一般责任保险未购买 → 提示进入第二阶段审核 |  
| 预计支出超过5万美元 | 需要高层管理人员的批准才能进入第三阶段 |  

**第一阶段退出条件：** 所有未标记的字段填写完整，且所有取消资格的条件都已解决。将审核结果记录在供应商跟踪系统中，并注明时间戳。

## 第二阶段：文件收集  

**负责人：** 运营部门/供应商（供应商提交文件，运营部门审核）  
**时间线：** 第2-5天  
**退出条件：** 收到所有所需文件，且文件有效期内  

### 2.1 — 按类别划分的文件清单  

**所有供应商（通用）**  
- [ ] W-9表格（美国供应商）或W-8BEN表格（国际供应商）  
- [ ] 保险证明（COI）——具体要求请参阅相关文档  
- [ ] 签署的保密协议（NDA）  
- [ ] 经营许可或同等注册证明  

**技术/软件即服务（SaaS）供应商（额外要求）**  
- [ ] SOC 2 Type II安全认证报告  
- [ ] 数据处理协议（DPA）——如果处理个人数据则需要  
- [ ] 分包商名单（如有）  
- [ ] 渗透测试报告（如果需要访问内部系统）  

**专业服务供应商（额外要求）**  
- [ ] 专业责任保险证明  
- [ ] 相关专业执照/认证证明  
- [ ] 推荐人信息（至少2位客户联系人）  

**设施/承包商（额外要求）**  
- [ ] 承包商许可证（根据所在州要求）  
- [ ] 员工工伤保险证明  
- [ ] 设备/工具保险证明（如有）  
- [ ] OSHA合规性证明（如有）  

**物流/运输供应商（额外要求）**  
- [ ] 货物保险证明  
- [ ] 运输公司编号（如果使用公路运输）  
- [ ] 运营许可证/MC编号  

### 2.2 — 文件审核标准**  

对于收到的每份文件：  
1. **验证真实性**——文件是否使用正式信头，是否已签名，是否来自认可的保险公司或机构？  
2. **检查有效期**——文件应在收到后60天内有效。如果即将过期，请在完成第二阶段前要求供应商更新。  
3. **确认覆盖范围**——文件是否涵盖了合同中的服务内容（例如，技术供应商的保险证明中必须包含“技术服务”）。  
4. **记录接收信息**——记录文件接收日期、文件类型、有效期以及审核人的签名。  

### 2.3 — 文件缺失的跟进流程**  

- **第0天：** 发送首次文件请求（使用模板A）  
- **第3天：** 如果文件不完整，发送提醒（使用模板B）  
- **第7天：** 如果仍未收到完整文件，联系供应商的高级联系人  
- **第10天：** 如果仍未收到回复，暂停入职流程，并通知内部请求人  

#### 模板A — 首次文件请求  
```
Subject: [Your Company] Vendor Onboarding — Documents Required

Hi [Contact Name],

Thank you for your interest in partnering with [Your Company]. To complete your onboarding, we need the following documents by [DATE — 5 business days]:

[List required documents from checklist above]

Please send to: [ops email or portal link]

Questions? Reply to this email or call [contact].

Best,
[Your Name]
[Your Company] Operations
```  

#### 模板B — 提醒邮件  
```
Subject: Action Required: Outstanding Documents for [Vendor Name] Onboarding

Hi [Contact Name],

[List missing items with checkboxes]
```  

## 第三阶段：内部审批流程  

**负责人：** 运营部门（负责流程分配）+ 部门负责人（负责审批）  
**时间线：** 第5-8天  
**退出条件：** 收到所有必要的审批意见  

### 3.1 — 审批流程**  

| 支出金额范围 | 需要的审批人员 |  
|------------|-------------------|  
| 年支出低于1万美元 | 仅部门负责人 |  
| 1万美元至5万美元 | 部门负责人 + 财务部门 |  
| 5万美元至10万美元 | 部门负责人 + 财务部门 + 运营总监/副总裁 |  
| 超过10万美元 | 所有审批人员 + 首席执行官/高管团队 |  
| 任何涉及数据访问的供应商 | 所有审批人员 + IT/安全部门 |  
| 任何涉及法律风险的供应商 | 所有审批人员 + 法律部门/法律顾问 |

### 3.2 — 审批材料**  

向每位审批人员发送以下材料：  
1. **供应商汇总表**（1页）：供应商名称、服务类别、预计支出金额、合同期限、关键联系人  
2. **第一阶段审核结果**：供应商的资格审核信息  
3. **第二阶段文件审核情况**：确认所有文件已收到及任何问题  
4. **风险提示**（如有）：诉讼、保险到期、证书缺失等  
5. **请求理由**：说明为什么需要该供应商（简短说明，3-5句话）  
6 **审批要求**：批准/拒绝/有条件批准  

### 3.3 — 审批条件**  

如果审批人员有条件性批准：  
- 将审批条件记录在供应商档案中  
- 设置提醒，要求在30天内核实条件是否满足  
- 对于轻微的条件问题，不要暂停入职流程——只需记录并跟踪即可  

如果审批被拒绝：  
- 记录拒绝原因  
- 通知内部请求人  
- 暂停入职流程  
- 如果供应商修改后重新提交，从第二阶段开始重新审核  

### 3.4 — 审批响应时间**  

审批人员有**3个工作日**的响应时间。如果未收到回复：  
- 第4天：运营部门发送提醒（“[供应商名称]需要尽快获得批准”）  
- 第6天：运营部门向上级审批人员求助  
- 第8天：默认审批结果为“暂缓”，并记录在案并通知请求人

## 第四阶段：系统设置与访问权限  

**负责人：** IT部门/运营部门  
**时间线：** 第8-10天  
**退出条件：** 供应商在所有系统中完成配置  

### 4.1 **系统配置清单**  

**财务/会计部门**  
- [ ] 将供应商信息添加到会计系统（如QuickBooks、NetSuite等）  
- [ ] 设置付款条款（例如Net 30、Net 60等）  
- [ ] 收集银行账户信息（通过安全方式传输，切勿通过电子邮件）  
- [ ] 将W-9表格/税务文件上传到会计系统  
- [ ] 为供应商分配采购订单处理流程的联系人  

**采购/合同部门**  
- [ ] 将签署的合同上传到合同管理系统（或共享文件夹）  
- [ ] 记录合同开始日期、结束日期和续签日期  
- [ ] 设置合同结束前90天的提醒  
- [ ] 为合同分配内部负责人  

**运营/项目管理部门**  
- [ ] 将供应商信息添加到相关项目组中  
- [ ] 记录工作范围并链接到供应商档案  
- [ ] 设置绩效评估计划（根据合同金额每季度或每年进行一次）  

**IT/安全部门（如果供应商需要系统访问权限）**  
- [ ] 根据最小权限原则设置访问权限  
- [ ] 如果供应商需要访问内部工具，配置单点登录（SSO）/双重身份验证（2FA）  
- [ ] 收集供应商的安全问卷信息  
- [ ] 设置访问权限，有效期与合同结束日期一致  

**合规性记录**  
- [ ] 在合规性跟踪系统中记录保险证明的到期日期  
- [ ] 记录经营许可的到期日期  
- [ ] 设置下一次合规性审核日期

### 跟踪系统自动化（如果使用电子表格）  

设置条件格式：  
- **红色高亮**：保险证明或经营许可在30天内到期  
- **黄色高亮**：文件或许可证在60天内到期  
- **橙色高亮**：如果超过7天仍未通过第二或第三阶段审核，标记为“流程停滞”

## VendorGate系统实施指南（前30天）  

按照以下步骤在您的组织中部署VendorGate系统：  

**第1周：基础设置**  
- [ ] 根据公司需求定制信息收集表  
- [ ] 确定审批流程（填写支出金额范围）  
- [ ] 创建供应商跟踪系统（将上述字段列表导入工具）  
- [ ] 设置文件收集文件夹结构（共享驱动器或供应商门户）  
- [ ] 使用公司品牌定制所有邮件模板  

**第2周：流程测试**  
- [ ] 用一个现有供应商测试入职流程，验证流程是否顺畅  
- [ ] 发现问题并更新检查清单  
- [ ] 向团队介绍五阶段流程及各部门的职责  
- [ ] 明确每个阶段的负责人  

**第3周：自动化设置**  
- [ ] 设置文件到期提醒（日历提醒或电子表格提醒）  
- [ ] 如果可能，将系统连接到会计系统  
- [ ] 创建用于文件提交的共享邮箱地址  

**第4周：正式启用**  
- [ ] 对所有新供应商启用VendorGate系统  
- [ ] 审查现有供应商的文件是否齐全，是否存在合规性问题  
- [ ] 为前5位重要供应商安排首次绩效评估

## 预期成果**  

完成整个入职流程后，应达到以下目标：  
1. ✅ 完整的供应商信息记录（所有字段填写完成）  
2. ✅ 所有所需文件收集齐全、经过审核并归档  
3. ✅ 所有审批记录齐全，包括审批时间和审批人信息  
4. ✅ 供应商在会计系统、合同系统和相关系统中完成配置  
5. ✅ 发送欢迎邮件并确认收到  
6. ✅ 设置文件续签的提醒  
7. ✅ 供应商跟踪系统更新，显示供应商的状态和关键日期  

**标准供应商的入职时间：** 10-12个工作日  
**优化后的入职时间：** 在运营部门专人协助下，5-7个工作日
