# 供应商绩效审计  
**框架：供应商绩效评分卡（VPS）**  
**输出：供应商评估结果、改进计划或终止合作建议**  

大多数供应商关系的恶化源于缺乏有效的评估机制。本季度审计系统为您提供了一种结构化的方法，用于评估所有关键供应商，及时发现潜在问题，并基于数据做出关于是否续签合同、重新谈判或更换供应商的决策。  

---

## 审计频率  
- **所有优先级供应商（年度合同价值 > 10,000 美元或对业务至关重要的供应商）**：每季度进行一次  
- **标准供应商**：每半年进行一次  
- **在发生重大事件时**（如 SLA 违反、安全问题、交付失败等）：立即启动审计  
- **合同续签前**（至少在合同到期前 60 天）：进行一次审计  

---

## 第一阶段：KPI 评分  
对每个评估维度进行 1-5 分的评分。请诚实评分——这是为了您的决策，而非供应商的感受。  

### 评估维度 1：交付与 SLA 履行情况（权重：30%）  
| 分数 | 评分标准 |  
|-------|---------|  
| 5   | 始终超出 SLA 要求；对任何问题都主动沟通；无意外故障。 |  
| 4   | 超过 95% 的时间符合 SLA；问题很少且能迅速解决。 |  
| 3   | 大多数时间符合 SLA；偶尔出现问题但能合理解决。 |  
| 2   | 经常违反 SLA；解决速度慢或需要升级处理。 |  
| 1   | 经常出现交付失败；SLA 只是形式上的要求，实际并未得到遵守。 |  

**所需证据**：工单数据、交付日志或事件记录。切勿凭记忆评分。  

### 评估维度 2：输出质量（权重：25%）  
| 分数 | 评分标准 |  
|-------|---------|  
| 5   | 输出质量超出预期；错误率接近于零；几乎不需要返工。 |  
| 4   | 输出质量稳定；小问题能得到及时处理。 |  
| 3   | 质量一般；需要一定程度的返工。 |  
| 2   | 质量不稳定；返工频繁；内部团队需花费时间处理供应商提供的成果。 |  
| 1   | 输出经常不符合标准；需要大量内部资源进行修正。 |  

### 评估维度 3：响应速度与沟通效率（权重：20%）  
| 分数 | 评分标准 |  
|-------|---------|  
| 5   | 始终能联系到供应商；主动报告问题；沟通清晰及时。 |  
| 4   | 在约定的 SLA 时间内响应；大部分时间都能主动沟通。 |  
| 3   | 响应一般；有时需要催促。 |  
| 2   | 响应缓慢；大部分沟通都需要您主动发起。 |  
| 1   | 联系不可靠；问题是由您发现的，而非供应商主动报告的。 |  

### 评估维度 4：性价比（权重：15%）  
| 分数 | 评分标准 |  
|-------|---------|  
| 5   | 投资回报率明显；成本低于市场平均水平；展现出显著的价值。 |  
| 4   | 性价比良好；成本与输出质量和合作关系相匹配。 |  
| 3   | 市场平均水平；既不便宜也不明显过贵。 |  
| 2   | 相对于所提供的价值或市场替代方案来说，价格偏高。 |  
| 1   | 相对于所获得的服务，价格过高；有其他替代方案能以更低成本提供相同或更好的服务。 |  

### 评估维度 5：战略契合度与规划一致性（权重：10%）  
| 分数 | 评分标准 |  
|-------|---------|  
| 5   | 与我们的业务目标高度一致；主动帮助我们实现目标。 |  
| 4   | 战略方向基本一致；了解我们的目标并据此调整服务。 |  
| 3   | 仅满足基本需求；没有超出合同范围的服务。 |  
| 2   | 在某些方面存在战略分歧；产品/服务方向与我们的需求不符。 |  
| 1   | 完全不匹配；产品/服务与我们的需求背道而驰。 |

## 第二阶段：综合评分与等级分类  
**评分计算方式：**  
```
VPS = (D1 × 0.30) + (D2 × 0.25) + (D3 × 0.20) + (D4 × 0.15) + (D5 × 0.10)
```  

**最高得分：5.0**  

| VPS 评分范围 | 评估等级 | 建议行动 |  
|-----------|------|-------------------|  
| 4.0 – 5.0 | 🟢 绿色 — 可信赖的合作伙伴 | 续签合同，考虑扩大合作范围或建立战略合作伙伴关系 |  
| 3.0 – 3.9 | 🟡 黄色 — 需关注 | 续签合同但需提出改进计划（针对评分最低的维度） |  
| 2.0 – 2.9 | 🟠 橙色 — 存在风险 | 重新谈判条款或开始寻找替代供应商；给予 60 天的改进期限 |  
| 1.0 – 1.9 | 🔴 红色 — 需更换供应商 | 开始更换流程；不再续签合同 |

## 第三阶段：问题日志审查  
在最终确定评分前，审查该供应商在评估期间的事件/工单记录：  
- 共计发生了多少起事件？其中还有多少未解决？  
- 平均解决时间是多少？与 SLA 对比如何？  
- 有哪起事件被标记为关键/高影响事件？  
- 这些事件是否对业务产生了负面影响（如收入损失、客户投诉、合规问题）？  

**事件严重性调整因素：**  
- 1 次以上未解决的关键事件 → 评估等级降一级  
- 3 次以上未解决的中等严重事件 → 无论 VPS 评分如何，都必须制定改进计划  

---

## 第四阶段：改进计划模板（适用于黄色/橙色等级）  
如果 VPS 评分低于 4.0，需制定正式的改进计划：  

**改进计划 — [供应商名称] — [季度]**  
- 审查期间：[开始日期] – [结束日期]  
- VPS 评分：[X.X] / 5.0  
- 评估等级：黄色 / 橙色  
- 审查日期：[距离今天 90 天]  

**识别的主要问题：**  
1. [具体问题及证据]  
2. [具体问题及证据]  

**所需改进措施：**  
1. [具体、可量化的改进措施] — 目标：[指标]，截止日期：[日期]  
2. [具体、可量化的改进措施] — 目标：[指标]，截止日期：[日期]  

**未完成改进的后果：**  
- 黄色等级：升级为橙色等级；同时开始寻找替代供应商。  
- 橙色等级：不再续签合同；立即启动更换流程。  

**注意事项：** 将该改进计划发送给供应商，并要求其书面确认。  

---

## 第五阶段：终止合作触发条件  
在满足以下任一条件时，立即启动更换供应商的流程：  
- VPS 评分低于 2.0  
- 连续两个季度处于橙色等级  
- 发生具有重大业务影响的重大事件且问题未得到解决  
- 供应商表示将停止提供该产品/服务  
- 市场上存在性价比更高的替代方案  
- 出现合规或安全问题  

满足上述条件时，立即开始寻找替代供应商，并设定明确的切换日期。
