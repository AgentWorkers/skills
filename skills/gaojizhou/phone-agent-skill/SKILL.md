# AutoGLM 手机代理技能

该技能允许 Codex 通过 AutoGLM 手机代理 SDK 来控制 Android 设备：执行点击、输入、滑动、滚动、启动应用程序、截图以及读取用户界面文本等操作。它主要用于自动化任务，例如端到端测试、数据收集或重现用户操作流程。

## 前提条件
- 一台已启用开发者模式和 USB 调试功能的 Android 设备或模拟器。
- 系统路径中包含 `adb` 工具，并且设备能够在 `adb devices` 列表中看到。
- 已安装 AutoGLM 手机代理 SDK（详情请参阅上游文档：https://github.com/zai-org/Open-AutoGLM）。
- 手机代理后台服务正在运行（在使用该技能之前，请先启动 SDK 提供的代理服务）。

## 设置步骤
1) 连接设备并验证连接性：`adb devices` 命令应至少列出一台设备。
2) 按照 SDK 的指南启动手机代理服务（该服务通常会绑定到您的机器上的某个主机/端口）。记下服务的 URL。
3) 将服务 URL 暴露给代理运行时环境，例如通过设置 `PHONE_AGENT_ENDPOINT=http://127.0.0.1:5000`（请根据实际情况调整主机和端口）。
4) 当 SDK 提示时，授予设备所需的权限（如覆盖显示/无障碍访问权限），以确保点击和文本输入操作能够正常执行。

## 使用方法
- 描述高层次的操作目标；代理会将其分解为具体的用户界面操作步骤。
- 提供应用程序名称或屏幕上的文本作为操作的定位依据（例如：“打开设置，搜索‘Wi-Fi’，然后关闭它”）。
- 在执行可能带来风险的操作时，请求用户确认是否需要截图。

示例指令：
- “打开 Play Store，搜索‘Signal’应用程序，并将搜索结果中的第一个链接分享给我。”
- “在 Twitter 应用中，进入设置 → 隐私设置，关闭位置精度选项，然后发送该选项状态的截图。”
- “启动我们的测试应用程序，使用提供的测试账号登录，并截取购买确认页面的截图。”

## 输出结果
- 代理执行的操作日志（点击/滑动/输入记录）、截图以及 SDK 返回的结构化观察结果。
- 后端出现的错误会直接显示出来，以便您快速排查问题。

## 故障排除
- 如果命令执行失败，请确认 `PHONE_AGENT_ENDPOINT` 可以访问，并且端口没有被防火墙阻止。
- 如果点击操作的位置不正确，请在 SDK 中重新校准设备分辨率，或重启无障碍访问服务。
- 如果未检测到设备，请重新连接 USB 数据线，确保 `adb` 具有相应的权限，并重新执行 `adb devices` 命令。

## 安全性与限制
- 该技能会执行真实的用户界面操作——请仅在测试设备或测试账号上使用。
- 避免执行需要生物特征认证的操作；SDK 无法绕过硬件的身份验证提示。
- 受网络影响的操作可能会因地区或应用程序版本的不同而有所差异；在可靠性至关重要的情况下，请提供明确的备用方案。
