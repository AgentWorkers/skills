# Jarvis Db Query Doctor 01  
提供一个具体且可重复使用的开发工作流程。  

## 输入参数  
- 期望的结果和截止日期  
- 限制条件（时间、工具、风险承受能力）  
- 可用的资源（代码、文档、截图、日志）  

## 执行步骤  
1. 以可衡量的标准定义成功与失败的标准。  
2. 在扩大工作规模之前，构建一个最小化的、可执行的方案。  
3. 在每个关键节点执行操作，并记录相应的证据。  
4. 在最终提出建议之前，通过质量检查。  
5. 提供可立即实施的成果及后续行动方案。  

## 质量检查标准  
- 所有的主张都必须有证据支持。  
- 明确说明假设和权衡因素。  
- 对于高风险操作，优先选择可逆的变更方案。  
- 每个后续步骤都应明确指定负责人及预计完成时间。  
- 如果主要计划失败，需准备备用方案。  

## 输出格式  
- 情况概述（最多5行）  
- 主要发现（按影响程度排序）  
- 行动计划（今日/本周内需完成）  
- 风险及应对措施  
- 有用的检查清单或具体操作指令  

## 示例提示  
- “使用此工具将我的杂乱笔记整理成一份可投入生产的计划。”  
- “运行该工具后，提供一个以风险优先的行动计划，并附带相应的验收标准。”
