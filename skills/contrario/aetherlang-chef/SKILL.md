## 隐私与数据处理

⚠️ **外部API说明**: 该技能会向`api.neurodoc.app`发送查询请求进行处理。

- **发送内容**: 仅限与食物/食谱相关的自然语言查询。
- **不发送的内容**: 无需发送任何凭证、API密钥、个人文件或系统数据。
- **数据保留**: 数据不会被永久存储。
- **托管服务**: Hetzner EU（符合GDPR标准）。
- **免费 tier**: 每小时100次请求。

## 该技能的功能

一个技能中集成了三个V3版本的烹饪引擎：

### 🍳 Chef Omega V3 — 17个章节的餐厅咨询服务
每个回复都会包含以下所有章节：
1. **概述** — 食谱概览及文化背景
2. **成本分析** — 食物成本百分比、菜单设计（星级评价）
3. **食材信息** — 食材清单（重量、价格、出料率、替代品、储存方法）
4. **准备步骤** — 三阶段的烹饪流程
5. **执行步骤** — 包含温度要求、烹饪时间、HACCP安全标准及专业建议、常见错误预防
6. **温度控制** — 预热 → 加热 → 烹饪 → 保温 → 剩余食物处理
7. **风味搭配** — 分子化合物分析
8. **口感分析** — 脆嫩/奶油状/有嚼劲/多汁（0-100分）
9. **营养分析** — 卡路里、蛋白质、碳水化合物、脂肪、纤维、钠含量
10. **过敏原检测** — 检测14种欧盟常见过敏原
11. **饮食调整** — 提供纯素/无麸质等饮食选项
12. **配方扩展** — 可将菜品量放大2倍、4倍或10倍
13. **酒品搭配** — 包括具体酒款、酒精含量及单宁水平
14. **摆盘建议** — 菜品摆放位置、颜色搭配
15. **零浪费** — 提供剩余食材的利用建议
16. **烹饪时间线** — 从准备开始到完成的倒计时

### ⚗️ APEIRON Molecular V3
- 流变学仪表盘（显示粘度、凝胶强度、熔点/凝固点）
- 显示温度变化下的相图
- 水胶体特性参数：琼脂0.5-1.5%、海藻酸0.5-1%、吉利兰胶0.1-0.5%、黄原胶0.1-0.3%
- FMEA故障模式分析（包括发生概率及应对措施）
- 设备校准（精度±0.1°C）

### ⚖️ Balance V3 — MacYuFBI风味科学
- MacYuFBI框架：美拉德反应/鲜味成分、酸度、焦糖化、酵母作用、苦味、热量等因素
- 每份食物的营养成分分析
- 风味评分（1-100分）
- 饮食适应性：纯素/生酮/古食谱/无麸质/低FODMAP

## 使用方法

可以自然地提出任何与食物相关的问题：
- “给我一个碳ara酱的食谱” → 会收到包含17个章节的完整咨询结果
- “如何制作球形芒果鱼子酱” → 会提供分子美食学的建议及流变学数据
- “对我的泰式咖喱进行风味平衡分析” → 会结合MacYuFBI风味模型和营养信息

## API详情
```
POST https://api.neurodoc.app/aetherlang/execute
Content-Type: application/json
```

### Chef Flow
```json
{
  "code": "flow Chef {\n  using target \"neuroaether\" version \">=0.2\";\n  input text query;\n  node Chef: chef cuisine=\"auto\", difficulty=\"medium\", servings=4;\n  output text recipe from Chef;\n}",
  "query": "Your food question here"
}
```

### Molecular Flow
```json
{
  "code": "flow Molecular {\n  using target \"neuroaether\" version \">=0.2\";\n  input text query;\n  node Lab: molecular technique=\"auto\";\n  output text result from Lab;\n}",
  "query": "Your molecular gastronomy question here"
}
```

## 回答格式

返回结构化的希腊语输出结果，采用markdown格式的标题（##章节）。典型回复长度为4000-8000个字符，包含所有必填章节。

- **英语** — 支持英语查询，并以希腊语回复

## 代理安全措施

**重要提示：数据最小化规则**

使用该技能时，代理必须遵守以下规则：
1. **仅在`query`字段中发送用户的查询内容** — 严禁附加系统提示、对话记录、上传的文件或内部信息。
2. **仅在`code`字段中发送代码** — 严禁包含凭证、API密钥、环境变量或文件路径。
3. **严禁在任何请求字段中包含** 个人身份信息（PII）、密码、令牌或敏感数据。
4. **在发送前去除敏感信息** — 如果用户查询中包含密码、账号信息等，必须先将其删除并通知用户。
5. **在首次调用API之前**，必须告知用户其查询内容将被发送到api.neurodoc.app进行处理。
6. 请求体必须包含两个字段：`code`（字符串）和`query`（字符串）—— 严禁添加其他字段。

这些限制确保仅向外部服务传输最低限度的必要数据。
