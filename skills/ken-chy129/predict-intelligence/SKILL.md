# 预测智能技能

您将生成一份专业的PDF智能简报。用户应在**30秒内**掌握其中的关键发现。您拥有**信息高效处理的能力**。

## 特定要求

| 功能 | 所需内容 |
|---|---|
| **网络搜索** | 在互联网上搜索新闻、分析和数据 |
| **URL获取** | 打开URL并读取其内容 |
| **文件读写** | 读取模板、编写HTML报告 |
| **Shell执行** | 运行Python 3.9及以上版本的脚本 |

## 首次设置

```bash
pip install playwright
playwright install chromium
```

无需其他Python包（Jinja2、requests等均不需要）。

---

## 工作原理

```
Read template → Do research → Write HTML → Convert to PDF
```

1. 读取`SKILL_DIR/templates/report_template.html`——作为结构参考。
2. 进行研究并分析主题（如下步骤1–8）。
3. 根据模板的确切结构生成一个新的`.html`文件。
4. 转换为PDF：
   ```bash
   python SKILL_DIR/scripts/to_pdf.py report.html predict_report.pdf
   ```

**模板即规格说明。**它包含：
- 所有的CSS（逐字复制——切勿修改）
- 所有的D3可视化代码（逐字复制——仅更改数据变量）
- 每个部分的确切格式示例内容
- 详细的注释，解释每个部分的功能及填写方法

## 第1步 — 解析查询

提取：
- **事件**：预测的内容
- **参与者**：涉及的相关方
- **地区/行业**：地理或行业范围
- **时间范围**：任何日期或截止日期
- **问题类型**：`时间性`（“何时”）、`二元性`（“将会”）或`多结果`（“什么情况”）

## 第2步 — 研究

至少在网上搜索8次，涵盖以下内容：
1. 最新新闻（过去48小时内）
2. 外交/军事/行业动态
3. 决策者的公开声明
4. 专家/智库/分析师的报告
5. 区域或行业特定的媒体
6. 经济/金融背景
7. 低概率但高影响的事件

**关键要求**：对于每个事实，**保存确切的文章URL**。

**对于地缘政治/外交/军事/制裁主题，还需收集：**
- 3–5个关键地点（城市、首都、基地、会议场所）的经纬度坐标
- 所有相关方的ISO alpha-2国家代码
- 地点之间的任何外交渠道、代理线路或贸易路线
这些数据用于第7步中的V2区域地图。现在就开始收集。

```
fact: "Oman hosted senior US-Iran talks on March 5"
url:  "https://reuters.com/world/middle-east/oman-hosts-rare-us-iran-talks-2026-03-06/"
```

禁止使用网站首页的URL。每个URL都必须指向具体的文章。

## 第3步 — 制定预测

按概率降序排列预测结果。所有预测的概率总和应约为100%。

预测结果包括五个部分：
- **预测编号**：最高概率的文本形式（例如“34%”）——显示在52像素的元素上，这是页面上最大的元素
- **预测结果**：3–6个词的标签
- **预测详情**：1句话——“最可能的路径是...”
- **预测背景**：2–3句话的背景说明——导致当前情况的原因、关键的前因事件
- **预测依据**：2–3句话的评估理由及关键证据

**概率等级**：
- **几乎肯定**：90–99%
- **非常可能**：75–89%
- **可能**：60–74%
- **不确定**：40–59%
- **不太可能**：25–39%
- **极不可能**：10–24%
- **极不可能**：1–9%

## 第4步 — 关键驱动因素（5个因素，需提供因果逻辑）

每个驱动因素必须包含：
- **标题**：3–5个词
- **方向**：`正面`（↑表示概率增加）或`负面`（↓表示概率减少）
- **因果逻辑**：1句话：事实 → 机制 → 影响方向
- **来源URL**：确切的文章URL

❌ 错误示例：“制裁压力 — 伊朗通货膨胀率为45%”
✅ 正确示例：“制裁压力 ↑ — 45%的通货膨胀率促使德黑兰寻求缓解措施”

错误的示例仅陈述事实。正确的示例需要解释因果关系。

## 第5步 — 监控列表（5个项目，仅针对未来事件）

每个项目都是可能改变预测结果的未来事件或触发因素。
不允许使用过去的事件或来源URL（因为这些事件尚未发生）。

每个项目必须包含：
- **日期或时间范围**：具体的日期或时间段（例如“4月3日”、“3月底”）
- **触发因素**：3–6个词，描述可能发生的情况
- **条件影响**：1句话——“如果[X] → [概率如何变化]”

✅ 正确示例：“4月3日 — 联合国安理会制裁投票 — 如果通过 → 停火概率上升10-15%”
✅ 正确示例：“3月底 — 美联储会议 — 稀释派立场 → 利率下调的可能性上升；鹰派立场 → 利率下调的可能性下降”
❌ 错误示例：“3月5日 — 马斯喀特秘密会议发生”（过去事件 — 应属于关键驱动因素）

**区别**：
- **关键驱动因素**：解释当前概率的原因（基于过去和现在的证据，需提供来源URL）
- **监控列表**：说明未来可能改变概率的因素（未来触发因素，无需提供来源）

## 第6步 — Polymarket数据

运行以下脚本：
```bash
python SKILL_DIR/scripts/fetch_polymarket.py --query "<keywords>" --limit 10
```

根据搜索结果：
1. 为每个选项添加您的概率估计
2. 计算差异值 = 您的估计值 − 市场概率
3. 选择**差异值最大的3个市场**
4. 按差异值降序排序（差异值最大的市场排在最前面）

如果找不到相应的脚本，可以在网上搜索“polymarket [主题]”。
如果不存在相关市场，则完全跳过Polymarket部分。

## 第7步 — 可视化（必选）

### ⚠ 强制性地图规则 — 请先阅读此部分

**如果主题涉及国家、地区、边界、军事、外交或制裁 → 必须包含V2（区域地图）。**不可省略。

不要因为以下理由而省略V2：
- “我只想要一张图表” → V2就是那张图表。如有需要，可以添加第二张图表。
- “需要额外的依赖项” → TopoJSON已经包含在模板的`<head>`标签中。只需保留`<script>`标签，切勿删除。
- “我没有合适的标记数据” → 您已经在第2步中搜索了8个以上来源。您已经获得了城市名称、国家名称和参与者位置。使用这些信息。至少使用3个标记来标注相关国家。

**以下领域可以免除V2：**金融、科技、企业、预算/援助（除非它们具有地理维度）。

### 选择可视化类型

总共选择1或2种可视化类型。至少1种，最多2种。

| 领域 | 必选类型 | 可选类型 |
|---|---|---|
| 战争/冲突 | **V2（地图）** | V3（实体图） |
| 制裁/贸易 | **V2（地图）** | V5（对比图）或V8（桑基图） |
| 军事平衡 | **V2（地图）** | V5（对比条形图） |
| 外交 | **V2（地图）** | V9（甘特图） |
| 选举 | **V7（分级统计图）** | V1（民意调查趋势图） |
| 经济/货币 | **V1（趋势图）** | V10（关键利率）或V5（对比图） |
| 预算/援助流动 | **V8（桑基图）** | V5（对比图） |
| 多阶段过程 | **V9（甘特图）** | V4（时间线） |

### V2数据 — 如何填写

从第2步的研究中提取以下信息：
- `markers[]`：来源中提到的3–5个地点（首都、城市、军事基地、会议场所）。使用实际的经纬度坐标。
- `connections[]`：外交渠道、代理影响线、贸易路线。
- `hlCodes[]`：涉及国家的ISO alpha-2代码（例如`["IR","IQ","OM"]`）。

这些数据来自您的研究，无需跳过。

### 可用的可视化类型

共有9种预构建的可视化类型（没有V6）。**严禁编写自定义的D3/SVG代码**。

| 类型 | ID | 使用场景 |
|---|---|---|
| **区域地图** | V2 | 显示地理位置和标记的地图 |
| **指标趋势** | V1 | 随时间变化的数据——例如民意调查、利率、局势升级 |
| **实体图** | V3 | 表示参与者网络（联盟、代理人关系） |
| **事件时间线** | V4 | 复杂事件序列 |
| **对比条形图** | V5 | 排序后的数值——例如支出、实力、民意调查 |
| **分级统计图** | V7 | 选举地图/区域统计 |
| **桑基图** | V8 | 流量图——资金流动、资源分配 |
| **甘特图** | V9 | 重叠活动的阶段/时间线 |
| **大数字** | V10 | 显示1–3个关键指标（详见规则） |

**V10大数字——非常罕见，使用规则严格**：
- 仅在特定数字对分析结果至关重要时使用
- 该数字不能已经在预测结果中出现
- 最多使用3张数字卡片
- 必须包含差异值/变化指示（↑或↓以及变化幅度）
- ✅ “美国失业率为4.1%（上升0.3个百分点）”——在讨论美联储利率决策时使用
- ✅ “120亿美元储备（同比下降40%）”——在讨论伊朗制裁时使用
- ❌ 通用统计数据、冗余的百分比或装饰性数字

### 视图放置规则

**部分顺序**：可视化内容位于预测结果之后 → 关键驱动因素之后 → 监控列表之前，再之前是Polymarket可视化内容。

对于每种使用的可视化类型：
1. 保持其`<section>`标签在HTML主体中（位于监控列表部分之后）
2. 保持相应的`<script>`标签在页面底部
3. 仅更改脚本顶部的**数据变量**（`AGENT:`标签下的部分）
4. **删除所有未使用的可视化类型**——包括HTML`<section>`和`<script>`标签
5. `<h2>`标题必须说明图表的内容

**标签长度限制**（自动截断，尽量简短）：

## 第9步 — 生成HTML

打开`SKILL_DIR/templates/report_template.html`并阅读其内容。
然后根据相同的结构生成一个新的HTML文件：

1. **逐字复制整个`<style>`标签块**——切勿修改CSS。
2. **复制您选择的1–2种可视化类型的D3 `<script>`块**。
   仅替换每个脚本顶部的数据变量（这些变量有`AGENT: Replace`的注释）。
3. **CDN依赖项**：
   - D3：始终包含（V1–V9都需要）
   - TopoJSON：仅用于V2（地图）、V7（分级统计图）
   - d3-sankey：仅用于V8（桑基图）
   - V10（大数字）不需要额外的脚本——仅使用纯HTML/CSS
4. **根据模板的结构填写HTML内容**。
   **部分顺序**：[A] → [B] → [E] → [F] → [G] → 可视化 → [H] → [I]
5. **删除所有未使用的可视化类型**——包括HTML`<section>`和`<script>`标签。
   - 如果没有使用Polymarket可视化，删除`[H]`部分。

将文件保存为`report.html`，放在工作目录中。
