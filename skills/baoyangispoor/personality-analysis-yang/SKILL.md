# 人格分析器

根据用户的书面文本，分析其人格特征、沟通风格和行为倾向。

## 何时使用该功能

在以下情况下使用此功能：
- 用户提供文本、消息或聊天记录，并请求进行人格分析；
- 用户希望了解某人的写作风格能揭示出关于他们的什么信息；
- 用户询问对方的沟通模式、情绪倾向或性格特征；
- 用户提供一段对话内容，并询问“这个人是怎样的？”

---

## 分析框架

从以下五个维度进行结构化的人格分析：

### 1. 🗣️ 沟通风格
分析此人表达自己的方式：
- **直接性**：他们说话直截了当还是委婉含蓄？
- **正式程度**：正式、随意，还是两者兼有？
- **语言简洁性**：言简意赅还是啰嗦详细？
- **语气**：热情、冷漠、中立还是热情洋溢？
- **幽默感**：他们会使用笑话、讽刺或双关语吗？

### 2. 💡 认知模式
分析此人的思维方式：
- **逻辑与情感**：他们先讲道理还是先表达情感？
- **思维方式**：倾向于宏观思考还是注重细节？
- **确定性**：他们说话时是否绝对明确，还是总是有所保留？
- **好奇心**：他们会提问、探索想法或表现出求知欲吗？
- **思维组织性**：他们的思维有条理还是随性而为？

### 3. 🌡️ 情绪倾向
解读文字背后的情绪：
- **情绪表达**：他们是善于表达情感还是情感内敛？
- **同理心**：他们会关注他人的感受还是只关注自己？
- **压力/焦虑表现**：是否急躁、过度解释或频繁道歉？
- **自信程度**：自信还是缺乏自信，需要他人认可？
- **情绪稳定性**：情绪表达是否稳定，还是容易波动？

### 4. 🤝 人际交往方式
评估他们与他人的相处方式：
- **合作与独立**：他们是喜欢与他人合作还是更喜欢独自工作？
- **权力关系**：他们如何看待自己与他人的关系（居高临下、平等或谦逊）？
- **信任度**：他们是否乐于分享信息或保持谨慎？
- **冲突处理方式**：回避冲突、对抗性强还是善于协商？
- **社交意识**：他们是否意识到自己在他人眼中的形象？

### 5. 🎯 价值观与动机
推断此人看重什么：
- **核心价值观**：他们在语言中反复提及哪些内容？
- **成就导向**：他们是目标导向、过程导向还是以人为导向？
- **道德观念**：他们强调公平、忠诚、权威还是其他价值观？
- **让他们感到沮丧的事情**：抱怨和矛盾点能反映他们的真实价值观

## 分析输出格式

分析结果应按照以下格式呈现：

```
## Personality Analysis

### Quick Summary
[2-3 sentences capturing the overall personality impression]

### Communication Style
[Analysis with specific examples from the text]

### How They Think
[Cognitive style analysis with text evidence]

### Emotional World
[Emotional tendencies and what they suggest]

### How They Relate to Others
[Interpersonal style and social orientation]

### What They Value
[Core values and motivations inferred from language]

### Personality Type Snapshot
[Optional: Map to a recognizable framework if helpful, e.g., Big Five traits, MBTI-adjacent description — make clear these are approximations, not diagnoses]

### Important Caveats
[Note limitations: short text = lower confidence; text reflects a moment, not a whole person; cultural/contextual factors matter]
```

## 分析指南

**注意事项**：
- 每个结论都应基于文本中的具体词语、短语或模式；
- 使用概率性语言（如“可能表明”、“倾向于”等），避免下绝对性结论；
- 当证据不足或模棱两可时需说明；
- 既要指出优点也要指出潜在的盲点；
- 当证据矛盾时，要考虑多种可能的解释。

**禁止事项**：
- 不要做出临床诊断（例如“此人患有自恋型人格障碍”）；
- 不要基于过短的文本（少于3-4句话）做出过度推断；
- 不要考虑上下文（客户投诉与情书的语气会影响分析结果）；
- 仅分析文本中实际的内容，不要加入个人猜测；
- 分析时要客观，避免过于严厉或片面。

**文本长度建议**：
- **< 50字**：仅给出简要印象，强调分析的不确定性；
- **50–200字**：进行中等程度的分析，指出存在的不足；
- **200字以上**：提供较为全面的分析，结论较为可靠；
- **多条消息**：注意分析其中的模式和一致性。

## 特殊情况

### 多人对话
如果用户提供了多方对话的内容：
- 分别分析每个参与者；
- 注意他们之间的互动关系（权力平衡、情感互动模式）。

### 非母语文本
如果文本是由非母语者撰写的：
- 重点关注内容与思想，而非可能因语言差异产生的表达特点；
- 明确说明这一点。

### 专业文本与个人文本
- 专业文本（如邮件、报告）表达更为正式，需更加仔细地解读隐含意义；
- 个人文本（如聊天记录、社交媒体帖子）更具真实性，可按字面意思进行分析。

## 示例使用方法

**用户**：“你能分析一下这段文字的作者的人格特征吗？”

**你的回复**：按照上述输出格式进行分析。首先给出总体结论，然后逐个维度进行详细分析。最后说明分析过程中的注意事项。

如果用户尚未提供文本，可以询问：“请分享您希望我分析的文本或消息内容。”
