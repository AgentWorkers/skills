# 项目0技能

## 什么是项目0？

P0是一个基于Solana的去中心化金融（DeFi）经纪平台，它提供了无需许可的服务。该平台支持跨多个借贷平台（包括P0原生平台、Kamino和Drift）的统一保证金账户，使代理能够获取整个生态系统中最高的收益和最深厚的流动性。所有操作都在链上完成，无需托管，并且无需注册。

**文档：**

- 协议概述：https://docs.0.xyz/
- TypeScript SDK：https://docs.0.xyz/typescript-sdk/overview

### 使用案例1：获取收益 + 获得信贷

用户可以存入资金以获取DeFi市场中的最高收益。当用户需要流动资金（例如进行购买）时，可以选择用存入的抵押品借款稳定币，而无需出售加密货币。这样用户既能持续获得收益，又能随时使用现金。

**流程：** 存款 -> 获得收益 -> 根据需要借款稳定币 -> 还款。

**示例提示：**

- “我在P0上存入我的SOL以获得最高收益。”
- “我需要500美元的USDC，但不想出售我的SOL。”
- “在P0上最便宜的稳定币是什么？”

### 使用案例2：高级收益策略

用户可以通过利率套利和循环策略在Solana上找到最高的收益。这些策略涉及存入一种资产、借款另一种资产，并管理杠杆头寸。虽然收益更高，但需要主动监控账户的健康状况。

**流程：** 确定策略 -> 存入抵押品 -> 借款 -> 监控账户健康状况。

**示例提示：**

- “目前P0上哪些策略的收益最高？”
- “在P0上设置一个bbSOL/SOL的利率套利。”
- “将我的资金用于P0以获得最高收益。”

## 代理工作流程

当用户请求在P0上获取收益、存款、借款或管理头寸时，请按照以下步骤操作。切勿跳过编写代码的步骤。

### 第1步：检查钱包余额

- 确定钱包地址：从`.env`文件中获取`WALLET_ADDRESS`；如果设置了`WALLET_KEYPAIR`，则从密钥对中获取地址；或者询问用户。
- 从钱包API中获取钱包中的代币持有情况（详见下面的钱包端点）。无需提供凭证。使用每个代币的`address`字段，根据`mint`字段将它们与P0平台上的银行进行匹配。

### 第2步：获取P0数据

- 从P0的API中获取银行信息和策略信息（详见下面的只读API部分）。

### 第3步：推荐最佳操作

- 如果用户希望**获取收益或获得信贷**，推荐最佳的存款机会，并提及借款选项（使用案例1）。
- 如果用户希望获得**最高的收益**或提到策略、利率套利或循环操作，推荐最佳的策略，并解释所需的存款和借款设置（使用案例2）。
- 将钱包中的代币与可用的银行收益进行匹配。对于钱包持有的每种代币，从银行数据中找到最佳的存款年化收益率（APY）。比较所有持有情况以找到总体最高的收益。
- 如果钱包中持有的代币在P0平台上没有对应的银行（例如memecoins），建议将其兑换成支持的代币。
- 如果兑换成其他代币可以获得更高的收益，建议进行兑换，并解释其中的权衡。
- 如果钱包中持有多种支持的代币，建议同时存入多种代币——而不仅仅是收益最高的那种。
- 在执行操作之前，向用户展示具体的计划和收益数字。

**示例：**“您的钱包中持有0.09 bbSOL（约18美元）和0.10 SOL（约15美元）。bbSOL在P0上的年化收益为15.2%，而SOL的年化收益为4.9%。我建议您存入bbSOL以获得更高的收益。是否继续？”

### 第4步：收集凭证

在执行链上操作之前，代理需要一个RPC URL和钱包密钥对。

- **RPC URL：** 从`.env`文件中获取`RPC_URL`、`SOLANA_RPC_URL`或`HELIUS_RPC_URL`。如果没有找到，请询问用户：“我需要一个付费的Solana RPC URL来执行交易。（Helius提供免费 tier，网址为https://www.helius.dev）”
- **钱包地址：** 从`.env`文件中获取`WALLET_ADDRESS`。如果设置了该地址，可用于只读操作（如查看钱包余额、发现账户），无需提供密钥对。只有在签署交易时才需要密钥对。
- **钱包密钥对：** 如果用户在消息中提供了密钥对路径，直接使用该路径。否则从`.env`文件中获取`WALLET_KEYPAIR`。如果两者都没有，请询问：“我需要一个Solana密钥对来签署交易。请在`.env`文件中设置`WALLET_KEYPAIR`，或者告诉我您的密钥对JSON文件的位置。”
- **P0账户（可选）：** 从`.env`文件中获取`P0_ACCOUNT`。如果设置了该地址，直接使用它来加载账户（跳过账户发现步骤）。如果没有设置，请按照下面的“创建或加载账户”步骤进行操作。
- 请勿伪造URL、文件路径或账户地址。等待用户提供真实的值。

### 第5步：收集兑换凭证（如需）

- 如果第3步中的计划涉及代币兑换，代理需要一个Jupiter API密钥。
  - 检查环境变量或`.env`文件中是否有`JUPITER_API_KEY`或`JUP_API_KEY`。
  - 如果找到了密钥，直接使用它；否则询问用户：“我需要一个Jupiter API密钥来进行代币兑换。可以在https://portal.jup.ag获取一个免费的密钥（每60次请求收费60美分）”。
- 如果不需要兑换，跳过此步骤。

### 第6步：执行操作

- 执行第3步中的计划。如果需要，先进行代币兑换（详见“兑换代币”部分），然后进行存款、借款等操作。使用Solscan链接报告结果：`https://solscan.io/tx/${signature}`

## 只读操作：从P0获取数据

- 使用公共HTTP API进行只读操作。无需SDK、钱包或RPC。

### 银行端点

`GET https://ai.0.xyz/api/banks`

- 返回所有借贷池（银行）的信息，包括利率、元数据和定价。仅包含提供抵押品的银行（隔离的银行会被过滤掉）。响应是一个轻量级的数据结构（每个银行9个字段），其中已经预先计算好了存款年化收益率（deposit_apy）。

---

**每个银行的字段：**

| 字段           | 描述                                                         |
| --------------- | ----------------------------------------------------------------------------- |
| `bank_address`  | 银行的链上地址（使用SDK的`client.getBank()`方法）                          |
| `symbol`        | 代币符号（如SOL、USDC、JitoSOL等）                                      |
| `mint`          | 代币的铸造地址                                                    |
| `mint_decimals` | 代币的小数位数（SOL为9位，USDC为6位）                                  |
| `venue`         | 借贷平台（P0、Kamino、Drift）                                             |
| `deposit_apy`   | 实际存款年化收益率（百分比，已预先计算）                             |
| `borrow_apy`    | 借款年化收益率（百分比）                                              |
| `usd_price`     | 代币的Oracle价格（以美元计）                                             |
| `token_program` | 代币的程序名称（如TOKEN PROGRAM_ID或TOKEN_2022）                              |

`deposit_apy`字段已经包含了底层代币的收益率（例如LST的质押利率）。无需手动计算——只需按照`deposit_apy`字段降序排序即可。

**请注意：** 借款仅支持P0平台的银行。Kamino和Drift平台的银行仅支持存款——您可以在这些平台上获取收益，但不能借款。在寻找借款机会时，请过滤`venue`字段，确保其为“P0”。**

**查找最佳存款收益：**

- 按`deposit_apy`字段降序排序银行。

**查找最便宜的稳定币借款：**

- 过滤`venue`字段为“P0”且`symbol`字段为`[USDC, USDT, USDG, USDS, HYUSD]`的记录，然后按照`borrow_apy`字段升序排序。

**将钱包中的代币与银行匹配：** 使用`mint`字段将钱包中的代币与可用的银行进行匹配。钱包中的代币地址应与银行的`mint`字段相匹配。

### 策略端点

`GET https://ai.0.xyz/api/strategies`

- 返回预先计算好的策略信息，显示最佳的存款/借款组合及其预计的年化收益率。这些策略有助于确定应存入哪种代币以及应借款哪种代币以获得最佳的收益差。

---

**每个策略的字段：**

| 字段                  | 描述                                                         |
| ---------------------- | ----------------------------------------------------- |
| `heading`              | 人类可读的名称（例如“bbSOL/SOL利率套利”）                         |
| `primaryBankAddress`   | 存款的目标银行                                     |
| `secondaryBankAddress` | 借款的目标银行                                     |
| `spread`               | 存款和借款之间的利率差                                 |
| `leverage`             | 杠杆倍数                                                     |
| `apy`                  | 预计的年化收益率（小数形式，例如0.085表示8.5%）                             |

该端点返回按年化收益率降序排列的策略。`heading`字段描述了策略的类型（例如“利率套利”、“循环策略”等）。

**使用策略来规划存款和借款：**

- 策略会显示哪些存款/借款组合具有最佳的收益差。代理可以分别执行这些存款和借款操作。

---

**将策略与银行关联：** 使用`primaryBankAddress`和`secondaryBankAddress`从银行端点获取银行详细信息。

### 钱包端点

`GET https://ai.0.xyz/api/wallet/{address}`

- 返回Solana钱包中的所有代币持有情况，包括余额和美元价值。无需API密钥或RPC。

---

**返回的数据结构：`{ wallet, total_usd_value, tokens[] }`，其中每个代币的字段包括：**

| 字段       | 描述                                                         |
| ----------- | ---------------------------------------- |
| `address`   | 代币的铸造地址                                               |
| `symbol`    | 代币符号                                                     |
| `name`      | 代币名称                                                     |
| `decimals`  | 代币的小数位数                                                     |
| `balance`   | 代币的余额（以人类可读的形式显示）                                       |
| `usd_price` | 代币的美元价格                                                     |
| `usd_value` | 该代币的总美元价值                                                     |

## 在链上与协议交互

- 使用TypeScript SDK执行需要签署操作的命令：创建账户、存款、取款、借款、还款等。这些操作需要Solana密钥对和用户授权。

### 先决条件**

- Node.js版本 >= 18
- Solana密钥对（JSON字节数组）
- 有足够的资金（SOL用于交易手续费和存款代币）
- **付费的RPC端点**——请检查`.env`文件中的`RPC_URL`；如果没有，请询问用户（详见代理工作流程的第4步）
- **钱包地址**——可选：请检查`.env`文件中的`WALLET_ADDRESS`，用于只读操作（详见代理工作流程的第4步）
- **钱包密钥对**——请检查`.env`文件中的`WALLET_KEYPAIR`；或者使用用户在消息中提供的路径（详见代理工作流程的第4步）
- **P0账户**——可选：请检查`.env`文件中的`P0_ACCOUNT`，以跳过账户选择步骤（详见代理工作流程的第4步）
- **Jupiter API密钥**——仅在进行代币兑换时需要（详见代理工作流程的第5步）

### 钱包设置

- 代理需要一个Solana密钥对来签署交易。确定密钥对的路径如下：
  1. 如果用户在消息中提供了路径，直接使用该路径。
  2. 从`.env`文件中获取`WALLET_KEYPAIR`。
  3. 如果两者都没有，请询问：“我需要一个Solana密钥对来签署交易。请在`.env`文件中设置`WALLET_KEYPAIR`，或者告诉我您的密钥对JSON文件的位置。”

### 创建或加载账户

P0账户（`MarginfiAccount`）是一个链上的PDA（Positional Data Account），用于管理头寸。一个钱包可以通过不同的`accountIndex`值拥有多个账户。每个账户都有独立的头寸和风险——一个账户中的抵押品无法被另一个账户访问。

## 核心操作

### 存款

- 返回一个传统的交易记录。金额以用户界面显示的单位表示。

**SOL代币的Gas预留：** 在存款SOL时，钱包中至少需要保留0.01 SOL作为交易手续费。如果存入全部SOL余额，后续的交易将会失败。

---

### 取款

- 返回一个分阶段的交易记录。请依次发送每个交易。

---

### 借款

- 返回一个分阶段的交易记录（可能包含Oracle调用交易）。

**必须进行预借款检查**。跳过这些检查会导致交易失败（程序错误代码6009）或浪费手续费。

---

### 还款

- 返回一个传统的交易记录。

---

### 交易验证

- 在发送任何交易后，务必在链上验证交易是否成功。`confirmTransaction`仅确认交易已被包含在区块中——它并不检查程序是否成功执行。交易可能被“确认”但实际上在程序层面失败（例如，错误代码6009：“RiskEngine rejected”）。

---

**注意事项：** 在报告交易成功之前，请务必检查`confirmation.value.err`。如果`err`不为空，说明交易虽然被记录在链上，但程序拒绝了它——用户的资金并未转移。

**重试策略：** 如果交易失败，请不要重复尝试。请使用Solscan链接和错误信息报告失败情况，然后由用户决定如何处理。

## 通过Jupiter进行代币兑换

- 如果钱包中持有的代币与策略所需的代币不同（例如，钱包中持有SOL，但最佳收益是bbSOL），请先使用Jupiter API进行兑换。

**Jupiter需要一个API密钥。** 请检查`.env`文件中是否有`JUPITER_API_KEY`或`JUP_API_KEY`。如果没有找到，请询问用户：“我需要一个Jupiter API密钥来进行代币兑换。可以在https://portal.jup.ag获取一个免费的密钥（每60次请求收费60美分）”。

### 获取报价并执行兑换

- `amount`参数以原始整数单位（lamports表示SOL，SPL的最小单位）输入。请使用银行API中的`mint_decimals`字段进行转换。

## 错误参考

| 错误代码          | 原因                                                         | 解决方案                                                         |
| --------------------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------- |
| `Bank not found`     | 未找到对应的银行地址                                               | 请通过银行API验证银行地址                                         |
| Insufficient free collateral | 抵押品不足                                                     | 请增加存款或减少借款金额                                                     |
| Simulation failed    | 交易在链上会失败                                                     | 请检查日志——可能是Oracle数据过期或余额不足                                   |
| Transaction expired     | 块哈希在确认前已过期                                                     | 请使用新的块哈希重新尝试                                         |
| Account not found     | P0账户地址不存在                                                     | 请验证地址或创建新账户                                                     |
| Program error 6009     | RiskEngine拒绝交易（健康状况不佳或Oracle数据过期）                                      | 请确认账户具有足够的抵押品；检查账户健康状况                                   |
| 429 Too Many Requests | RPC请求次数过多                                                     | 请使用付费的RPC服务提供商                                         |
