# 早晨简报

以一份包含所有重要信息的汇总开始你的一天。该功能会将你未读的邮件、当天的日程安排、待办任务以及当地天气信息整合成一份简洁的简报，通过 Telegram 或 Discord 发送给你，或保存为每日文件。

你可以在每天早上通过 cron 任务自动执行该功能，也可以根据需要随时手动触发以查看最新状态。

## 功能内容

- 未读邮件汇总（总数及按优先级排序的前 5 条）
- 当天的日程事件（包括时间与地点）
- 来自任务管理系统的待办任务或提醒
- 当地天气预报（温度、天气状况、降雨概率）

所有信息都集中在一条消息中，无需在多个应用之间切换。

## 设置

### 必需条件
- **邮件访问权限**：Gmail（通过 Gmail API 或其他邮件服务）或你的代理能够读取的任何邮件服务
- **日历访问权限**：Google 日历 API 或 Apple 日历

### 可选（但推荐）
- **天气信息**：无需 API 密钥。你的代理可以从任何免费来源（如 wttr.in、Open-Meteo 等）获取天气数据
- **任务管理系统**：如果你使用任务管理工具（如 Todoist、Asana、Notion 或纯文本文件），请告知代理任务的位置
- **发送渠道**：Telegram 机器人、Discord 机器人或文件输出。如果没有配置发送渠道，简报将保存在 `memory/briefings/YYYY-MM-DD.md` 文件中

### 配置

告知代理以下偏好设置（这些设置将存储在内存中）：

1. **你的时区**：确保 cron 任务在正确的本地时间执行
2. **简报发送时间**：你希望接收简报的时间（默认为当地时间上午 7:30）
3. **邮件账户**：需要扫描的收件箱
4. **日历**：需要检查的日历
5. **天气信息来源**：你的城市或邮政编码
6. **发送方式**：通过 Telegram、Discord 或文件发送
7. **格式偏好**：简洁版（使用项目符号）或详细版（包含邮件预览）

## 工作流程

当简报被触发（通过 cron 任务或手动请求）时，代理将执行以下步骤：

### 第一步：收集邮件信息
- 检查收件箱中的未读邮件
- 统计未读邮件的总数
- 根据发件人和邮件主题的紧急程度筛选出前 5 条重要邮件
- 显示每封邮件的发件人名称、主题以及简短的预览内容
- 标记出紧急邮件（关键词：urgent、ASAP、deadline、overdue、payment）

### 第二步：收集日程信息
- 从日历中获取当天的事件信息
- 显示每个事件的名称、开始时间、结束时间、地点（如有）以及参与者人数
- 突出显示下一个即将发生的事件（例如：“团队会议 45 分钟后开始”）
- 注意任何时间冲突的情况

### 第三步：收集任务信息
- 检查任务管理系统中今天到期的或逾期的任务
- 列出最重要的 5 个任务
- 显示每个逾期任务的逾期天数
- 如果未配置任务管理系统，则跳过此步骤

### 第四步：收集天气信息
- 获取指定地点的当天天气信息
- 包括当前温度、最高/最低温度、天气状况（晴天/多云/下雨）以及降雨概率
- 如有必要，添加实用提示（例如：“建议带伞——下午 3 点后有 80% 的降雨概率”

### 第五步：整理并发送简报
将所有信息整理成简洁的格式：

```
🌅 Morning Briefing — [Day, Month Date]

📧 EMAIL (X unread)
• [Sender] — [Subject] (preview)
• [Sender] — [Subject] (preview)
• [Sender] — [Subject] (preview)
🔴 [X] flagged as urgent

📅 CALENDAR
• 9:00 AM — Team standup (Google Meet)
• 11:30 AM — Client call with [Name] (Zoom)
• 2:00 PM — Dentist appointment
⏰ Next up: Team standup in 45 minutes

✅ TASKS
• [Task 1] — due today
• [Task 2] — due today
• [Task 3] — 2 days overdue ⚠️

🌤️ WEATHER — [City]
72°F / 22°C — Partly cloudy
High 78° Low 65° | Rain: 10%

Have a great day! ☕
```

通过配置的发送渠道发送简报，或保存到 `memory/briefings/YYYY-MM-DD.md` 文件中。

### 第六步：设置 cron 任务
配置 cron 任务，每天早上自动执行简报：

- **时间**：每天指定的时间（默认为当地时间上午 7:30）
- **任务**：执行早晨简报流程
- **输出**：将简报发送到指定的渠道

你的代理应使用其 cron 任务调度系统（Mac 上使用 LaunchAgents，Linux 上使用 cron，或代理内置的调度器）来自动触发该功能。

## 定制建议

- **添加新闻头条**：包含你所在行业的前三条新闻
- **添加股票/加密货币价格**：如果你关注特定股票/加密货币
- **添加团队动态**：从 Slack/Discord 频道获取信息
- **周末模式**：周六/周日使用不同的格式（跳过工作邮件，添加轻松的内容）
- **旅行模式**：在旅行时添加航班状态、酒店信息及当地提示
