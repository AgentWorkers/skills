# 自动化客户跟进系统

不要再因为疏于联系而错失商机了。该系统会自动读取您的客户或潜在客户列表，识别出近期未联系过的联系人，并自动生成个性化的跟进消息，确保不会有任何机会被遗漏。

您可以每周通过 cron 任务运行该系统，或者随时查看哪些联系人需要关注。

## 功能亮点：
- **识别逾期未联系的潜在客户**：系统会标记出 X 天内未收到您消息的联系人。
- **个性化跟进模板**：为每个逾期联系人生成定制的跟进消息。
- **优先级排序**：根据交易金额、最后一次联系时间以及潜在客户的阶段来决定跟进的优先级。
- **跟进统计**：显示总联系人数量、逾期联系人数量等详细数据。
- **沟通记录追踪**：记录您发送的内容、时间以及接收者信息。

## 设置步骤

### 必需条件：
- **客户列表**：一个 CSV 文件、Google Sheet 或任何代理能够读取的结构化列表。至少包含姓名、电子邮件（或电话号码）以及最后一次联系日期。列越多，个性化程度越高。

### 可选配置（推荐）：
- **发送权限**：如果您希望代理在获得您的批准后发送跟进消息（通过 Gmail、Outlook 或 Twilio 发送短信）。
- **CRM 集成**：如果您使用 CRM（如 HubSpot、Notion 或 Airtable），可以直接使用该系统，而无需使用 CSV 文件。
- **日历访问权限**：代理可以使用日历功能建议合适的会面时间。
- **通知渠道**：通过 Telegram 或 Discord 发送跟进摘要通知。

### 配置参数：
- **客户列表位置**：文件路径、Google Sheet URL 或 CRM 连接信息。
- **逾期阈值**：多少天未联系即被视为“逾期”（默认值：14 天）。
- **风险阈值**：多少天未联系即被视为“高风险”（默认值：30 天）。
- **跟进风格**：指定跟进消息的语气（专业、随意、友好或直接），以确保模板与您的沟通风格一致。
- **跟进时间**：指定系统运行的时间（默认值：每周一上午 9:00）。
- **每次运行的最大跟进数量**：每次运行最多生成多少条跟进消息（默认值：10 条）。
- **通知方式**：选择通过 Telegram、Discord 或文件发送跟进摘要。
- **行业/背景信息**：输入您的业务类型，以便跟进内容更具针对性（例如：“网页设计公司”、“房地产投资者”、“SaaS 企业创始人”）。

## 工作流程：
1. **加载客户列表**：从配置的来源（CSV、Google Sheet、CRM 或 Notion）读取客户列表。
2. **分类逾期联系人**：根据最后一次联系时间将联系人分为三类：
   - **高风险**：超过风险阈值（30 天以上）：这些潜在客户可能正在失去兴趣，需要立即跟进。
   - **逾期**：超过逾期阈值（14-30 天）：需要定期跟进，但无需等待太久。
   - **活跃**：在逾期阈值以内（< 14 天）：最近有联系，本次无需采取行动。
3. **排序与选择**：根据以下标准为所有逾期和高风险联系人分配优先级：
   - 交易金额：金额越高，优先级越高。
   - 逾期天数：逾期时间越长，优先级越高。
   - 交易阶段：距离成交越近，优先级越高。
4. **生成跟进消息**：为每个选中的联系人生成个性化跟进消息：
   - 根据联系人的公司和交易阶段调整内容。
   - 保持简洁：消息长度控制在 3-5 句以内。
   - 包含明确的行动号召（CTA），例如提出会面请求或分享有用的信息。
5. **编译跟进摘要**：将所有生成的跟进消息整理成摘要格式。
6. **发送与追踪**：通过指定渠道发送跟进摘要，并在下方显示原始消息供审核。
   - 收到用户批准后发送消息。
   - 发送后更新客户列表中的最后一次联系日期。
   - 在 `memory/follow-up-log/YYYY-MM-DD.md` 文件中记录沟通记录。

## 自定义建议：
- **多渠道跟进**：为同一联系人同时生成电子邮件和 LinkedIn 消息。
- **自动分类新联系人**：新联系人加入时自动分配交易阶段和优先级。
- **多步骤跟进流程**：设计多轮跟进策略（例如：第一天发送电子邮件，第三天发送 LinkedIn 消息，第七天发送电话提醒）。
- **跟踪成交情况**：标记交易是否成功，并记录转化率。
- **推荐机制**：对于活跃客户，在项目完成后建议请求推荐。
- **会前准备**：在预约通话前查看联系人的历史记录并准备谈话要点。
