# 智能邮件摘要

别再被收件箱里的邮件淹没了。这个智能工具会扫描你的未读邮件，将它们分类为不同的优先级，并为紧急邮件草拟回复内容——这样你只需花5分钟时间查看邮件，而不是45分钟来阅读它们。

你可以按照设定的时间表（早上和下午）运行该工具，或者在任何收件箱变得难以处理时随时触发它。

## 你将获得什么

- 所有未读邮件被分为4个优先级等级
- 为紧急邮件草拟了回复内容
- 识别并区分了新闻通讯和营销邮件
- 从邮件正文中提取了需要处理的行动事项
- 统计摘要（总未读邮件数、紧急邮件数量、最久未读邮件的时间）

## 设置

### 必需条件
- **电子邮件访问权限** — 需要能够通过Gmail API或任何你的代理可以读取的电子邮件服务访问你的Gmail账户。代理需要具有读取你的收件箱和列出未读邮件的权限。

### 可选条件
- **发送权限** — 如果你希望代理在获得你的批准后发送草拟的回复，还需要提供发送权限
- **发送渠道** — 可以通过Telegram/Discord发送邮件摘要，或者选择以文件形式输出
- **联系人信息** — 如果你有CRM或联系人列表，请告知代理这些信息。这有助于它优先处理来自重要联系人的邮件。

### 配置

告诉你的代理以下信息：
1. **电子邮件账户** — 要扫描的收件箱
2. **VIP列表** — 始终被视为高优先级的电子邮件地址或域名（例如老板、重要客户、家人）
3. **静音列表** — 始终被归类为低优先级的发件人（例如你不会阅读的新闻通讯、自动发送的通知）
4. **摘要发送时间** — 指定工具运行的时间（默认：当地时间上午8点和下午2点）
5. **发送方式** — 指定摘要的发送目的地
6. **回复风格** — 你撰写邮件的方式（正式、随意、直接、友好），以便草拟的回复符合你的风格
7. **每次处理的邮件数量上限** — 每次运行的最大邮件数量（默认：50封）

## 工作原理

### 第1步：扫描收件箱
- 获取所有未读邮件（数量不超过配置的上限）
- 对于每封邮件，提取以下信息：发件人、主题、接收日期、邮件正文预览（前200个字符）以及任何附件

### 第2步：分类
将每封邮件分为以下4个类别：

**🔴 紧急 — 需要在今天内处理**
- 来自VIP发件人，或邮件中包含“紧急”、“尽快”、“截止日期”、“逾期”、“需要行动”等关键词
- 包含直接针对你的问题、审批请求或时间敏感的内容
- 正文中提到了24小时内的截止日期

**🟡 需要处理 — 需要回复，但不紧急**
- 包含针对你的问题、信息请求或会议邀请
- 来自已知联系人或与工作相关的发件人
- 需要在2-3天内回复

**🔵 供参考 — 有空时阅读**
- 包含更新信息、状态报告、共享文档或被抄送的邮件
- 不包含直接针对你的问题或需要处理的行动事项
- 了解这些信息即可，但不会妨碍你的工作

**⚪ 噪音邮件 — 跳过或批量处理**
- 来自被标记为静音的发件人、营销邮件或自动发送的通知
- 包含新闻通讯、促销信息或社交媒体通知
- 处理方式：归档、建议取消订阅或忽略

### 第3步：提取行动事项
对于紧急和需要处理的邮件，提取具体的行动事项：
- 需要回答的直接问题
- 提到的截止日期
- 需要查看的文档
- 需要确认的会议
- 需要批准的请求

### 第4步：草拟回复（仅针对紧急邮件）
为每封紧急邮件草拟一条回复：
- 保持回复风格与用户设定的风格一致
- 回答邮件中的具体问题
- 保持简洁（最多3-5句话）
- 提供明确的下一步操作或答案
- 将回复标记为“草稿”状态 — 未经用户批准前不要发送

### 第5步：整理摘要
格式化邮件摘要：

```
📧 Email Digest — [Time]

📊 INBOX STATUS
Total unread: [X] | Urgent: [X] | Action needed: [X]
Oldest unread: [X days ago] from [sender]

🔴 URGENT ([X] emails)
1. [Sender] — [Subject]
   → [Action item / question]
   📝 Draft reply ready

2. [Sender] — [Subject]
   → [Action item / question]
   📝 Draft reply ready

🟡 ACTION NEEDED ([X] emails)
3. [Sender] — [Subject]
   → [What they need from you]

4. [Sender] — [Subject]
   → [What they need from you]

🔵 FYI ([X] emails)
5. [Sender] — [Subject] (status update)
6. [Sender] — [Subject] (shared doc)

⚪ NOISE ([X] emails)
[X] newsletters | [X] notifications | [X] marketing
💡 Consider unsubscribing from: [top 3 noisy senders]
```

### 第6步：发送
通过配置的渠道发送邮件摘要。如果生成了草拟的回复，会在摘要下方显示这些回复以供用户审核。
