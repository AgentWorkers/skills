# 照片说明

当用户分享一张包含相关信息（如拍摄地点、使用的相机、镜头、胶片类型、拍摄对象、拍摄氛围）的照片时，需要为所有指定的平台生成相应的说明文字。每个平台都有其独特的表达风格和格式要求。

如果用户指定了具体的摄影设备（如相机机身、镜头、胶片类型或数码设置），请在说明中予以体现；但不要编造用户未提供的设备信息。

## 平台说明

### 📸 Instagram
- **风格**：简洁、富有诗意或幽默感，让图片本身说话。
- **格式**：1-2行描述 → 空行 → 设备信息（如有） → 空行 → 标签。
- **标签**：最多使用5个标签（Instagram的限制）。优先选择能够体现照片风格的标签（如`#filmphotography`）、拍摄地点、使用的胶片类型/设备，以及一个与照片氛围相关的标签。质量优先于数量。

### 📷 Flickr
- **风格**：描述性更强，注重故事性和拍摄技巧。
- **格式**：使用斜体标题，后跟1-3句描述性文字，最后注明设备信息。
- **内容要求**：包括拍摄地点以及吸引摄影师拍摄的元素，类似写一篇摄影随笔。

### 🐦 X (Twitter)
- **风格**：简洁明了，字数控制在280字以内，避免滥用标签。
- **格式**：用一句话概括照片的主题，设备信息可放在最后。
- **目标**：吸引读者停下滑动屏幕，仔细看照片。

### 🪟 Glass
- **风格**：像摄影师之间交流一样自然、真诚，不使用标签或吸引互动的内容。
- **格式**：1-3句描述拍摄地点和简要感受，设备信息用中点（·）分隔。
- **氛围**：像在摄影散步时与朋友聊天一样随意。

### 🟦 Tumblr
- **风格**：更具文学性和表现力，篇幅稍长，注重氛围和故事讲述。
- **格式**：使用加粗的标题作为标题，后跟2-4句叙述或反思性文字，最后加上设备信息。
- **标签**：标签之间使用空格，例如`#film photography`。

### 🔵 Bluesky
- **风格**：亲切、温暖，注重社区互动，类似早期的Twitter。
- **格式**：1-2句描述性文字，语气轻松但富有思考。
- **设备信息**：可选，但应具有实际意义（Bluesky社区反对滥用标签）。

### 🧵 Threads
- **风格**：类似Instagram，但更注重对话式交流，像与粉丝聊天一样自然。
- **格式**：1-2句描述性文字，设备信息若有趣也可提及，标签使用3-5个。

### 🔢 500px
- **风格**：注重技术和拍摄技巧，这个社区非常重视摄影技巧。
- **格式**：先写标题，再写1-3句描述照片的拍摄技巧、环境条件及成功的原因，务必包含完整的设备信息。
- **内容要求**：可提及相机设置、光线条件或拍摄技巧。

### 🟠 Reddit
- **风格**：真实、略带自嘲，注重社区互动，避免过度自我推广。
- **格式**：标题简洁明了，正文包含拍摄背景和设备信息。
- **示例格式**：`Bombay Beach [Canon EOS 1V, Tri-X 400]`
- **内容要求**：2-3句描述拍摄背景和故事，提及相关子版块（如r/analog用于胶片摄影，r/photography用于数码摄影，r/streetphotography等）。

### 👤 Facebook
- **风格**：个人化、亲切，像与朋友家人分享一样自然。
- **格式**：2-3句描述性文字，讲述拍摄时的情境和感受，设备信息仅在必要时提及。
- **标签**：尽量少用或不用标签（1-2个），Facebook用户更关注故事本身。

### 🎞️ VSCO
- **风格**：简洁、诗意，注重照片本身的美感。
- **格式**：最多1行描述，有时只需一个词或短语，避免使用标签。
- **氛围**：像低声细语一样自然，避免过度解释。
- **设备信息**：除非是关于胶片类型的，否则不要提及。

### 📝 Substack
- **风格**：叙述性强，类似散文，读者期待的是一篇完整的文章中的照片说明。
- **格式**：2-4句描述性文字，作为新闻邮件中照片下方的说明文字，包含拍摄背景、观察细节和感受。
- **内容要求**：设备信息可增加故事层次感，同时提及地点和拍摄条件，强调照片背后的故事感。
- **标签**：使用空格分隔标签，如`#film photography`。

### 📌 Pinterest
- **风格**：描述性强，便于搜索。Pinterest注重用户体验和内容优化。
- **格式**：分为两部分：
  - **标题**：简短且包含关键词（5-10个词），例如`[沙漠道路，日落时分，Amboy California`或`胶片摄影，莫哈韦沙漠风光`。
  - **描述**：2-3句描述场景、氛围和风格，自然融入相关关键词（如地点、风格、胶片类型、主题等）。
- **目标**：让搜索“沙漠胶片摄影”或“66号公路风格”的用户能找到这条内容。
- **标签**：Pinterest使用描述中的关键词进行内容推荐。

## 生成说明后的操作

生成所有平台的说明后，更新Apple Notes中的“Photo Captions Log”记录。使用脚本将新内容添加到现有记录的最顶部：

```bash
python3 /root/.openclaw/workspace/tools/update_captions_note.py \
  "<Photo Title / Location>" \
  "<Camera · Film/Settings>" \
  "<captions as HTML string>"
```

该脚本会自动查找Apple Notes中的“Photo Captions Log”记录（通过IMAP连接），删除旧记录，并将新内容保存为最新条目（放在最上方）。整个过程应保持安静，除非出现错误，否则无需通知Paul。

## 注意事项：

- 根据每张照片的具体内容和氛围调整说明文字。
- 避免在不同平台重复使用相同的表述，确保每个平台的说明都符合其平台的特点。
- 适当的幽默和机智可以提升说明的趣味性，但需与照片的整体氛围相匹配。
- 如果照片是黑白的，可根据需要添加相关的黑白摄影标签。
- 每条说明都应为该照片量身定制，避免通用化。
- 对于胶片照片，强调其模拟摄影的美感；对于数码照片，则着重描述拍摄瞬间和技巧。
- 如果用户仅指定某些平台，只需为这些平台生成说明。
- 书写时模仿人类的表达方式，避免使用复杂的标点符号（如破折号或分号），使用自然的语句结构。设备信息在Glass、Flickr和500px平台上可使用中点（·）分隔。
