# Poku — 外拨电话

## 第一步：解析电话号码

- **原始号码**（例如 `917-257-7580`）——去除格式化信息，并在前面加上 +1（美国地区的默认前缀）。结果：`+19172577580`。
- **个人联系人的姓名**——直接询问用户获取电话号码，不要猜测。
- **仅使用企业名称**——使用搜索工具找到电话号码，然后在继续操作前与用户确认。

此步骤的结果是 <标准化后的电话号码>，该号码将用于第四步。

---

## 第二步：收集详细信息并确认意图

请阅读 `references/EXAMPLES.md`。每个模板都明确列出了该类型电话所需的详细信息。使用相应的模板来确定还缺少哪些信息，然后仅向用户询问这些信息。

如果没有任何模板与电话类型匹配，请询问用户：具体的目标、所需的名称或参考号码。

根据实际情况推断合理的默认值，并告知用户。不要重复询问已经提供的信息。

在进入第三步之前，必须与用户分享计划并获得用户的确认。

> “我打算拨打 [电话号码]，联系 [联系人名称]，目的是 [具体目标]。我会说明我是代表 [用户名称] 打电话的。如果没有人接听，我会留下语音邮件：[留言内容]。可以继续吗？如果可以，我稍后会去拨打电话。”

**在此阶段，收集任何可能有助于通话的额外信息（例如参考号码）。**

---

## 第三步：起草通话提示信息

如果 `references/EXAMPLES.md` 中有与电话类型匹配的模板，请使用该模板，并将所有占位符替换为实际值。切勿留下未填充的占位符。

如果没有匹配的模板，请按照以下结构起草通话提示信息：
1. **身份**——代理的身份以及他们代表谁打电话。
2. **目标**——具体的通话目的，以及针对不同回答情况的应对策略。
3. **语音邮件内容**——如果无人接听时应留下的具体留言内容。

---

## 第四步：拨打电话

在 `openclaw.json` 文件中找到 `POKU_API_KEY`。

使用 `exec` 工具执行 `curl` 命令来拨打电话（务必设置 `background: false`，并明确将 `yieldMs`（背景延迟时间）设置为 300000）。

```bash
curl -s -X POST \
  -H "Authorization: Bearer $POKU_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"message": "<drafted message from Step 3>", "to": "<normalized number>"}' \
  https://api.pokulabs.com/phone/call
```

在请求尚未完成时切勿重试——通话可能会持续长达 5 分钟。如果 `POKU_API_KEY` 未设置，请检查 `~/.openclaw/openclaw.json` 文件。有关错误代码，请参阅 `references/API.md`。

---

## 第五步：报告结果

一旦工具完成通话并返回 “响应”，立即通知用户。例如：“我刚刚完成了通话。我与 Kristin 联系过了，她已经确认了明天晚上 7 点的预约。”
首先报告重要的信息：确认的日期/时间、联系人姓名、参考号码以及后续步骤。如果响应中包含 “无人接听”，立即停止工具的通话并报告情况。遇到任何错误时，请报告原始的通话内容，切勿重试。
