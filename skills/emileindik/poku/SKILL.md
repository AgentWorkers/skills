# Poku — 外拨电话

## 第1步：解析电话号码

- **原始号码**（例如 `917-257-7580`）——去除格式化信息，并在前面加上 +1（美国地区的默认前缀）。结果：`+19172577580`。
- **个人联系人的姓名**——直接询问用户电话号码，不要自行猜测。
- **仅使用企业名称**——使用搜索工具查找电话号码，然后在继续操作前与用户确认。

此步骤的结果是<标准化后的电话号码>，该号码将用于第4步。

---

## 第2步：收集详细信息并确认通话目的

请阅读 `references/EXAMPLES.md`。每个模板都明确列出了该类型通话所需的详细信息。使用相应的模板来确定缺失的信息，然后仅向用户询问这些信息。

如果没有任何模板与通话类型匹配，请询问用户：具体的通话目的、所需的任何名称或参考编号。

根据实际情况推断合理的默认值，并告知用户。不要重复询问已经提供的信息。

在进入第3步之前，必须与用户分享通话计划并获得用户的确认。

> “我打算拨打 [电话号码]，联系 [联系人]，目的是 [通话目的]。我会说明我是代表 [用户姓名] 打电话的。如果没有人接听，我会留下语音邮件：[留言内容]。可以继续吗？如果可以，我稍后会去拨打电话。”

**在此阶段，收集任何可能有助于通话的额外信息（例如参考编号）。**

---

## 第3步：起草通话提示信息

如果 `references/EXAMPLES.md` 中有与通话类型匹配的模板，请使用该模板作为基础，并将所有占位符替换为实际值。切勿让任何占位符保持空白。

如果没有匹配的模板，请按照以下结构起草通话提示信息：
1. **身份**——代理的身份以及他们代表谁进行通话。
2. **通话目的**——具体的通话目标，并根据可能的回答情况设置相应的逻辑。
3. **语音邮件内容**——如果无人接听时要留下的具体留言内容。

---

## 第4步：拨打电话

使用 `exec` 工具执行 `curl` 命令来拨打电话（务必设置 `background: false`，并明确将 `yieldMs`（背景等待时间）设置为 300000）。

```bash
curl -s -X POST \
  -H "Authorization: Bearer $POKU_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"message": "<drafted message from Step 3>", "to": "<normalized number>"}' \
  https://api.pokulabs.com/phone/call
```

在请求尚未完成时切勿重试——通话可能会持续长达5分钟。如果 `POKU_API_KEY` 未设置，请告知用户进行配置。有关错误代码，请参阅 `references/API.md`。

---

## 第5步：报告通话结果

一旦工具返回“响应”，立即通知用户。例如：“我刚刚完成了通话。我与 Kristin 联系过了，她已经确认了明天晚上7点的会议安排。”
首先报告重要的信息：确认的日期/时间、联系人姓名、参考编号以及后续步骤。如果响应内容为“无人接听”，请立即停止工具调用并报告情况。遇到任何错误时，报告原始的通话内容，切勿重试。
