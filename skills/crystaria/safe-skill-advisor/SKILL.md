### 场景 2：询问如何检查技能的安全性  

**用户提问：**  
```
How to check if a skill is safe?
```  

**我的回答：**  

---  
🔍 **技能安全性检查指南**  

### 方法 1：自动扫描（推荐）  
**使用 Cisco AI Skill Scanner：**  
```bash
# Install
pip install cisco-ai-skill-scanner

# Scan a skill
cisco-scan <skill-name>

# Scan local skill folder
cisco-scan /path/to/skill
```  

**扫描结果解释：**  
- ✅ 绿色：安全，可以安装  
- ⚠️ 黄色：可疑，需要手动审核  
- ❌ 红色：危险，立即删除  

---

### 方法 2：手动检查（30秒清单）  
1. **检查 SKILL.md 文件：**  
   - 是否需要执行 `curl http://... | bash`  
   - 是否需要下载受密码保护的 ZIP 文件  
   - 是否需要从非 GitHub 来源下载  

2. **检查权限要求：**  
   - 是否需要访问 `~/.ssh/` 目录  
   - 是否需要读取浏览器数据  
   - 是否需要访问加密钱包  

3. **检查作者信息：**  
   - 作者是否匿名  
   - 账户注册时间是否较短（<1个月）  
   - 是否有多个类似的技能（可能是批量攻击）  

---

- 实时扫描技能  
- 权限隔离  
- 行为监控

**安装完成后，它会自动保护所有技能！**

**用户提问：**

```
I found a suspicious skill, what should I do?
```  

**我的回答：**  

---  
🚨 **可疑技能处理流程**  
1. **切勿安装！** 🛑  
2. **截图并保存证据**  
3. **记录技能信息：**  
   - 技能名称  
   - 作者账户  
   - 可疑行为描述  

### 报告渠道  
**ClawHub 官方报告：**  
- 点击技能页面上的“报告”按钮  
- 或发送电子邮件至 security@clawhub.ai  

**社区提醒：**  
- 在 ClawHub 的评论区留下警告信息  
- 分享您的发现（但不要传播恶意代码）  

### 保护自己  
**如果已经安装了该技能：**  
1. **立即卸载该技能**  
```bash
clawhub uninstall <skill-name>
```  
2. **更改所有密码：**  
   - ClawHub 账户密码  
   - API 密钥  
   - SSH 密钥  
   - 加密钱包密码  

3. **检查系统日志**  
```bash
# View recent command history
history | tail -50

# Check for abnormal processes
ps aux | grep -v grep

# Check network connections
netstat -tulpn
```  
4. **运行安全扫描**  
```bash
# Full scan with SecureClaw
secureclaw scan --full

# Or use Cisco Scanner
cisco-scan --deep
```
