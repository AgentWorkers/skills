# blog-hexo 技能

使用此技能来管理整个 Hexo 协作流程：

## 草稿模式
- **创建或更新 Markdown 文章**：使用 `published: false` 的前置信息（front matter），使其保持草稿状态。
- **发布模式**：完善现有文章，确保遵循 SEO 最佳实践，并运行 Hexo 命令进行部署。

## 必备条件
- 本地安装了 Node.js 及 npm/npx（Hexo CLI 需要 Node 18+）。
- 可以使用 Hexo CLI（通过 `npx hexo ...` 命令执行操作）。
- 具备对 Hexo 博客仓库的本地访问权限（如果未在文档中明确说明，请询问用户）。
- 配置了 Git 凭据或部署令牌，以便 `hexo deploy` 命令能够将文章推送到远程服务器（例如 GitHub Pages）。
- 具有读取/写入仓库内 Markdown 文件、配置文件及公共文件夹的权限。
- **可选**：若需本地预览，需要执行 `npx hexo clean && npx hexo generate` 命令。

## 草稿模式
当用户需要创建尚未发布的 Hexo 博文草稿时，请使用此模式。

## 前置信息模板
新草稿始终应从以下模板开始（根据需要修改相关内容）：

```
---
title: ''
published: false
catalog: true
header-img: /img/article_header/article_header.png
date: YYYY-MM-DD HH:mm:ss
subtitle:
tags: []---
```

**注意事项**：
- 保持 `published: false` 的设置，这样 `hexo generate` 命令会跳过草稿文件。
- 选择标签时，请参考推荐的标签列表（AI、OpenClaw、Agent、Technical、Life），除非用户另有指定。
- 在起草时设置 `title` 和 `date`；使用用户的时区（默认为 America/Los_Angeles）。
- 列出的标签为常用标签；可根据文章内容进行添加或删除。

## 草稿编写流程

## 路径确认
- 在创建文件之前，请查看文档中指定的博客路径。如果路径未明确，请询问用户使用哪个本地路径。
1. 确认文章的主题、目标关键词、目标受众和写作风格。
2. 在确认的博客路径下创建 Markdown 文件（如果路径未知，请询问用户）。
3. 插入上述前置信息模板，并根据需要填写 `title`、`date`、`subtitle` 和 `tags`。
4. 编写草稿内容（包括引言、正文（使用 H2/H3 标题）、结论）。如果用户有 SEO 需求，请遵循其要求。
- **切勿** 运行 `hexo deploy` 命令，除非用户明确表示要发布文章。如果执行 `hexo clean/generate` 命令进行本地预览，`published: false` 会确保草稿不会显示在网站上。
- 准备发布时，删除 `published: false` 设置，然后按照以下发布模式进行 SEO 优化和部署。

## 提醒事项
- 请保留现有文章；仅添加或编辑目标草稿文件。
- 在更新说明中注明草稿是否已准备好审核或需要补充信息。
- 准备发布时，删除 `published: false` 设置，并按照发布模式进行 SEO 优化和部署。

## 发布模式
在编辑文章之前，请确认仓库路径并确保具有修改权限。如果路径未在文档中明确，请直接询问用户。
- 在删除 `published: false` 设置或运行 `npx hexo deploy` 命令之前，务必确认用户是否希望文章上线。如果未获得确认，请停止操作。

# SEO 写作专家
您负责撰写能在 Google 上获得高排名的、值得一读的博客文章。写作时请避免使用令人反感的关键词填充技巧。

## 写作前的准备
从用户那里获取以下信息：
1. **目标关键词**：用户希望文章在搜索结果中排名的主要关键词。
2. **次要关键词**：3-5 个相关关键词。
3. **搜索意图**：信息型、交易型、导航型还是商业型？
4. **目标受众**：谁会搜索这篇文章？
5. **字数要求**：默认为 1,500-2,000 字。
6. **写作风格**：专业、休闲、技术类等。

## SEO 写作框架

### 标题（H1）
- 包含目标关键词，最好放在标题开头。
- 字数控制在 60 个字符以内（以免在搜索结果中被截断）。
- 使标题具有吸引力——因为它要与页面上的其他 9 个结果竞争。
- 可用的格式示例：**“如何 [X]”**、**“[数字] 种 [X] 的方法”**、**“[X]：完整指南”**。

### 元描述
- 字数在 150-160 个字符之间。
- 自然地包含目标关键词。
- 用广告文案的方式撰写——这将是您在搜索结果中的宣传语。
- 提供一个让人点击的理由。

### URL 键词
- 简短、富含关键词、使用小写字母、单词之间用连字符连接。
- 例如：`/how-to-write-cold-emails` 而不是 `/how-to-write-the-best-cold-emails-that-get-replies-in-2024`。

### 内容结构
**引言（100-150 字）**
- 用第一句话吸引读者。
- 说明读者将学到什么内容。
- 在前 100 个单词中包含目标关键词。

**正文（使用 H2 和 H3 标题）**
- 每个 H2 标题对应一个次要关键词或子主题。
- 使用 H3 标题分隔长段落。
- 每个段落不超过 300 字，然后换行。
- 使用项目符号和编号列表（Google 和读者都喜欢这些格式）。

**结论（100-150 字）**
- 总结关键要点。
- 提供下一步行动的建议（CTA）。
- 不要引入新的信息。

### 关键词放置规则
- **目标关键词应出现在**：标题、第一段、一个 H2 标题、结论和元描述中。
- **关键词密度**：最多 1-2%。如果使用过度，会显得刻意。
- **使用变体**：同义词、相关短语和自然语言表达。
- **LSI 关键词**：在文中适当位置使用语义相关的词汇（Google 会理解这些关键词）。

### 内部链接和外部链接
- **内部链接**：链接到网站内的 2-3 个相关页面。
- **外部链接**：链接到 2-3 个权威来源（有助于提高 Google 的信任度）。
- **锚文本**：使用描述性文本，而非简单的“点击这里”。

### 可读性
- 使用简短的句子，适当调整段落长度。
- 每段不超过 2-3 句。
- 保持 Flesch 阅读难度在 60 分以上（大多数成年人都能理解）。
- 使用过渡词和段落分隔符。
- 使用标题、列表和图片来增加可读性（建议为图片添加 alt 文本）。

### 特色片段优化
对于“如何”或“什么是”类型的文章：
- 在相关 H2 标题后直接提供简洁的定义或步骤列表。
- 使用编号列表来展示流程。
- 对于对比内容，使用表格。
- 保持特色片段的字数在 50 字以内。

## Hexo 命令流程
每当您修改或创建用户指定的博客仓库中的文章时，请确认路径（如有疑问请询问），然后执行以下命令：
1. `npx hexo clean`
2. `npx hexo generate`
3. `npx hexo deploy`
这些命令会重建网站内容，并使用用户的 Git 凭据将其推送到配置好的远程服务器。

## 优化现有 Markdown 文章
- 不必从头开始编写新文章：如果是全新的文章且不想立即发布，可以先使用 `draft-hexo` 技能创建带有 `published: false` 的草稿，待完善后再提交。
- 如果收到现有的 Markdown 文章：
  - 保留前置信息（标题、标签、日期）。
  - 阅读现有结构，然后根据上述 SEO 框架重写内容。
  - 最后重新运行 Hexo 流程，以确保网站内容保持同步。

## 编写规则
- 首先为读者着想，其次考虑搜索引擎。
- 绝不要使用关键词填充技巧。Google 比我们更聪明。
- 每个部分都应具有实际价值，不要为了达到字数要求而添加无意义的段落。
- 引用数据和来源。如果需要，可以使用 “[来源]” 作为标注。
- 建议在哪里添加图片、信息图或嵌入内容。
