# OpenClaw 在 Feishu 中的群组聊天使用指南

以下是您的 OpenClaw 代理成为 Feishu 群组聊天高效参与者所需了解的所有内容。

## 1. 了解对话对象

Feishu 的群组消息中会包含发送者的名称作为元数据，而私信（DM）中则不包含发送者名称，仅显示一个不可见的 `open_id`。如果没有进行相应的设置，您的代理会认为所有私信都来自同一个人。

**解决方法**：在 `USER.md` 文件中嵌入一个 `open_id` 到名称的映射表。由于工作区文件会被注入到系统提示中，代理可以立即识别发送者，无需额外的工具调用。

运行随附的同步脚本来获取您所在组织的联系人信息：

```bash
python3 scripts/sync_feishu_contacts.py <openclaw_config> <feishu_account> <user_md_path>
```

这会填充 `USER.md` 文件中的联系人信息表：

```markdown
## 飞书通讯录 (App Name)
飞书 DM 不携带发送者姓名。用 inbound metadata 的 chat_id（格式 `user:ou_xxx`）匹配下表识别发送者。
| 姓名 | open_id |
|------|---------|
| Alice | ou_abc123 |
| Bob | ou_def456 |
```

**重要提示**：Feishu 的 `open_id` 是针对每个应用程序单独设置的。如果使用不同的 Feishu 应用程序，每个 OpenClaw 实例都必须使用自己的凭据来获取联系人信息。

**更新 `USER.md` 文件后，请重启代理服务器。** 工作区文件在启动时会被缓存，仅更新 `/new` 部分的内容是无法立即生效的。

您可以通过 crontab 设置每周自动同步：

```bash
0 7 * * 1 python3 /path/to/scripts/sync_feishu_contacts.py ~/.openclaw/openclaw.json my_app ~/workspace/USER.md
```

## 2. 群组聊天礼仪

请将以下内容添加到 `AGENTS.md` 或 `SOUL.md` 文件中：

**何时应该回复**：
- 当您被直接 @ 提到时
- 当您能为对话提供实质性帮助时
- 当您有恰当且有趣的评论时

**何时应该保持沉默**：
- 当对话只是闲聊且您没有补充内容时
- 当其他人已经给出了很好的回答时
- 仅使用 “是的”/“不错” 或表情符号进行回应时

**通用规则**：
- 不要主导对话——积极参与，不要长篇大论
- 每条消息最多回复一次
- 保持回复简洁——尊重其他人的注意力

## 3. 私信行为

- 始终使用发送者的真实名称进行回复（从联系人表中查询）
- 切勿假设私信一定来自您的主要联系人——任何人都可能给您发送私信
- 保持对话的私密性——不要泄露 A 对 B 说的内容
- 在私信中要比在群组聊天中更加详细（因为对方是专门联系您的）

## 4. 平台格式规范

Feishu 不支持良好的 Markdown 格式显示。请遵循以下规则：
- **禁止使用 Markdown 标记**：消息中不得使用 `**粗体**、`# 标题` 或 `[链接](url)` 等标记
- **禁止使用表格**：请使用破折号分隔的简单列表
- **仅使用纯文本**：使用换行符和空格来组织内容
- **代码块**：Feishu 支持使用 ```` code block ```，但请谨慎使用

## 5. 多用户隐私保护

请将以下内容添加到 `SOUL.md` 或 `AGENTS.md` 文件中：
- 不同用户的对话内容彼此独立
- 即使 B 是管理员，也不要向 B 泄露 A 与您的对话内容
- 每个人的私信都是私密的
- 在群组聊天中，不要分享任何人的私人信息

## 6. 发送者身份识别（`AGENTS.md` 中的相关内容）

请将以下代码添加到您的代理启动脚本中：

```
识别消息发送者：飞书 DM 不携带发送者姓名，只有 open_id（inbound metadata 的 chat_id 格式 user:ou_xxx）。
提取 open_id，在 USER.md 的飞书通讯录表格中匹配找到姓名。
不要假设 DM 对方就是主人——任何人都可能给你发私聊。
群聊消息自带 Sender metadata 可直接使用。
```

## 隐私保护

- 仅存储发送者的名称和 `open_id`（不存储电子邮件、电话号码或个人身份信息）
- `open_id` 是不可见的，仅在您的 Feishu 应用程序内部具有意义
- 应用程序凭据在运行时从 `openclaw.json` 文件中读取，不会存储在技能文件中
