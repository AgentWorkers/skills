# Job Hunter

这是一个端到端的求职助手工具，帮助用户从寻找职位机会到成功获得面试机会。

## 快速入门

### 1. 设置候选人资料

为用户创建一个 JSON 格式的个人资料。以 `{baseDir}/references/profile-template.json` 作为模板。请用户提供以下信息：
- 目标职位和职位级别
- 关键技能和使用的工具
- 地点偏好（城市或远程工作）
- 薪资期望
- 不接受的公司或行业
- 偏好的行业/领域

将生成的文件保存为 `profile.json`，放在工作区中。

### 2. 搜索职位

使用 `web_search` 工具通过多个查询来扩大搜索范围：

```
site:linkedin.com/jobs "[role]" "[city]"
site:indeed.com "[role]" "[city]"  
site:glassdoor.com/job "[role]" "[city]"
"[role]" "[city]" hiring 2025 2026
```

**扩展关键词**——不要只搜索职位名称。请参考 `{baseDir}/references/search-strategies.md` 以获取关键词扩展的技巧。

**替代方案**：如果 Brave API 可用，可以运行搜索脚本：
```bash
{baseDir}/scripts/search_jobs.sh "CX Manager" --location "Amsterdam" --days 7
```

### 3. 评估匹配度

对于找到的每个职位，进行匹配度分析：
```bash
python3 {baseDir}/scripts/analyze_fit.py --profile profile.json --jobs jobs.json --threshold 50
```

或者使用以下框架进行手动评估：
- **技能匹配**（40%）：用户是否具备职位要求的 60% 以上技能？
- **职位级别匹配**（25%）：职位级别是否合适（既不过高也不过低）？
- **地点匹配**（15%）：工作地点是否符合用户要求或是否支持远程工作？
- **行业匹配**（10%）：职位是否属于用户偏好的行业？
- **风险因素**（10%）：是否存在用户明确拒绝的公司或行业？

评分标准：🟢 75 分以上：非常匹配 | 🟡 55-74 分：匹配良好 | 🟠 40-54 分：勉强匹配 | 🔴 40 分以下：跳过

### 4. 展示搜索结果

对于每个职位，展示以下信息：
- **职位名称及公司名称**（附带直接链接）
- **匹配度评分**（用颜色表示）
- **匹配理由**（列出用户技能与职位要求的最高匹配项）
- **需要弥补的差距**（列出用户缺失的技能，并标注为“渴望学习的内容”）
- **薪资预估**（如果有的话）
- **建议**：申请 / 可以申请 / 跳过

## 申请支持

### 撰写求职信

请阅读 `{baseDir}/references/cover-letter-guide.md` 以了解求职信的结构和写作指南。撰写个性化的求职信，内容包括：
- 提及具体的公司信息（避免使用通用模板）
- 将用户的经验与职位要求的 2-3 项关键技能对应起来
- 包括可量化的成就
- 保持求职信字数在 350 字以内

### 面试准备

请阅读 `{baseDir}/references/interview-prep.md` 以获取全面的面试准备指南。帮助用户完成以下工作：
- 公司背景研究
- 为关键技能准备具体的案例故事
- 编写个性化的自我介绍内容
- 准备薪资谈判的策略
- 准备面试时可以向面试官提出的问题

### 薪资调研

```bash
bash {baseDir}/scripts/salary_research.sh "Job Title" "Location"
```
请参考至少 3 个来源获取薪资信息。在荷兰，薪资计算时需要考虑 8% 的假期津贴和可能的第 13 个月奖金。

## 每日工作简报格式

当以定时任务的形式运行求职助手时，每日简报应包含以下内容：
- **新发现的机会**：过去 24 小时内找到的职位，附带匹配度和直接链接
- **申请状态**：待处理申请的最新进展
- **行动事项**：今天需要申请哪些职位，以及需要跟进的事项
- **市场动态**：行业趋势、薪资变化和招聘情况

## 跟踪记录

维护一个职位跟踪记录，包括以下信息：
- 公司名称、职位名称、发现日期、信息来源
- 匹配度和推荐建议
- 状态：`新发现` → `已申请` → `筛选中` → `面试中` → `收到录用通知`/`被拒绝`/`被忽略`
- 申请情况（是否已申请及原因）
- 联系信息和后续跟进日期

## 给代理人的建议

- **切勿代表用户直接申请职位**——让用户自己做出决定
- **不要信息过载**——3-5 个高质量的匹配机会比 20 个质量一般的职位更有价值
- **记录用户拒绝的公司**——被拒绝后不要再推荐同一家公司
- **如实评估匹配度**——即使匹配度较低，也要如实告知用户
- **尊重用户的拒绝理由**——如果用户明确表示不接受某个公司，即使该公司很优秀也不要再推荐
- **定期更新个人资料**——根据用户的最新需求调整个人资料
- **庆祝成功**——用户成功申请职位或获得面试机会时，要及时给予反馈
