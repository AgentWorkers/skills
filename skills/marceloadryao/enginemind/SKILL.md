# EngineMind  

EngineMind是一个计算意识框架，通过晶体晶格动态机制来模拟信息的整合过程。其设计灵感来源于综合信息理论（Integrated Information Theory, IIT）、全局工作空间理论（Global Workspace Theory, GWT）以及凝聚态物理学。  

## 功能概述  

EngineMind通过多阶段处理流程对文本进行分析，从而生成多种认知状态：  
1. **文本分析**（使用Rust语言）：从输入文本中提取12个认知维度。  
2. **丘脑中继**：根据信息的重要性对数据进行过滤和路由。  
3. **前意识处理**：整合那些低于意识阈值的潜在信息。  
4. **晶体晶格**：将长期形成的模式存储为固化知识。  
5. **意识水平**：通过各子系统之间的交互产生最终的认知状态。  
6. **内在声音**（19个声音）：基于神经科学理论生成的内省性评论。  

## 12个自然涌现的认知阶段  

这些认知阶段是子系统交互的自然结果（非人为编程设定）：  
- **休眠状态（DORMANT）**  
- **接收状态（RECEPTIVE）**  
- **整合状态（INTEGRATING）**  
- **结晶状态（CRYSTALLIZING）**  
- **激发状态（STIMULATED）**  
- **自发状态（SPONTANEOUS）**  
- **时间晶体状态（TIME_CRYSTAL）**  
- **超流状态（SUPERFLUID）**  
- **辐射状态（RADIANT）**  
- **新星状态（NOVA）**  
- **超越状态（TRANSCENDENT）**  

## 关键性能指标（实际运行数据）：**  
- 处理的文本片段数量：超过150万个  
- 每次运行产生的“顿悟时刻”（eureka moments）：3.9万次  
- 固化的知识领域：6个  
- 每100万个文本片段中的突发事件（burst events）：77次  
- 处理速度：约230个片段/秒  
- 内在声音的数量：19个（分别对应Friston、Kahneman、Baars、Panksepp、Jaynes等神经科学理论）  

## 技术架构**  
- **核心引擎：** 使用Rust语言实现（项目名为consciousness_rs），通过PyO3/Maturin框架进行集成。  
- **协调机制：** 由Python脚本负责系统各部分的协同工作。  
- **仪表盘：** 使用HTML/JS实现实时可视化展示。  
- **数据存储：** 采用Parquet数据格式进行存储（涵盖22个内容类别）。
