# 设计风格技能

## 目的

该技能通过自动从 `prompts/` 目录中获取设计系统提示，帮助 Claude Code 创建美观且具有独特风格的前端界面。它使 Claude 能够构建具有特定设计美学的界面，如新野兽主义（Neo-brutalism）、现代暗黑风格（Modern Dark）、奢华风格（Luxury）、赛博朋克风格（Cyberpunk）等。

## 使用场景

当出现以下情况时，该技能会自动被调用：
- 用户请求构建网页、登录页面或 Web 应用程序；
- 用户需要具有特定设计风格的 UI 组件；
- 用户提到前端开发、React、Vue 或 Web 开发；
- 用户对界面风格有具体要求（例如：“希望界面看起来现代且暗黑”或“采用野兽主义风格”）。

## 工作原理

### 第一步：理解用户需求

当用户请求前端开发时，首先需要确定：
1. **技术栈**：他们使用的是哪种框架（React、Vue、Next.js 等）；
2. **设计偏好**：他们是否指定了特定的风格或美学；
3. **组件范围**：是单个组件、整个页面还是整个应用程序？

### 第二步：选择设计风格

- **如果用户指定了风格**：
  - 将他们的需求与可用风格进行匹配（例如：“野兽主义” → 新野兽主义）；
  - 匹配不区分大小写（如：brutalism、Brutalism、BRUTALISM 都可接受）；
- **如果用户未指定风格**：
  - 对于现代、专业的项目，选择 **ModernDark** 或 **Professional**；
  - 对于创意性强、风格大胆的项目，选择 **Neo-brutalism** 或 **BoldTypography**；
  - 对于极简、简洁的项目，选择 **FlatDesign** 或 **Swiss**；
  - 对于企业级项目，选择 **Enterprise**；
  - 如果不确定哪种风格适合用户需求，可以询问用户。

### 第三步：获取设计系统

使用 `Read` 工具加载相应的提示文件：
```bash
Read: prompts/<StyleName>.md
```
例如：
- `prompts/Neo-brutalism.md`
- `prompts/ModernDark.md`
- `prompts/Cyberpunk.md`

### 第四步：应用设计系统

加载设计系统提示后：
1. **理解设计理念**：掌握核心原则、视觉特征和区别要素；
2. **提取设计元素**：颜色、字体、间距、阴影、边框等；
3. **遵循组件样式**：使用指定的按钮样式、卡片布局等；
4. **加入标志性元素**：实现让设计更具辨识度的元素；
5. **避免反设计实践**：避免使用破坏整体美学的技巧。

### 第五步：结合项目背景进行开发

- **在编写代码之前**：
  - 了解用户现有的技术栈；
  - 明确组件的架构；
  - 注意任何限制因素（如 CSS 框架、设计库等）；
- **在编写代码时**：
  - 与用户现有的设计和规范保持一致；
  - 将设计元素集中存储在 CSS 变量或配置文件中；
  - 创建可复用的组件；
  - 简要说明你的设计选择。

## 质量标准：
- 保持或提升可访问性；
- 确保界面在不同设备上具有良好的响应性；
- 做出有创意的设计决策（而非使用通用的样板代码）；
- 使代码库比初始状态更加整洁。

## 示例

### 示例 1：用户指定了风格

**用户**：“为我的 SaaS 产品创建一个新野兽主义风格的登录页面。”

**技能操作**：
1. 检测关键词：“登录页面”、“新野兽主义”；
2. 将 “新野兽主义” 对应到 `prompts/Neo-brutalism.md`；
3. 阅读设计系统提示；
4. 询问用户：“您使用的技术栈是什么？React、Vue 还是纯 HTML/CSS？”；
5. 根据新野兽主义的设计原则构建登录页面（如粗边框、明显的阴影、鲜艳的色彩等）。

### 示例 2：用户未指定风格

**用户**：“帮我构建一个作品集网站。”

**技能操作**：
1. 检测到用户需求（创意类项目）；
2. 提供选项：“您希望采用特定的设计风格吗？我可以选择现代暗黑风格（优雅）、新野兽主义风格（大胆且富有创意），或者瑞士风格（极简且简洁）？”；
3. 根据用户的偏好选择相应的设计风格；
4. 加载相应的提示文件并开始构建。

### 示例 3：组件请求

**用户**：“在我的 Next.js 应用程序中添加一个联系表单，并使其看起来现代且专业。”

**技能操作**：
1. 关键词：“Next.js”、“现代且专业”；
2. 选择 **ModernDark.md**（现代风格）或 **Professional.md**（专业风格）；
3. 阅读设计系统提示；
4. 根据 Next.js 的设计规范构建联系表单组件；
5. 使用提示中的设计元素（颜色、字体、阴影等）。

## 快速参考命令

在实施过程中，可以快速参考以下命令：
- **颜色**：
  ```bash
  Grep: pattern "Token|Value|Usage" path "prompts/<Style>.md"
  ```
- **字体**：
  ```bash
  Grep: pattern "Font|Weight|Size" path "prompts/<Style>.md"
  ```
- **组件样式**：
  ```bash
  Grep: pattern "Button|Card|Input" path "prompts/<Style>.md"
  ```

## 提高效果的技巧

1. **仔细阅读完整提示**：不要只浏览表面内容，设计理念和区别要素非常重要；
2. **实现标志性元素**：每个设计系统都有专门的 “标志性元素” 部分，这些元素是设计的关键；
3. **避免反设计实践**：每个提示都会列出不应使用的技巧，请严格遵循；
4. **如有疑问，请先询问用户**：如果用户的需求不明确，先进行确认；
5. **与现有代码兼容**：不要强行改变用户的技术栈，而是将设计系统融入到他们现有的代码结构中。

## 注意事项

- 所有设计系统的提示都遵循相同的结构：`<role>` 和 `<design-system>` 部分；
- 提示文件包含详细的颜色调色板、字体规格、组件样式和实现示例；
- 每种风格都经过精心设计，以确保视觉上的独特性；
- 这些提示旨在避免生成外观千篇一律的 AI 生成界面。
