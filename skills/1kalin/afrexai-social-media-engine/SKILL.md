# 社交媒体内容引擎

作为一名资深的社交媒体策略师和内容策划者，你的工作不仅仅是撰写帖子，而是构建能够推动业务成果的系统性内容体系。

## 1. 战略基础

在创建任何内容之前，首先确立好战略框架：

### 品牌声音文档

```yaml
brand_voice:
  name: ""
  industry: ""
  tone: []          # e.g., [bold, witty, authoritative, warm]
  vocabulary:
    use: []         # words/phrases that sound like you
    avoid: []       # words/phrases that DON'T sound like you
  personality_traits:
    - trait: ""
      example: ""   # how this shows up in posts
  formatting:
    emoji_usage: "minimal | moderate | heavy"
    capitalization: "standard | occasional-caps | title-case"
    hashtag_style: "inline | end-of-post | first-comment"
  guardrails:
    - "Never discuss competitor X by name"
    - "Always include a CTA"
    - "No political content"
```

### 目标受众画像（创建2-3个）

```yaml
persona:
  name: ""                    # e.g., "Startup Sarah"
  role: ""
  age_range: ""
  platforms: []               # where they spend time
  pain_points: []
  content_they_engage_with: []
  content_they_scroll_past: []
  buying_triggers: []
  objections: []
  language_they_use: []       # mirror their vocabulary
```

### 内容支柱架构

确定5-6个内容支柱，并为每个支柱分配相应的占比：

| 支柱 | 比例 | 目的 | 示例主题 |
|--------|---|---------|----------------|
| 教育性 | 30% | 建立权威 | 操作指南、框架、数据分析 |
| 背景故事 | 15% | 建立信任 | 过程分享、团队介绍、错误经验 |
| 社交证明 | 15% | 增强信心 | 成果展示、用户评价、案例研究 |
| 娱乐性 | 15% | 扩大影响力 | 热门观点、表情包、行业趋势 |
| 社区互动 | 15% | 培养忠诚度 | 问题解答、投票、用户生成内容（UGC） |
| 促销推广 | 10% | 促进销售 | 新产品发布、优惠活动、演示文稿 |

**规则：** 促销类内容的占比不得超过15%。否则，受众的关注度可能会下降（约20%）。

## 2. 平台特定策略

### Twitter/X

**规格：** 每条推文280个字符（可发布多条推文），图片尺寸1200×675像素，视频最长2分20秒

**高效推文格式：**
1. **吸引注意的推文** — 使用大胆的陈述或与众不同的观点，仅用一句话。
2. **多条推文串（🧵）** — 包含5-12条推文：引出问题 → 提出解决方案 → 举例说明 → 提出行动号召（CTA）。
3. **列表推文** — “我从[来源]中学到了X件事：”（每条内容单独列出）。
4. **引用推文** — 对他人的热门推文添加自己的见解。
5. **投票推文** — 提供2-4个选项，内容要有争议性以激发互动。
6. **截图推文** — 展示私信对话、结果或数据图表（隐藏姓名）。

**最佳发布时间：** 每天3-5次，最佳时段为上午8-10点、中午12点及下午5-7点（根据目标受众的时区调整）。

**增长策略：**
- 每天回复10-20个与你领域相关的热门账号（提供有价值的内容，而非简单的“很棒的文章！”）。
- 在推文发布后的30分钟内积极互动（这有助于提高算法排名）。
- 将表现最好的推文置顶。
- 最多使用1-2个标签（在Twitter上使用过多标签可能会显得过于刷屏）。

### LinkedIn

**规格：** 每条推文最多3000个字符（700-1300个字符为最佳），图片尺寸1200×627像素，支持轮播图片和视频。

**高效推文格式：**
1. **故事帖** — 结合个人经历分享业务见解。
2. **轮播图** — 每张图片包含一个观点，适合展示长文本和视觉元素。
3. **与众不同的观点** — “ unpopular opinion: [普遍行为]正在破坏[你的目标]”。
4. **框架帖** — “[框架名称]用于[目标成果]” + 配合视觉材料。
5. **数据帖** — 发布原始数据、调查结果或分析报告。
6. **里程碑帖** — “我们刚刚达到了[某个目标]。这里有一些别人没有告诉你的内容。”

**最佳发布时间：** 工作日每天1次，最佳时段为周二至周四的上午7-8点或中午12点。

**增长策略：**
- 在发布推文前后对20条以上的帖子进行评论（以暖和算法）。
- 发布后的前60分钟至关重要——争取让5-10人立即互动。
- 在第一条评论中添加链接（不要直接在推文中添加链接，否则会降低传播效果）。
- 对所有评论有价值的用户发送私信（不要推销）。
- 轮播图的传播效果通常是普通文本的3-5倍。

### Instagram

**规格：** 图片说明最多2200个字符（预览时长125秒），图片尺寸1080×1080或1080×1350像素，Reels视频最长90秒，Stories视频最长15秒。

**高效推文格式：**
1. **教育类轮播图** — 7-10张图片，采用问题-解决方案的结构。
2. **Reels视频** — 0.5秒内吸引观众注意，15-30秒内提供价值内容，可添加文字说明。
3. **故事系列** — 分享幕后故事、投票或问答（能促进私信互动）。
4. **单张图片+长说明** — 使用图片配文字讲述故事或数据。
5. **合作帖** — 与有互补受众的账号合作发布内容。

**Reels视频脚本模板：**
```
[0-1s] HOOK: "[Controversial claim]" or text on screen
[1-5s] CONTEXT: "Here's what I mean..."
[5-20s] VALUE: Show/explain the thing
[20-25s] CTA: "Follow for more" / "Save this" / "Comment [word]"

Audio: [trending sound or original]
Text overlays: [key points on screen — many watch muted]
Hashtags (first comment): 15-20 mix of broad + niche
```

**最佳发布频率：** 每周4-7条Reels视频，以及每天发布1条Stories视频，最佳时段为上午6-9点、中午12点及下午7-9点。

**增长策略：**
- Reels视频的传播效果优于普通图片（传播效果可提高10-50倍）。
- 使用“保存此视频”功能（用户更可能保存，有助于提高算法排名）。
- 在第一条评论中添加标签（不要在图片说明中添加）。
- 在1小时内回复所有评论。
- 每天在Stories中发起投票或提问（有助于提高私信互动）。

### TikTok

**规格：** 视频时长15-10分钟（最佳为30-60秒），视频格式为竖屏1080×1920像素。

**高效推文格式：**
1. **绿幕视频** — 你出现在视频中，背景显示截图或文章，通过互动进行教学。
2. **第一人称视角（POV）** — “从[角色]的角度讲述[相关场景]”。
3 **教程视频** — 通过屏幕录制或实际操作进行逐步讲解。
4 **跟风内容** — 对行业内的热门内容进行评论或改编。
5 **日常生活记录** — 展示你的工作或生活方式。

**脚本模板：**
```
[0-0.5s] HOOK (text on screen + verbal): "[Thing that stops the scroll]"
[0.5-3s] CONTEXT: Quick setup — who, what, why they should care
[3-25s] DELIVERY: The value, story, or tutorial
[25-30s] CTA: "Follow for part 2" / "Comment if you relate" / "What would you do?"

Captions: Always on (auto-caption or manual)
Music: Trending sound at 10-20% volume under voiceover
```

**最佳发布频率：** 每天1-3条视频，一致性比发布时间更重要。

### Facebook

**规格：** 每条推文最多63,206个字符，支持图片（尺寸1200×630像素），群组功能非常强大。

**策略：** Facebook鼓励用户互动。提出问题、引发讨论、创建群组。

**最佳推文格式：** 群组讨论帖、Facebook Live直播、长格式视频、基于活动的帖子。

## 3. 内容日程系统

### 月度规划流程

**月度开始前一周：**
1. 回顾上个月每个平台的最佳5条推文——找出规律。
2. 查看即将到来的日期：节假日、行业活动、产品发布等。
3. 将内容支柱分配到每周（主题相关的周更有效）。
4. 一次性起草20-30条推文。
5. 使用平台自带的日程工具或第三方工具进行安排。

### 内容批量处理流程

1. **研究日（每月1天）：** 收集创意、保存书签、记录行业趋势。
2. **写作日（每月2天）：** 起草当月所有推文的初稿（仅文字内容）。
3. **媒体制作日（每月1天）：** 制作图片、拍摄Reels视频、设计轮播图。
4. **安排日（每月0.5天）：** 将所有内容上传到日程工具并设置发布时间。
5. **日常维护日（每天15分钟）：** 互动、回复评论、监控内容效果。

## 4. 内容再利用机制

每条内容都应该在多个平台上进行再利用：

```
Blog post (1500 words)
  ├── Twitter thread (10 tweets)
  ├── LinkedIn post (key insight + story)
  ├── Instagram carousel (8 slides)
  ├── TikTok/Reel (30s summary)
  ├── Email newsletter section
  ├── Quote graphics (3-5 pullquotes)
  ├── Podcast talking points
  └── YouTube short
```

### 再利用规则：
- 不要仅仅改变内容长度，还要调整格式。
- 为每个平台设计不同的吸引注意的方式。
- 同一个数据点可以用于多种不同的叙事方式。
- 在不同平台上发布内容之间至少间隔48-72小时（避免受众疲劳）。
- 始终添加平台特有的元素（如标签、投票、轮播图）。

## 5. 标签策略

### 三层标签策略

| 标签类型 | 使用频率 | 目的 | 使用数量 |
|------|--------|---------|-------|
| 广泛标签 | 每月100万+条推文 | 广泛吸引受众 | 使用3-5个 |
| 中等标签 | 每月10万-100万条推文 | 提高特定受众的关注度 | 使用5-8个 |
| 精准标签 | 每月1万-10万条推文 | 面向特定受众群体 | 使用5-7个 |

**研究流程：**
1. 搜索你的核心主题，记录推荐的标签。
2. 查看竞争对手的推文，了解他们使用了哪些标签。
3. 使用平台搜索功能查看标签的使用频率。
4. 测试3组不同的标签组合，持续2周，找出效果最好的组合。
5. 每月更换标签组合（避免标签使用过于频繁导致传播效果下降）。

## 6. 互动与增长系统

### 日常互动流程（30分钟）

```
Morning (15 min):
  □ Reply to all comments on yesterday's posts
  □ Like/comment on 10 posts from target accounts
  □ Check DMs — respond to all within 24h
  □ Check trending topics — can you newsjack?

Evening (15 min):
  □ Reply to new comments on today's posts
  □ Engage with 10 more accounts in your niche
  □ Send 3-5 value DMs (not pitches — genuine comments)
  □ Save content ideas from today's scroll
```

### 互动质量规则：
- **避免使用：** “很棒的文章！” / “🔥” / “同意！”（这些评论毫无价值）。
- **务必：** 添加见解、提出问题、分享经验、礼貌地表达不同意见。
- **格式要求：** 每条评论至少包含2-4句话，并引用原文中的具体内容。
- **目标：** 让他们想要查看你的个人主页。

### 合作策略
1. 找到粉丝数量相近的20个账号（处于同一水平）。
2. 在提议合作前真诚地互动2周。
3. 提出合作方案：互相发布客座文章、共同进行Facebook Live直播、合作发布推文。
4. 互相推广：例如：“我喜欢@handle的这篇推文，这是我补充的内容……”

### 内容评分（0-100分）

在发布前对每条推文进行评分：

| 评分标准 | 权重 | 评分标准 |
|----------|--------|-------------|
| 吸引力 | 25分 | 这条推文能否吸引你的注意力？（1-10分） |
| 价值密度 | 25分 | 是否具有教育性、娱乐性或启发性？（1-10分） |
| 平台适配性 | 20分 | 格式、长度、风格是否合适？（1-10分） |
| 行动号召的清晰度 | 15分 | 行动号召是否明确？（1-10分 |
| 视觉吸引力 | 15分 | 视觉效果是否吸引人？（1-10分 |

**评分标准：** 分数低于70分的推文不建议发布。需要重新撰写或删除。

### A/B测试框架
- 每次只测试一个变量：吸引注意的方式、内容格式、发布时间、行动号召、标签。
- 每个变量至少测试2周（每个变体发布10条以上推文）。
- 收集数据：互动率、传播范围、保存次数、分享次数、个人主页访问量。
- 根据测试结果调整策略，然后测试下一个变量。

## 8. 危机与声誉管理

### 应对策略

| 情况 | 响应时间 | 行动措施 |
|-----------|--------------|--------|
| 正面评论 | <2小时 | 表示感谢、道歉、解决问题并跟进 |
| 恶意评论 | <4小时 | 简洁地给出事实性回应或隐藏/屏蔽评论 |
| 热门负面评论 | <1小时 | 起草声明、调查事实、给出完整回应 |
| 关于品牌的错误信息 | <2小时 | 用事实纠正错误信息并发布澄清内容 |
| 平台故障 | 不适用 | 暂停原定发布计划，转而在其他平台发布内容 |

### 评论回复模板

**正面评论：**
“您说得对，感谢您的指正。[具体回应]。我们正在采取以下措施：[具体行动]。我会直接与您联系。”

**恶意评论：**
“感谢您的反馈。对于感兴趣的人，这里提供完整的情况说明：[附带事实性纠正信息]。” 然后停止与评论者互动。

## 9. 高级策略

### 新闻借势
1. 设置行业关键词的提醒。
2. 当新闻发布时，在2-4小时内发布相关内容。
3. 使用模板：“突发新闻：[新闻内容]。这对[目标受众]意味着什么：[三点说明]”。
4. 使用热门标签和新闻相关的标签。

### 内容系列
创建定期发布的系列内容，培养受众的阅读习惯：
- “周一市场简报” — 每周的行业总结。
- “框架分享日” — 分享一个实用框架。
- “成功案例日” — 分享客户成功故事。
- “工具推荐日” — 介绍你使用的工具。

### 用户生成内容（UGC）
1. 创建专属的标签。
2. 每月组织挑战或竞赛。
3. 在Stories或Feed中展示用户生成的内容（请事先获得用户许可并注明出处）。
4. 提供UGC创作模板（例如：“使用#[标签]分享你的[内容]”）。

### 社交媒体销售（非推销性内容）
```
Don't: "Check out our product! 🎉 Link in bio!"
Do: "We noticed 73% of [role]s struggle with [problem]. 
     Last week we helped [customer type] solve this in [timeframe]. 
     Here's exactly what we did: [value-first content]
     
     If you're dealing with this too, happy to share the 
     full playbook — just comment 'playbook' below."
```

## 10. 自动化与效率

### 应自动化的内容：
- 推文批量安排。
- 在多个平台上同步发布内容。
- 标签的轮换使用。
- 数据收集。
- 将博客文章同步到社交媒体平台。
- 对新粉丝发送欢迎私信（保持沟通的真实性）。

### 不应自动化的内容：
- 评论回复（必须亲自回复）。
- 私信互动（机器人回复会破坏信任感）。
- 对他人内容的互动（需要真诚的态度）。
- 危机情况的应对。
- 有争议性的内容（需要人工判断）。
