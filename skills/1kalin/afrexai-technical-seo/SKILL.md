# 技术SEO精通

我们提供全面的技术SEO审计、修复和监控服务，涵盖从网站的可爬取性到核心网页性能（Core Web Vitals），以及国际SEO优化等所有搜索引擎所需的关键要素，以确保您的网站能够被正确地发现、爬取、索引和排名。

## 使用场景

- 需要对网站进行技术SEO审计
- 为什么Google没有索引我的页面？
- 需要修复核心网页性能问题
- 迁站前的SEO检查清单
- 网站加载速度慢/排名下降
- 需要检查robots.txt文件、站点地图（sitemap）和结构化数据标记（schema markup）

获取`[domain]/robots.txt`文件并评估其内容：

**常见错误：**
| 错误 | 影响 | 修复方法 |
|---------|--------|-----|
| `Disallow: /` 阻止所有页面被索引 | 无页面被索引 | 移除或缩小禁止范围 |
| 阻止CSS/JS文件被爬取 | 会导致页面渲染不良，从而影响排名 | 允许`/assets/`等路径被爬取 |
| 未在robots.txt中引用站点地图 | 爬取速度变慢 | 添加`Sitemap:`指令 |
| 未声明多个站点地图 | 部分页面无法被爬取 | 声明所有站点地图 |
| 错误地阻止了搜索或过滤页面的爬取 | 造成不必要的爬取负担 | 使用`Disallow: /*?`等模式进行限制 |

### 3.2 常见的索引障碍

| 障碍因素 | 检测方法 | 修复方法 | 优先级 |
|---------|-----------|-----|----------|
| `noindex`元标签 | 检查`<meta name="robots">`标签 | 移除该标签或将相关页面设置为可索引 |
| `X-Robots-Tag: noindex`头部信息 | 检查HTTP响应头部 | 从服务器配置中移除该头部信息 |
| 内容指向了错误的规范页面 | 检查`<link rel="canonical">`标签 | 修正指向正确的页面 |

| 页面没有内部链接（孤立页面） | 检查并添加内部链接 |
| 软404错误（200状态码但页面内容为空） | 检查页面内容 | 返回正确的404错误或添加内容 |

## 第6阶段：HTTPS与安全

**如果尚未启用HTTPS，请参考以下迁移 checklist：**
1. 获取SSL证书（Let's Encrypt是免费选项）
2. 在服务器或CDN上安装SSL证书
3. 将所有内部链接更新为HTTPS
4. 使用301重定向将所有HTTP链接重定向到HTTPS
5. 更新所有URL的协议为HTTPS
6. 更新站点地图中的URL
7. 更新Google搜索控制台中的相关设置
8. 更新robots.txt文件中的站点地图相关设置
9. 检查页面上是否存在混合内容（HTTP资源与HTTPS资源共存）
10. 确保所有设置无误后，启用HSTS头部信息

## 第8阶段：URL结构与网站架构

**网站架构的最佳实践：**
- 平面结构：从首页出发，所有页面都在3次点击范围内可访问
- 使用“中心辐射”式架构：分类页面链接到所有子页面，子页面也相互链接
- 在每个页面上实现路径导航（使用`BreadcrumbList`结构化数据）
- 使用`rel="next"`/`rel="prev"`进行分页，或通过渐进式增强（progressive enhancement）实现无限滚动
- 根据需要使用分面导航（faceted navigation）并设置相应的规范链接

## 第9阶段：国际SEO优化

### 9.1 hreflang设置

**hreflang设置模板：**

**规则：**
- 每个页面都必须引用所有语言版本，包括自身
- 必须包含`x-default`（用于处理未匹配的语言）
- 使用ISO 639-1编码语言代码，ISO 3166-1编码地区代码
- hreflang设置必须是双向的（如果A页面引用了B页面，B页面也必须引用A页面）
- 规范链接（canonical link）和hreflang设置必须一致——规范链接应指向相同语言版本的页面

### 9.2 国际化目标优化 checklist：

- 确保所有页面都设置了正确的hreflang（双向链接）
- 指定了`x-default`属性
- 确保内容已经过专业翻译
- 使用目标语言的货币、日期格式和电话号码
- 在搜索控制台（Search Console）中设置地理定位
- 在目标国家部署本地服务器或CDN节点
- 从目标地区获取本地链接
- 为不同语言提供专门的站点地图或站点地图章节

## 常见的技术SEO错误

| 编号 | 错误 | 重要性 | 修复方法 |
|---|---------|---------------|-----|
| 1. 阻止测试环境的页面被索引，且在上线后忘记解除限制 | 会导致没有页面被索引 | 在上线前检查robots.txt文件和`meta robots`设置 |
| 2. 使用302重定向而非301重定向 | 会导致链接权重分散 | 应使用301重定向 |
| 网站有多个首页版本（如www和非www、HTTP和HTTPS版本、带有 trailing slash的版本） | 会导致链接权重分散 | 选择其中一个规范版本并重定向其他版本 |
| 孤立页面没有内部链接 | 会导致爬虫无法找到这些页面 | 为这些页面添加内部链接 |
| 站点地图中包含不可索引的页面 | 会导致索引混乱 | 仅索引可被爬取的页面 |
| 图片缺少alt文本 | 会影响可访问性和SEO效果 | 为所有图片添加描述性alt文本 |
| 部署后未监控核心网页性能指标 | 可能导致性能下降 | 设置用户体验监控机制并接收警报 |
| 重定向链（如A→B→C→D） | 会导致爬取速度变慢和链接权重分散 | 将重定向路径简化为单步 |
| 大尺寸且未优化的图片 | 会导致LCP时间过长和页面加载速度慢 | 使用WebP或AVIF格式，并使用响应式图片源代码 |
| 未启用HTTPS或页面内容混合 | 会影响搜索引擎的信任度和排名 | 完全迁移至HTTPS |

## 特殊情况处理

### 电子商务网站（拥有数千个产品页面）：
- 使用分面导航机制，设置规范链接并设置不可索引的过滤规则
- 对于缺货页面，显示“缺货”信息（而非404错误）
- 使用`rel=canonical`链接将用户引导到集合页面，或实现“加载更多”功能

### 单页应用程序（Single Page Applications, SPA）：
- 必须进行预渲染或服务器端渲染（SSR）以优化SEO效果
- 确保HTML中存在`<a href>`标签（而不仅仅是通过JavaScript进行链接跳转）
- 在禁用JavaScript的情况下测试页面内容是否可正常显示

### 大型网站（拥有超过10万页）：
- 使用多个站点地图进行索引
- 优化爬取预算非常重要
- 考虑使用自动更新的动态站点地图
- 通过分析日志文件来了解实际的爬取行为

### 特定于WordPress的优化建议：
- 安装Yoast或RankMath插件以辅助技术SEO工作
- 检查主题是否正确生成结构化数据标记
- 减少插件使用（每个插件都会增加JS和CSS的加载负担）
- 使用对象缓存（如Redis）和页面缓存（如WP Super Cache/W3 Total Cache）

### 在Google算法更新后：
- 不要惊慌——等待2周，让算法效果稳定下来
- 比较受影响的页面和未受影响的页面，找出规律
- 检查Google搜索控制台中的手动操作记录
- 重点关注内容质量和用户体验（E-E-A-T），而不仅仅是技术修复
