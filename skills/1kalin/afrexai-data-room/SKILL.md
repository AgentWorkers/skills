# 数据室构建工具

该工具可帮助您为融资、并购或尽职调查创建结构化的虚拟数据室目录和文件层级结构。

## 使用场景
- 准备进行轮融资（种子轮至C+轮）
- 出售企业或进行并购尽职调查
- 回应投资者或收购方的文件请求
- 为董事会治理整理公司文件

## 工作原理

用户需提供以下信息：
- 公司所处的发展阶段
- 行业
- 融资轮次类型（或并购类型）
- 任何特定的投资者要求

工具将生成以下内容：

### 1. 文件夹结构
按类别组织的完整目录树：
- **公司资料**：公司章程、董事会会议记录、股本结构表、股东协议
- **财务资料**：利润表（过去3年及预测数据）、资产负债表、现金流量表、烧钱速度、收入明细、纳税申报表
- **法律资料**：知识产权分配情况、雇佣协议、保密协议（NDA）、诉讼记录、监管文件
- **商业资料**：客户合同、业务发展情况、客户流失数据、定价历史、合作伙伴协议
- **产品与技术**：架构文档、安全审计报告、系统运行时长记录、技术栈概览
- **人力资源与团队**：组织结构图、关键员工简历、薪酬汇总、期权计划、招聘计划
- **合规性资料**：数据隐私政策（GDPR/CCPA）、保险凭证、许可证、审计报告

### 2. 文件清单
每个文件夹都会附带优先级标记的清单：
- 🔴 **必备文件**：缺少这些文件，交易无法完成
- 🟡 **建议文件**：虽然不是必需的，但缺失会引发疑问
- 🟢 **可选文件**：体现公司的成熟度，非强制要求

### 3. 缺项分析
对比用户现有的文件与清单，标记缺失的文件，并标注紧急程度及预计制作时间。

### 4. 访问控制建议
- 哪些文件适合放入“预览版”（非保密协议阶段）
- 哪些文件需要签署保密协议（NDA）
- 哪些文件仅限管理层查看
- 实施水印和下载限制

## 规则
- 根据公司发展阶段调整文件内容的详细程度（例如，不要求初创企业提供过去3年的审计财务报告）
- 标注需要法律审核的文件
- 提供每份缺失文件的预计准备时间
- 以Markdown格式输出清单（可直接复制粘贴到Notion、Google Docs或实际的数据室平台中）

## 输出格式
Markdown格式，包含嵌套的清单。每个条目格式为：`- [ ] 文件名称`，并附带优先级标签和备注。
