# 施工计划工具

帮助您规划、确定范围并执行能够真正交付成果的敏捷冲刺。摒弃繁琐的仪式性流程。

## 功能介绍

该工具会根据您的待办事项列表（或任务清单）生成详细的冲刺计划，内容包括：
- 团队产能计算（团队规模 × 可用天数 × 专注度系数）
- 任务分配（包含缓冲时间）
- 施工目标及成功标准
- 每日站会模板
- 与关键指标相关的回顾会议提示

## 使用方法

只需向工具说明：“为[团队/项目]规划一个为期两周的冲刺”，并提供以下信息：
- 团队规模和成员的可用时间
- 待办事项列表（可粘贴或手动描述）
- 任何硬性截止日期或依赖关系

## 施工计划框架

### 1. 团队产能计算
```
Available hours = team_size × sprint_days × hours_per_day × focus_factor
focus_factor = 0.7 (accounts for meetings, interrupts, context switching)
```

### 2. 待办事项优先级排序（RICE评分法）

对每个任务进行评分：
- **影响范围**：该任务影响多少用户或流程？（1-10分）
- **重要性**：该任务对项目进展的推动作用有多大？（0.25, 0.5, 1, 2, 3）
- **估算准确性**：您对任务完成时间的预估有多可靠？（0.5, 0.8, 1.0）
- **所需工作量**：完成该任务所需的人天数

**RICE评分 = (影响范围 × 重要性 × 估算准确性) / 所需工作量**

按照评分从高到低排序，然后根据计算出的团队产能来安排任务。

### 3. 施工目标

目标应简洁明了且可衡量。例如：“确保新用户入职流程的80%在48小时内完成。”

### 4. 缓冲时间设置

预留20%的产能用于应对意外情况。如果所有任务都安排满了产能，那么团队就已经超负荷了。

### 5. 任务完成标准

每个任务都必须满足以下条件：
- [ ] 代码已审核并合并到项目中
- [ ] 测试通过
- [ ] 代码已部署到测试环境
- [ ] 产品负责人已签字确认

### 6. 每日站会（支持异步沟通）

每位成员需要汇报：
- 昨天完成了哪些工作
- 今天计划完成哪些工作
- 当前遇到的阻碍（如果有）

无需汇报“今天做了什么”，重点关注实际交付的成果。

### 7. 施工回顾会议（最长15分钟）

- **进度对比**：计划完成的任务点数与实际完成的任务点数
- **未完成的任务及原因**
- **改进措施**：选择一项流程优化措施（仅限一项）

### 8. 需警惕的常见问题

- 施工范围在冲刺过程中多次变更
- 直到冲刺最后两天才完成任何任务
- 未完成的任务占比超过计划工作量的30%
- 站会时间超过10分钟

## 为何有效

大多数冲刺计划失败的原因在于团队忽视了产能计算并过度承诺。该框架首先要求提供准确的数字，再根据优先级安排任务。设置20%的缓冲时间是为了确保团队能够按时完成承诺的任务。
