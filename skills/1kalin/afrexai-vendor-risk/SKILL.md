# 供应商风险评估

本工具用于评估第三方供应商在安全性、财务稳定性、合规性、运营依赖性以及数据管理方面的风险，并对这些风险进行评分与管理。专为采购团队、首席信息安全官（CISO）以及负责管理10家以上供应商的运营负责人设计。

## 使用方法

对每个关键供应商执行该评估，并将所有评分汇总至“投资组合风险视图”中。

## 评估框架

### 1. 供应商风险评分卡（5个维度，每个维度0-100分）

**安全性（0-100分）**
- 是否已实施SOC 2 Type II标准？（+20分）
- 在过去12个月内是否进行了渗透测试？（+15分）
- 是否有完善的事件响应计划？（+15分）
- 数据在存储和传输过程中是否进行了加密？（+15分）
- 是否强制要求所有访问都使用多因素认证（MFA）？（+10分）
- 供应商是否完成了安全问卷调查？（+10分）
- 供应商是否公开了其分包商名单？（+15分）

**财务稳定性（0-100分）**
- 收入趋势：增长（+25分）、持平（+10分）、下降（0分）
- 资金储备是否足以支持至少18个月的运营？（+20分）
- 客户集中度是否低于20%？（+15分）
- 是否公开了财务报表或经过审计？（+15分）
- 是否没有重大法律诉讼？（+15分）
- 信用评级是否合格？（+10分）

**合规性与监管要求（0-100分）**
- 是否持有相关的行业认证？（+20分）
- 是否遵守GDPR/CCPA等法规？（+20分）
- 是否签订了数据处理协议？（+15分）
- 监管审计记录是否良好？（+15分）
- 合同中是否包含审计条款？（+15分）
- 是否满足了数据存储的地域要求？（+15分）

**运营依赖性（0-100分**
- 是否有包含财务处罚的SLA？（+20分）
- 过去12个月的系统正常运行时间是否超过99.9%？（+20分）
- 是否每年都进行了灾难恢复测试？（+15分）
- 供应商是否成为业务的关键环节（存在单点故障风险）？（-20分）
- 是否有详细的迁移计划？（+15分）
- 供应商是否提供API或数据导出功能？（+15分）
- 是否对供应商的锁定风险进行了评估？（+15分）

**数据管理（0-100分）**
- 是否对数据进行分类？（+20分）
- 数据的保留和删除政策是否明确？（+20分）
- 是否能在72小时内收到数据泄露通知？（+20分）
- 是否能保证数据的可迁移性？（+15分）
- 供应商是否会在您的数据上进行人工智能/机器学习训练（可选择不参与此类训练；否则扣10分）？（+15分）
- 是否有访问日志和审计追踪机制？（+10分）

### 2. 风险等级分类

| 总分 | 风险等级 | 审查频率 | 应采取的措施 |
|--------|---------|-----------|-------------------------|
| 400-500 | 低风险 | 每年 | 进行常规监控 |
| 300-399 | 中等风险 | 每半年 | 需制定补救计划 |
| 200-299 | 高风险 | 每季度 | 提交给高层管理层处理，并寻找替代供应商 |
| 0-199 | 临界风险 | 每月 | 需在90天内制定退出计划 |

### 4. 供应商失败的成本

| 受影响领域 | 计算方式 |
|------------|-------------------------|
| 收入损失 | 日收入 × 预计的停机时间（天数） |
| 恢复成本 | 迁移费用 + 紧急采购费用 |
| 合规性罚款 | 由于供应商数据泄露而产生的罚款 |
| 声誉损失 | 客户流失率 × 客户生命周期价值（LTV） × 受影响的客户数量 |
| 运营中断 | 员工闲置期间的成本 × 恢复周期 |

### 5. 季度审查内容

- 自上次审查以来的评分变化（标记出评分下降超过10分的供应商）
- 供应商新增的分包商信息
- 供应商的SLA履行情况
- 安全事件或未遂事件
- 合同续签的时间表及谈判策略
- 其他供应商的基准情况

### 6. 需立即采取行动的警示信号

- 供应商被竞争对手收购
- 关键人员（如CISO、CTO）离职
- 系统停机时间超过SLA规定的期限（超过2个月）
- 遭到监管机构的调查或处罚
- 供应商拒绝完成安全问卷调查
- 供应商的数据泄露事件影响到其他客户
- 供应商突然提高价格（超过20%）

## 行业特定的供应商风险

| 行业 | 关键供应商类别 | 特定风险 |
|---------|-------------------------|---------------------------|
| 医疗保健 | 电子病历系统（EHR）、账单处理、远程医疗 | 遵守HIPAA/BAA标准的情况，患者健康信息（PHI）的泄露风险 |
| 金融服务 | 核心银行业务、支付系统、客户身份验证（KYC） | 遵守PCI DSS标准，合规性报告要求 |
| 法律服务 | 案件管理、电子数据发现（ediscovery） | 权限泄露，客户数据安全问题 |
| SaaS | 基础设施、身份验证、支付系统 | 系统故障的连锁反应，个人身份信息（PII）的安全风险 |
| 制造业 | 生产制造执行系统（MES）、供应链管理、物联网（IoT） | 专利侵权，生产中断 |
| 建筑行业 | 项目管理、安全措施 | 合规性文件缺失 |
| 电子商务 | 支付系统、订单履行、内容分发网络（CDN） | 遵守PCI标准，高峰期的系统可用性 |
| 人力资源 | 招聘管理系统（ATS）、背景调查、薪资处理 | 候选人的个人身份信息（PII）安全问题，AI筛选中的偏见 |
| 房地产 | 房地产信息管理系统（MLS）、交易管理、产权登记 | 金融欺诈，交易流程延误 |
| 专业服务 | 客户关系管理（CRM）、账单处理、文档管理 | 客户信息泄露 |
