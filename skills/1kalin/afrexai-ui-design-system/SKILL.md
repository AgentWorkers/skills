#### 第1步：选择颜色策略

| 颜色策略 | 适用场景 | 示例 |

| 分割对比色方案 | 颜色鲜艳但搭配协调 | 例如Notion应用的颜色方案 |
| 中性色+强调色 | 安稳专业，易于阅读 | 例如GitHub使用的白色和蓝色 |

### 颜色可访问性规则

| 对比度比率 | 适用场景 | WCAG等级要求 |
|---------------|---------|------------|
| 最低4.5:1 | 普通文本（小于18像素） | AA级 |
| 最低3:1 | 大字号文本（≥18像素，加粗，≥24像素） | AA级 |
| 最低3:1 | UI组件、图标 | AA级 |
| 最低7:1 | 普通文本 | AAA级 |

**切勿仅依赖颜色**——始终结合图标、标签或其他视觉元素来辅助信息传递。

#### 暗色模式策略

| 浅色模式 | 暗色模式 | 规则 |
|-----------|-----------|------|
| 背景为白色 | 不要使用纯黑色（使用灰色#111827） | 纯黑色会导致视觉疲劳 |
| 灰色背景下的文字颜色 | 灰色背景下的文字颜色 | 降低对比度，避免眼睛疲劳 |
| 主要颜色调整为500%亮度 | 辅助颜色调整为400%亮度 | 在暗色背景下调整颜色饱和度 |
| 边界框颜色 | 边界框颜色 | 边界框颜色 |

### 间距规则

- **相关元素**：间距为4-8像素（例如图标和标签）
- **分组元素**：间距为12-16像素（例如表单字段）
- **章节之间**：间距为32-64像素

- **卡片内部填充**：间距为16-24像素
- **保持一致性**：如果水平间距为16像素，垂直间距也应保持16像素或其倍数

## 第4阶段：组件设计模式

### 按钮层次结构

每个页面都需要明确的按钮层次结构，用户应能立即识别主要操作按钮。

| 按钮级别 | 样式 | 适用场景 | 每个页面最多使用数量 |
|-------|-------|---------|-------------|
| 主要按钮 | 实心背景，对比色文字 | 主要的CTA按钮、提交按钮、确认按钮 | 1-2个 |
| 辅助按钮 | 轮廓明显的背景或背景颜色较淡的按钮 | 替代操作按钮 | 2-3个 |
| 三级按钮/隐藏按钮 | 仅显示文本，无背景 | 取消按钮、关闭按钮、返回按钮 | 无限制 |
| 破坏性操作按钮 | 红色实心按钮或带轮廓的按钮 | 删除按钮 | 1个 |
| 图标按钮 | 图标+工具提示 | 工具栏按钮，紧凑型UI | 根据需要使用 |

**按钮状态提示：**
- [ ] 默认状态
- [ ] 鼠标悬停时状态（颜色变化或阴影效果）
- [ ] 活动/被点击状态（颜色加深，尺寸放大0.98倍）
- [ ] 高亮状态（键盘导航时显示高亮圈）
- [ ] 禁用状态（透明度50%，鼠标指针不可点击）
- [ ] 加载状态（显示加载指示器）

**最小触摸目标尺寸：44×44像素**（即使按钮在视觉上看起来更小）

### 表单设计规则

1. **输入框上方必须有标签**（避免内联或浮动标签，因为这样会影响可访问性）
2. **少于6个输入框的表单**使用单列布局（姓名等相关字段使用双列布局）
3. **输入框宽度应与预期输入内容长度相匹配**（例如电子邮件输入框应占据整个宽度，邮政编码输入框宽度较短）
4. **实时验证**：在用户离开输入框时立即显示错误信息
5. **错误信息**应具体说明问题（例如“电子邮件必须包含@符号”）
6. **必填字段**用“(可选)”标注，而非用*标记
7. **使用分隔符或标题将相关字段分组**
8. **逐步显示高级选项**：默认情况下不显示高级选项

### 模态框与对话框设计

- **最大宽度**：480像素（小尺寸），640像素（中等尺寸），800像素（大尺寸）
- **背景**：半透明的深色背景（点击时隐藏）
- **始终**在右上角放置关闭按钮（X）
- **焦点捕捉**：确保键盘焦点始终位于模态框内
- **动画效果**：淡入效果+轻微的尺寸放大（200-300毫秒）
- **移动设备**：模态框应从底部滑出
- **破坏性操作**：需要明确确认，使用红色按钮，并说明操作后果

### 空白状态处理

**永远不要显示空白页面。**每个空白状态都应包含以下元素：**

1. **插图或图标**：增加视觉吸引力
2. **标题**：说明该区域的用途
3. **描述**：解释为什么该区域为空
4. **提示信息**：指示用户下一步该做什么

---

### 加载状态处理

| 加载状态类型 | 适用场景 | 示例 |
|---------|------|---------|
| 骨架显示**：内容正在加载 | 显示与页面布局相匹配的灰色渐变图案 |
| 加载指示器**：短暂的操作（小于3秒） | 显示提交按钮或提示信息 |
| 进度条**：显示加载过程（例如文件上传、多步骤操作）
**乐观式UI**：对于低风险操作，使用提示按钮、切换按钮或发送消息等元素

**规则：优先使用骨架显示方式**——骨架显示可以减少用户的等待感。

### 响应式布局规则

| 组件 | 移动设备 | 平板设备 | 桌面设备 |
|-----------|--------|--------|---------|
| 导航栏 | 使用底部标签栏或汉堡菜单 | 侧边栏折叠 | 侧边栏展开或顶部导航栏 |
| 卡片布局 | 单列布局 | 双列布局 | 三到四列布局 |
| 表格 | 卡片式布局或水平滚动 | 在较少列数时仍可显示完整表格 |
| 侧边栏+内容 | 全宽布局 | 叠叠式侧边栏 | 持久性侧边栏 |
| 重要内容区域 | 文本下方显示图片 | 并排显示 | 带有额外边距的并排布局 |
| 模态框 | 全屏显示 | 居中显示 | 居中显示 |
| 表单布局 | 单列布局 | 单列布局 | 相关字段可以使用双列布局 |

### 移动设备的触控友好设计规则

- **最小触摸目标尺寸**：44×44像素（Apple设备）/48×48像素（Material Design规范）
- **元素之间的最小间距**：8像素
- **拇指操作区域**：主要操作按钮应位于屏幕底部60%的位置
- **滑动手势**：始终提供按钮替代方案
- **移动设备上避免依赖鼠标悬停来触发交互**

## 第7阶段：可访问性检查

### WCAG AA级标准（最低要求）

**可感知性：**
- **普通文本的对比度≥4.5:1，大字号文本的对比度≥3:1**
- **不要仅依赖颜色作为信息传递的唯一方式**（添加图标、图案、文字说明）
- **所有图片都应配有alt文本**（装饰性图片除外）
- **视频应配有字幕；音频应配有文字说明**
- **文本可放大至200%大小而不破坏页面布局**

**可操作性：**
- **所有功能均可通过键盘操作**
- **所有交互元素都应有明显的焦点指示**
- **提供“跳转到内容”链接**
- **没有会导致意外操作的元素**
- **交互操作有足够的时间（或提供延时选项）**
- **没有内容每秒闪烁超过3次**

**可理解性：**
- `<html>`标签应包含语言属性**
- **表单标签应与输入字段关联**
- **错误信息应明确指出问题并提供修正建议**
- **页面间的导航逻辑一致**
- **没有意外的页面布局变化**

**稳定性：**
- **使用语义正确的HTML结构**
- **正确使用ARIA属性（或完全不使用ARIA属性——优先使用语义化的HTML结构）**
- **使用屏幕阅读器（如VoiceOver、NVDA）进行测试**
- **在浏览器放大200%的情况下仍能正常使用**

### 语义化HTML使用指南

| 功能 | 使用方式 | 不建议使用的方式 |
|---------|-----|-----|
| 页面标题 | 使用`<header>`标签 | 使用`<div class="header">` |
| 导航栏 | 使用`<nav>`标签 | 使用`<div class="nav">` |
| 主要内容 | 使用`<main>`标签 | 使用`<div class="content">` |
| 节点 | 使用`<section>`标签 + 标题 | 使用`<div>`标签 |
| 文章/卡片 | 使用`<article>`标签 | 使用`<div class="card">` |
| 侧边栏 | 使用`<aside>`标签 | 使用`<div class="sidebar">` |
| 页面底部 | 使用`<footer>`标签 | 使用`<div class="footer">` |
| 按钮操作 | 使用`<button>`标签 | 使用`<div onclick>`或`<a href>` |
| 链接/导航 | 使用`<a href>`标签 | 使用`<button>`作为导航链接 |
- **项目列表**：使用`<ul>`或`<ol>`标签 | 使用带有换行符的`<div>`元素 |

## 第8阶段：设计评审标准（0-100分）

从以下维度对每个设计进行评分：

| 评估维度 | 权重 | 评分标准 |
|-----------|--------|----------|
| 视觉层次结构 | 20% | 用户能否在3秒内识别主要操作？标题/内容/CTA的层次结构清晰吗？ |
| 一致性 | 15% | 间距、颜色、边界半径、字体风格是否一致？组件是否可重复使用？ |
| 空白布局 | 15% | 各部分之间的间距是否合理？页面是否不拥挤？是否没有过度使用空白空间？ |
| 可访问性 | 15% | 对比度是否符合WCAG标准？是否可以通过键盘操作？是否使用了语义化HTML？ |
| 响应式设计 | 10% | 设计是否在移动设备、平板设备和桌面设备上都能正常使用？是否适合触控操作？ |
| 交互设计 | 10% | 是否定义了鼠标悬停、激活状态、加载状态等交互效果？ |
| 字体设计 | 10% | 文字行长度是否易于阅读？层次结构是否合理？字体大小是否一致？ |
| 颜色使用 | 5% | 颜色使用是否合理？是否使用了足够多的颜色？暗色模式是否可用？ |

**评分标准：**
- 90-100分：设计成熟，可以交付使用 |
- 75-89分：设计不错，需要少量改进 |
- 60-74分：功能基本可用，但需要进一步优化 |
- 60分以下：设计需要大幅修改

### 快速质量检查清单

在发布任何设计之前，请确保：

- **主要CTA按钮一目了然**
- **空白状态有明确的视觉提示**
- **存在加载状态（如骨架显示或加载指示器）**
- **错误状态有明确的提示信息**
- **移动设备布局经过测试**
- **暗色模式正常工作**
- **所有交互元素都有悬停效果和高亮状态**
- **表单验证功能正常**
- **所有文本的对比度都符合WCAG标准**
- **没有孤立的标题或标题中的单词过短**
- **设置了网站图标和页面标题**
- **图片都配有alt文本**
- **404错误页面已设计完成**

### 动画设计系统

| 动画类型 | 动画持续时间 | 动画效果 | 适用场景 |
|----------|----------|--------|-----|
| 微动画 | 100-200毫秒 | 动画效果为“ease-out” | 用于切换按钮、复选框、按钮点击等 |
| 中等长度动画 | 200-400毫秒 | 动画效果为“ease-out” | 用于面板、下拉菜单、工具提示等 |
| 长时间动画 | 400-700毫秒 | 动画效果为“ease-in-out”，并带有延迟效果 | 用于页面切换、模态框显示等 |
**减少动画使用：**始终遵循`prefers-reduced-motion`策略，避免不必要的动画效果。

### 性能优化设计

- **图片**：使用`loading="lazy"`属性，使用`srcset`进行响应式加载，采用WebP格式
- **字体**：根据实际使用的字符数量选择合适的字体，最多使用2个字体系列
- **显示在页面上方的元素**：将关键CSS代码内联显示，重要图片预先加载
- **图标**：使用SVG精灵图或图标字体，避免使用单独的PNG图片
- **动画效果**：仅使用`transform`和`opacity`属性（利用GPU加速），避免使用`width`、`height`、`top`、`left`等属性进行动画

### 国际化设计

- **支持右-to-left（RTL）语言**：使用合适的CSS属性（例如`margin-inline-start`而非`margin-left`
- **文本长度**：德语/芬兰语等语言的文本长度可能比英语长30-40%，请相应调整设计
- **日期/数字格式**：根据用户语言环境进行自定义
- **图标**：避免使用具有文化特定含义的符号
- **字符串长度**：考虑文本长度可能是英语的两倍
