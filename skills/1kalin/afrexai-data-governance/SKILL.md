# 数据治理框架

评估并改进您组织在六个领域的数据治理状况。

## 涵盖内容

1. **数据质量** — 完整性、准确性、一致性、及时性的评分
2. **数据目录管理** — 资产清单、数据 lineage 跟踪、元数据管理
3. **访问控制** — 基于角色的权限管理、最小权限原则、数据分类（公开/内部/机密/受限）
4. **合规性映射** — GDPR、CCPA、SOX、HIPAA、PCI-DSS、行业特定法规
5. **数据保留与生命周期管理** — 保留政策、归档计划、删除流程、法律保留要求
6. **AI/代理数据治理** — 培训数据的来源追踪、模型输入/输出记录、偏见检测、代理工作流程中的个人身份信息（PII）处理

## 使用方法

当需要评估数据治理情况时，请按照以下步骤操作：

1. 确定哪些领域是优先级（或评估全部六个领域）
2. 对每个领域的8项控制措施进行0-3分的评分：
   - 0 = 未实施
   - 1 = 临时/非正式的
   - 2 = 有文档记录但部分执行
   - 3 = 自动化且持续监控
3. 计算每个领域的得分（总分 / 24 × 100）
4. 计算整体治理得分（各领域得分的平均值）
5. 根据风险优先级生成改进计划

## 评分解读

| 得分 | 评分等级 | 应采取的行动 |
|-------|--------|--------|
| 0-25% | 严重不足 | 需立即采取补救措施 — 存在合规风险 |
| 26-50% | 发展中 | 需制定90天内的改进计划 |
| 51-75% | 管理良好 | 优化并自动化薄弱环节 |
| 76-100% | 优化完成 | 维护现状并与其他组织进行对比 |

## 第1领域：数据质量控制

1. 数据分析自动化（重复检测、格式验证）
2. 带有服务水平协议（SLA）阈值的质量仪表板
3. 数据质量问题的根本原因分析
4. 数据管理机制（为每个领域指定数据负责人）
5. 数据管道中的质量检查环节（在数据导入时拒绝不合格数据）
6. 业务规则验证（特定领域的逻辑检查）
7. 系统间的数据一致性验证（源数据与目标数据的匹配）
8. 数据质量趋势跟踪（每月的改进指标）

## 第2领域：数据目录管理控制

1. 自动化资产发现（数据库、API、文件、SaaS）
2. 具有统一定义的业务术语表
3. 数据 lineage 跟踪（从来源到使用的全过程）
4. 业务用户的搜索和发现界面
5. 元数据丰富化（标签、分类、敏感度标记）
6. 目录覆盖度跟踪（记录的资产占比）
7. 使用情况分析（谁访问了什么数据、访问频率）
8. 与BI/分析工具的集成（支持目录查询）

## 第3领域：访问控制

1. 基于角色的访问控制（RBAC）并定期审查
2. 数据分类的严格执行
3. 最小权限原则（默认权限设置）
4. 访问请求和审批流程
5. 特权账户管理（监控管理员账户）
6. 访问权限的定期重新认证
7. 异常访问模式的检测
8. 自动化权限撤销（角色变更或离职时立即解除访问权限）

## 第4领域：合规性映射

1. 相关法规清单（按地区和行业划分）
2. 控制措施与法规要求的对应关系
3. 数据处理记录（如GDPR第30条等规定）
4. 同意管理（收集、存储、撤回流程）
5. 数据主体权利的处理（访问、删除、数据迁移）
6. 跨境数据传输合规性（SCC、数据保护水平评估）
7. 数据泄露通知程序（GDPR要求72小时内通知）
8. 定期合规性审计（内部和第三方审计）

## 第5领域：数据保留与生命周期管理

1. 根据数据类型制定保留策略（合同要求、法规规定、运营需求）
2. 自动化归档流程（从热数据到冷数据再到删除）
3. 法律保留要求的管理（用于法律诉讼）
4. 删除数据的验证（确认删除并保留审计记录）
5. 存储成本优化（根据访问频率分层存储）
6. 备份和恢复测试（定期进行恢复演练）
7. 数据最小化（仅收集必要的数据）
8. 废弃系统的处置流程

## 第6领域：AI/代理数据治理

1. 培训数据的来源追踪（来源、用户同意、偏见检测）
2. 模型输入/输出记录（输入和输出的数据）
3. 代理工作流程中的个人身份信息处理（PII的检测和屏蔽）
4. 输出结果的准确性验证
5. 代理决策的审计跟踪（自动化决策的可解释性）
6. 数据反馈机制（人工审核代理对数据的修改）
7. 供应商的数据共享协议（第三方API对数据的访问权限）
8. 合成数据的政策（生成数据的用途和方式）

## 不良数据治理的代价

| 风险 | 平均成本 | 预防成本 |
|------|-------------|-----------------|
| GDPR罚款 | 430万美元（2025年平均值） | 每年4.5万至12万美元 |
| 数据泄露 | 488万美元（IBM 2025年案例） | 每年6万至20万美元 |
| 审计失败 | 15万至50万美元的补救费用 | 每年3万至8万美元 |
| 数据决策失误 | 15%至25%的收入损失 | 每年2万至6万美元 |
| AI偏见事件 | 200万至500万美元（包括法律费用和品牌损失） | 每年2.5万至7.5万美元 |

## 改进优先级矩阵

始终按照以下顺序进行改进：
1. **合规性差距** — 合规罚款可能造成重大损失
2. **访问控制** — 数据泄露会迅速破坏信任
3. **AI治理** — 风险增长最快的领域
4. **数据质量** — 数据质量直接影响最终结果
5. **目录管理** — 无法管理的数据无法有效治理
6. **数据保留** — 存储成本会累积，法律风险也会增加
