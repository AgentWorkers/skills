# 人工智能合规性准备评估

本工具用于评估组织在遵守人工智能（AI）相关法规方面的准备情况，并生成一份可操作的合规性实施路线图。

## 适用场景
- 在审计前评估组织的AI合规性状况
- 为欧盟《人工智能法案》（2026年8月生效）、美国卫生与公共服务部（HHS）的AI相关要求以及NIST的AI风险管理框架（AI RMF）做准备
- 为AI系统的部署制定治理路线图
- 评估当前AI使用带来的合规风险

## 使用方法

当需要评估AI合规性准备情况时，请收集以下信息：

### 必需输入信息
1. **行业**（法律、医疗保健、金融服务、保险、建筑、制造业、政府等）
2. **公司规模**（员工人数或收入范围）
3. **正在使用的AI系统**（例如：聊天机器人、文档审核工具、欺诈检测系统、招聘工具、客户服务系统、分析工具等）
4. **所涉及的司法管辖区**（仅限美国、受欧盟法规约束、同时受多个司法管辖区约束或全球范围）

### 可选输入信息
- 当前的治理框架（如有）
- 预计的审计日期
- 已获得的合规认证（如SOC2、ISO 27001、HIPAA等）
- 正在使用中的AI供应商/工具的数量

## 评估框架

对每个评估维度进行1-5分的评分（1表示没有控制措施，5表示成熟度最高）：

### 8个评估维度
1. **风险分类**：是否已根据欧盟《人工智能法案》/NIST的标准对AI系统进行风险分级？
2. **文档管理**：是否为每个AI系统准备了技术文档、模型说明以及数据来源记录？
3. **人工监督**：对于高风险决策，是否制定了明确的人工干预流程？
4. **偏见与公平性**：是否定期进行偏见审计、使用公平性指标，并进行差异性影响测试？
5. **数据治理**：训练数据是否具有明确的来源记录，是否获得了用户同意，数据保留和删除政策是否完善？
6. **事件响应**：是否制定了针对AI相关事件的应对方案、报告流程以及回滚计划？
7. **供应商管理**：是否对AI供应商进行了风险评估，并在合同中明确了AI治理要求？
8. **审计追踪**：是否对AI辅助产生的结果进行了日志记录、可解释性分析以及决策过程的追踪？

### 评分标准
- **35-40分**：合规准备就绪——只需解决一些小问题
- **25-34分**：部分准备到位——需要在特定领域进行大量改进
- **15-24分**：风险较高——多个维度存在重大缺陷
- **8-14分**：情况危急——在监管审查前需要立即采取行动

## 输出格式

生成的报告内容包括：
1. **执行摘要**：总体评分、风险等级以及最突出的三个问题
2. **各维度评分**：包含每个维度的评分、相关证据以及问题描述
3. **法规适用情况**：列出适用的相关法规及其关键截止日期：
   - 欧盟《人工智能法案》：2026年8月2日（针对高风险系统）
   - 美国卫生与公共服务部AI透明度要求：2026年4月3日（医疗保健领域）
   - NIST AI风险管理框架：持续实施中（适用于联邦承包商及最佳实践要求）
   - 各州关于AI的法规：因行业而异
