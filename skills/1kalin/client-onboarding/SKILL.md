# 客户入职检查表生成器

本工具可为任何服务型企业生成一份全面、定制化的客户入职检查表，涵盖客户信息收集、系统设置、沟通流程、关键里程碑以及成功评估等内容。

## 使用场景
- 开始新的客户合作项目时  
- 在整个团队中统一入职流程  
- 通过改善初次印象来降低客户流失率  
- 制定服务交付的标准操作流程（SOP）  

## 工作原理  
向用户询问以下信息：  
1. **业务类型**（代理机构、SaaS服务、咨询服务、托管服务等）  
2. **提供的服务内容**  
3. **客户规模**（小型企业、中型企业、大型企业）  
4. **预计入职周期**  

根据这些信息，生成以下结构化的检查表：  

### 第一阶段：入职前（入职前一天）  
- [ ] 合同签署并确认付款  
- [ ] 指定内部负责团队（客户经理、技术负责人）  
- [ ] 发送包含服务期望的欢迎邮件  
- [ ] 请求客户提供系统/工具/平台的访问凭证  
- [ ] 安排启动会议  
- [ ] 向内部团队介绍客户的目标、限制条件及相关利益方  

### 第二阶段：启动阶段（入职第1-3天）  
- [ ] 完成启动电话会议，明确目标、时间表及成功评估标准  
- [ ] 确定关键利益方并记录联系方式  
- [ ] 协议沟通频率（每周电话会议、Slack、电子邮件）  
- [ ] 共享项目计划/路线图  
- [ ] 确定并完成首个可交付成果（需在第一周内完成）  
- [ ] 将客户信息添加到项目管理工具中  

### 第三阶段：系统设置与配置（第1周）  
- [ ] 完成技术设置和环境配置  
- [ ] 连接并测试所有必要的集成功能  
- [ ] 完成数据迁移或导入（如适用）  
- [ ] 设置面向客户的仪表板或报告系统  
- [ ] 发送首个可交付成果  
- [ ] 客户反馈：确认已感受到服务价值  

### 第四阶段：系统稳定阶段（第2-4周）  
- [ ] 完成首次每周沟通  
- [ ] 收集并处理客户反馈  
- [ ] 向客户交付相关文档和操作指南  
- [ ] （如适用）完成培训  
- [ ] 记录并共享问题解决流程  
- [ ] 建立服务性能基准  

### 第五阶段：过渡至日常维护阶段（第2个月以后）  
- [ ] 从入职阶段过渡到日常运营支持  
- [ ] 核对实际成果与初始目标  
- [ ] 进行客户满意度调查（NPS或定性评估）  
- [] 发现潜在的追加销售或扩展机会  
- [] 在内部完成入职流程的回顾总结  

## 输出格式  
以Markdown格式提供检查表，用户可将其复制到自己的项目管理工具中。根据具体业务类型和服务内容对各项内容进行个性化调整。  

## 定制规则：  
- **针对SaaS服务**：重点关注技术集成、API配置及用户权限管理  
- **针对咨询服务**：重点强调需求分析、利益方识别及交付里程碑  
- **针对托管服务**：重点关注服务水平协议（SLA）、监控机制及问题处理流程  
- **针对代理机构**：重点强调品牌规范、审批流程及内容更新计划
