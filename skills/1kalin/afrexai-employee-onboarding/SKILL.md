# 员工入职生成器  
为任何职位生成一个结构化的90天入职计划，涵盖入职前准备、第一天、第一周、30天/60天/90天的关键节点、导师分配以及绩效评估指标。  

## 提供的内容：  
- **入职前检查清单**：IT设备配置、工作空间设置、欢迎礼包、所需文件  
- **第一天日程安排**：入职培训安排、第一天体验设计  
- **第一周计划**：团队介绍、工具使用指南、首个任务分配  
- **30天/60天/90天关键节点**：针对职位的具体目标及明确的成功标准  
- **导师/伙伴计划**：匹配机制、定期沟通机制、问题上报流程  
- **经理指导模板**：每周一对一沟通模板、反馈框架、需要注意的警示信号  
- **合规性跟踪**：所需培训完成情况、政策确认、认证截止日期  

## 使用方法：  
提供以下信息：  
- 职位名称及所属部门  
- 团队规模及汇报结构  
- 工作方式（远程、混合办公或办公室办公）  
- 行业（用于合规性要求）  
- 员工需要使用的特定工具或系统  

系统会生成完整的入职包，包括检查清单、时间表、模板和跟踪表格。  

## 重要性：  
- 20%的员工离职发生在入职后的前45天内  
- 结构化的入职流程能将员工留存率提高82%（Brandon Hall Group研究）  
- 通过正式的入职计划，新员工达到全产能的速度加快34%  
- 每个失败的新员工在入职第一年内会给公司带来15,000至25,000美元的损失  

## 入职流程：  

### 入职前准备（收到录用通知后至第一天）  
**IT与访问权限：**  
- [ ] 笔记本电脑/设备已订购并配置  
- [ ] 创建了电子邮件账户  
- [ ] 已获得Slack/Teams等聊天工具的访问权限  
- [ ] 所有职位所需的SaaS工具均已配置  
- [ ] 已获取VPN和安全凭证  
- [ ] 已获得办公场所出入权限  

**人力资源与行政：**  
- [ ] 已签署并归档录用通知  
- [ ] 背景调查已完成  
- [ ] 已发送税务表格（W-4/P45等）  
- [ ] 已发送福利注册资料  
- [ ] 员工已确认收到员工手册  
- [ ] 已填写紧急联系人信息  

**欢迎环节：**  
- [ ] 经理发送个性化欢迎邮件  
- [ ] 通过邮件分配并介绍导师  
- [ ] 第一周的工作日历已预先填写  
- [ ] 已起草团队Slack频道介绍信息  

### 第一天  
| 时间 | 活动 | 负责人 |  
|------|----------|-------|  
| 9:00 | 欢迎会及工作空间参观 | 经理 |  
| 9:30 | IT设备设置确认 | IT/导师 |  
| 10:00 | 人力资源培训（福利政策） | 人力资源部门 |  
| 11:00 | 团队介绍 | 经理 |  
| 11:30 | 与导师共进咖啡 | 导师 |  
| 12:00 | 团队午餐 | 全体团队 |  
| 13:00 | 职位概述与期望说明 | 经理 |  
| 14:00 | 工具使用指南 | 导师 |  
| 15:00 | 首个任务分配 | 经理 |  
| 16:00 | 当天工作总结 | 经理 |  

### 第一周：  
- 第2-3天：深入了解团队流程和现有项目  
- 第4天：跟随资深团队成员体验日常工作流程  
- 第5天：完成首个小型任务（难度适中），周五向经理汇报  

### 30天关键节点：  
- 完成所有必需的培训与认证  
- 独立掌握团队工作流程及工具使用  
- 提交2-3个小型成果  
- 与关键跨部门人员建立联系  
- 经理评估：进展顺利/需要支持/存在问题  

### 60天关键节点：  
- 独立负责一个工作流程或项目  
- 在团队会议中能提出有价值的意见  
- 发现并改进某个流程  
- 收到并采纳了第一轮反馈  
- 经理评估：表现优异/进展顺利/需要支持  

### 90天关键节点：  
- 在职位上完全发挥效能（根据职位KPI衡量）  
- 完成试用期评估（如适用）  
- 制定了未来6个月的个人发展计划  
- “净推荐值”（Net Promoter Score）：“你会向朋友推荐这家公司吗？”  
- 经理评估：表现优秀/符合期望/需要制定进一步发展计划  

### 经理每周一对一沟通模板（示例）：  
```
## Weekly Check-in — [Name] — Week [X]

### How are you feeling? (1-10)
### What went well this week?
### What's been confusing or frustrating?
### What do you need from me?
### Goals for next week:
### Any concerns? (theirs or yours)
```  

### 需立即上报的警示信号：  
- 未提前通知缺席第一天  
- 入职前两周内表现消极（关闭摄像头、回答简短、缺席会议）  
- 导师反映员工与团队文化不适应  
- 未在入职第14天内完成必要培训  
- 经理在入职第7天内未进行任何一对一沟通  

### 入职绩效评估指标：  
| 指标 | 目标 | 测量方式 |  
|--------|--------|-------------|  
| 完成首个任务的时间 | < 5个工作日 | 经理记录 |  
| 达到全产能的时间 | < 90天 | 基于KPI的评估 |  
| 90天留存率 | > 95% | 人力资源数据 |  
| 新员工满意度（调查） | > 4.2/5.0 | 30天调查结果 |  
| 经理满意度 | > 4.0/5.0 | 90天调查结果 |  
| 培训完成率 | 在入职第14天内达到100% | 学习管理系统数据 |  

## 输出格式：  
系统可生成以下格式的文件：  
- **Google Sheets/Excel**：带有条件格式的跟踪表格  
- **Notion数据库**：以阶段为列的看板  
- **Markdown格式的检查清单**：适用于GitHub或内部文档  
- **Slack工作流程**：每个关键节点的自动提醒  
- **电子邮件序列**：分两周发送的入职引导邮件系列
