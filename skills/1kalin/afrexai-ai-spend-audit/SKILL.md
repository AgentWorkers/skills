# 人工智能支出审计

审计您公司的AI支出——发现浪费，衡量投资回报率（ROI），并合理调整您的工具栈。

## 适用场景
- 季度AI预算审查
- 在续订AI工具订阅之前
- 当AI支出超过收入的3%且没有明确的ROI时
- 在评估自建与购买AI功能的决策时

## 框架

### 第一步：盘点所有AI支出项目
将所有AI支出分类如下：

| 类别 | 例子 | 常见浪费情况 |
|----------|----------|---------------|
| **基础模型** | OpenAI、Anthropic、Google API密钥 | 40-60%（未使用的资源、选择了错误的模型等级） |
| **带AI的SaaS** | Salesforce Einstein、HubSpot AI、Notion AI | 30-50%（启用了功能但未使用） |
| **定制开发** | 内部机器学习团队、模型微调、RAG流程 | 25-45%（重复工作、过度设计） |
| **基础设施** | GPU实例、向量数据库、嵌入计算 | 35-55%（配置过高、开发实例始终处于运行状态） |
| **数据与训练** | 数据标注服务、训练数据、合成数据 | 20-40%（一次性成本被不必要的重复使用） |

### 第二步：对每个工具进行评分（0-100分）

**使用频率评分（0-30分）**
- 0：无人使用
- 10：不到25%的授权用户使用
- 20：25-75%的用户使用
- 30：超过75%的用户使用，每天都在使用

**ROI评分（0-40分）**
- 0：没有可衡量的业务影响
- 10：节省了时间但无法直接带来收入/成本节省
- 20：可衡量的成本降低（节省的成本小于支出的2倍）
- 30：明确的ROI（节省的成本是支出的2-5倍）
- 40：高ROI（节省的成本是支出的5倍以上）

**可替代性评分（0-30分）**
- 0：属于通用产品（有10种以上成本更低的替代品）
- 10：存在一些替代品
- 20：替代品较少，切换成本适中
- 30：不可替代，深度集成

**行动阈值：**
- 评分0-30分：**立即终止使用**
- 评分31-50分：**重新评估**——重新谈判或寻找替代方案
- 评分51-70分：**优化**——调整使用级别或规模
- 评分71-100分：**继续使用**——每季度进行监控

### 第三步：优化模型成本

对于每个基于API的AI工具，检查以下方面：
1. **模型选择**：是否使用了GPT-4而GPT-3.5就足够了？是否使用了Claude Opus而Sonnet更合适？
   - 规则：使用满足质量要求的最低成本模型
   - 测试：使用较低成本的模型运行100个生产查询，测量质量差异

2. **缓存**：是否在重新处理相同或类似的查询？
   - 语义缓存可以减少20-40%的API调用次数
   - 完全匹配缓存可以进一步减少5-15%的调用次数

3. **批量处理与实时处理**：哪些请求确实需要秒级响应？
   - 在大多数提供商那里，批量处理可以节省50%的成本
   - 将非紧急请求排队到批量处理窗口

4. **令牌优化**：
   - 剪减系统提示（每个令牌在大量使用时都会产生费用）
   - 使用结构化输出以减少响应所需的令牌数量
   - 为每个用例设置令牌使用上限

### 第四步：整合供应商

分析具有重叠功能的工具：

```
Current State → Target State
─────────────────────────────────────────
ChatGPT Teams + Claude Pro + Gemini → Pick ONE primary + ONE backup
Jasper + Copy.ai + ChatGPT for content → Single content tool
3 different vector databases → Consolidate to 1
Internal embeddings + OpenAI embeddings → Standardize on one
```

**整合后的成本节省**：通常可节省总AI支出的25-40%。

## 公司规模基准（2026年）

| 公司规模 | 典型AI支出 | 典型浪费 | 可回收的成本 |
|-------------|-----------------|---------------|-------------|
| 10-25名员工 | 每月2,000美元至8,000美元 | 35-50% | 每月700美元至4,000美元 |
| 25-50名员工 | 每月8,000美元至25,000美元 | 30-45% | 每月2,400美元至11,000美元 |
| 50-200名员工 | 每月25,000美元至80,000美元 | 25-40% | 每月6,000美元至32,000美元 |
| 200-500名员工 | 每月80,000美元至300,000美元 | 20-35% | 每月16,000美元至105,000美元 |
| 500名以上员工 | 每月300,000美元以上 | 15-30% | 每月45,000美元至300,000美元 |

## 需警惕的信号
- AI支出增长速度超过收入增长速度（不可持续）
- 同一类别中存在超过3个重复的工具
- AI SaaS许可证没有使用情况记录
- GPU实例全天候运行用于开发/测试工作负载
- 为初创企业级别的使用量支付企业级费用
- 没有对不同模型等级进行A/B测试
- “创新预算”没有具体的成功指标

## 行业差异
- **SaaS/技术行业**：如果AI支出用于产品开发，其占比可更高（5-8%）
- **专业服务行业**：重点关注可计费的工时效果——每1美元的AI支出应至少节省5美元的人工成本
- **制造业**：AI支出应与缺陷减少或产量提升相关联
- **医疗行业**：合规成本可能导致支出增加20-30%——在评估浪费前需考虑这一点
- **金融服务行业**：模型风险管理会增加15-25%的成本——这是合理支出
- **电子商务行业**：应根据订单量来衡量AI支出——随着业务量增长，支出应相应减少
