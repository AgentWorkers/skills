# 应收账款管理器

自动化应收账款（AR）工作流程：包括账龄分析、收款优先级排序、付款跟进、现金申请以及坏账预测。

## 功能概述

1. **应收账款账龄报告**：将未结清的发票按照0-30天、31-60天、61-90天、90天以上四个时间段进行分类，并附上风险评分。
2. **收款优先级队列**：根据欠款金额、逾期天数和风险等级对逾期账户进行排序，以确定最佳的跟进顺序。
3. **付款提醒邮件**：自动生成专业的提醒邮件（从友好提醒到正式通知，最后是催收通知）。
4. **现金申请匹配**：将收到的付款与未结清的发票进行匹配，并处理金额差异。
5. **坏账预测**：利用历史付款模式和账龄趋势来预测可能的坏账损失。
6. **应收账款周转天数（DSO）跟踪**：计算应收账款周转天数，并提供行业基准数据以供分析。

## 使用方法

向您的智能代理说明您的需求：
- “对我们的未结清发票进行账龄分析。”
- “请对我们的收款队列进行优先级排序，我们应该先催收哪些账户？”
- “为[客户名称]生成一份60天逾期的提醒邮件。”
- “我们的DSO为47天，这与SaaS行业的基准相比如何？”
- “预测第一季度的坏账风险。”

## 应收账款账龄分类

| 分类 | 风险等级 | 应采取的行动 |
|--------|-----------|--------|
| 当前（0-30天） | 低 | 监控 |
| 31-60天 | 中等 | 发送友好提醒 |
| 61-90天 | 高 | 提升处理级别并发送书面通知 |
| 90天以上 | 危急 | 发出最终催款通知或进行坏账核销评估 |

## 收款优先级计算公式

```
Priority Score = (Invoice Amount × 0.4) + (Days Overdue × 0.3) + (Customer Risk Score × 0.3)
```

客户风险评分（1-10分）的依据包括：
- 付款历史（平均付款天数）
- 过期发票的数量
- 信用额度的使用情况
- 行业内的违约率

## 付款提醒邮件分级

### 第1天（发票到期日）
主题：发票#[NUM]——今日到期，请付款
语气：友好，提供信息

### 第7天（逾期1周）
主题：友好提醒——发票#[NUM]已逾期
语气：温和但明确

### 第30天（逾期1个月）
主题：请付款——发票#[NUM]已逾期30天
语气：专业且正式

### 第60天（逾期2个月）
主题：紧急——发票#[NUM]严重逾期
语气：严肃，提及滞纳金和服务影响

### 第90天以上（最终通知）
主题：最终通知——发票#[NUM]需立即付款
语气：正式，提及将转交给催收部门

## 行业应收账款周转天数（DSO）基准

| 行业 | 良好DSO | 平均DSO | 较差DSO |
|----------|----------|-------------|----------|
| SaaS/软件 | <30 | 30-45 | >60 |
| 专业服务 | <35 | 35-55 | >70 |
| 制造业 | <40 | 40-60 | >75 |
| 建筑业 | <45 | 45-70 | >90 |
| 医疗保健 | <35 | 35-50 | >65 |

## 坏账预测模型

根据账龄阶段估算坏账概率：
- 当前阶段：违约率为1-2%
- 31-60天：5-8%
- 61-90天：15-25%
- 90-120天：30-50%
- 120天以上：50-80%

根据预测的坏账概率，对应未结清金额计提损失准备金。

## 现金申请规则

1. **完全匹配**：付款金额与某张未结清的发票完全一致。
2. **组合匹配**：付款金额与多张发票的总金额相匹配。
3. **付款金额不足**：付款金额低于发票金额——标记为争议或需审核扣款。
4. **付款金额超出**：付款金额高于发票金额——将款项用于最旧的未结清发票或提供信用额度。
5. **未找到匹配项**：未找到匹配项——暂缓处理，48小时内进行核实。

## 输出格式

生成应收账款报告时，包含以下内容：
- 总未结清应收账款金额
- 账龄分布（金额及占比）
- 按优先级排序的十大逾期账户
- 当前DSO与3个月前的对比情况
- 预计的坏账风险
- 建议采取的行动（应联系谁、发送什么信息）
