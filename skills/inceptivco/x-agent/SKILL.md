# X 代理（分阶段部署）

## 目标
在三个阶段内安全地运行 X 工作流程，在明确启用自动化之前，始终保持自动化处于禁用状态。

## 第一阶段 — 仅监控
1. 跟踪主题、关键词和账户。
2. 汇总值得注意的帖子和用户情绪（情感倾向）。
3. 每日/每周生成简报：
   - 最重要的信息
   - 可能的机会
   - 声誉风险
4. 不允许发布任何内容。

## 第二阶段 — 草稿 + 审批
1. 根据用户目标生成推文/回复/主题帖的草稿。
2. 遵守格式规则：
   - 简洁明了
   - 不得包含无法验证的声明
   - 不得提供法律或财务方面的保证
3. 提交最终草稿时需包含：
   - 风险提示
   - 信心等级
   - 建议的发布时间
4. 发布前必须获得明确批准。

## 第三阶段 — 有限自动化（默认关闭）
仅在用户确认后启用。

**自动化限制：**
1. 仅允许执行预先批准好的操作流程。
2. 设定硬性上限：
   - 每天的最大帖子数量
   - 每小时的最大回复数量
3. 禁止发布的类别：
   - 政治相关内容（除非获得明确许可）
   - 法律/医疗/财务建议相关的语言
4. 需要遵守静默时段和冷却时间规则。
5. 必须设置全局关闭开关。

## 必须定义的防护措施
在授予发布权限之前，必须明确以下内容：
1. 账户权限：仅监控 | 草稿+审批 | 有限自动化
2. 每天的最大帖子数量
3. 每小时的最大回复数量
4. 静默时段（根据时区设置）
5. 被禁止的主题/短语
6. 关闭开关的命令及操作者

## 基础操作流程
1. 新闻反应帖（单条推文）
2. 研究总结帖（3–5条帖子）
3. 回复分类处理（待审批队列）
4. 每日总结帖

## 部署检查清单
1. 创建或选择 X 账户。
2. 在本地环境中配置 API 凭据（切勿在聊天中传输）。
3. 先以仅监控模式运行 3–7 天。
4. 进入草稿+审批模式运行 1–2 周。
5. 仅为一个操作流程启用有限自动化。
6. 每周进行审查并调整相关限制。

## 输出格式
### 监控简报
- 发生了什么
- 为什么这很重要
- 建议的应对措施

### 草稿包
- 草稿内容
- 备选方案 A/B
- 风险提示
- 建议的发布时间

### 自动化报告
- 执行的操作
- 被跳过的操作（及其原因）
- 使用的速率限制情况
- 被阻止的违规行为
