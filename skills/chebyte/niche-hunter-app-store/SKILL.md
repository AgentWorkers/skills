# NicheHunter Ultra — 市场情报模式

**适用场景：**  
- 在VPS上运行的OpenClaw（无界面显示）  
- 需要与Telegram聊天系统进行交互  
- 不需要使用交互式浏览器  

---

# 所需工具  

| 工具        | 必需条件 | 用途                                      |  
|------------|------------|------------------------------------------|  
| `web_search`   | 必需        | 用于查找图表、竞争对手信息、用户评价及收入数据             |  
| `web_fetch`    | 可选        | 从指定URL中提取结构化内容                      |  

**注意：** 如果`web_search`不可用，请停止执行后续流程。  

---

# 执行规范（严格遵循）  

- 最多允许执行18次`web_search`请求。  
- 总共分析的URL数量不得超过20个。  
- 每个细分市场最多分析8个竞争对手。  
- 每个应用最多分析20条用户评价（优先考虑评分1星和3星的评价）。  
- 禁止重复查询相同的数据。  
- 如果数据受到付费限制，仅尝试使用一个替代来源。  
- 如果收集到的数据质量较低（信号较弱），则降低评估的可靠性。  

---

# 工作流程  

1. **市场细分**  
2. **市场需求分析**  
3. **竞争对手研究**  
4. **市场差距识别**  
5. **定量评分**  
6. **市场情报报告生成**  
7. **产品需求文档（PRD）编写**（用户选择后完成）  

**每个步骤都必须生成相应的检查点。**  

---

# 检查点格式  

--- CHECKPOINT ---  
**步骤：** {1–7}  
**市场细分：** {具体细分市场}  
**分析的顶级竞争对手：** [列出主要竞争对手]  
**选定的机会：** null | “[应用名称]”  
**下一步：** {下一步操作}  
--- END CHECKPOINT ---

# 第1步 — 市场细分  

依次询问以下问题：  
1. 需要解决什么问题？  
2. 该产品是面向消费者还是企业（B2B）？  
3. 产品是否需要大量依赖人工智能技术，还是可以简化设计？  
4. 预计每月的收入目标是多少？  

将这些问题整理成明确的细分市场描述：  
**目标用户群体 + 使用痛点 + 市场背景 + 盈利模式。**  
（此步骤需要生成检查点。）  

---

# 第2步 — 市场需求分析  

使用`web_search`工具查找：  
- 官方应用商店中的产品信息  
- 该细分市场中的热门应用  
- 评分汇总数据  

提取以下信息：  
- 应用名称  
- 评分总数  
- 评分星级  
- 盈利模式  
- 产品定位概述  

将应用分为3–5个细分市场。  

**判断标准：**  
- 评分超过10万：市场主导产品  
- 评分在1万至10万之间：需求旺盛  
- 评分在1万至1万之间：细分市场较为成熟  
- 评分低于500：市场潜力较小或产品表现不佳  
（此步骤需要生成检查点。）  

---

# 第3步 — 竞争对手研究  

针对每个细分市场，分析最多5–8个竞争对手：  
**收集数据：**  
- 评分情况  
- 产品在相关图表中的排名  
- 应用发布时间（如果可获取）  

**财务数据：**  
- 定价策略  
- 订阅费用等级  
- 公开披露的收入信息  

**用户反馈数据：**  
- 用户对产品的投诉（尤其是1星评价）  
- 用户在使用产品时遇到的问题  
- 用户体验（UX）方面的不足  

**收入数据：**  
- 直接收入来源（如果存在）  
- 通过其他方式估算的安装量  

（此步骤需要生成检查点。）  

---

# 收入估算模型（无直接数据时使用）  

**免费模式：**  
**预估安装量 = 评分总数 × 100**  

**付费模式：**  
**预估安装量 = 评分总数 × 40**  
**每月收入 = 安装量 × 3% × 订阅价格**  

**评估可靠性：**  
**高 / 中等 / 低**  

**注意：** 不要将估算结果作为事实直接呈现给用户。  

---

# 第4步 — 市场差距识别  

识别以下问题：  
- 用户反复提到的问题  
- 产品缺失的功能  
- 定价过高  
- 用户体验方面的缺陷  
- 未被充分满足的市场需求  

**明确产品的主要优势（1–2个核心差异点）**  
**次要的优势（可选）**  
（此步骤需要生成检查点。）  

---

# 第5步 — 定量评分  

**评分标准（0–10分）：**  
- **市场需求强度**（35%）  
- **市场差距明显程度**（30%）  
- **盈利潜力**（20%）  
- **产品设计简洁性**（15%）  

**加权得分：**  
**市场需求 × 0.35** +  
**市场差距 × 0.30** +  
**盈利潜力 × 0.20** +  
**产品设计简洁性 × 0.15**  

**评分结果必须基于实际观察到的数据。**

# 第7步 — 投资者级别的MVP产品需求文档（用户选择后编写）  

文档结构：  
1. 执行摘要  
2. 市场验证情况  
3. 目标用户群体（2–3个）  
4. 产品的主要优势  
5. MVP功能列表  
6. 界面设计（包含各个功能模块）  
7. 盈利策略  
8. 技术栈：  
  - Expo SDK 5.2及以上  
  - TypeScript  
  - Expo Router  
9. 设计规范：  
  - 必须使用Hex颜色  
  - 字体样式  
  - 组件设计风格  
10. 关键绩效指标（KPIs）  
11. 风险及应对措施  

**编写要求：**  
- 详细描述所有UI组件  
- 提供模拟数据示例  
- 文档内容实用且易于复制  
- 适合Rork团队直接使用  

（此步骤需要生成检查点。）

# 使用Telegram时的注意事项：  
- 避免使用过长的表格  
- 采用项目符号列表格式  
- 将长文本分多段展示  
- 每条消息的字数控制在2000字以内  
- 使用用户熟悉的语言进行沟通  
- 文本内容需简洁明了
