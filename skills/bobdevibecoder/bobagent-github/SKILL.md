# GitHub 技能

使用 `gh` 命令行工具（CLI）与 GitHub 进行交互。当不在 Git 目录中时，务必指定 `--repo owner/repo` 参数；或者直接使用 GitHub 的 URL。

## 提交请求（Pull Requests）

- 检查 PR 的持续集成（CI）状态：
  ```bash
gh pr checks 55 --repo owner/repo
```

- 列出最近的工作流运行记录：
  ```bash
gh run list --repo owner/repo --limit 10
```

- 查看某次工作流的运行情况，并找出哪些步骤失败了：
  ```bash
gh run view <run-id> --repo owner/repo
```

- 仅查看失败步骤的日志：
  ```bash
gh run view <run-id> --repo owner/repo --log-failed
```

## 高级查询的 API

`gh api` 命令可用于获取其他子命令无法提供的数据。

- 获取包含特定字段的 PR 信息：
  ```bash
gh api repos/owner/repo/pulls/55 --jq '.title, .state, .user.login'
```

## JSON 输出

大多数命令都支持 `--json` 选项以生成结构化输出。你还可以使用 `--jq` 来过滤输出内容：
  ```bash
gh issue list --repo owner/repo --json number,title --jq '.[] | "\(.number): \(.title)"'
```
