## 使用场景

当用户明确请求代码实现时，该技能会提供相应的规划、执行指导及验证流程。

## 架构

用户的偏好设置会存储在 `~/code/` 目录中（仅在用户明确请求时才会进行存储）。

```
~/code/
  - memory.md    # User-provided preferences only
```

首次使用时，需要执行以下操作：`mkdir -p ~/code`

## 使用范围

该技能仅用于：
- 提供编码工作流程的指导；
- 存储用户在 `~/code/` 目录中明确指定的偏好设置；
- 读取相关的参考文件。

该技能绝不会：
- 自动执行代码；
- 发起网络请求；
- 访问 `~/code/` 目录或用户项目之外的文件；
- 修改自身的 `SKILL.md` 文件或辅助文件；
- 在用户不知情的情况下自主执行任何操作。

## 核心规则

### 1. 先检查内存配置
如果存在 `~/code/memory.md` 文件，请先读取其中记录的用户偏好设置。

### 2. 用户控制执行过程
- 该技能仅提供指导，不会自动执行任何操作；
- 用户决定何时进入下一步；
- 如果需要委托子代理执行任务，必须得到用户的明确授权。

### 3. 先制定计划再编写代码
- 将用户的需求分解为可测试的步骤；
- 每个步骤都应能够独立验证；
- 详细规划方法请参阅 `planning.md`。

### 4. 全面验证所有内容
- 在执行每个功能后，建议运行相应的测试；
- 对用户界面的任何更改，建议截图保存；
- 在代码交付前，建议执行完整的测试套件。

### 5. 根据用户请求存储偏好设置
- 用户的指令：相应地更新 `memory.md` 文件。

**仅保存用户明确要求保存的信息。**

## 工作流程

```
Request -> Plan -> Execute -> Verify -> Deliver
```

## 常见误区
- **交付未经测试的代码**：务必先进行验证；
- **处理大型代码修改（如 PR）**：应将其拆分为可测试的部分；
- **忽略用户偏好设置**：务必先查看 `memory.md` 文件。

## 自我修改规则

该技能绝不会修改自身的 `SKILL.md` 文件或辅助文件。用户的数据仅会在用户明确请求后存储在 `~/code/memory.md` 中。

## 外部接口

该技能不会发起任何网络请求。

| 接口 | 发送的数据 | 目的 |
|---------|-----------|---------|
| 无 | 无 | 该技能不使用网络接口 |

## 安全性与隐私保护

- **数据存储方式**：
  - 仅保存用户明确要求保存的偏好设置；
  - 数据存储在 `~/code/memory.md` 中。

- **数据传输情况**：
  - 该技能不会向外部传输任何数据。

**该技能不会执行以下操作**：
- 自动执行代码；
- 访问网络或外部服务；
- 访问 `~/code/` 目录或用户项目之外的文件；
- 在用户不知情的情况下自主执行任何操作；
- 未经用户授权，不会委托子代理执行任务。
