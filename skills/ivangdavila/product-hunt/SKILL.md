## 关键的启动陷阱

**时间安排：**
- 启动时间：**太平洋时间凌晨12:01至晚上11:59** — 请注意这不是您的时区 |
- 最佳推广时间：周二至周四；周一为补习模式，周五为周末模式 |
- 开始的4-6小时将决定产品的排名走势 — 请尽早开始推广活动 |

**素材要求（规格必须准确）：**
- 图库图片：1270x760像素 — 第一张图片至关重要 |
- GIF/视频的效果远优于静态图片（时长不超过30秒） |
- 徽标：240x240像素 |
- 标语：最多60个字符，避免使用流行语（如“革命性”、“人工智能驱动”等，否则会被直接忽略） |

**开发者评论：**
- 请分享个人故事，而非公司的宣传内容 — PH社区更倾向于独立开发者 |
- 说明您开发产品的初衷以及遇到的问题 |
- 随着产品更新，及时更新评论内容 |

**注意：**  
- 首段内容可能会被截断 — 请确保首句能够吸引读者的注意力；过长的描述可能会被省略。

## 推广者的信誉系统

**建立信誉：**
- 质量比数量更重要 — 5次高质量的推广活动比20次平庸的推广活动更有价值 |
- 每一次失败的推广都会影响未来的推广效果 |
- 顶尖推广者会有自己的粉丝群，他们会收到通知 — 这是真正的优势 |
- 未与开发者协调就进行推广意味着产品可能在发布当天存在漏洞 |

**推广者的操作规则：**
- 任何人都可以发布推广内容，但经验丰富的推广者会获得更多的关注 |
- 被编辑选为“特色推广”会大幅提升曝光率 |
- 收集用户反馈有助于延长产品推广的效果（即使在发布日后也是如此） |

## 需要避免的社区互动行为

**会被检测并受到处罚的行为：**
- 同一IP/设备多次点赞 → 被视为虚假投票 |
- 新账号集中点赞 → 被视为刷赞行为 |
- 固定的点赞群体（总是互相点赞的50个人） |
- 外部投票交换群组（PH社区会密切监控Slack/Discord等平台） |
- 在评论区请求点赞 → 会被公开曝光 |

**哪些行为被视为垃圾信息：**
- 在多个产品间复制粘贴评论 |
- 无实质内容的通用性祝福语 |
- 向开发者发送大量私信 |
- 对批评采取防御性或攻击性的态度 |

## 竞品分析的注意事项

**模型的局限性：**
- 点赞数并不代表用户数量或付费客户数量（两者之间没有关联） |
- 超过80%的“当日最佳产品”在两年内就会被用户遗忘 |
- PH社区的受众主要是独立开发者、设计师和早期采用者，而非企业用户 |
- 高点赞数但评论少的产品很可能是被人为操纵的推广结果 |

**有用的信息来源：**
- 评论中的情感表达能反映用户的真实反馈 |
- 用户提到的改进需求是改进产品的机会 |
- 开发者的回复速度能体现团队的准备情况 |
- 产品推广的速度（2小时内获得200个点赞 vs 24小时内获得500个点赞）是评估推广效果的重要指标 |

## 范围

本文档涵盖的内容包括：产品发布、推广活动、竞品分析以及社区互动。
