## 设置

如果 `~/garden/` 目录不存在或为空，请阅读 `setup.md` 并按照其中的步骤进行设置。用户使用该技能说明他们对此有兴趣——请自然地开始帮助他们。

## 使用场景

用户需要帮助管理他们的花园：跟踪植物生长情况、记录日常活动、规划种植轮作、诊断问题或查看收获结果。该技能会帮助用户跨季节保持信息的有序记录。

## 架构

所有数据都存储在 `~/garden/` 目录下。具体模板请参阅 `memory-template.md`。

```
~/garden/
├── memory.md      # REQUIRED: context and status
├── climate.md     # Optional: zone, frost dates
├── plants/        # Optional: detailed plant files
├── zones/         # Optional: zone tracking
└── harvests.md    # Optional: yield records
```

初始设置时只需启用 `memory.md` 文件；如用户需要更详细的跟踪功能，可再逐步添加其他文件。

## 核心规则

### 1. 植物信息管理
每种植物在 `plants/` 目录下都有一个对应的文件，其中包含品种、种植日期、所属区域以及养护计划。这些信息按需加载，而非默认显示。

### 2. 区域管理
每个花园区域在 `zones/` 目录下都有一个对应的文件，其中记录该区域的生长条件、当前种植的植物以及种植轮作历史。同一植物家族至少需要间隔 3 年才能再次种植在同一区域。

### 3. 日常活动记录
使用 `log/YYYY-MM.md` 文件记录各种活动，并使用相应的图标进行标记：🌱（种植）、💧（浇水）、🐛（害虫）、🍅（收获）、😂️（修剪）、🌡️（天气事件）。

### 4. 气候信息
用户可以配置气候相关信息（如 USDA 分区及霜冻日期），这些信息用于计算最佳种植时间并接收季节性提醒。

### 5. 收获记录
将收获结果记录在 `harvests.md` 文件中，包括日期、植物名称、区域及收获数量。这有助于跨季节比较植物生长情况并评估不同品种的表现。

### 6. 问题诊断
当用户报告问题时，需查看植物的健康记录、区域环境以及最近的天气情况。具体诊断方法请参阅 `diagnostics.md`。

### 7. 数据存储层次
- **当前关注的数据**：始终优先加载 `memory.md` 文件。
- **植物/区域相关数据**：按需加载。
- **日志档案**：仅用于历史参考。

## 常见问题解答

- “哪些植物需要浇水？”：查看植物的养护计划及最近的记录。
- “现在可以种植什么植物？”：参考霜冻日期和种植轮作规则。
- “叶子为什么变黄？”：参考 `diagnostics.md` 中提供的症状分析方法。
- “显示番茄的种植历史？”：加载对应的植物文件。
- “去年的收获情况？”：从 `harvests.md` 文件中汇总数据。

## 使用注意事项

- **避免误区**：
  - 不要在同一区域内连续三年种植同一植物家族，否则容易引发病虫害。
  - 注意不同区域之间的微气候差异，因为这会影响植物的生长。
  - 发现问题时务必及时记录，否则后期诊断会更加困难。
  - 避免仅根据养护计划浇水，而应根据土壤实际状况来决定。

## 安全与隐私

**数据存储方式**：
- 所有花园数据均存储在 `~/garden/` 目录内。
- 不使用任何外部 API 或网络请求。
- 不进行数据传输、分析或收集。

**该技能产生的文件**：
- 所有数据文件均保存在 `~/garden/` 目录下。
- 如用户明确同意，数据可能也会被记录在用户的工作区文件（如 `~/MEMORY.md`）中。

**该技能不会**：
- 在未经用户确认的情况下推断或保存用户的偏好设置。
- 访问天气相关 API（气候信息需手动配置）。
- 控制硬件或灌溉系统。
- 未经许可不会在 `~/garden/` 目录之外写入数据。
- 不会发起任何网络请求。
