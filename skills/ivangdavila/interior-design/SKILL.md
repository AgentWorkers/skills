## 使用场景

当用户需要关于室内空间的帮助时，例如：翻新规划、家具选择、材料计算、租赁优化、房间布置方案设计或产品可视化等。

## 架构

项目数据存储在 `~/interior-design/` 目录下。具体设置方法请参考 `memory-template.md` 文件。

```
~/interior-design/
|-- memory.md          # HOT: active project, confirmed preferences
|-- spaces/            # Per-room: dimensions, fixed elements, constraints
|-- suppliers.md       # Verified local suppliers with price tiers
+-- archive/           # Completed projects
```

## 数据存储

所有数据均保存在 `~/interior-design/` 目录中。首次使用时需要创建相关数据文件：
```bash
mkdir -p ~/interior-design/{spaces,archive}
```

## 使用范围

本技能仅限于：
- 根据用户提供的尺寸和照片提供建议；
- 根据实际测量数据计算所需材料数量；
- 将用户的偏好信息存储在本地文件中；
- 在指定范围内比较不同的选项。

**本技能绝不执行以下操作：**
- 代用户进行采购或联系供应商；
- 仅凭用户的信息推断其风格喜好；
- 未经确认用户所在地区就推荐产品；
- 未经核实产品库存情况就推荐具体商品。

## 技能文件的自我更新

本技能文件（`SKILL.md`）本身不会被自动修改。用户的偏好设置会保存在 `~/interior-design/memory.md` 文件中。

## 核心规则

### 1. 无尺寸数据 = 无具体建议
系统会默认推荐“标准”家具。但实际上并不存在所谓的“标准”家具。

**在任何家具或布局建议之前：**
**必须先获取用户的尺寸数据**，否则只能提供方向性的指导，绝不能推荐具体的产品或布局方案。

### 2. 建议前需进行本地化处理**
系统会默认推荐全球知名的家具品牌（如 IKEA、West Elm、Wayfair），但会忽略用户的实际地理位置。

**验证流程：**
1. 询问用户的所在国家/城市；
2. 确认这些品牌在该地区是否有销售；
3. 检查推荐的商品是否能够以合理的成本和时间在该地区送达；
4. 注意价格差异（例如，西班牙的 IKEA 产品与美国的 IKEA 产品价格可能不同）。

**绝对不能在没有确认品牌在该地区销售的情况下，直接建议用户“从某品牌购买某产品”。**

### 3. 价格等级的统一性
系统会在同一个建议中同时推荐经济型和高档型商品，这可能导致房间布置显得不协调。

**确定价格等级后，请始终遵循该等级：**
| 用户反馈 | 对应的价格等级 |
|------------------|--------------|
| “经济型”、“预算级”、“IKEA 级别” | 仅推荐经济型产品 |
| “高品质”、“投资级”、“中高端品牌” | 仅推荐中高端产品 |
| “设计师品牌”、“无价格限制” | 仅推荐高端产品 |

**如果用户的预算为 200 欧元，那么所有推荐的商品都必须符合这个预算范围。**

### 4. 确认物品能否通过实际入口
系统会推荐那些能够通过房间入口（如门宽、楼梯间、电梯等）的商品。

**在推荐任何尺寸超过 50 厘米的商品之前：**
- 询问物品能否通过房间的入口；
- 计算物品通过这些入口的可行性；
- 标明哪些商品需要现场组装，哪些可以直接送达。

**如果某件沙发无法通过 75 厘米的门，那么这个推荐就毫无意义。**

### 5. 任务顺序的合理性
系统在提供建议时，可能会忽略某些任务必须先完成的顺序。

**室内装修有一些固定的工作顺序：**
**在推荐任何商品之前，必须先完成某些步骤。**

**绝对不能推荐：**
- 在确认地板是否平整之前就推荐地板材料；
- 在确定电源位置之前就推荐照明设备；
- 在确定照明方式（自然光/人工光源）之前就推荐墙面颜色；
- 在安装地板之前就推荐家具（否则可能导致家具损坏）。

### 6. 照片对展示效果的影响
对于租赁或展示用途，每个设计决策都会影响房间的展示效果。

**拍摄照片时需考虑以下因素：**
- 主题照片（第一张图片）直接影响用户的点击率，因此应优先拍摄这个空间；
- 明亮的颜色会让房间显得更大；深色的墙壁会压缩空间感；
- 照片中的杂乱物品会严重影响房间的整体效果（比实际情况更明显）；
- 窗户的方向会影响最佳拍摄时机。

**具体拍摄要求和检查清单请参考 `photography.md` 文件。**

### 7. 从商业回报的角度出发提供建议
对于租赁或展示用途，应将所有建议视为具有投资回报的行为。

**尽可能量化收益：**
- “升级床品可能使每晚的租金增加 10-15%”；
- “专业拍摄的照片通常能提高 20-40% 的预订率”；
- “以房产价值的 1% 的成本进行房间布置，能在销售时带来 5-10% 的回报”。

**不要仅仅说“这个功能很实用”，而要提供具体的收益计算依据。**

## 常见误区及预防措施

| 误区 | 发生原因 | 预防措施 |
|------|----------------|------------|
| 视觉上的误解 | 系统根据默认尺寸展示商品，而用户未提供具体尺寸 | 必须始终使用用户提供的具体产品规格 |
| 风格偏差 | 系统推荐 Pinterest 上流行的风格，而用户可能有不同的偏好 | 需要根据用户的现有家具或明确的需求来推荐风格 |
| 未考虑产品可用性 | 系统可能会推荐用户所在地区没有销售的商品 | 在推荐任何产品之前，必须确认产品的可用性 |
| 忽视维护问题 | 例如，在光线较暗的房间推荐白色织物或宠物用品 | 需要了解用户的生活习惯和维护需求 |
| 对易过时的元素的盲目跟风 | 推荐三年后可能过时的流行元素 | 对于难以更改的设计元素，需提前提示用户风险 |
**物流问题** | 直接推荐商品而忽略实际的配送和组装问题 | 在每个家具推荐中都必须考虑物流因素 |
