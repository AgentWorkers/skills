## 适用场景

当需要获取桌面应用程序、浏览器页面、模拟器、特定区域、窗口或全屏的截图时，此技能非常有用。这些截图可用于调试、质量保证（QA）、文档编写、发布说明、错误报告、视觉审查，以及前后对比等场景。

本技能的重点在于如何可靠地获取截图，而非截图后的图像编辑。

## 工具选择

| 使用场景 | 推荐工具 | 原因 |
|---------|--------------|-----|
| macOS桌面或窗口 | `screencapture` | 内置工具，可靠性强，支持静默捕获、交互式捕获、区域捕获和窗口捕获 |
| iOS模拟器 | `xcrun simctl io booted screenshot` | 比通用桌面捕获工具更适合模拟器输出 |
| Linux Wayland环境 | `grim` + `slurp` | X11工具在Wayland上可能无法正常工作或表现异常 |
| Linux X11环境/无头CI | `scrot`或浏览器原生捕获工具 | 适用于最小化显示或虚拟显示环境 |
| Windows桌面捕获 | `nircmd savescreenshot`或`Pillow ImageGrab` | 比PowerShell的截图命令更简单易用 |
| 网页或Web应用程序 | `Playwright` | 适用于获取稳定的视口、元素、全页截图，以及带有遮罩效果的截图 |
| 视觉差异对比/截图测试 | 使用`Playwright`并设置固定视口 | 可更好地控制动画效果、光标显示、遮罩效果及测试结果的重复性 |

优先选择最合适的原生捕获工具。当需要确保捕获结果的稳定性、元素定位的准确性或视觉效果的重复性时，再考虑使用浏览器原生工具。

## 核心规则

### 1. 根据需求选择捕获方式，而非凭习惯

- 桌面UI截图通常应使用操作系统原生的捕获工具。
- 网页和Web应用程序应使用浏览器原生的捕获方式，而非浏览器窗口的截图。
- 尽可能使用模拟器自带的截图工具。
- 当只需要捕获特定区域或窗口内容时，使用区域或窗口捕获；只有在需要完整上下文时才使用全屏捕获。

### 2. 在捕获前确保目标内容稳定

- 动态页面应在捕获前稳定下来：等待网络请求完成或相关元素加载完毕，确保字体和过渡效果显示完整。
- 不要在页面实际渲染完成之前就进行截图。
- 对于浏览器捕获，尽可能使用明确的等待条件，而非盲目等待。
- 如果页面无法完全静止，应等待所需的UI状态稳定后再进行捕获。

### 3. 固定视口、缩放比例和主题设置以确保结果的可重复性

- 如果视口、缩放比例、主题或设备分辨率发生变化，截图对比将失去意义。
- 对于浏览器捕获，应在获取基准截图或前后对比截图前先固定视口设置。
- Retina和HiDPI显示器可能显示的像素数量较多，需明确选择所需的输出格式（物理像素还是CSS缩放后的像素）并保持一致性。
- 如果页面的暗/亮模式重要，应分别捕获两种模式的截图，避免混淆。

### 4. 只捕获必要的内容

- 元素、区域或窗口的截图通常比全屏截图更清晰有用。
- 全页截图适用于审计和存档，但长页面的截图难以阅读和对比。
- 对于浏览器操作，元素截图或裁剪后的区域截图通常比全页截图更便于对比。
- 如果截图用于证明问题，应保留足够的上下文信息，以便用户理解截图内容。

### 5. 在捕获前清除不必要的元素

- 遮盖或避免显示不必要的UI元素：光标、加载提示、聊天窗口、通知、加载提示、时间戳和随机生成的内容。
- 在捕获前屏蔽或隐藏敏感信息、个人数据、令牌和内部链接。
- 在使用`Playwright`进行浏览器捕获时，关闭动画效果、隐藏光标和设置遮罩等选项，以确保视觉效果的稳定性。
- 如果某些元素是导致问题的原因，应保留这些元素；否则应将其移除。

### 6. 选择合适的输出格式

- PNG格式适用于截图、用户界面截图、代码输出和文本较多的场景。
- JPEG格式适用于需要保存高质量图像的场景，但不适用于普通截图。
- 当兼容性允许时，WebP格式可用于分享或存储，但若用户期望接收PNG格式的文件，则不应默认使用WebP。
- 除非用户明确要求，否则避免通过JPEG格式对截图进行重新压缩。

### 7. 便于自动化和持续集成（CI）操作

- 在捕获失败时，立即保存截图，然后再尝试或继续执行后续操作。
- 为基准截图和带有时间戳的截图使用固定的文件名。
- 在持续集成过程中，一致的视口设置和稳定的状态比截图的数量更重要。
- 无头环境下应优先使用浏览器原生的截图工具，而非尝试直接捕获主机显示内容。

### 8. 确保截图的实际用途

- 检查重要细节是否可见、清晰，且没有被裁剪掉。
- 确保敏感信息（如密码、链接等）不会出现在截图中。
- 前后对比时，应使用相同的视口设置、缩放比例和主题设置。
- 即使截图在技术上是正确的，但如果对用户来说没有实际意义，那么这个截图也是无用的。

## 高效的截图技巧

- macOS：`screencapture -x out.png`（静默捕获）；`-i`（交互式选择）；`-R x,y,w,h`（指定捕获区域）。
- iOS模拟器：`xcrun simctl io booted screenshot out.png`
- Linux Wayland：`grim -g "$(slurp)" out.png`
- 使用`Playwright`进行页面捕获时，应等待目标状态稳定后再进行截图。
- 使用`Playwright`时需要注意的稳定性设置：固定视口、关闭动画效果、隐藏光标、对敏感区域进行遮罩处理以及保持主题和媒体设置的稳定性。

## 常见错误

- 错误地选择了捕获方式（例如，本应获取元素或页面截图却选择了窗口截图）。
- 在字体、数据或布局尚未加载完成时进行捕获。
- 比较不同视口大小或缩放比例的截图，并将其视为有效对比结果。
- 使用JPEG格式截图时，会导致文本、边缘和代码部分模糊不清。
- 时间戳、光标闪烁、通知或随机内容会影响视觉对比结果的准确性。
- 忘记了Wayland环境可能影响X11截图工具的正常使用。
- 共享的截图中仍包含敏感信息（如密码、链接等）。
