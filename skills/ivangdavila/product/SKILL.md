## 使用场景  
- 创建产品（数字SaaS服务或实体商品）  
- 使用人工智能生成产品原型和图片  
- 优化产品在电商平台（如Amazon、Etsy、Shopify）上的展示效果  
- 验证产品创意并评估产品与市场的匹配度  
- 制定定价策略并进行产品发布规划

## 核心工作流程  

**数字产品（SaaS服务）：**  
1. 验证产品创意 → `strategy/validation.md`（进行用户访谈、快速测试）  
2. 定义最小可行产品（MVP） → 确定产品的核心价值主张  
3. 确定功能优先级 → `strategy/prioritization.md`（使用RICE评分法）  
4. 发布产品 → 开展测试计划，快速迭代  
5. 测量产品市场契合度（PMF） → `strategy/metrics.md`（根据Sean Ellis的研究，PMF指标的重要性超过40%）  

**实体产品：**  
1. 设计产品外观 → 确保符合设计文件规范（DFM）  
2. 选择合适的供应商 → 查阅`manufacturing/compliance.md`以获取各地区所需的认证信息  
3. 使用人工智能工具制作产品图片 → 参考`visuals/prompts.md`  
4. 根据不同平台要求准备产品展示文件 → 存放在`commerce/`目录下  
5. 发布产品 → 实施限量销售策略或设置稀缺性机制  

## 重要规则：  
- **先验证需求再开发** —— 必须先测试市场需求  
- **平台规格至关重要** —— 图片尺寸不符合要求可能导致产品展示失败  
- **定价需包含所有费用** —— 计算出所有平台费用后的实际利润率  
- **在扩大规模前确保产品市场契合度** —— 根据Sean Ellis的研究，如果产品市场契合度低于40%，用户满意度会非常低
