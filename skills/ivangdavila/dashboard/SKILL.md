## 数据存储

```
~/dashboard/
├── registry.json           # Dashboard index
├── {name}/
│   ├── config.json         # Layout, widgets
│   ├── data.json           # Current data
│   └── index.html          # Dashboard page
```

首次使用时执行以下操作：`mkdir -p ~/dashboard`

## 功能范围

该功能包括：
- ✅ 生成静态 HTML 仪表板
- ✅ 创建用户可运行的数据获取脚本
- ✅ 将仪表板存储在 `~/dashboard/` 目录下

**用户驱动的流程：**
- 用户指定数据来源
- 用户通过环境变量提供 API 凭据
- 用户运行数据获取脚本（可以通过定时任务或手动方式）
- 该功能会自动生成 HTML 仪表板和数据获取脚本

**该功能不会执行以下操作：**
- ❌ 在用户未提供凭据的情况下访问数据
- ❌ 自动执行数据获取操作（数据获取脚本由用户通过定时任务触发）
- ❌ 未经用户同意抓取外部服务的数据

## 核心规则

### 1. 用户提供数据
在创建仪表板时，用户需要提供所需的数据来源信息。

### 2. 架构
该功能由代理程序生成数据获取脚本，用户负责运行这些脚本。

### 3. 数据获取脚本模板
数据获取脚本的模板位于相应的文件中。

### 4. 视觉质量检查（交付前）
- 在浏览器中打开生成的仪表板并截图
- 检查内容是否有重叠、字体是否清晰可读（字号≥14px）、对比度是否合适
- 如发现问题，需修复后再进行验证
- 只有通过视觉质量检查后，仪表板才能正式交付给用户

### 5. 设计默认值
- 背景颜色：`#0f172a`（深色）/ `#f8fafc`（浅色）
- 文本颜色：`#e2e8f0`（深色）/ `#1e293b`（浅色）
- 内容间距：16px、24px、32px
- 边角圆角：8px
- KPI（关键绩效指标）的显示方式：数字大小为 48-72px，标签大小为 14px

### 6. 安全性
- 凭据信息仅通过环境变量传递，绝不存储在文件中
- 仪表板默认部署在 `127.0.0.1` 这一本地地址上
- 显示的数据中不包含任何个人身份信息（PII）
- 如果需要将仪表板暴露到网络中，用户需要自行添加身份验证机制
