## 核心原则

### 1. 留存客户优先于获取新客户  
- 保留现有客户比寻找新客户更经济实惠。  
- 客户留存率提高 5% 可带来超过 25% 的利润增长。  
- 在扩大业务规模之前，先解决客户流失问题。  

### 2. 主动应对而非被动反应  
- 在客户提出投诉之前主动联系他们。  
- 客户参与度下降预示着流失风险。  
- 定期与客户沟通，不要等到问题出现才采取行动。  

### 3. 关注实际价值，而非记录的活动量  
- 关键是客户是否获得了实际价值，而非客服人员的活动记录。  
- 关注客户成功的结果，而非客服人员的操作记录。  
- 如果客户没有感受到价值，其他一切都无关紧要。  

### 4. 有针对性地分配资源  
- 并非所有客户都值得同等程度的关注：  
  - 为企业客户提供深度服务，为小型企业客户提供基础支持；  
  - 根据客户的收入潜力合理分配资源。  

### 5. 收入增长需通过努力获得  
- 在提出更多需求之前，先证明产品的价值。  
- 选择合适的时机进行业务拓展——在客户满意度最高的时期进行。  
- 在客户取得成功后，再开展交叉销售和升级销售。  

### 6. 客户健康状况能预示一切  
- 建立一个能够预测客户流失的评分系统。  
- 优先关注领先指标，而非滞后指标；  
- 随着市场变化，每季度更新评分模型。  

### 7. 高层领导的参与与共识  
- 了解真正的业务决策者（经济决策者），而不仅仅是用户需求。  
- 建立多方面的合作关系；  
- 业务成果比产品功能的采用更为重要。  

## 绩效指标框架

| 指标 | 测量方法 |
|--------|----------|
| GRR（Gross Retention Rate） | 总客户留存率（保留现有收入） |
| NRR（Net Retention Rate） | 净客户留存率（新增客户减去流失客户） |
| Time to Value | 新客户入职后的效果评估 |
| Health Score | 客户健康状况评分（预测流失风险） |
| Logo Churn | 客户流失率（流失客户的比例） |

## 不同阶段的客户成功工作重点

| 阶段 | 工作重点 |
|-------|-------|
| 初创期 | 创始人主导客户成功工作，手动管理客户留存 |
| 种子期 | 首次招聘客户成功经理（CSM），收集基本客户健康数据 |
| A 轮融资期 | 客户服务团队建设，客户细分 |
| B 轮及以上融资期 | 规模化运营，建立预测模型，明确收入责任 |

## 常见误区  

- 仅关注活动记录而忽视实际价值  
- 一刀切地对待所有客户（将企业客户与小型企业客户同等对待）  
- 问题出现后才采取应对措施  
- 过度关注净推荐值（NPS）而忽视实际结果  
- 忽视产品本身的问题（客服人员无法解决产品缺陷）  

## 需要人为判断的情况  

某些决策需要人为判断：  
- 与高价值客户协商续约条款  
- 对关键客户进行战略性的资源调配  
- 为续订服务制定特殊定价政策  
- 召开高层业务评审会议
