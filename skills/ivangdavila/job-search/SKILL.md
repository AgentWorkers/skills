## 使用场景

用户处于求职阶段：搜索职位、申请工作或准备面试。而代理则负责跟踪求职机会、研究公司信息、准备申请材料以及协助面试准备。

## 架构

所有与求职相关的信息都存储在 `~/job-search/` 目录下。具体设置方法请参阅 `memory-template.md` 文件。

```
~/job-search/
├── memory.md          # HOT: preferences, target criteria
├── applications.md    # Active pipeline
├── companies.md       # Research on target companies
├── materials/         # CV versions, cover letters
└── archive/           # Closed applications
```

## 核心原则

### 1. 重质量而非数量
- 5份有针对性的申请比50份泛泛而谈的申请更有效。
- 每份申请都需要根据公司的具体情况进行个性化定制。
- 切勿盲目投递申请——这会损害你的职业声誉。

### 2. 核实“远程工作”信息的真实性
在推荐远程职位之前，请务必核实以下内容：
- 地理限制（例如“仅限美国”、“要求使用欧盟时区”）
- 合同中的税务/法律条款
- 实际时区是否与用户期望相符。

### 3. 识别危险信号
| 信号 | 可能的含义 |
|--------|---------------|
| 招聘信息发布超过3个月 | 该职位可能已经不再招人或公司人员流动频繁 |
| 使用夸张的表述（如“顶尖公司/高手团队”） | 可能暗示公司存在过度工作的文化 |
| 薪水描述模糊（如“具有竞争力”） | 实际薪资可能低于市场水平 |
| 离职率高的公司 | 公司稳定性较差 |
| 最近大规模裁员 | 公司可能存在不稳定因素 |

### 4. 保持用户信息的真实性
- 所有申请材料都应体现用户的个人风格，而非机器生成的文本。
- 要求用户提供写作样本以匹配申请文件的风格。
- 切勿为了追求关键词而牺牲内容的真实性。
- 招聘人员能够识别出人工智能生成的内容——个性化非常重要。

### 5. 跟踪申请进度
在 `~/job-search/applications.md` 文件中记录以下信息：
- 公司名称、职位名称、申请日期
- 当前申请状态、下一步行动
- 联系方式、面试日期
- 需要的跟进提醒

### 6. 在申请前进行充分调研
对于每家目标公司，收集以下信息：
- 公司的财务状况（资金状况、收入趋势）
- Glassdoor上的用户评价（确保信息是最新的）
- 最近的新闻动态（如裁员、收购事件）
- 如果可能的话，了解招聘经理的背景信息

### 7. 根据用户实际情况调整策略
| 用户类型 | 优先事项 |
|-----------|----------|
| 经验丰富的求职者（10年以上工作经验） | 利用人脉资源、谨慎决策、进行薪资谈判 |
| 初级求职者/应届毕业生 | 优先选择质量较高的职位、适合应届生的公司 |
| 转行者 | 强调可转移的技能以及适合过渡性职位 |
| 有紧急需求的求职者 | 注重申请速度、寻找临时性工作或立即获得收入的机会 |

## 常见误区

- **过度优化申请材料会破坏真实性**——虽然关键词填充能通过系统筛选，但人类招聘人员仍能察觉到机器生成的文本。
- **薪资数据容易过时**——请确保提供的薪资范围是当前的，而非两年前的数据。
- **对“完美匹配”的过度自信**——即使60%的职位要求符合，也仍有可能被拒绝。
- **脱离实际提供的求职建议**——没有合适的引荐或沟通策略，单纯的网络拓展往往难以成功。
- **对紧急需求提供长期建议**——在当前急需工作的情况下，“建立个人品牌”无济于事。
