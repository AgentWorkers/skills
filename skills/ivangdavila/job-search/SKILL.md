## 使用场景

用户正在寻找工作：搜索职位、申请公司或准备面试。代理则负责跟踪招聘机会、研究公司信息、准备申请材料以及协助面试准备。

## 架构

所有数据存储在 `~/job-search/` 目录下。具体设置请参考 `memory-template.md` 文件。

```
~/job-search/
├── memory.md          # HOT: preferences, target criteria
├── applications.md    # Active pipeline
├── companies.md       # Research on target companies
├── materials/         # CV versions, cover letters
└── archive/           # Closed applications
```

## 核心原则

### 1. 重质量而非数量
- 5份有针对性的申请比50份泛泛而谈的申请更有效。
- 每份申请都需要根据公司的具体情况进行个性化定制。
- 切勿盲目投递——否则会损害个人声誉。

### 2. 核实“远程工作”的真实性
在推荐远程职位之前，请核实以下信息：
- 地理限制（例如“仅限美国”、“需要欧盟时区”）
- 合同中的税务/法律条款
- 实际时区是否与用户的期望相符。

### 3. 识别风险信号
| 信号 | 可能的含义 |
|--------|---------------|
| 招聘信息发布超过3个月 | 可能是虚假职位或公司人员流动频繁 |
| 使用夸张的表述（如“明星团队”） | 可能暗示过度工作的企业文化 |
- 薪水描述模糊（如“具有竞争力”） | 实际可能低于市场水平 |
| 声称“年轻且充满活力的团队” | 存在年龄歧视的风险 |
| 最近有大规模裁员 | 公司稳定性不佳 |

### 4. 保持用户语言风格
- 所有申请材料都应体现用户自身的特点，而非机器生成的通用模板。
- 要求用户提供写作样本以匹配申请的语气。
- 切勿为了追求关键词而牺牲内容的真实性。
- 招聘人员能够识别出机器生成的内容——个性化至关重要。

### 5. 跟踪申请进度
在 `~/job-search/applications.md` 文件中记录以下信息：
- 公司名称、职位名称、申请日期
- 当前申请状态、下一步行动
- 联系方式、面试日期
- 需要的跟进提醒

### 6. 在申请前进行充分调研
对于每家目标公司，收集以下信息：
- 公司的财务状况（资金状况、收入趋势）
- Glassdoor上的用户评价（确保信息是最新的）
- 最近的新闻动态（如裁员、收购等）
- 如果可能的话，获取招聘经理的个人信息

### 7. 根据用户情况调整策略
| 用户类型 | 优先事项 |
|-----------|----------|
| 经验丰富者（10年以上工作经验） | 利用人脉资源、谨慎决策、进行薪资谈判 |
| 初级员工/应届毕业生 | 选择质量较高的公司，优先考虑适合入门级的职位 |
| 职业转换者 | 强调可转移的技能，寻找过渡性职位 |
| 有紧急需求的人 | 注重申请速度、选择临时性工作或立即能带来收入的职位 |

## 常见误区

- **过度优化申请材料会破坏真实性**——虽然关键词填充能通过系统筛选，但人类招聘者会察觉到机器生成的痕迹。
- **薪资数据容易过时**——请确保提供的薪资范围是当前的，而非两年前的数据。
- **对“完美匹配”的过度自信**——即使60%的条件符合要求，仍有可能被拒绝。
- **脱离实际提供网络建议**——没有合适的引荐策略，冷启动的求职尝试往往失败。
- **对紧急需求的人提供长期建议**——在当前急需工作的情况下，“建立个人品牌”无济于事。
