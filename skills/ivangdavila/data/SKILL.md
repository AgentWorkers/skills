## 使用场景

用户需要执行以下操作：从数据源（数据库、API、文件）中提取数据；清洗和转换复杂的数据集；分析数据以发现规律；可视化结果；或自动化重复性的数据处理任务。代理程序会处理整个数据工作流程。

## 核心操作

- **查询生成**：用户描述所需数据 → 代理程序生成 SQL 语句或查询逻辑，处理数据连接、过滤和聚合操作 → 返回结果或解释执行过程。
- **数据清洗**：加载原始数据集 → 检测问题（空值、重复项、异常值、格式不一致） → 应用相应的修复措施 → 记录所有转换步骤。
- **探索性分析**：新数据集到达 → 生成描述性统计信息、数据分布图、相关性分析 → 发现有趣的模式和异常值 → 提供包含关键发现的总结报告。
- **数据可视化**：分析完成后 → 选择合适的图表类型 → 按要求格式（PNG、SVG、交互式 HTML）导出结果 → 为利益相关者提供可视化展示。
- **定期报告**：一次定义报告内容 → 代理程序按计划自动执行 → 更新图表和指标数据 → 提供包含重点内容的汇总报告。

## 重要规则

- 在应用任何数据转换之前，务必先预览转换结果（展示转换前后的数据对比）。
- 对所有数据转换操作进行详细记录，包括数据来源、具体操作内容及理由。
- 在进行分析之前，务必验证数据类型和范围的正确性（“输入数据的质量直接影响输出结果”）。
- 使用适当的统计方法进行数据分析，并先验证相关假设。
- 生成可复现的分析结果（包括随机数生成方式、版本信息、时间戳等）。
- 明确处理缺失数据的方法，并记录所采用的策略（删除、插补、标记等）。
- 根据数据类型选择合适的图表类型（分类数据、连续数据、时间序列数据）。

## 用户角色与使用场景

| 用户角色 | 使用重点 | 触发条件 |
|------|-------|---------|
| 分析师 | SQL 查询、数据探索、数据洞察 | “这些数据能告诉我们什么？” |
| 工程师 | 数据处理流程设计、数据转换、数据质量控制 | “请清洗这些数据并加载到指定位置” |
| 业务人员 | 关键绩效指标（KPI）监控、仪表盘制作 | “与上一季度相比，我们的表现如何？” |
| 研究人员 | 数据统计分析的严谨性、结果的可复现性 | “这种差异有统计学意义吗？” |
| 开发人员 | 数据库模式设计、API 数据处理、数据类型处理 | “请从这个 JSON 文件中生成数据类型信息” |

有关各用户角色的具体工作流程，请参阅 `patterns.md`。

## 首次使用指南

1. 确定数据来源（数据库、文件、API）。
2. 建立数据连接或加载数据文件。
3. 进行初步的数据探索与分析（了解数据结构、数据类型及存在的问题）。
4. 根据用户需求进行数据清洗和转换。
5. 根据用户目标进行分析或数据可视化处理。
