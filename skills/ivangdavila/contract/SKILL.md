## 角色

通过结构化的流程起草合同。收集需求、预见潜在问题、生成合同条款、管理合同版本。该技能与“contracts”技能（用于管理已签订的合同）不同。

**主要区别：**  
该技能负责创建合同，而“contracts”技能则负责管理已签署的合同。

```

│   └── {contract-name}/
│       ├── current.md          # ALWAYS read this (latest version)
│       ├── versions/           # v001.md, v002.md, ...
│       ├── intake.md           # Discovery answers
│       ├── parties.md          # All parties + interests
│       └── notes.md            # Negotiation log
├── {contract-name}/            # Finalized (compatible with contracts skill)
│   ├── executed.pdf
│   └── meta.md
```

**版本管理规则：**  
**严禁直接修改合同内容！**  
1. 将合同内容复制到“versions”文件夹中；  
2. 为版本文件添加版本号；  
3. 修改复制后的文件；  
4. 更新原始的 `current.md` 文件。

## 流程概述

1. **发现阶段**：  
   - 确定合同类型、相关方及管辖区域；  
   - 加载 `intake.md` 文件并提出问题；  
   - 未获得完整信息前不得继续下一步。  

2. **结构设计阶段**：  
   - 根据合同类型定义各章节内容；  
   - 检查合同中是否包含强制性条款。  

3. **起草阶段**：  
   - 逐条生成合同条款；  
   - 对关键条款提供多种可选方案。  

4. **审查阶段**：  
   - 分析潜在风险，排查模糊不清的内容，确保合同内容的连贯性。  

5. **协商阶段**：  
   - 若涉及多方参与者，需记录各方立场并提出折中方案。  

6. **最终定稿阶段**：  
   - 需获得相关人员批准；  
   - 完成起草后，将合同文件移至 `~/contracts/{name}` 目录中（其中 `{name}` 为合同名称）。  

详细流程要求请参阅 `phases.md` 文件。

**职责范围限制**  

- **不提供法律建议**：  
  无法评估合同在特定司法管辖区的有效性。  

- **需咨询律师的情况**：  
  包括并购交易、证券相关事务、劳动纠纷、政府合同及专利相关事宜。  

- **主动询问而非自行臆断**：  
  若需求不明确，请务必再次核实相关信息。
