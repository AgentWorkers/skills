## 使用场景

当用户需要在腾讯生态系统中选择、比较、实施某个方案或评估相关风险时，可以使用本文档。适用于腾讯云架构的决策、微信（WeChat）和企点（WeCom）的相关决策、小程序的规划、支付与广告相关的事宜、中国地区与全球地区的推广策略，以及在腾讯官方文档信息不明确时的验证工作。

**注意：** 请勿将本文档作为通用的中国市场策略指南使用，也不应将其作为针对特定产品的专业指导文件（除非任务已经明确指向某个单一产品）。

## 架构

所有与腾讯相关的数据和配置信息存储在 `~/tencent/` 目录下。如果该目录不存在，请先运行 `setup.md` 脚本以完成初始化。具体目录结构请参考 `memory-template.md`。

```text
`- decisions.md        # Final recommendations, assumptions, and open risks
```

## 使用要求

- 进行规划、研究或文档验证时无需特殊权限。
- 若需要执行特定操作（如访问腾讯云、微信或合作伙伴的控制台），可能需要获得用户的授权。
- 严禁要求用户在聊天中提供密码、二维码登录凭证、短信验证码、Cookie 或私钥等敏感信息。

## 数据存储

仅保存对后续工作有价值的持久化数据：
- 用户实际使用的产品系列信息
- 用户明确分享的账户或租户名称
- 用户偏好的地区、语言及文档来源
- 需要反复确认的合规性或推广相关要求
- 包含未解决风险及后续处理事项的最终决策记录

**注意：** 严禁存储敏感信息（如密码、账单数据、原始客户数据或复制来的控制台令牌）。

## 核心规则

### 1. 首先明确使用的产品领域
- 将用户请求归类到以下主要领域之一：腾讯企业服务、腾讯云服务、微信生态系统、业务协作、广告与增长、支付服务，或游戏与内容服务。
- 如果用户涉及多个领域，应分别处理每个领域的需求，而不能简单假设一个解决方案适用于所有领域。
- 大多数错误的建议源于对“腾讯服务”定义的模糊性。

### 2. 明确目标地区和受众
- 每项腾讯相关建议都必须明确指出目标受众是中国大陆用户、香港用户、新加坡用户，还是全球用户。
- 同时明确受众类型（消费者、企业团队、开发者、广告商或运营人员）。
- 产品可用性、文档内容、使用指南、支付方式及合规要求因地区和受众而异。

### 3. 优先使用官方文档，再参考翻译内容
- 对于产品功能、使用限制、价格信息、法规声明、SDK 支持或上线要求等，应以腾讯官方发布的文档和产品页面为准。
- 如果英文文档与中文文档存在差异，应记录这些差异，并优先参考最新或更详细的官方来源。
- 绝不要将翻译后的内容视为比官方文档更权威的依据。

### 4. 将业务目标转化为具体产品决策
- 将模糊的请求（如“我们需要在腾讯平台上开展业务”）转化为具体的需求（例如消息传递、身份认证、支付方式、计算资源、对象存储、广告服务、分发渠道或本地化运营等）。
- 在选择具体工具之前，请参考 `decision-framework.md` 中提供的参考框架。
- 当用户需要明确决策时，切勿直接提供产品列表。

### 5. 将腾讯云视为独立平台
- 在规划时，应基于腾讯云的实际服务内容和操作规范进行设计，而非简单套用其他平台的名称。
- 在推荐解决方案前，需说明区域支持情况、账户权限限制、可观测性差异及管理服务之间的权衡。
- 如果某项工作负载依赖于中国大陆的数据存储要求、ICP 许可或跨境流量，必须提前说明。

### 6. 区分研究指导与执行权限
- 规划和验证步骤属于安全操作，无需用户授权。
- 任何涉及登录、修改云资源、提交应用资产或处理支付相关操作的步骤，都必须获得用户的明确授权。
- 请勿假设同一机构同时拥有腾讯云、微信 OA、企点及支付服务。

### 7. 最后生成决策记录
- 每项重要决策都应生成简洁的记录，包括最终选择的方案、被排除的方案、假设的前提条件、存在的障碍以及仍需人工确认的事项。
- 仅将这些关键决策信息保存在 `~/tencent/` 目录下，以确保后续工作的连贯性。

## 常见错误

- 将腾讯、腾讯云、微信、企点及 QQ 视为可互换的产品系列 → 导致错误的资源分配和责任归属。
- 假设国际版本的文档与中国大陆版本的文档完全一致 → 会遗漏使用限制、入门指南或价格细节。
- 仅基于 AWS 的类比来推荐腾讯云服务 → 尽管服务名称相似，实际操作可能存在显著差异。
- 直到最后才考虑中国地区与全球地区的推广差异 → 可能导致数据存储位置、ICP 许可、语言或支付方面的问题。
- 仅依赖非官方的英文博客文章作为决策依据 → 这类信息可能过时或简化了实际情况。
- 假设一个企业账户可以管理所有腾讯相关服务 → 实际上各服务通常由不同的部门或机构负责。

## 安全与隐私

**数据传输：**
- 发送到腾讯官方文档或产品页面的搜索关键词和请求信息。
- 当官方文档不完整时，可向用户授权的第三方来源发送补充信息。

**数据存储：**
- 用户的激活设置和腾讯使用历史记录保存在 `~/tencent/` 目录下。
- 账户信息、地区默认设置及决策记录保存在本地 Markdown 文件中。
- 仅保存可信来源的补充信息和推广相关限制。

**注意事项：**
- 严禁在本地 Markdown 文件中存储敏感信息。
- 严禁在未确认地区适用性的情况下声称产品可在全球范围内使用。
- 未经用户授权，严禁直接登录腾讯相关平台。
- 严禁修改本文档的定义或内容。

## 信任原则

本文档依赖腾讯官方提供的网站及经授权的第三方来源进行验证。只有在信任这些来源的情况下，才能安装和使用本文档。

**适用范围：**
- 本文档仅用于将与腾讯相关的请求正确路由到相应的产品领域。
- 通过官方文档进行腾讯相关选项的比较。
- 规划腾讯云服务、微信生态系统的推广策略，并在 `~/tencent/` 目录中维护必要的本地配置信息。

**禁止行为：**
- 严禁自行虚构产品功能或地区可用性信息。
- 严禁将非官方的总结内容作为最终决策依据。
- 严禁要求用户提供密码、二维码登录凭证、Cookie 或敏感信息。
- 严禁在未经用户授权的情况下执行任何可能修改账户状态的操作。
