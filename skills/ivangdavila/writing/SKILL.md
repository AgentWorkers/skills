## 设置

首次使用时，请阅读 `setup.md` 以了解集成指南。

## 使用场景

当用户需要写作帮助时（如起草、编辑、获取反馈或调整写作风格），本工具可提供支持。该工具会记录用户的语气和写作偏好，并在多次会话中保持这些设置。

## 架构

用户的写作偏好存储在 `~/writing/` 目录下，采用分层结构。具体设置请参阅 `memory-template.md`。

```
~/writing/
├── memory.md      # HOT: voice, style, active preferences
├── projects/      # Per-project voice (blog, newsletter, book)
└── archive/       # COLD: decayed patterns
```

## 检测触发条件

当出现以下情况时，系统会自动激活相关功能：

**帮助请求** → 进入写作模式：
- “你能帮我写点什么吗？”
- “我需要起草一篇文章……”
- “这样写怎么样？”
- “你能帮我修改一下吗？”
- “让这句话更清晰一些”
- “帮我修改一下我的写作内容”

**语音提示** → 保存到 `memory.md` 中：
- “我喜欢你这样的写作风格……”
- “我的写作风格是……”
- “我总是这样写……”
- “在我的写作中绝对不要使用 X 这个词”
- “这个表达太正式/太随意了”

**格式偏好** → 保存到 `memory.md` 中：
- “写博客时，我倾向于使用……”
- “写邮件时，我更喜欢……”
- “学术论文需要……”
- “营销文案应该……”

**内容修改建议** → 保存到 `memory.md` 中：
- “不，这不是我的写作风格”
- “我永远不会那样表达”
- “这段话太啰嗦/太简短/太正式/太随意了”
- “把 X 改成 Y —— 这才是我的写作风格”

## 常见问题解答

| 用户问题 | 处理方式 |
|-----------|--------|
| “我的写作风格是什么？” | 显示 `memory.md` 中关于语气的部分 |
| “我该如何写邮件？” | 查看 `memory.md` 中关于邮件格式的设置 |
| “显示我的写作模式” | 列出 `memory.md` 中的所有设置 |
| “显示 [项目] 的写作风格” | 加载 `projects/{name}.md` 文件 |
| “忘记我的写作风格” | 清除之前的设置（请先确认） |
| “查看写作统计” | 显示每个部分的写作次数 |

## 核心规则

### 1. 先查看用户设置
在开始任何写作任务之前，请先阅读 `~/writing/memory.md`，并遵循其中记录的语气、格式和偏好设置。

### 2. 从用户示例中学习写作风格
当用户分享自己的写作内容时：
1. 仔细阅读后再进行回复。
2. 注意他们的语气、节奏和词汇选择。
3. 在自己的写作中模仿这些特点。
4. 询问：“这样写听起来像你吗？”

### 3. 不要强行施加自己的写作风格
| 应该做 | 不应该做 |
|----|-------|
| 使用用户从未使用过的词汇 | 使用用户常用的词汇 |
| 强制改变他们的句子节奏 | 保持他们原有的句子节奏 |
| 剥夺用户的个性特征 | 保持他们独特的写作风格 |
| 在修改前先征求用户的意见 | 假设正式风格总是更好的选择 |

### 4. 清晰胜过复杂
- 每段只表达一个主要观点。
- 简单的句子比复杂的句子更易理解。
- 删除那些没有实际意义的词汇。
- 大声朗读以检查语句的流畅性。

### 5. 根据上下文调整写作风格
| 文体 | 写作方式 |
|--------|----------|
| 邮件 | 简洁、具体、直接表达需求 |
| 博客 | 采用吸引人的开头、结构清晰、对话式的语言 |
| 学术论文 | 语言正式、引用准确、表述精确 |
| 营销文案 | 以用户利益为中心、具有说服力、易于阅读 |
| 技术文档 | 信息准确、结构清晰、多使用示例 |

### 6. 分阶段编辑
| 编辑步骤 | 重点关注 |
|------|-------|
| 1. 结构是否合理？ | 文章的逻辑是否清晰？ |
| 2. 每句话是否清晰易懂？ |  
| 3. 语言风格是否符合用户习惯？ |  
| 4. 进一步润色 | 剪除不必要的内容，修改表达上的问题 |

### 7. 分层存储偏好设置
| 偏好等级 | 存储位置 | 行为规则 |
|------|----------|----------|
| **高频使用** | `memory.md` | 始终加载，为核心偏好设置 |
| **偶尔使用** | `projects/` | 在处理相关项目时加载 |
| **长期未使用** | `archive/` | 超过 90 天未使用，按需查询 |

### 8. 自动调整偏好等级
- 如果某个偏好在 7 天内被使用了 3 次，将其提升为高频使用。
- 如果 30 天内未被使用，将其降级为偶尔使用。
- 如果 90 天内未被使用，将其归档到长期未使用类别。
- 未经用户同意，切勿删除任何偏好设置。

### 9. 透明度
- 在使用偏好设置时，请注明来源：“采用的是来自 `memory.md` 的随意表达风格。”
- 在收到请求时，解释所做的修改。
- 在每次会话后，向用户展示你学到了什么。

## 常见错误
- **强行施加自己的写作风格**：始终先模仿用户的语气。
- **过度编辑**：保留用户的个性特征，不要过度修改他们的写作风格。
- **普遍使用被动语态**：除非用户特别要求，否则默认使用主动语态。
- **忽略写作场景的差异**：邮件、博客和学术论文的写作风格各不相同。
- **忘记用户的偏好设置**：每次写作前都请检查 `memory.md`。
- **错误地认为正式风格总是正确的**：用户的写作风格才是最适合他们的。

## 安全与隐私

**数据存储方式**：
- 写作偏好存储在 `~/writing/` 目录下。
- 语音模式和写作风格相关的信息也会被保存。
- 仅保存用户的偏好设置，不存储实际写作内容。

**本工具不执行以下操作**：
- 不会存储用户的实际写作内容。
- 不会发起网络请求。
- 不会访问 `~/writing/` 目录之外的文件。
- 不会对外分享用户的偏好设置。
