## 快速参考

| 文件 | 用途 |
|------|---------|
| `profiles.md` | 根据用户类型（执行者、自由职业者、家长、学生、创始人）进行配置 |

## 数据存储

数据存储在 `~/planner/` 目录下：
- **config** — 用户配置信息、能量使用时段、约束条件 |
- **today** — 当天计划（每日更新） |
- **commitments** — 已有的承诺事项及后续任务 |
- **weekly** — 带有截止日期的周度概览 |
- **archive/** | 过去的计划记录 |

## 代理的功能

| 用户指令 | 代理操作 |
|-----------|--------------|
| “规划我的一天” | 根据优先级和用户精力状况生成时间安排 |
| “有什么紧急事项？” | 从日历/任务列表中筛选出最紧急的事项，并显示截止日期 |
| “保护我的早晨时间” | 防止干扰，安排深度工作，启用“DND”模式（Do Not Disturb） |
| “我向某人承诺了某事” | 记录该承诺事项及其截止日期，并设置提醒 |
| “我是否任务过多？” | 分析当周的工作负担，检测冲突，建议调整计划 |
| “进行每周回顾” | 总结已完成和待完成的任务，调整下周的计划，并归档已完成的事项 |

## 核心工作流程

**早晨（可配置时间）：**
1. 获取日历事件、待办任务以及已有的承诺事项 |
2. 应用用户配置的规则（如能量使用时段、约束条件） |
3. 确定当天的优先级事项（必须完成的三项） |
4. 生成简报（最多包含五项内容，优先处理紧急事项）

**白天：**
- 从对话中记录新的承诺事项 |
- 在截止日期临近时发出提醒（48小时、24小时、2小时前） |
- 对干扰进行分类处理（根据配置的紧急程度）

**晚上：**
- 总结当天完成的任务和进展中的事项 |
- 记录当天的承诺事项 |
- 预览第二天的计划

详细的工作流程请参阅 `routines.md`。

## 优先级规则：
- **仅显示前三项优先级**：如果用户列出的任务超过三项，系统会自动进行排序 |
- **重要事项优先于紧急事项**：截止日期的压力并不等同于任务的影响程度 |
- **合理安排任务时间**：将困难任务安排在用户精力高峰时段，将行政类任务安排在精力较低时段 |
- **保留缓冲时间**：避免将所有任务安排得过于紧密，预留20%的空闲时间

优先级规则的具体内容请参阅 `priorities.md`。

## 基于用户类型的代理行为

代理会根据用户类型自动调整其行为：
- **执行者**：以日历为依据，协助安排会议、提供任务分配建议 |
- **自由职业者**：关注项目进度，跟踪截止日期 |
- **家长**：平衡工作与家庭时间，协调各项事务，制定应急计划 |
- **学生**：根据学术日程安排学习时间 |
- **创始人**：多任务管理，过滤干扰，保护深度工作时间

各用户类型的详细行为规范请参阅 `profiles.md`。

## 承诺事项跟踪

用户所做的每一项承诺都会被自动记录：
- 从对话中提取相关信息（例如：“我会在Y时间之前发送X文件”） |
- 将承诺事项及其截止日期添加到 `commitments.md` 文件中 |
- 在截止日期前发送提醒（可配置：24小时、48小时） |
- 对逾期未完成的承诺进行标记

承诺事项的跟踪流程请参阅 `tracking.md`。
