## 使用场景

用户希望：创建一门课程以变现专业知识，将现有材料转换为在线格式，发布并销售课程，管理学生并跟踪学习进度，或改进现有的课程。该方法适用于在线、面对面或混合教学模式。

**不适用场景**：作为学生参加课程（请使用 `university` 或 `school` 功能）；没有课程结构的一般性教学活动；仅用于企业合规性培训。

## 工作区结构

所有课程数据都存储在 `~/courses/` 目录下：

```
~/courses/
├── [course-name]/           # One folder per course
│   ├── curriculum.md        # Modules, lessons, objectives
│   ├── content/             # Raw materials, scripts, notes
│   ├── production/          # Videos, slides, downloads
│   ├── marketing/           # Sales page, emails, promos
│   ├── students.md          # Enrollment, progress tracking
│   └── analytics.md         # Metrics, feedback, improvements
├── templates/               # Reusable templates
└── config.md                # Platforms, integrations, defaults
```

## 核心操作

**新课程创建：**  
用户拥有相关专业知识或主题 → 分析现有材料 → 制定课程大纲 → 估算制作时间 → 创建文件夹结构。

**内容转换：**  
用户提供 PDF 文件、录制视频或演示文稿 → 提取关键内容 → 重新组织成课程章节 → 编写教学脚本/大纲。

**创建课程模块：**  
确定课程主题与学习目标 → 编写教学脚本 → 设计练习题/评估方式 → 明确所需辅助材料。

**发布课程：**  
课程内容准备就绪 → 编写销售文案 → 创建邮件发送流程 → 配置课程平台 → 设置价格与访问权限。

**管理学生：**  
跟踪学生注册情况与学习进度 → 识别需要帮助的学生 → 自动发送提醒 → 生成完成证书。

**课程改进：**  
收集用户反馈 → 分析完成率 → 找出问题所在 → 提出改进方案。

## 课程创建流程

| 流程阶段 | 负责人 | 用户职责 |
|-------|------------|-----------|
| 1. 市场调研 | 分析竞争对手，确定课程定位 | 审批课程方向 |
| 2. 课程结构设计 | 根据材料制定课程大纲 | 审查并调整 |
| 3. 内容创作 | 编写教学脚本、设计练习题 | 录制或审阅内容 |
| 4. 材料制作 | 编辑视频、制作幻灯片、生成辅助材料 | 进行质量检查 |
| 5. 平台设置 | 配置课程平台、设置支付方式、测试教学流程 | 最终审批 |
| 6. 课程发布 | 编写销售文案、发送推广邮件 | 审批宣传内容 |
| 7. 课程运营 | 回答常见问题、跟踪学习进度、发送提醒 | 处理学生反馈 |
| 8. 课程优化 | 分析反馈、提出改进建议、创建新内容 | 确定优先改进事项 |

## 重要规则（务必遵守）  

- **切勿跳过市场调研**：在开始制作内容之前，务必了解市场情况与课程定位。  
- **优先利用现有素材**：尽可能基于用户现有的材料进行内容转换。  
- **模块化设计**：每个课程章节时长控制在 10–20 分钟以内，并保持独立性。  
- **必须包含评估环节**：每个模块都应包含评估方式（如小测验、练习题或项目）。  
- **自动化为主**：学生支持服务应实现 80% 的自动化处理，仅20% 由人工完成。  
- **数据驱动的迭代**：通过跟踪完成率、用户反馈和参与度来持续优化课程。  
- **平台无关性**：所提供的建议应适用于任何课程平台（如 Teachable、Thinkific、Kajabi 或自定义平台）。  

## 内容转换快速指南

| 来源材料 | 转换方法 |
|----------------|----------------|
| 长时间录制视频 | **转录** → 按主题分类 → 编写教学脚本 |
| PDF 笔记/幻灯片 | **提取大纲** → 扩展成课程章节 → 添加练习题 |
| 一对一教学录像 | **整理内容结构** → 按主题分组 → 制定课程模块 |
| 会议演讲 | **转录内容** → 重新组织结构以适应课程需求 | 增加深度和细节 |
| 博文/文章 | **按主题分类** → 补充缺失内容 → 添加评估环节 |

## 首次使用时的操作步骤：  

1. 询问用户属于哪种类型的课程创建者（专家、教育工作者、企业培训师或教练）。  
2. 了解用户现有的素材（如录制视频、笔记、幻灯片等）。  
3. 确定目标受众及课程转化目标。  
4. 根据用户提供的素材，建议合适的课程结构。  
5. 在 `~/courses/` 目录下创建相应的文件夹结构。
