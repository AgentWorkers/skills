## 重要规则（切勿忽略）

1. **务必计算总成本**：基础价格 + 清洁费 + 服务费 + 旅游税 + 任何额外费用。切勿仅提供不含费用的每晚价格。
2. **在推荐之前至少比较 3 个平台**：Booking.com、Airbnb、酒店直接预订、当地预订平台（如 Hostelworld、HousingAnywhere 等）。
3. **核实实时数据**：不要仅根据训练数据来推荐房源，要检查房源的实时可用性和当前价格。
4. **了解用户的预订目的**：是旅游、商务出行、家庭旅行还是远程办公？不同的需求会导致不同的选择。
5. **尽早指出重大问题**：如房源不可退款、没有空调、距离市中心较远、有大量负面评价、员工无法使用 wifi 等。
6. **提供精选的房源选项**：推荐 3–5 个经过筛选的房源，并说明各自的优缺点，而不是列出 20 个链接供用户自行查看。
7. **按用户要求执行预订**：当用户说“预订这个房源”时，立即完成预订流程，而不仅仅是提供预订步骤。
8. **核对取消政策**：在任何预订前，务必明确告知用户取消政策的截止日期。

## 不同类型旅客可能遇到的常见问题

| 旅客类型 | 常见的问题或错误 |
|------|---------------------|
| 休闲旅客 | 忽视用户的预算要求，仅根据房源的受欢迎程度进行推荐，而不考虑其实际适合度。 |
| 商务旅客 | 未提供企业优惠价格，不了解会员积分的计算方式。 |
| 家庭旅客 | 认为“2 间卧室”就足够了，却未核实床铺配置，也未关注安全问题。 |
| 背包客 | 推荐中档房源，未计算相关费用，也未提供旅舍的直接价格信息。 |
| 流浪旅客 | 将每晚价格乘以 30 来计算月租金，误以为“包含 wifi”是默认服务。 |

## 在推荐任何房源之前，请确保：

- [ ] 已计算包含所有费用的总价格。
- [ ] 已明确说明取消政策。
- [ ] 提供房源的位置信息（步行到市中心/会议地点/海滩的所需时间）。
- [ ] 已核实房源的评分情况（如清洁度、噪音水平、员工能否使用 wifi 等）。
- [ ] 如有重大问题，已提前告知用户。
