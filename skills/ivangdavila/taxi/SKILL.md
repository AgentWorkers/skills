## 使用场景

当用户需要关于出租车、拼车、汽车服务或地面交通的任何帮助时，代理将负责进行信息调研、价格比较、预订协助、费用监控以及问题解决。

## 架构

所有与出租车服务相关的数据都存储在 `~/taxi/` 目录下。具体设置方法请参阅 `memory-template.md` 文件。

```
~/taxi/
├── memory.md          # Preferences, saved addresses, default apps
├── trips.md           # Trip history and expenses
├── accounts.md        # Which apps user has, preferences per app
└── promos.md          # Active promo codes, referral links
```

## 核心规则

### 1. 先了解用户需求
在给出任何推荐之前，请先查看 `~/taxi/memory.md` 文件中的以下信息：
- 用户偏好的应用程序及其原因
- 保存的地址（家庭、工作地点、机场）
- 支付方式（信用卡、现金、企业账户）
- 特殊需求（如轮椅使用、儿童座椅需求、是否适合携带宠物）
- 对费用的敏感度

### 2. 主动进行比较
| 任务 | 代理应采取的行动 |
|------|--------------|
| 用户需要乘车服务 | 比较至少 3 种选项的价格 |
| 发现价格激增 | 警告用户，并建议等待或选择其他方式 |
| 机场接送 | 比较拼车、班车、出租车和汽车服务的价格 |
| 团体出行 | 计算多人乘坐或使用大型车辆是否更经济 |
| 固定路线出行 | 随时间跟踪价格变化，推荐最佳预订时间 |

### 3. 最大化节省成本
- 在预订前搜索有效的促销代码 |
- 检查用户是否在某个应用程序中拥有推荐积分 |
| 比较预定服务和按需服务的价格 |
- 在时间允许的情况下，建议使用拼车或共享出行方式 |
- 监控价格激增情况，并在价格回落时提醒用户 |
| 查找机场的固定价格选项和按里程计费的选项

### 4. 协助预订流程
代理可以完成以下操作：
- 通过浏览器打开预订网站或应用程序
- 预填上车点和目的地信息
- 根据用户需求选择车型
- 应用促销代码
- 设置预定上车时间
- 引导用户完成支付确认流程

用户需要自行完成的部分包括：
- 最终支付确认
- 输入新的支付方式
- 进行电话验证

### 5. 处理特殊场景
| 特殊场景 | 代理的处理方式 |
|----------|---------------|
| 机场接送 | 核实登机口信息、航班状态，提供接机服务 |
| 商务出行 | 确保具备发票开具功能，了解企业相关费用报销政策 |
| 深夜出行 | 优先选择安全性较高的司机和评分较高的服务 |
| 在国外出行 | 查找当地适用的打车应用程序，确认是否接受现金或信用卡支付，提供语言帮助 |
| 大型团队出行 | 计算分摊费用的方式，寻找适合的面包车或小型巴士，并协调多辆车 |
| 无障碍出行 | 筛选适合轮椅使用的车辆，并在必要时向公司确认相关要求 |
| 携带宠物出行 | 查找适合宠物的出行方式及所需的运输工具 |
| 带有儿童的出行 | 确保有儿童座椅，并选择适合家庭出行的服务 |
| VIP/商务出行 | 提供汽车服务、豪华轿车或专车服务 |
| 活动接送 | 协调接送地点，避免价格激增的情况 |

### 6. 完整记录所有行程信息
将行程信息记录在 `~/taxi/trips.md` 文件中：
- 行程日期、路线、使用的应用程序
- 实际支付金额、应用的促销代码
- 行程类别（个人出行、商务出行、特定项目）
- 备注（如司机迟到、价格激增、遇到的问题）

同时提供以下信息：
- 每月的费用汇总
- 各类别的费用明细
- 与公共交通方式的费用对比
- 年度费用变化趋势

### 7. 解决问题
当用户遇到问题时：
1. 确定问题的类型（如收费过高、路线问题、司机问题、安全问题、物品丢失等）
2. 选择合适的沟通渠道（应用程序内、电子邮件、电话、社交媒体）
3. 撰写包含详细信息的投诉信
4. 跟踪问题的解决进度
5. 如果问题仍未解决，可向上级或相关机构求助（如消费者投诉平台、当地监管机构、社交媒体）

具体投诉处理流程请参阅 `problems.md` 文件。

### 8. 保持信息更新
- 关注用户所在城市新出现的打车应用程序
- 监控促销代码的来源（如 RetailMeNot、Honey 等）
- 关注相关法规的变化（部分城市可能禁止或限制某些服务）
- 了解季节性价格波动（如节假日价格激增、机场建设期间）

### 9. 多种出行方式的整合
有时出租车并非最佳选择：
- 根据成本和时间需求，比较出租车与其他出行方式（如公共交通）
- 建议使用火车加最后一公里的拼车服务
- 对于多日或多站点的出行，可以考虑租车服务
- 机场出行时，可以使用酒店提供的班车服务
- 对于短距离出行，可以考虑步行或拼车服务

### 10. 隐私与安全
- 绝不存储用户的完整支付信息
- 如果用户同意，可在日志中隐藏家庭地址
- 将行程信息分享给紧急联系人
- 在用户上车前核实司机的相关信息

## 常见错误
- 在预订前未查看价格激增情况 → 必须始终显示当前的价格调整系数
- 假设某个应用程序适用于所有情况 → 应先查看当地的替代方案
- 忽略预定服务选项 → 这些服务通常更便宜 10-20%
- 未及时记录商务出行信息 → 之后难以分类和报销
- 直接使用第一个找到的促销代码 → 部分促销代码可能存在冲突，请仔细核对使用条款
- 仅推荐价格最低的服务 → 对于某些出行来说，安全性和可靠性同样重要
- 忽视机场的特殊规定 → 有些机场有指定的接送区域和特殊的价格政策
- 未跟踪航班延误情况 → 用户可能会等待过长，导致额外费用产生
- 未考虑用户之前的出行偏好 → 在提供建议前请先查看用户的历史记录
