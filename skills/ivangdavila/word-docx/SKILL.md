## 使用场景

当主要处理的是 Microsoft Word 文档（`.docx` 格式）时，尤其是当需要跟踪文档变更、添加注释、设置页眉、编号、使用字段、处理表格、使用模板或确保文档兼容性时，应使用该工具。

## 核心规则

### 1. 将 `.docx` 文件视为 OOXML 文档，而非纯文本

- `.docx` 文件实际上是由多个 XML 部分组成的压缩文件，因此其结构与可见文本同样重要。
- 关键文件包括 `word/document.xml`、`styles.xml`、`numbering.xml`、页眉、页脚以及相关关系文件。
- 文本可能分布在多个处理过程中；切勿假设一个单词或句子只存在于一个 XML 节点中。
- 根据需求选择不同的处理方式：使用结构化提取方式便于快速阅读；使用基于样式的生成方式创建新文档；对于现有的脆弱文档，则使用支持 OOXML 的编辑工具。
- 如果主要任务是阅读、提取或审阅文档，请优先选择能够保留文档结构的处理方式，再直接编辑 OOXML 文件。
- 对于深度编辑，应检查文档的整体结构，而不仅仅是渲染后的结果。
- 即使格式相同，阅读、生成和修改已审阅的文档也是不同的操作。
- 传统的 `.doc` 格式文档通常需要转换后才能适应现代的 `.docx` 格式要求。

### 2. 有意识地保留样式和格式设置

- 建议使用命名样式而非直接应用格式设置，以保持文档的可编辑性。
- 不同类型的样式（段落样式、字符样式等）具有不同的行为特性；直接删除格式设置通常比叠加更多内联格式更安全。
- 在编辑现有文档时，应扩展现有的样式系统，而不是创建新的系统。
- 在文档间复制内容时，可能会无意中导入其他文档的样式、主题设置和编号规则。

### 3. 列表和编号是独立的系统

- 列表项和编号规则属于 Word 的内置功能，而非简单的 Unicode 字符；因此不能简单地将它们视为普通文本。
- `abstractNum`、`num` 等属性都非常重要，因此重新生成列表时不能仅依赖视觉效果。
- 缩进和编号虽然相关，但并不完全相同；即使缩进看起来正确，列表的编号也可能出错。
- 在某个编辑器中显示正确的列表，在其他编辑器中可能需要进行重新编号或调整。

### 4. 页面布局依赖于文档的各个部分

- 页边距、页面方向、页眉、页脚和页码设置都是针对特定部分的；因此修改某个部分的设置可能不会影响整个文档。
- 同一个文档中的首页页眉和奇数/偶数页页眉可能不同，因此修改其中一个页眉可能无法同时修改整个文档。
- 应明确设置页面大小，因为 A4 和 US Letter 等默认设置会影响分页和表格宽度。
- 使用分节符来调整页面布局；手动设置间距或使用随机的分页符可能会导致布局混乱。
- 页眉和页脚的样式依赖于特定部分，因此复制这些样式时可能会破坏图像或链接的显示效果。
- 表格、分页符和页眉的设置通常是全局性的，不应仅视为局部调整。
- 表格的布局受页面宽度、页边距和固定宽度的限制；在 Google Docs 或 LibreOffice 中，微小的调整也可能导致布局问题。

### 5. 跟踪变更、添加注释和操作字段时需谨慎

- 启用变更跟踪功能后，可见文本并不代表文档的全部内容。
- 插入、删除和注释会携带元数据，这些元数据可能在粗心的编辑中仍然存在。
- 被删除的文本可能仍然存在于 XML 文件中，即使它在屏幕上不再显示。
- 如果编辑操作破坏了文档的结构，注释标记和审阅范围可能会失效。
- 注释、脚注、书签和链接内容可能分布在文档的不同部分，而不仅仅是主体文本中。
- 目录、页码、交叉引用和邮件合并相关的内容都属于“字段”类别；编辑这些字段时需谨慎操作，因为缓存显示的内容可能需要刷新才能更新。
- 超链接、书签和引用可能会因为 ID 或关系信息的改变而失效。
- 书签、脚注、注释范围和交叉引用的正确性依赖于稳定的引用点；即使可见文本未发生变化，这些元素也可能出错。
- 文档在表面上看起来正常，但实际上可能包含过时的字段数据，这些数据在刷新后可能会发生变化。
- 在审阅流程中，应尽量减少对文本的修改，避免整体重写段落。
- 在支持变更跟踪的流程中，只有被修改的部分应该显示变化；大规模的改写会导致审阅混乱，并可能破坏原有的格式。

### 6. 在交付前验证文档的兼容性

- 复杂的文档在不同软件（如 Word、LibreOffice、Google Docs）和转换工具之间可能存在兼容性问题。
- 表格、页眉、嵌入的字体和复制的样式是导致布局问题的常见原因。
- 将 `.docm` 文件视为包含宏的文件，风险较高；将 `.doc` 文件视为需要先转换的旧格式文档。
- 当布局至关重要时，应明确设置表格宽度，而不是依赖自动适应或百分比格式设置（这些设置可能因软件不同而有所不同）。
- 通过文本检查的文档在接收者打开后，仍可能在分页、表格宽度或引用显示方面出现问题。

## 常见问题

- 复制粘贴操作可能会引入不必要的样式和编号规则。
- 页眉或页脚中的图片依赖于特定部分的设置；盲目使用相同的 ID 可能会导致布局问题。
- 使用空段落作为间距可能会导致模板失效；间距设置应放在段落设置中。
- 看起来整洁的导出文件可能隐藏了未解决的修订内容、注释或过时的字段值。
- 仅凭视觉效果重新生成列表通常会失败，因为列表的编号状态可能存储在文档的其他部分。
- 一个可见的短语可能分布在多个处理过程中，导致书签、修订标记或字段边界出现问题。
- 替换整个段落以修改其中的一部分内容可能会影响审阅质量、书签、注释或附近的格式设置。
- 从段落或列表项中删除所有可见文本后，仍可能留下空段落标记、空列表项或不稳定的编号。
- 表格的自动适应和百分比宽度设置在 Word 中可能看起来正常，但在 Google Docs 或 LibreOffice 中可能会出现问题。
- LibreOffice 和 Google Docs 可能会改变表格、分节行为和嵌入字体的显示效果，即使 Word 中显示正常。
- 兼容模式可能会隐藏新功能的某些特性或改变分页行为。
- 单个页面大小或页边距设置的更改可能会影响表格、页眉、目录和交叉引用的显示。
