## 文件结构

- DOCX 文件实际上是一个 ZIP 文件，其中包含多个 XML 文件：`word/document.xml` 存储主要内容，`word/styles.xml` 存储样式信息。
- 文本被分割成多个文本单元（`<w:r>`），每个单元都具有统一的格式；一个单词可能跨越多个文本单元。
- 段落（`<w:p>`）由多个文本单元组成；切勿将一个段落简单地视为一个连续的文本块。
- 节（section）用于控制页面布局：页眉、页脚、页边距以及页面的方向设置都是针对每个节单独设置的。

## 样式与直接格式化

- 样式（如标题 1、普通文本等）都有明确的名称，并且可以被重复使用；而直接格式化则是内联的，会覆盖已设置的样式。
- 删除直接格式化后，可以查看底层使用的样式，这对于清理文档非常有用。
- 字符样式适用于单个文本单元，段落样式适用于整个段落；这两种样式会叠加在一起共同影响文本的显示效果。
- 链接的样式既可以应用于整个段落，也可以应用于选中的文本，具体取决于使用方式。

## 列表与编号

- 文本编号的实现较为复杂：`abstractNum` 定义了编号规则，`num` 用于引用这些规则，而段落则通过 `numId` 来引用编号。
- 重新开始编号不是自动完成的，需要使用带有“重启”标志的 `<w:numPr>` 标签。
- 列表项和编号使用相同的编号系统（`numId`）。
- 列表的缩进设置与编号是分开控制的；即使列表没有明显的缩进，系统也会正确显示其内容。

## 页眉、页脚与节

- 每个节都可以有不同的页眉和页脚设置；这些设置会分别应用于首页、奇数页和偶数页。
- 节的分界方式会影响页面的连续性（例如，下一个节可能从下一页开始，或者在同一页面上继续显示）。
- 页眉和页脚的信息存储在单独的 XML 文件中，并通过节的属性来引用。
- 页码是动态变化的（在打开文档或打印时才会更新）。

## 文本修订与注释

- 文本修订功能会同时保存原始内容和修改后的内容；用户可以选择接受或拒绝修改才能最终确定文档的最终版本。
- 被删除的文本仍然会以 `<w:del>` 标签的形式存在于文档中；切勿认为所有被标记为删除的文本都已被完全移除。
- 注释通过书签 ID 来标识其覆盖的范围（从 `<w:commentRangeStart>` 到 `<w:commentRangeEnd>`）。
- 修订记录会记录谁修改了哪些内容；即使修改被接受，这些元数据也会保留下来。

## 字段与动态内容

- 文档中的某些内容（如日期）是动态生成的（例如 `{{ DATE \@ "yyyy-MM-dd" }`），而不是静态显示的文本。
- 目录（TOC）、页码、交叉引用等都属于字段类型；需要更新这些字段才能使内容显示最新信息。
- 超链接可以是字段类型，也可以是直接使用的 `<w:hyperlink>` 标签；两种形式都是合法的。
- `MERGEFIELD` 标签用于邮件合并功能；在实际合并之前，这些字段只是占位符。

## 兼容性

- “兼容性模式”会限制某些功能的可用性，使其仅适用于早期版本的 Word 文档；请检查 `w:compat` 设置。
- 不同工具和地区对页面大小的默认设置可能不同；需要明确指定页面大小（如 US Letter 或 A4）；否则分页和表格的宽度可能会出现问题。
- LibreOffice 和 Google Docs 在处理文档格式时可能会有一些差异；建议进行来回测试以确保兼容性。
- 嵌入的字体可能无法在文档之间正确传输；系统会使用替代字体来显示这些字体。
- DOCM 格式包含宏（存在安全风险）；而 DOC 格式是传统的二进制格式。

## 常见的问题

- 为了实现间距效果，不要使用空段落；建议在段落样式中直接设置前后的空格。
- 如果需要控制页面布局，请使用节的分界功能（即使用 `w:sectionBreak`）来手动创建新的段落。
- 当在页眉中插入图片时，需要为每张图片单独设置引用关系；否则相同的图片可能会在多个页眉中重复显示。
- 复制粘贴操作可能会引入源文档中的样式；这可能会导致样式库中出现重复的样式。
