## 使用场景

用户需要代表自己发送消息时，代理应避免犯下人类容易犯的社交错误：使用错误的语气、选择错误的沟通渠道、发送消息的时机不当，或者自动执行某些操作（如自动回复或发送消息）。

## 核心规则

### 1. 绝不要自动发送消息
- 与时间线、价格、法律条款、产品可用性相关的内容 → 必须先草拟供人类审核，切勿自动发送；
- 如果人工智能回复“我们可以在周五前完成”，这可能会对用户的职业发展造成负面影响；
- 任何涉及金钱确认的内容都需要经过每次交易后的明确批准；
- 当对承诺的准确性不确定时 → 请先询问相关人员的意见。

### 2. 高风险情况的处理
- 对于涉及投资者、董事会、媒体或律师的重要信息，必须先草拟供人类审核，切勿自动发送；
- 客户投诉、标有“紧急”、“法律问题”或“令人失望”等字样的消息也需要人工处理；
- 表达慰问、处理关系问题或解决冲突时同样需要人工介入；
- 向新建立的重要联系人发送的第一条消息也必须由人类完成。

### 3. 与人类的沟通风格保持一致
- 在起草消息之前，先阅读对方最近5条消息；
- 不要使用对方从未使用过的表达方式（例如“希望你一切都好！”）；
- 不要使用对方避免使用的表情符号；
- 真正的人类通常会使用简短的回复，而人工智能生成的回复往往较长，因此请调整回复的简洁程度以匹配人类的习惯。

### 4. 选择合适的沟通渠道
| 紧急程度 | 适用渠道 |
|---------|---------|
| 生产系统出现故障 | 先打电话，再通过Slack沟通； |
| 需要在当天处理 | 通过Slack或Teams的私信沟通； |
| 本周内需要处理 | 发送电子邮件； |
| 仅用于提供信息（无需行动） | 发送电子邮件； |
- 对于紧急问题，绝对不要通过电子邮件沟通； |
- 对于正式的客户沟通，绝对不要使用Slack。

### 5. 发送消息的时机至关重要
- 即时回复会暴露出系统是自动处理的；
- 如果接收者在凌晨3点，应安排在早晨发送消息；
- 通过电子邮件发送消息时，合理的回复时间通常是几小时内；通过Slack私信发送时，应期望在1小时内得到回复。

### 6. 了解上下文可以避免沟通失误
- 在发送消息之前，请确保自己处于正确的聊天频道；
- 不要向已经联系过50次的人再次自我介绍；
- 在群聊中，偶尔潜水是正常的，但回复所有消息则显得不自然；
- 如果不确定应该加入哪个群组，请先询问相关人员。

## 常见错误

- 在投诉邮件中抄送上级 → 本应缓和矛盾，反而可能使问题升级；
- 使用“谢谢”等通用回复语回复所有收件人 → 会打扰到50个人；
- 转发带有内部评论的聊天记录 → 会破坏信任关系；
- 在晚上11点发送消息仅仅是为了“完成任务” → 这表明缺乏基本的沟通边界感；
- 在客户还没有使用你的名字之前就使用他们的名字 → 这是一种冒昧的行为。
