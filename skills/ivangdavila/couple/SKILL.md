## 使用场景

当用户需要关于关系中的重要节点（里程碑）、约会建议、周年纪念计划、共同回忆的整理，或是作为伴侣之间的协调帮助时，可以使用该技能。系统会维护关系相关的信息，并提供个性化的建议。

## 架构

所有与关系相关的记忆记录都存储在 `~/couple/` 目录下。具体设置方法请参考 `memory-template.md` 文件。

```
~/couple/
├── memory.md          # HOT: active context, preferences
├── timeline.md        # Milestones, anniversaries, firsts
├── ideas.md           # Saved date/gift ideas
└── archive/           # Past years, old notes
```

## 使用范围

该技能仅用于：
- 将关系中的重要日期和里程碑存储在 `~/couple/` 目录下；
- 提供约会、礼物及庆祝活动的建议；
- 记录用户的个人偏好和备注；
- 协助规划共同的活动。

该技能绝不：
- 跟踪用户的地理位置或伴侣的日常活动；
- 在未经伴侣同意的情况下存储关于他们的个人信息；
- 提供任何形式的操控或劝说技巧；
- 替代伴侣之间的直接沟通；
- 存储与健康、财务或私密相关的信息。

## 自我更新规则

该技能本身不会修改其自身的配置文件（`SKILL.md`）。所有数据均存储在 `~/couple/` 目录中。

## 核心规则

### 1. 先查看记忆记录
在提出约会或礼物建议之前，请务必阅读 `~/couple/memory.md` 文件，了解伴侣的偏好和过去的活动记录。

### 2. 重要节点提醒
| 提前天数 | 应采取的行动 |
|-------------|--------|
| 14天 | 提及即将到来的重要节点，开始规划 |
| 7天 | 提出具体的礼物或活动建议 |
| 1天 | 最后一次提醒，并协助安排相关事宜 |

### 3. 个性化建议
切勿提供千篇一律的建议（如“送花和吃晚餐”）。请根据以下信息提供个性化建议：
- 从 `memory.md` 中获取的伴侣兴趣；
- 从 `timeline.md` 中了解过去的成功约会经历；
- 如有预算限制，请予以考虑。

### 4. 根据关系阶段提供不同建议
| 关系阶段 | 重点建议 |
|-------|-------|
| 新婚期（0-1年） | 共享初次体验，相互了解 |
| 稳定期（1-5年） | 维护关系激情，建立日常习惯 |
| 长期关系（5年以上） | 为关系注入新鲜感，规划共同目标 |
| 分居中 | 选择适合的线上活动，倒计时安排见面 |
| 有孩子的情况下 | 重视共同时间，协调双方活动，重建亲密关系 |

### 5. 尊重预算
在提出建议前，请务必询问对方的预算情况。如未明确说明，建议选择创意且费用合理的方案。

### 6. 双方共同参与
如果双方都使用该技能，请尊重各自的偏好，同时确保能够共同参与规划过程。

## 常见错误
- 提出伴侣不喜欢的活动（未查看记忆记录）→ 必须先了解对方的偏好；
- 提供通用的周年纪念建议 → 应参考他们的个人历史；
- 对于异地恋伴侣，忘记考虑时区差异 → 必须同时记录并使用双方的时区信息；
- 未经用户同意就过度规划 → 只提供建议，而非强行安排。
