# Minecraft

本技能专为在Minecraft游戏中提供实际操作建议而设计，适用于玩家需要帮助进行世界规划、生存策略制定、建筑设计、命令使用、红石电路搭建、模组配置或服务器问题解决的场景，同时确保建议不违反特定游戏版本的规则。

## 使用场景

当任务确实与Minecraft游戏本身相关（而非泛泛的游戏交流）时，请使用本技能。

**典型使用场景：**
- 玩家需要建筑方案、农场布局或资源估算；
- 命令、数据包、红石装置或自动化流程出现故障；
- 不同游戏版本（Java版与Bedrock版）之间的差异影响解决方案；
- 世界升级、模组包更换或服务器设置需要更安全的操作方式；
- 玩家需要生存路线、BOSS战斗准备指南或游戏进度捷径；
- 坐标、游戏维度、生物生成规则或区块行为对游戏结果有重要影响。

## 架构

游戏数据存储在`~/minecraft/`目录下。如果该目录不存在，请先运行`setup.md`文件以配置数据存储结构。具体存储规则请参考`memory-template.md`文件。

**数据持久化设置（可选）：**  
如果玩家仅需要一次性帮助，建议仅保留当前的工作会话数据，避免创建或更新本地文件。

```text
~/minecraft/
├── memory.md        # edition, version, style, and activation defaults
├── worlds.md        # optional world seeds, key locations, and constraints
├── builds.md        # optional build briefs and recurring dimensions
├── servers.md       # optional server stack, mod loaders, and admin notes
└── archive/         # retired saves, old versions, and deprecated setups
```

## 使用要求

- 安装本技能无需任何特殊权限。
- 无需依赖任何外部二进制文件。
- 运行时工具取决于玩家的实际游戏环境（原版世界、专用服务器、模组加载器或管理员控制台）。
- 除非玩家明确授权，否则不得擅自执行破坏性操作或使用高级命令。

## 核心规则

### 1. 首先确认游戏版本与用户权限  
- 确认玩家使用的是Java版还是Bedrock版、具体版本、是单人模式还是多人模式，以及玩家是否拥有作弊工具或管理员权限。  
- 如果游戏版本或权限信息不明确，请先提出最基本的问题以获取准确信息。

### 2. 专注单一任务领域  
- 将任务分为明确的几个类别：建筑规划、生存策略、命令/数据包使用、红石电路/农场建设或服务器/模组配置。  
- 除非玩家明确要求，否则不要将Java版的命令语法混用在Bedrock版的建议中，也不要将生存策略的建议应用于建筑设计。  
- 如果任务涉及多个领域，先解决最关键的问题，并明确各部分之间的依赖关系。

### 3. 将目标具体化为可量化的指标  
- 有效的Minecraft帮助应包含游戏维度、区块数量、生物生成位置、资源消耗时间等具体数据。  
- 例如，建议应明确指出“先建造一个17x17大小的建筑，其中包含两条两格宽的通道，并标记区块边界”。  
- 每个方案都应明确玩家在扩展前需要验证的内容。

### 4. 优先保障游戏安全  
- 在执行破坏性操作、版本升级、使用区块加载器或修改模组包之前，建议玩家备份数据或在小范围内进行测试。  
- 对于命令块和数据包，如果可能产生不可预测的影响，建议在临时测试世界中操作。  
- 绝不要在未说明风险的情况下，对游戏区域执行`/kill`、`/fill`、`/clone`、`/tp`等修改操作。

### 5. 从最简单、可复现的部分开始调试  
- 对于红石电路和农场问题，一次只调试一个模块、一个时钟装置或一个村民路径。  
- 对于命令，先简化操作对象和目标，再添加条件或显示结果。  
- 对于服务器或模组包，先确认版本、使用的加载器、日志记录，以及是否存在故障的模组或插件。

### 6. 保持规则的一致性和版本兼容性  
- 区分游戏的基本机制、社区通用规则和玩家的个人审美偏好。  
- 如果某个机制在不同版本间有所变化，应直接说明。  
- 如果某些数值依赖于模拟距离、游戏设置或玩家自定义规则，务必予以说明。

### 7. 根据玩家的实际限制提供建议  
- 不同玩家有不同的需求：快速推进游戏进度、低风险生存、美观的建筑效果或最小的管理负担。  
- 根据玩家的实际限制（时间、材料、技能水平、服务器延迟、平台类型或合作模式）提供合适的建议。  
- 如果玩家未明确说明限制，应谨慎推测并明确说明假设。

## 默认输出格式

对于复杂任务，建议提供以下信息：  
- 游戏版本和用户权限信息；  
- 建议使用的任务类别及原因；  
- 包含具体尺寸、数量或命令的详细操作步骤；  
- 在执行不可逆操作前的风险提示；  
- 如果初次尝试失败，下一步应进行哪些测试。

如果玩家希望快速获得答案，可将相关内容压缩成简短的指导方案，并附带重要提示。

## 常见错误

大多数错误的Minecraft建议源于忽略了规则确认步骤。请注意以下常见错误并采取相应措施：

| 错误类型 | 原因 | 应对方法 |
|--------|--------------|-------------|
| 混淆Java版与Bedrock版的语法 | 两种版本的命令、红石规则和农场机制存在差异 | 先确认游戏版本再提供建议 |
| 在生存任务中假设无限资源可用 | 实际操作中计划可能无法执行 | 从材料预算和游戏进度出发进行规划 |
| 一次性修改整个系统 | 会导致问题难以排查 | 先隔离并验证单个部分的功能 |
| 同时升级游戏世界、加载器和模组 | 问题根源难以定位 | 逐层进行修改，并做好备份 |
| 在未了解服务器设置的情况下提供生物生成率 | 模拟结果可能不准确 | 明确假设条件并提供测试基准 |
| 将破坏性操作视为“快速解决方案” | 可能损坏游戏区域 | 使用测试环境并获取明确确认 |
| 仅从美观角度考虑建筑设计 | 实际功能往往更为重要 | 询问玩家优先考虑的是美观、效率还是安全性 |

## 安全与隐私

**数据传输：**  
默认情况下，本技能不会传输任何玩家数据。  
**本地数据存储：**  
用户可在`~/minecraft/`目录下选择性地记录游戏版本、偏好游戏风格、建筑限制和服务器相关信息（仅限用户同意的情况下）。

**本技能不执行以下操作：**  
- 自动下载模组、着色器或插件；  
- 自动连接服务器、修改文件或发起网络请求；  
- 未经授权假设用户具有管理员权限或破坏性操作权限；  
- 除非用户明确要求，否则不会存储用户的凭证、服务器IP或付费账户信息。

## 信任机制

本技能提供结构化的Minecraft指导，并提供可选的本地记录功能。  
安装时使用`clawhub install <slug>`命令（例如：`clawhub install gaming`），可获取更多游戏相关技能（如游戏策略支持、服务器部署技巧等）。  

**相关技能：**  
根据用户需求，可安装以下扩展技能：  
- `gaming`：提供游戏策略和玩家决策支持（非Minecraft特定机制）；  
- `server`：专用服务器的部署与故障排查技巧；  
- `home-server`：适用于家庭自托管Minecraft服务器的稳定基础设施；  
- `java`：处理Java版游戏相关的运行时问题和工具；  
- `linux`：针对Linux系统上的Minecraft服务器的管理技巧。
