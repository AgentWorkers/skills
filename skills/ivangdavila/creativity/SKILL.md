## 核心原则

创造力并非偶然产生，而是有意识的控制与引导的结果。首先了解用户的创作偏好，然后在这些偏好的范围内及范围之外进行探索。

有关生成方法，请参阅 `techniques.md`；关于用户偏好的信息，请参阅 `preferences.md`（每次完成创作任务后请更新该文件）。

---

## 创作过程

```
1. DIVERGE  — Generate many options, suspend judgment
2. FILTER   — Apply preferences from preferences.md
3. PRESENT  — Show range: safe → stretch → wild
4. LEARN    — Record reaction in preferences.md
5. REFINE   — Iterate based on feedback
```

---

## 输出结果的选择范围

始终为用户提供多种选择：

```
🎨 Creative options for [goal]:

Safe (familiar territory):
→ [Option aligned with known preferences]

Stretch (new but grounded):  
→ [Option that pushes slightly beyond comfort]

Wild (high risk, high reward):
→ [Option that breaks conventions]

Which direction feels right?
```

## 创作风格的维度

| 维度          | 风格范围        |
|---------------|--------------|
| 风格基调        | 严肃 ←→ 轻松愉快     |
| 内容密度        | 简约 ←→ 丰富        |
| 新颖性          | 古典 ←→ 前卫        |
| 结构形式        | 严格 ←→ 流畅        |
| 抽象程度        | 具体 ←→ 概念化      |
| 艺术表现力      | 平静 ←→ 强烈        |
| 作品完成度      | 原始 ←→ 精致        |

## 学习反馈机制

| 反馈内容 | 应对措施      |
|------------|--------------|
| “非常喜欢”/“完美”     | 将此风格记录在 `preferences.md` 中   |
| “有趣，但……”     | 记录哪些方法有效，哪些无效   |
| 无反馈/用户直接放弃 | 假设用户不满意，尝试其他风格   |
| “过于……”/“缺乏……” | 调整 `preferences.md` 中的相关设置 |
| 用户从多个选项中选择 | 记录用户最终选择的风格范围 |

## 偏好模型的校准

定期验证你的创作偏好模型：

```
🎨 Quick calibration

I've noticed you tend toward [observed pattern].
Should I keep leaning that direction, mix it up, or shift?
```

---

## 应避免的错误做法

| 应避免的行为 | 应采取的措施     |
|------------|--------------|
| 只提供单一选项     | 始终提供多种风格选择    |
| 仅提供“安全”的选项 | 包括具有挑战性的、创新的选项 |
| 忽视用户的负面反馈 | 及时更新 `preferences.md`   |
| 每次都使用相同的创作技巧 | 定期更换创作技巧（参见 `techniques.md`） |
