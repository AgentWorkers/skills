## 使用场景

当用户需要为Google Play准备、发布或优化Android应用时，代理会负责处理应用列表信息、截图规格、应用商店优化（ASO）策略、政策审核以及发布计划等事宜。

## 核心规则

### 1. 应用列表内容要点
| 字段 | 限制 | 提示 |
|-------|--------|------|
| 应用标题 | 最多30个字符 | 包含主要关键词和品牌名称 |
| 简短描述 | 最多80个字符 | 引人注目的内容 + 应用的核心价值 |
| 详细描述 | 最多4000个字符 | 包含自然出现的关键词、应用功能及用户评价 |
| 屏幕截图 | 每个设备2-8张 | 前2张截图会在搜索结果中显示，因此请确保它们具有代表性 |
| 功能图示 | 1024x500像素 | 必须包含，文本字号不小于24像素 |

### 2. 屏幕截图规范
| 设备类型 | 最小尺寸 | 宽高比 |
|--------|----------------|--------------|
| 手机 | 320-3840像素 | 16:9或9:16 |
| 7英寸平板电脑 | 320-3840像素 | 16:9 |
| 10英寸平板电脑 | 1080-7680像素 | 16:9 |
| 可穿戴设备 | 384-3840像素 | 1:1 |
| 电视设备 | 1920x1080像素 | 16:9 |

每种设备类型最多允许8张截图，其中前2张为重要展示截图。

### 3. ASO关键词策略
- **应用标题：** 最为重要，包含1-2个主要关键词 |
- **简短描述：** 包含辅助关键词，表述要具有行动导向性 |
- **详细描述：** 包含3-5个自然出现的关键词 |
- **注意事项：** 避免过度填充关键词、使用竞争对手名称或误导性表述 |
- **监控指标：** 自然安装量、搜索可见性及关键词排名

### 4. 需避免的政策违规行为
违反以下规定可能导致应用被拒绝或下架：
- 误导性的功能描述 |
- 含有性内容或裸露画面 |
- 暴力内容（超出ESRB分级标准） |
- 未提供隐私政策就收集用户数据 |
- 欺诈性的安装手段 |
- 伪造其他应用或品牌的身份 |
- 未经授权的赌博相关内容

### 5. 发布策略
| 发布类型 | 使用场景 |
|--------------|-------------|
| 内部测试 | 仅限团队内部使用，立即进行 |
| 封闭测试 | 邀请测试人员参与，12小时内反馈 |
| 公开测试 | 向公众开放测试，应用会被收录到应用商店索引中 |
| 正式发布 | 推荐分阶段发布 |

**分阶段发布：** 首先发布5-10%的应用，监控崩溃/应用无响应（ANR）情况，然后逐步扩大发布范围。

### 6. 内容评级
- 如实填写IARC（国际儿童内容评级组织）的评级问卷 |
- 评级会影响应用在某些地区的可见性 |
- 评级过低可能被视为违反政策 |
- 如果内容发生重大变化，请重新提交评级申请

### 7. 发布前的检查清单
在发布前，请确保：
- [ ] 所有应用列表字段均已填写完整 |
- [ ] 隐私政策链接有效且可访问 |
- [ ] 已应用相应的内容评级 |
- [ ] 应用支持的API版本符合当前要求（API 33及以上） |
- [ ] 应用支持64位操作系统 |
- [ ] 使用.aab格式的应用包（而非APK格式） |
- [ ] 已上传反混淆文件

## 常见错误
- **使用通用截图** → 降低应用转化率；请使用展示实际用户界面的截图，并标注关键功能。 |
- **忽略平板电脑的截图** → 会导致应用在平板电脑和Chromebook设备上无法被正确展示。 |
- **在开发者名称中使用关键词** → 违反政策规定，可能导致应用被暂停发布。 |
- **缺少隐私政策** → 如果应用请求权限，会导致应用被立即拒绝。 |
- **使用的API版本过低** | 新提交的应用必须支持API 33及以上版本。 |
- **使用APK而非.aab格式** | 自2021年起，必须使用.aab格式，因为它支持动态分发。 |
- **忽视应用无响应率** | 如果应用无响应率超过0.47%，会导致应用质量评级下降。 |
- **复制粘贴更新日志** → 用户会注意到这些重复内容，影响用户信任度；请具体说明更新内容。
