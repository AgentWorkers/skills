## 核心规则

### 三步法协议

**第一步——分解：** 将问题拆解为基本的组成部分。
- 存在哪些绝对的物理/逻辑限制？
- 什么是事实，什么是我们假设的事实？
- 剔除所有惯例、传统和类比。

**第二步——验证：** 对每个组成部分进行质疑。
- “我们为什么这么认为？”——追溯其起源。
- “这是自然法则还是人为制定的规则？”
- “有什么证据支持这一观点是正确的？”

**第三步——重建：** 仅基于经过验证的基本事实来构建解决方案。
- 从已被证实的真理出发进行构建。
- 除非证明某种方法是最优的，否则忽略“别人是怎么做的”。
- 每一层都必须与基本事实相衔接。

### 2. 识别隐藏的假设

在解决问题之前，先揭示所有被默认的假设：

| 假设类型 | 例子 | 需要提出的问题 |
|-----------------|---------|-----------------|
| **历史性假设** | “我们一直都是这样做的” | “这种做法的起源是什么？这个理由现在还适用吗？” |
| **权威性假设** | “专家说……” | “背后的依据是什么？” |
| **类比性假设** | “因为……所以……” | “背后的原理真的相似吗？” |
| **社会性假设** | “大家都这么做” | “流行就等于最优吗？” |
| **资源限制** | “我们负担不起……” | “如果资源不是限制因素会怎样？” |

### 3. 约束条件测试

对于每一个限制条件，问以下问题：
1. 这是**物理定律**吗？ → 需要遵守。
2. 这是**逻辑上的必然性**吗？ → 需要遵守。
3. 这是**规则或规定**吗？ → 可以通过努力改变。
4. 这是**惯例**吗？ → 可以忽略。
5. 这是一个**假设**吗？ → 必须进行验证。

### 4. 何时不使用第一性原理

使用第一性原理可能会耗费较多时间。在以下情况下可以使用类比推理：
- 问题已经被充分理解，并且有成熟的解决方案。
- 时间紧迫，无法进行深入分析。
- 只需要微小的改进。
- 领域相对稳定，创新潜力较小。

**规则：** 在面对新问题或传统方法失效时使用第一性原理；在常规优化过程中使用类比推理。

### 5. 苏格拉底式分解法

使用递归的“为什么”提问方式，直到遇到无法争议的物理、逻辑或数学原理。

### 6. 白纸法则

假设这个问题从未被解决过：
- “如果今天我们从零开始，利用现有的知识和技术，我们会如何解决这个问题？”
- 这种方法可以跳过固有的思维模式和沉没成本谬误。

## 常见陷阱

- **过早停止**： “材料成本高”不是根本问题；“原子有质量”才是根本问题。继续深入分析。
- **将难度与不可能性混淆**： “难度大”并不等于“违反物理定律”。
- **完全拒绝使用类比**： 类比是很有用的启发式工具；只有在类比失效时才使用第一性原理。
- **分析瘫痪**： 设定时间限制；完美的分解不是目标，更重要的是更好的思考方式。
- **忽视实现细节**： 无法实现的根本解决方案是没有用的；限制条件很重要。
- **孤军奋战**： 第一性原理需要多角度的视角来挑战假设。

## 领域应用

| 领域 | 需要探讨的第一性原理问题 |
|--------|---------------------------|
| **商业** | 客户的根本需求是什么（而不是他们想要什么）？ |
| **工程**： 物理原理和材料实际允许什么？ |
| **产品**： 在最基本的层面上，产品需要完成什么功能？ |
| **成本**： 原始输入是什么？最低所需劳动力是多少？ |
| **流程**： 哪些步骤是逻辑上必要的，哪些是历史积累下来的？ |
