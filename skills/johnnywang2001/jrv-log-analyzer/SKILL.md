# 日志分析器

用于分析任何基于文本的日志文件，以检测错误模式、错误严重程度分布以及异常情况。

## 快速入门

```bash
python3 scripts/analyze_logs.py <logfile>
python3 scripts/analyze_logs.py app.log --top 20 --severity ERROR
python3 scripts/analyze_logs.py server.log --json --since "2026-03-01"
```

## 主要功能

- **错误严重程度分类** — 自动从日志行中识别 FATAL（致命错误）、ERROR（错误）、WARN（警告）、INFO（信息）和 DEBUG（调试）级别
- **错误特征提取** — 通过去除变量部分（如 IP 地址、UUID、PID、时间戳）来对相似错误进行分组
- **异常检测** — 标记错误率超过平均值两倍的小时
- **时间戳解析** — 支持 ISO 8601、syslog 和 nginx/apache 格式的时间戳
- **灵活的输出方式** — 可生成便于阅读的报告，或使用 `--json` 格式输出以便进一步处理

## 命令行选项

| 选项          | 描述                                      |
|--------------|-----------------------------------------|
| `--top N`       | 显示最常见的错误模式（默认值：15）                         |
| `--severity LEVEL` | 过滤最低严重程度的错误（FATAL, ERROR, WARN, INFO, DEBUG）       |
| `--json`       | 以结构化 JSON 格式输出结果                         |
| `--since TIMESTAMP` | 仅分析指定时间戳之后的日志行                         |

## 使用流程

1. 在目标日志文件上运行分析器
2. 查看错误严重程度分布，了解系统整体运行状况
3. 检查最常见的错误模式，找出重复出现的问题
4. 分析异常发生的时间段
5. 使用 `--json` 格式的输出结果，将其导入其他工具或报告中
