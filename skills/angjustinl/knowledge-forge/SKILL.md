# Knowledge Forge

Knowledge Forge 使用一个四步转换引擎，将原始经验转化为可复用的认知资产。

## 核心概念

经验是丰富的，但答案却很稀缺。大多数专家在自己的领域内非常出色，但当他们展示文档或上台演讲时，其他人往往难以理解他们的观点。问题不在于缺乏经验，而在于这些经验没有被有效地“建模”（即没有被转化为可跨场景应用的抽象结构）。

所谓“建模的经验”，是指那些已经被抽象成能够在不同情境中应用的认知结构。我们的技能正是负责完成这一转化过程的。

## 转换引擎

当用户提供原始材料（例如案例研究、个人总结、商业文档、演讲草稿或任何形式的经验描述）时，系统会按以下四个步骤依次执行转化过程：

### 第一步：视角转换——“我的经验” → “你的挑战”

1. **识别用户取得的成就**；
2. **将其重新定义为受众面临的** **普遍性挑战**；
   - 提取用户解决的核心问题；
   - 去除与特定领域相关的细节；
   - 用受众在工作中能识别的语言重新表述这个挑战；
   - 使受众产生共鸣：“是的，我也有这样的困扰”。

**需要回答的关键问题：**“这个经验解决了受众已经存在的哪些问题？”

有关建模模式和示例的详细信息，请参阅 [modeling-patterns.md](references/modeling-patterns.md)。

### 第二步：经验建模——具体故事 → 可复用结构

原始经验往往只是一个个具体的故事。我们需要将其转化为一个**通用模型**，使其能够在不同场景中发挥作用。

1. **发现隐藏在具体案例中的结构模式**；
2. 为这个模式起一个易于记忆、简洁的名称（例如：“100→10→1 营销漏斗”）；
3. **验证**：该模型是否适用于受众关心的至少2-3个其他领域？

**需要回答的关键问题：**“这个经验背后的底层结构是什么？它是否独立于具体领域而具有普遍性？”

有关原型建模及前后对比示例的详细信息，请参阅 [modeling-patterns.md](references/modeling-patterns.md)。

### 第三步：叙事重构

按照以下顺序重新构建故事内容：

1. **挑战呈现**：明确地展示这个普遍性挑战，让受众感受到紧迫感。这部分内容需要重点突出；
2. **模型介绍**：将抽象出的模型作为新的思考框架，强调思维方式的转变（例如角色变化、认知模型的升级），而非具体的工具或操作步骤；
3. **经验验证**：利用原始故事来证明模型的有效性，但不要让故事本身成为核心内容。

**原则**：重点应放在“道”（决策背后的逻辑）上，而非“术”（具体的操作方法）。工具和步骤容易遗忘，真正能传递的是思维方式的转变。

有关设计认知转变技巧的详细信息，请参阅 [challenge-design.md](references/challenge-design.md)。

### 第四步：核心信息的提炼

设计一个简洁、具体且具有启发性的句子，作为最终输出的核心内容。

一个好的核心信息应满足以下要求：
- **具体性**：不是模糊的口号（如“更聪明地工作”），而是能够引发深刻思考的表述（例如“人工智能不是节省时间，而是改变了你参与的游戏规则”）；
- **易记性**：足够简洁，便于受众记住并应用到自己的工作中；
- **启发性**：当应用于受众的实际情境时，能够引发新的思考。

**需要回答的关键问题：**“如果受众忘记了其他所有内容，那么有哪一句话能够直接改变他们的思维方式？”

将这个核心信息放在输出内容的最高潮部分，让它成为整个转化过程的自然结果，而不是一个孤立、脱离上下文的口号。

## 输出结果

完成内部转化后，生成最终的输出结果。

### 输出格式的确定

如果用户指定了输出格式，请按照其要求进行；否则，系统会根据上下文自动选择合适的格式：

| 信号 | 格式 |
|--------|--------|
| “演讲”、“分享”、“课程”、“研讨会” | 演讲稿 |
| “文章”、“博客文章”、“文档” | 文章或文档 |
| “总结”、“知识卡片” | 简洁的知识卡片 |
| 其他模糊或未指定的格式 | 默认为知识卡片 |

有关输出模板和结构设计的详细信息，请参阅 [output-formats.md](references/output-formats.md)。

### 输出内容结构

无论采用何种格式，所有输出结果都必须包含以下要素：

1. **挑战**：用受众能够理解的语言表述的普遍性问题；
2. **模型**：具有记忆点的抽象结构；
3. **证据**：将原始经验转化为证明模型有效性的依据；
4. **核心信息**：能够引发受众思考的那一句话。

### 转换日志

在输出结果之后，附加一个简短的 `## 转换日志`，记录转化过程中做出的关键决策：

```  
## 转换日志  

- **视角转换**：[原始表述] → [面向受众的挑战]  
- **提取的模型**：[模型名称及简要描述]  
- **叙事重点**：[哪些内容被弱化，哪些内容被强调]  
- **核心信息**："[那个具有启发性的句子]"  
```

这个日志有助于用户理解转化过程，并根据需要进一步优化内容。
