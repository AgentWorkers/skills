# Minimalist Notion × OpenAI UI

打造简洁、低干扰性的产品用户界面（UI），具备清晰的层次结构和符合操作者使用习惯的设计。

## 核心原则

1. 优先考虑内容的清晰度，而非装饰性。
2. 布局要简洁明了，同时要足够丰富，以满足复杂工作的需求。
3. 通过排版、间距和对比度来构建层次结构。
4. 避免使用过多的视觉元素（如浓重的阴影、花哨的渐变或随意的色彩点缀）。
5. 每个元素都必须有其存在的合理理由。

## 工作流程

1. 用一句话描述屏幕的功能目的。
2. 确定一个主要用户操作。
3. 首先使用间距和排版来构建界面的基本框架（不依赖颜色）。
4. 仅根据状态或含义添加有意义的色彩。
5. 按照 `references/qa-checklist.md` 中的清单进行检查。

## 强制性的样式规范

- 每个屏幕最多使用一种强调色。
- 边缘半径：8–12px（保持一致的比例）。
- 阴影效果：要柔和，避免使用复杂的“水滴”效果。
- 动画效果：持续时间120–180毫秒，采用缓动效果，且仅用于功能性展示。
- 空闲状态应提示用户下一步的操作。
- 错误状态必须提供修复指引。

## 输出要求（在生成UI规范或代码时）

务必包含以下内容：

- **功能目的**：该屏幕旨在实现什么目标。
- **界面结构**：各个区域和层次关系。
- **使用的设计元素**：字体大小、间距、颜色等。
- **交互状态**：鼠标悬停、焦点、禁用、加载、错误等状态。
- **可访问性**：键盘操作和色彩对比度的注意事项。
- **实现指南**：具体的Tailwind/CSS/React实现建议。

确保输出内容既实用又简洁，便于直接用于开发。
