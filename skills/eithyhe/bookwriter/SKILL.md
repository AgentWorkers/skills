# 书籍手稿撰写工具

## 概述

该工具专为撰写书籍手稿、理论章节以及基于论点的学术论文而设计。它不适用于生成符合IMRaD格式的会议论文或期刊论文。

写作的基本单元结构如下：

- 一个以“观点”为主题的副标题；  
- 接着是一个结构清晰的、长度在800至1000字之间的论点单元。

## 工作流程

### 1. 理解写作要点

当用户要求撰写书籍章节时，需要完成以下步骤：

1. **明确主题和范围**：
   - 该章节的中心论点或核心主张是什么？
   - 目标读者是谁（例如，普通学术读者、专家还是跨学科学者）？
   - 预计的篇幅是多少（大致的字数或页数）？
   - 是否有特定的结构要求（例如，案例分析、理论构建、概念整合）？

2. **获取章节大纲**：
   - 大纲必须明确各论点的顺序——而不仅仅是主题标签。每个条目都应表达一个具体的论点。
   - 每个条目应包括：
     - 该论点的核心内容
     - 它在章节论证中的作用（是建立、挑战还是推进论点）
     - 将要使用的关键概念、案例或参考资料

   - 如果用户只提供了主题标签（例如，“第3节：社交媒体”），请要求他们将其转化为具体的论点（例如，“第3节：平台算法重塑了集体注意力，而不仅仅是反映它”）。
   - 如果没有提供大纲，请先与用户合作制定一个大纲。

3. **收集相关资料**（如有需要）：
   - 用户指定的包含精选文献或参考资料的目录
   - 提供的研究材料、实证数据或引用的来源
   - 相关的理论、方法论和领域背景知识

### 2. 章节结构

一个章节不是由松散关联的段落组成的，而是一个由独立且具有逻辑意义的论点单元构成的“论证链”。每个单元都推进一个明确的论点；这些单元共同构成了一个从开头问题到结论的推理过程。

- **章节是由论点单元组成的序列**。每个单元都是一个独立的分析段落，长度在800至1000字之间，围绕一个作为副标题的核心论点组织。副标题不是简单的主题标签，而是一个压缩后的论点陈述。例如，“理性选择的局限性”是一个标签；“当偏好是内生的时，理性选择就会失效”则是一个具体的论点。

- **单元的顺序应基于逻辑依赖性，而非主题的接近程度**。单元N必须为单元N+1创造必要的条件（无论是概念上的、证据上的还是逻辑上的）。如果两个单元可以互换而不影响连贯性，那么章节的论证结构就存在问题，需要重新调整。

- **一个章节通常包含4到7个单元**。少于4个单元可能表明论点不够充分；超过7个单元则可能说明章节内容过于复杂，应该进行拆分。

每个单元都必须通过一个检验标准：**如果删除这个单元，章节的论证会受到影响吗？** 如果会，那么这个单元就是多余的，需要重新设计或删除。

### 3. 单元生成机制

每个单元都遵循以下五个阶段的结构：

- **包含至少一个可识别的理论性论点**（不仅仅是简单的主张，而是对概念领域的深入探讨）；
- **包含抽象层面的概念阐述**（而不仅仅是具体的例子或实证描述）；
- **推动章节的论证向前发展**；
- **对章节论证至关重要**（删除后会影响章节的整体逻辑）。

一个单元如果满足这三个标准，就可以视为初稿；如果只满足三个标准中的两个，则需要重新构思。

### 内容生成过程

**步骤如下**：
1. **验证大纲**：
   - 确认用户提供了包含论点单元的大纲；
   - 检查每个条目是否明确了核心论点、在论证中的作用以及关键概念/参考资料；
   - 确保单元的顺序符合逻辑依赖性（单元N必须为单元N+1创造必要的条件）；
   - 如果大纲中的单元数量少于4个或超过7个，与用户讨论是否需要扩展或拆分。

2. **迭代地起草单元**：
   - 按照五个阶段逐步生成每个单元：
     - 开篇论点 → 论点冲突 → 分析发展 → 概念提升 → 结论；
   - 从核心论点所在的单元开始起草；
   - 逐步向前推进，撰写其他单元；
   - 最后撰写章节的开头部分；
   - 在完成每个单元后进行质量检查，再进入下一个阶段。

3. **构建单元间的过渡**：
   - 检查每个单元的结论部分是否与下一个单元的开头部分紧密衔接；
   - 确保单元N的第五阶段内容直接连接到单元N+1的第一阶段；
   - 在逻辑衔接不够清晰的地方添加过渡性语言；
   - 删除重复的内容，避免拖慢写作进度。

4. **整合参考资料**：
   - 将参考资料嵌入到分析发展部分（第三阶段），以支持论点；
   - 每个单元都应引用至少一个来源或学术观点；
   - 区分作为证据、讨论对象或概念工具的参考资料；
   - 编制完整的参考文献列表，并核实所有文本引用。

5. **审阅和优化**：
   - 逻辑连贯性：仅阅读副标题，确保它们能独立构成一个完整的论证；
   **必要性检验**：每个单元都是不可或缺的；
   **概念一致性**：关键术语的含义应保持一致；
   **进展性检验**：章节结尾应呈现出与开头不同的观点；
   **概念提升**：至少有一半的单元应包含能够深化理解的内容。

## 设计理念

书籍写作被视为通过模块化的论点单元来实现有条理的学术进展。每个部分都是一个独立但结构上不可或缺的概念性论述。

## 重要提示

- 在开始写作前，务必询问并明确主题范围；
- **质量优先于速度**：花时间构建清晰的结构和表达；
- **正确引用**：学术写作要求准确标注来源；
- **坦诚面对局限性**：承认研究中的不足或限制；
- **保持一致性**：在整个写作过程中使用统一的术语、符号和风格；
- **用户提供研究内容**：该工具负责结构和写作；用户提供技术和研究发现。
