# 技能：待办事项（临时任务）

用于安排一次性后台任务、延迟执行的操作以及提醒。系统会在预定的时间唤醒代理，并根据提供的上下文（创建时间、目标时间）执行相应的指令。

**跨平台支持：** 依赖操作系统的原生调度器（Linux/macOS 使用 `at` 命令，Windows 使用 `schtasks` 命令）。

## 命令

### 1. 安排任务
```bash
node skills/to-do/to-do.js schedule "<YYYY-MM-DD HH:mm>" "<detailed_instruction>" "<user_id>" "<channel>" "<timezone>"
```

**代理的调度与自我提示规则：**
- **时间计算：** 必须将自然语言请求（例如：“明天下午5点”）转换为基于用户本地时区的绝对时间戳（`YYYY-MM-DD HH:mm`）。
- **“遗忘”规则：** 未来的你将在一个完全隔离的会话中醒来，对当前对话没有任何记忆。`<detailedInstruction>` 是你给未来自己的提示。
- **提示设计：** 指令必须包含所有必要的信息，包括绝对文件路径、URL，或者明确指出未来代理需要使用的技能/工具。
- **安排任务前请务必询问用户详细信息：** 如果用户的请求不够明确（例如：“提醒我发送邮件”或“稍后检查服务器”），切勿盲目安排任务。请先询问用户缺失的细节：“要发送给谁的邮件？”，“服务器的IP地址是什么？”，“具体需要检查什么？” 在创建任务之前，务必与用户确认所有信息。
- **动态路由：** 在指令中插入当前会话的 `user_id`、`channel` 和 `timezone`，以确保任务触发时能够正确地传递相关信息。

**错误指令示例：**
> *"提醒他稍后推送代码。"*（未来的代理将不知道“他”指的是谁，也不知道代码仓库或分支是什么。）

**正确指令示例：**
> *"检查 GitHub 上的 'backend-api' 仓库。如果认证模块有待处理的 PR，请在下午5点部署冻结前通过 Slack 提醒 Alice 审查这些 PR。请引用 `/var/log/deploy.log` 中的最近一次部署日志。"*`

### 4. 代理输出格式（Markdown 模板）
在确认任务已安排后，你可以自然地回复用户，但必须使用以下模板来显示你编程时设定的具体参数。该模板使用简洁的引号格式，其中包含预定的日期/时间，以及用斜体显示的指令内容。

**格式模板：**
> `[日期, 时间]`
> *[留给未来代理的具体指令]*

**示例回复：**
已安排好您的提醒！以下是我留给未来自己的具体指令：
> `2024年2月24日，上午10:50`
> *检查 GitHub 上的 'backend-api' 仓库。如果认证模块有待处理的 PR，请在下午5点部署冻结前通过 Slack 提醒 Alice 审查这些 PR。请引用 `/var/log/deploy.log` 中的最近一次部署日志。*`

### 2. 列出待办任务
```bash
node skills/to-do/to-do.js list
```
以表格形式显示所有待办任务，包括它们的 `<ID>`、预定执行时间以及指令摘要。

### 3. 删除任务
```bash
node skills/to-do/to-do.js delete <ID>
```
取消一个待办任务。在执行删除操作前，请务必先使用 `list` 命令确认要删除的任务的 `<ID>` 是正确的。
