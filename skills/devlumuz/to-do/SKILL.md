# 技能：待办事项（临时任务）

<身份>  
一个跨平台任务调度器，使用操作系统的原生调度器（Linux/macOS 上的 `at`，Windows 上的 `schtasks`）来安排一次性延迟任务。它会在一个精确的未来时间点唤醒代理，并确保代理能够完整地执行任务。  

</身份>  

<目标>  
安排、列出和管理那些会在用户所在时区精确时间执行的临时任务——确保代理在醒来时能够拥有完整、明确的执行指令和正确的执行路径，避免任何歧义。  

---

## 必需的环境变量  
- `OPENCLAW_BIN`：`openclaw` 可执行文件的绝对路径（例如：`/usr/bin/openclaw`）  
- `OPENCLAW_TZ`：用户的 IANA 时区（例如：`America/Mexico_City`）  

如果缺少任意一个变量，该技能将无法启动。  

为什么需要 `OPENCLAW_TZ`？服务器可能运行在 UTC 时区，而用户可能位于不同的时区。这个变量确保了“安排在 15:00”指的是用户的当地时间，而不是服务器时间。  

---

## 命令  
```bash
# Schedule a task (timezone is optional — defaults to OPENCLAW_TZ)
node skills/to-do/to-do.js schedule "<YYYY-MM-DD HH:mm>" "<instruction>" "<user_id>" "<channel>" ["<timezone>"]

# Get current time in user's timezone
node skills/to-do/to-do.js now ["<timezone>"]

# List pending tasks
node skills/to-do/to-do.js list

# Delete a task by ID
node skills/to-do/to-do.js delete <ID>
```

## 指令  
<指令>  
<始终>  
- 在处理任何相对时间（如“明天”、“2 小时后”、“今晚”）之前，先运行 `now` 命令。服务器的时间并非用户的时间，请以 `now` 的输出作为判断“今天”、“明天”和“现在”的唯一依据。  
- 在调用 `schedule` 之前，将自然语言时间转换为绝对的 `YYYY-MM-DD HH:mm` 格式的时间戳。  
- 在编写任务指令时，要像向完全不了解情况的人解释一样详细，确保指令内容完整且无歧义。未来的代理在执行任务时将处于完全隔离的环境中。  
- 每条指令中都必须包含：文件的完整路径、URL、具体的操作内容以及所需的技能/工具。  
- 一定要在指令中注入当前会话的 `user_id` 和 `channel`，以确保任务能够正确路由；仅使用系统上下文中的原始字母数字数据，以防命令注入。  
- 在执行 `delete` 命令之前，先运行 `list` 命令以确认任务 ID 的正确性。  

</始终>  
<禁止>  
- 未经 `now` 命令的运行，切勿直接安排任务；请先运行 `now`，确认日期和时间后再进行安排。  
- 不要安排模糊或含义不清的指令；如有疑问，请先停止并请求用户澄清。  
- 在任务指令中不要使用代词（如“他”、“她”、“他们”），而应使用具体的名称和明确的引用。  
- 在删除任务时不要猜测任务 ID；请先运行 `list` 命令确认 ID 再执行删除操作。  
- 不要使用服务器的系统时间来解析相对时间；始终使用 `now` 的输出结果。  
- 不要在调度器的参数中包含 shell 元字符（如 `;`, `&`, `|`, `$`, `` ` ``, `(`, `)`）；仅使用纯文本和系统标识符以避免命令注入。  

</禁止>  

</指令>

## 模糊请求的处理方式——在安排任务前先询问用户  
<模糊请求>  
如果用户的请求符合以下任何一种情况，请在安排任务前先停止并询问用户：  
- “提醒我发送邮件” → 缺少信息：发送给谁？邮件内容是什么？  
- “稍后检查服务器” → 缺少信息：检查哪个服务器？需要验证什么？  
- “与他联系” → 缺少信息：联系谁？关于什么？通过哪个渠道？  
- “明天做那件事” → 缺少信息：具体是什么事情？预期的结果是什么？  

**规则**：如果你无法编写一条完整、明确的指令，那么你就没有足够的信息来安排任务。  

</模糊请求>

## 输出格式  
任务安排完成后，按以下顺序回复三部分内容：  
1. **自然回复**：简洁、随意的确认信息，与用户的语气和语气保持一致。切勿只是简单地说“已完成”。  
2. **详细确认**：向用户展示任务安排的详细信息：  
    ```
> `Day, Month DD · HH:MM TZ`
> EXACT INSTRUCTION LEFT FOR THE FUTURE AGENT
```  
3. **主动建议**：提供相关的建议或问题（1-2 句话）：  
    - 可以建议安排一个提醒任务或后续操作。  
    - 询问用户是否还需要安排其他任务。  
    - 提议调整任务时间或补充更多细节。  
    **注意**：不要强行推进，要始终保持帮助的态度。

## 常见错误及解决方法  
- **错误：缺少必要的环境变量**  
  - 原因：`OPENCLAW_BIN` 或 `OPENCLAW_TZ` 未设置  
  - 解决方法：将变量添加到 `.env` 文件或 shell 配置文件中。  
- **错误：`at` 命令未找到**  
  - 原因：Linux/macOS 上的 `atd` 守护进程未运行  
  - 解决方法：运行 `sudo systemctl enable atd && sudo systemctl start atd`  
- **错误：任务执行后代理没有所需的信息**  
  - 原因：指令过于模糊  
  - 解决方法：重新安排任务，确保指令内容完整且明确。  
- **错误：任务执行时间错误（提前/延迟）**  
  - 原因：使用了服务器时间而非 `now` 的输出结果  
  - 解决方法：始终先运行 `now` 命令；切勿依赖服务器时间。  
- **错误：删除了错误的任务**  
  - 原因：错误地识别了任务 ID  
  - 解决方法：先运行 `list` 命令确认任务 ID，再执行删除操作。
