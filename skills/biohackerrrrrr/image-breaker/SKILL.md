# 图片转换工具（Image Breaker）

该工具可将文档、PDF文件、图片以及网页内容转换为结构化的Markdown笔记，并保存到工作区中，同时实现与Obsidian笔记系统的同步。

## 工作流程

### 1. 提取内容

- **对于URL或PDF文件：**  
  使用相应工具（如`web_fetch`）提取文件内容。

- **对于图片：**  
  使用图像处理工具提取图片中的文本或数据。

- **对于已分析过的内容：**  
  直接使用原始数据。

### 2. 构建内容结构

将提取的内容转换为格式规范的Markdown格式：

- **需要创建的章节包括：**
  - **概述**：文档或内容的主要内容是什么？
  - **关键要点**：主要内容的要点列表。
  - **详细说明**：包含子章节和标题的详细解释。
  - **参考范围/标准**（如适用）：用于展示数值数据的表格。
  - **行动事项**：如何使用这些信息。
  - **来源**：原始文件的URL或文件名称。

- **格式指南：**
  - 使用表格展示数值数据（如参考范围、标准、对比结果）。
  - 使用项目符号列表列出关键要点。
  - 使用标题（`##`、`###`）来组织内容。
  - 对于技术性内容，使用代码块进行展示。
  - 首次出现的重要术语需加粗显示。

### 3. 保存并同步

使用正确的元数据（frontmatter）将Markdown笔记保存到工作区，并自动同步到Obsidian笔记系统中：

### 标签分配

根据内容自动分配3个最相关的标签：

- **常见标签：**
  - `research`：学术论文、研究资料、参考文献
  - `bloodwork`：实验室检测结果、生物标志物
  - `nmr`：核磁共振（NMR）相关的数据分析
  - `cholesterol`：胆固醇及相关脂质信息
  - `peptides`：特定肽类（如BPC-157、TB-500等）
  - `supplements`：维生素、矿物质、化合物
  - `protocols`：治疗/优化方案
  - `founders`：与企业/创业者健康相关的内容
  - `longevity`：抗衰老、寿命相关
  - `performance`：认知/身体性能优化
  - `training`：锻炼、训练计划
  - `toku`：纳豆激酶（nattokinase）、Toku Flow相关内容

优先选择与内容最匹配的标签。

## 输出目录

- **默认目录：`research/image-breaker-notes/`
- **根据内容类型选择的特定目录：**
  - 研究文档 → `research/papers/` 或 `research/protocols/`
  - 实验室检测结果 → `research/bloodwork/`
  - 营销材料 → `content/references/`
  - 锻炼内容 → `research/training/`
  - 商业文档 → `projects/business-docs/`

请根据内容类型选择合适的目录。

## 示例用法

- 用户提供Labcorp的NMR检测报告URL：
  1. 使用`web_fetch`提取报告内容。
  2. 将内容整理成Markdown格式，包括：
    - NMR检测的测量项目概述。
    - 关键参考范围表格。
    - 数据解读指南。
    - 与标准脂质的对比结果。
  3. 分配标签：`bloodwork`、`nmr`、`research`。
  4. 将笔记保存为`research/image-breaker-notes/2026-02-10-nmr-lipid-panel-reference.md`。
  5. 将笔记同步到Obsidian系统的`ImageBreaker/2026-02-10-nmr-lipid-panel-reference.md`目录。
  6. 向用户提供文件路径和Obsidian链接。

## 最佳实践：
- **始终先提取内容**：在构建结构之前，先使用`web_fetch`或其他工具提取数据。
- **编写详细的笔记**：不仅要包含原始数据，还要提供上下文信息。
- **使用表格展示数据**：用于展示参考范围、对比结果和标准。
- **智能分配标签**：最多分配3个最相关、最具体的标签。
- **合理选择输出目录**：确保目录结构与工作区的组织方式一致。
- **自动同步**：用户希望笔记能够在Obsidian系统中方便地被引用。
- **提供文件位置信息**：向用户提供工作区和Obsidian系统中的文件路径。
