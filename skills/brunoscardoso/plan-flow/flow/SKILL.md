# 流程：自动飞行模式（Autopilot Mode）的切换

该功能用于启用或禁用自动飞行模式。启用后，功能请求会自动执行完整的工作流程，而无需手动触发命令。

## 使用方法

```
/flow -enable       Enable autopilot mode
/flow -disable      Disable autopilot mode
/flow -status       Show current state
/flow               Same as -status
/flow -help         Show help
```

## 工作原理

**状态**：存储在 `flow/.autopilot` 文件中（该文件为空标记文件）。创建该文件即可启用自动飞行模式，删除该文件即可禁用。

### 启用自动飞行模式时

用户的所有操作都会被分类如下：

| 类别 | 信号类型 | 处理方式 |
|----------|---------|--------|
| 以 `/` 开头的命令 | 按常规方式执行，不触发自动流程 |
| 问题/探索性请求 | 用户询问代码相关问题或需要解释 | 按常规方式回答，不触发自动流程 |
| 简单任务 | 单个文件、修复方案明显、复杂度为 0-2 | 直接执行，不触发自动流程 |
| 功能请求 | 新功能请求、涉及多个文件、复杂度大于 3 | **触发完整的工作流程** |

### 完整工作流程（共 6 个步骤）：

1. **检查集成契约** → 查找相关的集成契约 |
2. **需求收集** → 收集用户需求并提问 |
3. **制定计划** → 制定实施计划并获取用户批准 |
4. **执行计划** → 根据任务复杂度分组执行各个阶段 |
5. **代码审查** | 审查所有未提交的代码变更 |
6. **归档** | 将相关文件移至 `flow/archive/` 目录，并提示用户进行清理 |

### 强制性检查点

即使在自动飞行模式下，以下步骤也会暂停以等待用户输入：

| 检查点 | 执行时机 | 原因 |
|------------|------|-----|
| 需求收集与解答 | 第 2 步 | 用户必须回答问题 |
| 计划制定前的验证 | 第 2 步到第 3 步 | 在执行计划之前，必须准备好需求文档 |
| 计划批准 | 第 3 步 | 用户必须批准计划才能继续执行 |

### 禁用自动飞行模式时

命令会按常规方式单独执行：
`/discovery` → `/create-plan` → `/execute-plan` → `/review-code`

## 切换模式的命令：

### 启用自动飞行模式：
1. 创建 `flow/.autopilot` 文件（文件内容为空）
2. 向用户确认自动飞行模式已启用

### 禁用自动飞行模式：
1. 删除 `flow/.autopilot` 文件
2. 向用户确认自动飞行模式已禁用

### 当前状态查询：
1. 检查 `flow/.autopilot` 文件是否存在
2. 显示当前状态（启用/禁用）

## 重要规则：

| 规则 | 说明 |
|------|-------------|
| **状态存储方式** | 状态信息存储在 `flow/.autopilot` 文件中（创建文件启用模式，删除文件禁用模式） |
| **无其他副作用** | 该命令仅用于管理标记文件，不会触发任何工作流程步骤 |
| **切换后停止** | 切换模式后，立即停止所有操作并等待用户输入 |
