# 仪表板

这是一个统一的网络终端，用于任务管理、队列处理和系统监控。

## 概述

这是一个单页仪表板，集成了以下功能：
- **任务队列**：查看和管理待处理的任务
- **系统监控**：CPU使用率、内存使用情况、系统负载以及运行时间
- **ZeroTier状态**：网络连接信息
- **输出流**：最近的日志记录

## 快速启动

```bash
# Start dashboard
dashboard start 3853
```

然后访问：http://localhost:3853

## 特点

### 实时监控
- CPU使用率（带有进度条显示）
- 内存使用率（带有进度条显示）
- 系统负载平均值
- 系统运行时间

### 任务队列管理
- 查看待处理/正在处理的任务
- 完成当前任务
- 清空队列
- 每3秒自动刷新一次

### ZeroTier集成
- 连接状态
- ZeroTier IP地址
- 网络信息

### 输出流
- 最近的日志记录
- 可过滤日志来源

## 命令行接口（CLI）命令

| 命令 | 描述 |
|---------|-------------|
| `start [port]` | 启动Web服务器 |
| `status` | 快速查看CLI状态 |

## API接口

| 接口 | 方法 | 描述 |
|----------|--------|-------------|
| `/` | GET | 主要仪表板信息 |
| `/raw` | GET | JSON格式的系统状态信息 |
| `/api/complete` | POST | 完成任务 |
| `/api/clear` | POST | 清空队列 |

## 集成方式

该仪表板整合了以下组件的数据：
- `task-queue` 组件
- `system-monitor` 组件
- `output-streamer` 组件
- `zerotier-deploy` 组件

## 使用场景

1. **操作仪表板**：在一个界面中监控所有系统状态
2. **任务管理**：查看并完成待处理的任务
3. **快速状态检查**：通过CLI命令快速获取系统状态
4. **ZeroTier访问**：快速获取ZeroTier的IP地址
