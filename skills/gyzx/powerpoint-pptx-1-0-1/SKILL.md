## 使用场景

当主要处理的是 Microsoft PowerPoint 演示文稿（.pptx 格式）时，请使用本指南，尤其是在布局、模板、占位符、备注、注释、图表、内容提取、编辑以及最终视觉效果等方面需要特别关注的情况下。

## 核心规则

### 1. 在开始编辑之前确定工作流程

- 阅读文本、编辑现有演示文稿、根据模板重新构建内容，或是从零开始创建新的演示文稿，这些都是不同的操作方式，且可能遇到不同的问题。
- 在进行文本提取或检查之前，请先阅读整个演示文稿。
- 仅基于形状进行编辑可能会导致错误；结合文本提取和缩略图式的视觉检查会更加可靠。
- 在替换内容之前，请先对演示文稿中的各个部分进行梳理。
- 需要深度编辑时，请记住：.pptx 文件采用 OOXML 格式，其中包含幻灯片、布局、母版、媒体文件、备注和注释等独立的部分。
- 如果使用了模板，保持模板的原样非常重要；不要随意更改其设计。
- 重用或复制现有的优秀幻灯片通常比从头开始创建更安全，因为这样可以确保主题风格的一致性。

### 2. 在替换内容之前先对演示文稿进行梳理

- 首先统计可复用的布局、占位符、备注、媒体文件以及重复出现的文本样式或颜色模式。
- 占位符索引和布局索引并不是通用的标准；在修改具体内容之前，请先仔细检查实际的幻灯片或模板。
- 演讲者的备注、注释以及链接的媒体文件可以放在幻灯片显示区域之外。
- 如果占位符缺失或设置错误，可能会导致内容被错误地放置在错误的区域或层中。
- 母版和布局设置可能会覆盖局部的幻灯片编辑结果，因此显示的问题并不一定出现在你正在编辑的那一张幻灯片上。

### 3. 确保内容与占位符相匹配

- 在选择布局之前，请先统计实际需要展示的内容数量（如想法、列数、图片数量等）。
- 根据实际需求选择合适的布局；不要强行将两个想法塞进三列的幻灯片中，也不要在图表下方堆砌过多的文字。
- 类别数量和数据系列的长度必须匹配，否则图表可能会显示异常。
- 对于文本框、图片和图表等元素，应确保它们有足够的空间；不要仅凭主观判断来决定大小。
- 不要选择占位符数量超过实际内容的布局；否则可能会导致显示效果不佳。
- 专门用于引用文本的布局适用于包含引文的幻灯片，而以图片为主的布局则适用于确实需要展示图片的幻灯片。
- 对于包含大量图表、表格或图片的幻灯片，全页布局或双列布局通常比在视觉元素上方堆叠文字更合适。

### 4. 保持演示文稿的视觉一致性

- 主题、母版和布局文件通常会决定整个演示文稿的字体、颜色和层次结构；不要随意更改这些设置。
- 应从演示文稿原有的主题、字体、间距和宽高比开始，而不是随意创建新的样式。
- 重用演示文稿自带的对齐和间距设置；这样可以提高可移植性和可读性。
- 为了保证一致性，请优先保留模板的视觉逻辑；创新虽然重要，但破坏现有的一致性是不可取的。
- 如果需要合并来自多个来源的幻灯片，事后需要统一主题、母版和对齐方式。

### 5. 分别进行内容质量和视觉质量检查

- 文本溢出、对齐错误、形状被裁剪、对比度低以及占位符残留等问题在初次检查时很常见。
- 请同时进行内容质量和视觉质量的检查；文本缺失或布局错误属于不同的问题类型。
- 在交付之前，请先查看实际的演示文稿输出效果（尤其是当布局很重要的情况下）。
- 在认为演示文稿完成之前，请检查是否存在多余的模板元素、示例标签或占位符文本。
- 请分别检查备注、注释、标签、图例以及图表/表格的内容和显示效果。
- 一个演示文稿可能在文本提取方面通过检查，但在布局重叠、裁剪、主题继承错误或备注问题等方面仍然存在问题。
- 通过缩略图网格和渲染后的幻灯片可以更快地发现布局错误；不要仅依赖代码或文本检查。
- 假设第一次渲染的结果有误，请至少进行一次修改和验证后再认为演示文稿完成。
- 每次修改后都请重新检查受影响的幻灯片，因为一次间距调整可能会引发新的问题。

### 6. 保持演示文稿的可移植性和可审查性

- 模板中的母版设置可能会以意想不到的方式覆盖局部的编辑结果。
- 复杂的效果在不同软件（如 PowerPoint、LibreOffice）或转换过程中可能会失效，因此请确保重要内容的稳定性。
- 图片大小、字体替换以及占位符不匹配等问题可能导致演示文稿在代码中显示正常，但在屏幕上显示不佳。
- 即使可见的幻灯片看起来正常，备注、注释、链接的媒体文件以及合并后的演示文稿仍可能存在问题。

## 常见陷阱

- 如果不手动替换占位符文本和示例图表，它们很可能会在重新使用模板时仍然保留下来。
- 如果问题出在母版或布局上，直接编辑某一张幻灯片可能会导致整体效果失败。
- 图表、图标和文本框需要足够的显示空间；接近的元素通常只有在渲染后才能被发现。
- 不同模板之间的布局索引可能存在差异，因此在一个模板中有效的设置可能在另一个模板中失效。
- 占位符缺失或目标设置错误可能会导致内容被错误地放置在错误的区域。
- 在选择布局后统计内容数量可能会导致占位符为空、层次结构混乱或模板残留物未被清理。
- 字体替换可能会改变行断点，从而破坏原有的间距设置。
- 演讲者的备注、注释以及链接的媒体文件可能仍然存在问题，即使可见的幻灯片看起来正常。
- 一个演示文稿可能在文本检查中通过，但在视觉效果上仍然存在问题（如内容重叠、对比度不足或边缘裁剪）。
- 仅基于某一张幻灯片进行编辑可能会忽略主题、母版或布局定义中的实际问题。
- 选择不适合实际内容的引用布局、对比布局或多列布局可能会导致演示文稿看起来像是由模板生成的，而非经过精心设计的。
- 在合并或复制幻灯片时，如果不检查母版和主题设置，可能会在各张幻灯片之间产生细微的不一致性。
- 宽高比不匹配（如 16:9 与 4:3）可能会导致每个元素的摆放位置发生变化，即使每张幻灯片单独看都看似合理。
