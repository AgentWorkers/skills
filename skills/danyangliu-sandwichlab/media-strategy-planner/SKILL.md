# 广告媒体策略

## 目的
核心任务：
- 明确广告目标、选择合适的广告渠道、合理分配预算

该技能专为广告工作流程设计，旨在提供可执行的计划而非泛泛而谈的建议。

## 适用场景
当用户需要以下内容时，请使用此技能：
- 与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长决策
- 针对以下平台的广告策略：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP/程序化广告
- 具体需求：目标设定、广告渠道选择、预算分配

**高频关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价策略、流量、转化率、营销漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP

## 输入要求
**必填项**：
- 目标：增长目标及关键绩效指标（KPI）的优先级
- 预算范围：测试预算和扩展预算
- 广告渠道范围：需要包含的广告渠道

**可选项**：
- 目标受众细分
- 创意素材库
- 季节性因素
- 政策限制

## 输出内容
1. 策略概览
2. 广告渠道的角色定义
3. 预算与出价计划
4. 测试方案
5. 扩展与停止规则

## 工作流程
1. 明确目标层次结构（主要和次要KPI）
2. 根据营销漏斗阶段分配广告渠道的角色
3. 根据预期效果和风险分配预算
4. 设计测试方案和学习周期
5. 制定扩展、暂停和终止规则

## 决策规则
- 如果KPI之间存在冲突，优先考虑预算效率而非广告量
- 如果某个广告渠道的数据支持不足，先分配最低额度的测试预算
- 如果目标受众范围广泛，采用模块化创意和分层定位策略

## 平台注意事项
主要支持的广告平台：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP/程序化广告

**平台使用建议**：
- 建议针对不同平台制定具体的策略；不要将所有渠道合并为一个通用计划
- 对于Meta和TikTok Ads，优先考虑创意内容的测试频率
- 对于Google Ads和Amazon Ads，重点关注需求捕获和用户意图分析
- 对于DSP/程序化广告，重点关注受众控制和投放频率管理

## 限制与注意事项
- 严禁伪造数据或预测结果
- 将观察到的事实与假设区分开来
- 对每个建议的行动使用可衡量的语言进行描述
- 在存在支出风险的情况下，必须包含至少一种回退或止损机制

## 故障处理与升级流程
- 如果关键输入信息缺失，仅要求提供最低限度的必要信息
- 如果平台限制导致冲突，需展示权衡方案及安全默认值
- 如果信心度较低，需明确标注并提供验证清单
- 如果出现高风险问题（如政策变更、计费问题、数据跟踪异常），需通过结构化的流程进行上报

## 代码示例
### 策略矩阵（YAML格式）
```yaml
objective: 提高投资回报率（ROAS）
channels:
  - name: Meta
    role: 激发用户需求
  - name: Google Ads
    role: 捕获用户需求
budget_split:
  Meta: 0.55
  Google Ads: 0.45
```

### 测试方案示例
```yaml
cell_id: T1
variable: 目标受众细分
success_metric: 每次点击成本（CPA）
```

## 实例
### 示例1：调整广告渠道组合
**输入**：
- 预算固定为5万美元
- 连续两周投资回报率（ROAS）下降

**输出重点**：
- 预算重新分配方案
- 测试方案
- 停止损失条件

### 示例2：以创作者为主导的扩展策略
**输入**：
- 目标：在保持投资回报率（ROAS）稳定的前提下增加流量
- 广告渠道：TikTok Ads + YouTube Ads

**输出重点**：
- 广告渠道的角色分配
- 预算投放节奏
- 创意内容的更新频率

### 示例3：针对现有客户群的再营销策略
**输入**：
- 新客户开发效果不稳定
- 拥有稳定的现有客户群

**输出重点**：
- 再营销方案的设计
- 避免覆盖现有客户群的策略
- 分阶段实施计划
