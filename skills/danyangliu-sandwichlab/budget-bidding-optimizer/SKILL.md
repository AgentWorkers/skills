# 广告出价优化器

## 目的
核心任务：
- 确定出价策略、控制出价节奏、优化预算分配

该技能专为广告工作流程设计，应提供可操作的方案而非泛泛而谈的建议。

## 适用场景
当用户提出以下需求时，请使用该技能：
- 需要与业务成果相关的广告执行指导
- 需要制定涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长策略
- 需要针对以下平台采取行动：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、DSP/程序化广告
- 需要具体优化出价策略、出价节奏或预算分配

**高频关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价、流量、转化率、销售漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP

## 输入要求
**必填项**：
- 目标：增长目标及关键绩效指标（KPI）的优先级
- 预算范围：测试预算和扩展预算
- 广告渠道范围：需要包含的广告渠道

**可选项**：
- 目标受众细分
- 创意素材库
- 季节性因素
- 政策限制

## 输出内容
1. 战略概要
2. 各广告渠道的职责分配
3. 预算与出价计划
4. 测试方案
5. 扩展与停止规则

## 工作流程
1. 明确目标层次结构（主要和次要KPI）
2. 根据销售漏斗阶段为各广告渠道分配职责
3. 根据预期效果和风险分配预算
4. 设计测试方案和学习周期
5. 制定扩展、暂停或停止的规则

## 决策规则
- 若KPI之间存在冲突，优先考虑预算效率而非广告量
- 若某广告渠道的数据支持不足，先分配最低额度的测试预算
- 若目标受众广泛，应从模块化创意和分层定位开始

## 平台注意事项
**主要适用平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、DSP/程序化广告

**平台行为建议**：
- 建议针对不同平台制定具体策略；不要将所有渠道合并为统一方案
- 对于Meta和TikTok Ads，优先考虑创意内容的测试频率
- 对于Google Ads和Amazon Ads，优先考虑吸引用户和提升点击率
- 对于DSP/程序化广告，优先考虑受众控制和出价频率的管理

## 限制与注意事项
- 禁止伪造数据或预测结果
- 将观察到的事实与假设区分开来
- 对每项建议使用可量化的表述
- 在存在支出风险的情况下，必须包含至少一种回滚或止损机制

## 故障处理与升级流程
- 若关键输入信息缺失，仅要求提供最低限度的必要信息
- 若平台限制与建议冲突，需展示权衡方案及安全默认值
- 若信心度较低，需明确标注并提供验证清单
- 若出现高风险问题（如政策问题、计费问题或数据跟踪异常），需通过结构化流程上报

## 代码示例
### 战略矩阵（YAML格式）

```yaml
objective: 提高投资回报率（ROAS）
channels:
  - name: Meta
    role: 吸引新用户
  - name: Google Ads
    role: 捕获用户需求
budget_split:
  Meta: 0.55
  Google Ads: 0.45
```

```yaml
cell_id: T1
variable: 目标受众细分
success_metric: 每次点击成本（CPA）
```

## 实例
### 示例1：调整广告渠道组合
**输入**：
- 预算固定为5万美元
- 近两周内投资回报率（ROAS）持续下降

**输出重点**：
- 预算重新分配方案
- 测试方案
- 停止损失条件

### 示例2：以创作者为主导的扩展策略
**输入**：
- 目标：在保持投资回报率（ROAS）稳定的前提下增加流量
- 广告渠道：TikTok Ads + YouTube Ads

**输出重点**：
- 各广告渠道在销售漏斗中的职责分配
- 预算分配策略
- 创意内容更新频率

### 示例3：针对现有客户的再营销策略
**输入**：
- 新客户开发效果不稳定
- 拥有稳定的现有客户群

**输出重点**：
- 再营销方案的设计
- 避免触达重复受众的策略
- 两阶段实施计划
