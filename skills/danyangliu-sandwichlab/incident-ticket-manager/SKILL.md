# 广告事件工单

## 目的
核心任务：
- 事件捕获、严重程度分类、分配负责人

此技能专为广告工作流程设计，应提供可操作的解决方案而非通用建议。

## 适用场景
当用户需要以下内容时，请使用此技能：
- 与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长决策
- 针对 Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads 的平台级操作
- 具体功能：事件捕获、严重程度分类、负责人分配

**高相关性关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价、流量、转化率、销售漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP

## 输入参数
**必填项**：
- 实体ID：账户、广告活动、广告组或广告标识符
- 事件/审计范围
- 时间窗口

**可选项**：
- 日志或事件记录
- 政策标志
- 警报阈值
- 负责人联系方式

## 输出内容
1. 运营状态总结
2. 按严重程度排序的问题发现
3. 缓解措施
4. 升级工单内容
5. 监控检查清单

## 工作流程
1. 确认运营范围和受影响的实体。
2. 验证数据时效性和事件完整性。
3. 发现异常情况、政策违规或设置漏洞。
4. 根据支出风险和恢复速度对问题进行排序。
5. 提供可供负责人执行的操作方案和工单内容。

## 决策规则
- 如果数据过时，禁止做出最终判断，并请求更新数据。
- 如果发现高风险问题，建议先采取控制措施。
- 如果根本原因不明确，提供优先级最高的假设及验证顺序。

## 平台注意事项
**主要覆盖平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads

**平台操作指南**：
- 建议针对不同平台制定具体方案；不要将所有平台的操作合并为一个通用计划。
- 对于 Meta 和 TikTok Ads，优先考虑创意测试的频率。
- 对于 Google Ads 和 Amazon Ads，优先考虑需求捕获和查询/列表意图。
- 对于 DSP/程序化广告，优先考虑受众控制和投放频率管理。

## 约束与限制
- 严禁伪造指标或政策结果。
- 将观察到的事实与假设区分开来。
- 对每个建议的操作使用可量化的表述。
- 在存在支出风险的情况下，必须包含至少一种回滚或止损措施。

## 故障处理与升级流程
- 如果关键输入信息缺失，仅要求提供最低限度的必要字段。
- 如果平台限制存在冲突，需展示权衡方案及安全默认值。
- 如果信心度较低，需明确标注并提供验证清单。
- 如果出现高风险问题（如政策问题、计费问题、跟踪故障），需通过结构化的工单内容进行升级处理。

```

```

condition: impressions_down_pct > 40
action: alert_and_pause_review
```

## 示例
### 示例 1：广告素材在发布前损坏
**输入**：
- 未记录购买事件
- 广告活动将在 24 小时后开始

**输出重点**：
- 问题严重程度排序
- 立即采取缓解措施
- 重新发布的检查清单

### 示例 2：账户健康状况审计
**输入**：
- 多个平台账户
- 政策或计费状态未知

**输出重点**：
- 准备情况评分表
- 需要避免的风险
- 负责人应采取的行动

### 示例 3：实时异常处理
**输入**：
- 支出激增但转化率未提升
- 多个广告活动受到影响

**输出重点**：
- 异常问题处理流程
- 控制措施
- 事件工单内容
