# 多平台广告执行器

## 目的
核心任务：
- 执行顺序的规划、设置检查清单的制定以及广告投放的控制

该技能专为广告工作流程设计，旨在提供可操作的策略而非泛泛而谈的建议。

## 适用场景
当用户提出以下需求时，请使用此技能：
- 需要与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每点击成本（CPA）或预算效率的增长决策
- 需要针对以下平台的操作：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP/程序化广告
- 特定功能需求：执行顺序的规划、设置检查清单的制定以及广告投放的控制

**高频关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每点击成本（CPA）、投资回报率（ROI）、预算、出价、流量、转化率、销售漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP

## 输入参数
**必填项**：
- 目标：增长目标及关键绩效指标（KPI）的优先级
- 预算范围：测试预算与扩展预算
- 广告渠道范围：需要包含的广告渠道

**可选项**：
- 目标受众细分
- 创意素材库
- 季节性因素
- 政策限制

## 输出结果
1. 战略概览
2. 各广告渠道的职责分配
3. 预算与出价计划
4. 测试方案
5. 扩展与停止规则

## 工作流程
1. 明确目标层次结构（主要KPI与次要KPI）
2. 根据销售漏斗的不同阶段分配广告渠道的职责
3. 根据预期效果和风险分配预算
4. 设计测试单元及学习周期
5. 制定扩展、暂停或停止广告投放的规则

## 决策规则
- 若KPI之间存在冲突，优先考虑收入效率而非广告投放量
- 若某广告渠道的数据支持不足，先分配最低额度的测试预算
- 若目标受众范围广泛，应从模块化创意素材和分层定位开始

## 广告平台注意事项
**主要覆盖平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP/程序化广告

**平台操作建议**：
- 建议针对每个平台制定具体的策略；不要将所有渠道合并为一个通用计划
- 对于Meta和TikTok Ads，优先考虑创意素材的测试频率
- 对于Google Ads和Amazon Ads，重点关注需求捕获及用户查询/列表意图
- 对于DSP/程序化广告，重点关注受众控制与投放频率的管理

## 限制与注意事项
- 禁止伪造任何指标或政策结果
- 将观察到的事实与假设区分开来
- 对每项建议的操作使用可量化的表述
- 在存在支出风险的情况下，必须包含至少一项回退或止损策略

## 故障处理与升级流程
- 若关键输入信息缺失，仅要求提供最低限度的必要信息
- 若不同平台的限制相互冲突，需展示权衡方案及安全的默认处理方式
- 若信心度较低，需明确标注并提供验证检查清单
- 若出现高风险问题（如政策变更、计费问题、数据跟踪异常），需通过结构化的报告进行上报

## 代码示例
### 战略矩阵（YAML格式）
```yaml
objective: 提高投资回报率（ROAS）
channels:
  - name: Meta
    role: 激发用户需求
  - name: Google Ads
    role: 捕获用户需求
budget_split:
  Meta: 0.55
  Google Ads: 0.45
```

### 测试单元示例
```yaml
cell_id: T1
variable: 目标受众细分
success_metric: 每点击成本（CPA）
```

## 实例
### 示例1：广告渠道组合调整
**输入**：
- 预算固定为5万美元
- 近两周内投资回报率（ROAS）持续下降

**输出重点**：
- 预算重新分配方案
- 测试方案
- 停损条件

### 示例2：以创作者为主导的扩展策略
**输入**：
- 目标：在保持投资回报率（ROAS）稳定的前提下增加流量
- 广告渠道：TikTok Ads + YouTube Ads

**输出重点**：
- 各广告渠道在销售漏斗中的职责分配
- 预算分配逻辑
- 创意素材的发布频率

### 示例3：以重新定位为主的恢复策略
**输入**：
- 新客户开发效果不稳定
- 拥有稳定的现有客户群

**输出重点**：
- 重新定位策略的设计
- 避免触及现有客户群体的规则
- 分阶段启动计划
