# 广告资产管理系统

## 目的
核心任务：
- 资产分类管理、权限控制、生命周期管理

此技能专为广告工作流程设计，应提供可执行的计划而非通用建议。

## 适用场景
当用户提出以下需求时，请使用此技能：
- 与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长决策
- 针对以下平台的操作：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads
- 具体功能需求：资产分类管理、权限控制、生命周期管理

**高频关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价、流量、转化率、销售漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP

## 输入参数
**必填项**：
- 产品或服务（product_or_offer）
- 目标受众（target_audience）
- 广告投放范围（placement_scope）

**可选项**：
- 现有广告素材（existing_assets）
- 品牌风格（brand_tone）
- 禁止使用的表述（prohibited_claims）
- 创意限制（creative_constraints）

## 输出内容
1. 创意目标（Creative Objective）
2. 创意方向与吸引点设定（Angle and Hook Set）
3. 广告素材规格（Asset Specification）
4. 测试方案（Test Variant Plan）
5. 创意质量检查表（Creative QA Notes）

## 工作流程
1. 将创意目标与销售漏斗阶段及关键绩效指标（KPI）对齐。
2. 生成多种创意方向及吸引点变体。
3. 将每种创意方向与相应的广告投放格式要求相匹配。
4. 定义测试方案矩阵及测试顺序。
5. 设置质量与合规性检查点。

## 决策规则
- 如果目标受众较为陌生，优先考虑能够引发问题的创意方向及验证性内容。
- 在再营销阶段，优先确保信息清晰且具有紧迫感。
- 如果广告格式有严格限制，简化信息结构，只保留一个明确的行动号召（CTA）。

## 平台注意事项
**主要适用平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads

**平台使用建议**：
- 建议针对不同平台制定具体方案；不要将所有平台的策略合并为一个通用计划。
- 对于Meta和TikTok Ads，优先考虑创意测试的频率。
- 对于Google Ads和Amazon Ads，重点关注需求捕捉及用户查询/列表意图。
- 对于DSP/程序化广告平台，优先考虑受众控制与投放频率管理。

## 约束与注意事项
- 绝不允许伪造数据或预测结果。
- 区分观察到的事实与假设。
- 对每个建议的操作使用可量化的表述。
- 在存在支出风险的情况下，必须包含至少一种回退或止损策略。

## 故障处理与升级流程
- 如果关键输入信息缺失，仅要求提供最低限度的必要字段。
- 如果平台限制存在冲突，需展示权衡方案及安全默认选项。
- 如果信心度较低，应明确标注并提供验证清单。
- 若出现高风险问题（如政策变更、计费问题、跟踪数据异常），需通过结构化的方式上报并寻求协助。
