# 深度广告研究

## 目的
核心任务：
- 研究工作流程的协调、数据来源的规划以及研究结果的整合。

该技能专注于广告相关工作流程，其输出应为可执行的计划，而非泛泛而谈的建议。

## 适用场景
当用户提出以下需求时，请使用此技能：
- 与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长决策
- 针对以下平台的操作建议：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、DSP/程序化广告

相关关键词：
- 广告、广告活动、增长策略、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价策略、流量、转化率、销售漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、DSP

## 输入要求
必填项：
- 研究问题
- 假设集
- 决策截止日期

可选项：
- 数据来源偏好
- 置信度目标
- 需排除的假设
- 研究深度（即研究的详细程度）

## 输出内容
1. 研究计划
2. 证据表格
3. 假设评估结果
4. 战略性结论
5. 可执行的后续实验方案

## 工作流程
1. 将研究问题分解为可验证的假设。
2. 制定数据来源和证据收集计划。
3. 评估证据的有效性和冲突点。
4. 整合这些证据对广告策略的影响。
5. 输出决策结果及后续实验建议。

## 决策规则
- 如果证据质量较低，应说明局限性并避免做出过于绝对的结论。
- 如果假设之间存在冲突，根据证据的有效性和时效性进行排序。
- 如果决策截止日期临近，应提供基于最佳努力的建议，并附带风险说明。

## 平台注意事项
主要研究范围：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、DSP/程序化广告

平台操作建议：
- 建议根据具体平台制定针对性的策略；不要将所有平台合并为统一的通用计划。
- 对于Meta和TikTok Ads，优先考虑创意内容的测试频率。
- 对于Google Ads和Amazon Ads，重点关注需求捕捉和用户搜索/浏览意图。
- 对于DSP/程序化广告，重点关注受众控制和投放频率的管理。

## 限制与注意事项
- 绝不允许伪造数据指标或政策结果。
- 将观察到的事实与假设区分开来。
- 对每个提出的行动使用可量化的语言进行描述。
- 在存在支出风险的情况下，必须包含至少一种回退或止损策略。

## 故障处理与升级流程
- 如果关键输入信息缺失，仅要求提供最低限度的必要信息。
- 如果平台限制相互冲突，需展示权衡方案及安全的默认选项。
- 如果置信度较低，应明确标注并提供验证清单。
- 如果出现高风险问题（如政策变更、计费问题或数据跟踪异常），需通过结构化的报告进行上报。

## 代码示例
### 研究计划 YAML 示例
```yaml
hypothesis: 创作者主导的视频内容能在第一周内提升投资回报率（ROAS）
sources: [平台数据、竞争对手案例、内部测试结果]
confidence_target: 中等至高
```

### 证据记录示例
```yaml
source: campaign_2026_q1
finding: 每次点击成本（CPA）下降了18%
confidence: 中等
```

## 示例说明
### 示例 1：深度竞争对手分析
输入：
- 需要分析竞争对手过去三个月的创意内容和推广策略
- 研究平台：Meta + TikTok Ads

输出内容：
- 证据表格
- 规律总结
- 战略性建议

### 示例 2：假设验证测试
输入：
- 团队认为广泛的目标定位总能带来更好的效果
- 相关证据较为复杂（正面和负面都有）

输出内容：
- 假设的详细分解
- 根据证据强度排序的结论
- 建议的后续实验方案

### 示例 3：董事会层面的战略汇报
输入：
- 需要为下一季度的广告渠道方向提供建议
- 预算有所增加

输出内容：
- 多种方案选项
- 带有风险权重的建议
- 适合决策的总结报告
