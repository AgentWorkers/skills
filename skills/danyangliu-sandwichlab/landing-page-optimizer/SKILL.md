# 广告着陆页优化器

## 目的
核心任务：
- 提升转化率的设计、点击率（CTA）测试、页面迭代计划

该技能专为广告工作流程设计，应提供可操作的方案，而非泛泛而谈的建议。

## 适用场景
当用户提出以下需求时，请使用该技能：
- 与业务成果相关的广告执行指导
- 涉及收入、投资回报率（ROAS）、每次点击成本（CPA）或预算效率的增长决策
- 针对以下平台的操作：Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads
- 具体功能需求：提升转化率的设计、点击率测试、页面迭代计划

**高频关键词**：
- 广告、广告活动、增长、收入、利润
- 投资回报率（ROAS）、每次点击成本（CPA）、投资回报率（ROI）、预算、出价、流量、转化率、营销漏斗
- Meta、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads

## 输入要求
**必填项**：
- 目标：增长目标和关键绩效指标（KPI）的优先级
- 预算范围：测试预算和扩展预算
- 平台范围：需要包含的广告渠道

**可选项**：
- 目标受众细分
- 创意素材库
- 季节性因素
- 政策限制

## 输出内容
1. 战略概要
2. 广告渠道角色定义
3. 预算和出价计划
4. 测试矩阵
5. 扩展和停止规则

## 工作流程
1. 定义目标层次结构（主要和次要KPI）
2. 根据营销漏斗阶段分配广告渠道的角色
3. 根据预期效果和风险分配预算
4. 设计测试方案和学习周期
5. 设置扩展、暂停和停止的规则

## 决策规则
- 如果KPI之间存在冲突，优先考虑收入效率而非流量
- 如果某个渠道的数据支持不足，先分配最低的测试预算
- 如果目标受众广泛，从模块化创意和分层定位开始

## 平台注意事项
**主要覆盖平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads

**平台操作建议**：
- 建议针对不同平台制定具体策略；不要将所有渠道合并为一个通用计划
- 对于Meta和TikTok Ads，优先考虑创意内容的测试频率
- 对于Google Ads和Amazon Ads，优先考虑捕获用户需求和展示意图
- 对于DSP/程序化广告平台，优先考虑受众控制和投放频率的管理

## 限制与注意事项
- 严禁伪造数据或预测结果
- 将观察到的事实与假设区分开来
- 对每个建议的操作使用可量化的表述
- 在存在支出风险的情况下，至少包含一个回退或止损条件

## 故障处理与升级流程
- 如果缺少关键输入信息，仅要求提供最低限度的必要字段
- 如果平台限制相互冲突，展示权衡方案和安全的默认选项
- 如果信心度较低，明确标注并提供验证清单
- 如果出现高风险问题（如政策问题、计费问题、跟踪数据异常），通过结构化的报告进行上报
