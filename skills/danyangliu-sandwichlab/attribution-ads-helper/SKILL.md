# 归因辅助工具

## 目的
核心任务：
- 诊断不同渠道之间的归因差异。
- 比较归因窗口的假设及其对预算的影响。
- 建立实用的归因决策框架以优化资源分配。
- 提供与归因结果一致的、可操作的预算分配建议。

## 适用场景
当用户需要以下内容时，请使用此工具：
- 归因模型比较
- 不同渠道之间的ROAS（投资回报率）/CAC（每次点击成本）冲突
- 在归因不确定性下进行预算决策
- 跟踪数据分析和模型解释支持

**相关关键词**：
- 归因、跟踪、模型、预测
- ROAS、CPA（每次点击成本）、收入、分配、预算
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、DSP（数字广告平台）

## 输入参数
**必填参数**：
- `channel_metrics_by_window`（按窗口划分的渠道指标）
- `attribution_windows`（归因窗口设置）
- `conversion_event_definitions`（转化事件定义）
- `decision_context`（决策背景信息）

**可选参数**：
- `offline_conversion_data`（离线转化数据）
- `holdout_or_incrementality_data`（保留数据或增量数据）
- `MMM_or_ltv_inputs`（货币化模型或生命周期价值输入数据）
- `confidence_threshold`（置信度阈值）

## 输出结果
1. **归因不匹配地图**：显示各渠道归因结果之间的差异。
2. **窗口敏感性分析**：分析不同归因窗口对结果的影响。
3. **决策安全的KPI视图**：提供可靠的KPI数据。
4. **预算重新分配建议**：基于分析结果提出预算调整方案。
5. **验证实验计划**：为未解决的问题制定实验验证方案。

## 工作流程
1. 规范化事件和转化事件的定义。
2. 比较每个归因窗口下的表现。
3. 量化模型差异对决策结果的影响。
4. 提出带有置信度标签的分配建议。
5. 为未解决的问题制定验证实验计划。

## 决策规则
- 如果各渠道的归因结果存在显著差异，采用混合归因策略。
- 如果某个渠道对用户点击次数非常敏感，减少仅依赖“最后点击”数据的决策方式。
- 如果有增量数据支持，优先考虑这些数据而非代理指标。
- 如果不确定性仍然较高，将预算分配到多个测试组中以降低风险。

## 平台说明
**主要适用平台**：
- Meta（Facebook/Instagram）、Google Ads、TikTok Ads、YouTube Ads、Amazon Ads、Shopify Ads、DSP（数字广告平台）

**平台使用建议**：
- 对每个平台明确区分归因窗口的设置。
- 区分平台自报的数据和统一计算的归因结果。

## 限制与注意事项
- 不要在同一结论中混合使用不一致的转化事件定义。
- 对于需要长时间考虑才能转化的产品，需标注时间延迟效应。
- 当模型差异较大时，避免做出二元化的决策。

## 故障处理与升级流程
- 如果事件分类不一致，首先输出规范化检查清单。
- 如果离线转化数据不可用，标记潜在问题并采取保守的决策策略。
- 如果预算决策涉及高风险，必须通过实验来验证结果。
