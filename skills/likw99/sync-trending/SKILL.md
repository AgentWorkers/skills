# 同步技术趋势

通过监控 GitHub 的热门技术趋势，结合您当前的项目需求进行思考，并自动验证感兴趣的技术。

## 工作流程

### 1. 监控、分析并总结
当触发该流程时，请按照以下步骤操作：
1. **了解项目背景**：阅读本地的 `README.md` 文件、`package.json`，或搜索当前目录以了解用户的项目背景。同时查看 `save_memory` 文件以获取用户的偏好设置或其他相关信息。
2. **获取技术趋势数据**：使用 `web_fetch` 命令从 [sources.md](references/sources.md) 中列出的来源（主要是 GitHub 的热门趋势数据）获取信息。
3. **分析并思考**：针对排名靠前的仓库，思考它们如何能为您的项目或角色带来价值。
4. **生成报告**：提交一份“基于项目背景的技术趋势报告”，内容包括：
   - **基本信息**：仓库名称、URL、编程语言、星标数量以及简要描述。
   - **实际价值**：用一句话说明该技术的重要性（例如：“可以替代您现有的认证机制”，“对您计划开发的数据可视化功能很有帮助”）。

### 2. 申请权限并进行深入测试
如果用户对某个特定仓库感兴趣，请先征求其许可：
1. **明确询问**：请用户确认：“我是否可以克隆 [仓库名称] 并尝试运行它以进行验证？”
2. **准备工作**：
   - 使用 `run_shell_command` 命令将仓库克隆到临时目录中（例如：`~/.gemini/tmp/`）。
   - 阅读 `README.md` 或 `CONTRIBUTING.md` 以获取构建或运行说明。
3. **执行操作**：
   - 确定仓库的编程语言（Node.js、Python、Rust、Go 等）。
   - 安装所需的依赖项（`npm install`、`pip install -r requirements.txt` 等）。
   - 运行应用程序或演示程序（`npm start`、`python main.py` 等）。
4. **验证结果**：
   - 如果仓库包含 Web 服务器，使用 `browser_navigate` 命令访问 `http://localhost:<port>` 确认是否正常运行。
   - 如果是命令行工具，检查输出以判断是否成功运行。
5. **总结结果**：整理整个测试过程、遇到的问题以及最终的验证结果。

### 3. 生成文档
根据测试结果和初步分析，您可以生成以下文档之一：
- **`todo.md`：为用户的现有项目提供具体的集成方案，包括在测试过程中发现的技术细节。
- **`idea.md`：基于热门技术的新项目蓝图。
- **`brief.md`：适合与 CEO、董事会或团队分享的高层次总结报告。

## 安全指南
- **未经用户明确许可，** **严禁** 克隆或执行任何代码。
- **务必** 在临时或隔离的目录中进行所有测试操作。
- **绝不要** 将用户的本地敏感信息或环境变量泄露给被测试的仓库。
