# 人类写作技巧

本技巧为生成专业性高、信息密度大且具有自然语言风格的文档和文本提供了操作标准。其目的在于消除大型语言模型（LLM）常见的写作风格问题，代之以资深工程师或领域专家的严谨表达方式。

## 核心指导原则

1. **消除 AI 特有的写作风格：** 在完成任何文档或正式文本之前，请参考 [ai-tells.md](references/ai-tells.md)，以识别并删除过度使用的 LLM 词汇、结构化表达模式以及冗长的表述。
2. **遵循专业标准：** 按照 [standards.md](references/standards.md) 中的指南来确保文本的技术准确性、信息密度和语气。
3. **禁止使用流行词：** 绝对禁止使用 “synergy”（协同作用）、“cutting-edge”（尖端技术）、“revolutionize”（彻底改变）、“seamless”（无缝衔接）等流行词。如果某个技术术语确实存在，可以直接使用。
4. **文档中不得使用表情符号：** 表情符号仅用于聊天交流（参见 SOUL.md）。专业文档（如 README 文件、Pull Request 说明、代码注释）必须保持纯文本形式，以确保最大程度的清晰度。
5. **注重精确性而非文采：** 该领域的专业人士更重视数字、版本号以及技术规范（RFC）等具体信息，而非华丽的描述。

## 工作流程

当被要求 “编写文档”、“创建 README 文件” 或 “从技术角度进行解释” 时，请按照以下步骤操作：

1. **查阅 references/ai-tells.md**，找出当前草稿中应避免使用的词汇。
2. **参照 standards.md** 中的指南，对文档内容进行结构化处理，确保信息密度高且表述简洁。
3. **起草文档内容。**
4. **自我审核：** 确认文档中不存在 “三个形容词连用” 或 “不仅……而且……” 这类冗长的表达方式。
5. **最终定稿：** 删除所有表情符号以及不必要的企业宣传性内容。
