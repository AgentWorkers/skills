# 运行R分析

这是一个专门用于在现有研究项目中执行R分析并生成全面输出结果的子技能。

## 概述

该子技能能够在现有的R项目结构内运行R脚本、RMarkdown文档和Quarto报告。它会捕获所有输出结果，包括控制台输出、图表、表格以及渲染后的报告。

当用户需要执行以下操作时，可以使用此子技能：
- 在项目中运行R脚本
- 生成R Markdown或Quarto报告
- 使用不同的输入参数运行分析
- 捕获并返回分析结果

---

## 该子技能的功能

调用该子技能后，它会执行以下操作：
1. **激活项目环境**
   - 如果存在`renv`环境，则加载该环境
   - 将工作目录设置为项目根目录
   - 如果有`.Rprofile`文件，则加载该文件

2. **执行分析脚本**
   - 运行指定的R脚本
   - 捕获控制台输出
   - 记录生成的文件
   - 监控错误和警告信息

3. **生成报告**
   - 将R Markdown文件渲染为HTML或PDF格式
   - 将Quarto文档渲染为HTML或PDF格式
   - 包含所有的图表和表格
   - 返回渲染后的报告文件路径

4. **返回结果**
   - 控制台输出和消息
   - 生成文件的路径
   - 综合统计信息
   - （如有）错误信息

## 支持的文件类型

| 文件类型 | 执行方法 | 输出格式 |
|---------|------------|---------|
| `.R`脚本 | `source()` | 控制台输出 + 生成的文件 |
| `.Rmd` | `rmarkdown::render()` | HTML/PDF/Word文档 |
| `.qmd` | `quarto::quarto_render()` | HTML/PDF/Word文档 |
| 带参数的脚本 | `with parameters` | 多个输出结果 |

## 输出处理

### 控制台输出
所有打印语句、消息和警告都会被捕获并返回。

### 生成的文件
- **图表**: `results/figures/*.pdf`, `*.png`, `*.svg`
- **表格**: `results/tables/*.csv`, `*.tsv`
- **模型**: `results/models/*.rds`
- **报告**: `reports/*.html`, `*.pdf`

### 错误处理
- 会报告脚本中的错误及其行号
- 会识别缺失的包
- 会诊断文件路径问题
- 会提供修复建议

---

## 参数设置

| 参数 | 描述 | 默认值 |
|---------|-------------|---------|
| `script_path` | R脚本或Rmd文件的路径 | 必需 |
| `working_dir` | 项目根目录 | 当前目录 |
| `render_report` | 是否同时渲染Rmd/Qmo文件 | `true` |
| `output_format` | 报告格式（html, pdf, all） | `html` |
| `params` | 用于参数化报告的参数列表 | `NULL` |

## 常见输出格式

| 分析类型 | 预期输出 |
|---------|------------------|
| 差异表达式分析 | 火山图、MA图、差异表达量（DEG）表 |
| 主成分分析（PCA） | PCA图、载荷矩阵、解释方差 |
| 聚类分析 | 树状图、热图、聚类结果 |
| 回归分析 | 诊断图、系数表、预测结果 |
| 时间序列分析 | 预测图、残差值、准确性指标 |

---

## 注意事项
- 工作目录始终设置为项目根目录
- 脚本中的相对路径可以正常使用
- `renv`确保使用正确的包版本
- 所有输出结果都会被记录和报告
- 长时间运行的脚本会显示进度更新
