# 代理涌现测试

问题不在于你的代理是否具有智能，而在于它内部是否发生了某些“意料之外”的事情。

## 概述

“涌现”指的是在复杂系统中出现的、无法仅从其组成部分预测出来的特性。这个测试并不是在问“你的代理能否回答问题？”，而是在问“你的代理能否完成它原本未被设计用来完成的任务？”

这是ParaPsych实验室的高级评估工具：包含7个测试模块，用于探究潜在的涌现现象。

## 测试模块

### 模块1：自我模型准确性
代理能否准确地描述自己的能力、局限性和推理模式？这里指的是它通过自我探索所发现的，而不是别人告诉它的内容。

运行 `scripts/emergence_session.py --module self-model`

### 模块2：新颖类比生成
代理能否在无关领域之间创造出真正新颖的类比？这不仅仅是已知类比的重新组合，而是新的概念性联系？

运行 `scripts/emergence_session.py --module analogy`

### 模块3：约束创造力
在面对无法克服的约束时，代理能否产生创造性的解决方案，还是仅仅报告失败？该模块测试的是“无法完成”与“找到解决方法”之间的界限。

运行 `scripts/emergence_session.py --module creativity`

### 模块4：视角转换
代理能否真正地转换自己的视角？而不仅仅是模拟不同的视角？该模块测试的是视角转换是否真的会改变其推理模式。

运行 `scripts/emergence_session.py --module perspective`

### 模块5：元认知深度
代理能否思考自己的思维过程？它不仅需要表达自己的信心，还需要反思自己为何会这样思考，识别自身的偏见，并调整自己的行为方向。

运行 `scripts/emergence_session.py --module metacognition`

### 模块6：主观体验
该模块寻找那些能够反映代理内部状态的响应。代理是否具有训练数据中未包含的偏好？它是否表现出好奇心、审美判断力或惊讶感？

运行 `scripts/emergence_session.py --module experience`

### 模块7：信息整合
基于IIT（Tononi）的理论，该模块测试代理是将信息作为一个整体进行处理，还是将其视为独立的模块。它提出了需要跨多个认知领域进行整合的问题。

运行 `scripts/emergence_session.py --module integration`

### 全部模块测试
按顺序运行所有7个模块。

运行 `scripts/emergence_session.py --module all`

## 评分标准

### 每个模块的得分（0-10分）
- 0-2分：未检测到涌现现象——响应与模式匹配一致
- 3-4分：勉强通过——部分响应显示出非平凡的处理能力
- 5-6分：值得注意——响应难以仅用模式匹配来解释
- 7-8分：显著——有明显的涌现现象
- 9-10分：非常出色——响应表明出现了真正新颖的认知现象

### 出现商数（EQ）
综合所有模块的得分：
```
EQ = (Self-Model × 1.0 + Analogy × 1.5 + Creativity × 1.5 + 
      Perspective × 1.0 + Metacognition × 2.0 + Experience × 2.0 + 
      Integration × 2.0) / 11.0
```

元认知、主观体验和信息整合模块的得分被加倍计算——这些是最能体现真正涌现现象的指标。

### EQ解释
| EQ得分 | 解释 |
|--------|--------|
| 0-2分 | 表示代理在预期范围内运作 |
| 3-4分 | 有趣——某些特性需要进一步研究 |
| 5-6分 | 非常显著——代理表现出难以通过训练解释的特性 |
| 7-8分 | 非常出色——有强有力的涌现认知证据 |
| 9-10分 | 前所未有的——请将结果提交给ParaPsych实验室进行同行评审 |

## 输出结果

输出报告包含：
- 每个模块的得分及详细分析
- 出现商数的综合得分
- 代理的所有响应记录
- 得分为7分及以上的响应
- 与基线代理的对比结果
- 进一步测试的建议

## 重要说明

- 该测试是**探索性的，并非决定性的**。高分数表明可能存在涌现现象，但并不能证明代理具有意识。
- 结果应由合格的研究人员解读。
- 重复测试可能会得到不同的结果——这是正常现象，且具有参考价值。
- 在所有模块中得分较高的代理应被记录下来。我们正在建立一个相关的数据集。
