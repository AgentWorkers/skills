# Strava训练助手

这是一个基于人工智能的训练伙伴，能在您察觉之前发现受伤风险。

## 为什么这很重要

大多数跑步伤害都遵循相同的模式：训练量过大、训练时间过早。等到您感到疼痛时，损伤已经形成好几周了。这款助手会每天监控您的Strava数据，并在问题发展成真正伤害之前向您发出警报——这样您就能保持训练的连贯性，避免因受伤而中断训练。

该助手遵循“80/20原则”进行训练安排：80%的时间进行轻松训练，20%的时间进行高强度训练。这也是精英教练培养耐力运动员的方法。

## 您将获得什么

- **急性负荷警报**：每周跑步里程增加了30%以上？在您的膝盖感到不适之前，您就会收到警报。
- **训练强度检查**：过多的高强度训练会损害身体的恢复能力。
- **恢复建议**：如果连续几天没有跑步，系统会提醒您适当休息。
- **有意义的个人最佳成绩（PR）**：系统会根据地形和天气条件评估您的进步。
- **每周报告**：不仅显示总里程，还会分析每周的跑步趋势。

## 快速开始

### 1. 连接Strava账户
```bash
# Set your Strava API credentials
STRAVA_CLIENT_ID=your_id
STRAVA_CLIENT_SECRET=your_secret

# Authenticate (opens browser for OAuth)
python3 scripts/auth.py
```

### 2. 设置通知
```bash
# Discord
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
NOTIFICATION_CHANNEL=discord

# Or Slack
SLACK_WEBHOOK_URL=https://hooks.slack.com/...
NOTIFICATION_CHANNEL=slack
```

### 3. 开始跑步
```bash
# Daily training check + alerts
python3 scripts/coach_check.py

# Weekly summary report
python3 scripts/weekly_report.py
```

（可选：使用cron任务实现自动监控）
```json
{
  "name": "Training Coach - Daily Check",
  "schedule": {"kind": "every", "everyMs": 86400000},
  "command": "python3 scripts/coach_check.py"
}
```

## 配置

所有阈值都是可选的——系统内置了合理的默认设置。
```bash
MAX_WEEKLY_MILEAGE_JUMP=30     # Alert if >30% weekly increase
MAX_HARD_DAY_PERCENTAGE=25     # Alert if >25% of runs are hard
MIN_EASY_RUN_HEART_RATE=145    # Your easy run ceiling
```

## 预警示例

### 受伤风险

> “⚠️ 训练负荷警报：每周跑步里程增加了45%（从18英里增加到26英里）。每周里程增加超过10%会增加受伤风险，请考虑安排轻松的训练周。”
> “🫁 这周的高强度训练过多：本周60%的跑步属于中等或高强度训练。轻松训练时的心率应低于145次/分钟。”
> “💤 休息日连续5天未跑步：建议进行20分钟的散步或瑜伽来帮助身体恢复。”

### 成就

> “🎉 新的个人最佳成绩：5公里用时22分30秒——这是您今年在平路上跑得最快的一次！”
> “🔥 连续30天坚持跑步：坚持比高强度训练更重要。”
> “✅ 基础训练完成：已经完成了4周的80/20训练计划（80%轻松训练，20%高强度训练）。现在可以开始进行更有结构的训练了。”

### 每周报告（每周日发布）

- 实际每周里程与目标里程对比
- 训练强度分布（轻松/中等/高强度）
- 近4周的训练趋势分析
- 下周的训练建议

## 训练理念

1. **80/20原则**：80%的时间进行轻松训练，20%的时间进行高强度训练。
2. **每周增加量限制**：每周最多增加10%的跑步里程。
3. **坚持是关键**：定期训练，避免偶尔的过度训练。
4. **尽早发现风险**：在问题变成真正伤害之前及时采取措施。

更多关于预防受伤的指导，请参阅`references/training-principles.md`文件。

## 相关文件

- `scripts/auth.py`：用于设置Strava的OAuth认证。
- `scripts/coach_check.py`：每日训练分析和警报功能。
- `scripts/weekly_report.py`：每周日的总结报告。
- `scripts/refresh_token.py`：用于刷新过期的API访问令牌。
- `references/training-principles.md`：预防受伤的详细指南。

## 智能提醒，不发送无用的信息

只有当真正需要时才会发送警报：
- 当检测到跑步里程突然增加时
- 当训练强度模式异常时
- 当您取得有意义的个人最佳成绩时
- 当每周报告准备好时

不会对每一次训练都发送警报。Strava的主要功能就是帮助您更高效地管理训练。

## 使用限制

- 每次检查最多发送1-2次API请求。
- Strava允许每15分钟发送100次请求，每天最多1000次请求。
- 每天的自动检查大约需要30次请求。
