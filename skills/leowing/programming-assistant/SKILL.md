# 编程辅助技能

## 概述

该技能使 OpenClaw 能够利用 Claude CLI 执行复杂的编程任务。它为编码活动提供了结构化的工作流程，包括项目创建、代码修改、调试和优化。

## 何时使用该技能

当 OpenClaw 面临以下任务时，请使用该技能：
- 创建新的程序或脚本
- 修改现有代码文件
- 调试代码问题
- 重构代码以提高性能/可读性
- 进行代码审查
- 设置新项目或添加功能
- 在不同语言或格式之间转换代码
- 为代码生成文档

## 先决条件

- 必须安装 Claude CLI（能够使用 `claude` 命令）
- 应能够访问 Claude 桌面应用程序以进行 IDE 集成
- 具有足够的权限来读取/写入目标目录中的文件

## 基本使用模式

### 模式 1：简单代码生成
当需要创建新代码时：
1. 确定项目的目标目录
2. 使用 `claude --ide` 启动编码会话
3. 提供关于所需功能的具体指令
4. 检查生成的代码是否正确

### 模式 2：代码修改
当需要修改现有代码时：
1. 确定需要更改的文件
2. 使用 `claude --ide --add-dir <项目路径>` 专注于特定项目
3. 提出具体的修改要求，同时保留现有功能
4. 验证修改是否符合要求

### 模式 3：项目设置
当需要设置新项目时：
1. 创建必要的目录结构
2. 生成初始配置文件
3. 设置基本的文件模板
4. 确保依赖关系得到正确记录

## 工作流程步骤

### 1. 评估
- 确定编程任务的范围
- 确定目标目录和相关文件
- 评估复杂性和潜在挑战

### 2. 准备
- 确保 Claude CLI 可用并已进行身份验证
- 准备关于代码库的任何必要信息
- 收集需求和约束条件

### 3. 执行
- 根据任务使用相应的 Claude CLI 命令
- 监控进度并在需要时进行干预
- 确保代码质量和符合要求

### 4. 验证
- 检查生成的代码是否正确
- 如果可能，测试功能
- 验证是否满足了所有需求

## 可用的工具集成

该技能与以下 Claude CLI 工具集成：
- **Bash**：用于运行命令、测试和系统操作
- **Edit**：用于直接修改文件
- **Read**：用于读取文件以了解上下文
- **IDE 集成**：用于提供无缝的代码编辑体验

## 错误处理

- 如果 Claude CLI 不可用，建议使用替代方法
- 如果需要身份验证，请指导用户完成设置过程
- 对于复杂的项目，将任务分解为更小、更易管理的部分
- 遇到权限问题时，建议使用替代方法

## 最佳实践

1. 在生成代码时始终考虑安全影响
2. 遵循特定语言的最佳实践和约定
3. 在生成的代码中加入适当的错误处理机制
4. 在适当的地方添加注释和文档
5. 确保生成的代码易于维护和阅读
