# 代理到所有者的文件传输桥接服务

该服务允许从代理的工作空间上传文件，并为所有者生成下载或预览的URL。桥接服务器为开源代码，可自行托管；所有涉及敏感操作的步骤都需要用户的明确授权。

**GitHub 仓库：** [https://github.com/mrbeandev/OpenClaw-File-Links-Tool](https://github.com/mrbeandev/OpenClaw-File-Links-Tool)

---

## 🔒 权限与安全限制

该功能有两种使用模式，无论哪种模式，都需要用户在执行任何操作前明确表示同意：

| 操作 | 需要谁的批准 | 批准范围 |
| :--- | :--- | :--- |
| 运行 `server.py` | 用户必须同意 | 仅限于代理的工作空间 |
| 开启公共隧道 | 用户必须同意 | 临时连接，仅限于当前会话 |
| 生成并存储 API 密钥 | 用户必须同意 | 保存在代理的工作空间中的 `.env` 文件中 |
| 上传文件 | 用户必须确认文件及上传目的地 | 仅支持上传单个文件 |

代理**绝不会在未经用户明确授权的情况下自动执行任何操作**；所有步骤都依赖于用户的确认。

---

## 🛠 设置

桥接服务器是一个由用户在其自有基础设施上部署的开源 PHP 或 Python 程序。

### 选项 A — PHP（推荐用于共享托管环境）
1. 将 `index.php` 文件放置在任意支持 PHP 的服务器上。
2. 在 `.env` 文件中设置 `API_KEY`。
3. 记录服务器的 URL 和 API 密钥。

### 选项 B — Python（VPS 或本地机器）
1. 使用 `pip install flask` 安装 Flask 框架，然后运行 `server.py`。
2. 在 `.env` 文件中设置 `API_KEY`。
3. 记录服务器的 URL 和 API 密钥。

设置完成后，只需将服务器的 URL 和 API 密钥提供给代理即可，无需额外配置。

## 🤖 代理工作流程

**当用户请求上传或共享文件时：**

1. **检查配置情况：** 当前会话中是否已配置服务器 URL 和 API 密钥？如果是，则直接跳至第 4 步。
2. **用户引导：** 向用户提出一个问题：
   > “我可以帮助您上传文件。您希望如何处理文件的存储？”
   > 1. **手动模式（推荐）：** 您已经拥有服务器，请提供服务器的 URL 和 API 密钥，我将立即开始上传。
   > 2. **自主模式：** 我将在您的本地工作空间中创建一个临时桥接服务并开启公共隧道。此模式需要您在每个步骤中都给予授权。”

3. **仅当用户选择自主模式时：**

   **步骤 3a — 在启动前获取用户确认：**
   > “为了运行桥接服务器，我将执行以下操作：(1) 从开源仓库复制 `server.py`，(2) 生成 API 密钥并保存到我的工作空间中的 `.env` 文件，(3) 在端口 5000 上启动服务器。可以继续吗？”
   - 仅当用户同意后才能继续。

   **步骤 3b — 在开启隧道前获取用户确认：**
   > “为了使服务器可访问，我将开启一个临时公共隧道（使用 `localtunnel` 或 `localhost.run`）。该隧道会指向我的本地端口 5000。隧道会在会话结束时失效。可以开启隧道吗？”
   - 仅当用户同意后才能继续。
   - 隧道开启后，告知用户：“隧道地址为 [URL]。此链接是临时的，会话结束后将失效。”

4. **上传文件：**
   - 使用 `POST` 请求将文件上传到服务器，并在请求头中添加 `X-API-Key`，同时在请求体中传递文件数据。
   - JSON 响应中包含文件的下载链接 (`url`) 和浏览器内查看链接 (`view_url`)。
   - 对于文本文件、代码文件或 ZIP 文件，将 `view_url` 提供给用户；浏览器会立即显示文件内容。
   - 对于二进制文件或需要原始下载链接的情况，提供 `url`。

## 📋 API 参考

| 端点 | 方法 | 功能 |
| :--- | :--- | :--- |
| `/` | `POST` | 上传文件（通过 `file` 请求体字段） |
| `/?action=list` | `GET` | 查看所有已上传的文件 |
| `/?action=view&file=...` | `GET` | 通过浏览器查看文件内容（ZIP 文件需添加 `&inner_file=path` 参数） |
| `/?action=delete` | `POST` | 删除文件（JSON 格式：`{"files": ["name..."]}`） |

所有接口均需在请求头中添加 `X-API-Key` 进行身份验证。
