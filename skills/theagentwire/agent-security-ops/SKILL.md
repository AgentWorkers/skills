# agent-security-ops

这是一套专为独立创业者设计的代码仓库安全加固工具包，通过一个命令即可设置预提交钩子（pre-commit hooks）、执行秘密信息扫描（secret scanning）以及实现持续监控（continuous monitoring）功能。

## ⚠️ 重要提示：`--no-verify` 参数的注意事项

> 使用 `git commit --no-verify` 可以绕过预提交钩子，这会跳过所有钩子的执行，包括秘密信息扫描。
>
> **建议：**
> 1. **除非你已确认没有秘密信息被提交到仓库中，否则切勿使用 `--no-verify` 参数**。
> 2. **建议在持续集成（CI）流程中添加额外的扫描机制**（例如使用 TruffleHog），以确保即使绕过了预提交钩子，也能检测到秘密信息。
> 3. **在使用 `--no-verify` 后，务必运行 `scan.sh` 命令**，以确保没有秘密信息被遗漏。

### 快速入门

执行以下命令后，该工具包将：
1. 安装 TruffleHog（安装时会使用 SHA256 校验和来确保版本的正确性，可通过环境变量 `TRUFFLEHOG_VERSION` 自定义版本）。
2. 设置一个预提交钩子，用于在提交前检查是否存在秘密信息。
3. 更新 `.gitignore` 文件，以屏蔽常见的秘密信息存储路径（如 `.security-ops/`、`.terraform/` 等目录）。
4. 执行首次秘密信息扫描（包括 Git 历史记录和文件系统中的未跟踪文件）。

### 命令说明

所有脚本都支持 `--help` 和 `--version` 参数：

#### `setup.sh` — 一次性代码仓库安全加固
- **特性：** 该命令可重复执行，不会覆盖现有的预提交钩子，只会将其添加到现有钩子列表中。

#### `scan.sh` — 全面安全扫描
- **扫描内容：**
  - **秘密信息：** 使用 TruffleHog 检查所有存在的秘密信息（已验证的秘密会特别标记）。
  - **文件系统：** 扫描文件系统中的未跟踪文件。
  - **常见秘密信息模式：** 包括 AWS、GitHub、Anthropic、Slack、OpenAI、Stripe、Google、Twilio、SendGrid、npm、Vault 等服务的秘密信息。
  - **低置信度的秘密信息模式：** 检查数据库 URL、密码/密钥、Bearer 令牌、Firebase、Supabase、JWT 等。
  - **`.gitignore` 文件检查：** 使用 `git check-ignore` 验证配置文件中的秘密信息屏蔽规则是否有效。
  - **依赖项审计：** 使用 `npm audit` 或 `pip audit` 检查依赖项中的安全问题（输出结果为 JSON 格式）。
  - **文件权限检查：** 查找可被全局读取的配置文件（如 `.env`、`.pem`、`.key` 等）。
  - **开放端口检查：** 列出正在监听的端口，并标记异常端口（在 macOS 上可能需要使用 `sudo` 权限）。
  - **环境变量检查：** 检查 shell 配置文件中的硬编码秘密信息。
  - **`.env` 文件检查：** 在 `$HOME`、桌面目录和下载目录中查找 `.env` 文件。
  - **Docker 秘密信息：** 检查 Dockerfile 和 `compose` 文件中的硬编码秘密信息。
  - **SSH 配置检查：** 检查 `~/.ssh` 文件的权限设置。

#### `monitor.sh` — 适用于 Cron 任务的监控脚本
- **工作原理：** 通过比较扫描结果与前一次扫描的结果来检测变化（基于哈希值），发生变化时输出 1，未变化时输出 0。使用原子文件写入和 flock 机制来防止脚本同时运行。

### 发现问题怎么办？

| 发现的问题 | 应采取的措施 |
|---------|-----------|
| **Git 中存在已验证的秘密信息** | 立即更换相应的凭据，并使用 `git filter-repo` 或 BFG 命令从 Git 历史记录中删除该凭据。 |
| **Git 中存在未验证的秘密信息** | 进一步核实，可能是误报或凭据已过期，仍需更换。 |
| **匹配到常见秘密信息模式** | 将该秘密信息移至 `.env` 文件或使用专门的秘密管理工具进行管理，并确保其被 `.gitignore` 文件屏蔽。 |
| **匹配到低置信度的秘密信息模式** | 手动核实其真实性。 |
| `.gitignore` 文件中缺少必要的屏蔽规则** | 重新运行 `setup.sh` 命令以添加缺失的屏蔽规则。 |
| **文件具有全局可读权限** | 将文件的权限设置为仅所有者可读。 |
| **检测到异常开放的端口** | 查找对应的进程（使用 `lsof -i :<port>`），必要时停止该进程。 |
| **Shell 配置文件中存在秘密信息** | 将秘密信息移至 `.env` 文件或使用 `1Password` 等工具进行管理，并删除相应的 `export` 语句。 |
| **Docker 中存在硬编码的秘密信息** | 使用 Docker 的秘密管理机制（如 `${VAR}` 语法）或 `.env` 文件来存储秘密信息。 |
| **SSH 配置问题** | 运行 `setup.sh --fix-ssh` 命令修复问题，或手动修改 `.ssh` 文件的权限。 |
| **发现不安全的 Git 远程仓库** | 使用 `git remote set-url` 命令更改远程仓库的访问权限。 |
| **检测到公共仓库** | 如果是意外情况，可以使用 `gh repo edit --visibility private` 命令将仓库设置为私有。 |

### 功能限制

- **注意：** 此工具仅基于字符串匹配进行安全检查，无法检测到经过混淆或动态生成的秘密信息。
- **不支持静态/动态应用程序安全测试（SAST/DAST）**。
- **对 Terraform、Kubernetes 或 CloudFormation 的支持有限**：仅能通过基本模式在 `.tf`/`.tfvars` 文件中进行扫描。
- **TruffleHog 的验证依赖于服务可用性**：如果相关服务不可用，某些秘密信息可能显示为“未验证”状态。
- **端口扫描仅检测当前正在监听的端口**，无法检测防火墙规则或网络暴露情况（在 macOS 上可能需要使用 `sudo` 权限）。
- 对 `$HOME` 目录的 `.env` 文件也会进行扫描，但仅作为警告，不计入仓库的安全问题统计。

### 扫描范围

| 扫描类别 | 使用的工具 | 覆盖范围 |
|----------|------|----------|
| 代码中的秘密信息 | TruffleHog | 当前文件及完整的 Git 历史记录（已验证的秘密会特别标记） |
| 文件系统中的秘密信息 | TruffleHog | 未跟踪的文件 |
| 常见秘密信息模式 | `grep` | 多种服务（如 AWS、GitHub、Anthropic、Slack 等） |
| 低置信度的秘密信息模式 | `grep` | 数据库 URL、密码、Bearer 令牌、Firebase、Supabase、JWT 等 |
| `.gitignore` 文件 | `git check-ignore` | `.env*`、`.pem`、`.key`、`.pfx`、凭证文件等 |
| 依赖项安全问题 | `npm/pip audit` | 包中的已知安全漏洞 |
| 文件权限问题 | `find` 命令 | 可被全局读取的敏感文件 |
| 开放端口 | `lsof/ss` | 检查正在监听的端口 |
| 环境变量中的秘密信息 | `grep` | Shell 配置文件和 `.env` 文件中的硬编码秘密信息 |
| Docker 秘密信息 | `grep` | Dockerfile 和 `compose` 文件中的硬编码秘密信息 |
| SSH 配置问题 | `stat` 命令 | `.ssh` 文件的权限设置 |

### 安全模型

- **二进制文件验证：** 使用 TruffleHog 并通过 SHA256 校验和来验证文件的完整性。
- **失败时阻止提交：** 如果 TruffleHog 无法正常运行，提交操作将被阻止。
- **禁止通过包管理器（brew）安装**：仅允许直接下载安装，以防止供应链攻击。
- **版本固定：** 使用 `TRUFFLEHOG_VERSION` 以确保使用的是固定版本的 TruffleHog。
- **自我排除机制：** 脚本会自动从扫描范围内排除自身文件。

## 常见问题解答

**这个工具是什么？**
`Agent Security Ops` 是一套用于保护代码仓库安全的工具包，它通过设置预提交钩子和执行全面的安全扫描来防止 AI 工具泄露秘密信息。它可以检测并阻止 API 密钥、令牌、密码等敏感数据被提交到 Git 历史记录中。

**它解决了什么问题？**
AI 工具在运行过程中会生成和处理各种凭据（如 API 密钥、令牌、数据库 URL 等）。如果没有相应的安全措施，这些信息可能会被意外提交到代码仓库或日志中。该工具可以在提交时以及需要时自动执行安全扫描。

**使用要求是什么？**
需要具备 Bash 命令行环境、Git 工具以及 TruffleHog（会自动安装）。该工具支持 macOS 和 Linux 系统，无需额外的 API 密钥或外部服务。

**10 项安全扫描包括哪些内容？**
- Git 中的提交文件
- 环境配置文件（`.env` 文件）
- 配置文件
- 最近的 Git 历史记录
- 常见的秘密信息模式
- AWS/GCP/Azure 等服务的凭据
- 私有密钥
- 数据库 URL
- API 令牌

**该工具适用于哪些 AI 工具框架？**
适用于所有使用文件写入功能的 AI 工具，无论使用的是 OpenClaw、Claude Code、Cursor、Aider 还是其他工具。

**可以定时执行扫描吗？**
可以。你可以将此工具与 Cron 任务结合使用，定期对代码仓库进行安全扫描。扫描结果会以结构化格式输出，便于自动化监控。
