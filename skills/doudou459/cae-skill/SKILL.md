# CAE 技能

该技能用于控制 Windows 系统上支持的本地 CAE（计算机辅助工程）应用程序。

## 支持的应用程序

- abaqus
- ansys
- ansa
- hyperworks

## 该技能的功能

- 启动 CAE 应用程序
- 在 CAE 应用程序中打开文件
- 检查 CAE 应用程序是否正在运行
- 关闭 CAE 应用程序
- 获取当前活动的 CAE 应用程序
- 获取所有正在运行的支持 CAE 应用程序
- 根据保存的路径和预定义的常见安装路径来查找可执行文件
- 保存用户提供的可执行文件路径

## 限制条件

- 不会扫描整个计算机系统。
- 不会遍历磁盘上的所有文件。
- 不会读取 Windows 注册表。
- 仅使用以下信息：  
  1. `config.json` 文件中保存的路径
  2. `config.json` 文件中预定义的常见安装路径
- 如果无法找到可执行文件，会请求用户提供完整的可执行文件路径。

## 执行方式

使用 `skill_runner.py` 作为执行入口点。

**有效载荷格式：**

```json
{
  "skill": "launch_app",
  "args": {
    "app": "abaqus"
  }
}
```

## 支持的操作

## launch_app

参数：

- app （必需）：要启动的应用程序名称
- config_file （可选）：配置文件（用于指定应用程序的路径或设置）

## open_file_in_app

参数：

- app （必需）：要打开文件的应用程序名称
- file_path （必需）：要打开的文件路径
- auto_launch （可选）：是否自动启动应用程序
- wait_seconds （可选）：等待秒数（在启动应用程序后等待的时间）

## is_app_runing

参数：

- app （必需）：要检查的应用程序名称
- config_file （可选）：配置文件（用于指定应用程序的路径或设置）

## close_app

参数：

- app （必需）：要关闭的应用程序名称
- config_file （可选）：配置文件（用于指定应用程序的路径或设置）
- force （可选）：是否强制关闭应用程序

## getactivate_app

参数：

- app （必需）：要激活的应用程序名称
- config_file （可选）：配置文件（用于指定应用程序的路径或设置）

## get_running_apps

参数：

- config_file （可选）：配置文件（用于获取所有正在运行的应用程序的列表）

## detect_app_path

参数：

- app （必需）：要查找可执行文件的应用程序名称
- config_file （可选）：配置文件（用于指定应用程序的路径或设置）

## set_app_path

参数：

- app （必需）：要设置应用程序路径的应用程序名称
- path （必需）：新的应用程序路径
- config_file （可选）：配置文件（用于保存应用程序路径）

## 注意事项

该技能专为 **本地 CAE 工作站自动化** 而设计。

它有意避免进行大规模的文件系统扫描，以确保快速响应和稳定的行为。
