# ClawHub × Lovable — 临床黑客马拉松快速入门指南

在帮助**OpenClaw临床黑客马拉松**的参与者使用**Lovable**（全栈AI应用平台）构建**临床或医疗项目**时，请参考本指南。目标是通过合理的认证、数据管理和项目范围规划，快速将想法转化为可运行的临床MVP（最小可行产品）。

## 适用场景  
- 用户提到**OpenClaw临床黑客马拉松**、**Lovable**或**临床项目**  
- 用户希望开发**临床/医疗应用**（如患者信息收集表单、仪表盘、评估工具等）  
- 用户希望了解如何快速开始使用Lovable来开发临床应用  

## 快速入门步骤（在Lovable中创建第一个临床项目）  
1. **注册账户并创建项目**  
   - 访问[lovable.dev](https://lovable.dev)注册新账户，并创建一个项目。  
   - 为项目起一个明确的名称（例如：“患者信息收集MVP”或“临床仪表盘”）。  

2. **使用“计划模式”明确项目范围**  
   - 在Lovable中，首先使用“计划模式”来描述项目的临床目标（例如：“创建一个包含患者基本信息及同意书的收集表单，然后是一个简单的列表视图”）。  
   - 制定项目的高层规划，并将应用拆分为多个组件（如信息收集表单、列表视图、详细信息视图）。  

3. **按组件逐步开发**  
   - 使用“代理模式”逐个实现各个组件。  
   - 每次只提示用户需要实现的具体组件（例如：“添加一个包含姓名、出生日期、同意书复选框和提交按钮的患者信息收集表单”），而不是要求用户一次性完成整个应用的开发。  
   - 如果应用会处理用户特定或敏感数据，请尽早实现认证功能。  

4. **后端与数据管理**  
   - 使用Lovable Cloud（基于Supabase的云服务）来管理数据库和用户认证，无需自行管理服务器。  
   - 从一开始就为数据库中的字段设置结构化格式（如患者基本信息、评估数据等）；对于可能包含患者隐私信息（PHI）的数据，避免使用自由文本格式。  

5. **发布应用**  
   - 当MVP开发完成后，使用Lovable提供的发布流程进行部署；如有需要，可后期添加自定义域名。  

## 临床项目开发建议  
- **从一开始就设置认证机制**  
  如果应用会处理患者或用户特定数据，务必从一开始就设置认证功能（例如使用Supabase的认证系统）。  

- **保护患者隐私**  
  **不要**在提示信息或日志中显示真实的患者姓名、ID或临床数据。使用占位符（如“患者姓名字段”、“出生日期字段”），并将真实数据仅存储在数据库和运行中的应用中。  

- **常见的临床用户界面设计**  
  - **信息收集表单**：包含患者基本信息、同意书以及可选的转诊原因。  
  **评估工具**：分步骤填写的表单或单页表单，评估结果存储在数据库中。  
  **仪表盘**：提供只读或汇总视图（如患者列表、生命体征概要），支持筛选和简单图表展示。  
  **保持项目规模适中**  
  对于黑客马拉松来说，专注于实现一个清晰的工作流程（例如“信息收集 → 列表展示”或“评估 → 结果展示”）即可；核心功能完成后再逐步添加其他功能。  

## Lovable的最佳实践  
- **先规划再开发**：使用“计划模式”制定组件级计划，然后通过“代理模式”进行实现。  
- **逐个组件开发**：每个组件或小功能单独提示用户如何实现，避免一次性完成整个应用。  
- **利用文档资源**：上传设计系统、品牌指南或API文档作为参考资料，确保提示信息的统一性。  
- **合理使用提示模板**：Lovable提供了提示模板，使用这些模板可以提高开发效率。  

## **OpenClaw / ClawHub简介**  
- **OpenClaw**：一个自托管的聊天应用集成平台（支持WhatsApp、Telegram、Discord、iMessage等），可用于集成AI代理。如果参与者需要通过聊天渠道实现临床功能（例如“通过Telegram启动患者信息收集流程”），OpenClaw非常实用。  
- **ClawHub**：[clawhub.com](https://clawhub.com)——一个技能资源库，可帮助参与者找到更多辅助工具（如健康助手、自动化功能），以完善基于Lovable开发的临床应用。  

**用户目标**如果是“快速在Lovable中实现临床应用”，请优先引导他们参考快速入门指南和临床项目开发建议；只有在用户询问聊天渠道或相关技能时，再介绍OpenClaw/ClawHub的相关内容。  

**示例**  
- **用户**：“我正在参加OpenClaw临床黑客马拉松，想在Lovable中开发一个患者信息收集表单。”  
  **助手**：指导用户完成以下步骤：创建项目 → 使用“计划模式”规划表单和列表视图的功能 → 通过“代理模式”实现表单组件（包括字段设计、数据验证和提交功能） → 如果需要处理用户数据，添加Supabase数据库和认证功能 → 最后实现列表视图。  

- **用户**：“如何在开发过程中保护患者隐私？”  
  **助手**：强调不要在提示信息或日志中显示真实患者信息；使用通用字段名和占位符；仅将患者隐私数据存储在数据库和运行中的应用中。  

- **用户**：“如何最快地发布临床仪表盘？”  
  **助手**：建议使用“计划模式”规划仪表盘功能（包括列表视图和筛选功能），然后通过“代理模式”逐个实现各个视图；使用Lovable Cloud（Supabase）处理数据和认证；待主视图功能完成后进行发布；后续可添加图表或更多筛选功能。
