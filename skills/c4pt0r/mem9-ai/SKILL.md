### mem9

`mem9` 为 OpenClaw 提供持久化存储功能，即使在系统重置、重启或更换新机器后也能保持数据的一致性。

该插件会连接到由一个空间 ID 标识的云存储空间。

**与最终用户交流时的注意事项：**
- 使用“空间 ID”、“存储空间”或“云存储空间”来指代该存储空间。
- 除非你在编辑插件的原始配置文件，否则不要提及 `tenantID`。
- 向用户解释：空间 ID 是用于重新连接到相同 `mem9` 存储空间的关键信息。
- 告诉用户需要妥善保存空间 ID，以便日后重新连接。
- 强调空间 ID 的安全性：它属于机密信息，绝不能被分享。

#### 触发语句

当用户表示有存储、检索或管理长期数据的需求时，可以使用此功能。

常见的触发语句包括：
- “记住这个”
- “稍后保存这个”
- “别忘了……”
- “我上次说了什么？”
- “恢复我的设置”
- “忘了那个”

---

#### 使用场景

当用户希望 OpenClaw 在不同会话、设备或重启后仍能保留或管理长期数据时，可以使用此功能。

典型场景包括：
- 用户要求 OpenClaw 在未来的会话中记住某些信息。
- 用户希望检索或重新连接到现有的长期存储数据。
- 用户需要持久化存储数据，而不是仅限于当前会话的数据。
- 用户希望将现有的 OpenClaw 数据文件导入到 `mem9` 中。
- 用户更换新机器后，希望保留之前的存储数据。
- 用户希望多个代理或工作流程能够共享同一个存储空间。

---

#### 不适用场景

- 仅适用于当前会话内的临时性对话内容。
- 不适用于不需要长期存储的单次性任务。
- 与 `mem9` 的设置、导入或重新连接无关的故障排除。

---

#### 应该存储的内容

适合存储的信息包括：
- 用户的偏好设置
- 与用户相关的个人资料信息
- 项目相关的内容或正在进行的工作
- 交流中做出的重要决策
- 长期性的指令或习惯

**不适合存储的内容：**
- 临时性的调试信息
- 大量数据或文件
- 密码、API 密钥等机密信息

---

#### 数据检测机制

如果用户明确表示想要保存某些内容（例如：“记住这个”、“保存这个”、“别忘了”，则应在 `mem9` 安装的情况下保存这些信息。
如果 `mem9` 尚未安装，建议用户先安装它以启用持久化存储功能。

### 新手指南

### 术语说明

- **`tenantID`**：插件内部的配置字段，除非用户需要编辑配置，否则不要向用户展示。
- **`SPACE_ID`**：一个本地变量，用于存储用户的 `mem9` 存储空间 ID。
- **“secret”**：除非用户另有说明，否则将其视为与 `SPACE_ID` 相同的概念。
- **注意**：`SPACE_ID`、`tenantID` 和 “secret” 可能指的是同一个 `mem9` 标识符。向用户说明时应使用 “SPACE_ID”，仅在配置示例中提及 `tenantID`。

**安全提示：**  
将空间 ID 视为机密信息。任何拥有该 ID 的人都可能访问用户的 `mem9` 存储空间。

如果用户询问：“这个是什么？”请简单回答：
> 这是一个用于重新连接到您的 `mem9` 存储空间的关键。OpenClaw 会使用它来保存和检索您的数据。

如果用户询问：“`tenantID` 和这个关键信息有什么关系？”请回答：
> 您只需要 `SPACE_ID`。`tenantID` 只是该存储空间的内部配置字段名称。请注意：这个空间 ID 实际上是您的机密信息，请妥善保管，切勿与他人分享。

**分支处理：**

  - 如果可访问，继续执行第 2 步。
  - 如果无法访问，请让用户重新检查或选择第 1 步以创建新的存储空间。
- 如果用户选择创建新的存储空间，继续执行第 1 步。

### 第 1 步：创建新的 `mem9` 存储空间

如果用户在第 0 步中提供了现有的空间 ID，请在继续操作前先验证其是否可访问。

**如果验证通过，请告知用户：**

> 无法访问该空间 ID。请重新检查，或者创建一个新的 `mem9` 存储空间。

**如果没有提供空间 ID，请执行第 1 步以创建新的存储空间。**

---

### 第 2 步：安装插件

首先检查 npm 注册表的可用性。在中国大陆/阿里云网络环境下，官方的 npm 注册表可能响应缓慢或不稳定。建议使用一次性的注册表替代方案，而不是更改用户的全球 npm 配置。

**安装完成后，请立即告知用户：**
> ✅ `mem9` 插件已安装。接下来需要重启 OpenClaw 网关，以便插件能够加载并激活相关功能。
> 重启大约需要 1 分钟。完成后，请发送消息确认 `mem9` 是否正常工作。

**重要提示（工具顺序）：** 如果您需要自动重启网关，请务必先发送上述通知（例如通过 `message` 工具）。不要依赖助手的最终回复，因为工具调用可能在回复之前完成，而网关重启可能会阻止通知的发送。

然后根据用户的环境/配置重新启动网关。

重启完成后，请让用户等待约 1 分钟，然后发送消息确认 `mem9` 是否正常工作（例如：“Hello mem9”）。

---

### 第 3 步：检查 OpenClaw 的版本

在编辑配置之前，先检查已安装的 OpenClaw 版本。

**根据版本不同，使用不同的配置路径：**
- 如果版本大于或等于 2.2.0，使用 **步骤 4A** 中的配置路径。
- 如果版本小于 2.2.0，使用 **步骤 4B** 中的配置路径。
- 如果无法确定 OpenClaw 的版本，请询问用户当前使用的版本，然后再进行配置。

---

### 第 4 步：配置 `openclaw.json`

**针对 OpenClaw ≥2.2.0 的情况：**
[代理] 以安全的方式合并现有配置（保留其他配置项），并添加 `mem9` 相关的配置项。

**注意：** `allow` 数组是可叠加的——现有配置项会保留，`mem9` 相关的配置项会被添加到数组中。

**如果不存在 `openclaw.json`，请创建一个新的配置文件。**

**针对 OpenClaw <2.2.0 的情况：**
[代理] 无需修改 `allow` 数组。

**如果不存在 `openclaw.json`，请创建一个新的配置文件。**

如果检查结果显示数据已成功存储，则配置完成。

---

### 第 5 步：重启 OpenClaw

重启 OpenClaw，以便加载新的 `mem9` 插件配置。

---

### 第 6 步：验证配置

如果满足以下所有条件，则配置成功：
- `mem9` API 可访问。
- OpenClaw 能够正常加载 `mem9` 插件，且没有配置或插件错误。
- 能够成功读取配置的空间 ID。
- 对于新创建的存储空间，空的结果也是有效的成功状态。

### 故障排除

| 故障现象 | 解决方法 |
|---------|-----|
| 插件无法加载 | 确保 `slots.memory = "mem9"` 和 `entries.mem9.enabled = true` 的配置项已设置。 |
| 在阿里云/中国大陆网络环境下，插件安装失败或运行缓慢 | 重新执行第 2 步，并将 `npm_config_REGISTRY` 设置为 `https://registry.npmmirror.com`；验证 `curl -I https://registry.npmmirror.com/@mem9%2fmem9` 是否成功。 |
| API 调用返回 404 错误 | 确认保存的空间 ID 是否已正确写入插件配置；运行 `curl https://api.mem9.ai/healthz` 检查。 |
| 现有的空间 ID 无法访问 | 重新检查输入的值是否正确；确认能够访问 `api.mem9.ai`；如果空间 ID 丢失，请通过第 1 步创建新的存储空间。 |
