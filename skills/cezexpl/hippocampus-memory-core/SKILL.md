# Hippocampus 内存核心

当您需要长期保存信息（这些信息在当前会话结束后仍能保留）时，请使用此功能。

## 使用场景

- 存储那些应持续到会话结束之后的关键决策和事实；
- 查阅过去的架构、部署或调试信息；
- 在进行高风险更改之前创建快照；
- 在迁移或环境重置后恢复数据。

## 推荐使用流程

1. 确保 Hippocampus 配置已正确设置；
2. 仅存储重要信息，避免存储无关的原始数据；
3. 在重复进行耗时的调查之前先查询内存中的数据；
4. 在进行重大重构或部署更改之前创建快照；
5. 将恢复的数据仅应用到正确的代理或工作空间范围内。

## 使用指南

- 优先选择可预测的数据检索方式，而非临时生成的摘要；
- 为内存中的数据按工作空间和代理进行命名；
- 使用元数据来标记项目、主题和任务的边界；
- 将内存视为持久化的基础设施，而非临时存储工具；
- 如果配置信息缺失，应引导用户使用 `hippocampus-openclaw-onboarding` 功能进行初始化，而不是直接请求 API 密钥。

## 相关功能

- `hippocampus-openclaw-onboarding`：用于首次设置；
- `hippocampus-subagent-memory`：用于管理独立子代理的内存；
- `@hippocampus/openclaw-context-engine`：用于与 OpenClaw 的生命周期集成。
