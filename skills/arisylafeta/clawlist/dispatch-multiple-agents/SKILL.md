# 分配多个代理任务

当你有多个独立任务需要处理时，不要依次执行它们，而是应该同时分配代理任务让它们并行工作。

## 适用场景

**适用情况：**
- 需要处理3个或更多独立任务；
- 这些任务之间没有状态或资源的共享；
- 任务之间不存在依赖关系；
- 时间效率至关重要。

**不适用情况：**
- 任务之间存在关联（解决其中一个任务可能会影响到其他任务）；
- 任务需要共享文件或资源（这可能导致冲突）；
- 所有任务都需要完整的系统上下文；
- 任务必须按照特定顺序执行。

## 实施步骤

### 1. 确定独立的任务领域

根据任务内容将它们分组：
- 任务A：研究竞争对手A
- 任务B：研究竞争对手B
- 任务C：研究竞争对手C

每个任务都是独立的。

### 2. 为每个代理任务指定具体任务

为每个代理任务分配以下内容：
- **明确的任务范围**；
- **所需的所有相关信息**（避免让代理任务自行寻找所需资料）；
- **明确的输出结果**；
- **禁止操作的内容**（明确指出哪些部分不可修改）。

### 3. 并行执行任务

使用`sessions_spawn`函数来同时启动这些代理任务。

### 4. 审查与整合结果

当代理任务完成并返回结果后：
- 逐一查看每个任务的结果；
- 确认没有冲突或重复的信息；
- 整合所有收集到的信息；
- 提交总结报告。

## 与其他技能的结合使用

可以在工作流程中结合使用`dispatch-multiple-agents`功能：

```
brainstorming → write-plan → dispatch-multiple-agents → verify-task
                                   ↓
                            doing-tasks (per subagent)
```

## 示例

**问题：**研究5个竞争对手

**任务分配：**
- 代理1 → 竞争对手A
- 代理2 → 竞争对手B
- 代理3 → 竞争对手C
- 代理4 → 竞争对手D
- 代理5 → 竞争对手E

**结果：**几分钟内即可完成全部分析。
