# 聊天协议中的上下文范围标签

这是一种轻量级、可移植的约定，用于明确指定聊天的上下文边界。

## 复制/粘贴快捷指南（一个屏幕显示）

- `[ISO: <主题>]`：表示这条消息是一个全新的讨论，没有之前的项目/主题背景。
- `[SCOPE: <主题>]`：将讨论范围限制在指定的主题内。
- `[GLOBAL]`：允许跨主题使用之前的讨论内容（需要明确指出使用了哪些内容）。
- `[NOMEM]`：不要将本次对话中的信息存储到长期记忆中。
- `[REM]`：需要将某些偏好设置或决策保存下来（这需要依赖内存后端；否则仅作为建议）。

示例：
- `[ISO: marketing][NOMEM]：为 OpenClaw 草拟 5 个广告方案；不要将这些信息存储到长期记忆中。`
- `[SCOPE: openclaw-mem]：解释为什么 A 通道出现了问题；请将讨论范围限定在这个主题内。`
- `[GLOBAL][REM]：记住：除非另有说明，否则显示的时间应为亚洲/台北的时间。`

## 快速入门

1) 在消息的开头添加一个或多个标签。
2) 推荐的标签使用顺序是：**范围标签** 然后是 **内存标签**。
3) 其余部分可以正常书写。

**可选**：如果你的助手支持命令式快捷方式，可以使用 `/ctx` 或 `/context_def` 来显示这个快捷指南。

## 标签解析规则

- 标签必须出现在用户消息的开头。
- 可以同时使用多个标签。
- 标签不会覆盖安全策略、工具访问控制、审批流程或平台规则。

### 矛盾的标签组合

某些标签组合可能会产生冲突（例如 `[ISO]` 和 `[GLOBAL]`，或者 `[REM]` 和 `[NOMEM]`）。

**推荐的处理方式**：
- 对于冲突的标签组合，以最后一个标签的定义为准。
- 如果组合含义不明确，请提出问题以获取澄清。

## 支持的标签

### 隔离/范围

- `[ISO: <主题>]` / `[Isolated Context: <主题>]`：
  - 将此讨论视为一个全新的主题，不要引入其他对话或项目的背景信息。
  - 除非用户明确提供，否则不要使用其他对话或项目的上下文。
  - 允许一些通用的安全规则和用户偏好设置（如时区设置、“未经批准不得应用更改”等）的隐式延续。

- `[SCOPE: <主题>]` / `[Scoped Context: <主题>]`：
  - 将讨论范围限制在指定的主题内。
  - 如果在指定范围内缺少某些信息，请提出问题以获取澄清。

- `[GLOBAL]` / `[Global Context OK]`：
  - 允许跨主题使用之前的讨论内容。
  - 在使用之前的讨论内容时，需要明确指出使用了哪些内容。

### 内存处理

- `[NOMEM]` / `[No Memory]`：
  - 不要将本次对话中的信息存储到长期记忆中。

- `[REM]` / `[Remember]`：
  - 表示需要将消息中的偏好设置或决策保存下来。
  **注意**：实际保存这些信息需要依赖主机代理启用内存支持。

## 默认行为（未使用标签的情况）

- 在跨主题混合讨论时要谨慎操作。
- 如果用户对信息泄露感到不满，建议使用上述标签。

## 跨平台/聊天界面注意事项

- Telegram 的斜杠命令不能包含连字符（-）。
- 使用 `/context_def`（下划线形式），而不是 `/context-def`。
- 斜杠命令可能会与其他机器人或功能冲突。
- 如果 `/ctx` 已被占用，可以直接使用标签语法（该语法在所有平台上都有效）。
- 标签本身只是文本，它们在 Telegram、Discord、Slack 和 WhatsApp 上的使用方式相同。
- 如果聊天界面自动格式化了括号，只需将标签放在消息的开头即可。
