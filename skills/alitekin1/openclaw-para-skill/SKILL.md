# PARA 组织技巧

使用 PARA 方法来保持工作区的整洁和有序。

## 结构
- `/root/clawd/1-Projects/`：有截止日期或目标的正在进行中的项目。
- `/root/clawd/2-Areas/`：正在处理的职责（例如：财务、健康、个人发展）。
- `/root/clawd/3-Resources/`：兴趣爱好和相关参考资料。
- `/root/clawd/4-Archives/`：已完成的项目或不再使用的资源/资料。

## 规定：自动组织
对于所有的文件操作，你必须遵循 PARA 结构。除非是核心身份文件（如 AGENTS.md、MEMORY.md 等），否则切勿将新文件放在根目录 `/root/clawd/` 中。

1. **新项目文件？** 在 `1-Projects/` 下创建一个子文件夹。
2. **长期参考资料或数据？** 放入 `3-Resources/`。
3. **已完成的工作？** 移动到 `4-Archives/`。
4. **系统/配置更新？** 保存在 `2-Areas/` 中。

## 工作流程
1. **识别**：确定文件或目录属于哪个类别。
2. **移动**：使用 `mv` 命令将其放入正确的 PARA 文件夹中。
3. **维护**：定期审查项目，并将已完成的项目移至 `Archives` 文件夹。

## 分类指南
- **项目**：机器人开发、特定的作业任务、正在撰写的报告。
- **职责领域**：财务记录（`MIANJY`）、电子邮件配置、技能提升相关资料。
- **资源**：研究论文、学习用图表、代码片段。
- **档案**：旧报告、已完成的学期文件。
