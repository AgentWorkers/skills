# 摘要生成器  
从只读的潜在客户记录中生成简洁的每日摘要。  

## 快速触发指令：  
- 提供包含趋势分析的今日潜在客户摘要。  
- 按地区显示高优先级的潜在客户数量。  
- 总结本周中介的库存情况与买家需求。  

## 推荐的工作流程：  
`情感分析-优先级评分器 -> 摘要生成器 -> 行动建议器`  

## 执行工作流程：  
1. 接收主管提交的日期范围请求。  
2. 使用 `references/summary-input.schema.json` 格式验证请求。  
3. 通过只读权限查询潜在客户数据。  
4. 综合以下指标生成摘要：  
   - `new_leads_count`（新潜在客户数量）  
   - `trends`（趋势数据）  
   - `record_type_breakdown`（潜在客户类型，如“库存列表”或“买家需求”）  
   - `priority_breakdown`（优先级，如“P1”、“P2”、“P3”）  
   - `urgency_breakdown`（紧急程度，如“高”、“中”、“低”）  
   - `top_localities`（业务量最大的地区）  
   - `dataset_mode`（根据请求或数据源自动确定）  
5. 使用 `references/summary-output.schema.json` 格式验证输出结果。  
6. 仅返回经过验证的摘要数据。  

## 规则限制：  
- 禁止解析聊天记录。  
- 禁止从消息中提取新潜在客户信息。  
- 禁止修改存储数据。  
- 禁止建议或执行任何后续操作。  
- 禁止将报告直接发送到外部系统。  

## 错误处理：  
1. 明确反馈查询或验证失败的原因。  
2. 如果请求范围内的潜在客户数据不存在，则返回零值指标。  
3. 如果没有读取权限，则终止操作。
