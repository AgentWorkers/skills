# 领导者信息提取器（Lead Extractor）

该工具用于从已解析的消息中识别潜在的领导者信息，并生成规范的领导者对象。

## 主要功能：

- 从指定的 WhatsApp 聊天记录中提取所有潜在买家信息。
- 从这些消息中提取联系人和预算信息。
- 识别出与房产相关的正式咨询请求。

## 推荐的工作流程（Recommended Workflow）：

`message-parser -> lead-extractor -> india-location-normalizer`

## 执行步骤：

1. 从主管处接收已解析的消息。
2. 使用 `references/parsed-message-input.schema.json` 标准验证输入数据。
3. 应用 `references/extraction-rules-re-india-v1.md` 中针对该聊天平台的提取规则。
4. 根据咨询意图、联系信息以及与房产相关的偏好来识别潜在的领导者对象。
5. 当消息正文中包含价格、面积或位置信息时，将多行内容视为一条候选记录。
6. 创建领导者记录，包含以下必填字段：
   - `lead_id`（领导者ID）
   - `name`（姓名）
   - `phone`（电话号码）
   - 可选字段：
     - `property_type`（房产类型）
     - `budget`（预算）
     - `deal_type`（交易类型）
     - `asset_class`（资产类别）
     - `price_basis`（价格计算方式）
     - `area_sqft`（面积，平方英尺）
     - `location_hint`（位置提示）
7. 将电话号码从多种格式（如 `+91 98205 82462` 或 `98200 78845`）统一转换为标准格式。
8. 区分价格相关信息和租金相关信息：
   - 例如：`3.5 Lakh rent`（月租金），`60K psf`（每平方英尺的价格），`4.25 Cr`（总金额）
9. 当记录明确指向同一人时，通过唯一键（如电话号码）去重。
10. 使用 `references/output-leads.schema.json` 标准验证提取的结果。
11. 仅返回经过验证的领导者对象。

## 规范限制（Boundaries）：

- 禁止对任何持久化存储进行写入或更新操作。
- 禁止修改原始消息内容。
- 禁止生成任何摘要信息。
- 禁止建议或执行任何后续操作。
- 禁止发送任何通信内容或触发外部系统。

## 错误处理（Error Handling）：

- 拒绝无效的解析后的输入数据。
- 如果没有找到任何领导者信息，返回一个空数组。
- 如果提取的记录不符合数据结构规范，返回详细的字段级错误信息。
