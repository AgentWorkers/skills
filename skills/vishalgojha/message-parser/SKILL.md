# 消息解析器

将原始的聊天数据转换为格式规范的解析后的消息对象数组。

## 执行工作流程

1. 接收以纯文本、JSON格式提供的WhatsApp聊天数据，或由Supervisor已加载的文件内容作为输入。
2. 检测并解析输入数据的格式。
3. 将每个聊天事件规范化为以下三个字段：
   - `timestamp`（遵循RFC 3339标准的日期时间字符串）
   - `sender`（非空字符串）
   - `content`（字符串；允许消息内容为空）
4. 保持消息的原始顺序；如果时间戳重复，仍保留原始的发送顺序。
5. 根据`references/parsed-message-array.schema.json`文件验证解析结果。
6. 仅返回经过验证的消息对象数组。

## 遵守限制

- 禁止推断或提取任何潜在的线索（leads）。
- 禁止将数据写入数据库或文件。
- 禁止生成任何摘要或行动计划。
- 禁止发送或安排任何外发通信。
- 禁止绕过Supervisor的路由机制。

## 处理错误

1. 对格式错误的条目返回明确的解析错误信息。
2. 如果源代码行信息可用，应包含对应的行号。
3. 如果解析结果不符合规范，立即终止程序执行并返回错误信息。
