# 以色列股票分析

## 概述

本技能提供对在特拉维夫证券交易所（TASE）交易的以色列股票和交易型开放式指数基金（ETFs）的全面分析，涵盖基本面分析（财务状况、企业质量、估值）、技术分析（指标、趋势、形态）、同行比较，并生成详细的投资报告。通过网络搜索获取实时市场数据，并应用结构化的分析框架。支持使用希伯来语和英语进行分析。

## 支持的标识符

本技能支持三种方式来识别以色列证券：

1. **TASE股票代码**（例如：ALRT、DSCT、TASE、PELEOT）
2. **ETF编号**（以色列ETF的识别编号，例如：510893、770001）
3. **公司名称**（希伯来语或英语，例如：“בנק לאומי”（Bank Leumi）、“טבע”（Teva））

## 数据来源与获取策略

使用可用的数据采集工具（MCPs）和直接API调用来收集以色列证券的当前市场数据：

**主要获取的数据：**
1. **当前股票/ETF的价格和交易数据**（价格、成交量、52周价格范围，单位为新以色列谢克尔（ILS））
2. **财务报表**（损益表、资产负债表、现金流量表，单位为ILS）
3. **关键指标**（市盈率（P/E）、每股收益（EPS）、股息收益率、账面价值、净资产收益率（ROE）、债务比率）
4. **分析师评级和目标价格**（以色列和国际分析师的评级）
5. **最新新闻和动态**（希伯来语财经新闻来源）
6. **同行/竞争对手数据**（用于以色列市场比较）
7. **技术数据**（移动平均线、相对强弱指数（RSI）、MACD等）

**数据获取方法（优先顺序）：**

**选项1：捆绑脚本（推荐）**
- 使用 `scripts/fetch_tase_data.py`（Python，功能更强大）
- 或 `scripts/fetch_tase_data.sh`（Shell，更简洁）
- 两者都支持：
  - TASE股票代码（ALRT、DSCT、TASE等）
  - ETF编号（510893、770001等）
  - 希伯来语公司名称（例如：“בנק לאומי”、“בנק הפועלים”等）
  - 多个API来源（Finnhub、Alpha Vantage）
  - 如果API未配置，则自动使用模拟数据

**选项2：配置API密钥**
- 在 **finnhub.io** 注册以获取实时TASE数据
- 设置环境变量：`export FINNHUB_API_KEY=your_key`
- 脚本将自动使用该密钥获取实时数据
- 其他无需配置——脚本会提供模拟数据

**选项3：可用的数据采集工具（MCPs）**
- 如果配置了财务数据MCP，则使用它来获取TASE数据
- 示例：Bloomberg MCP、Yahoo Finance MCP、TASE API MCP
- 通过以下命令调用：`mcporter call <server>.<arguments>`

**选项4：直接API调用**
- TASE API（如果可用）：直接通过tase.co.il API进行HTTP调用
- ISA（以色列证券管理局）数据API
- 公共财务数据API（Alpha Vantage、Finnhub）

**数据来源优先级：**
1. 配置并可用的数据采集工具（MCPs）
2. 直接API调用（TASE或ISA）
3. 公共财务数据API（Alpha Vantage、Finnhub等，支持以色列数据）
4. 公司投资者关系页面（通过curl自动获取）

**标识符解析：**
- 对于股票代码的查询：直接查询代码符号（ALRT、DSCT、TASE等）
- 对于ETF编号：将ETF编号映射到股票代码或通过ISA数据库查询
- 对于希伯来语名称：翻译或查询公司数据库以获取股票代码

**数据来源质量：**
- TASE官方API或数据库
- ISA（以色列证券管理局）的文件和数据库
- 公司投资者关系页面（希伯来语和英语）
- 以色列银行发布的资料
- 直接财务数据提供商

**货币说明：** 所有以色列证券均以新以色列谢克尔（ILS/₪）计价。除非特别要求转换为美元，否则价格和财务指标均以ILS显示。

## 分析类型

本技能支持四种类型的分析。确定用户需要哪种类型的分析：

1. **基本股票信息** - 提供关键指标的快速概览
2. **基本面分析** - 深入分析企业状况、财务状况和估值
3. **技术分析** - 图表形态、指标、趋势分析
4. **综合报告** - 结合所有方法的完整分析

## 分析工作流程

### 1. 基本股票信息

**使用场景：** 用户请求关于以色列证券的快速概览或基本信息

**步骤：**
1. 确定证券（股票代码、ETF编号或希伯来语名称）
2. 查找当前股票/ETF的数据（价格、成交量、市值）
3. 收集关键指标（市盈率、每股收益、股息收益率、账面价值）
4. 获取52周价格范围和年初至今的表现
5. 查找最新新闻或重要动态（优先考虑希伯来语来源）
6. 以简洁的摘要格式呈现（如果查询语言为希伯来语，则同时提供双语版本）

**输出格式：**
- 公司/基金描述（1-2句话，使用适当的语言）
- 当前价格（ILS）和交易指标
- 关键估值指标（表格）
- 最新表现
- 最新新闻（如有）
- 股息信息（如适用）

### 2. 基本面分析

**使用场景：** 用户需要对以色列股票进行财务分析、估值评估或企业评估

**步骤：**
1. **收集全面的财务数据：**
   - 收入、收益、现金流量（3-5年的趋势，单位为ILS）
   - 资产负债表指标（债务、现金、营运资金）
   - 盈利能力指标（利润率、净资产收益率、资本回报率）
   - 股息政策和历史
2. 阅读 `references/fundamental-analysis.md` 以了解以色列市场背景和分析框架
3. 阅读 `references/financial-metrics.md` 以获取指标定义、计算方法和以色列市场的特殊考虑因素
4. **分析企业质量：**
   - 在以色列/区域市场中的竞争优势
   - 管理层表现
   - 行业地位
   - 监管环境（ISA、以色列银行等）
5. **进行估值分析：**
   - 计算关键比率（市盈率与以色列市场平均水平、市净率、企业价值与息税折旧摊销前利润比率）
   - 与历史平均水平比较
   - 与同行群体比较
   - 估算合理价值范围（单位为ILS）
6. **识别风险：**
   - 公司特定风险
   - 市场/宏观风险（以色列特定风险：政治、安全、货币）
   - 财务数据中的警示信号
   - 监管风险
7. **根据 `references/report-template.md` 的结构生成输出（如适用，提供希伯来语翻译）**

**关键分析点：**
- 盈利能力趋势（利润率是否改善/下降）
- 现金流质量（现金流与收益）
- 资产负债表强度（债务水平与息税折旧摊销前利润比率、流动性）
- 增长可持续性
- 估值与以色列同行和历史平均水平的比较
- 股息的可持续性和增长轨迹

### 3. 技术分析

**使用场景：** 用户请求对TASE证券进行技术分析、图表形态或交易信号

**步骤：**
1. **收集技术数据：**
   - 当前价格（ILS）和近期价格走势
   - 成交量趋势（股票数量和ILS）
   - 移动平均线（20日、50日、200日）
   - 技术指标（相对强弱指数、MACD、布林带）
2. **阅读 `references/technical-analysis.md` 以了解指标定义、形态和以色列市场的特殊考虑因素**
3. **确定趋势：**
   - 上升趋势、下降趋势或盘整趋势
   - 相对于TA25指数（特拉维夫25指数）的趋势强度
4. **定位支撑和阻力水平：**
   - 最近的高点和低点（ILS）
   - 移动平均线水平
   - 心理价位（整数价位）
5. **分析指标：**
   - 相对强弱指数：超买（>70）或超卖（<30）
   - MACD：交叉和背离
   - 成交量：确认或背离信号
   - 布林带：收缩或扩张
6. **识别图表形态：**
   - 反转形态（头肩顶/底、双顶/底）
   - 继续形态（旗形、三角形）
7. **生成技术展望：**
   - 当前趋势评估
   - 需关注的关卡（ILS）
   - 风险/回报分析
   - 短期和中期展望

**解读指南：**
- 用多个指标确认信号
- 考虑成交量的验证作用
- 注意价格与指标之间的背离
- 始终识别风险水平（止损点以ILS为单位）

### 4. 综合投资报告

**使用场景：** 用户请求详细报告、投资建议或对以色列证券的全面分析

**步骤：**
1. **收集数据**（如基本信息部分所述）
2. **执行基本面分析**（按照上述流程）
3. **执行技术分析**（按照上述流程）
4. **阅读 `references/report-template.md` 以了解完整报告结构**
5. **综合分析结果：**
   - 整合基本面和技术分析结果
   - 制定买入/持有/卖出建议
   - 评估风险/回报
   - 考虑以色列市场的宏观环境
6. **生成建议：**
   - 买入/持有/卖出评级
   - 目标价格（ILS单位及时间范围）
   - 确信程度
   - 入场策略
   - 股息考虑（如适用）
7. **按照模板结构创建报告**

**报告必须包括：**
- 包含建议的执行摘要
- 公司/基金概述
- 投资论点（买入/持有/卖出观点）
- 基本面分析部分
- 技术分析部分
- 估值分析
- 风险评估（包括以色列特定风险）
- 催化因素和时间表
- 股息分析（如适用）
- 结论

## 股票比较分析

**使用场景：** 用户请求比较两只或更多以色列股票（例如：“比较Bank Leumi和Bank Hapoalim”）

**步骤：**
1. **收集所有证券的数据：**
   - 对每个股票代码/标识符执行数据收集步骤
   - 确保时间范围一致且所有数据以ILS为单位
2. **阅读 `references/fundamental-analysis.md` 和 `references/financial-metrics.md`
3. **创建并列比较：**
   - 业务模式比较
   - 财务指标表（所有关键比率，单位为ILS）
   - 估值指标表
   - 增长率比较
   - 资产负债表强度
   - 股息收益率比较
4. **识别相对优势：**
   - 每家公司的优势
   - 定量优势
   - 行业定位
5. **技术比较：**
   - 相对强度与TA25指数的比较
   - 动量比较
   - 哪只股票的技术状况更好
6. **生成建议：**
   - 哪只证券更具吸引力及原因
   - 考虑基本面和技术因素
   - 投资组合配置建议
   - 风险调整后的回报评估

**输出格式：** 遵循 `references/report-template.md` 中的“比较报告结构”

## 语言支持

本技能支持使用希伯来语和英语进行分析：

**希伯来语查询：**
- 主要以希伯来语回复
- 使用正确的希伯来语财务术语（例如：“שווי שוק”表示市值，“דיבידנד”表示股息）
- 为清晰起见，同时使用希伯来语和英语的公司名称
- 使用希伯来语来源（Calcalist、TheMarker、Globes）提供市场背景

**英语查询：**
- 用英语回复
- 可以引用希伯来语来源，但需翻译关键发现
- 使用英语财务术语

**混合语言：**
- 如果查询混合使用两种语言，以主要请求的语言回复
- 始终用两种语言清晰标注公司/股票代码名称

## 输出指南

**通用原则：**
- 使用表格展示财务数据和比较结果（便于阅读）
- 突出显示关键指标和发现
- 包括数据来源和日期
- 尽可能量化数据
- 提供买入和卖出观点
- 明确说明假设和不确定性
- 价格和财务指标始终使用ILS单位

**格式要求：**
- 使用标题区分不同部分
- 使用表格展示指标、比较结果和历史数据
- 使用项目符号列出列表和因素、风险
- 使用粗体显示关键发现和重要指标
- 以百分比显示增长率、回报率和利润率
- 货币单位统一格式化（ILS用₪表示，十亿用“B”表示，百万用“M”表示）

**语气：**
- 客观公正
- 承认不确定性
- 用数据支持观点
- 避免夸张表述
- 清晰说明风险
- 考虑以色列市场背景
