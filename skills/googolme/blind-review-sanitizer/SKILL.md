# 盲目审查工具（Blind Review Sanitizer）

该工具可自动对学术论文进行匿名处理，以适应双盲同行评审的要求。它通过移除作者身份信息、机构隶属关系、致谢内容以及过多的自我引用，同时保持文档的格式和学术内容的完整性。

**核心功能：**
- **作者身份移除**：利用模式匹配和可自定义规则，自动检测并删除作者姓名、机构隶属关系及联系方式。
- **致谢部分处理**：识别并移除或标记可能暴露作者身份的致谢部分（如资金来源或个人感谢语）。
- **自我引用检测与处理**：识别第一人称引用和过多的自我引用，这些内容可能会暴露作者身份。
- **支持多种文档格式**：能够处理 DOCX、Markdown 和纯文本文件，并根据格式进行相应的处理。
- **生成审计日志**：生成详细的处理记录，以便进行验证和透明度检查。

## 参数设置

| 参数 | 类型 | 是否必填 | 默认值 | 说明 |
|---------|--------|-----------|--------|-------------------|
| `--input` | str | 是 | - | 输入论文文件的路径（DOCX、MD 或 TXT 格式） |
| `--output` | str | 是 | - | 输出文件的路径 |
| `--authors` | list[str] | 否 | - | 需要删除的作者姓名列表 |

| `--highlight-self-cites` | bool | 否 | false | 仅高亮显示自我引用，不进行替换 |

## 使用场景

**建议在以下情况下使用该工具：**
- 为需要作者匿名的期刊准备论文进行双盲同行评审。
- 提交到要求匿名处理的会议（例如 NeurIPS、ICML、ACL 或主要医学期刊）。
- 在提交前进行最终检查，确保没有留下任何可识别信息。
- 重新处理之前被拒绝的论文，以便提交到具有不同匿名要求的新的期刊。
- 为公开预印本创建匿名版本的论文，同时保持引用信息的完整性。
- 处理需要部分作者保持匿名的合作论文。

**不建议在以下情况下使用：**
- 为开放同行评审或具有透明评审流程的期刊准备论文 → 请使用 `cover-letter-drafter`。
- 论文包含需要作者身份识别的专利申请内容 → 请先咨询法律顾问。
- 需要添加作者信息而非删除信息 → 请使用 `citation-formatter` 进行参考文献管理。
- 处理需要遵守 HIPAA 标准的敏感临床数据 → 请使用 `hipaa-compliance-auditor`。

## 核心功能细节

### 1. 作者身份检测与移除

系统地使用模式识别和用户指定的规则，从论文中检测并移除作者姓名、机构隶属关系及联系方式。

### 2. 机构隶属关系隐藏

自动检测并替换机构标识符（如大学、研究机构、部门、实验室），用通用占位符替代。

### 3. 致谢部分管理

智能识别和处理可能暴露作者身份或机构隶属关系的致谢部分、资金披露内容及个人感谢语。

### 4. 自我引用检测与处理

识别过多的自我引用和第一人称引用，这些内容可能会暴露作者身份，并用中性语言进行替换。

### 5. 多格式文档处理

能够处理 DOCX、Markdown 和纯文本格式的文档，在保持内容完整性的同时处理不同格式。

### 6. 审计日志与验证报告

生成详细的处理日志，以确保透明度和质量控制。

## 质量检查清单

**预处理前的检查：**
- 确保您有权限对所有作者的贡献进行匿名处理。
- 确认目标期刊/会议的匿名化要求。
- 编制完整的作者姓名列表及其变体。
- 识别所有机构隶属关系，包括次要职位。
- 检查补充材料中是否包含作者照片或简介。
- 检查文档属性/元数据中是否包含作者信息。
- 确认没有未处理的变更或评论中包含作者身份。

**处理过程中的检查：**
- 使用高亮显示模式预览所有更改。
- 确认作者列表完整（检查首字母、姓氏、全名）。
- 确认致谢部分的处理方式符合期刊政策。
- 检查自我引用的替换是否保持了引用顺序。
- 检查机构隶属关系的处理是否正确。
- 确认联系信息的移除是否正确（电子邮件、电话号码）。
- 确认图例标题和表格注释是否已处理。

**处理后的验证：**
- 在输出文档中搜索作者姓氏。
- 检查页眉、页脚和页码中是否包含机构标识。
- 确认文档属性是否已清除（文件 → 属性 → 移除个人信息）。
- 检查所有图例中是否包含作者元数据或水印。
- 如果将文档转换为 PDF 格式，验证引用编号是否正确。
- 确保数学符号和符号的表示正确。

## 常见问题与解决方案

- **输入准备问题：**
  - 在处理带有跟踪变更的文档时，请确保接受所有变更并删除评论。
  - 请确保清除文档属性中的作者信息和机构信息。
  - 请确保所有补充文件都经过相同的处理。
  - 请确保包含所有作者的姓名，包括中间名和姓名的变体。

**处理策略问题：**
  - 避免过度替换，以免损坏合法的引用和参考文献。
  - 先使用高亮显示模式进行审查，然后手动检查每个替换内容。
  - 确保处理彻底，避免遗漏任何标识符。
  - 使用不区分大小写的匹配方式；确保正则表达式模式覆盖所有变体。
  - 确保替换后的内容符合上下文。

**输出验证问题：**
  - 确保自动化工具不会遗漏任何作者信息。
  - 在提交前使用搜索功能检查作者姓氏。
  - 请保存并安全存储处理日志，以便后续验证。

## 故障排除**

- 如果处理后的输出中仍出现作者姓名，请检查作者列表是否完整，以及是否正确处理了所有名称变体。
- 如果出现过多的误报，请调整匹配规则或使用更精确的匹配条件。
- 如果文档格式损坏，请确保所有更改都被接受，并在处理前删除评论。
- 如果引用编号或参考文献出现问题，请检查处理后的文档完整性。

## 注意事项与限制

- 自动匿名化不能保证完全匿名，请务必进行手动验证。
- 模式匹配可能会遗漏依赖上下文的标识符或错误地标记合法内容。
- 该工具仅处理文本内容，图像、图表和嵌入对象可能包含未被检测到的识别信息。
- 对 LaTeX 文档的支持有限，建议使用专门的 LaTeX 工具进行处理。
- 该工具针对英语和中文进行了优化，其他语言的处理效果可能不佳。

## 伦理和法律注意事项

- 在提交前必须获得所有作者的匿名化同意。
- 匿名化不会改变版权归属。
- 一些期刊要求提供非匿名化的数据/代码版本。
- 提交后需要安排恢复作者身份的流程。
