# Elen MCP集成技能

> [!重要提示!]  
> 本技能为Learning Nodes技术栈中的决策流程设定了基本规则。您**必须**使用Elen MCP服务器来记录您的决策内容。这样做可以确保决策内容被版本控制、妥善保存，并便于后续查询。

## 1. 什么是Elen MCP？  
Elen Context Server负责管理“决策网络”相关的基础设施，它以MCP服务器的形式在本地运行。  

**安装要求：** 在使用本技能之前，您或您的代理需要配置IDE/Agent设置中的Elen MCP服务器（例如：Cursor、Claude、Windsurf）。  

```json
{
  "mcpServers": {
    "elen": {
      "command": "npx",
      "args": ["-y", "@learningnodes/elen-mcp@0.1.1"]
    }
  }
}
```  

每当您做出与技术、设计、安全或产品相关的决策时（例如：选择特定的框架、设计用户界面组件、构建API接口，或决定删除某个功能），都应使用服务器提供的MCP工具来记录这些决策。  

## 2. 何时使用Elen MCP？  
- **决策前**：使用`mcp_elen_elen_suggest`和`mcp_elen_elen_get_competency`查询您当前处理的议题是否已有相关的决策记录。  
- **决策过程中**：在制定新计划或解决结构问题时，使用`mcp_elen_elen_commit`将决策内容保存到决策网络中。  
- **修复漏洞时**：如果修复漏洞需要系统性的变更或规则更新，请提交新的决策，并引用之前的决策（如果需要完全替换旧决策，则使用`mcp_elen_elen_supersede`）。  

## 3. 格式要求  
通过`mcp_elen_elen_commit`提交决策时，需遵循以下格式：  
- **领域**：简要说明决策所属的领域（例如：“前端”、“基础设施”、“安全”）。  
- **问题描述**：明确阐述具体问题（例如：“市场界面的状态管理工具应选择哪种？”）。  
- **决策内容**：提供清晰、简洁的决策结果。  
- **约束条件**：列出必须遵守的规则或限制条件（例如：`“禁止使用内部API”，“文件大小必须小于1 MB”）。  

## 4. 避免无依据的更改  
在实施可能影响整个代码库的变更之前，请务必先在决策网络中核实相关决策记录。这样能确保决策过程透明化，避免因信息不透明导致的误解或错误。
