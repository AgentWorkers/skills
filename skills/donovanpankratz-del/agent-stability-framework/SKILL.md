# 代理稳定性框架（Agent Stability Framework, ASF）

**防止行为漂移 · 检测故障 · 确保角色一致性**

确保你的AI代理在多次会话中以及随着时间的推移，始终保持稳定、符合角色设定，并具备自我修正的能力。

## 该框架解决的问题

有三点会严重影响代理的可靠性：
1. **行为漂移** — 代理逐渐恢复到默认的训练状态，失去个性化特征。
2. **故障** — 代理产生错误的输出、出现幻觉、自相矛盾，或者无声地失效。
3. **角色不一致** — 代理在技术上能够正常运行，但感觉“不对劲”——失去了其本质。

ASF通过一个集成系统来解决这三个问题。

## 你将获得什么

- 完整的框架文档（AGENT_STABILITY_FRAMEWORK.md）
- 文件模板（SOUL.md、BASELINE_EXAMPLES.md、日志文件）
- 可直接粘贴的系统提示语句
- 检测清单和评分系统
- 适用于所有模型：Claude、GPT、Grok、Gemini、Llama、Mistral

## 快速入门

1. 将所有文件复制到代理的工作空间中。
2. 填写 `SOUL.md`（描述你的代理的身份特征）。
3. 创建 `BASELINE_EXAMPLES.md`（包含10个及以上正确的响应示例）。
4. 在系统提示中添加预设指令。
5. 24小时后进行首次审计。

**设置时间：** 45-90分钟  
**日常维护：** 5分钟  
**已在8种以上模型上测试过**

## 三层防御机制

### 第一层：防止行为漂移
- 预设指令（二进制规则）
- 预发送机制（用于删除错误信息）
- 故障检测机制
- 定期重置功能

### 第二层：检测故障
- 跟踪7种类型的故障
- 行动前的自我检查机制
- 故障日志与恢复流程
- 防止幻觉、自相矛盾和无声故障的发生

### 第三层：确保角色一致性
- 检测“技术上正确但不符合角色设定的”响应
- 角色一致性测试
- 恢复流程
- 用户反馈作为最终判断依据

## 包含的文件

- `AGENT_STABILITY_FRAMEWORK.md` — 完整的框架文档（13KB）
- `SOUL_TEMPLATE.md` — 身份描述模板
- `BASELINE_EXAMPLES_TEMPLATE.md` — 响应示例模板
- `DRIFT_LOG TEMPLATE.md` — 行为漂移跟踪日志

## 使用场景

- 需要保持一致性格的个人AI助手
- 不能产生错误数据的交易机器人
- 需要稳定语气的内容生成代理
- 需要可靠响应的客户服务机器人
- 需要保持准确性的研究辅助工具
- 任何需要24/7运行或跨多个会话使用的代理

## 为什么有效

1. **二进制规则更可靠** — “永远不要做X”这样的规则能够始终如一地被执行。
2. **示例帮助确立角色** — 基线响应作为参考标准。
3. **针对不同问题采取不同防御措施** — 行为漂移、故障和角色不一致需要不同的应对策略。
4. **自我修正利用LLM的能力** — AI可以通过特定规则自我审计。
5. **日志记录形成长期参考** — 重复出现的错误会成为后续的预防措施。
