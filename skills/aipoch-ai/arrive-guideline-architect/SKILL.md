# ARRIVE 指南架构师

## 概述

这是一个基于人工智能的协议设计工具，能够生成符合 ARRIVE 2.0 指南（动物研究：体内实验报告）的动物研究方案。该工具可生成结构化的文档，用于伦理审查、透明报告以及促进可重复的科学研究。

**核心功能：**
- **协议生成**：生成完全符合 ARRIVE 2.0 要求的研究方案
- **样本量计算器**：进行统计功效分析并提供理由说明
- **合规性检查器**：验证现有协议是否符合 ARRIVE 标准
- **随机化方案生成器**：生成并记录分配策略
- **伦理支持**：提供 IACUC 协议模板和动物福利相关文档
- **报告模板**：帮助准备包含所有必要元素的论文

## 使用场景

**✅ 在以下情况下使用该工具：**
- 设计需要伦理批准的新动物研究
- 准备提交给 IACUC（机构动物护理和使用委员会）的申请
- 撰写需要符合 ARRIVE 标准的期刊论文（如 PLOS、Nature 等）
- 验证现有协议的透明度和完整性
- 培训研究人员了解动物研究的最佳实践
- 规划需要标准化方案的多站点研究
- 审查用于申请资助的协议

**❌ 请勿在以下情况下使用：**
- 人类临床试验 → 使用 `clinical-protocol-designer`
- 仅限细胞培养的体外研究 → 不适用 ARRIVE 标准
- 野外野生动物研究 → 使用专门的野生动物研究指南
- 兽医临床案例 → 使用兽医病例报告标准
- 系统评价/荟萃分析 → 使用 PRISMA 指南

**集成方式：**
- **上游工具**：`sample-size-power-calculator`（统计设计）
- **下游工具**：`iacuc-protocol-drafter`（伦理提交）、`manuscript-prep-assistant`（论文辅助工具）

## 核心功能

### 1. ARRIVE 2.0 协议生成器

生成涵盖所有 10 个关键要素的完整协议：

**生成内容：**
1. **研究设计**：实验组、时间线、评估终点
2. **样本量**：包括功效计算及理由说明
3. **纳入/排除标准**：动物选择标准
4. **随机化**：分配方法（软件/硬件实现）
5. **盲法设计**：谁、何时、如何实施盲法
6. **结果指标**：主要、次要和探索性终点
7. **统计方法**：分析计划、使用的统计软件、显著性水平
8. **实验动物**：物种、品系、性别、年龄、体重、来源
9. **实验步骤**：详细的实验方法及时间安排
10. **结果报告**：数据展示模板

### 2. 样本量计算器

进行符合 ARRIVE 标准的统计功效分析，并提供理由说明：

**特点：**
- **效应量选择**：Cohen's d、比值比、风险比
- **多重比较**：Bonferroni、FDR 校正
- **退出率调整**：考虑预期的样本流失情况
- **理由说明文本**：自动生成样本量选择的理由
- **功效曲线**：为不同样本量生成功效计算结果

### 3. 合规性检查器

检查现有协议是否符合 ARRIVE 2.0 标准：

**输出结果：**

**验证内容：**
- **所有 10 个关键要素**：所有出版物都必须满足这些要求
- **顶级期刊推荐的内容**：部分顶级期刊有额外要求
- **特定期刊要求**：针对不同出版商的定制检查

### 4. 随机化与盲法设计生成器

生成并记录分配方案：

**支持的方法：**
- 简单随机化
- 分组随机化（固定/随机分组大小）
- 分层随机化（按性别、年龄、基线分组）
- 考虑协变量的随机化策略

## 常见研究模式

### 模式 1：药物疗效研究

**治疗干预研究模板：**

**关键注意事项：**
- 包含阳性对照以验证实验效果
- 设计多个剂量以确定剂量-反应关系
- 根据预期的效应量进行功效计算
- 样本量设计需考虑疾病变异性

### 模式 2：毒理学研究

**安全性评估模板：**

**关键注意事项：**
- 根据最大耐受剂量（MTD）选择剂量
- 设立恢复组以评估可逆性
- 对所有高剂量组和对照组动物进行全面的临床病理学检查

### 模式 3：行为学研究

**神经科学/行为学研究模板：**

**关键注意事项：**
- 设计对照实验顺序以消除学习效应
- 采用盲法视频分析以避免偏见
- 提供标准化的实验环境（光照、噪音控制）
- 对实验者进行培训并测试其操作的一致性

### 模式 4：外科手术模型研究

**基于手术程序的研究模板：**

**关键注意事项：**
- 提供详细的手术步骤和时间安排
- 提供全面的围手术期护理
- 明确的人道主义评估标准
- 设立假手术对照组以评估手术效果
- 按照 IACUC 指南进行疼痛管理

## 质量检查清单

**研究前：**
- [ ] **必须**：在开始研究前获得 IACUC 批准
- [ ] 样本量具有足够的统计功效（≥80%）
- [ ] 随机化方法记录清晰且可重复
- [ ] 所有评估者的盲法方案明确
- [ ] 人道主义评估标准明确
- [ ] 纳入/排除标准预先指定

**研究过程中：**
- [ ] 严格遵循随机化方案
- [ ] 保持盲法（仅在安全评估时揭盲）
- [ ] 记录所有动物的情况（使用 CONSORT 格式）
- [ ] 记录并报告不良事件
- [ ] 在预定时间点收集样本

**报告时：**
- **论文中涵盖所有 10 个关键要素**  
- **使用 CONSORT 格式记录实验流程**  
- **提供原始数据（或数据共享声明）**  
- **披露利益冲突**  
- **注明资金来源**

## 常见问题

**设计方面的问题：**
- ❌ **对照组设置不当** → 无法区分处理组与混杂因素的影响  
  - ✅ 必须包含适当的对照组（空白组、阳性对照组、假手术组）

- ❌ **采样方法不科学** → 导致选择偏差  
  - ✅ 必须将动物随机分配到各处理组

- ❌ **评估过程不透明** → 评估者可能存在偏见  
  - ✅ 尽可能采用盲法评估结果

**样本量方面的问题：**
- ❌ **未进行功效计算** → 研究效力不足，可能导致假阴性结果  
  - ✅ 必须事先计算样本量并提供理由

- ❌ **未考虑样本流失** → 最终样本量可能过小  
  - ✅ 必须考虑预期的样本流失率（通常为 10-20%）

**报告方面的问题：**
- ❌ **选择性报告结果** → 会导致发表偏倚  
  - ✅ 必须预先登记所有主要的和次要的评估终点

**参考资料**

参考资料位于 `references/` 目录中：
- `arrive_2.0_guidelines.md`：官方的 ARRIVE 2.0 指南和解释
- `sample_size_guidelines.md`：动物研究的统计方法指南
- `species_specific_requirements.md`：针对小鼠、大鼠、斑马鱼的特定要求
- `journal_compliance.md`：不同期刊的出版要求
- `statistical_methods.md：常见研究设计的统计方法指南
- `iacuc_templates.md`：伦理委员会申请模板
- `example_protocols.md：已发布的符合 ARRIVE 标准的协议示例

**脚本**

脚本位于 `scripts/` 目录中：
- `main.py`：协议生成的命令行工具
- `arrive_builder.py`：核心协议生成脚本
- `sample_size.py`：样本量计算脚本
- `randomization.py`：随机化方案生成脚本
- `validate.py`：合规性检查脚本
- `checklist_generator.py`：交互式检查清单工具
- `export.py`：多格式输出脚本（PDF、Word、Markdown）

**限制说明：**
- **基于模板生成**：生成的协议为标准格式；高度专业化的研究可能需要定制
- **不提供统计分析**：仅计算样本量，不执行实际数据分析
- **无实时监控**：仅生成协议，不跟踪实验过程
- **物种适用性**：主要针对小鼠和大鼠设计；其他物种可能需要调整
- **法规差异**：不同机构的 IACUC 要求可能有所不同，可能需要根据实际情况进行定制

---

**🐾 请记住：3R 原则（替代、减少、优化）是伦理研究的基石。本工具支持减少实验动物数量（优化样本量）和优化实验设计，但在可能的情况下，请优先考虑使用替代方法（如体外实验、计算机模拟）。**

## 参数设置

| 参数 | 类型 | 默认值 | 说明 |
|-----------|------|---------|-------------|
| `--interactive` | 标志 | - | **交互模式**：通过引导式提示运行工具（使用 `input()` 进行用户交互。适合初次使用或复杂研究设计） |
| `--input` | 字符串 | 必填 | 输入 JSON 文件路径（批量/自动化模式） |
| `--output` | 字符串 | "protocol.md" | 输出文件路径 |
| `--validate` | 字符串 | 必填 | 验证现有协议文件是否符合标准 |
| `--checklist` | 字符串 | 必填 | 生成符合 ARRIVE 2.0 标准的检查清单 |
| `--format` | 字符串 | "markdown" | 输出格式：markdown、pdf 或 docx |

**使用方式：**
- **自动化模式（推荐用于持续集成/持续部署）**：使用 `--input` 和 JSON 配置文件
- **交互模式**：使用 `--interactive` 通过提示进行设置
