# Vibe Notion

这是一个TypeScript命令行工具（CLI），它允许AI代理和人类通过Notion的非官方私有API与Notion工作空间进行交互。该工具支持对页面、数据库、区块进行完整的CRUD操作（创建、读取、更新、删除），以及搜索和用户管理。

> **注意**：此工具使用的是Notion的内部/私有API（`/api/v3/`），与官方的公共API不同。如需访问官方API，请使用`vibe-notionbot`。

## 使用哪个CLI

该工具提供了两个CLI版本，请根据您的需求选择合适的版本：

| | `vibe-notion` | `vibe-notionbot` |
|---|---|---|
| API | 非官方私有API | 官方Notion API |
| 认证 | 从Notion桌面应用自动提取`token_v2` | 通过环境变量`NOTION_TOKEN`（集成令牌） |
| 身份 | 以用户身份运行 | 以机器人身份运行 |
| 设置 | 无需设置——凭证自动提取 | 需要在notion.so/my-integrations创建集成 |
| 数据库操作 | `add-row`、`update-row` | 通过`page create --database`创建 |
| 视图管理 | `view-get`、`view-update` | 不支持 |
| 工作空间列表 | 支持 | 不支持 |
| 稳定性 | 私有API——Notion更新可能导致故障 | 官方API——稳定版本 |

**决策流程**：

1. 如果安装了Notion桌面应用 → 使用`vibe-notion`（此CLI）  
2. 如果设置了`NOTION_TOKEN`但没有桌面应用 → 使用`vibe-notionbot`  
3. 如果两者都可用 → 更推荐使用`vibe-notion`（功能更丰富，无需设置）  
4. 如果两者都没有 → 请用户安装其中一个工具。

## 重要提示：仅使用CLI

**切勿直接调用Notion的内部API**。始终使用`vibe-notion` CLI提供的命令。不要直接向`notion.so/api/v3/`发送原始HTTP请求，也不要使用任何Notion客户端库。直接调用API可能会导致凭证泄露，并可能触发Notion的滥用检测机制，从而导致用户账户被封禁。

如果`vibe-notion`不支持您需要的功能，请告知用户，并代表他们向[devxoul/vibe-notion](https://github.com/devxoul/vibe-notion/issues)提交功能请求。在提交之前，请删除所有可能识别用户或工作空间的信息（如ID、姓名、电子邮件、令牌、页面内容等），仅使用通用占位符。

## 重要提示：切勿编写脚本

**切勿编写任何脚本（Python、TypeScript、Bash等）来自动化Notion操作**。`batch`命令已经可以处理任意规模的操作。编写脚本来循环调用API是错误的做法——请使用`batch`命令并配合`--file`参数。

**适用情况**：
- 需要创建100多行数据时  
- 需要在新创建的行之间建立关联时（请参考[批量操作策略](#bulk-operations-strategy)  
- 操作规模较大，无法用单个命令完成时  

如果出现这种情况，请停止并使用`batch`命令。

## 快速入门

首次使用时，凭证会自动从Notion桌面应用中提取，无需手动设置。

> **重要提示**：所有在特定工作空间内执行的命令都需要`--workspace-id`参数。可以使用`vibe-notion workspace list`来获取您的工作空间ID。

## 认证

运行任何命令时，凭证（`token_v2`）会自动从Notion桌面应用中提取，无需API密钥、OAuth或手动提取。

在macOS系统中，首次使用时系统可能会请求访问Keychain——这是正常的，用于解密cookie。

提取到的`token_v2`会存储在`~/.config/vibe-notion/credentials.json`文件中，文件权限设置为`0600`。

## 内存管理

代理会维护一个名为`~/.config/vibe-notion/MEMORY.md`的文件，用于在会话之间保存持久化数据。此文件由代理管理，CLI不会读取或写入该文件。您可以使用`Read`和`Write`工具来管理这个内存文件。

### 读取内存

**在每个任务开始时**，使用`Read`工具读取`~/.config/vibe-notion/MEMORY.md`文件，以加载之前发现的工作空间ID、页面ID、数据库ID和用户设置。

- 如果文件不存在，可以忽略它，并在有可用信息时创建该文件。
- 如果文件无法读取（权限问题或目录缺失），也可以忽略该文件，不会导致错误。

### 写入内存

在发现有用信息后，使用`Write`工具更新`~/.config/vibe-notion/MEMORY.md`文件。以下情况需要写入文件：
- 发现工作空间ID时  
- 发现页面ID、数据库ID或集合ID时  
- 用户提供了别名或设置（例如：“将此数据库称为Tasks DB”）  
- 发现页面/数据库结构（父子关系、哪些页面包含哪些数据库）  

**注意**：不要存储以下内容**：
- `token_v2`、凭证、API密钥或任何敏感数据  
- 请勿存储完整的页面内容（仅存储ID和标题）  
- 除非是持久性引用（如数据库区块），否则不要存储区块级别的ID。

### 处理过时数据

如果某个存储的ID导致错误（例如页面未找到或访问被拒绝），请从`MEMORY.md`中删除该ID。不要盲目信任存储的数据，遇到问题时请重新搜索。

### 批量操作策略

对于大量操作（数十到数百条记录），请使用`--file`参数来避免超出Shell的参数限制。

**多步骤处理**：当新创建的行之间需要建立关联时（例如，行A引用行B），可以分两步进行：
1. **第一步**：创建所有行（不包括关联信息），将所有`database.add-row`操作写入一个JSON文件。
2. **第二步**：使用第一步生成的ID，创建另一个JSON文件，其中包含设置关联信息的`database.update-row`操作。

这样就可以避免编写复杂的脚本。

### 速率限制

Notion对其API设置了速率限制。批量操作会按顺序执行，因此大量操作（超过30个）可能会触发“429 Too Many Requests”错误。为了避免这个问题：
- 将大量操作分成每批25-30个操作进行
- 如果批量操作中途失败，请重新运行仅包含未完成操作的批次
- `batch`命令的输出会显示哪些操作成功了——可以使用`index`字段来确定从哪里继续执行

### 使用`--backlinks`的时机

`--backlinks`参数可以显示某个页面/数据库链接到的其他页面/数据库。这对于高效导航非常有用。

**在以下情况下使用`--backlinks`**：
- **追踪关联关系**：搜索结果可能是选择选项、枚举值或关联目标时，`--backlinks`可以立即显示所有引用该目标的页面。
- **查找引用**：找到一个页面后，需要知道其他页面是否引用了它或链接到了它。
- **反向查找**：不需要遍历所有数据库来查找引用某个页面的记录，可以直接使用目标页面的`--backlinks`。

### 分页

支持分页的操作可以通过`has_more`和`next_cursor`字段来实现：

- `block.children`：基于游标的分页。使用上一次响应中的`next_cursor`值作为`--start-cursor`参数。
- `search`：基于偏移量的分页。使用`next_cursor`值（一个数字）作为`--start-cursor`参数。
- `database.query`：使用`--limit`参数来控制页面数量。`has_more`表示还有更多结果，但私有API不支持基于游标的分页——通过增加`--limit`参数来获取更多页面。

### `vibe-notion: command not found`

如果未安装`vibe-notion`包，请使用包管理工具直接运行它。如果知道用户偏好的包管理工具，请直接使用该工具。

**限制事项**：
- 自动提取凭证功能仅支持macOS和Linux系统。Windows系统的DPAPI解密功能尚未实现。
- `token_v2`使用的是非官方内部API，如果Notion对其进行修改，可能会导致功能失效。
- 该工具使用的是私有/非官方API，Notion官方并不支持它。
