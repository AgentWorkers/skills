# Vibe Notion

这是一个TypeScript命令行工具（CLI），它允许AI代理和人类通过Notion的非官方私有API与Notion工作区进行交互。该工具支持对页面、数据库、区块、搜索内容以及用户管理进行完整的CRUD（创建、读取、更新、删除）操作。

> **注意**：此工具使用的是Notion的内部/私有API（`/api/v3/`），与官方的公共API不同。如需访问官方API，请使用`vibe-notionbot`。

## 使用哪个CLI

此包提供了两个CLI工具。根据您的需求选择合适的工具：

| | `vibe-notion` | `vibe-notionbot` |
|---|---|---|
| API | 非官方私有API | 官方Notion API |
| 认证 | 从Notion桌面应用中自动提取`token_v2` | 环境变量`NOTION_TOKEN`（集成令牌） |
| 身份 | 以用户身份运行 | 以机器人身份运行 |
| 设置 | 无需设置——凭证自动提取 | 需要手动在notion.so/my-integrations创建集成 |
| 数据库操作 | `add-row`、`update-row` | 通过`page create --database`创建 |
| 视图管理 | `view-get`、`view-update`、`view-list`、`view-add`、`view-delete` | 不支持 |
| 工作区列表 | 支持 | 不支持 |
| 稳定性 | 私有API——Notion更新可能导致功能失效 | 官方API版本更稳定 |

**决策流程**：
1. 如果安装了Notion桌面应用 → 使用`vibe-notion`。
2. 如果设置了`NOTION_TOKEN`但没有桌面应用 → 使用`vibe-notionbot`。
3. 如果两者都可用 → 优先选择`vibe-notion`（功能更丰富，无需设置）。
4. 如果两者都不具备 → 请用户安装其中一个。

## 重要提示：仅使用CLI

**切勿直接调用Notion的内部API**。始终使用`vibe-notion` CLI提供的命令。不要直接向`notion.so/api/v3/`发送原始HTTP请求，也不要使用任何Notion客户端库。直接调用API可能会导致凭证泄露，并可能触发Notion的滥用检测机制，从而导致用户账户被封禁。

如果`vibe-notion`不支持您需要的功能，请告知用户，并代表他们向[devxoul/vibe-notion](https://github.com/devxoul/vibe-notion/issues)提交功能请求。在提交之前，请删除所有可能识别用户或其工作区的真实数据（如ID、姓名、电子邮件、令牌、页面内容等），使用占位符代替，并在问题描述中仅提及缺失的功能。

## 重要提示：切勿编写脚本

**切勿编写Python、TypeScript、Bash等脚本来自动化Notion操作**。`batch`命令已经可以处理任意规模的大量操作。编写脚本来循环调用API是错误的做法——请使用`batch`命令并配合`--file`参数。

**适用情况**：
- 需要创建100多行数据时
- 需要在新创建的行之间建立关联时（请参考[批量操作策略](#bulk-operations-strategy)
- 操作规模过大，无法用单个命令完成时

如果您有这样的需求，请停止并使用`batch`命令。

## 快速入门

首次使用时，凭证会从Notion桌面应用中自动提取，无需手动设置。

> **重要提示**：所有在特定工作区内执行的命令都需要`--workspace-id`参数。可以使用`vibe-notion workspace list`来查找您的工作区ID。

## 认证

运行任何命令时，凭证（`token_v2`）会从Notion桌面应用中自动提取，无需使用API密钥、OAuth或手动提取凭证。

在macOS系统上，首次使用时系统可能会请求访问Keychain——这是正常的，用于解密cookie。

提取到的`token_v2`会存储在`~/.config/vibe-notion/credentials.json`文件中，文件权限设置为`0600`。

## 内存管理

代理会维护一个名为`~/.config/vibe-notion/MEMORY.md`的文件来存储跨会话的持久化数据。此文件由代理管理，CLI不会读取或写入该文件。您可以使用`Read`和`Write`工具来管理这个内存文件。

### 读取内存

**在每个任务开始时**，使用`Read`工具读取`~/.config/vibe-notion/MEMORY.md`文件，以加载之前发现的工作区ID、页面ID、数据库ID和用户偏好设置。
- 如果文件尚不存在，可以忽略它，并在有可用信息时再创建。
- 如果文件无法读取（权限问题或目录缺失），也可以忽略它，不会导致错误。

### 写入内存

在发现有用信息后，使用`Write`工具更新`~/.config/vibe-notion/MEMORY.md`文件。需要写入的情况包括：
- 发现工作区ID后
- 发现页面ID、数据库ID或集合ID后
- 用户提供了别名或偏好设置后
- 发现页面/数据库结构后（例如父子关系、哪些数据库属于哪些页面）

**需要存储的内容**：
- 带有名称的工作区ID
- 带有标题和父级信息的页面ID
- 带有标题和父级信息的数据库/集合ID
- 用户指定的别名（如“Tasks DB”、“Main workspace”）
- 常用的视图ID
- 父子关系（哪些数据库属于哪些页面）
- 用户在交互过程中表达的偏好设置

**禁止存储的内容**：
- 绝不要存储`token_v2`、凭证、API密钥或任何敏感数据
- 绝不要存储完整的页面内容（仅存储ID和标题）
- 除非是持久化引用（如数据库区块），否则不要存储区块级别的ID

### 处理过时数据

如果存储的ID导致错误（例如页面找不到或访问被拒绝），请将其从`MEMORY.md`文件中删除。不要盲目信任存储的数据——在数据异常时请重新搜索。

### 批量操作

可以通过一次CLI调用执行多个写入操作。这样可以在一次操作中创建、更新或删除多个对象，从而节省令牌并减少网络请求次数。

**支持的操作**（共14个）：
- `page.create`：创建页面
- `page.update`：更新页面标题、图标或内容
- `page.archive`：归档页面
- `block.append`：向父区块添加区块
- `block.update`：更新区块
- `block.delete`：删除区块
- `block.move`：移动区块到新位置
- `comment.create`：创建评论
- `database.create`：创建数据库
- `database.update`：更新数据库标题或架构
- `database.delete-property`：删除数据库属性
- `database.add-row`：向数据库添加行
- `database.update-row`：更新数据库行的属性
- `block.upload`：将文件作为图片或文件区块上传

### 批量操作策略

**多步骤处理**：当新创建的行之间需要建立关联时（例如，行A需要引用行B），可以分两步进行：
1. **第一步**：创建所有行（不包含关联信息），将所有`database.add-row`操作写入一个JSON文件。
2. **第二步**：使用第一步得到的ID，创建另一个JSON文件，其中包含设置关联信息的`database.update-row`操作。

这样就可以避免编写复杂的脚本。

### 请求速率限制

Notion对其API有速率限制。批量操作是顺序执行的，因此大量操作（30个以上）可能会触发“429 Too Many Requests”错误。为了避免这个问题：
- 将大量操作分成每批25-30个的操作
- 如果批量操作中途失败，请仅重新运行剩余的操作
- `batch`命令的输出会显示哪些操作成功了——使用`index`字段来确定从哪里继续执行

### 方案提示（`$hints`）

当数据库架构出现问题时，`database get`和`database query`命令可能会返回一个`$hints`数组。这些提示提供了可操作的修复方案，请按照提示进行修复。

**当 `$hints` 存在时**：仔细阅读每个提示，并执行建议的修复命令。损坏的属性可能会导致Notion应用程序崩溃。常见的问题包括：
- **已删除但仍在架构中的属性**：通常无害，但表明存在历史问题。
- **损坏的引用**：引用的集合已删除或丢失，可能会导致Notion应用程序崩溃。
- **损坏的关联**：目标集合缺失，也可能导致Notion应用程序崩溃。

如果 `$hints` 不存在，说明架构是正常的——无需采取任何操作。

**何时使用`--backlinks`**

### 分页

返回列表的命令支持分页功能，可以通过`has_more`和`next_cursor`字段实现：
- `block.children`：基于游标。将上一次响应中的`next_cursor`值作为`--start-cursor`参数传递。
- `search`：基于偏移量。将`next_cursor`值（一个数字）作为`--start-cursor`参数传递。
- `database query`：使用`--limit`参数来控制页面数量。`has_more`表示还有更多结果，但私有API不支持基于游标的分页——增加`--limit`参数可以获取更多页面。
