# OpenClaw Tally

该技能将AI的使用方式从简单的“统计令牌数量”转变为关注“任务完成的经济性”。它不再问“有多少令牌？”，而是会问“完成某项任务需要多少资源，这样做是否值得？”

## 🛠️ 安装

### 1. 使用OpenClaw命令安装（推荐）
告诉OpenClaw：“安装openclaw-tally技能。”代理会自动处理安装和配置过程。

### 2. 手动安装（通过CLI）
如果您更喜欢使用终端，请运行以下命令：
```bash
clawhub install openclaw-tally
```

## 安全与隐私声明

- **消息处理机制**：该技能会注册一个`message-post`钩子，并处理**所有收到的消息**。
- **仅限本地处理**：所有处理操作都在本地完成，不会向任何外部服务器发送数据。
- **消息内容处理**：任务检测器会通过正则表达式匹配来读取消息内容，以识别任务的开始/完成/失败信号。**消息内容本身不会被存储**，只有元数据（令牌数量、使用的模型、会话ID、复杂度评分）会被保存到数据库中。
- **存储方式**：默认使用SQLite数据库（路径为`~/.openclaw/tally/tally.db`），也可以指定自定义路径进行测试。
- **依赖库**：需要`better-sqlite3`（Node.js的原生插件）。安装过程中会通过`npm install`命令触发原生构建过程。
- **权限要求**：无需网络访问权限，也不需要执行权限。文件系统访问范围仅限于`~/.openclaw/tally/`目录。

## 功能介绍

- **自动从消息流中检测任务**（第一层：任务检测器）
- **跨会话、子代理及定时任务记录任务的成本信息**（第二层：任务记录系统）
- **计算每项任务、每个模型及每个定时任务的效率评分（Task Efficiency Score, TES）**（第三层：分析引擎）

## 命令接口

- `/tasks list` — 显示最近的任务列表，包括状态、成本和效率评分
- `/tasks stats` — 显示指定时间段的统计信息
- `/tasks this-week` — 显示本周的任务概览
- `/tasks show <task_id>` — 查看特定任务的详细信息
- `/tasks report --dimension model` — 提供模型的效率报告
- `/tasks cron-health` — 检查定时任务的执行效率和健康状况

## 任务复杂度分级

- **L1（基础级）**：单轮处理，仅使用文本，无需使用任何工具
- **L2（常规级）**：需要多轮处理或调用1–3个工具
- **L3（高级级）**：需要使用多个工具、进行文件读写操作以及调用外部API
- **L4（复杂级）**：涉及子代理、定时任务处理以及跨会话的操作

## 任务效率评分（TES）

```
TES = quality_score / (normalized_cost × complexity_weight)
```

- **> 2.0** 🟢 优秀
- **1.0–2.0** 🟡 良好
- **0.5–1.0** 🟠 低于平均水平
- **< 0.5** 🔴 较差
- **0.0** ⚫ 失败

## 使用方法

安装该技能后，它会自动连接到`message-post`事件处理机制。您可以使用上述`/tasks`命令来查询和分析任务数据。所有数据都会存储在`~/.openclaw/tally/tally.db`文件中。
