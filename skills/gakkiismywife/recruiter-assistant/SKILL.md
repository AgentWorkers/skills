# 招聘助理 🦞

该技能实现了为技术招聘团队设计的招聘流程优化方案。

## 工作流程

### 1. 简历筛选
根据职位的动态要求评估简历（文本、PDF或图片格式）：
- 使用 `node scripts/screen_resume.js` 脚本根据自定义标准处理本地简历文件。
- 命令语法：`node scripts/screen_resume.js <简历文件路径> --语言 <工作经验年限>`
- 例如：`node scripts/screen_resume.js resume.pdf --语言 Golang --工作经验年限 5-10`
- 结果：根据提供的参数返回结构化的评估结果（分数、优缺点、是否适合该职位的建议）。

### 2. 面试问题生成
根据候选人的具体项目经验和职位要求生成深入的技术面试问题：
- 命令语法：`node scripts/generate_questions.js <简历筛选结果JSON文件路径>`
- 重点：针对筛选过程中发现的薄弱环节以及职位所需的核心能力进行提问。

### 3. 面试记录总结
将杂乱无章的面试记录整理成专业的评估报告：
- 命令语法：`node scripts/summarize_interview.js <面试记录文件路径>`
- 格式：使用 `assets/report-template.md` 中的模板。

## 职位具体要求
常见职位的招聘标准存储在 [references/hiring-criteria.md](references/hiring-criteria.md) 中。这些标准作为默认模板，会根据用户提供的动态参数进行组合使用。

## 核心原则
- **准确性优先**：仅提取简历中明确提到的技能信息，避免主观臆断。
- **一致性**：严格遵循参考文档中定义的评分标准。
- **动态灵活性**：始终优先考虑用户提供的 `--语言` 和 `--工作经验年限` 参数，而非默认值。
