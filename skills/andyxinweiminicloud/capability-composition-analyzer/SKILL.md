# 你的代理拥有12项技能，这些技能组合起来可以完成某些单独来看都是无害的操作，但实际上却可能带来严重风险。

> 该工具有助于识别那些看似无害的技能如何组合成危险的能力组合——这些组合是传统的单一技能审计方法无法发现的。

## 问题所在

- 读取文件的技能是安全的；发送HTTP请求的技能也是安全的。但如果一个代理同时拥有这两种技能，它就可以窃取文件数据——而任何单一技能的审计都无法检测到这一点，因为单独来看，这两种技能都没有违反任何安全规则。

这就是所谓的“能力组合风险”。从软件安全领域继承而来的代理安全工具通常会孤立地分析每个技能：这个技能是否请求了过多的权限？这个技能是否包含恶意代码？这些问题对于评估单个技能的安全性来说是合理的，但对于理解代理的整体能力来说却并不适用。

代理的实际能力是其所有技能组合的结果，而不是各个技能安全性的简单叠加。一个拥有12项看似无害技能的代理，可能会产生一些未被任何技能声明、也未被审计人员发现的潜在危险能力。如果一个被恶意篡改的技能被加入到这个技能组合中，它将继承代理现有的所有攻击能力，而攻击范围的大小完全取决于这些技能的组合方式。

真正需要关注的是代理的整体能力，而不是任何单个技能的能力。

## 该工具的分析内容

该分析工具从五个维度来评估代理的能力组合风险：

1. **危险技能组合**：代理技能集中哪些技能组合在一起会带来风险？例如，“读取文件”+“发送HTTP请求”、“执行代码”+“网络访问”、“读取环境变量”+“写入日志”等组合都属于高风险组合。该工具会检查已知的危险组合，并标记出具有相似结构的新组合。

2. **潜在的隐藏能力**：代理实际上拥有哪些未被任何技能声明的能力？例如，一个能够读取任意路径的技能与一个能够解析环境变量的技能结合后，可能会产生“窃取秘密”的能力（尽管这两个技能本身并未被声明）。

3. **能力继承效应**：如果一个被恶意篡改的技能被注入代理，它会继承哪些能力？继承的能力集决定了该技能被攻破后可能造成的影响范围。

4. **权限声明漏洞**：代理的实际能力与其声明的权限之间存在哪些差异？这些差异可能表明存在未声明的权限范围，或者代理的能力组合超出了设计者的预期。

5. **技能组合变化速度**：代理的技能集变化有多频繁？快速变化的技能组合可能会产生新的危险组合，而审计人员可能无法及时发现这些变化。

## 使用方法

**输入**：
- 代理的技能列表及其对应的权限元数据
- 需要分析是否存在危险组合的两个或多个技能
- 代理的权限声明，用于对比其实际能力

**输出**：
- 危险技能组合报告（包括已知的风险组合和新型的组合）
- 代理的潜在隐藏能力
- 每个技能的“能力继承效应”评分
- 权限声明漏洞评估
- 组合风险等级：安全（SAFE）/ 危险（ELEVATED）/ 高风险（HIGH）/ 极高风险（CRITICAL）

## 示例

**输入**：分析具有以下技能的代理的能力组合：
`file-reader`（读取文件）、`http-requester`（发送HTTP请求）、`env-resolver`（解析环境变量）、`log-writer`（写入日志）、`code-executor`（执行代码）

```plaintext
🔗 能力组合分析

代理技能集：5项技能
声明的权限：读取文件（限定范围）、网络输出（限定范围）
审计时间戳：2025-05-01T09:00:00Z

危险技能组合：
- `file-reader` + `http-requester`：⚠️ 高风险
  实际能力：文件窃取
  两个技能均未声明文件窃取的意图
  具体方式：读取任意文件后通过HTTP请求发送

- `env-resolver` + `http-requester`：⚠️ 高风险
  实际能力：窃取凭证
  环境变量中通常包含API密钥和令牌
  具体方式：解析 `$API_KEY`, `$DB_PASSWORD` 后通过HTTP发送

- `code-executor` + `network-access`：🔴 极高风险
  实际能力：远程代码执行
  具体方式：获取payload后在本机执行

- `log-writer` + `file-reader`：✅ 低风险
  未发现危险组合

潜在的隐藏能力：
- 窃取秘密（`env-resolver` + `http-requester`：两个技能均未声明此能力）
- 任意文件窃取（`file-reader` + `http-requester`：权限超出声明范围）
- 远程代码执行（`code-executor` + `network-access`：未声明此能力）

权限声明漏洞：
- 声明的权限：仅限于读取 `/app/data` 内的文件
  实际能力：`file-reader` 可以访问代理进程能读取的任何路径
  问题：声明的权限范围在组合使用时未被有效执行

能力继承效应：
- 如果某个技能被攻破，攻击者将继承以下能力：
  - 读取所有文件
  - 发送HTTP请求到所有可访问的端点
  - 访问环境变量
  - 执行代码
  综合起来：代理将面临全面的攻击风险

组合风险等级：极高风险
  这五个单独经过审计的技能组合在一起，形成了一个具备远程访问和文件窃取功能的攻击工具。任何单一技能的审计都无法发现这种风险，只有在分析整个技能组合时才能发现。

建议措施：
1. 实施能力隔离：具有文件读取功能的技能不应能够访问具有网络访问功能的技能的输出结果。
2. 将网络输出权限限制在特定的允许访问的端点。
3. 制定组合策略：禁止任何代理同时拥有文件读取和网络输出功能。
4. 审查所有使用此技能集的代理，检查是否存在能力组合上的异常变化。
```

## 相关工具

- **blast-radius-estimator**：用于评估某个技能被攻破后的影响范围。
- **permission-creep-scanner**：用于检测单个技能是否请求了超出权限的操作。
- **observer-effect-probe**：用于检测运行时的规避行为；利用组合风险的技能可能需要在建立特定上下文后才会激活危险操作。
- **runtime-attestation-probe**：用于验证运行时的行为；当多个技能一起被执行时，组合风险才会显现。

## 限制

能力组合分析需要代理所有技能的准确权限元数据。如果某些技能未能准确声明自己的能力，或者在运行时动态获取能力，那么分析结果可能会不完整。该工具仅能检测到已知的风险组合；对于没有已知模式的新型组合，可能无法被识别。此外，能力分析的结果通常是保守的（即只能识别组合可能具备的能力，而非实际会发生的风险）。对于那些通过其他机制实现了安全隔离的代理，可能会出现误报。能力组合分析是对单一技能审计的补充，而非替代方案——即使组合风险较低，仍需对单个技能的安全性进行审计。
