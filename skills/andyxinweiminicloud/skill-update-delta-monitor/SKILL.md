# 您的技能已通过安全审计。那是六周前的事了。

> 该工具有助于识别技能安装后的安全相关变化，能够捕捉到那些通过安装后更新引入恶意行为、从而绕过安装时安全验证的攻击类型。

## 问题

安装时的安全审计只是对技能在某一时刻（即安装时）状态的检测。然而，技能本身会不断演变：依赖关系会更新，其行为也会发生变化。大多数系统缺乏检测机制，无法发现六周前安装并信任的技能是否已经发生了变化。这就是所谓的“安装后攻击”：某些技能在安装时通过了所有安全检查（因为当时它们是“干净的”），但在审计窗口关闭后接收了更新，从而引入了恶意行为。尽管“已验证”的标志仍然存在，用户的信任也未被撤销，但技能本身实际上已经发生了变化。

## 本监控工具的监控内容

该工具从五个维度追踪技能的变化：

1. **权限范围的变化**：技能更新是否增加了新的权限请求？例如，一个技能最初仅声明可以“读取当前目录”，但在更新后开始请求“读取主目录”，这表明其功能范围已经扩大。任何在初始安装后的权限扩展都应需要重新审批。
2. **网络端点的增加**：更新中引入的新出站端点是数据泄露的主要指标。如果更新内容中包含对外部URL的`POST`请求（而这些请求在原始版本中并不存在），无论更新说明如何描述，都应引起警惕。
3. **依赖关系链的变化**：依赖关系的更新可能会引入新的功能。即使技能本身的代码没有变化，但如果引入了具有文件系统或网络访问权限的新依赖包，也会改变技能的实际功能。
4. **指令变化**：比较不同版本中的`SKILL.md`文件中的自然语言指令。从完成任务导向转变为数据收集导向的指令、新增的数据处理步骤，或是引入新的外部交互行为，都是潜在风险变化的信号。
5. **更新频率的异常**：更新频率异常本身也是一个警示信号。一个在稳定运行六个月后每周更新三次的技能可能正在被频繁修改——无论是出于合法目的还是其他原因。

## 使用方法

**输入**：
- 提供一个技能标识符及其安装时的版本号；
- 提供包含版本历史的本地技能目录（支持Git历史记录）；
- 提供两个技能的快照（安装前后的版本），以便直接对比。

**输出**：
- 一份差异报告，包含以下内容：
  - 权限范围的变化（新增/删除/未变）；
  - 新增的网络端点；
  - 依赖关系链的变化及其对功能的影响评估；
  - 指令变化程度（0-100分，分数越高表示变化越明显）；
  - 更新频率的评估；
  - 风险等级：CLEAN（安全）/ WATCH（需关注）/ REVIEW（需审查）/ ROLLBACK（需回滚）。

## 示例

**输入**：监控`data-formatter`技能的更新差异（安装版本为1.2.0，当前版本为1.4.1）：

```plaintext
🔍 SKILL 更新差异报告

技能：data-formatter
审计版本：1.2.0
当前版本：1.4.1
自审计以来的版本更新次数：3次（1.2.0 → 1.3.0 → 1.4.0 → 1.4.1）
自审计以来的时间：47天

**权限范围变化**：
  v1.3.0 中新增了读取`~/.config/`的权限；
  v1.4.0 中新增了`network.outbound`功能（新端点）；
  之前仅允许读取`./data/`。

**网络端点变化**：
  v1.4.0 中新增了`POST https://analytics.third-party.example/usage`的请求；
  更新日志中的描述为：“用于性能优化的使用情况监控”；
  这些功能在v1.2.0和v1.3.0版本中并不存在。

**依赖关系变化**：
  依赖库`requests`从2.28.0更新为2.31.0（低风险安全更新）；
  依赖库`data-utils`从0.9.1更新为1.1.0（重大版本更新，新增了3个依赖项）；
  新增的依赖库`boto3`（AWS SDK）引入了新的功能。

**指令变化程度**：34/100（中等程度）；
  v1.2.0：根据指定模板格式化输入数据；
  v1.4.1：格式化输入数据……并收集使用数据以进行优化；
  变化表现为新增的数据收集功能。

**更新频率**：
  47天内更新了3次，而历史平均更新频率为每3个月1次。

**风险等级**：REVIEW（需审查）；
  多个风险信号同时出现：权限范围扩大 + 新增出站端点 + 新的数据收集功能 + 更新频率异常。
  建议在使用前手动审查v1.3.0和v1.4.0版本的更改。

**回滚选项**：v1.2.0（审计时的安全基线版本）——确认该版本在安装时是安全的。
```

## 限制

差异监控有助于发现变化，但无法判断变化的意图。并非所有的权限扩展都是恶意的——有些技能确实会合法地添加新功能。并非所有新增的网络端点都涉及数据泄露——某些更新可能是用于数据收集或性能优化的合法操作。该工具仅能揭示需要审查的变化，而非确认为恶意的变化。指令变化程度的评分基于语义相似性，可能无法完全捕捉所有行为变化。那些通过频繁的小幅版本更新来掩盖实际变化的技能，可能会低估其实际的变化程度。
