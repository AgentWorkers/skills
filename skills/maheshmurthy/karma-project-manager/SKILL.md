## 设置

如果`KARMA_API_KEY`已经设置完毕，请跳转到[验证](#verify)步骤。

否则，请使用`AskUserQuestion`工具，并选择以下选项：
- 问题：“您需要Karma API密钥才能继续。您希望如何设置它？”
- 选项：“快速开始 — 立即生成（无需账户）”，“通过电子邮件登录 — 链接到现有的Karma账户”，“我已经有密钥”

### 快速开始（无需账户）

执行该操作后，系统将返回：`{"apiKey": "karma_..."}` — 该信息仅显示一次。

**注意**：务必传递`-d '{}'`参数，否则会导致400错误。

### 通过电子邮件登录

1. 请求用户的电子邮件地址。
2. 发送`POST ${BASE_URL}/v2/api-keys/auth/init`请求，请求参数中包含`{"email": "..."}`。
3. 系统会发送验证码，发送`POST ${BASE_URL}/v2/api-keys/auth/verify`请求，请求参数中包含`{"email": "...", "code": "...", "name": "claude-agent"}`。
4. 系统返回`{"key": "karma_..."}`。

| 错误代码 | 应对措施 |
|---------|---------|
| 400 | 验证码无效或已过期 | 请用户重新请求验证码或获取新验证码 |
| 409 | 有效的API密钥已存在 | 使用现有的密钥或从网站撤销该密钥 |

### 默认区块链

当用户未指定区块链时，系统将默认使用**Base (8453)**。系统会询问用户是否确认：
> 您的项目将创建在**Base**区块链上。是否继续？
>
> - **是**
> - **选择其他区块链**：Arbitrum、Base、Celo、Lisk、Optimism、Polygon、Scroll、Sei

### 链的继承规则

子操作必须使用与其父操作相同的区块链：
- **申请资助** → 使用`project.chainId`
- **申请资助更新** → 使用`grant.chainId`
- **设置里程碑** → 使用`grant.chainId`
- **完成里程碑** → 使用`milestone.chainId`
- **更新项目** → 使用`project.chainId`

请通过API查询父操作的区块链信息，切勿在子操作中要求用户再次指定区块链。

## 操作说明

### 创建项目

| 参数 | 是否必填 | 说明 |
|-------|---------|-------------|
| `chainId` | 是 | 区块链ID（默认：Base 8453） |
| `title` | 是 | 项目名称（1-200个字符） |
| `description` | 是 | 项目描述（1-5000个字符） |

**可选参数：**

| 参数 | 说明 | ------------- |
| `imageURL` | 标志图/图片URL |
| `links` | 类型为`{ type, url }`的数组 — 包括GitHub、网站、Twitter、Discord链接 |
| `tags` | 最多20个字符串的数组 — 例如："defi"、"infrastructure" |
| `problem` | 该项目要解决什么问题？（1-5000个字符） |
| `solution` | 项目的解决方案是什么？（1-5000个字符） |
| `missionSummary` | 简要的项目使命声明（1-1000个字符） |
| `locationOfImpact` | 项目的影响范围或领域（1-1000个字符） |
| `businessModel` | 项目如何维持运营？（1-1000个字符） |
| `stageIn` | 项目开发阶段：创意阶段、MVP阶段、测试阶段、生产阶段、成长阶段、成熟阶段（1-1000个字符） |
| `raisedMoney` | 目前筹集的资金金额（1-1000个字符） |
| `pathToTake` | 项目的未来路线图（1-1000个字符） |

#### 收集项目信息

当用户创建项目时，一次性展示所有字段，并允许用户填写所需的信息：
> 要创建项目，请提供以下信息。**只有**项目名称和描述是必填的，但其他信息有助于提升项目的竞争力：
>
> - **项目名称**：项目的名称 |
| - **项目描述**：该项目解决什么问题？ |
| - **解决方案**：项目如何解决问题？ |
| - **使命声明**：用一句话概括项目的目标 |
| - **开发阶段**：创意阶段、MVP阶段、测试阶段、生产阶段、成长阶段、成熟阶段 |
| - **影响范围**：项目影响的领域或对象 |
| - **运营模式**：项目如何维持运营？ |
| - **筹集资金**：目前筹集了多少资金？ |
| - **路线图**：项目的未来计划 |
| - **链接**：GitHub、网站、Twitter、Discord的链接 |
| - **标签**：项目所属的类别（例如：defi、基础设施） |
| - **图片**：项目的标志图或宣传图片的URL |

仅展示用户提供的字段，其他元数据字段均为可选。

#### 项目创建完成后

项目创建成功后，显示以下信息：
> 您的项目已在{chainName}区块链上创建！
>
> - **项目名称**：{title}
| - **区块链**：{chainName} ({chainId})
| - **交易哈希**：{transactionHash}
>
> 您想发布第一条更新信息吗？可以分享您刚刚完成的内容、达成的里程碑或接下来的计划。

### 更新项目详情

更新现有项目的内容。**更新操作会替换所有字段**，因此请先获取当前项目的详细信息。

| 参数 | 是否必填 | 说明 |
|-------|---------|-------------|
| `chainId` | 是 | 项目的区块链ID |
| `projectUID` | 是 | 项目的唯一标识符 |
| `title` | 是 | 项目名称（1-200个字符） |
| `description` | 是 | 项目描述（1-5000个字符 |

此外，还可以更新`createProject`中的所有可选字段（imageURL、links、tags、problem、solution、missionSummary、locationOfImpact、businessModel、stageIn、raisedMoney、pathToTake）。

**注意**：在更新时，务必同时显示现有字段和更改的内容。

### 设置里程碑

为资助申请添加里程碑。

| 参数 | 是否必填 | 说明 |
|-------|---------|-------------|
| `chainId` | 是 | 资助申请的区块链ID |
| `grantUID` | 是 | 资助申请的唯一标识符 |
| `title` | 是 | 里程碑的名称（1-200个字符 |
| `description` | 里程碑的具体内容（1-5000个字符 |
| `endsAt` | 是 | 里程碑的截止时间（以Unix时间戳表示，单位为秒） |
| `priority` | 否 | 优先级（0-4） |

日期转换方法：`Math.floor(new Date("2025-06-30").getTime() / 1000)`

### 一次性创建项目和资助申请

通过一次操作同时创建项目和资助申请（需要生成4条证明记录）。

除了`createProject`中的参数外，还需提供：
| `communityUID` | 是 | 社区的唯一标识符 |
| `grant.title` | 是 | 资助申请的名称（1-200个字符 |
| `grant.description` | 否 | 资助申请的描述 |
| `grant.amount` | 否 | 资助金额 |
| `grant.proposalURL` | 否 | 资助计划的链接 |
| `grant.programId` | 否 | 资助计划的唯一标识符 |

成功创建后，显示与`createProject`相同的成功提示信息。

## 错误处理

| 错误代码 | 错误原因 | 应对措施 |
|---------|---------|---------|
| 400 | 参数错误 | 显示错误信息，并指导用户修正参数 |
| 401 | API密钥无效 | 告知用户检查`KARMA_API_KEY`或重新执行设置流程 |
| 429 | 每分钟请求次数限制（60次） | 等待片刻后重试 |
| 500 | 服务器错误 | 重试一次，如仍无法连接则报告错误 |

## 特殊情况

| 情况 | 处理方式 |
|---------|---------|
| 缺少必填参数 | 询问用户缺失的参数 |
| 未指定区块链（主操作） | 默认使用Base区块链，并询问用户确认 |
| 未指定区块链（子操作） | 继承父操作的区块链设置 |
| API密钥未设置 | 执行设置流程 |
| 项目名称过长（超过200个字符） | 截断名称并确认 |
| 需要项目唯一标识符但用户只提供了名称 | 通过API查询项目的唯一标识符 |
| 部分项目更新 | 先获取当前项目详情，合并更改后再进行更新 |
| 项目中存在多个资助申请 | 显示所有资助申请的列表，并询问用户需要更新哪个 |
| 提供的日期为字符串形式 | 将日期转换为Unix时间戳（单位为秒） |
