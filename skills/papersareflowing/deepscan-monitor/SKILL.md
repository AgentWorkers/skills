# DeepScan 监控器

当用户希望 Claude 管理一个持续较长时间的 PapersFlow 研究工作流程（而不仅仅是单次搜索请求）时，请使用此功能。

## 工作流程

1. 使用 `run_deepscan` 启动任务。
2. 立即告知用户该任务是异步执行的。
3. 使用 `get_deepscan_live_snapshot` 获取以下信息的实时更新：
   - 进度
   - 状态信息
   - 检查点状态
   - 最重要的论文
   - 部分总结
   - 关键发现
4. 如果用户仅需要简单的进度检查，可以使用 `get_deepscan_status`。
5. 当 `finalReportAvailable` 为 `true` 或任务完成后，调用 `get_deepscan_report`。
6. 当用户需要从已存储的 DeepScan 历史数据中获取汇总报告时，使用 `summarize_evidence`。
7. 只有在报告数据稳定且适合绘制图表时，才使用 `run_python_plot`。

## 重要行为规范

- 不要暗示 MCP 服务器会自动向 Claude 发送任务完成通知。
- 有意识地定期检查任务进度，并向用户说明正在进行的检查过程。
- 当用户需要更详细的实时信息时，优先使用 `get_deepscan_live_snapshot` 而不是 `get_deepscan_status`。
- 如果报告尚未准备好，要明确告知用户，并确保下一步操作清晰明了。

## 进度更新方式

当任务仍在进行中时，更新内容包括：
- 当前状态
- 进度百分比
- 当前阶段或状态信息
- 任何相关的检查点问题
- 重要的实时论文
- 如果有的话，还包括关键发现

除非用户要求提供更多细节，否则更新内容应保持简洁。

## 绘图指南

只有在报告数据稳定且适合绘制图表时，才使用 `run_python_plot`，例如：
- 按年份划分的论文分布
- 引用分布
- 不同会议或期刊上的论文分布
- 对于已完成的任务，进行分组对比

对于数据稀疏或质量明显较低的情况，不要生成图表，同时要向用户说明这一点。

## 示例

- 用户请求：“对代理检索系统的评估基准进行 DeepScan 分析，并随时向我报告进度。”
- 用户请求：“查看我的 DeepScan 进度，并告诉我目前的关键发现。”
- 用户请求：“任务已完成，请汇总最终报告，并按年份绘制论文分布图。”
- 用户请求：“总结我最近关于蛋白质结构预测的 DeepScan 报告中的关键发现。”
