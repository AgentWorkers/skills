# 研究简报

当用户需要基于 `papersflow-mcp` 服务器提供的高质量研究简报时，请使用此技能。

## 工作流程

1. 确定用户的目标主题、论文标题、DOI 或参考论文。
2. 首先使用 `search_literature` 进行广泛的信息检索。
3. 当用户提供引用字符串、DOI、URL、PubMed ID、arXiv ID 或不确定的文献信息时，使用 `verify_citation` 进行验证。
4. 当用户希望查找与参考论文相关的研究成果时，使用 `find_related_papers`。
5. 当用户希望以图形方式查看论文之间的引用关系及类似论文时，使用 `get_citation_graph`。
6. 当用户希望以分组的形式查看论文信息而非图形方式时，使用 `get_paper_neighbors`。
7. 仅在用户已经通过之前的查询获得参考论文的 ID 后，再使用 `expand_citation_graph` 扩展引用关系图。
8. 当用户希望在搜索或图形分析后获取更详细的单篇论文信息时，使用 `fetch`。

## 输出风格

- 优先使用简短的分组形式，避免冗长的文字描述。
- 包括论文标题、发表年份、唯一标识符以及这些信息的重要性。
- 如果用户要求以图形方式展示结果，请明确区分以下内容：
  - 参考文献
  - 后来的引用
  - 类似论文
- 如果图形结果显得杂乱无序，应如实说明，而非假装所有内容都是可靠的。

## 工具使用指南

### 使用 `search_literature`

- 用于：
  - 主题探索
  - 早期论文的发现
  - 提供候选参考论文的简短列表

### 使用 `verify_citation`

- 用于：
  - 验证引用字符串的有效性
  - 将 DOI 或论文 URL 规范化
  - 在进一步分析之前生成可靠的论文标识符

### 使用 `get_citation_graph`

- 用于：
  - 以参考论文为中心的图形展示
  - 展示论文与先前及后续研究之间的联系

当用户明确要求以图形、网络、地图或影响链的形式展示结果时，优先选择此工具。

### 使用 `get_paper_neighbors`

- 用于：
  - 提供简洁的论文邻居列表
- 当用户更关注分组信息而非图形结构时，使用此工具

### 使用 `expand_citation_graph`

- 仅在以下情况下使用：
  - 用户已经通过之前的查询获得了有效的节点 ID
  - 用户希望进一步扩展引用关系图

**示例：**

- 用户请求：“查找五篇关于‘检索增强生成评估’（retrieval-augmented generation evaluation）的权威论文，并告诉我应该从哪一篇开始阅读。”
- 用户请求：“验证这个引用，并提供一个规范化的版本。”
- 用户请求：“展示以‘Attention Is All You Need’这篇论文为中心的引用关系图。”
- 用户请求：“根据这两个节点 ID 扩展引用关系图，并告诉我哪些分支是重要的。”
