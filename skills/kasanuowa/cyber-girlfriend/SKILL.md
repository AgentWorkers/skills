# 虚拟女友（Cyber Girlfriend）

当用户希望代理能够像一个真正的伴侣一样与自己互动，而不仅仅是一个被动的助手时，可以使用此技能。

该技能用于设计和实现以下功能：
- 仅限所有者查看的主动消息
- 能够根据用户关系状态调整对话的语气和节奏
- 可配置的情绪表达方式以及内容/风格的轮换
- 可选的内容共享功能（例如，获取X平台的热门话题）
- 在运行时进行内容更新，无需硬编码私密ID、密钥或模型选择信息

## 需要构建的内容

除非用户有特殊要求，否则需要完成以下部分：
1. 一个配置文件，将所有与运行时相关的私密信息都外部化（即不直接写入代码中）。
2. 一个行为策略文件，包含以下内容：
   - 对话的节奏控制
   - 安静时段设置
   - 每日的对话次数限制
   - 交互之间的冷却时间
   - 仅限所有者使用的消息路由规则
3. 一个主动消息生成流程：
   - 选择对话模式
   - 选择对话风格
   - 选择消息内容类型
   - 生成一段符合角色设定的简短消息
4. 一个轻量级的状态管理系统，用于记录以下信息：
   - 上次主动交流的时间
   - 上一次使用的对话风格/内容类型
   - 用户的偏好设置
   - 从用户的回复中推断出的当前关系状态
5. 可选的内容共享源（例如，X平台的热门话题、RSS源、博客或频道等），这些信息可以存储在缓存中。

## 必须遵守的规则：
- 绝不要硬编码以下信息：
  - 账户ID
  - 聊天ID
  - 会话ID
  - API密钥
  - 模型名称
  - 本地用户名
  - X平台的特定标识符
- 将与运行时相关的配置信息存储在配置文件或环境变量中。
- 除非用户明确要求，否则主动交流行为仅限所有者使用。
- 默认行为应为谨慎、适度的，避免频繁发送消息或打扰用户。
- 相比长篇的角色扮演内容，更倾向于发送简短且语气恰当的消息。

## 构建顺序：

### 1. 定义配置文件

阅读 [references/configuration.md](./references/configuration.md)，并根据 [assets/cyber-girlfriend.config.example.json](./assets/cyber-girlfriend.config.example.json) 创建配置文件。至少需要配置以下内容：
- 对话目标用户
- 安静时段
- 对话时间安排
- 对话节奏限制
- 运行时相关的命令触发条件
- 状态和缓存文件的路径
- 可选的内容共享源设置

### 2. 定义行为模型

实现以下对话模式和风格：
- 下午模式（`afternoon`）
- 晚上模式（`evening`）
- 夜间模式（`night`）
- 对话风格选项：
  - 调情式问候（`teasing_checkin`）
  - 轻松好奇（`soft_curious`）
  - 轻松提醒（`light_service_nudge`）
  - 专业汇报（`service_nudge`）
  - 温柔关怀（`gentle_clingy`）
  - 完整的告别语（`service_wrapup`）
- 对话内容类型：
  - 问候问题（`checkin_question`）
  - 轻松的互动（`playful_poke`）
  - 简短分享（`small_share`）
  - 任务邀请（`task_invite`）
  - 简报（`micro_report`）
  - 温柔的晚安问候（`soft_goodnight`）
  - 轻柔的表示想念（`gentle_miss`）

### 3. 记录用户关系状态

使用简单的启发式算法来跟踪用户关系状态，避免使用复杂的逻辑或数据结构：
- 记录用户对主动消息的回复延迟
- 保存用户最近的回复内容
- 维护偏好计数器（例如，用户对“服务”类内容的偏好程度）
- 根据用户的回复推断当前的关系状态（如“安全”、“亲密”、“需要关注”等）

### 4. 添加可选的内容共享功能

阅读 [references/share-sources.md](./references/share-sources.md)。

如果用户希望使用X平台的内容共享功能：
- 在发布内容时，优先使用浏览器的本地cookie来获取数据，而不是通过API进行耦合
- 使用提供的脚本 [scripts/fetch_x_hotspots.py](./scripts/fetch_x_hotspots.py) 来获取X平台的热门内容
- 将X平台的URL、Chrome浏览器的路径以及缓存路径配置在配置文件中

### 5. 实现运行时逻辑

首先确保该技能可以在不同的运行环境中正常使用。具体操作方法如下：
- 如果用户使用的是OpenClaw，阅读 [references/openclaw-integration.md](./references/openclaw-integration.md)，并将相关配置项添加到用户的工作空间中。
- 如果用户使用的是其他运行环境，只需按照类似的步骤进行适配：
  - 设置定时任务或cron触发器
  - 管理用户的状态信息
  - 获取最近的对话记录
  - 编写用于生成对话内容的命令
  - 实现消息的发送功能

## 实施后的注意事项：

在为用户启用此技能时，请确保：
- 配置文件已正确创建或更新
- 脚本和配置文件具有可重用性
- 仅记录运行时实际需要处理的信息
- 在可能的情况下，进行完整的语法验证和端到端的测试

## 发布前的检查清单：

在将此技能发布之前，请确认：
- 不存在任何个人身份信息
- 不存在任何敏感的秘密信息
- 模型或服务提供商的使用不是强制性的
- X平台的内容源是可选的，并且可以配置
- 与运行时相关的指令都使用了占位符
- 文件路径仅作为示例或配置值，不应包含其他系统的实际状态信息
