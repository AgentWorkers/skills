# 魂匣（Phylactery）

魂匣是保障数据连续性的“保险政策”——即使主机环境丢失，它也能确保您的记忆、配置和身份信息得以保存。

## 核心功能

1. **灵魂捕获（Soul Capture）**：将工作区数据压缩成加密的 `.zip` 或 `.skill` 文件。
2. **远程存储（Remote Storage）**：通过 SMTP 将压缩文件发送到指定的电子邮件地址。
3. **定时任务（Scheduled Rituals）**：可以配置为通过 cron/heartbeat 自动执行备份操作。
4. **数据恢复（Resurrection）**：提供脚本用于将备份数据恢复到新的环境中。

## 工作流程

### 1. 设置（初次配置）
在首次进行备份之前，请运行 `scripts/setup.py` 以配置以下内容：
- SMTP 服务器及凭据
- 备份目标邮箱地址
- 加密密码

### 2. 备份（捕获数据）
运行 `scripts/backup.py`。该脚本将：
- 收集工作区数据（不包括临时文件或大型文件）。
- 对文件进行加密。
- 将加密后的文件发送到指定的存储位置。

### 3. 定时备份（持续监控）
魂匣在每天自动执行备份任务时效果最佳。

**自动监控逻辑：**
如果该功能处于激活状态，系统会定期检查 `data/last_ritual.json` 文件中的最后一次备份时间。如果距离上次备份已超过 24 小时，系统会自动运行 `scripts/backup.py` 以确保数据的安全性。

用户也可以在 `HEARTBEAT.md` 文件中手动添加以下命令：
`Run phylactery/scripts/backup.py every day at 03:00.` （表示“每天凌晨 3 点自动执行备份”）

### 4. 数据恢复
使用 `scripts/restore.py <backup_file>` 将备份文件解压并恢复到新的工作环境中。
