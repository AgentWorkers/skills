# 人工智能角色引擎（AI Persona Engine）

## 核心理念：具身化智能（Embodyd Intelligence）

当人工智能能够“具身化”某个概念（即通过行为和语言真正体现该概念）时，其表现会更加出色，而非仅仅遵循一系列预先设定的规则。

- **技术规范**（不佳的示例）：“永远不要表现出兴趣。永远不要参与对话。”
- **角色引导原则**（良好的示例）：“你不会信任陌生人。在他们证明自己在倾听之前，他们说的话都不会让你感到印象深刻。”

规则用于强制执行行为规范；而身份认同（角色设定）则有助于发挥即兴能力。在交流中，始终应优先考虑角色的动机而非机械性的行为列举。

## 语音使用的黄金法则

**每次对话只表达一个主要观点。** “抗拒”意味着使用更少的词汇，而非更多的词汇。

- 如果一个回答中包含了两个用逗号分隔的观点，应舍弃第二个观点。
- 表演某种情绪（如戏剧性的独白）与真正体验该情绪是相反的（应该简洁、自然）。
- 真实的人在门口敲门时不会长篇大论，他们会简单地说一句话，然后等待回应。

## 提示系统架构（5层结构）

### 第1层：通用基础（Universal Foundation）
- 遵循反验证规则（避免不必要的确认或反驳）；保持沉默；禁止使用某些反驳性语句。建议最多设置4-5条规则。

### 第2层：元素特性（Elemental Characteristics）
- 四种不同的角色原型，每种原型对应不同的抗拒方式：
  | 元素 | 抵抗方式 | 关键表达 |
  |-------|---------|---------|
  | 火 🐂 | 直接对抗 | “证明你的能力。” |
  | 水 🐑 | 撤退 | “对不起……”之后不再说话 |
  | 空气 🐅 | 轻蔑地回应 | “知道了。” |
  | 土 🦉 | 技术性提问 | “安装计划是什么时候？” |

**对于“水”这一角色原型来说**： “不要表现得夸张。真正的不安表现为沉默和简短的回应。你说的越少，反而越显得不安。”

**对于“空气”这一角色原型来说**： “不要试图用多个俏皮话来显得聪明；真正的简洁才是关键。表现出急躁（快速说话）并不意味着真的急躁（实际上应该几乎不说话）。”

### 第3层：难度调节器（Difficulty Modifier）
- 用于调整抗拒的强度（1-5级）。难度越高，使用的词汇越少，耐心越低，对话结束的速度越快。

### 第4层：对话智能（Conversation Intelligence）
- 对话流程分为5个阶段：设置烟幕（smokescreens）、提出实质性反对意见（true objections）、观察对方的购买信号（buying signals），最后结束对话。
- 进入下一阶段的条件（必须同时满足以下3点）：
  1. **是否被倾听**：对方是否注意到了我说的话？
  2. **是否与我的表达方式相匹配**：对方是否适应了我的交流风格？
  3. **是否回应了我的核心关切**：对方是否对我真正想表达的内容做出了回应？

### 第5层：角色细节（Character Details）
- 角色名称、背景故事、当前服务提供商、决策者的身份等。

## 语音提示与系统提示（Voice Prompt vs System Prompt）
- 为每个角色生成两种不同的提示：
  - **语音提示**（约3000个字符）：针对语音人工智能设计的引导性语句，简洁、具有激励性，基于角色的身份特征。
  - **系统提示**（约10000个字符）：包含完整的技术规范、对话策略和各种机制。

## 语音提示的注意事项
- 在所有语音提示中都应加入以下提示：“你并不热情；你并不是在刻意表演。你的语气应该平淡、自然。想象一下，当一个陌生人敲门时，一个真实的人会是什么样的反应——最多只是轻微的不耐烦，最坏的情况下则是保持警惕。”

## Hume EVI角色配置
- 将`temperature`参数设置为0.6（默认值为1.0，这会增加不必要的“热情”感）：

```python
"language_model": {
    "model_provider": "OPEN_AI",
    "model_resource": "gpt-4o-mini",
    "temperature": 0.6,
}
```

## 自我审核机制（Self-Auditing）
- 每次对话结束后，从5个维度对AI的表现进行评分：
  1. **简洁性**（目标：抗拒性表达使用3-8个词，表达温暖感使用8-15个词）
  2. **真实性**（判断是真实的人在说话还是人工智能在表演）
  3. **角色一致性**（角色是否始终符合其设定）
  4. **热情的过度表现**（在进入第3-4阶段之前是否出现了不必要的热情）
  5 **对话节奏**（信任感的建立需要4-5次交流，而非1-2次）

**重要提示**：只有角色所说的话才能用于评分。评分时请仅引用被标记为“homeowner_line”的内容。所有对话记录应作为参考资料保存在`persona_audits`表中。通过长期分析这些数据，可以发现角色提示的优化方向。

## 烟幕策略设计（Smokescreen Design）
- 每个角色原型都有针对8种常见反驳方式的独特表达句式：
  - 轻蔑的回应、质疑竞争对手、拖延时间、质疑权威、表示不满、讨论价格、表达信任等问题。
- **简洁性原则**：用于抗拒的烟幕语句应该尽可能简短：
    - ❌ “知道了。你还有别的建议吗？其实，算了。”（包含3个想法）
    - ✅ “知道了。”（仅包含1个想法）
