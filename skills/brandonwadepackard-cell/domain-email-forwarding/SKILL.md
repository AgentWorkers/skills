# 域名邮箱转发

将来自自定义域名的邮件转发到可访问的收件箱中。当原始邮箱位于你拥有的域名上，但你未主动为其提供邮箱托管服务时，这一功能对于账户恢复非常有用。

## 使用场景

- **账户恢复**：目标平台将验证码发送到 `user@yourdomain.com`，但你的邮箱托管服务处于关闭状态。
- **捕获所有邮件**：将某个域名的所有邮件转发到一个统一的收件箱中。
- **节省成本**：将域名邮件转发到 Gmail，而无需支付额外的邮箱托管费用。
- **临时用途**：只需接收一条验证码，之后即可恢复原有设置。

## 决策矩阵

| 注册商/DNS 服务 | 是否提供免费转发 | 设置时间 | 注意事项 |
|---|---|---|---|
| **GoDaddy** | ✅ 内置功能 | 5 分钟 | 需要启用“转发状态”选项 |
| **Cloudflare** | ✅ 提供邮件转发功能 | 10 分钟 | 域名的 DNS 必须托管在 Cloudflare 上 |
| **Namecheap** | ✅ 内置功能 | 5 分钟 | 免费提供最多 100 条转发的服务 |
| **ImprovMX** | ✅ 免费 tier | 15 分钟 | 适用于任何注册商，只需修改 MX 记录 |
| **Google Workspace** | ❌ 需付费 | 30 分钟 | 对于单纯的转发需求来说过于复杂 |

## GoDaddy

### 如果已有邮箱账户（例如，使用 Email Essentials 计划）

1. 登录 GoDaddy（网址：`productivity.godaddy.com`）
2. 进入 **管理 → 邮箱转发**（侧边栏）
3. 找到相应的域名 → 点击现有转发规则的 **编辑**（铅笔图标）
4. 更改 “转发邮件到” 的地址为你的 Gmail 或其他可访问的邮箱地址
5. 点击 **保存**
6. **重要提示**：确保 “转发状态” 选项处于 **开启** 状态。

### 如果没有邮箱账户（仅用于域名转发）

1. 登录 GoDaddy（网址：`account.godaddy.com/products`）
2. 找到你的域名 → 点击 **管理 DNS**
3. GoDaddy 可能提供无需完整邮箱计划的免费转发服务
4. 进入 **邮箱转发** 部分 → 添加新的转发规则

### 注意事项：
- 用户级别的转发设置界面可能会卡住（显示加载动画）。建议使用 **管理员级别** 的转发设置页面（网址：`/#/admin/email/forwarding`）。
- 外部地址的转发功能需要手动启用 “转发状态” 选项（该选项不是自动开启的）。
- 设置更改后 1-5 分钟内生效（由于已有邮箱账户，无需等待 DNS 传播）。

## Cloudflare 邮件转发

如果你的域名 DNS 已经托管在 Cloudflare 上，这是最佳选择。

### 设置步骤：
1. 登录 Cloudflare 控制面板 → 选择你的域名 → 点击 **邮件转发**
2. 点击 **启用邮件转发**
3. Cloudflare 会自动添加所需的 MX 和 TXT 记录。
4. 创建转发规则：
   - 自定义转发地址：`user@yourdomain.com`
   - 转发目标地址：`your@gmail.com`
5. Cloudflare 会发送一封验证邮件到目标邮箱，请点击邮件中的链接完成验证。
6. 规则即刻生效。

### 捕获所有邮件（Catch-all 功能）

- 启用 “捕获所有邮件” 功能，可将该域名的所有邮件转发到一个统一的收件箱中。
- 适用于不知道具体接收验证码邮箱地址的情况。

### 注意事项：
- 需要将域名 DNS 托管在 Cloudflare 上。
- 目标邮箱地址必须经过验证（点击验证邮件中的链接）。
- 如果现有邮箱托管服务的 MX 记录与 Cloudflare 的设置冲突，Cloudflare 会发出警告。

## Namecheap

1. 登录 Namecheap → 进入域名列表 → 选择你的域名
2. 点击 **“邮箱转发”** 标签（或选择 **管理 → 邮件设置**）
3. 从下拉菜单中选择 “邮箱转发” 功能
4. 添加转发规则：`user` → `destination@gmail.com`
5. 保存设置。

### 注意事项：
- Namecheap 的免费套餐支持每天最多 100 条邮件的转发。
- 选择 “邮箱转发” 功能后，MX 记录会自动配置。

## ImprovMX（适用于任何注册商）

适用于任何域名注册商。免费 tier 支持每天 25 条邮件的转发。

### 设置步骤：
1. 访问 `improvmx.com` → 输入你的域名
2. 添加转发别名：`user@yourdomain.com` → `destination@gmail.com`
3. ImprovMX 会提供 MX 记录的配置指南（请参考 **```
   MX mx1.improvmx.com (priority 10)
   MX mx2.improvmx.com (priority 20)
   ```**）
4. 在你的注册商处添加相应的 MX 记录。
5. 等待 DNS 传播完成（通常需要 5 分钟到 48 小时）。

### 注意事项：
- 免费套餐仅支持接收邮件，不支持发送邮件。
- DNS 传播可能导致首封邮件延迟几分钟到几小时才能送达。
- 需要删除与现有邮箱服务冲突的 MX 记录。

## 验证步骤

设置转发规则后，务必进行验证：

1. 用另一个账户向 `user@yourdomain.com` 发送一封测试邮件。
2. 检查目标邮箱的收件箱（以及垃圾邮件文件夹）。
3. 如果测试邮件成功送达，说明转发功能正常，可以继续进行账户恢复操作。
4. 如果 5 分钟后仍未收到测试邮件：
   - 检查 GoDaddy 的 “转发状态” 选项是否处于开启状态。
   - 确认 MX 记录是否正确（使用 `dig MX yourdomain.com` 命令检查）。
   - 检查目标邮箱的垃圾邮件文件夹。
   - 如果刚修改了 MX 记录，请等待 DNS 传播完成。

## 账户恢复后的处理

账户恢复后：
- 将账户的默认邮箱地址更新为你的主要邮箱（Gmail），以避免再次使用转发功能。
- 决定是否继续使用转发功能：如果需要，可保留该设置（以便将来接收该域名的邮件）；否则可恢复到原始设置。
- 记录账户的邮箱地址和登录方式，以备将来参考。
