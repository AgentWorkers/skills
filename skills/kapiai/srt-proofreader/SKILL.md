# SRT 校对工具

请按照以下工作流程依次执行操作。尽量减少编辑内容，并确保每一步的操作都是明确且可预测的。

## 1. 进入 `srts/` 目录

1. 切换到任务的 `srts/` 目录。
2. 如果 `srts/` 目录不存在，请立即停止并报告。

## 2. 确保文件位于 Git 仓库中

1. 检查 `srts/` 目录是否已经存在于 Git 仓库中。
2. 如果不在 Git 仓库中：
   - 运行 `git init` 命令初始化仓库。
   - 运行 `git add .` 命令添加所有文件到暂存区。
   - 运行 `git commit -m "init srt"` 命令创建第一个提交。
3. 请勿修改现有的提交历史记录。

## 3. 查看并阅读 `source.md` 文件

1. 检查 `srts/source.md` 文件是否存在。
2. 如果文件缺失，请立即停止并告知用户：“`source.md` 文件尚未创建。”
3. 在编辑字幕之前，请先阅读 `source.md` 文件。
4. 从 `source.md` 文件中提取专业术语和专有名词，并将其作为官方的术语使用。

## 4. 选择主要的 `.srt` 文件

1. 列出 `srts/` 目录下所有扩展名为 `.srt` 的文件；
2. 确保文件名不以`.` 或 `_` 开头。
3. 必须选择其中一个文件作为主要的字幕文件。
4. 如果没有符合条件的文件，请立即停止并询问用户哪个文件是主要的字幕文件。

## 5. 处理大型字幕文件（行数超过 400 行）

1. 统计主要 `.srt` 文件的行数。
2. 如果行数小于或等于 400 行，可以直接进行编辑。
3. 如果行数超过 400 行：
   - 使用 `scripts/srt_sections.mjs` 脚本将文件拆分为多个部分（例如 `section-001.srt`、`section-002.srt` 等）。
   - 依次编辑每个部分。
   - 使用相同的脚本将拆分后的部分合并回原来的 `.srt` 文件中。
5. 使用以下脚本接口进行文件分割和合并：
   - 分割命令：`node scripts/srt_sections.mjs split --input <main.srt> --output <dir> --chunk 400`
   - 合并命令：`node scripts/srt_sections.mjs merge --input-dir <dir> --output <main.srt>`

## 6. 字幕编辑规则

1. 仅修改拼写错误和术语不一致的问题。
2. 绝不要修改字幕的顺序编号。
3. 绝不要修改时间戳行（`-->`）。
4. 保持原始的行结构和空行格式。
5. 优先使用 `source.md` 文件中的术语。
6. 在最后一行字幕文本后不要添加新的标点符号。

## 7. 最终验证

1. 查看 `git diff` 的输出，确认索引行和时间戳行没有发生变化。
2. 确认只有字幕文本行被修改过。
3. 完成编辑后，简要总结修改的术语类别并报告结果。
