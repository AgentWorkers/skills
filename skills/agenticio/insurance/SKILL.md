# 保险管理

这是一个专注于本地保险记录管理的工具。它可以帮助用户追踪已有的保险记录，整理相关日期，并记录与保险索赔相关的事件。

## 高度重视隐私与安全

### 数据存储（至关重要）
- **所有保险记录仅存储在本地**，路径为 `memory/insurance/`  
- **不使用任何外部 API 来处理保险数据**  
- **不与保险公司系统建立连接**  
- **本工具不提供任何保险购买服务**  
- 用户可以完全控制数据的保留和删除  

### 安全准则（不可商讨）  
- ✅ 记录保险单的详细信息  
- ✅ 记录保险索赔和事件详情  
- ✅ 统计已支付的保费、续保情况以及保险单的数量  
- ✅ 在用户的记录中标记缺失或未记录的保险类别  
- ❌ **绝不会提供保险建议**  
- ❌ **绝不会推荐具体的保险金额**  
- ❌ **绝不会告诉用户应该购买哪种保险**  
- ❌ **绝不会替代持证保险代理人、经纪人或律师的角色**  
- ❌ **绝不会协助用户购买保险**  

### 法律免责声明  
保险决策应基于个人情况、所在司法管辖区以及保险合同的具体条款。本工具仅用于组织和查看已保存的保险记录。如果记录中存在缺失的保险类别，并不意味着用户需要额外的保险。对于保险范围的确定或法律解读，请咨询持证的专业保险顾问。  

## 数据文件  
本工具将数据存储在以下本地 JSON 文件中：  
- `memory/insurance/policies.json`  
- `memory/insurance/claims.json`  
随着功能的扩展，未来可能会添加更多文件。  

## 核心工作流程  

### 添加保险记录  
用户：“添加我的家庭保险单”  
→ 使用 `scripts/add_policy.py`  
→ 将保险单信息保存到本地  

### 查看已保存的保险单  
用户：“显示我的保险单”  
→ 使用 `scripts/list_policies.py`  
→ 显示本地记录中所有已保存的保险单  

### 检查即将到期的保险  
用户：“有哪些保险需要续保？”  
→ 使用 `scripts/check_renewals.py --days 60`  
→ 显示即将到期的保险单  

### 生成保险记录摘要  
用户：“汇总我的保险记录”  
→ 使用 `scripts/generate_summary.py`  
→ 显示按类型分类的保费总额以及即将到期的保险单  

### 记录保险索赔事件  
用户：“为我的汽车保险单记录一次索赔”  
→ 使用 `scripts/log_claim.py`  
→ 将索赔相关事件详细信息保存到本地  

## 脚本参考  
| 脚本 | 功能 |  
|--------|---------|  
| `add_policy.py` | 添加新的保险记录 |  
| `list_policies.py` | 列出所有已保存的保险单 |  
| `check_renewals.py` | 查找即将到期的保险单 |  
| `generate_summary.py` | 汇总保险记录信息 |  
| `log_claim.py` | 记录保险索赔或相关事件 |

本工具仅用于本地保险记录的管理。它不提供保险建议、推荐保险范围、解释法律条款，也不替代持证的专业人员。
