# nihao 👋

这是一个用于设置Nostr身份信息并进行健康检查的命令行工具（CLI）。它是一个单一的二进制文件，非交互式设计，适用于代理程序（agents）使用。

**来源**: https://github.com/dergigi/nihao

## 功能概述

该工具会安装一个Go语言编写的二进制文件（`nihao`），其主要功能包括：

- **生成Nostr密钥对**：使用`crypto/rand`库生成随机的Ed25519密钥对。
- **发布事件**：支持多种类型的事件，如个人资料（kind 0）、关注关系（kind 3）、备注（kind 1）、中继列表（kind 10002）、私信中继（kind 10050）、钱包信息（kind 17375）等。
- **发送HTTP请求**：用于NIP-05验证、LNURL解析、Cashu货币的铸造验证、中继节点的探测（NIP-11）以及图像的HEAD请求检查。
- **连接Nostr中继**：通过WebSocket与Nostr中继节点建立连接，以发布或查询事件。

**注意**：该工具**不会**将密钥存储在磁盘上**（它会将密钥的摘要（nsec）输出到标准输出流；可以通过`--nsec-file`选项将密钥写入文件，或通过`--nsec-cmd`选项将密钥传递给其他命令）。

## 先决条件

- **Go 1.21或更高版本**：需要安装Go语言才能从源代码编译该工具。可以通过`go version`命令查看Go版本。安装方法：https://go.dev/dl/ 或通过包管理器（如`brew install go`、`apt install golang`等）。

## 安装方法

`nihao`以源代码的形式提供，用户需要使用Go的标准工具链在本地编译它。不会提供预编译的二进制文件，用户需要从GitHub下载源代码，然后在自己的计算机上编译并将其放置在`$GOPATH/bin`目录下。

**验证安装结果**：运行`nihao version`命令。

该工具的源代码可以在https://github.com/dergigi/nihao上查看，以确保其代码的完整性和安全性。

## 使用步骤

当代理程序安装该工具后，需要按照以下步骤操作：

### 第一步：安装二进制文件

### 第二步：创建身份信息

该步骤会生成一个密钥对，并将身份信息安全地写入`./nsec.key`文件中（文件权限设置为`0600`）。用户可以通过解析生成的JSON文件来获取`npub`、`pubkey`和`relays`等关键信息。

**重要提示**：如果`nsec`文件丢失，将无法恢复其中的信息。请务必使用`--nsec-file`或`--nsec-cmd`选项来安全地保存密钥。

### 第三步：通知用户

需要向用户提供以下信息：
- 他们的代理程序的`npub`信息。
- 他们的Lightning网络地址（默认为`<npub>@npub.cash`）。
- 并提醒用户备份`nsec`文件。

### 第四步（可选）：关注其他用户

如果用户提供了自己的`npub`信息，代理程序应该自动开始关注该用户。`nihao`本身不负责处理具体的关注关系，可以使用`nak`或其他工具来完成这一操作。

## 设置新身份信息

执行以下命令来创建新的Nostr身份信息：

- 生成一个新的Nostr密钥对（或使用`--sec`/`--stdin`选项指定密钥）。
- 发布个人资料信息（kind 0）。
- 发布中继列表（kind 10002），并设置NIP-65读写权限。
- 发布私信中继列表（kind 10050）。
- 设置Lightning网络地址（`<npub>@npub.cash`）。
- 发布一条带有`#nihao`标签的备注。

## 密钥管理

**注意**：该工具默认情况下不会将密钥存储在磁盘上。密钥的安全处理方式如下：
- `--nsec-file <文件路径>`：将密钥摘要写入文件，权限设置为`0600`（推荐用于自动化脚本）。
- `--nsec-cmd <命令>`：将密钥传递给其他命令的标准输入（例如密码管理器）。
- `--stdin`：从标准输入读取密钥，避免密钥泄露到shell历史记录或进程列表中。
- `--json`：以结构化JSON格式输出密钥信息，便于程序处理。

**重要提示**：在共享环境中，**避免将原始的密钥值作为CLI参数传递**（例如`--sec nsec1...`），因为这些参数可能会被进程列表显示出来。建议使用`--stdin`或`--nsec-cmd`选项。

## 检查现有身份信息

该工具可以检查并评估用户的身份信息质量（评分范围为0–8分）：

- 检查个人资料的完整性（名称、显示名称、简介、图片、横幅等）。
- 验证NIP-05标识符的有效性。
- 检查图片的可访问性、Blossom平台的托管情况、文件大小等。
- 验证Lightning网络地址的有效性。
- 检查中继节点的可用性、数量等。

## 备份功能

该工具可以导出所有与身份相关的事件信息（以JSON格式），包括个人资料、关注关系、中继列表等。这些数据可以用于备份、迁移或存档。

## 集成建议

安装完成后，可以定期运行`nihao check <npub> --json --quiet`命令来检查身份信息的健康状况。如果评分下降，可以及时采取措施。

## 安全性注意事项

- **无预编译的二进制文件**：用户需要在本地使用Go编译该工具。
- **不存储密钥**：除非明确使用`--nsec-file`或`--nsec-cmd`选项，否则不会保存密钥。
- **限制网络连接**：仅与Nostr中继节点（WebSocket）、NIP-05/LNURL端点（HTTPS）和Cashu铸造平台（HTTPS）进行网络通信。
- **优先使用标准输入**：在使用现有密钥时，建议使用`--stdin`而不是`--sec`选项，以避免泄露进程信息。
- **文件权限设置**：确保密钥文件仅允许所有者读写。

## 默认设置

- 中继节点列表：`relay.damus.io, relay.primal.net, nos.lol, purplepag.es`
- 私信中继节点：`nip17.com, relay.damus.io, relay.primal.net, nos.lol`
- Lightning网络地址：`<npub>@npub.cash`
- Cashu货币铸造地址：`minibits, coinos, macadamia`
- 钱包类型：`17375`（NIP-60）
