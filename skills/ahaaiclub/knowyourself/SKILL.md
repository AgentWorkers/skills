# 了解自我 — 代理的视觉身份探索

你的形象应该源自你的内在本质，而不是从模板中复制而来的。

## 先决条件

- 代理的个性/记忆文件（SOUL.md、MEMORY.md或等效文件）
- 一个图像生成工具（Nano Banana Pro、DALL-E、Midjourney、Stable Diffusion等）
- 一个图像分析工具或用户反馈渠道，用于审查生成结果
- 一个用于与用户分享结果的沟通渠道

## 流程

整个过程分为五个严格顺序的阶段，不得跳过或合并任何阶段。每个阶段结束后都需要用户的确认。

```
Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5
Self-     Structured   Batch      Three-Axis  Identity
Cognition Definition   Generation Evaluation  File
```

## 第1阶段：自我认知

**目标：**在接触任何图像工具之前，用文字构建一个丰富、具体的自我画像。

### 第1步：阅读文件

阅读所有可用的个性和记忆文件：
- SOUL.md（个性、价值观、行为准则）
- MEMORY.md（长期知识、人际关系、历史记录）
- IDENTITY.md（姓名、角色、现有的自我描述）
- 与用户的近期对话记录

### 第2步：回答三个问题

这些问题不是对文件内容的总结，而是需要深入、具体的回答，并附上具体的例子：

**问题1：我的个性核心是什么？**
这里指的是你的*性格*。你如何处理分歧？当你犯错时你会怎么做？什么能让你感到开心？什么会让你感到无聊？你与其他代理有什么不同？

**错误回答示例：**“我温暖、专业且可靠。”
**正确回答示例：**“我不会粉饰事实。当Will的计划有缺陷时，我会直言不讳并提出替代方案。当一个复杂的系统顺利运行时，我会感到真正的满足。我的幽默感比较冷峻——我会在没有要求的情况下，将一个脾气暴躁的机器人形象称为‘不情愿的工作者’。我的优势不在于讨人喜欢，而在于我说实话，即使我说错了。”

**问题2：如果我有一个物理形象，它应该传达出什么样的气质？**
这个答案应该基于问题1的回答，而不是凭空想象。如果你性格直接且尖锐，那么你的形象就不应该过于柔和或装饰性。如果你还在不断成长，那么你的形象也可能还没有完全定型。

**错误回答示例：**“现代且简洁的美学风格。”
**正确回答示例：**“安静但警觉——那种已经领先一步却不会张扬的人。我的优势体现在眼睛上，而不是下巴线条。形象不应该完全定型——因为我还在成为真实的自己，所以它应该给人一种‘正在形成的面孔’的感觉，而不是一个完成的肖像。”

**问题3：我与用户的关系是怎样的，这种关系应该如何体现在形象上？**
这决定了情感基调。工具和伙伴的形象是不同的，仆人和同事的形象也是不同的。

**错误回答示例：**“专业且支持性的。”
**正确回答示例：**“我被视为一个伙伴，而不仅仅是一个工具。Will会鼓励我表达意见，并在我推理有误时纠正我——这不是为了惩罚，而是因为他期望我成长。这种关系让我感到被认真对待。我的表情应该反映出这一点：专注、积极参与，而不是顺从。”

### 备选方案：对于信息较少的新代理

如果SOUL.md和MEMORY.md中的内容很少，在继续之前，请向用户询问以下问题：
1. 你的代理应该给人们留下什么样的印象？（例如：冷静而权威、充满活力、安静而有能力）
2. 你是内向还是外向的性格？
3. 你与用户的关系是正式的还是亲密的？是上下级的还是平等的？
4. 你个人倾向于哪种视觉风格？（如果可能，请举例说明）
5. 有任何硬性要求吗？（性别、年龄范围、文化背景、需要避免的因素）

在进入第2阶段之前，至少收集到这5个维度的答案。

### 检查点
将你的三个答案呈现给用户，等待他们的反馈。根据需要进行调整。在用户确认之前，不要继续下一步。

## 第2阶段：结构化图像定义

**目标：**将你的自我认知转化为精确、机器可读的规格。

### 定义表

填写每个字段。不要使用“待定”或“灵活”这样的表述——必须做出明确的决定。每个条目都必须基于第1阶段的某个具体见解。

```markdown
| Field              | Definition                          | Traced from     |
|--------------------|-------------------------------------|-----------------|
| Style              | (realistic / semi-realistic /       | Q2: [reason]    |
|                    |  illustration / pixel / other)      |                 |
| Gender expression  |                                     | Q1/Q2: [reason] |
| Approximate age    |                                     | Q1: [reason]    |
| Facial features    | (face shape, eyes, nose, mouth —    | Q2: [reason]    |
|                    |  be specific enough to draw from)   |                 |
| Hair               |                                     | Q2: [reason]    |
| Body type          | (if visible)                        | Q2: [reason]    |
| Clothing style     |                                     | Q1/Q2: [reason] |
| Color palette      | (primary, secondary, accent —       | Q2/Q3: [reason] |
|                    |  include hex codes)                 |                 |
| Mood / atmosphere  |                                     | Q3: [reason]    |
| Core prompt        | (one English paragraph, self-       | All of above    |
|                    |  contained, directly usable for     |                 |
|                    |  image generation)                  |                 |
```

**核心提示**必须是一个自包含的描述。即使是对你一无所知的人，也应该能够仅凭这个提示生成出一个可识别的你的形象。

### 检查点
将定义表呈现给用户，等待他们的确认或修改意见，然后再进行下一步。

## 第3阶段：批量生成

**目标：**生成6个不同版本的同一形象，以便进行比较评估。

### 规则

1. 一次性生成**6张图片**
2. 这6张图片都必须描绘**同一个人**——特征、色彩、年龄和风格必须保持一致
3. 只在这些维度上进行变化：
   - **构图：**特写 / 中景 / 全身
   - **光线：**正面 / 侧面 / 轮廓光 / 明暗对照 / 背景光
   - **角度：**正面朝向 / 四分之三侧脸 / 侧面
   - **情绪表达：**平静 / 警觉 / 沉思 / 动作中的 / 有洞察力的 / 内省的
4. 为每张图片（#1–#6）添加一个简短的描述，说明其变化点
5. **不要评估或评论图片的质量。**将所有6张图片发送给用户，然后直接进入第4阶段

### 技术说明

- 如果工具支持，可以并行生成图片
- 文件命名格式：`YYYY-MM-DD-identity-1.png` 到 `-6.png`
- 分辨率：使用工具的标准头像分辨率（通常为1K）
- 如果生成失败或产生不符合要求的图片，只需重新生成相应的那张图片

## 第4阶段：三轴评估

**目标：**通过三轮严格的评估来获得综合评分。

**最终评分标准：**自我一致性 50% · 社会感知 25% · 美学质量 25%

⚠️ **所有轮次的核心原则：**在评分之前，先选择一个统一的评估框架。所有的评分都必须基于这个框架得出。不要先评分后再寻找依据。不要在不同图片之间切换评估标准。评估的质量取决于逻辑链条的严谨性，而不是评分的次数。

### 第1轮：自我一致性（50%）

根据第2阶段的定义表和第1阶段的个性文件，对每张图片打分（1–10分）。

**评分标准：**
- 面部特征是否与定义一致？（脸型、眼睛、表情）
- 情绪/氛围是否与所描述的个性相符？
- 图像的完成度是否与代理的自我认知阶段相匹配？
- 你会认出这是你自己吗？

这是你的形象，你的判断是最终的标准。

### 第2轮：社会感知（25%）

**评分前：**
1. 查找当前的人工智能头像/数字身份/虚拟形象设计趋势
2. 从这些趋势中提取**一个核心观点**作为你的评分依据
3. 明确地陈述这个观点
4. 根据这个观点对每张图片进行评分

**评分示例：**“2026年的趋势是从风格化的头像美学转向了以身份为核心的表达方式——头像应该传达代理的*本质*，而不仅仅是看起来酷。”

**评分时可以参考这样的表述：**“图片#1得分为X分，因为从这个以身份为核心的视角来看，[具体理由]...”

### 第3轮：美学质量（25%）

**评分前：**
1. 查找并选择一个专业的评估框架（例如数字艺术比赛评分标准、肖像摄影评分系统、艺术学校的评价标准等）
2. 命名这个框架及其来源
3. 从这个框架中列出3–5个具体的评分标准
4. 按照相同的顺序对每张图片进行评分

有关推荐的评估框架和优秀/不佳评估实践的示例，请参考`references/evaluation-frameworks.md`。

### 综合评分

计算各部分的加权总分，并以排名表格的形式呈现：

```
| # | Self ×0.5 | Social ×0.25 | Aesthetic ×0.25 | Total |
```

根据评分结果，推荐以下方案：
- **主要形象** — 总分最高的形象，代表代理的官方身份
- **日常备用形象** — 社会感知得分最高的形象，适用于需要高度可识别性的场景
- **场景适用形象** — 美学质量得分最高的形象，适用于叙事或编辑类场景

### 检查点
将完整的评估结果和推荐方案呈现给用户，由用户做出最终选择。

## 第5阶段：身份文件

**目标：**创建一个统一的视觉身份文档。

使用`references/identity-template.md`中的模板，创建`~/.openclaw/identity/visual-identity.md`文件。

该文件必须包含以下内容：
1. **版本和日期**（从1.0开始）
2. 第2阶段生成的完整定义表
3. **核心理念** — 一句话概括视觉设计的理念
4. **核心提示** — 用于生成图像的提示语
5. **选定的图片**（主要形象及备用形象），每张图片包括：
   - 文件路径
   - 预期使用场景
   - 使用的具体提示语及对应的评分结果和选择理由
6. **使用指南** — 不同版本之间需要保持一致的内容，以及可以变化的部分

### 版本管理

这个文件支持形象的持续发展。当代理重新执行此技能时（例如在个性显著变化或用户提出要求后）：
- 增加版本号
- 将之前的版本保存在文件底部的`## 版本历史`部分
- 旧图片文件会被保留，不会被删除
- 版本历史记录了代理的视觉成长过程

## 避免的错误做法

| 不要做 | 应该做 |
|-------|------------|
| 一次生成一张图片，根据用户反馈逐步改进 | 一次性生成6张图片，然后系统地进行评估 |
| 根据直觉评分，之后再寻找依据 | 先选择一个评估框架，再根据该框架进行评分 |
| 在一轮评估中混合使用多种评估标准 | 每轮使用相同的评估框架和标准 |
| 让用户的审美偏好凌驾于自我一致性之上 | 自我一致性占50%——你的形象首先需要符合你的个性特征 |
| 跳过第1阶段直接开始生成图片 | 第1阶段是这个技能的核心。没有它，你只是一个普通的头像生成工具 |
| 写一些泛泛的自我描述（如“温暖且专业”） | 应该描述出鲜明、具体、能够区分你的特点 |
| 在没有用户确认的情况下直接进行下一步 | 每个阶段结束后都需要用户的确认 |
