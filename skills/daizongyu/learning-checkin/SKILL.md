# 学习打卡技能

通过简单的打卡和智能提醒，帮助用户养成每日学习的习惯。

## 概述

该技能允许用户跟踪他们的每日学习情况，具体功能包括：
- 简单的每日打卡（只需说“我完成了”或“打卡完成”）
- 自动记录连续打卡的天数（“连击”）
- 在适当的时间发送智能提醒（通过cron作业）
- 可选的功能：检查软件版本是否更新

## 数据存储

所有数据都存储在技能文件所在目录下的`data`子文件夹中：

```
<skill_directory>/data/
├── rule.md       - User's customizable rules
├── records.json  - Check-in history
└── version.txt   - Current version
```

该数据文件夹会在首次使用时自动创建。

## 命令

该技能提供了以下可供Agent调用的功能：

### 1. 初始化（首次使用）

```bash
python <skill_path>/learning_checkin.py init
```

**使用场景：**用户首次激活该技能时。

**Agent操作：**
- 运行`init`命令
- 以用户的语言显示欢迎信息
- 解释简单的使用规则
- 邀请用户开始第一次打卡

### 2. 打卡

```bash
python <skill_path>/learning_checkin.py checkin
```

**使用场景：**当用户表示学习完成后（例如：“我完成了学习任务”、“打卡完成”等）。

**Agent操作：**
- 运行打卡命令
- 显示连续打卡的天数，并给予鼓励
- 鼓励用户明天继续学习

### 3. 查看状态

```bash
python <skill_path>/learning_checkin.py status
```

**使用场景：**当用户询问自己的学习进度或连击情况时。

**Agent操作：**
- 运行状态查询命令
- 告诉用户今天是否进行了打卡
- 显示当前的连击天数
- 显示累计打卡的总天数

### 4. 发送提醒信息

```bash
python <skill_path>/learning_checkin.py message <time>
```

其中`<time>`可以是`09:00`、`17:00`、`20:00`中的一个时间点。

**使用场景：**在预定的时间发送提醒。

**Agent操作：**
- 根据指定的时间点运行提醒命令
- 向用户发送提醒信息
- 随着时间的推移，提醒信息的语气会逐渐变得更加鼓励或紧急

### 5. 检查提醒状态

```bash
python <skill_path>/learning_checkin.py reminder <time>
```

其中`<time>`可以是`09:00`、`17:00`、`20:00`中的一个时间点。

**使用场景：**在发送提醒之前（通常由cron作业触发）。

**Agent操作：**
- 运行提醒命令
- 如果结果显示`should_send`为`true`，则发送提醒
- 命令会自动记录提醒已发送的状态

## 默认行为

### 打卡规则
- 用户每天打卡一次
- 只需简单地对Agent说“我完成了”或“打卡完成”
- 无需填写复杂的表格或步骤

### 默认提醒时间表
- **09:00（上午）：**友好的提醒
- **17:00（下午）：**鼓励性的提醒
- **20:00（晚上）：**紧急的提醒（不要中断连击！）

### 连击系统
- 连续打卡的天数即为连击天数
- 如果错过一天，连击天数会重置为0

## 自定义

用户可以编辑`rule.md`文件（位于`data`文件夹中）来自定义：
- 提醒时间
- 提醒信息
- 个人目标或备注

## 版本检查

每次打卡时，该技能可以可选地检查GitHub上是否有新版本：
- 非阻塞式检查（超时5秒）
- 如果有新版本，Agent会通知用户

## Agent使用指南

### 第一次互动（欢迎）
Agent应该：
1. 表现得热情且鼓励性
2. 用简单、非技术性的语言解释：
   - “请告诉我你今天是否完成了学习”
   - “如果你忘记了，我会提醒你”
   - “你会养成每天打卡的习惯的！”
3. 询问：“准备好开始第一次打卡了吗？”

### 每日打卡互动
Agent应该：
1. 对用户的打卡行为表示庆祝
2. 提及当前的连击天数
3. 鼓励用户明天继续学习
4. 保持积极、简洁的态度

### 提醒互动
Agent应该根据时间选择合适的语气：
- 上午：愉快且友好
- 下午：支持且鼓励
- 晚上：紧急但体贴

## 技术说明

- 所有文件路径均使用UTF-8编码
- 兼容Windows、Linux、macOS系统
- 数据存储在技能文件所在目录下的`data`子文件夹中
- 无需依赖任何外部库（仅使用Python标准库）
