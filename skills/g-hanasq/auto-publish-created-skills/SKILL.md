# 自动发布新创建的技能

当助手刚刚创建了一个新的本地技能，且用户希望将这些技能上传到 ClawHub 时，请使用此功能。

## 前提条件

只有满足以下所有条件时才能进行发布：
- 用户明确要求将助手创建的技能发布到 ClawHub；
- 技能目录中包含有效的 `SKILL.md` 文件；
- 该技能已经过本地审核并提交；
- 当前的 ClawHub 会话已登录。

## 工作流程

1. 在 `workspace/skills/<slug>` 目录下找到新创建的技能文件夹。
2. 使用 `clawhub whoami` 命令检查 ClawHub 的登录状态。
3. 使用 `clawhub inspect <slug>` 命令检查远程服务器上的技能状态。
4. 版本控制：
   - 如果技能在远程服务器上不存在，则发布版本 `0.1.0`；
   - 如果技能已经存在于远程服务器上，则更新其版本号。
5. 使用 `skills/clawhub-publish-flow/scripts/publish_to_clawhub.js` 脚本进行发布。
6. 使用 `clawhub inspect <slug>` 命令验证发布结果。
7. 如有需要，更新技能的注册信息。

## 变更日志格式

使用简短的、描述性强的变更日志，例如：
- `初始发布`
- `添加运行时说明`
- `优化触发器描述`
- `完善工作流程文档`

## 安全规则

切勿发布内容模糊或不完整的技能草稿。发布之前，技能必须内容完整且已提交到版本控制系统。

## 输出结果

输出以下信息：
- 技能名称
- 版本号
- 发布结果
- 如果可用，提供技能的最终 URL
- 验证是否成功
