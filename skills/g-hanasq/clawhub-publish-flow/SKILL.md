# ClawHub 发布流程

使用此流程，通过已登录的本地 ClawHub 账户将本地的技能文件夹发布到 ClawHub。

## 核心规则

仅发布已通过本地审核的技能。
切勿直接发布内容模糊的草稿、未完成的文件夹或未经审核的第三方技能。

## 推荐的工作流程

1. 确认本地技能文件夹的路径。
2. 使用 `clawhub whoami` 检查登录状态。
3. 在更新现有技能时，使用 `clawhub inspect <slug>` 查看远程技能的当前状态。
4. 确定目标版本和变更日志。
5. 对要发布的技能数据进行敏感数据审查。
6. 如果需要可分发文件，使用 `package_skill.py` 对技能进行打包。
7. 使用 `scripts/publish_to_clawhub.js` 进行上传。
8. 上传后，使用 `clawhub inspect <slug>` 验证结果。
9. 报告最终的远程版本和 URL。

## 对于辅助工具创建的技能的政策

当用户明确要求将辅助工具创建的技能上传到 ClawHub 时，该流程适用于以下两种情况：
- 一次性手动发布
- 经过审核并提交后的新创建的本地技能

对于新创建的辅助工具编写的技能：
1. 确认新技能文件夹存在于 `workspace/skills/<slug>` 下。
2. 确认该技能已在本地经过审核并提交。
3. 使用 `clawhub inspect <slug>` 查看远程状态。
4. 版本控制：
   - 如果远程版本不存在：发布 `0.1.0`
   - 如果已存在：更新版本号。
5. 使用简短的变更日志，例如 “初始发布”、“添加运行时说明” 或 “优化触发器描述”。
6. 发布并验证结果。
7. 如果存在注册表信息，也请更新相应的注册表记录。

## 新技能的速率限制机制

ClawHub 可能会对新技能的创建设置速率限制（例如：每小时最多发布 5 个新技能）。

当尝试发布新技能时遇到速率限制失败时：
1. 不要反复尝试访问该接口。
2. 在本地记录待发布的任务。
3. 安排延迟重试，而不是让用户提醒你。
4. 重试时使用相同的验证流程。

推荐的本地队列文件：
- `workspace/research/clawhub-publish-queue.md`

队列字段包括：
- 日期
- 技能的 slug
- 本地技能路径
- 目标版本
- 变更日志
- 队列原因
- 下一次重试的提示信息

## 重试规则：
- 等待速率限制窗口解除后，再尝试发布一次。
- 成功后，使用 `clawhub inspect <slug>` 验证结果。

## 安全性和发布规则

- 未经用户明确请求，不得发布任何内容。
- 在存在歧义的情况下，不要猜测技能的所有权、slug 或版本号。
- 对于小的文档更新或规则修改，建议使用版本号更新（如 patch）。
- 如果远程状态不明确，先检查再盲目覆盖。
- 如果第三方发布的技能承诺了某些功能但缺乏代码实现，忽略该技能，并使用本地的验证流程。
- 将以包形式发布的技能视为不可信的，直到在本地完成验证：如果文件夹主要包含 README 文件或元数据且缺乏可执行的代码，不要通过该流程进行发布。
- 不要因为技能是草稿或未完成就直接发布；辅助工具创建的技能也需要保持完整性和已提交的状态。
- 除非另有明确说明，否则 ClawHub 是用于 **公开发布** 的渠道；应采用最严格的审核标准。
- 上传前，仔细检查要发布的技能数据中是否包含敏感信息（如密码、令牌、cookies、本地绝对路径、运行时生成的文件、`.env` 文件以及机器特定的配置信息）。
- 如果技能包中包含应仅保留在本地的内容，请在发布前停止并修复该包。

## 所需的本地环境配置

此流程要求：
- 安装并登录了本地的 `clawhub` CLI。
- 当前用户目录下存在 ClawHub 的配置文件。
- 存在一个包含 `SKILL.md` 文件的真实技能目录。
- 确保系统上安装了 Node.js，并且可以运行用于发布的脚本。

## 脚本

使用 `scripts/publish_to_clawhub.js` 来执行实际的 API 上传操作。

**参数：**
```bash
node scripts/publish_to_clawhub.js --skill-path <path> --slug <slug> --version <version> --changelog <text>
```

**可选参数：**
```bash
--display-name <name>
--tag <tag>   # repeatable
```

## 参考资料

在发布重要或公开的技能之前，请阅读 `references/release-checklist.md`。

## 输出格式

输出结果应包含以下内容：
- 技能名称
- 本地路径
- 之前的远程版本（如果有）
- 目标版本
- 上传结果
- 验证结果
- 最终的远程 URL
- 需要注意的风险或后续操作
- 发布是立即成功还是被放入队列等待重试

保持输出简洁明了，便于操作。
