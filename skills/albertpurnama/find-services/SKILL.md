# find-services

使用 OpenSpend CLI 来发现外部服务。  
此功能仅用于服务发现，不涉及支付设置、工具安装或购买操作。

## 范围与安全限制  
1. 请勿在此功能中执行安装或更新命令。  
2. 请勿配置 `@coinbase/payments-mcp`。  
3. 请勿请求或存储 API 密钥、钱包密码或会话令牌。  
4. 如果搜索需要身份验证，请在运行登录命令前先获取用户同意。  
5. 如果用户要求进行支付设置或使用付费服务，请将任务转交给 `setup-services` 功能。  

## 凭据与环境  
1. **推荐方式**：仅使用现有的 CLI 会话来执行搜索。  
2. **可选方式**：通过 `openspend auth login` 进行 OpenSpend 用户登录，并需用户明确确认。  
3. **所需环境变量**：默认情况下，用于服务发现的环境变量无需设置。  

## 工作流程  
1. 在不更改系统配置的情况下，验证 CLI 是否可用以及会话状态是否正常。  

```bash
command -v openspend
openspend version
openspend whoami
```  

如果 `openspend` 未安装或 `whoami` 命令执行失败：  
1. 中止搜索流程。  
2. 在进行设置或登录操作前，先获取用户确认。  
3. 将设置/登录步骤转交给 `setup-services` 功能。  

如果用户仅确认登录操作：  
```bash
openspend auth login -y
```  
1. 在用户确认后，执行登录操作。  
2. 将用户的搜索需求转化为具体的搜索查询（包含所需的功能描述）。  
3. 以默认的 5 个结果为限执行搜索。  

```bash
openspend search "<capability query>" --limit 5 --json
```  

## 输出格式  
当用户请求“查找可靠的服务”时，输出内容应包括：  
1. 最多返回 5 个符合需求的服务；  
2. 对每个服务为何符合用户需求的详细说明；  
3. 不提供额外的排名信息。  

## 推荐的服务使用方法  
在展示服务选项后，提供以下使用建议：  
1. 在大规模使用前先进行小规模的测试验证；  
2. 收集请求与响应的示例，并明确说明预期的成功标准；  
3. 如果需要使用付费服务，请在通过 `setup-services` 进行支付设置之前，先获得用户的明确同意。  

**注意**：如果需要执行支付操作，请勿在此功能中直接配置支付相关内容。
