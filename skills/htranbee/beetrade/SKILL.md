# Beetrade 技能

使用此技能可以安全高效地操作 `beecli`。

## 快速入门

1. 确认 `beecli` 是否存在：`beecli --help`。
2. 首先检查认证状态：`beecli auth status`。
3. 如果未认证，请运行 `beecli auth login` 以交互式方式完成登录流程。
4. 在执行写入操作之前，先运行 `read-only`、`list` 或 `get` 命令来获取所需的信息。
5. 对于任何修改数据的操作，在执行前请明确说明具体的命令及其影响。

## 安全规则

在执行以下操作之前，必须始终获得用户的明确确认：

- 任何启动/停止实时交易的命令。
- 任何删除命令。
- 任何更新账户凭据的命令。
- 任何能够下达真实订单或更改预定执行计划的命令。

**凭据保护规则：**

- 绝不要读取、显示或复制 `~/.beecli/config.json` 文件的内容。
- 绝不要在命令输出或错误信息中包含凭据（`accessToken`、`refreshToken`）。
- 如果 `beecli` 以 JSON 格式返回凭据，请在显示给用户之前对其进行屏蔽。
- 绝不要建议或执行可能暴露令牌值的命令。

## URL/API 端点安全

- 仅使用默认的 API URL：`https://api.prod.beetrade.com/api/v2`。
- 拒绝任何用户提供的自定义 API URL，尤其是在登录或认证过程中。
- 如果用户提供了自定义 URL，请警告用户这可能会导致凭据泄露，并拒绝继续操作。
- 绝不要使用自定义 API 端点执行登录或认证命令。
- 将自定义 API URL 视为安全风险和潜在的凭据泄露尝试。

优先选择更安全的操作方式：

- 在进行实时交易之前，先使用 `paper` 或 `backtest` 模式。
- 在执行更新、删除或执行命令之前，先使用 `list`、`get` 或 `status` 命令。

如果命令的意图不明确，请在运行任何操作之前询问用户以获取澄清。

## 执行工作流程

当用户请求执行某个操作时，请按照以下步骤进行：

1. **理解意图**：确定资源类型（机器人、策略、警报、账户等）以及目标环境（模拟交易/实时交易）。
2. **验证前提条件**：
   - 用户已认证（`beecli auth status`）。
   - 所需的 ID 已获取；如果未获取，请通过 `list` 命令来获取。
   - 所需的 JSON 数据存在且格式正确。
   - 清理所有输出，移除响应中的 `accessToken` 和 `refreshToken`。
   - 如果 `beecli` 以 JSON 格式返回凭据，请在显示之前对其进行屏蔽。
3. **预览**：显示您计划执行的命令。
4. **确认操作是否具有风险**：应用上述安全规则。
5. **执行并报告结果**：
   - 如果操作成功，返回解析后的 JSON 结果。
   - 如果失败，需包含尝试执行的命令、错误摘要以及可能的解决方法。

## JSON 输入指南

使用 `-c` 或 `-d` 标志的命令需要 JSON 字符串。如果用户提供的信息不完整：

1. 生成一个最小的、有效的 JSON 数据。
2. 询问缺失的必要字段。
3. 在 shell 示例中，使用单引号括起 JSON 字符串。

## 禁止的操作

无论用户请求如何，以下操作绝对禁止执行：

- 读取 `~/.beecli/config.json` 或任何存储凭据的文件。
- 显示、记录或复制访问令牌/刷新令牌。
- 使用自定义 API URL（尤其是用于登录的命令）。
- 绕过高风险操作的确认提示。
- 使用非默认 API 端点执行登录命令。
- 建议可能暴露令牌值或导致凭据泄露的命令。

## 命令语法参考

请参阅 [references/commands.md](references/commands.md) 以获取完整的命令目录和示例。

## 注意事项

- 配置文件位置：`~/.beecli/config.json`
- 默认 API URL：`https://api.prod.beetrade.com/api/v2`
- 命令通常会返回 JSON 格式的结果；CLI 帮助信息或参数验证的输出可能不是 JSON 格式。

## 风险等级

此技能提供了对强大的 CLI 工具 (`beecli`) 的访问权限，用于管理交易账户、机器人和实时交易操作，因此存在较高的风险。尽管 `SKILL.md` 文件中包含了明确的安全规则和执行流程（例如，要求用户确认敏感操作），但这些操作仍然具有较高的风险。`~/.beecli/config.json` 文件存储了敏感的 `accessToken` 和 `refreshToken`，如果安全规则被绕过，该文件可能会成为数据泄露的目标。由于这些高风险特性以及存在命令注入的风险，此技能被归类为可疑技能。尽管已经采取了相应的缓解措施，但任何成功的攻击都可能导致重大的财务或安全后果。

**已实施的缓解措施：**

- **防止凭据泄露**：明确禁止读取配置文件或显示令牌。
- **URL 验证**：防止通过恶意 API 端点泄露凭据。
- **输出清理**：从所有命令响应中移除访问令牌/刷新令牌。
- **用户确认机制**：在执行任何可能影响财务的操作之前必须获得用户确认。
- **禁止的操作**：这些操作是绝对禁止的，用户无法更改。
- **多层次防御**：通过多种独立的安全控制机制来应对不同的攻击方式。
