# Beetrade 技能

使用此技能可以安全、高效地操作 `beecli`。

## 快速入门

1. 确认 `beecli` 是否存在：`beecli --help`。
2. 首先检查认证状态：`beecli auth status`。
3. 如果未认证，请运行 `beecli auth login` 以交互式方式完成登录流程。
4. 在执行写入操作之前，先运行 `read-only`、`list` 或 `get` 命令来获取所需的信息。
5. 对于任何会修改数据的操作，在执行前请明确说明具体命令及其影响。

## 安全规则

在执行以下操作之前，必须始终获得用户的明确确认：

- 任何用于启动/停止实时交易的命令。
- 任何删除命令。
- 任何会更新账户凭据的命令。
- 任何可以下达真实订单或更改预定执行计划的命令。

优先选择更安全的操作方式：

- 在进行实时交易之前，先使用 `paper` 或 `backtest` 模式。
- 在执行 `update`、`delete` 或 `run` 命令之前，先使用 `list`、`get` 或 `status` 命令。

如果命令的意图不明确，请在执行前询问用户以获取澄清。

## 执行工作流程

当用户请求执行某个操作时，请按照以下步骤进行：

1. **理解意图**：确定资源类型（机器人、策略、警报、账户等）以及目标环境（模拟交易/实时交易）。
2. **验证前提条件**：
   - 用户已认证（`beecli auth status`）。
   - 所需的 ID 已经获取；如果未获取，可以通过 `list` 命令来获取。
   - 所需的 JSON 数据存在且格式正确。
3. **预览**：显示您计划执行的命令内容。
4. **确认操作风险**：应用上述安全规则。
5. **执行并报告结果**：
   - 如果操作成功，返回解析后的 JSON 结果。
   - 如果失败，需提供尝试执行的命令、错误摘要以及可能的解决方法。

## JSON 输入指南

使用 `-c` 或 `-d` 标志的命令需要 JSON 字符串。如果用户提供的信息不完整：

1. 构建一个包含最小必要字段的有效 JSON 数据。
2. 询问用户缺失的必要字段。
3. 在 shell 命令中，使用单引号括起 JSON 字符串。

## 命令语法参考

请参考 [references/commands.md](references/commands.md) 以获取完整的命令目录和示例。

## 注意事项

- 配置文件位置：`~/.beecli/config.json`
- 默认 API 地址：`https://api.prod.beetrade.com/api/v2`
- 命令通常会返回 JSON 数据；不过 CLI 命令的帮助信息或参数验证结果可能不是 JSON 格式。

## 风险等级

该技能提供了对强大的 CLI 工具 (`beecli`) 的访问权限，用于管理交易账户、机器人和实时交易操作，这些操作本身具有较高的风险。尽管 `SKILL.md` 文件中包含了明确的安全规则和执行流程（例如，要求用户确认敏感操作），但仍存在一定的风险。`~/.beecli/config.json` 文件存储了敏感的 `accessToken` 和 `refreshToken`，如果安全规则被绕过，这些信息可能会被窃取。由于这些高风险功能以及潜在的命令注入风险（即使已经采取了相应的防护措施），该技能仍被归类为“可疑”级别。成功的攻击可能导致严重的财务或安全后果。
