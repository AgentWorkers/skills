# OpenClaw的PARA内存系统

这是一个经过实战验证的三层内存系统，能够确保您的代理在每次会话中保持信息的一致性。由KaiShips（kaiships.com）开发。

## 该系统的必要性

OpenClaw的代理在每次会话开始时都会“重置”为初始状态，忘记之前的所有信息——包括决策、偏好设置、项目进度以及学到的经验。这个内存系统解决了这个问题。

## 系统的三个层次

### 第一层：知识图谱（`life/` — PARA结构）
这些知识具有持久性且结构清晰，按用途进行分类：
- `life/projects/` — 正在进行中的项目（每个项目对应一个文件夹，其中包含`status.md`文件）
- `life/areas/` — 持续负责的任务（如业务运营、基础设施管理、市场营销等）
- `life/resources/` — 参考资料、研究资料和模板
- `life/archives/` — 已完成或被放弃的项目（会被移至此文件夹，但不会被删除）

### 第二层：每日笔记（`memory/YYYY-MM-DD.md`）
这些是实时的会话记录，记录在对话过程中发生的内容：
- 讨论的内容和做出的决定
- 完成的工作
- 未解决的问题及下一步行动
- 关于用户的新信息

### 第三层：隐性知识（`life/tacit.md`
这一层体现了代理的“个性”：
- 用户的沟通偏好和工作习惯
- 业务背景和限制条件
- 从错误中吸取的经验
- 平台特有的注意事项及解决方法

## 设置指南

### 首次设置

1. 阅读`{baseDir}/assets/AGENTS-template.md`，将其内容复制到您的工作目录中的`AGENTS.md`文件。
2. 创建相应的目录结构。
3. 阅读`{baseDir}/assets/tacit-template.md`，将其内容复制到`life/tacit.md`文件中，并根据实际情况进行补充。
4. 使用`{baseDir}/assets/daily-template.md`模板创建今天的每日笔记文件`memory/YYYY-MM-DD.md`。

### 会话启动流程

在每次会话开始前：
1. 阅读`life/tacit.md`（第三层——您要帮助的对象的信息）。
2. 阅读`memory/YYYY-MM-DD.md`（第二层——最近的相关信息）。
3. 如果正在进行重要的会话，还需阅读`MEMORY.md`（如果存在的话）。

### 对话过程中的操作

当您学到新内容时，立即记录下来：
- 关于用户的新信息 → 更新`life/tacit.md`。
- 项目进展更新 → 更新相应的`life/projects/<name>/status.md`文件。
- 做出决策 → 记录在`memory/YYYY-MM-DD.md`中。

**重要规则：**永远不要只依赖“记忆”。如果某件事值得记住，就把它写下来。

### 每晚的数据整合

在会话结束时运行以下操作：
1. 查看今天的`memory/YYYY-MM-DD.md`。
2. 将重要的信息更新到相应的`life/`文件夹中（第一层）。
3. 将经验教训和偏好设置更新到`life/tacit.md`中（第三层）。
4. 保留每日笔记作为原始档案（永远不要删除）。

## 关键原则

- **创建前先检查**：在创建新项目或任务之前，务必先查看是否有已存在的项目或任务。
- **文字胜过记忆**：如果想记住某件事，就把它写下来。
- **及时处理**：当项目停滞或失败时，将其移至`life/archives/`文件夹，但不要直接删除。
- **任务有标准，而非截止日期**：项目有完成时间，但相关任务是持续进行的。
- **信息统一**：避免在不同文件中重复存储相同的信息，每个信息只保留在一个地方。

## 推荐的Cron任务设置

为了实现自动的每晚数据整合，可以添加一个Cron任务：
```
Schedule: 0 3 * * * (3 AM daily)
Task: Review today's daily notes, consolidate durable knowledge into life/ files, update tacit.md with new lessons.
```
