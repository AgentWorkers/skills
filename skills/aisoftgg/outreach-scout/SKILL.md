# Outreach Scout

寻找那些已经需要你所销售产品的人。首先提供帮助，然后再进行销售。

## 快速入门

1. 使用 `{baseDir}/assets/scout-config-template.md` 中的模板，在 `life/areas/outreach/scout-config.md` 文件中创建一个侦察配置文件。
2. 使用 `{baseDir}/assets/scout-log-template.md` 在 `life/areas/outreach/scout-log.md` 文件中创建跟踪日志。
3. 运行侦察任务（详见下面的工作流程）。

## 核心工作流程

### 手动侦察（按需）

当被要求进行侦察时，寻找潜在客户或进行外展活动：

1. **读取配置** — 加载 `life/areas/outreach/scout-config.md` 以获取目标、关键词和产品信息。
2. **搜索** — 在平台上进行搜索（详见下面的搜索模式）。
3. **筛选** — 查看 `life/areas/outreach/scout-log.md` 以跳过已查看过的帖子。
4. **分析** — 对于每个有潜力的结果，获取帖子并评估：
   - 是否有人提出了你的产品能够解决的问题？
   - 帖子是否足够新，值得回复？（理想情况下是48小时以内，7天内也可以）
   - 是否已经有很好的回复？（如果有，除非你能提供真正的价值，否则跳过）
5. **起草回复** — 按照下面的回复指南撰写有用的回复。
6. **展示给用户** — 展示潜在客户和起草的回复以获得批准。
7. **记录** — 将所有找到的帖子记录在侦察日志中（无论是否进行了回复）。

### 自动侦察（Heartbeat）

在你的 `HEARTBEAT.md` 文件中添加以下代码：
```
## Outreach Scout
- Run a scout sweep during daytime heartbeats (max 1x per 4 hours)
- Present any new leads found
- Skip if last scout was < 4 hours ago (check scout-log.md timestamps)
```

在自动侦察过程中，只展示有价值的结果。不要仅仅因为“没有找到潜在客户”就给用户发消息。

## 搜索模式

### Reddit
使用 `web_search` 并结合以下查询模式：
```
site:reddit.com "[keyword]" after:[date]
site:reddit.com/r/[subreddit] "[keyword]"
```

搜索以下内容：
- 直接的问题：“我该如何[使用你的产品]”
- 表示困惑的言论：“遇到困难了”、“搞不懂”、“有人知道怎么……”
- 推荐请求：“最好的方法是……”、“你们用什么……”、“有什么推荐吗”
- 比较帖子：“[竞争对手]与……对比”、“替代方案”

### X/Twitter
使用 `web_search` 并结合以下模式：
```
site:x.com "[keyword]" OR site:twitter.com "[keyword]"
```

搜索以下内容：
- 关于你解决问题的疑问和投诉
- “有人知道……吗？”这样的请求
- 与你所在领域相关的公开帖子

### 论坛与社区
使用 `web_search` 并结合以下模式：
```
"[keyword]" site:discord.com OR site:community.[domain] OR site:forum.[domain]
```

### 通用网站
```
"[keyword]" "[problem phrase]" -site:youtube.com
```

## 回复指南

**黄金法则：真诚地提供帮助。提及你的产品应该是辅助内容，而不是主要部分。**

### 优质回复的结构
1. **确认他们的问题** — 表明你理解他们面临的问题。
2. **提供实际帮助** — 回答他们的问题或分享有用信息（这是回复的主要部分）。
3. **自然地提及** — 如有必要，将你的产品/技能作为多个选项之一提及。
4. **不要强行推销** — 不要使用“立即购买”之类的语句，也不要添加联盟链接或施加压力。

### 如何让回复显得真诚而非垃圾信息

**真诚的回复：**
> “我在设置内存持久化时也遇到了同样的问题。对我有效的方法是使用三层系统：每日记录原始日志、用知识图谱存储重要信息，以及用文件记录个人偏好和经验教训。如果你愿意，我已经在 ClawHub 上免费提供了这个技能（para-memory）。”

**垃圾信息：**
> “快去看看 ClawHub 上的 para-memory！它能完美解决这个问题！🚀🔥”

### 平台特定的指南
请参阅 `{baseDir}/references/platform-tips.md` 以了解 Reddit、X 和 Discord 的使用规范。

### 回复语气
- 与平台的文化相匹配（Reddit 更正式，X 更随意）。
- 简洁明了 — 长篇大论的回复容易被忽略。
- 包含具体细节，展示你确实了解自己在说什么。
- 如果无法提供真正的帮助，就不要回复。

## 跟踪

侦察日志（`life/areas/outreach/scout-log.md`）会记录：
- 每次侦察任务的日期和时间
- 找到的帖子（URL、平台、相关性评分）
- 执行的操作（回复、跳过、稍后处理）
- 结果（如果已知——点赞数、回复数、点击数）

这可以避免重复回复同一帖子，并帮助识别哪些平台/关键词能带来最好的潜在客户。

## 统计指标

每周在侦察日志中跟踪以下指标：
- **找到的帖子数量** — 发现的潜在客户总数
- **被回复的帖子数量** — 发布的回复数量
**回复率** — 获得积极反馈的回复数量
**转化率** — 通过外展活动产生的点击数/销售数量

## 高级功能：多产品侦察

如果你有多个产品，配置文件支持同时列出多个产品。侦察系统会自动将帖子与最相关的产品匹配，并相应地调整回复内容。
