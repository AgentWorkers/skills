# here.now

**技能版本：1.8.3**

**功能说明：**  
可以从任何文件或文件夹创建一个实时可访问的网站。仅支持静态托管。

**安装或更新方法（推荐）：**  
`npx skills add heredotnow/skill --skill here-now -g`  

**注意：**  
对于通过仓库链接或本地项目安装的情况，无需添加 `-g` 参数。

## **系统要求**  
- 必需的软件：`curl`、`file`、`jq`  
- 可选的环境变量：`$HERENOW_API_KEY`  
- 可选的凭证文件：`~/.herenow/credentials`  

## **创建网站**  
（相关代码块未提供，可参考原文说明。）  

创建的网站会生成一个实时访问的 URL（例如：`https://bright-canvas-a7k2.here.now/`）。  

**工作流程：**  
1. 创建或更新网站内容；  
2. 上传文件；  
3. 完成配置。  
只有当配置步骤成功完成后，网站才会正式上线。  

**注意事项：**  
- 如果未使用 API 密钥，创建的网站为临时网站，有效期为 24 小时。  
- 使用保存的 API 密钥后，网站将永久保存。  

**文件结构：**  
- 对于 HTML 网站，将 `index.html` 文件放置在发布的目录根目录下（不要放在子目录中）。该目录的内容将成为网站的根目录。  
- 例如：发布 `my-site/` 时，确保 `my-site/index.html` 存在，而不是发布包含 `my-site/` 的父目录。  
- 也可以直接发布原始文件（如图片、PDF、视频、音频等）。

**API 密钥存储：**  
脚本从以下来源读取 API 密钥（优先选择最先匹配的来源）：  
1. `--api-key {key}` 参数（仅用于持续集成/脚本环境，避免在交互式使用中使用）  
2. 环境变量 `$HERENOW_API_KEY`  
3. 凭证文件 `~/.herenow/credentials`（推荐使用）  

**重要提示：**  
收到 API 密钥后，请立即保存（自行执行保存操作），切勿让用户手动执行。避免在交互式会话中通过命令行参数传递密钥；建议使用凭证文件进行存储。  
切勿将凭证文件或状态文件（`~/.herenow/credentials`、`.herenow/state.json`）提交到版本控制系统中。

每次创建或更新网站后，脚本会更新 `./herenow/state.json` 文件：  
（相关代码块未提供，可参考原文说明。）

**用户须知：**  
- 始终分享当前脚本运行生成的 `siteUrl`。  
- 通过脚本的 `stderr` 输出中的 `publish_result.*` 行来了解授权模式。  
- 如果 `publish_result.auth_mode=authenticated`，说明网站是永久保存的（无需提供声明 URL）。  
- 如果 `publish_result.auth_mode=anonymous`，说明网站有效期为 24 小时，请提供声明 URL（如果 `publish_result.claim_url` 不为空且以 `https://` 开头），以便用户永久保存该链接；请告知用户声明令牌仅提供一次且无法恢复。  
- **切勿** 让用户查看 `.herenow/state.json` 以获取声明 URL 或授权状态。  

## **使用限制：**  
|                | 临时网站          | 授权网站                |
| -------------- | ------------------ | ---------------------------- |
| 最大文件大小    | 250 MB             | 5 GB                         |
| 有效期        | 24 小时           | 永久保存（或自定义过期时间）    |
| 每 IP 小时请求限制 | 5 次              | 免费账户：60 次/小时；高级账户：200 次/小时 |
| 是否需要账户    | 不需要                 | 需要账户（在 here.now 获取密钥）    |

## **获取 API 密钥：**  
要将临时网站升级为永久网站，请按照以下步骤操作：  
1. 请求用户的电子邮件地址。  
2. 向用户发送一次性登录码。  
3. 告知用户：“请在收件箱中查找来自 here.now 的登录码，并将其粘贴到这里。”  
4. 验证登录码后获取 API 密钥。

## **域名管理：**  
在 `here.now` 上，用户可以拥有自己的子域名（例如 `yourname.here.now`），这些子域名用于路由到用户的网站。  
- 子域名端点：`/api/v1/handle`  
- 子域名格式：小写字母/数字/连字符，长度为 2-30 个字符，开头和结尾不能有连字符。  

## **自定义域名：**  
可以自定义域名（例如 `example.com`），并使用它来托管网站：  
- 自由账户：支持 1 个自定义域名；  
- 高级账户：支持最多 5 个自定义域名。  
- 域名相关端点：`/api/v1/domains` 和 `/api/v1/domains/:domain`

## **链接设置：**  
链接可以将网站连接到用户的子域名或自定义域名：  
- 链接端点：`/api/v1/links` 和 `/api/v1/links/:location`  
- 路径参数示例：`__root__`  
- 更改会全局生效（最快 60 秒内，通过 Cloudflare 的 KV 系统传播）。  

**示例：**  
- 链接到用户的子域名：`/api/v1/links/my-subdomain`  
- 链接到自定义域名：`/api/v1/links/example.com/docs`
