# here.now

**技能版本：1.8.0**

**功能简介：**  
可以从任意文件或文件夹创建一个实时可访问的网站。仅支持静态托管。

**安装或更新方法（推荐）：**  
`npx skills add heredotnow/skill --skill here-now -g`  

**注意事项：**  
- 如果没有安装 `npm`，请参考 [替代安装指南](https://here.now/docs#install-skill)。

## **系统要求**  
- 必需的软件：`curl`、`file`、`jq`  
- 可选的环境变量：`$HERENOW_API_KEY`  
- 可选的凭证文件：`~/.herenow/credentials`

**工作流程：**  
1. 创建或更新网站内容；  
2. 上传文件；  
3. 完成配置后，网站才会正式上线。  

**注意事项：**  
- 未设置 API 密钥时，创建的网站为临时网站（有效期 24 小时）；  
- 设置 API 密钥后，网站将永久保存。  

**文件结构：**  
- 对于 HTML 网站，将 `index.html` 文件放在发布的目录根目录下（不要放在子目录中）。  
- 如果发布的是包含 `index.html` 的文件夹，请直接发布整个文件夹（例如：`my-site/`）。  

**文件类型支持：**  
- 单个文件：自动提供丰富的查看器（支持图片、PDF、视频、音频格式）；  
- 多个文件：会自动生成目录列表，并提供文件夹导航和图片库。

## **API 密钥存储**  
脚本从以下途径读取 API 密钥（优先级依次为：  
1. `--api-key {key}` 参数（仅用于持续集成/脚本环境）；  
2. 环境变量 `$HERENOW_API_KEY`；  
3. 凭证文件 `~/.herenow/credentials`（推荐使用）。  

**密钥安全提示：**  
收到 API 密钥后，请立即保存（请自行执行保存操作），切勿让用户手动操作。避免在交互式环境中通过命令行参数传递密钥；建议使用凭证文件进行存储。  
切勿将凭证文件或状态文件（`~/.herenow/credentials`、`.herenow/state.json`）提交到版本控制系统中。

## **用户须知：**  
- 请始终分享当前脚本生成的网址（`siteUrl`）。  
- 查看脚本输出中的 `publish_result.*` 行以确定网站的身份验证模式：  
  - 如果 `publish_result.auth_mode=authenticated`，说明网站为永久保存状态（无需提供 claim URL）；  
  - 如果 `publish_result.auth_mode=anonymous`，说明网站有效期为 24 小时，请提供 claim URL（如果 `publish_result.claim_url` 不为空且以 `https://` 开头）。  
- 请勿让用户自行查看 `.herenow/state.json` 文件以获取 claim URL 或身份验证状态。  

## **使用限制：**  
| 功能        | 临时网站            | 已认证网站                          |
|-------------|------------------|--------------------------------------|
| 最大文件大小    | 250 MB             | 5 GB                              |
| 有效期       | 24 小时             | 永久保存（或自定义过期时间）                    |
| 每 IP 小时请求量 | 5 次              | 免费账户：60 次/小时；高级账户：200 次/小时             |
| 是否需要账户    | 无需               | 需要 API 密钥（可在 here.now 获取）                    |

## **获取 API 密钥：**  
要将临时网站升级为永久网站，请按以下步骤操作：  
1. 获取用户的电子邮件地址；  
2. 向用户发送一次性登录码；  
3. 用户验证登录码后，获取 API 密钥（请用户自行保存）。

## **快速参考：**  
- **处理与链接：**  
  - 用户可以在 `here.now` 上创建自定义子域名（例如：`yourname.here.now`），用于路由访问网站；  
  任何拥有 API 密钥的账户均可申请子域名；  
  相关 API 端点：`/api/v1/handle`；  
  链接生成端点：`/api/v1/links` 和 `/api/v1/links/:location`；  
  路径参数的默认处理方式是 `__root__`。  

## **自定义域名：**  
- 可使用自己的域名（例如：`example.com`）并对其进行配置；  
  需要付费计划（高级账户及以上）。  
  **添加自定义域名：**  
    （代码示例略）  
  包括 DNS 配置指导（指向 `fallback.here.now`）；SSL 会自动配置。  

**域名状态检查：**  
域名状态为 `pending`（等待 DNS 验证和 SSL 启用），验证通过后状态变为 `active`。  

**链接设置：**  
创建链接时需指定域名参数；  
- 空的 `location` 参数会生成默认首页（`https://example.com/`）；  
- 例如：`location="docs"` 会生成 `https://example.com/docs/`。  

**域名列表：**  
可列出所有已使用的自定义域名。  

**注意事项：**  
- 免费账户最多支持 1 个自定义域名；高级账户支持最多 5 个域名；  
- 大多数域名使用 ALIAS 记录（CNAME）进行配置；  
- 子域名（如 `docs.example.com`）也可使用标准 CNAME 记录；  
- 相关 API 端点：`/api/v1/domains` 和 `/api/v1/domains/:domain`。
