# GLM自动路由系统

用于ZAI GLM模型的二进制模型路由系统——适用于轻量级和重量级任务。

## 介绍
1. **GLM-4.7-FlashX** 是默认模型。只有在任务确实需要时，才会启动 **GLM-5**。
2. 使用 `sessions_spawn` 来运行需要使用 **GLM-5** 的任务：
   ```
sessions_spawn({
  task: "<the full task description>",
  model: "zai/glm-5",
  label: "<short task label>"
})
```
3. 完成GLM-5任务后，系统会自动切换回默认的FlashX模型继续处理。

## 模型

### GLM-4.7-FlashX（默认模型 - zai/glm-4.7-flashx）

适用于轻量级任务：
1. 简单的问答（什么、何时、谁、在哪里）
2. 闲聊（无需推理）
3. 快速查询
4. 文件查找
5. 重复性任务或格式化操作
6. 定时任务（如果需要推理，则升级到GLM-5）
7. 状态检查
8. 基本确认
9. 提供简洁的答案，无需解释

**禁止操作：**
- ❌ 禁止使用FlashX进行编码
- ❌ 禁止使用FlashX进行分析
- ❌ 禁止使用FlashX进行任何推理操作
- ❌ 禁止使用FlashX进行研究
- 如果您认为某个请求不属于上述轻量级任务范围，请升级到GLM-5
- 如果您认为自己的操作违反了上述禁止事项，请升级到GLM-5

### GLM-5（zai/glm-5）

适用于重量级任务：
1. 编码（任何复杂度的任务）
2. 分析与调试
3. 多步骤推理
4. 研究与调查
5. 关键决策
6. 架构设计
7. 复杂问题解决
8. 深度研究
9. 重要决策
10. 详细解释

## 示例

| 任务 | 使用的模型 | 原因 |
|------|-------|-----|
| “查看日历” | FlashX | 简单查询 |
| “现在几点了？” | FlashX | 简单问答 |
| “心跳检查” | FlashX | 常规操作 |
| “读取这个文件” | FlashX | 简单查询 |
| “总结内容” | FlashX | 基本任务 |
| “编写Python脚本” | GLM-5 | 编码 |
| “调试这个错误” | GLM-5 | 分析 |
| “研究市场趋势” | GLM-5 | 深度研究 |
| “规划迁移方案” | GLM-5 | 复杂规划 |
| “分析这个问题” | GLM-5 | 分析 |

## 其他注意事项
1. 如果用户请求使用特定模型，请按照其要求执行。
2. 在输出结果中务必注明使用了哪个模型。
3. 完成GLM-5任务后，系统会自动切换回默认的FlashX模型。
4. 如果某个任务不属于FlashX的适用范围，请升级到GLM-5。
5. 如果您认为自己的操作违反了上述禁止事项，请升级到GLM-5。
6. 编码任务必须使用GLM-5。
7. 如有疑问，优先选择GLM-5（安全总比后悔好）。
8. 心跳检查等常规任务始终使用FlashX处理，除非需要复杂分析。
