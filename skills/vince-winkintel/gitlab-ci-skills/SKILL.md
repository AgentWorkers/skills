# GitLab CLI 技能

此技能指向与 GitLab CLI 使用相关的特定子技能（`glab-*`）。

## 可用技能

- `glab-auth`：用于 GitLab CLI 的身份验证
- `glab-alias`：为 GitLab CLI 命令创建别名
- `glab-api`：与 GitLab API 交互的命令
- `glab-attestation`：处理 GitLab 的身份验证和授权
- `glab-changelog`：查看 GitLab 的变更日志
- `glab-check-update`：检查 GitLab 仓库的更新状态
- `glab-ci`：配置和运行持续集成（CI）流程
- `glab-cluster`：管理 GitLab 的集群
- `glab-completion`：为 GitLab CLI 命令提供自动补全功能
- `glab-config`：配置 GitLab 的设置
- `glab-deploy-key`：生成和管理部署密钥
- `glab-duo`：支持 GitLab 的双人协作功能
- `glab-gpg-key`：生成和管理 GitLab 的 GPG 密钥对
- `glab-help`：显示 GitLab CLI 的帮助信息
- `glab-incident`：报告和处理 GitLab 中的问题/事件
- `glab-issue`：创建和跟踪 GitLab 仓库中的问题
- `glab-iteration`：管理 GitLab 问题的迭代过程
- `glab-job`：执行特定的 GitLab 任务
- `glab-label`：为 GitLab 问题添加标签
- `glab-mcp`：管理多仓库配置
- `glab-milestone`：设置和跟踪 GitLab 项目的里程碑
- `glab-mr`：管理合并请求（Merge Request）
- `glab-opentofu`：使用 OpenTelemetry 进行性能监控
- `glab-release`：发布新的 GitLab 仓库版本
- `glab-repo`：操作 GitLab 仓库
- `glab-schedule`：安排 GitLab 仓库的定期任务
- `glab-securefile`：管理安全文件
- `glab-snippet`：提取 GitLab 仓库中的代码片段
- `glab-ssh-key`：管理 SSH 密钥对
- `glab-stack`：查看 GitLab 仓库的代码依赖关系
- `glab-token`：生成和管理 GitLab 的访问令牌
- `glab-user`：管理 GitLab 用户
- `glab-variable`：配置 GitLab 的变量
- `glab-version`：显示 GitLab 的版本信息
