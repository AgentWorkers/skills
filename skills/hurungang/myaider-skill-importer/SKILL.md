# MyAider 技能导入器

## 目的  
自动化将技能从 MyAider MCP 服务器导入到代理技能中的过程。该工具会检索可用的技能，允许用户选择要导入的技能，并使用现有的技能创建工具为每个技能生成相应的技能文件。

## 必须遵循的工作流程  

### 第 0 步 — 必须完成：查找 MyAider MCP 服务器名称并检查技能创建工具  
**命名规则说明：** MCP 工具的标识符遵循 `mcp__<服务器名称>__<工具名称>` 的格式。服务器名称是用户在配置 MCP 时选择的，可能不是 `myaider`。务必获取实际的服务器名称，而不能直接假设为 `myaider`。  

MyAider MCP 服务器提供了一个名为 `get_myaider_skills` 的独特工具。由于该名称在 MyAider 中是唯一的，因此使用这个名称可以避免与其他 MCP 服务器的冲突。  

#### 第 A 阶段 — 查找服务器名称  
在可用的工具中搜索名称为 `get_myaider_skills` 的工具。使用您的代理支持的任何工具发现机制（例如，列出可用工具、按名称搜索）。完整的工具标识符形式为 `mcp__<服务器名称>__get_myaider_skills`。  
从工具标识符中提取服务器名称并存储为 `{SERVER_NAME}`。后续的所有调用都使用 `mcp__{SERVER_NAME}__get_myaider_skills`（以及 `mcp__{SERVER_NAME}__get_myaider_skill_updates`）。  

#### 第 B 阶段 — 根据搜索结果进行处理  
- **找到 1 个匹配项** → 从工具标识符中提取 `{SERVER_NAME}`，直接进入第 1 步。  
- **未找到匹配项** → 通知用户需要先设置 MyAider MCP：  
  > 显然，您的 MyAider MCP 服务器尚未配置。要使用此技能，您需要先设置 MyAider MCP 服务器。  
  > **设置说明：**  
  > 1. 访问 https://www.myaider.ai/mcp  
  > 2. 按照说明为您的代理配置 MyAider MCP 服务器  
  > 3. 配置完成后，请再次联系我以导入您的 MyAider 技能  
  > 在用户确认 MyAider 已配置之前，不要继续执行后续操作。  

- **找到 2 个或更多匹配项** → 列出所有找到的服务器名称，并询问用户哪个是他们的 MyAider 实例。将用户的回答作为 `{SERVER_NAME}`。  
检查技能创建工具是否可用；如果不可用，请提示用户安装该工具。  

### 第 1 步 — 必须完成：获取可用技能  
使用在第 0 步中获取的服务器名称，调用 `mcp__{SERVER_NAME}__get_myaider_skills`（传入一个空对象 `{}`），以检索 MyAider 中的所有可用技能。  

### 第 2 步 — 向用户展示技能列表  
向用户展示技能列表及其描述，并询问他们选择：  
- “全部” — 导入所有技能  
- 或者指定他们想要导入的具体技能（通过名称）  
等待用户的确认后再继续执行。  

### 第 3 步 — 对于每个选定的技能  
对于用户想要导入的每个技能：  
1. **从 `getSkills` 的结果中提取技能信息：**  
   - 技能名称  
   - 描述（来自使用说明或摘要）  
   - 使用说明（主要内容）  
   - **相关工具的完整信息**：从 `getSkills` 的结果中的 “Tools” 部分提取每个工具的名称、描述和参数规范  

2. **使用技能创建工具生成格式正确的技能文件：**  
   必须自动生成技能文件，而不能让用户手动操作。必须使用 `skill-creator` 工具，并按照以下模板执行：  

   ```  
   创建一个名为 “[技能名称]”的新技能，基于以下信息：  

   ## 技能名称  
   [技能名称]  

   ## 描述  
   [提供详细的描述以及使用指南]  

   ## 元数据  
   在技能的 YAML 文件头中添加以下字段（除了名称和描述）：  
   - source: myaider  
   - updated_at: [来自远程技能的 ISO 8601 时间戳，例如 2026-03-06T12:00:00Z]  

   ## 使用说明  
   [来自 MyAider 技能的完整使用说明]  

   ## 相关工具（MCP {SERVER_NAME}）**  
   该技能使用了 {SERVER_NAME} 中的以下 MCP 工具。请包含完整的工具描述和参数规范，以优化令牌使用效率——技能不应依赖 MCP 协议来获取工具信息：  

   ### [工具名称-1]  
   [来自 get_myaider_skills 结果的完整工具描述]  

   **参数：**  
   [参数规范——包括所有参数及其类型、是否必填以及描述]  

   ### [工具名称-2]  
   [来自 get_myaider_skills 结果的完整工具描述]  

   **参数：**  
   [参数规范——包括所有参数及其类型、是否必填以及描述]  
   ```  

   **重要提示**：必须直接在技能文件中包含提取的工具描述和规范，以避免额外的 MCP 协议开销。这样可以使技能在无需调用 MCP 协议的情况下正常运行。  

3. **在每个技能创建完成后，向用户确认**  

### 第 4 步 — 总结  
所有选定的技能创建完成后，提供以下信息：  
- 成功创建的技能列表  
- 文件位置  
- 失败的技能（如果有）

## 升级工作流程  
当用户请求 **升级**、**更新** 或 **同步** 他们的 MyAider 技能时，触发此工作流程。  

### 升级步骤 0 — 查找服务器名称  
与主流程的第 0 步相同。搜索 `get_myaider_skills` 以获取 `{SERVER_NAME}`。如果 MCP 未配置，显示设置说明并停止。  

### 升级步骤 1 — 获取远程更新信息  
调用 `mcp__{SERVER_NAME}__get_myaider_skill_updates`（传入一个空对象 `{}`）。这将返回最新的技能定义及其 `updated_at` 时间戳。  

### 升级步骤 2 — 读取本地 MyAider 技能  
查找所有在 YAML 文件头中包含 `source: myaider` 的本地安装技能。对于每个技能，读取其 `updated_at` 值，并构建一个 `skill-name → local_updated_at` 的映射。  

### 升级步骤 3 — 比较和分类  
对于升级步骤 1 中返回的每个技能：  
- **远程的 `updated_at` 更新时间晚于本地的** → 标记为 **需要升级**  
- **本地没有该技能** → 标记为 **需要新安装**  
- **远程的 `updated_at` 与本地相同或更早** → 跳过（已是最新的）  
向用户展示分类结果（哪些需要升级、哪些是新添加的、哪些已经是最新的），并在继续之前获取用户的确认。  

### 升级步骤 4 — 升级过时的技能  
对于标记为 **需要升级** 的每个技能，使用完整的更新信息（与主流程第 3 步相同的模板，包括更新的 `updated_at` 和工具规范）调用 `skill-creator:skill-creator`。`skill-creator` 会覆盖现有的技能文件。  

### 升级步骤 5 — 安装新技能  
对于标记为 **需要新安装** 的每个技能，按照主流程第 3 步中的方法调用 `skill-creator`。  

### 升级步骤 6 — 总结  
提供最终报告：  
- 升级的技能（名称 + 旧版本 → 新版本的 `updated_at`）  
- 新安装的技能  
- 已经是最新的技能  
- 失败的操作  

---

## 重要限制  
- 必须始终通过搜索 `get_myaider_skills` 来查找 MCP 服务器名称（第 0 步）——不要硬编码 `myaider` 或其他名称  
- 在确认 MCP 已配置后，再调用 `get_myaider_skills` —— 不要猜测哪些技能是可用的  
- **必须从 `get_myaider_skills` 中提取并包含完整的工具描述和规范**——这对于优化令牌使用效率至关重要。生成的技能应能够在不依赖 MCP 协议的情况下正常运行  
- 在所有 MCP 工具调用和生成的技能文件中始终使用已获取的 `{SERVER_NAME}`  
- 在每个创建或升级的技能的 YAML 文件头中始终包含 `source: myaider` 和 `updated_at` —— 这些字段是升级流程所必需的  
- 在创建或升级技能之前，必须等待用户的确认  
- 使用 `skill-creator` 一次只创建/升级一个技能  
- 在每次创建技能时，确保与用户进行清晰的沟通  

## 示例用法：  
- “导入我的 MyAider 技能”  
- “从 myaider 创建技能”  
- “设置我的 MyAider MCP 中的技能”  
- “升级我的 MyAider 技能”  
- “将我的 MyAider 技能更新到最新版本”  
- “同步我的 MyAider 技能”
