# 记忆管理技巧

使用严格的双层模型来组织记忆信息。

## 核心规则

将 `MEMORY.md` 视为一个小型、永久性的索引文件。

- 不要将所有带有日期的记忆文件都复制到 `MEMORY.md` 中。
- 不要将每日日志转化为永久性记忆记录。
- 除非某个信息具有长期价值且需要在每个新会话中都显示，否则将其保存在 `memory/YYYY-MM-DD.md` 文件中。

## 记忆模型

### 1. 永久性记忆：`MEMORY.md`

仅保留那些在每个会话中都值得查看的信息：
- 稳定的用户偏好设置
- 持久性的身份或关系信息
- 长期有效的项目配置
- 重要的工作区规则
- 仍然有效的跨会话待办事项
- 可重复使用的操作性信息（这些信息在未来可能会再次用到）

该文件应保持简洁、无重复内容，并便于快速浏览。

### 2. 日常记忆：`memory/YYYY-MM-DD.md`

在此文件中保存特定于会话的详细信息：
- 每日工作日志
- 中间推理过程摘要
- 一次性实验结果
- 临时调试笔记
- 每日的成果和进度记录
- 以后可能有用的详细记录（但不是每个会话都必需的）

在压缩或整理记忆信息时，应保留这些文件，而不是将所有内容都转移到永久性记忆中。

## 决策规则：是否需要将信息转移到 `MEMORY.md` 中

在将任何内容写入 `MEMORY.md` 之前，请问自己：
- 这些信息在未来多个会话中是否仍然重要？
- 是否需要在会话开始时立即加载这些信息？
- 这些信息是否足够稳定，不会很快导致文件变得混乱？

只有当答案明确为“是”时，才将信息转移到 `MEMORY.md` 中。

如果某项信息主要用于记录历史、提供上下文或作为审计追踪，应将其保存在带有日期的文件中。

## 应该转移到 `MEMORY.md` 中的内容

仅转移以下简洁明了的信息：
- 用户偏好的文本格式（短段落、简洁的表达）
- 必须始终遵守的工作区路径或权限规则
- 影响未来执行的当前活跃项目规则
- 持续多天的待办事项

## 不应该转移到 `MEMORY.md` 中的内容

默认情况下，不要转移以下内容：
- 完整的每日总结
- 每日的变更日志
- 详细的任务执行记录
- 调试过程记录
- 重复出现的笔记
- 已经包含在 `MEMORY.md` 中的信息
- 所有已发布的成果或状态检查结果

## 压缩工作流程

当用户要求压缩或整理记忆信息时，请按照以下步骤操作：
1. 分别扫描 `MEMORY.md` 和带有日期的记忆文件。
2. 首先清理并删除 `MEMORY.md` 中的重复内容。
3. 直接压缩带有日期的文件。
4. 保留 `memory/YYYY-MM-DD.md` 文件中的有用历史记录。
5. 仅将少量持久性信息转移到 `MEMORY.md` 中。
6. 如果不确定某项信息是否属于永久性内容，将其保留在原始文件中。

## 合并工作流程

如果使用合并或迁移操作，请注意：
- 选择性地合并内容，而不是全部合并。
- 仅提取持久性信息。
- 跳过每日总结、日志和冗长的标题。
- 避免为每个日期创建重复的章节。
- 更倾向于更新现有的永久性章节，而不是添加新的基于日期的章节。

## 质量标准

`MEMORY.md` 应满足以下要求：
- 体积足够小，以便在启动时快速加载
- 按照持久性主题进行结构化，而不是按日期排序
- 不包含无关的历史信息
- 仅包含必须了解的信息

如果某个选项会导致 `MEMORY.md` 的体积增大，但并不会提高其可重用性，那么将该内容保留在带有日期的文件中。

## 安全性注意事项：
- 在进行文件操作之前，请验证文件路径的合法性。
- 仅在工作区的记忆文件范围内进行操作。
- 在执行破坏性压缩之前，请先创建备份。
- 在删除或丢弃记忆信息时，尽可能保留可恢复的备份。
