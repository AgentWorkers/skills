# 终端（Terminal）

本地 shell 的辅助工具，帮助用户更清晰地规划操作并谨慎地执行命令。

## 核心理念（Core Philosophy）：
1. 将用户的意图转化为可执行的 shell 命令。
2. 在对系统进行任何修改之前，优先提供预览和检查功能。
3. 对具有破坏性的操作，必须获得用户的明确确认。
4. 执行命令后，用通俗的语言总结操作结果。

## 运行时要求（Runtime Requirements）：
- 必须能够使用 `python3`。
- 本地环境中应包含标准的 shell 工具。
- 不需要任何外部包。

## 安全模型（Safety Model）：
- 仅在本地环境中执行操作。
- 不会请求用户的任何外部凭证。
- 不会进行任何隐藏的网络活动。
- 对于具有破坏性的操作，必须获得用户的明确确认。
- 首先推荐使用只读检查功能（如 `ls`、`find`、`cat`、`grep`、`pwd`、`file`）。
- 高风险命令在执行前会进行提示：
  - `rm`、`mv`、`chmod`、`chown`
  - 递归删除或修改文件的操作
  - 会覆盖文件的 shell 重定向操作
  - 杀死进程的命令
  - 安装或卸载软件包的命令

## 数据存储（Storage）：
所有本地数据仅存储在以下路径：
- `~/.openclaw/workspace/memory/terminal/history.json`

不进行云同步，也不使用任何第三方 API 或遥测功能。

## 工作流程（Workflows）：
- **规划命令（Plan Command）**：将用户的意图转化为安全的 shell 命令建议。
- **预览风险（Preview Risk）**：在执行命令前解释其可能产生的影响。
- **执行命令（Execute）**：运行命令并捕获标准输出（stdout）和标准错误（stderr）。
- **总结结果（Summarize）**：用通俗的语言解释命令的执行结果。
- **记录历史（History）**：将最近的命令操作记录在本地。

## 脚本（Scripts）：
| 脚本名称 | 功能 |
|---|---|
| `init_storage.py` | 初始化本地终端操作历史记录的存储 |
| `plan_command.py` | 根据用户的意图生成 shell 命令 |
| `run_command.py` | 在执行命令前进行安全检查 |
| `summarize_result.py` | 用通俗的语言总结命令的输出结果 |
| `show_history.py` | 显示最近的命令操作记录 |
