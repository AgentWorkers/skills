# Planner：将不确定性转化为可行的计划。

## 核心理念
1. 计划应减少不确定性，而非增加额外的工作负担。
2. 首先明确各种约束条件，然后再制定计划。
3. 大型计划应被分解为多个阶段、里程碑和具体的下一步行动。
4. 良好的计划制定能在执行开始前带来清晰的方向。

## 运行时要求
- 必须安装 Python 3，并确保它可以作为 `python3` 可用。
- 不需要任何外部包。

## 数据存储
所有数据仅存储在本地，路径如下：
- `~/.openclaw/workspace/memory/planner/plans.json`
- `~/.openclaw/workspace/memory/planner/archive.json`

不支持外部同步，也不使用云存储或第三方规划 API。

## 计划类型
- `trip`：旅行计划，包括行程安排、预订顺序和预算管理。
- `week`：每周计划，包含主题、时间安排和优先级。
- `project`：多阶段的路线图，包含里程碑。
- `launch`：项目启动计划，包括准备工作和时间表。
- `decision`：在各种选项中根据约束条件和权衡进行选择。

## 核心字段
- `goal`：用户想要实现的目标。
- `constraints`：预算、时间、精力、截止日期和依赖关系等限制因素。
- `phases`：计划的宏观阶段。
- `milestones`：具体的检查点。
- `next_steps`：需要立即执行的行动。
- `notes`：额外的规划信息。

## 主要工作流程
- **制定计划**：`add_plan.py`
- **查看最佳计划结构**：`suggest_plan.py`
- **更新计划**：`update_plan.py`
- **汇总当天的计划内容**：`daily_plan.py`
- **查看所有活跃的计划**：`weekly_map.py`

## 脚本说明
| 脚本 | 功能 |
|---|---|
| `init_storage.py` | 初始化本地存储文件 |
| `add_plan.py` | 创建新的计划 |
| `suggest_plan.py` | 显示当前的最佳计划或下一个执行阶段 |
| `update_plan.py` | 更新计划的状态、日期、备注或阶段信息 |
| `daily_plan.py` | 显示活跃的计划及接下来的行动步骤 |
| `weekly_map.py` | 查看所有计划的内容和进度
