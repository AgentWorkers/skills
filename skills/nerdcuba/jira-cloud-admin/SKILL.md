# Jira Admin Cloud（第一阶段）

## 本技能的作用
- 提供**Jira Cloud**管理的实用指南（包括JSM和Confluence的相关内容）。
- 提供模块检查清单及官方参考资料。
- 根据用户需求，以**英语/西班牙语**进行回复。

## 推荐的操作流程（快速）
1. 确定**项目类型**：公司管理型还是团队管理型。
2. 明确用户的目标（权限、工作流程、字段设置、自动化功能、JSM配置、报告需求）。
3. 查阅`references/`目录中的**模块检查清单**。
4. 如需更多详细信息，请查阅Atlassian的官方文档。

## 模块与参考资料
- **Atlassian基础知识** → `references/atlassian-cloud-links.md`
- **检查清单** → `references/checklists.md`
- **JSM（Jira Service Management）** → `references/jsm-links.md`
- **Confluence** → `references/confluence-links.md`
- **专家资源** → `references/experts-links.md`

## 关键检查清单（概述）
- **权限设置**：角色与权限的关系、权限方案、用户组、权限粒度。
- **工作流程**：状态转换规则、验证器、后续操作、条件设置。
- **字段与界面**：字段的用途、界面设计、必填字段、对项目的影响。
- **自动化功能**：项目级/全局自动化规则、使用限制、审计机制。
- **JSM**：请求类型、服务级别协议（SLAs）、任务队列、项目管理门户。
- **报告功能**：共享筛选条件、仪表盘、数据可视化工具。

## 注意事项
- 请始终优先处理**Jira Cloud**的相关内容（而非数据中心版本）。
- 请以Atlassian的官方文档作为确认依据。
- 回答内容应简洁明了且具有实际操作指导意义。
