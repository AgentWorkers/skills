# 自适应技能工厂

使用此工具来保持代理（agents）的效率，同时避免因重复使用技能而导致系统臃肿。

## 工作流程

1. **分析需求**：
   - 对于一次性、简单的任务，无需创建新的技能。
   - 对于重复出现的工作流程、敏感的操作或特定领域的任务，可重用现有的技能。
   - 只有当相同的需求可能频繁出现时，才创建新的技能。

2. **优先查找现有技能**：
   - 首先查看`<workspace>/skills`目录中的工作区技能。
   - 然后检查从配置的额外技能目录中加载的共享技能。
   - 如果现有技能能够满足需求，直接重用它们。

3. **利用网络资源**：
   - 如果网络连接和ClawHub可用，在创建新技能之前先进行搜索。
   - 精准搜索与当前工作流程完全匹配的技能。
   - 安装最精简、最相关的技能，而非包含大量功能的技能包。
   - 避免安装与其它角色职责重叠的技能。

4. **创建本地技能**：
   - 将特定于角色的技能保存在`<workspace>/skills/<skill-name>`目录中。
   - 将跨角色可重用的技能保存在`D:\internal-hub\skills`目录中。
   - 保持`SKILL.md`文件简洁明了，仅记录必要的步骤和内容。
   - 仅在确实能节省工作量的情况下，才包含脚本、参考资料或相关资源。

5. **验证与优化**：
   - 编辑技能内容后进行验证。
   - 当工作流程发生变化或代理遇到问题时，及时更新技能。
   - 删除不再有用的旧技能或简化其内容。

## 角色职责划分

- **技术技能**：仅用于代码编写、部署、自动化、调试、网站开发及脚本编写。
- **智能技能**：仅用于研究、数据验证、政策分析、新闻收集或竞争对手分析。
- **贸易技能**：仅用于供应商管理、价格谈判、最小订购量（MOQ）处理、交货期计算、采购流程、库存管理、加密货币分析及金融交易分析。
- **管理技能**：仅用于任务分配、进度报告、审核及执行监督。

## 避免的事项

- **避免创建通用性过强的技能**。
- **避免创建与其它角色职责重复的技能**。
- **避免安装无关的技能包**，即使它们可用。
- **避免将技能文件用作存储杂项信息的工具**。
