# 宠物看护服务客户信息表生成器

该工具专为宠物看护服务企业生成专业、精美的PDF信息表。用户只需提供最少的输入信息，即可获得一份完整、符合企业品牌形象的PDF文件。

## 适用场景

当用户有以下需求时，请使用此工具：
- 需要创建宠物看护服务的客户信息表；
- 希望制作新的客户问卷或入职表格；
- 需要为业务准备宠物信息表；
- 希望在接待新客户时展现出专业形象；
- 需要宠物寄养服务表、遛狗服务表或临时看护服务检查表。

## 主要功能

| 功能 | 详细描述 |
|---------|-------------|
| **可填写的PDF字段** | 提供交互式文本字段和复选框，支持在线填写 |
| **7种颜色主题** | 可选择薰衣草色、海洋蓝、森林绿、玫瑰粉、日落橙、中性灰和午夜黑 |
| **多宠物支持** | 可生成包含1至10个宠物信息的表格 |
| **服务模板** | 为宠物寄养、遛狗和临时看护服务提供专用模板 |
| **家庭访问信息** | 包含门禁密码、报警设置、WiFi信息及停车指南（适用于上门看护服务） |
| **配置文件支持** | 支持使用YAML格式保存企业设置以供重复使用 |

## 第一步：收集用户信息

请用户提供以下信息：
1. **企业名称**（必填）——例如：“Happy Paws Pet Sitting” |
2. **看护人员姓名**（可选）——会显示在授权信息中 |
3. **提供的服务**——宠物看护、遛狗、寄养、临时看护等 |
4. **服务地点**——用于表格标题的城市/州 |
5. **联系方式**——用于页脚的电子邮件、电话或网站 |
6. **颜色主题**（可选）——从7种可选主题中选择 |
7. **服务类型**（可选）——普通服务、寄养服务或遛狗服务 |
8. **宠物数量**（可选）——如果客户有多只宠物 |

如果某些信息缺失，可使用默认值或保留空白字段。

## 第二步：生成PDF文件

```bash
python scripts/generate_form.py \
  --business-name "Happy Paws Pet Sitting" \
  --sitter-name "Jane Smith" \
  --services "Dog Walking, Boarding, Drop-in Visits" \
  --location "Austin, TX" \
  --contact "jane@happypaws.com | (512) 555-1234" \
  --theme ocean \
  --service-type boarding \
  --pets 2 \
  --output "/mnt/user-data/outputs/client_intake_form.pdf"
```

### 命令行参数说明

| 参数 | 详细描述 | 默认值 |
|--------|-------------|---------|
| `--business-name` | 用于表格标题的企业名称 | "Your Business Name" |
| `--sitter-name` | 用于授权信息的看护人员姓名 | （空） |
| `--services` | 服务类型（逗号分隔） | "Pet Sitting, Dog Walking, Boarding" |
| `--location` | 服务地点（城市/州） | （空） |
| `--contact` | 联系方式 | （空） |
| `--theme` | 颜色主题 | 薰衣草色（默认） |
| `--service-type` | 服务类型（普通/寄养/遛狗） | "general" |
| `--pets` | 宠物数量（1-10） | 1 |
| `--fillable` | 是否启用可填写的PDF字段 | （默认启用） |
| `--no-fillable` | 生成仅用于打印的表格 | （无） |
| `--no-home-access` | 省略家庭访问信息部分 | （无） |
| `--config` | 配置文件路径 | （无） |
| `--output` | 输出PDF文件名 | client_intake_form.pdf |
| `--list-themes` | 显示可用主题并退出 | （无） |

### 使用配置文件

为重复使用，创建一个`config.yaml`文件：

```yaml
business_name: "Happy Paws Pet Sitting"
sitter_name: "Jane Smith"
services: "Dog Walking, Boarding, Drop-in Visits"
location: "Austin, TX"
contact: "jane@happypaws.com"
theme: ocean
service_type: general
num_pets: 1
fillable: true
```

然后使用以下命令生成PDF文件：

```bash
python scripts/generate_form.py --config config.yaml --output client_form.pdf
```

## 第三步：向用户展示结果

生成完成后，通过`present_files`功能将PDF文件分享给用户，并告知他们：
- 表格包含可填写字段，客户可在线填写；
- 表格也可用于打印，方便手写填写；
- 可根据企业品牌自定义颜色主题（使用`--list-themes`参数）；
- 如有宠物需要照顾，请使用`--pets 2`或更多参数来生成相应数量的宠物信息页；
- 建议用户保存配置文件以方便后续使用。

## 表格内容说明

### 第1页——宠物主人信息
- 主人姓名、地址、电话、电子邮件
- 紧急联系人（姓名、关系、电话）
- 养殖医生信息及24小时紧急联系电话
- 被授权的接宠物人员
- 沟通偏好（更新频率、联系方式）

### 第2页——家庭访问信息（可选，默认包含）
- 进入方式（锁箱密码、车库门禁码）
- 报警设置及解除警报方法
- WiFi网络和密码
- 停车指南
- 温控系统/暖通空调设置
- 禁止进入的区域和家庭规则

### 第3页——宠物信息
- 宠物名称、品种、年龄、体重、毛色
- 性别（及绝育情况）
- 微芯片和许可证编号
- 防蚤/蜱虫预防措施

### 第4页——疫苗接种信息
- 疫苗接种情况（通过复选框选择）：狂犬病、DHPP、犬瘟热、FVRCP、FeLV、犬流感
- 疫苗有效期（可填写）
- 其他需要接种的疫苗（可选）

### 第5页——健康与行为信息
- 宠物过敏情况、健康状况
- 药物信息（名称、剂量、使用频率、使用说明）
- 行为评估：对陌生人/儿童/其他狗/猫的反应
- 宠物的恐惧源、分离焦虑程度
- 注意事项（如逃跑行为）
- 知晓的指令、召回能力
- 如何进行如厕训练及训练计划
- 睡眠地点、宠物笼子使用情况

### 第6页——喂养与日常护理
- 食物品牌/类型、喂食量、储存位置
- 喂食时间表（通过复选框选择）
- 是否允许喂零食
- 宠物的运动需求

### 第7页——特定服务信息（非通用服务）
- **寄养服务**：送养/接养时间、所需物品清单
- **遛狗服务**：遛狗时的行为要求、路线偏好、遛狗时长
- **临时看护服务**：每次看护时的任务清单

### 第8页——授权与协议
- 紧急医疗护理授权
- 运输授权
- 允许使用宠物照片或发布到社交媒体的授权
- 取消服务政策确认
- 责任声明
- 客户签名、填写日期、打印姓名
- 办公场所使用说明

## 可用颜色主题

| 主题 | 描述 |
|-------|-------------|
| `lavender` | 薰衣草色与桃色搭配 |
| `ocean` | 宁静的蓝色与沙黄色 |
| `forest` | 自然的鼠尾草绿与金色 |
| `rose` | 温暖的粉色与棕褐色 |
| `sunset` | 珊瑚橙色与天蓝色 |
| `neutral` | 专业的灰色与柔和的蓝色 |
| `midnight` | 高雅的板岩色与温暖的金色 |

运行`--list-themes`命令可查看所有颜色选项的代码。

## 系统要求

- Python 3.8及以上版本 |
- `reportlab`库（用于PDF生成） |
- `pyyaml`库（可选，用于配置文件）

## 安装依赖库

```bash
pip install reportlab pyyaml
```

## 使用建议

- 对于优先采用数字化工作流程的情况，建议保持“可填写字段”功能启用（默认设置），以便客户可以通过手机或电脑填写表格；
- 如果客户需要纸质表格，可以使用`--no-fillable`参数；
- 如果家中有多只宠物，请使用`--pets 3`参数生成相应的宠物信息页；
- 提供寄养服务的商家，请使用`--service-type boarding`参数；
- 提供遛狗服务的商家，请使用`--service-type walking`参数；
- 建议用户保存配置文件，以便每次生成表格时保持一致性。
