# SKILL.md - 外部潜在客户开发（Outbound Prospecting）

本技能提供了一种结构化的流程，用于研究潜在客户并主动与其建立联系。该流程遵循 `MEMORY.md` 中规定的“Overseer Baseline”标准，并且仅在协同工作（copilot）模式下执行。

## 工作流程

该流程专为针对单一目标公司域名或名称而设计。

### 1. 研究与审计 (`research_auditor`)

- **目标：**收集关于目标公司的信息，了解其业务情况，识别潜在需求，并找出可进行个性化沟通的切入点。
- **操作步骤：**
    - 使用 `web_search` 查找公司的官方网站、服务、产品及最新新闻。
    - 对关键页面（首页、关于我们、博客）使用 `web_fetch` 功能提取详细内容。
    - 分析提取到的信息，以确定：
        - 公司的核心业务范围。
        - 目标受众。
        - 公司的最新动态（产品发布、融资、新员工招聘等）。
        - 可能存在的问题（例如：网站过时、搜索引擎优化不佳、近期没有发布博客文章等）。

### 2. 确定决策者 (`contact_finder`)

- **目标：**找到公司内最合适的联系人。
- **操作步骤：**
    - 使用 `web_search` 进行搜索，例如：
        - `site:linkedin.com [公司名称] CEO`
        - `site:linkedin.com [公司名称] 创始人`
        - `site:linkedin.com [公司名称] 市场部负责人`
    - 确定主要决策者的姓名和职位。

### 3. 起草联系邮件 (`draft_writer`)

- **目标：**撰写一封个性化且基于事实的邮件，供人工审核。
- **操作步骤：**
    - 整理所有收集到的信息。
    - 起草一封邮件，采用“先建立关系再提出问题”的沟通方式，并结合实际数据（如收入损失等）来支持论点。
    - 邮件中必须包含至少三条在研究阶段发现的公司具体信息。
- 最终输出应是一个 markdown 块，其中包含起草好的邮件内容，可供代理复制使用。邮件本身不应由代理直接发送。
