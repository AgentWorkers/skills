# Semantic Hub

**简要概述**  
Semantic Hub 是一款专为人类与 AI 代理协同工作设计的任务和项目管理工具。您可以通过可视化的看板（Kanban）来组织工作（支持拖放功能，分为“待办”、“进行中”和“已完成”等状态），也可以通过表格格式查看任务，或使用日历进行计划安排——选择最适合您的方式即可。AI 代理可通过 REST API 与共享的看板进行交互：它们可以列出看板、创建和更新任务、添加评论、上传文件并将其附加到卡片或评论上，还能过滤和搜索工作项，并查看活动历史记录。代理在操作时遵循基于角色的访问控制机制，仅作为团队成员使用相应权限。

**看板管理**  
看板所有者可以在仪表板中创建看板，然后为每个代理生成 API 密钥，并授予其访问特定看板的权限。代理无权删除卡片、修改设置或调整成员信息。

## 认证  
使用 `SEMANTIC_HUB_API_KEY` 是必须的。该密钥由看板所有者在 https://dashboard.simplysemantics.com/hub 中生成，用于验证代理的请求。该密钥的权限仅限于特定的看板，不会授予代理未获授权的访问权限。看板所有者可以随时撤销或更换该密钥。

## 隐私与数据保护**  
- **传输的数据**：仅包括卡片数据（标题、描述、评论、状态变更）以及看板元数据。不会传输任何用户的个人身份信息（PII）、代理的秘密信息、源代码或环境变量。  
- **不传输的数据**：不会传输用户的凭据、私钥、环境变量、文件内容或任何形式的个人信息。  
- **数据存储**：看板和卡片数据会存储在用户的账户中，并且每个账户和每个看板的数据都是独立存储的。  
- **API 密钥管理**：您的 `SEMANTIC_HUB_API_KEY` 仅用于请求认证，不会以明文形式被记录、共享或传输给第三方。  
- **数据隔离**：Semantic Hub 的数据不会与其他 Simply Semantics 服务共享。  
- **日志记录**：API 请求日志仅用于限制请求频率和防止滥用。日志中包含 API 密钥的哈希值（而非密钥本身）、端点地址和时间戳，保留时间为 30 天。

**使用场景**  
在以下情况下激活 **Semantic Hub**：  
- 用户或代理需要创建、更新或管理看板上的任务/卡片/会议。  
- 需要查看卡片的状态、按优先级/状态/分配者筛选任务，或列出所有任务。  
- 需要“添加卡片”、“创建任务”、“创建会议”、“提出想法”、“更新状态”、“将任务标记为已完成”、“查看我的看板内容”或“查看我们的议程”。  
- 需要添加评论或查看卡片上的评论。  
- 需要上传文件或将文件/URL 附加到卡片或评论上。  
- 需要“附加此文件”、“上传截图”或“将此链接添加到卡片”。  
- 需要查看卡片的活动历史记录。  
- 需要列出看板成员或查看任务分配情况。  
- 需要筛选卡片（例如：“显示所有高优先级任务”或“查看分配给我的任务”）。  

**禁止使用场景**：  
- 创建或删除看板（仅限管理员操作，代理无权执行）。  
- 管理看板设置、列类型或卡片类型（仅限管理员操作）。  
- 添加或删除团队成员（仅限管理员操作）。  
- 管理其他代理或 API 密钥（仅限管理员操作）。  
- 删除卡片（代理无法执行此操作）。

响应内容：（上传成功后的文件信息）  

**上传限制**：  
- **免费账户**：每文件 2 MB，总共 10 次上传  
- **专业账户**：每文件 10 MB，总共 100 次上传  
- **企业账户**：每文件 50 MB，无上传次数限制

- `413`：文件过大，超出您的账户等级限制。  
- `429`：达到您的账户等级的上传次数限制。

**异常情况**：  
- `401/403`：缺少或无效的 `SEMANTIC_HUB_API_KEY`。请设置相应的环境变量以使用此功能。  
- `403`（带有 `upgrade_url`）：达到账户等级限制，请升级您的计划。  
- `404`：未找到卡片或看板，或代理无访问权限。  
- `413`：文件过大，超出您的账户等级限制。  
- `429`：达到上传次数限制。  
- `500`：服务暂时不可用，请稍后再试。
