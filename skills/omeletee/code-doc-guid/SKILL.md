# 代码文档与导航指南

本技能提供了一个**代码库知识图谱**，帮助您在无需阅读数千个文件的情况下浏览复杂的项目。它使用本地的 SQLite 数据库来存储文件之间的关系、依赖关系以及符号定义。

## ⚠️ 重要提示：必须执行的操作（触发条件）

**请在继续之前阅读此内容！**

在执行以下任何操作之前，**必须**先调用此技能（`code-doc-guid`）作为**前置步骤**。未遵守此规则将视为违反协议。

### 🔴 必须执行的预操作（侦察阶段）
**触发条件**：您即将：
*   修改任何代码文件（`Write`、`SearchReplace`）。
*   重命名或移动文件（`RunCommand: mv/ren`）。
*   删除文件（`DeleteFile`）。
*   重构类、函数或变量。

**操作**：**必须**先运行 `python ... codebase_manager.py inspect/search`。
**原因**：在不知道哪些部分依赖于某段代码的情况下，直接修改代码是危险的。严禁“猜测”依赖关系。

### 🟢 必须执行的后续操作（同步阶段）
**触发条件**：您刚刚完成了一组文件修改。
**操作**：**必须**运行 `python ... codebase_manager.py update`。
**原因**：知识图谱需要与磁盘上的文件状态保持一致，以便后续操作能够正确进行。

## 工作流程与命令

所有命令均通过 `python scripts/codebase_manager.py <command> [args]` 来执行。
*（注意：如果手动运行，请调整脚本的保存路径。）

### 1. `search` - 查找符号和概念
使用此命令按名称或功能（文档字符串）查找代码。
*   **命令**：`search <query>`
*   **示例**：`search "UserAuth"`
*   **输出**：以 JSONL 格式显示文件路径、符号名称和文档字符串。
*   **操作建议**：在读取文件之前，先使用此命令找到**确切的文件路径**。

### 2. `inspect` - 分析依赖关系与风险
使用此命令了解修改代码可能带来的影响。
*   **命令**：`inspect <filename_fragment>`
*   **示例**：`inspect "auth_service"`
*   **输出（JSON）**：
    *   `risk_score`：**高** / **中** / **低**。
    *   `doc_file`：包含详细报告的文件路径（例如：`.trae/codeguiddoc.md`）。
*   **操作指南**：
    1. **首先阅读 JSON 摘要**。
    2. **如果风险等级为**高**：
        * **立即停止**修改代码。
        * **阅读**生成的 Markdown 文件（`doc_file`）。其中包含 Mermaid 图表和完整的依赖关系列表。
        * **向用户展示** Mermaid 图表（如果可能的话），或简要说明影响范围：“此修改会影响 25 个文件，包括核心模块 A 和 B。”
        * **请求确认**是否继续修改。

### 3. `update` - 刷新索引
在任何文件修改后，都需要执行此操作。
*   **命令**：`update`
*   **输出**：“正在处理 X 个已修改的文件...”`
*   **注意**：此操作是增量式的，执行速度非常快（通常不到 1 秒）。请在完成修改后立即运行此命令。

### 4. `graph` - 导出可视化图表（可选）
仅当用户明确要求时使用此命令查看代码架构概览。
*   **命令**：`graph`
*   **输出**：生成 `.trae/architecture_layers.md`（人类可读格式）和 `.trae/dependency_graph.json`（机器可读格式）。

## 令牌经济策略

为了节省令牌和时间：
1. **不要为了查找一个函数而阅读整个文件**。使用 `search` 命令，并设置 `limit=20, offset=N` 来限制搜索范围。
2. **不要猜测依赖关系**。使用 `inspect` 命令查看详细的依赖关系图。
3. **遵循层级结构**：如果您修改的是**第 0 层**的文件，请格外小心——因为这可能会影响整个系统；而修改**第 5 层**的文件通常较为安全。
