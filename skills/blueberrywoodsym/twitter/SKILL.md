# **Tweet Writer 技能**

## 概述

该技能可帮助您撰写具有病毒式传播力的、具有说服力的推文和话题串，这些内容经过优化，能够更好地适应 X 的算法。它结合了经过验证的文案写作框架、病毒式传播的钩子公式以及实时研究方法，让您能够根据自己所在领域的成功案例来创作内容。

**关键词**：Twitter、X、推文、话题串、病毒式内容、社交媒体、互动性、钩子、文案写作

## 工作流程

### 第一阶段：领域研究（至关重要）

在撰写任何推文之前，您必须研究用户所在领域的病毒式传播案例。

**研究步骤：**
1. **确定领域/主题** — 用户正在撰写关于什么内容？
2. **搜索病毒式传播的案例** — 使用 WebSearch 搜索：
   - `"[领域] 病毒式推文示例`
   - `"[领域] 在 X 上走红的话题串`
   - `"[主题] 最具影响力的推文`
   - `site:twitter.com OR site:x.com "[领域关键词]" 高互动量`
3. **分析模式** — 提取以下内容：
   - 成功的钩子样式
   - 内容结构
   - 语气和风格
   - 使用的具体数字/结果
   - 促进互动的呼吁行动（CTA）
4. **记录分析结果** — 在开始写作前创建一份简要的分析报告

**示例研究提示：**
```
Searching for: "SaaS founder viral tweets"
              "startup advice twitter thread viral"
              "tech entrepreneur best tweets engagement"
```

## X 的算法（2026 年版）

了解算法的奖励机制至关重要：

### 互动性层级（从高到低）
1. **回复** — 最重要的信号
2. **引用推文** — 高价值，表明您的内容引发了讨论
3. **书签** — 强烈的价值信号
4. **转发** — 内容传播的信号
5. **点赞** — 基础互动指标

### 时间敏感性
- **第一个小时至关重要** — 如果 60 分钟内没有获得关注，后续的关注度会显著下降
- **高峰时段**：工作日的上午 9-11 点和下午 7-9 点，周末的上午 9-11 点
- 新鲜内容优先展示 — X 优先展示最新内容

### 停留时间
X 会跟踪用户在您的内容上停留的时间。停留时间越长，传播范围越广。
- 话题串能自然延长用户的停留时间
- 视觉内容能让用户更长时间地关注帖子
- 有吸引力的钩子能阻止用户快速滚动页面

### 格式优化
- **原生视频**：优先于外部链接
- **图片/轮播图**：比纯文本的互动性高出 2 倍
- **话题串**：比单条推文的互动性高出 3 倍
- **投票**：能提高用户参与度

### 应避免的内容
- **外部链接**：会受到严重惩罚（尤其是对于非高级账户）
- **通用内容**：缺乏独特性，难以传播
- **要求互动**：如“点赞和转发”会降低传播效果

## 钩子公式（最关键的部分）

您的钩子决定了推文成功与否的 80-90%。您只有大约 1 秒的时间来吸引用户的注意力。

### 强有力的陈述
```
"Nobody talks about this, but..."
"Unpopular opinion: [controversial take]"
"Everything you've been told about [X] is wrong."
"[Common belief] is a myth. Here's the truth:"
```

### 具体的结果
```
"I [specific result] in [specific timeframe]. Here's how:"
"[Number] [achievement] in [timeframe]. The breakdown:"
"From [bad state] to [good state] in [time]. Thread:"
```
**示例**：“我在 90 天内从 0 关注者增长到了 5 万关注者。以下是我的具体方法：”

### 好奇心缺口
```
"I found a [adjective] [topic] hack that no one talks about..."
"The one thing [type of person] gets wrong about [topic]"
"Why most people fail at [X] (and how to fix it)"
```

### 问题式钩子
```
"Want to know the real secret to [X]?"
"What if everything you knew about [X] was wrong?"
"Ever wonder why [common frustration]?"
```

### 故事式钩子
```
"3 years ago I was [bad state]. Today I [good state]."
"I almost quit [X]. Then this happened:"
"The story of how I [achievement] (with $0 budget):"
```

### 模式中断式钩子
```
"Everyone says [X]. They're wrong."
"Stop doing [common practice]. Do this instead:"
"Delete [common thing]. Here's why:"
```

### 列表式钩子
```
"[Number] [things] that will [benefit] (thread):"
"[Number] lessons from [experience/achievement]:"
"The [number] [category] I wish I knew earlier:"
```
**示例**：“上周，7 个 AI 工具为我节省了 20 多小时的工作时间：”

## 易传播的推文格式

### 格式 1：列表式文章（互动性最高）
```
Hook: "[Number] [things] that [benefit]:"

1. [Item] — [Brief explanation]
2. [Item] — [Brief explanation]
...
[CTA or summary]
```

### 格式 2：相反观点
```
Hook: "[Popular belief] is wrong."

Here's why: [2-3 sentences of reasoning]

What actually works: [Your alternative]
```

### 格式 3：前后对比
```
[Time period] ago: [Bad state]
Today: [Good state]

The difference? [One key insight]
```

### 格式 4：框架式推文
```
Hook: "The [Name] Framework for [Result]:"

Step 1: [Action]
Step 2: [Action]
Step 3: [Action]

[Optional: brief expansion on each]
```

### 格式 5：填空式推文
```
"The most underrated skill for _____ is _____."
"If I could only use one tool for [X], it would be _____."
```
*能引发大量回复*

### 格式 6：通用体验式推文
```
"When you finally [common experience/realization]"
"Why does nobody talk about [shared frustration]?"
"That moment when [relatable situation]"
```

## 话题串的结构（7 条推文的最佳组合）

### 话题串模板

**推文 1（钩子）**：
- 最有吸引力的见解或结果
- 包含具体的数字
- 明确表示这是一个话题串：“🧵” 或 “(thread)”

**推文 2（背景信息）**：
- 展开钩子的内容
- 说明为什么这很重要
- 激发更多好奇心

**推文 3-6（核心内容）**：
- 每条推文包含一个关键见解
- 使用编号格式（1/、2/ 等）
- 每 3-4 条推文添加视觉元素（图片、图表）
- 每条推文都应具有独立的价值

**推文 7（总结/呼吁行动）**：
- 总结关键要点
- 指出内容的实际应用价值

**推文 8（呼吁行动）**：
- 提出一个问题（引发回复）
- 引用第一条推文（促进转发）
- 提供个人资料或新闻通讯的链接

### 话题串写作规则
1. 每条推文都应引导用户点击下一条
2. 避免冗余内容 — 每个字都应具有意义
3. 句子要简短（每条推文不超过 250 个字符）
4. 推文应具有吸引力
5. 为推文编号（如 2/12、3/12 等）

## 推文文案写作框架

### PAS（问题 → 激发兴趣 → 提供解决方案）
**最可靠的互动性公式**

```
[Problem]: You're [specific situation]
[Agitate]: And it's costing you [consequence]
[Solution]: Here's what works: [your answer]
```

### AIDA（注意力 → 兴趣 → 愿望 → 行动）
**最适合促销内容**

```
[Attention]: Hook that stops scroll
[Interest]: "Here's what most people don't realize..."
[Desire]: "Imagine if you could [benefit]"
[Action]: "DM me [X] to get started"
```

### BAB（之前 → 之后 → 连接）
**最适合讲述转变故事的内容**

```
[Before]: I was [bad state]
[After]: Now I'm [good state]
[Bridge]: The difference? [Your insight/solution]
```

## 说服原则

应用以下原则让您的推文更具吸引力：

**具体性** — “增长 23%” 比 “大幅增长” 更有效
- 具体的数字增加可信度
- 明确的时间框架增加紧迫感
- 详细信息让声明更可信

**社会证明** — “500 多位客户” 比 “许多客户” 更有说服力
- 来自真实用户的结果
- 用户/关注者的数量
- 可识别的名字/品牌

**好奇心缺口** — 创造信息不对称性
- 提供有价值的信息但不完全揭示
- 承诺具体的结果
- 使用“大多数人忽略了……”

**争议性** — 挑战现有观点
- “主流观点是错误的”
- 相反的观点能引发讨论
- 避免冒犯性语言 — 旨在引发思考

**共鸣性** — 用户能产生共鸣的经历
- 使用“当你意识到……”这样的表达
- 共同的困扰
- 普遍的困扰

## 增长技巧

### 30 天专注子主题策略
选择您所在领域的一个狭窄子主题，并连续 30 天只发布关于该主题的内容。

**示例**：如果您从事市场营销工作，可以专注于“邮件主题行”一个月。
结果：X 的算法会认为您是该领域的权威。

### 回复策略
专注于引发回复，而非点赞或转发。
- 提出问题
- 发布填空式推文
- 发布引人讨论的“热门观点”
- 算法会认为您是讨论的发起者

### 互动窗口
- 每天发布 3-5 条推文
- 每天与 20 个以上的账户互动（有意义的回复）
- 在发布后一小时内回复评论

### 80/20 规则
- 80% 的内容应具有实际价值（非促销内容）
- 20% 的内容用于推广
- 以价值为先能建立信任，从而促进转化

## 推文长度指南

- **单条推文**：字符数不超过 110 个效果最佳
- **话题串**：每条推文不超过 250 个字符
- **为什么短文本更有效**：易于阅读，便于引用，适合移动设备

---

## 常见误区及避免方法

**内容过于泛泛** — 从“成功技巧”改为“这 3 个冷门邮件模板让我本周获得了 10 次会议机会”

**没有钩子** — 从背景信息开始而非直接展示影响力

**要求互动** — 如“点赞和转发！”会降低传播效果

**主推文中的外部链接** — 将链接放在回复中

**没有具体数字** — 如“我快速增长了”不如“我在 63 天内增长了 12,847 名关注者”

**过于推销** — 内容的价值比例过低，显得过于商业化

**没有呼吁行动** — 话题串结尾没有明确的下一步行动

## 如何使用该技能

当用户请求帮助撰写推文时：
1. **询问背景信息**：
   - 他们所在的领域/主题是什么？
   - 目标是什么？（互动、关注者数量、转化率）
   - 关键信息或见解是什么？
   - 需要包含哪些具体结果或数字？

2. **研究阶段**（使用 WebSearch）：
   - 在他们的领域内搜索病毒式传播的案例
   - 识别成功的写作模式
   - 记录有效的钩子和结构

3. **提供创作选项**：
   - 提供 2-3 种不同的钩子版本
   - 使用合适的框架（如 PAS、AIDA 等）
   - 在可能的情况下使用具体的数字

4. **优化内容**：
   - 检查字符数量
   - 强化钩子内容
   添加互动引导或呼吁行动

5. **提供多种版本**：
   - 单条推文版本
   - 适合的话题串版本
   - 测试不同的钩子类型
