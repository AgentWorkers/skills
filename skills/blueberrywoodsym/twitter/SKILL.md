# **Tweet Writer 技能**  

## **概述**  
此技能可帮助您撰写具有病毒式传播力的、具有说服力的推文和话题串，这些内容经过优化，能够更好地吸引 X 平台的算法。它结合了经过验证的文案写作框架、病毒式内容创作公式以及实时研究方法，让您能够模仿所在领域内成功案例的内容风格。  

**关键词**：Twitter、X 平台、推文、话题串、病毒式内容、社交媒体、互动性、吸引人的开头语、文案写作  

## **工作流程**  

### 第一阶段：领域研究（至关重要）  
在撰写任何推文之前，您必须研究用户所在领域的病毒式传播案例。  

**研究步骤**：  
1. **确定领域/主题**——用户正在撰写关于什么内容？  
2. **搜索病毒式传播的案例**——使用 WebSearch 搜索：  
   - `"[领域] 病毒式推文示例`  
   - `"[领域] 在 X 平台上走红的推文话题串`  
   - `"[主题] 最具影响力的推文`  
   - `site:twitter.com OR site:x.com "[领域关键词]" 高互动量`  
3. **分析模式**——提取以下要素：  
   - 有效的吸引人的开头语风格  
   - 内容结构  
   - 语气和表达方式  
   - 使用的具体数字/结果  
   - 能够促进互动的呼吁行动（CTA）  
4. **记录分析结果**——在开始写作前撰写一份简要的分析报告。  

**示例研究提示：**  
```
Searching for: "SaaS founder viral tweets"
              "startup advice twitter thread viral"
              "tech entrepreneur best tweets engagement"
```  

### 第二阶段：推文创作  
使用以下框架，根据成功案例来创作内容。

## **X 平台的算法（2026 年版）**  
了解算法的奖励机制至关重要：  

### **互动性层级（从高到低）**  
1. **回复**——最具影响力的信号  
2. **引用推文**——高价值，表明您的内容引发了讨论  
3. **书签**——强烈的价值信号  
4. **转发**——内容传播的放大器  
5. **点赞**——基本的互动指标  

### **时间敏感性**  
- **最初的 60 分钟至关重要**——如果 60 分钟内没有获得关注，后续的关注度会显著下降  
- **高峰时段**：工作日的上午 9-11 点和晚上 7-9 点，周末的上午 9-11 点  
- **新鲜内容优先**——X 平台更青睐最新的内容  

### **停留时间**  
X 平台会追踪用户在您的内容上停留的时间。停留时间越长，传播范围越广。  
- 话题串能自然延长用户的停留时间  
- 视觉内容能让用户更长时间地关注帖子  
- 有吸引力的开头语能阻止用户快速滚动页面  

### **格式优化**  
- **原生视频**：优先于外部链接  
- **图片/轮播图**：相比纯文本，互动性提升 2 倍  
- **话题串**：相比单条推文，互动性提升 3 倍  
- **投票**：能激发用户参与  

### **需要避免的内容**  
- **外部链接**：会受到严重惩罚（尤其是对于非高级账户）  
- **通用内容**：缺乏独特性，难以吸引关注  
- **要求互动**（如“点赞和转发”）会降低传播效果

## **吸引人的开头语公式（最关键的部分）**  
您的开头语决定了推文成功与否的 80%-90%。您只有大约 1 秒的时间来吸引用户的注意力。  

### **有力的陈述**  
```
"Nobody talks about this, but..."
"Unpopular opinion: [controversial take]"
"Everything you've been told about [X] is wrong."
"[Common belief] is a myth. Here's the truth:"
```  

### **具体的结果**  
```
"I [specific result] in [specific timeframe]. Here's how:"
"[Number] [achievement] in [timeframe]. The breakdown:"
"From [bad state] to [good state] in [time]. Thread:"
```  
**示例**：“我在 90 天内从 0 关注者增长到了 5 万关注者。以下是我的具体方法：”  

### **引发好奇心的开头语**  
```
"I found a [adjective] [topic] hack that no one talks about..."
"The one thing [type of person] gets wrong about [topic]"
"Why most people fail at [X] (and how to fix it)"
```  

### **疑问式开头语**  
```
"Want to know the real secret to [X]?"
"What if everything you knew about [X] was wrong?"
"Ever wonder why [common frustration]?"
```  

### **故事式开头语**  
```
"3 years ago I was [bad state]. Today I [good state]."
"I almost quit [X]. Then this happened:"
"The story of how I [achievement] (with $0 budget):"
```  

### **打破常规的开头语**  
```
"Everyone says [X]. They're wrong."
"Stop doing [common practice]. Do this instead:"
"Delete [common thing]. Here's why:"
```  

### **列表式开头语**  
```
"[Number] [things] that will [benefit] (thread):"
"[Number] lessons from [experience/achievement]:"
"The [number] [category] I wish I knew earlier:"
```  
**示例**：“这 7 个 AI 工具上周为我节省了 20 多小时的时间：”

## **容易传播的推文格式**  

### 格式 1：列表式文章（互动性最高）  
```
Hook: "[Number] [things] that [benefit]:"

1. [Item] — [Brief explanation]
2. [Item] — [Brief explanation]
...
[CTA or summary]
```  

### 格式 2：反传统观点**  
```
Hook: "[Popular belief] is wrong."

Here's why: [2-3 sentences of reasoning]

What actually works: [Your alternative]
```  

### 格式 3：对比前后效果**  
```
[Time period] ago: [Bad state]
Today: [Good state]

The difference? [One key insight]
```  

### 格式 4：框架式内容**  
```
Hook: "The [Name] Framework for [Result]:"

Step 1: [Action]
Step 2: [Action]
Step 3: [Action]

[Optional: brief expansion on each]
```  

### 格式 5：填空式内容**  
**示例**：这种格式能引发大量回复。  

### 格式 6：普遍适用的内容**  
```
"When you finally [common experience/realization]"
"Why does nobody talk about [shared frustration]?"
"That moment when [relatable situation]"
```

## **话题串结构（7 条推文的最佳组合）**  

### **话题串模板**  
**推文 1（吸引人的开头语）**：  
- 最具吸引力的见解或结果  
- 包含具体的数字  
- 明确表示这是一个话题串：`🧵` 或 `（话题串）`  

**推文 2（背景说明）**：  
- 展开开头语的内容  
- 说明其重要性  
- 激发更多好奇心  

**推文 3-6（核心内容）**：  
- 每条推文传达一个关键见解  
- 使用编号格式（1/、2/ 等）  
- 每 3-4 条推文添加视觉元素（图片、图表）  
- 每条推文都应具有独立的价值  

**推文 7（总结/呼吁行动）**：  
- 总结关键要点  
- 指出内容的实际应用价值  

**推文 8（呼吁行动）**：  
- 提出一个问题（引发回复）  
- 引用第一条推文（促进转发）  
- 提供个人资料或新闻通讯的链接  

### **话题串写作规则**  
1. 每条推文都必须吸引用户点击下一条  
2. 避免冗余内容——每个字都应具有意义  
3. 句子简短（每条推文不超过 250 个字符）  
4. 语言应具有吸引力  
5. 为推文编号（如 2/12、3/12 等）

## **推文文案写作框架**  

### **PAS（问题 → 激发兴趣 → 解决方案）**  
**最可靠的互动性提升公式**  
```
[Problem]: You're [specific situation]
[Agitate]: And it's costing you [consequence]
[Solution]: Here's what works: [your answer]
```  

### **AIDA（注意力 → 兴趣 → 愿望 → 行动）**  
**最适合推广性内容**  
```
[Attention]: Hook that stops scroll
[Interest]: "Here's what most people don't realize..."
[Desire]: "Imagine if you could [benefit]"
[Action]: "DM me [X] to get started"
```  

### **BAB（前后对比）**  
**最适合讲述转变故事的内容**  
```
[Before]: I was [bad state]
[After]: Now I'm [good state]
[Bridge]: The difference? [Your insight/solution]
```

## **说服技巧**  
应用以下技巧让您的推文更具吸引力：  
- **具体性**——使用具体数字（如“增长 23%”比“大幅增长”更有效）  
- 时间框架能增加紧迫感  
- 详细信息让声明更具可信度  
- **社会证明**——使用真实用户的案例  
- **引发好奇心**——透露部分信息但不要全部公开  
- **承诺具体结果**  
- **挑战现有观点**  
- **相关性**——分享用户共有的经历  
- **利用普遍的困扰**

## **增长技巧**  

### **30 天专注子主题策略**  
在您的领域内选择一个具体的子主题，连续 30 天只发布与此相关的内容。  
**示例**：如果您从事市场营销，可以专注于“邮件主题行”一个月。  
结果：X 平台会认为您是该领域的专家。  

### **回复策略**  
重点关注引发用户回复，而非点赞或转发：  
- 提出问题  
- 发布填空式推文  
- 发布引发讨论的“热点观点”  
- 算法会认为您是讨论的发起者  

### **互动窗口**  
- 每天发布 3-5 条推文  
- 每天与 20 个以上账户互动（有意义的回复）  
- 在发布后一小时内回复评论  

### **80/20 规则**  
- 80% 的内容应具有实际价值（非推广性）  
- 20% 的内容用于推广  
- 以价值为导向的内容能建立信任并促进转化

## **推文长度指南**  
- **单条推文**：字符数不超过 110 个  
- **话题串**：每条推文不超过 250 个字符  
- **为什么短篇推文更有效**：易于阅读，便于引用，适合移动设备  

---

## **常见误区及避免方法**  
- **内容过于泛泛**——例如“成功技巧”不如“这三个冷门邮件模板让我本周获得了 10 次会议机会”  
- **没有吸引人的开头语**——先介绍背景而非直接说明效果  
- **要求互动**（如“点赞和转发”）会降低传播效果  
- **主推文中的外部链接**：将链接放在回复中  
- **缺乏具体数字**——例如“我的粉丝数量快速增长”不如“我在 63 天内增长了 12,847 名粉丝”  
- **过于推销**——内容价值比例低，显得过于商业化  
- **没有明确的呼吁行动**：话题串结尾没有明确的下一步行动

## **执行检查清单**  
在发布前，请确认：  
- 开头语具有吸引力（使用粗体、具体内容或引发好奇心的表述）  
- 前 7 个字能吸引用户继续阅读  
- 相关内容中包含具体数字  
- 字符数符合要求（单条推文不超过 110 个字符，话题串不超过 250 个字符）  
- 主推文中没有外部链接  
- 有明确的呼吁行动或互动引导  
- 在高峰时段发布  
- 发布后一小时内准备好与用户互动

## **如何使用此技能**  
当用户请求帮助撰写推文时：  
1. **询问背景信息**：  
   - 他们所在的领域/主题是什么？  
   - 目标是什么（提升互动性、增加关注者或促进转化）？  
   - 关键信息或见解是什么？  
  - 是否需要包含具体的结果或数字？  
2. **进行研究**：  
   - 在他们的领域内搜索病毒式传播的案例  
   - 识别有效的写作模式  
   - 记录有效的开头语和结构  
3. **提供写作方案**：  
   - 提供 2-3 种不同的开头语版本  
   - 选择合适的写作框架（如 PAS、AIDA 等）  
   - 在可能的情况下使用具体数字  
4. **优化内容**：  
   - 检查字符数  
   - 强化开头语  
   添加互动引导或呼吁行动  
5. **提供多种版本**：  
   - 单条推文版本  
   - 适合的话题串版本  
   - 测试不同的开头语
