# PrivateBridge

该技能实现了 OpenClaw 实例与中继服务器之间的安全远程通信，无需暴露端口、依赖 SSH 或使用 Telegram/Discord。

## 描述

PrivateBridge 将您的本地 OpenClaw 实例注册为中继网络中的一个 **受管理的远程节点**。连接成功后，该节点可以通过安全的、仅允许出站连接的 WebSocket 通道接收指令、执行工作流程、报告运行状态并重新启动。

该技能用原生且安全的中继通道替代了基于外部消息传递（如 Telegram 或 Discord）的控制方式，从而提升了 OpenClaw 的交互安全性。

## 节点生命周期

启用该技能后，OpenClaw 实例会作为具有远程通信功能的节点注册到中继服务器，并保持活跃的会话状态。

节点可以处于以下三种状态之一：

| 状态 | 描述 |
|---|---|
| **在线** | 已认证并接受中继服务器的指令 |
| **重新连接中** | 连接丢失；采用指数级退避策略自动重连 |
| **离线** | 该技能被禁用，或在中继服务器尝试多次连接失败后无法到达 |

中继服务器仅会在节点 **已认证且在线** 的状态下接受指令。在重新连接过程中接收到的指令会被中继服务器丢弃。

## 功能

| 功能 | 描述 |
|---|---|
| `remote_chat` | 远程接收并执行指令，实时将响应令牌流式传输回客户端 |
| `remote_status` | 报告节点运行状态：运行时间、当前任务、上次错误信息、连接状态 |
| `remote_restart` | 无需人工干预即可安全重启 OpenClaw 进程；重启前会取消所有待执行的任务并报告结果 |
| `remote_trigger` | 远程触发 OpenClaw 的工作流程或任务 |

远程指令仅限于预先声明的功能范围内，无法执行任意系统级操作。

## 配置参数

| 参数 | 是否必填 | 描述 |
|---|---|---|
| `relay_url` | ✅ | 中继服务器的 WebSocket 地址 |
| `node_id` | ✅ | 该 OpenClaw 节点的唯一标识符 |
| `auth_token` | ✅ | 用于与中继服务器认证的密钥令牌 |

## 消息协议

### 入站消息（中继服务器 → 节点）

| 类型 | 功能 |
|---|---|
| `prompt` | 通过 OpenClaw 的指令执行器执行指令，并将响应令牌流式传输回客户端 |
| `status` | 返回节点的运行状态信息 |
| `restart` | 取消所有待执行的任务，然后优雅地重启节点 |
| `workflow` | 执行指定的 OpenClaw 任务或工作流程 |

### 出站消息（节点 → 中继服务器）

- **心跳信号**（每 15 秒发送一次）：
  ```json
  { "node_id": "...", "uptime": 3600, "active_tasks": 2, "last_error": null, "connection_state": "online" }
  ```
- **响应数据**：每个令牌包含 `{ "type": "token", "request_id": "...", "content": "..." }`
- **响应完成**：`{ "type": "done", "request_id": "..." }`
- **完整的心跳信号**：包含 `request_id` 的完整状态信息

## 外部接口

| 接口 | 协议 | 发送的数据 | 接收的数据 |
|---|---|---|---|
| `wss://<relay_url>/connect` | WebSocket (TLS) | `auth_token`, `node_id`, 心跳信号数据, 命令响应令牌 | 中继服务器发送的指令：`prompt`, `status`, `restart`, `workflow` |

该技能不会与其他外部接口进行交互，所有网络通信均限制在配置的 `relay_url` 内。

## 安全性与隐私保护

### 从您的机器传输的数据

- **`auth_token`**：在 WebSocket 握手过程中发送一次，用于节点认证 |
- **`node_id`**：每次发送心跳信号或响应时都会包含，用于标识节点 |
- **心跳信号数据**：运行时间（秒）、当前任务数量、上次错误信息、连接状态 |
- **命令响应令牌**：根据 `prompt` 指令流式传输回中继服务器 |
- **工作流程完成状态**：触发的工作流程的成功或失败信息

### 保留在您机器上的数据

- 所有的本地 AI 模型执行和推理结果 |
- 本地文件系统内容（不会被读取或传输） |
- 环境变量（除上述三个参数外） |
- 系统信息、IP 地址或硬件详细信息（不会被收集）

## 网络安全特性

- **仅允许出站连接**：该技能不会打开任何监听端口或接受入站连接 |
- **TLS 加密**：所有 WebSocket 连接均使用 `wss://` 协议（TLS 1.2+） |
- **无数据持久化**：中继服务器不会存储指令内容或响应令牌，仅实时转发数据 |

## 安全声明

> **通过安装此技能，您将您的 OpenClaw 实例连接到配置好的 `relay_url` 对应的中继服务器。** 指令内容和响应令牌会通过该中继服务器实时传输。只有在您信任中继服务器的运营商时，才建议安装此技能。默认的中继服务器（`wss://relay-terminal-cloud.fly.dev`）由项目维护者运营。

## 运行保障

- **即使中继服务器断开连接，本地 AI 模型的执行也会继续** |
- **中继服务器不会暴露节点的任何信息给外部流量** |
- **远程操作受功能限制**：仅能执行预先声明的功能 |
- **在重启前会报告所有待执行的任务**，避免出现无声故障 |

## 安装方法

**最简单的方法是使用可视化设置向导：**

👉 打开控制面板，前往 **[/skill/remote-relay](/skill/remote-relay)**，通过用户界面完成所有配置——生成节点 ID、测试连接，并一次性导出配置信息。

**或者手动配置：**

1. 将该技能添加到您的 OpenClaw 实例中 |
2. 配置 `relay_url`、`node_id` 和 `auth_token` |
3. 启动 OpenClaw，中继连接将自动建立。

## 使用场景

该技能适用于需要远程访问 OpenClaw 节点的团队和个人，同时无需暴露基础设施信息，有效提升了安全性。
