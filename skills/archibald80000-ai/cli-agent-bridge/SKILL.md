# CLI Agent Bridge（Windows配套组件）

该目录是`cli-agent-bridge`项目的公开发布版本，由ClawHub团队提供。

其主要用途包括：

- 在ClawHub平台上展示技能信息并帮助用户发现新的技能；
- 为OpenClaw技能提供详细的描述和使用指南；
- 解释本地配套组件的部署方式及工作原理。

请注意，该组件本身并非一个独立的运行时程序包。

## 此次发布的版本包含的内容：

- 用于ClawHub和OpenClaw平台识别技能的元数据；
- 公开的技术文档；
- 关于服务提供商行为、文件操作及审批规则的参考说明。

## 此次发布的版本不包含的内容：

- PowerShell执行器；
- 批量启动工具；
- 本地安装脚本；
- 本地测试脚本；
- 嵌入式的AI服务提供商二进制文件。

上传该目录后，用户并不会自动获得以下功能：

- PowerShell执行环境；
- 本地版本的`gemini`、`claude`或`codex`工具；
- 对工作区文件进行读写操作的权限。

## 本地配套组件的预期功能：

完整的本地配套组件部署应具备以下功能：

- 为`gemini`、`claude`和`codex`提供本地化的CLI接口；
- 允许用户控制工作区的文件读取、列表和检查是否存在；
- 对文件创建、写入和追加操作进行审批控制；
- 支持从文件中读取提示信息，以处理长文本或包含特殊字符的输入内容。

## 服务提供商的状态：

此次发布的文本版本本身无法直接执行任何服务提供商的功能。

在本地配套组件部署环境中：

- `gemini`服务提供者已通过本地验证；
- `codex`服务提供者已通过本地验证；
- `claude`服务提供者的可用性取决于用户的本地Claude CLI认证状态及使用配额。

## 文件系统权限限制：

此次发布的文本版本本身不具备文件系统访问权限。在本地配套组件中，所有文件系统操作都必须在指定的工作区根目录内进行，并且需要经过明确的权限审批才能执行写入操作。
