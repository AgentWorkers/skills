# YouTube 文本转录流程（简化版）

## 快速入门

当用户需要以下服务时，请使用此流程：
- 生成并清理 YouTube 视频的文字转录内容；
- 校正转录内容中的说话者信息（区分采访者、插话者和主要发言者）；
- 提供保留时间戳和说话者信息的翻译后的文本转录文件；
- 将处理后的文件打包成可共享的文件夹。

## 最小化的工作流程：

1. **转录**
   - 使用现有的转录工具对目标视频进行转录（建议使用单词级别的转录方式，以确保说话者信息的准确性）。
   - 保留所有原始转录结果。

2. **说话者信息校正（影响较小）**
   - 保持默认的主要发言者对应关系不变；
   - 仅在校正明显错误的说话者标识时，调整插话或提问部分的归属；
   - 确保行数和时间戳保持不变。

3. **翻译（可选）**
   - 逐行翻译文本内容；
   - 保持原有的格式：`[HH:MM:SS] 发言者：文本`；
   - 除非用户特别要求简化，否则保留专业术语。

4. **打包文件以供交接**
   - 创建以下文件夹结构：`transcripts/`、`artifacts/`、`MANIFEST.txt`；
   - 根据用户需求，提供一份审核过的原始转录文件和一份翻译后的文件。

## 本流程禁止的操作：
- 不要引入复杂的自动化机制；
- 除非用户明确要求，否则不要更改行数或时间戳的结构；
- 不要随意更改说话者的身份标签，应尽量保守地进行调整。

## 质量检查：
- 核对最终转录文件的行数是否与原始视频一致；
- 确保所有生成的文件都存在且命名清晰；
- 确保所有关键的处理步骤都在 `MANIFEST.txt` 文件中有所记录。
