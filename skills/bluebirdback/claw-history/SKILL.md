# Claw 的发展历程

本文档旨在呈现 Claw 从诞生至今最完整、可追溯的发展时间线。

## 核心原则

不要默认使用当前会话的历史记录，而应从最早可获得的证据开始重建整个发展过程。

## 所需数据来源（按顺序）：

1. 最早和最新的 `memory/YYYY-MM-DD*.md` 文件（确定最早和最新的记录日期）。
2. 用于记录长期重要事件的 `MEMORY.md` 文件。
3. 可用的会话清单/历史记录（包括主会话和子代理会话）。
4. 当前的对话记录/工具日志。

如果某个数据来源无法获取，请明确说明该数据缺失的情况。

## 工作流程：

1. 从记录中确定最早的已知时间戳（即 Claw 的“诞生时间”）。
2. 收集所有可获取时间段内的确认操作记录。
3. 删除重复的条目。
4. 按时间顺序对条目进行排序（从最早到最新）。
5. 为每个条目标记状态：
   - **已确认**（有直接证据支持），
   - **推测**（基于间接或部分证据）。
6. 提供一个总结，说明哪些操作被记录下来，哪些没有被记录。

## 输出格式：

- **诞生时间**：最早的已知时间戳及数据来源。
- **覆盖范围**：记录的时间段及检查的数据来源。
- **按时间顺序排列的历史记录（从最早到最新）：**
  - `YYYY-MM-DD HH:MM (TZ) — [已确认|推测] 操作 — 结果`
- **缺失的数据或限制**：未记录的时间段、无法访问的日志、存在的不确定性。
- **可信度**：高 / 中等 / 低

## 质量要求：

- 绝不要伪造时间戳或操作记录。
- 优先使用有证据支持的条目，而非主观的叙述性总结。
- 如果具体时间未知，应使用最可靠的记录方式（日期或会话顺序）进行标注。
- 尽可能提供数据来源的详细信息（文件名、行号或日志引用）。
- 如果用户要求提供“所有操作记录”，则应同时包括重要的里程碑事件和日常操作记录。
