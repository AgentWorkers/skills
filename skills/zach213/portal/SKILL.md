# 门户（Portal）

可以将任何 URL 转换为可共享的实时浏览器会话。观众将在云虚拟机中看到一个真实的浏览器界面——每个会话时长为 10 分钟。

**观看模式（Watch）**：AI 会点击、滚动页面并进行讲解。  
**播放模式（Play）**：观众可以自由浏览，但 AI 会屏蔽不允许访问的区域。

## 快速参考（Quick Reference）

| 情况 | 操作 |
|---|---|
| 用户请求创建门户或演示网站 | 从步骤 1 开始 |
| 公共网站（登录页面、文档、营销内容） | 跳过登录步骤，直接进入步骤 3 |
| 需要登录的网站（仪表板、SaaS 服务、管理员页面） | 先调用 `save_login`（步骤 2） |
| 本地文件或 localhost | 将文件压缩为 ZIP 格式并使用 Base64 编码，作为 `ptl.entry.source` 传递 |
| Chrome 扩展程序 | 将扩展程序文件压缩为 ZIP 格式，并设置 `entry.url` 以用于测试 |
| 用户希望观看演示 | 选择“观看模式”（Watch mode）并调用 `create_script` |
| 用户希望自由浏览 | 选择“播放模式”（Play mode）并调用 `create_script` |
| 用户希望录制自己操作的过程 | 调用 `record_demo`，用户将在托管的浏览器中录制操作 |
| 用户希望选择被屏蔽的元素 | 调用 `pick_selectors`，用户可以在托管的浏览器中点击相应元素 |
| 门户正在创建中 | 调用 `make_portal` 并等待结果 |
| 会话尚未完成 | 调用 `get_session`（该操作会在服务器端阻塞 30 秒，需持续调用） |
| 需要查看会话回放 | 调用 `get_portal_sessions`，会返回对话记录和录制 URL |
| 用户需要更多使用权限 | 调用 `buy_credits`，系统会跳转到 Stripe 结账页面 |

## 向用户发送 URL

当任何工具返回需要用户访问的 URL（如 `verification_url`、`hosted_url` 或其他链接）时，必须将其发送到当前的聊天窗口中。  
**禁止** 使用 `open` 或 `xdg-open` 等 shell 命令——用户当前使用的是消息通道（如 WhatsApp、Telegram 等），而不是本地桌面环境。  
只需在回复中包含 URL，用户会在自己的设备上点击该链接即可。

## 工作流程（Workflow）

请按以下步骤操作，切勿跳过步骤 4（用户审核步骤）。

### 步骤 1 — 登录认证（Authentication）

调用 `portal_status`。如果未登录，则调用 `portal_login`。  
此操作会返回 `verification_url` 和 `device_code`。**将 `verification_url` 发送给用户**，以便他们可以登录。  
每隔 5 秒使用 `device_code` 调用 `portal_login_check`，直到用户完成登录。  
新用户会获得 3 个创建权限和 10 个观看权限。

### 步骤 2 — 分类网站类型（Classify the Site）

**询问用户**：确定该网站是否需要登录。  
- **公共网站**：直接进入步骤 3。  
- **需要登录的网站**：执行以下操作：  
  - 调用 `save_login`，系统会返回 `hosted_url`，**将其发送给用户**，以便他们可以登录并访问网站。  
  - 持续调用 `get_session`，直到状态变为“ready”。**无需询问用户是否完成登录**——系统会自动通知。登录完成后，获取 `saved_state_id`。  
- **本地文件**：将项目文件（排除 `node_modules`、`.git`、`dist` 文件夹）压缩为 ZIP 格式，并使用 Base64 编码，然后作为 `ptl.entry.source` 传递，同时设置 `entry.type` 为“local_file”。

### 步骤 3 — 生成内容（Generate Content）

如果用户未指定具体需求，提供以下 4 个选项：  
1. **观看模式（Watch）**：使用 AI 脚本（适用于登录页面）。  
2. **观看模式（Watch）**：用户自行录制操作过程。  
3. **播放模式（Play）**：使用 AI 选择的元素进行浏览。  
4. **播放模式（Play）**：用户自行选择要操作的元素。  

#### 观看模式（Watch Mode）：  
调用 `create_script`。系统会自动处理后续流程并直接返回完整的演示内容，无需再次调用 `get_script`。  
#### 观看模式（Watch Mode）——用户录制自己操作：  
调用 `record_demo`，并将生成的 `hosted_url` 发送给用户，以便他们在托管的浏览器中录制自己的操作。  
#### 播放模式（Play Mode）：  
- 使用 AI 选择的元素进行浏览：调用 `create_script`，并设置 `mode` 为“play”。  
- 用户自行选择元素：调用 `pick_selectors`，并将生成的 `hosted_url` 发送给用户，以便他们在托管的浏览器中点击元素。  

### 步骤 4 — 与用户共同审核（Review with User，强制要求）

**此步骤必不可少**。向用户展示演示内容的全部内容：  
- **观看模式**：每个场景都会附带讲解文字和操作步骤；  
- 提供示例问答对，确认用户的回答是否正确；  
- 显示 AI 发出的问候语和知识总结。  
- **播放模式**：显示被屏蔽的元素和允许访问的 URL；  
- 显示 AI 发出的问候语和相关知识内容。  
**询问用户**：“您希望给这个演示内容起什么名字？看起来可以了吗？”  
**注意**：为 URL 创建一个简洁的名称（使用小写字母、连字符，避免空格）。  

### 步骤 5 — 部署门户（Deploy the Portal）

使用完整的 PTL 规范调用 `make_portal`，此操作需要消耗 1 个创建权限。  
系统会自动处理后续流程，直到门户准备好，然后生成最终的门户 URL 并发送给用户。

### 步骤 6 — 部署后的操作（Post-Deployment Operations）：  
- **添加呼叫行动（CTA）按钮**：调用 `configure_portal`，并传入 `cta_text` 和 `cta_url`。  
- **获取嵌入代码片段**：调用 `configure_embed`，并传入 `allowed_origin`。  
- **查看会话回放**：调用 `get_portal_sessions`。  
- **排查问题**：调用 `get_creation_logs`。

## PTL 规范（PTL Specifications，最低要求）  

服务器会自动填充 `version`、`region`、`entry.type` 等参数。在调用 `make_portal` 之前，无需调用 `normalize_ptl` 或 `validate_ptl`——系统内置了验证机制。  

**重要规则（Important Rules）：**  
- **禁止** 使用内置的 canvas 工具来创建门户——请使用此插件提供的 `portal_*` 系列工具。  
- **切勿自行猜测 CSS 选择器**——仅使用 `create_script` 或 `pick_selectors` 返回的选器信息。  
- **切勿自动访问需要登录的网站**——对于需要登录的网站，仅获取单页内容。  
- **在创建门户之前，务必展示演示内容并获取用户确认**。  
- **持续调用 `get_session`——该操作会在服务器端阻塞 30 秒，无需询问用户是否完成操作。  
- **在所有点击操作中传递 `inner_text` 参数**——当动态页面上的选择器失效时，此参数可作为备用方案。  
- **在当前门户创建过程中，禁止创建第二个门户**——请调用 `get_portal` 来获取最新信息。  
- **通过聊天窗口将 URL 发送给用户**——禁止使用 shell 命令来打开 URL；用户会在消息通道中自行点击链接。
