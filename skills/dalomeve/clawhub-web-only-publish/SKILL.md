# 仅通过 Web 发布到 ClawHub

通过 ClawHub 的 Web 控制面板进行发布，无需使用 CLI 登录。

## 问题

使用 CLI 登录会导致以下问题：
- 认证循环失败
- 令牌过期
- 设备交互流程复杂化
- 会话管理开销增加

## 工作流程

### 1. 先决条件

- 浏览器已登录到 https://clawhub.ai
- 技能文件夹中包含 SKILL.md 文件
- 技能文件中不包含任何敏感信息（如密钥）

### 2. Web 发布步骤

1. 访问 https://clawhub.ai/upload
2. 确认已登录（用户名可见）
3. 填写表单：
   - Slug：`skill-name`
   - 显示名称：`Skill Name`
   - 版本：`1.0.0`
4. 点击“选择文件夹” -> 选择技能目录
5. 等待验证（SKILL.md 文件被识别）
6. （可选）添加变更日志
7. 点击“发布技能”
8. 获取发布结果 URL

### 3. 备用方案（无需 CLI 登录）

如果浏览器上传失败：
- 使用现有的 CLI 令牌（如果已经登录过）
- 运行命令：`clawhub publish <path> --version 1.0.0`
- **注意**：不要运行 `clawhub login`

## 可执行任务的完成标准

| 标准 | 验证方式 |
|----------|-------------|
| 技能 URL 可访问 | 访问该 URL，返回 200 OK 状态码 |
| 名称与 SKILL.md 文件中的名称一致 | 技能文件的标题与列表中的名称相同 |
| 版本正确 | URL 显示版本号为 v1.0.0 |
| 未使用 CLI 登录 | 日志中不存在 `clawhub login` 的记录 |

## 隐私/安全注意事项

- 技能文件中不包含任何敏感信息（如 API 密钥、令牌等）
- 在发布前扫描文件中的敏感内容
- 仅使用相对路径

## 使用场景

- 当需要将技能发布到 ClawHub 时
- 当 CLI 登录失败或无法使用时
- 当浏览器会话已激活时
