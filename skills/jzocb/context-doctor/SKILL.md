# Context Doctor

用于诊断并可视化上下文窗口资源的分配情况。

## 快速启动

运行可视化脚本：

```bash
python3 scripts/context-doctor.py
```

该脚本会自动检测以下信息：
- 工作区路径（`~/.openclaw/workspace` 或 `OPENCLAW_WORKSPACE` 环境变量）
- OpenClaw 的版本（通过 `openclaw --version` 命令获取）
- 安装的技能（包括系统级和工作区级的技能）
- 启动文件的大小及状态

## 输出部分

### 1. 工作区文件
显示每个启动文件的信息：
- **状态**：✓ 正常 / ✗ 未找到 / ⚠ 文件内容被截断（符号链接目标缺失）
- **字符数与令牌数量**：文件的原始大小及估计的令牌数量（每个字符占用 4 个令牌）
- **可视化条形图**：根据文件大小显示相应的比例

### 2. 安装的技能
列出系统和工作区技能目录中所有存在的 SKILL.md 文件。这些技能是按需加载的，并非在启动过程中加载的，仅用于提供参考。

### 3. 令牌预算
详细分析启动过程中的资源消耗情况：
- 系统提示信息（框架相关）
- 工作区文件（扫描结果）
- 技能列表（元数据描述）

显示启动过程中消耗的资源百分比以及剩余的可用对话空间。

## 诊断提示

| 提示 | 含义 | 应对措施 |
|--------|---------|--------|
| 🔴 文件内容被截断 | 文件长度超过最大限制，已自动截断 | 减少文件大小或分割文件内容 |
| 🟡 文件未找到或符号链接损坏 | 检查文件路径，修复符号链接 |
| 🟢 启动过程中资源消耗低于 10% | 系统运行正常 | 无需采取任何操作 |
| 🟠 启动过程中资源消耗超过 15% | 系统资源使用率较高 | 检查工作区文件是否存在冗余内容 |

## 图像输出（用于聊天或分享）

脚本会生成一张 PNG 格式的图像（无需截图）：

```bash
python3 scripts/context-doctor.py --png /tmp/context-doctor.png
```

该脚本通过 Rich SVG 格式生成一张类似终端界面的彩色图像。需要安装以下工具：`rich`（`pip3 install rich`）以及 `rsvg-convert`（`brew install librsvg`）或 `cairosvg`（`pip3 install cairosvg`）之一。

**代理工作流程**：当用户在聊天中询问上下文系统的运行状态时，使用 `--png` 选项运行脚本并直接发送生成的图像。如果操作成功，输出路径会显示在标准输出（stdout）中。
