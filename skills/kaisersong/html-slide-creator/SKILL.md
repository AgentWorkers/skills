# 幻灯片制作工具

该工具能够生成完全在浏览器中运行的、无依赖关系的HTML演示文稿。它引导用户从最初的构思到最终制作出精美的、带有动画效果的幻灯片集，通过“展示而非描述”的方式帮助用户确定视觉风格，而无需编写任何代码。

## 核心理念

1. **无依赖关系**：仅使用包含内联CSS/JS的单一HTML文件，无需依赖npm或构建工具。
2. **展示而非描述**：在用户看到选项之前，他们很难表达自己的设计偏好。因此，工具会生成视觉预览，而不是让他们回答诸如“你想要极简风格还是大胆风格？”这样的抽象问题。
3. **独特的设计**：避免使用通用的人工智能设计元素（如Inter字体、紫色渐变、千篇一律的标题布局）。每个幻灯片集都应该具有独特的设计感。
4. **适应视口**：幻灯片必须完美适应视口大小，不允许滚动。如果内容超出视口范围，应将其分割到多个幻灯片中展示，而不是压缩显示。如果演示文稿在滚动过程中出现卡顿，那就说明设计有缺陷——这也是我们设置内容密度限制的原因。
5. **先规划后生成**：对于复杂的幻灯片集，使用`--plan`命令先创建`PLANNING.md`大纲；然后使用`--generate`命令生成HTML。将思考过程与执行步骤分开，有助于提高结构清晰度，减少返工。

## 命令参数

通过解析命令参数来确定执行模式：

- **`--plan [提示信息]`**：规划模式。会检查`resources/`目录下的文件，分析提示信息，并生成`PLANNING.md`文件。**注意：此时不会生成HTML。**
- **`--generate [说明信息]`**：生成模式。如果存在`PLANNING.md`文件，则会先读取该文件（跳过第1/2阶段），然后生成HTML。
- **`--export pptx [--scale N]`**：使用内置脚本将最近修改的HTML文件导出为PPTX格式。需要Node.js版本≥18。
- **不使用任何参数**：自动检测模式（执行第0阶段）。

---

## 规划模式（`--plan`）

1. 扫描`resources/`目录下的文本/Markdown文件，并记录其中的图片信息。
2. 从提示信息中提取主题、目标受众、风格偏好、语言类型以及幻灯片数量。
3. 根据[references/planning-template.md](references/planning-template.md)模板起草规划大纲。
4. 将生成的文件保存为`PLANNING.md`，放在工作目录中。
5. 向用户展示幻灯片数量、结构以及关键决策内容，并征求用户的确认。
6. **注意：此时不会生成HTML。**

---

## 导出模式（`--export pptx`）

1. 在当前目录中查找最新的`.html`文件。
2. 运行以下命令：`python3 <skill-path>/scripts/export-pptx.py <presentation.html> [output.pptx]`
3. 提供生成的PPTX文件的路径和幻灯片数量。

该工具使用Playwright库，并利用用户系统自带的Chrome浏览器（无需额外下载Chromium浏览器）。它会捕捉每张幻灯片的像素级截图，并将这些截图组合成PPTX文件。只需安装`pip install playwright python-pptx`即可，无需安装Node.js或下载额外的浏览器插件。

## 第0阶段：模式检测

确定用户的需求：

- **模式A：新建演示文稿**：首先检查是否存在`PLANNING.md`文件。如果存在，则直接进入第3阶段，跳过第1/2阶段。
- **模式B：PPT转换**：用户已有`.ppt`或`.pptx`文件——进入第4阶段。
- **模式C：优化现有演示文稿**：读取现有的HTML文件，理解其结构后进行优化。在添加新内容时，务必检查内容是否适合当前幻灯片的显示范围；如果内容超出范围，应将其分割到多个幻灯片中展示，并及时通知用户。

---

## 第1阶段：内容收集

**首先，静默扫描`resources/`目录**。如果找到该目录，读取其中的文本/Markdown文件并记录图片信息（作为背景资料）。此时无需用户执行任何操作。

然后通过一次性的`AskUserQuestion`操作收集所有相关信息（包括5个问题），以避免来回沟通的麻烦：

- **用途**（单选）：产品推介、教学教程、会议演讲、内部演示
- **长度**（单选）：简短（5-10页）、中等（10-20页）、较长（20页以上）
- **内容准备情况**（单选）：所有内容已准备好、仅有初步笔记、仅提供主题信息
- **图片来源**（单选）：无需图片、来自`./assets`目录、其他来源（用户可指定路径）
- **是否支持内联编辑**（单选）：支持——允许在浏览器中直接编辑文本并自动保存（推荐）；不支持——仅用于展示用途

如果用户提供了内容，请在提交表单后让他们分享这些内容。

### 图片评估

如果用户选择“无需图片”，则跳过此步骤。仅使用文本的演示文稿同样可以制作出高质量的演示文稿——CSS生成的渐变、形状和排版效果同样能够创造出吸引人的视觉效果。

如果用户提供了图片：
1. 列出`resources/`目录下的所有图片文件。
2. 使用`Read`函数查看每张图片。
3. 对每张图片进行评估，标记为“可用”或“不可用”（并说明原因，例如图片模糊、与主题无关或损坏），同时记录图片的内容和主要颜色。
4. 根据评估结果初步设计幻灯片的结构。注意：不要先设计幻灯片再插入图片。例如：如果有3张可用产品截图，就可以设计3张以这些截图为内容的幻灯片。
5. 向用户展示评估结果和初步设计的幻灯片结构，然后通过`AskUserQuestion`获取用户的确认（例如选择“Bold Signal”风格）。

---

## 第2阶段：风格选择

大多数人难以用语言表达自己的设计偏好。此时会生成3个预览版本，让用户进行选择——这正是该工具的亮点所在。

### 预设风格选项

通过`AskUserQuestion`询问用户：
- **“请展示各种风格选项”**：根据用户的回答生成3个预览版本
- **“我已经有具体的设计想法了”**：直接展示预设的风格选择器（包括“Bold Signal”、“Dark Botanical”、“Notebook Tabs”、“Pastel Geometry”等选项，还有“其他”选项供用户查看全部预设）

**预设风格的详细信息请参阅[STYLE-DESC.md](STYLE-DESC.md)：**

| 预设风格 | 风格特点 | 适用场景 |
|--------|------|----------|
| Bold Signal | 自信、视觉冲击力强 | 适用于产品推介、主题演讲 |
| Electric Studio | 清晰、专业 | 适用于机构演示文稿 |
| Creative Voltage | 充满活力、复古现代风格 | 适用于创意类项目 |
| Dark Botanical | 优雅、精致 | 适用于高端品牌 |
| Notebook Tabs | 整洁、条理清晰 | 适用于报告或评论类内容 |
| Pastel Geometry | 友善、易于理解 | 适用于产品介绍 |
| Split Pastel | 有趣、现代感强 | 适用于创意机构 |
| Vintage Editorial | 机智、个性鲜明 | 适用于个人品牌 |
| Neon Cyber | 未来感强、科技感十足 | 适用于科技初创公司 |
| Terminal Green | 以开发者为中心 | 适用于开发工具或技术相关的演示文稿 |
| Swiss Modern | 极简、精准 | 适用于企业或数据相关的演示文稿 |
| Paper & Ink | 文学气息浓厚、富有层次感 | 适用于故事类内容 |

**风格与情绪的对应关系：**

| 情绪 | 推荐风格 |
|------|---------------|
| 感到印象深刻/自信 | Bold Signal、Electric Studio、Dark Botanical |
| 兴奋/充满活力 | Creative Voltage、Neon Cyber、Split Pastel |
| 平静/专注 | Notebook Tabs、Paper & Ink、Swiss Modern |
| 受到启发/感动 | Dark Botanical、Vintage Editorial、Pastel Geometry |

### 预览生成

在`.claude-design/slide-previews/`目录下生成3个名为`style-a/b/c.html`的HTML文件。每个文件都包含一个标题幻灯片（约50-100行内容），用于展示字体样式、颜色搭配和动画效果。

如果在第1.2步中找到了可用的品牌Logo，需将其（以Base64编码的形式）嵌入到每个预览文件中。让用户看到自己品牌在不同风格下的呈现效果，有助于他们做出更个性化的选择。

**注意禁止使用以下元素：**Inter/Roboto/Arial作为显示字体；避免使用通用的紫色背景渐变；避免使用千篇一律的居中标题布局。

每个预览文件都会附带简短的描述，然后通过`AskUserQuestion`询问用户更喜欢哪种风格（A/B/C或混合使用）。

## 第3阶段：生成演示文稿

根据第1阶段收集的内容和第2阶段选择的风格来生成最终的演示文稿。如果存在`PLANNING.md`文件，那么该文件就是生成演示文稿的依据，此时可以跳过第1和第2阶段。

### 如果选择了“Blue Sky”风格，请使用预设模板

**请参考[references/blue-sky-starter.html](references/blue-sky-starter.html)作为基础模板。**该模板已经包含了10个常见的视觉元素（如圆形图标、背景纹理、网格效果、动态文字效果、云状标题、计数器等）。你需要做的操作如下：
1. 在`:root`样式中设置`--slide-count`参数，将其设置为实际的幻灯片数量。
2. 将`#track`部分中的示例幻灯片内容替换为自己的实际内容，同时遵循注释中提供的幻灯片布局规则。
3. 在JavaScript代码中填充`orbPositions`数组，每个幻灯片的值格式为`[l1,t1,s1, l2,t2,s2, l3,t3,s3]`。

**注意：**不要重新编写与视觉效果相关的CSS代码，也不要修改幻灯片的布局结构。所有内容应使用预先定义的组件类进行展示，例如`.g`（玻璃卡片）、`.gt`（渐变文字）、`.pill`（信息框）、`.stat`（统计信息）、`.divider`（分隔符）、`.cols2/3/4`（列布局）、`.ctable`（表格）、`.co`（警告提示）、`.warn`（警告信息）、`.info`（信息提示）、`.layer`（背景层）等。

### 其他风格的处理方式

请参考[references/html-template.md]文件，其中包含了所需的HTML结构、JavaScript代码示例以及动画效果的实现方法。

**同时请参阅[STYLE-DESC.md]文件**，了解关于视口适应性的CSS设置、响应式布局规则、风格预设的详细信息，以及CSS使用中的注意事项（例如：不要直接使用`-clamp()`函数，而应使用`calc(-1 * clamp(...)`）。

### 视口适配

每个幻灯片的高度必须精确等于视口的高度（`100vh`或`100dvh`）。如果内容超出视口范围，应将其分割到多个幻灯片中展示，绝对不允许在单个幻灯片中出现滚动效果。

**内容密度限制：**

| 幻灯片类型 | 最大内容量 |
|------------|---------|
| 标题幻灯片 | 1个标题 + 1个副标题 |
| 内容幻灯片 | 1个标题 + 4-6个列表项 |
| 功能介绍幻灯片 | 1个标题 + 最多6个信息卡片（2×3或3×2排列） |
| 代码演示幻灯片 | 1个标题 + 8-10行代码 |
| 引用幻灯片 | 1条引用（最多3行） + 出处说明 |
| 图片幻灯片 | 1个标题 + 1张图片（图片高度最多为60vh） |

如果不确定如何处理内容，可以直接将幻灯片分割。

### 图片处理流程（如用户未提供图片，则跳过此步骤）

对于第1.2步中识别出的可用图片，需要根据实际情况进行处理（例如对Logo进行圆形裁剪、调整大图片的大小、为图片添加适当的边框等）。处理图片时请使用`Pillow`库。图片文件应使用相对路径（例如`assets/...`），除非用户明确要求生成独立的文件。

**注意事项：**
- 不要在多个幻灯片中重复使用同一张图片（Logo可以用于标题和结尾部分）。
- 如果图片的颜色与整体风格不匹配，务必添加与风格相匹配的边框或阴影效果。

### 代码质量要求

- 为每个代码部分添加注释，说明其功能、存在的理由以及修改方法。
- 使用语义化的HTML结构（如`<section>`、`<nav>`、`<main>`等标签）。
- 为导航元素和交互控件添加ARIA属性。
- 确保代码支持`@media (prefers-reduced-motion: reduce)`媒体查询规则，以优化动画效果。

## 第4阶段：PPT转换

请参考[references/pptx-extraction.md]文件，了解如何将生成的HTML文件转换为PPTX格式。

1. 运行提取脚本，获取每张幻灯片的标题、内容、图片和备注信息。
2. 向用户展示提取后的内容，并确认其显示效果是否正确。
3. 在处理完内容后，再次执行第2阶段（风格选择）的流程。
4. 生成最终的HTML文件，确保保留所有的文本、图片（来自`assets/`目录）以及幻灯片的顺序。如果需要，还可以添加演讲者的备注信息（以HTML注释的形式）。

---

## 最后步骤

1. 如果使用了内联编辑功能，请删除`.claude-design/slide-previews/`目录下的文件。
2. 如果演示文稿包含8张以上的幻灯片或是通过`PLANNING.md`文件生成的，请生成演讲者的备注内容（每张幻灯片对应2-4句话的提示内容）。
3. 打开生成的HTML文件（格式为`.html`）。
4. 提供演示文稿的总结信息。

```
Your presentation is ready!

📁 File: [filename].html
🎨 Style: [Style Name]
📊 Slides: [count]

Navigation: Arrow keys or Space · Scroll or swipe · Click dots to jump

To customize: edit :root variables at the top of the CSS for colors, fonts, and spacing.

To export as PPTX: run `/slide-creator --export pptx` (requires Node.js ≥ 18)
```

**如果用户选择了内联编辑功能，请告知他们可以通过将鼠标悬停在页面的左上角或按下“E”键进入编辑模式。**

## 不同情感风格对应的视觉效果建议

| 情感风格 | 推荐的视觉效果 |
|---------|-----------|
| 戏剧性/电影感强 | 缓慢的淡入效果（1-1.5秒）、深色背景、全屏显示的图片、视差效果 |
| 科技感/未来感强 | 蓝色荧光效果、粒子动画、网格图案、等宽字体、带有特殊效果的文字 |
| 有趣/友好的风格 | 柔和的动画效果、圆角设计、淡色调、浮动或抖动的动画效果 |
| 专业风格 | 微妙的动画效果（200-300毫秒）、简洁的无衬线字体、简约的装饰风格、以数据为中心的展示 |
| 平静/极简风格 | 极慢的动画效果、大量的空白空间、柔和的色调、使用有衬线的字体 |

## 使用示例：

- **示例1：为新的人工智能初创公司制作演示文稿**：
  - 用户填写简单的表单：指定用途为产品推介、内容为初步笔记、允许内联编辑。
  - 工具会评估用户提供的图片资源（其中4张图片可用，1张图片质量不佳，会给出说明）。
  - 根据用户的描述和提供的图片共同设计幻灯片结构，并通过用户反馈进行优化。
  - 选择“Neon Cyber”风格后，工具会生成3个预览版本，用户最终确认了这个风格。
  - 使用`Pillow`库处理图片，生成符合视口要求的HTML文件，并添加演讲者的备注内容。
  - 打开生成的演示文稿。

- **示例2：将PPTX文件转换为网页格式**：
  - 运行提取脚本，提取图片和内容。
  - 根据用户的反馈再次调整风格和内容。
  - 生成最终的HTML文件并展示给用户。
  - 完成演示文稿的制作。
