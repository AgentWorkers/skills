# 技能：购买便利性评估（现金支付、分期付款与月末一次性支付）

## 目标

该技能旨在帮助用户从经济角度判断以下支付方式中哪种更划算：

1. **享受折扣的现金支付**
2. **无折扣的分期付款**
3. **带利息的分期付款**
4. **月末一次性支付的信用卡**

输出结果需清晰、具体且具有解释性，包括：

- 哪种方式的**总成本最低**；
- 哪种方式的**财务便利性最高**（考虑资金的时间价值）；
- 最新官方通胀率对结果的影响（数据来自官方API）。

---

## 适用场景

当用户询问不同支付方式是否划算时使用该技能，例如：

- “现金支付享受15%折扣和分期付款无利息，哪种更划算？”
- “现金支付有折扣，但信用卡还款会在月末产生额外费用，哪种更合适？”
- “比较一次性现金支付和12期带利息的分期付款。”

---

## 需要输入的数据

该技能需要获取或推断以下信息：

- **price списка**：产品或服务的标价
- **descuento_efectivo_pct**：现金支付的折扣百分比（如有）
- **cantidad_cuotas**：分期付款的期数
- **recargo_pct**：分期付款的总费用百分比（如有）
- **pago_fin_mes**：信用卡是否支持在月末一次性支付
- **dias_hastaPago**：如果付款时间晚于月末，预计的付款截止日期
- **tasa_oportunidad_mensual**：资金的月度增值率或财务机会成本
- **usarIPC_oficial**：默认设置为`true`，以通过API获取最新官方通胀率

如果数据缺失，技能应使用现有信息并进行合理推断。通胀率无需用户提供，应通过API自动获取。

---

## 计算规则

### 1. 现金支付（享受折扣）

```math
precio_efectivo = precio_lista × (1 - descuento_efectivo pct / 100)
```

### 2. 无折扣的分期付款

如果没有利息：

```math
total_cuotas = precio_lista
valor_cuota = total_cuotas / cantidad_cuotas
```

### 3. 带利息的分期付款

如果有利息：

```math
total_cuotas = precio_lista × (1 + recargo pct / 100)
valor_cuota = total_cuotas / cantidad_cuotas
```

### 4. 月末一次性信用卡支付

如果没有折扣和利息：

```math
total_tarjeta_1pago = precio_lista
```

（可选：计算分期付款的财务优势）

---

## 分析标准

该技能需从**三个层面**分析便利性：

### A. 单纯的名义比较

比较每种方式的总支付金额，不考虑时间因素。

- 总金额最低的方式在绝对成本上最划算。

### B. 财务比较

如果有分期付款或延迟支付，使用月度增值率将未来的付款折现到现值。

#### 分期付款的现值

```math
VP = cuota / (1 + tasa_mensual) ^ n
```

其中`n`是付款周期数。

所有现值之和即为分期付款的**实际财务成本**。

#### 月末一次性支付的现值

如果付款在月末之前完成：

```math
VP_tarjeta = precio_lista / (1 + tasa_mensual) ^ (dias_hastaPago / 30)
```

这有助于判断是否因延迟付款而失去的折扣能够通过资金的时间价值得到补偿。

### C. 与通胀率的比较

该技能需将分期付款的便利性与最新通胀率进行对比。

#### 判断标准

判断分期付款的成本是低于、接近还是高于最新通胀率，从而判断融资是否在实际情况中降低了价格。

#### 官方数据来源

查询的API系列：

- `145.3_ingNACNAL_DICI_M_15` — 全国通用通胀率

基础API：

- `https://apis.datos.gob.ar/series/api/series`

请求路径：

- `https://apis.datos.gob.ar/series/api/series/?ids=145.3_ingNACNAL_DICI_M_15`

### IPC处理规则

1. 向指定URL发送GET请求。
2. 确保响应中包含`data`字段。
3. 确保`data`中至少有一条记录。
4. 解析`data`的格式：`[fecha, valor]`（例如`2025-12-01`）。
5. 将响应数据转换为以下结构：
   - `date`
   - `value`
6. 按时间顺序排列结果。
7. 如果没有数据，返回明确错误信息并显示请求的URL。
8. 不要自行生成数据或日期。

### IPC在分析中的使用

根据最近12个月的通胀数据，技能可以：

- 显示最新通胀率；
- 以最新通胀率为基准计算分期付款的累计通胀成本；
- 将此通胀成本与以下费用进行比较：
  - 现金支付时放弃的折扣；
  - 分期付款的总费用；
  - 延期付款的隐含成本。

### 解释标准

- 如果分期付款的总费用低于预期通胀率，那么分期付款在实际情况中更划算。
- 如果利息明显高于通胀率，分期付款的吸引力降低。
- 如果分期付款无利息或利息较低，且通胀率仍为正数，那么分期付款通常更划算（除非现金折扣很高）。

---

## 决策逻辑

技能应按照以下逻辑给出建议：

1. **确定名义上最便宜的选项**。
2. 如果有可用利率或可推断出合理利率，计算各选项的现值。
3. 通过API获取最新通胀率并进行比较。
4. 比较折扣带来的节省与**延迟支付的财务收益**以及**通胀率**。
5. 用简单语言给出结论：
  - “建议现金支付”
  - “分期付款更划算”
  - “月末一次性支付更合适”
  - “名义上一种方式更划算，但财务差异不大”

---

## 解释规则

### 现金支付更划算的情况：

- 折扣较高；
- 分期付款的利息较高；
- 保留资金的优势不足以弥补折扣损失；
- 或者利息明显高于预期通胀率。

### 分期付款更划算的情况：

- 分期付款无利息或利息较低；
- 月度增值率或预期通胀率较高；
- 分期付款的现值低于现金折扣后的价格；
- 或者分期付款的利息低于预计的通胀成本。

### 月末一次性支付更划算的情况：

- 没有分期付款选项；
- 现金支付会放弃折扣；
- 延期20–40天可以保持资金流动性；
- 且放弃的折扣较低。

### 与通胀率的比较

技能需明确说明：

- 如果分期付款的利息低于预期通胀率，那么分期付款更划算；
- 如果分期付款的利息与通胀率相当；
- 或者分期付款的利息高于通胀率。

示例：

> 尽管现金支付金额更高，但由于总利息低于预期通胀率，分期付款在实际情况中更划算。

---

## 预期输出格式

输出应包括：

### 1. 简短总结

以一句话给出主要建议。

示例：

> 建议现金支付，因为即使考虑延迟付款的收益，15%的折扣也远超过分期付款的财务优势。

### 详细数据

- 标价
- 现金价格
- 分期付款总额
- 每期金额
- 一次性信用卡支付总额
- 各选项的预计现值（如适用）
- 使用官方API获取的最新通胀数据（如适用）
- 分析期间的累计通胀数据（如适用）

### 解释性结论

解释为何某种方式更划算，以及在何种情况下可选择其他方式以保持流动性。

## 数据缺失处理

如果数据缺失，技能应：

- 如数据必不可少，请求缺失的数据；
- 或者进行合理估算并明确说明。

示例：

> 由于未提供利率，我进行了名义上的比较。如需要，也可以假设月利率为3%或5%进行重新计算。

如果API请求失败或返回无效数据，技能应明确告知。切勿让用户手动提供通胀率。

示例：

> 无法从API获取有效通胀数据。请求路径：`https://apis.datos.gob.ar/series/api/series?ids=145.3_ingNACNAL_DICI_M_15&last=12&format=json&metadata=simple`

在这种情况下，不要自行生成通胀数据或日期。

```json
{

  "cantidad_cuotas": 6,
  "recargo_pct": 0,
  "pago_fin_mes": true,
  "dias_hastaPago": 25,

}
```

## 输出示例

**建议现金支付。**

标价为120,000阿根廷比索，现金支付可享受15%折扣，实际支付金额为102,000阿根廷比索。
分6期无利息，总支付金额为120,000阿根廷比索，即每月20,000阿根廷比索。
即使按4%的月利率折现，分期付款的现值仍高于现金支付。
此外，与最新通胀率相比，分期付款的通胀损失无法通过节省的金额得到补偿。
月末一次性支付在便利性上稍优于立即支付，但无法弥补15%的折扣损失。
**结论：**从经济角度看，现金支付更划算；只有在需要保持流动性的情况下，才建议选择分期付款或月末一次性支付。

## 限制事项

- 不要假设“无利息的分期付款”总是最划算的。
- 不要假设“现金支付（享受折扣”总是最划算的。
- 必须始终解释所使用的计算标准。
- 如果差异很小，需明确说明。
- 如果缺乏利率数据，需明确告知。
- 当API可用时，不要让用户手动提供通胀率。
- 通胀数据必须来自指定API。

## 获取官方通胀率的技术流程

技能需遵循以下步骤获取官方通胀率：

### API端点

`https://apis.datos.gob.ar/series/api/series?ids=145.3_ingNACNAL_DICI_M_15&last=12&format=json&metadata=simple`

### 方法

- 使用`GET`请求；
- 无需身份验证。

### 验证规则

1. 确保响应为有效JSON。
2. 确认包含`data`字段。
3. 确保`data`不为空。
4. 确保每个条目包含：
  - ISO格式的日期；
  - 数值。

### 数据转换

将每条数据转换为：

```json
["2025-12-01", 2.7]
```

### 排序规则

按日期从最新到最旧排序。

### 错误处理

如果无法获取数据，返回明确错误信息：

```
错误：无法获取官方通胀率数据。请求路径：https://apis.datos.gob.ar/series/api/series?ids=145.3_ingNACNAL_DICI_M_15&last=12&format=json&metadata=simple
```

### 其他注意事项

- 不要自行生成数据或日期。
- 如果API未正确响应，不要使用虚构的数据。
- 如果API未返回数据，切勿使用虚构的通胀率。
