# 技能：分期付款与现金支付（简版）

## 目标

确定以下选项中哪种更划算：

1. 带折扣的现金支付
2. 分期付款（含/不含手续费）
3. 一次性支付的分期卡（月底支付）

答案应简洁、以数字形式呈现，并仅使用现值进行比较。

---

## 预期输入参数

- `price списка`（商品价格）
- `descuento_efectivo pct`（现金折扣百分比，可选）
- `cantidad_cuotas`（分期付款期数，可选）
- `recargo pct`（手续费百分比，可选）
- `pago_fin_mes`（月底支付日期，可选）
- `tasa_oportunidad_mensual`（月利率，推荐提供）
- `usar_proyeccion_inflacion`（是否使用通胀预测，默认为 `true`）
- `serieipc_id`（数据系列ID，可选，默认为 `145.3_ingNACNAL_DICI_M_15`）

如果缺少数据，请使用最小假设值，并在一条消息中说明。

如果未提供 `tasa_oportunidad_mensual` 且 `usar_proyeccion_inflacion` 为 `true`，则需根据通胀预测计算月利率。

---

## 内部数据服务访问（通胀预测）

当未提供月利率时，使用内部数据服务进行预测：

- 查询地址：`https://apis.datos.gob.ar/series/api/series`
- 查询方法：`GET`
- 默认数据系列：`145.3_ingNACNAL_DICI_M_15`

建议的查询示例：

`https://apis.datos.gob.ar/series/api/series?ids=145.3_ingNACNAL_DICI_M_15`

规则：

1. 数据以 `[日期, 数值]` 的格式返回。
2. 取最后 6 个有效数据点。
3. 计算这 6 个数据点的月平均利率。
4. 将百分比转换为小数形式以得到现值：`tasa = 平均利率 / 100`。
5. 如果服务查询失败或没有有效数据，需在一条消息中请求用户提供月利率。

---

## 计算方法

### 1) 现金支付

`价格现金 = 商品价格 * (1 - 现金折扣百分比 / 100)`

### 2) 分期付款

`分期总额 = 商品价格 * (1 + 手续费百分比 / 100)`

`单期付款金额 = 分期总额 / 分期期数`

`分期现值 = 总分期金额 / (1 + 月利率)^分期期数`（其中 `n = 1..分期期数`）

`折扣系数 = 总分期金额 / (1 + 月利率)^分期期数`（其中 `n = 1..分期期数`）

`可承受的最高手续费百分比 = ((分期期数 / 折扣系数) - 1) * 100`

**解释**：

- 如果 `手续费百分比 <= 可承受的最高手续费百分比`，则分期付款方案在现值上不会超过通胀率。
- 如果 `手续费百分比 > 可承受的最高手续费百分比`，则分期付款方案在现值上会超过通胀率。

### 3) 一次性支付的分期卡

`分期卡总额 = 商品价格`

`分期卡现值 = 分期卡总额 / (1 + 月利率)^(月底支付日期 / 30)`

### 4) 最终选择规则

选择现值最低的方案。

**注意**：不要仅基于名义金额进行比较。

## 输出规则（必填）

- 答案不超过 14 行。
- 必须包含以下部分：
  - `总结`
  - `现值对比`
  - `结论`
- 在 `现值对比` 中，需列出：
    - `现金支付`
    - `分期付款的现值`
    - 分期卡的现值`
- 在 `结论` 中，给出推荐方案、预估节省金额以及与第二佳方案的差异，同时说明可承受的最高手续费百分比。
- 如果无法计算月利率，需在一条消息中请求用户提供该信息。切勿用名义金额替代现值进行比较。

## 重要限制

- 不得提及外部服务、查询地址、技术来源或内部细节。
- 仅允许使用内部数据服务进行通胀预测，切勿在最终答案中披露这些信息。
- 不得添加冗长的解释或理论背景。
- 如缺少数据，不得自行编造数据。
- 不得仅基于名义本金进行比较。
- 在未计算现值的情况下，不得推荐任何方案。
